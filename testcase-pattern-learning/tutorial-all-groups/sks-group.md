# SKS 组历史测试用例补充规则

## 适用范围

本文件只用于 SKS 组相关 feature 的补充规则，适用于以下主题：

1. SKS 集群创建、节点组、原地更新、扩缩容
2. CNI / 网络配置、子网 / CIDR / 网关校验
3. GPU / 算力资源、节点组资源变更
4. 独立 ELF-CSI、裸 K8s / SKS 集群的存储联动
5. SKS 系统服务、可观测性关联、监控与告警

使用方式：

1. 先读 `all-groups-common.md`。
2. 如果当前 feature 属于 SKS、Kubernetes 集群管理、节点组、CNI、GPU、CSI、系统服务或 K8s 平台联动类，再读本文件。

## 来源与范围

本补充规则主要基于：

1. `former_cases/8_SKS/442_Master.md`
   主要提供 SKS 集群配置校验、GPU、独立 ELF-CSI、节点更新、可观测性关联等 testcase 模式。

主要覆盖以下能力：

1. 集群创建表单和网络参数校验
2. GPU 资源、节点组滚动更新、卡资源不足 / 充足
3. 独立 ELF-CSI 在 SKS 与裸 K8s 场景的部署、参数校验、存储类 / PVC / 快照 / 克隆
4. K8s 版本升级、原地更新、CPU / 内存热更新
5. SKS 关联可观测性、系统服务状态、重试与恢复

## SKS 组特有结论

1. SKS 组 testcase 通常先按“集群生命周期或插件生命周期”拆，再按部署形态、版本、资源和参数差异展开矩阵。
2. 只要 feature 涉及 K8s 参数，就默认补“空值 / 格式错误 / 冲突配置 / 缺省值 / 不支持值”。
3. SKS 组很强调“集群资源是否足够”和“更新是否触发原地变更或滚动变更”，不能只看任务成功。
4. 一旦 feature 涉及系统服务、监控、CSI 或插件，默认补“部署成功”“配置错误失败”“升级后回归”“资源或状态异常下恢复”。

## SKS 组 feature 类型与默认关注点

### 1. 集群创建与参数校验类

典型对象：
子网 CIDR、Service CIDR、Pod CIDR、网关、节点组基础参数。

默认关注维度：

1. 空值
2. 格式错误
3. 参数冲突
4. 默认值
5. 在线校验提示 / 提交校验提示

重点验证：
字段校验是否准确、提示文案是否清楚、非法配置是否被及时阻止。

### 2. 资源与节点组更新类

典型对象：
GPU 数量、节点数、CPU / 内存、磁盘大小、原地更新。

默认关注维度：

1. 资源充足 / 不足
2. 先创建新节点 / 原地热更新
3. CPU / 内存 / 磁盘差异
4. 已有工作负载 / 无工作负载
5. 升级前 / 升级后

重点验证：
是否触发正确的更新路径、资源不足时是否失败、热更新后系统状态是否一致。

### 3. CSI / 存储联动类

典型对象：
独立 ELF-CSI、storageClass、PVC / PV、快照、克隆、默认存储策略。

默认关注维度：

1. SKS 集群 / 裸 K8s
2. 在线 / 离线 / 私有镜像仓库
3. 参数为空 / 非法 / 缺省 / 支持值 / 不支持值
4. Filesystem / Block
5. 卷存在 / 不存在、访问模式一致 / 不一致
6. 升级前 / 升级后

重点验证：
部署结果、控制器 / node-plugin 状态、PVC / PV 制备结果、报错信息和升级后回归。

### 4. 系统服务 / 可观测性关联类

典型对象：
SKS 系统服务、OBS 关联、系统服务异常、重试机制、告警与监控页面。

默认关注维度：

1. 系统服务正常 / 异常
2. 关联成功 / 失败 / 重试
3. 工作负载集群 / 系统服务
4. 前置状态满足 / 不满足
5. 监控图表 / 告警 / 关联对象状态

重点验证：
关联任务结果、重试行为、关联后监控展示、异常恢复后的状态收敛。

## SKS 组特有补充规律

1. 标题尽量写清“对象 + 参数或资源条件 + 结果”，例如“子网 CIDR 与 Service CIDR 重叠时，创建集群被阻止并提示冲突”。
2. 涉及插件或控制器时，不只验部署成功，还要验 pod 状态、默认资源配置、后续功能可用性。
3. 涉及卷、快照、克隆时，默认补“创建 / 恢复 / 克隆 / 删除 / 报错 / 升级后回归”。
4. 只要 feature 涉及集群更新，就默认补“资源足够成功”“资源不足失败”“升级 / 更新后配置是否保留”。

每写一个新的 SKS 组 feature，建议最少补以下 10 类：

1. 主流程
2. 参数校验
3. 缺省值 / 空值 / 非法值
4. 资源足够 / 不足
5. 版本升级 / 原地更新
6. 控制器 / 插件状态
7. 卷 / 快照 / 克隆路径
8. 系统服务 / 可观测性联动
9. 报错与重试
10. 收尾闭环：回退、恢复、状态同步、监控展示

## SKS 组最简执行摘要

1. 先拆集群或插件生命周期，再拆参数、版本、资源矩阵。
2. 参数类默认展开空值、格式、冲突、缺省。
3. 资源类默认展开“足够成功 / 不足失败”。
4. 插件 / CSI 不只验部署，要验后续功能和 pod 状态。
5. 系统服务与可观测性联动必须覆盖异常恢复和重试。
