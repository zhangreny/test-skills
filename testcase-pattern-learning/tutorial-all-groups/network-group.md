## 适用范围

本文件只用于网络组相关 feature 的补充规则，适用于以下主题：

1. 网络路径 / 转发
2. 安全 / 策略 / 白名单
3. 端口控制 / 连通性
4. 网关 / VPC / LB / FIP / VDS
5. 网络故障恢复与链路稳定性

使用方式：

1. 先读 `all-groups-common.md`。
2. 如果当前 feature 属于网络 / 安全 / 端口控制 / 连通性 / 转发链路类，再读本文件。

## 网络组特有结论

1. 网络组 testcase 需要先按“路径或对象”拆问题空间，再按变量做矩阵。
2. 网络组比其他组更依赖同/跨节点、同/跨子网、协议、对象、版本、主备这些变量差异。
3. 网络组不能只写 happy path，必须同时覆盖失败路径、恢复路径、展示联动和升级兼容。
4. 如果历史样本里已经稳定出现“成功 / 失败”“修改前 / 修改后”“故障中 / 恢复后”的成对结构，生成新用例时默认继承这种成对写法。

## 网络组 feature 类型与默认关注点

### 1. 网络路径 / 转发类

典型对象：
VPC 通信、网关转发、FIP、对等连接、L2/L3 连通性、策略生效路径、LB 流量路径。

默认关注维度：

1. 同节点 / 跨节点
2. 同子网 / 不同子网
3. 同 VPC / 跨 VPC
4. VLAN / VPC / 混合网络
5. 路由网关 / NAT 网关 / L2 网关 / 对等连接 / FIP
6. IPv4 / IPv6
7. TCP / UDP / ICMP / 特定协议
8. 单 IP / 多 IP
9. 正常连通 / 因策略失败 / 因未关联失败 / 因配置错误失败
10. 修改前 / 修改后

重点验证：
路径是否可达、转发是否正确、策略是否生效、失败是否符合预期。

### 2. 策略 / 对象匹配类

典型对象：
DFW、安全组、标签、IP 组、Pod 安全组、隔离、白名单、日志开关。

默认关注维度：

1. 对象类型：标签 / 安全组 / IP 组 / Pod 安全组 / IP 地址
2. 动作类型：创建 / 编辑 / 删除
3. 策略类型：放行 / 拒绝 / 全局白名单 / 诊断隔离 / 完全隔离
4. 匹配方向：源 / 目标 / 任意端
5. 成员变化：增 / 删 / 改
6. 命中行为：生效 / 不生效 / 计数变化 / 日志生成
7. 规则状态：启用 / 停用
8. 升级前存量 / 升级后新增
9. 关联对象存在 / 被删除 / 被修改

重点验证：
对象匹配是否正确、策略结果是否正确、编辑后是否即时生效、日志和计数是否一致。

## 网络组特有补充规律

1. 先按功能树或网络路径把问题空间拆开。
2. 再按关键变量做矩阵，不漏同/跨节点、同/跨子网、协议、对象、主备、版本。
3. 标题尽量把前提和结果写明，让 case 自解释。
4. 不只测功能成功，还要测展示、日志、故障恢复、许可限制和升级兼容。

每写一个新的网络组 feature，建议最少补以下 10 类：

1. 主流程：创建、编辑、删除、启用、停用
2. 关键组合：节点位置、子网关系、VPC/VLAN、协议类型
3. 对象差异：标签、安全组、IP 组、Pod、网关、VDS
4. 正反结果：成功与失败成对覆盖
5. 生命周期：升级前后、切换前后、关联前后、日志开关前后
6. 故障恢复：节点、网口、服务、交换机、ARP/转发表
7. 展示观测：UI、日志、报表、筛选、告警
8. 资源限制：许可、配额、版本限制、禁用态
9. 兼容性：平台、架构、OS、旧版本升级路径
10. 收尾闭环：删除后归还、取消关联后状态、卸载后资源清理

## 网络组最简执行摘要

如果只保留网络组最重要的补充规则，优先保留下面 8 条：

1. 先拆路径或对象，再拆变量。
2. 网络组默认显式展开节点、子网、协议、版本、主备差异。
3. 一条 case 只验证一个核心结果。
4. 标题必须有对象、条件、结果。
5. 正向和反向场景默认成对。
6. 网络类优先拆路径、拓扑、对象、协议。
7. 故障类必须写恢复，不只写故障。
8. 升级类必须写存量、默认值、路径、回归。
