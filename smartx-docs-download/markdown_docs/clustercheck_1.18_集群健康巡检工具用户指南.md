# clustercheck/1.18/集群健康巡检工具用户指南
## 关于本文档

# 关于本文档

本文档介绍了如何使用集群健康巡检工具对集群的健康状况进行检测。

阅读本文档需了解 SMTX OS 和 SMTX ZBS 软件，并具备丰富的数据中心操作经验。

## 概述

# 概述

在 SMTX OS 和 SMTX ZBS 集群的日常运维中，需经常对集群的运行状况进行巡检，并根据巡检结果对集群的健康状况进行分析，以排除集群的故障和隐患。由于手动巡检的操作较为繁琐，且缺乏标准流程。因此，建议使用集群健康巡检工具（以下简称“巡检工具”）对集群进行自动巡检。

巡检工具可以对 SMTX OS 和 SMTX ZBS 集群的硬件、系统服务、管理程序、存储、网络运行的功能和性能进行自动检查，并生成检查报告供运维人员查看和分析，检查报告包含集群各方面的状态数据，并针对风险项提供了解决方案，可帮助运维人员发现集群存在或潜在的问题，保障集群的健康运行。

巡检工具同时支持通过命令行和 CloudTower 界面操作执行集群巡检，可以根据用户实际场景灵活使用。

- 如用户的集群较多，集群之间管理网络可以连通，并且不同集群的节点之间可以通过管理网络进行 SSH 连接，建议在节点上部署多集群巡检工具，通过命令行执行多集群巡检。
- 如用户只有一个集群或集群版本[版本过低](/clustercheck/1.18/cluster_check_release_notes/release-notes#%E7%89%88%E6%9C%AC%E9%85%8D%E5%A5%97%E8%AF%B4%E6%98%8E)，请在节点上部署单集群巡检工具，通过命令行执行单集群巡检。
- 如用户的所有集群都与 CloudTower 关联，建议在 CloudTower 上安装 Alpha 插件，在界面上执行集群巡检，执行一次巡检任务就能完成关联的所有集群的健康巡检。

> **注意**：
>
> 巡检前请确保 SMTX OS 节点、SMTX ZBS 节点和 CloudTower系统服务虚拟机上没有安装第三方插件，如有请将第三方插件卸载。

## 在节点上通过命令行执行巡检

# 在节点上通过命令行执行巡检

您可以在 SMTX OS 或 SMTX ZBS 集群的节点上部署巡检工具，在节点上执行命令进行集群健康巡检。

巡检工具可部署在：

- 运行 SMTX OS（ELF 平台）的物理节点
- 运行 SMTX OS（VMware ESXi 平台）的 SCVM
- 运行 SMTX ZBS 的物理节点

请查阅本版本的《集群健康巡检工具发布说明》确认巡检工具与集群的[版本配套关系](/clustercheck/1.18/cluster_check_release_notes/release-notes#%E7%89%88%E6%9C%AC%E9%85%8D%E5%A5%97%E8%AF%B4%E6%98%8E)，SMTX OS 或 SMTX ZBS 集群版本满足多集群巡检工具的适配要求，且用户集群较多时，建议部署多集群巡检工具，多集群巡检也支持巡检单个集群；集群版本过低，或用户只有一个集群时，仅支持在每个集群内单独部署单集群巡检工具进行集群巡检。

## 在节点上通过命令行执行巡检 > 多集群巡检 > 部署多集群巡检工具执行巡检

# 部署多集群巡检工具执行巡检

**前提条件**

- 部署巡检工具的节点和其他待巡检集群的任一节点的管理网络联通并且节点之间可以通过管理 IP 进行 SSH 连接。
- 如果解压安装包的目标路径下存在 turbot\_clusters 文件夹，需手动删除该文件夹。
- 建议使用 root 账户登录并执行如下操作，请提前获取 root 账户的密码。

**注意事项**

首次部署多集群巡检工具执行巡检任务需要根据以下操作步骤逐一完成；如果已经部署过多集群巡检工具，请直接参照第 3 步开始操作。

**操作步骤**

1. 在本地电脑上向部署多集群巡检工具的节点上传多集群巡检工具安装包。

   - Linux/macOS 系统

     在本地电脑执行如下命令，将多集群巡检工具安装包上传至节点。其中 `filename` 为安装包的名称，`management_ip` 为节点的管理 IP。

     `scp filename smartx@management_ip:/home/smartx/`
   - Windows 系统

     可使用 WinSCP 界面工具将安装包上传至节点。
2. 在节点中执行如下命令解压安装包。其中 `filename` 是安装包的名称。

   `tar -zxf filename`
3. 执行以下命令进入程序目录。

   `cd turbot_clusters`
4. 在配置文件 **config.yaml** 中填写需要进行巡检的集群的配置信息。配置文件 **config.yaml** 默认为空。

   - (选填) customer 和 service\_provider 下分别填入客户和服务提供方的相关信息，此信息将会同步到每个集群的巡检报告中。
   - clusters 下需要填入每个集群的相关信息：

   | 配置参数 | 说明 |
   | --- | --- |
   | name | 集群名称。不能和其他集群名称相同 |
   | management\_ip | 集群中任一节点的管理 IP |
   | username | SSH 连接时使用的用户名 |
   | pwd | SSH 连接时使用的密码 |
   | arch | 集群的架构类型，目前支持 x86\_64 和 aarch64 两种架构 |

   配置文件示例如下：

   ```
   customer:
     company_name: ""
     company_address: ""
     contact_user: ""
     contact_phone: ""
     contact_email: ""
   service_provider:
     company_name: ""
     contact_user: ""
     contact_phone: ""
     contact_email: ""
   clusters:
     - name: sre-17-11-turbot-res
       management_ip: 192.168.17.11
       username: smartx
       pwd: "password"
       arch: x86_64
     - name: dogfood-19-95
       management_ip: 192.168.18.40
       username: smartx
       pwd: "password"
       arch: aarch64
   ```

   > **注意**：
   >
   > 需要填写至少一个待巡检集群的相关信息。
5. 执行以下命令开始部署多集群巡检工具并执行多集群健康巡检。

`./clusters_check.sh`

若终端显示 `Clusters turbot check has done!` ，表示巡检工具部署成功并且巡检任务完成。

> **说明**:
>
> 如果过程中出现了 `FAILED - RETRYING` 字样的输出，说明正在进行巡检任务，属于正常输出。

# 下载多集群巡检结果

巡检任务完后，可在 `/tmp/collect_clusters_result` 目录中查看到多集群巡检结果。

```
cp -r /tmp/collect_clusters_result /home/smartx/ ; scp -r smartx@management_ip:~/collect_clusters_result/ ./
```

## 在节点上通过命令行执行巡检 > 多集群巡检 > 查看多集群巡检结果 > 查看多集群健康巡检报告

# 查看多集群健康巡检报告

多集群健康巡检报告中汇总了本次巡检所有集群的状态、资源使用率以及风险，并为每条风险项提供了针对性的解决方案，可帮助用户快速了解集群信息、明确优化方向，确保集群的资源和运行处于最优状态。

多集群巡检报告主要包含**巡检总结**和**风险详情**，汇总各个集群的版本信息、资源使用状况和风险分布情况。

对于各个集群表现异常的巡检项，按照由高到低的风险被分为 `P1`、`P2`、`P3` 和 `P4` 四个级别。报告对每一个风险巡检项的异常表现会说明原因，并给出操作建议。用户可根据说明进行相应的操作，以排除集群故障和潜在风险。

各级别的风险定义如下：

| 风险级别 | 风险定义 |
| --- | --- |
| `P1` | 存在严重影响系统正常运行的问题，建议立即采取行动对问题进行修复。 |
| `P2` | 存在紧急的资源或服务状态异常，建议在巡检完成后立即采取行动进行处理。 |
| `P3` | 检测到可能出现的潜在风险，不需要立即采取行动进行修正，但需要在日常运维注意此问题。 |
| `P4` | 配置不是最优方案，或不符合最佳实践。 |

> **说明**：
>
> 巡检报告自动生成的目录部分页码显示错误，需手动更新目录。

## 在节点上通过命令行执行巡检 > 单集群巡检 > 部署单集群巡检工具 > 部署场景

# 部署场景

部署单集群巡检工具的场景主要有以下三种，在三种场景中的部署操作完全相同。

- 若集群未部署过单集群巡检工具，请直接参考部署步骤安装本版本的单集群巡检⼯具。
- 若集群已部署了旧版本单集群巡检工具，建议按照部署步骤安装本版本，旧版本将被自动覆盖。

  > **说明**：
  >
  > - 在集群任⼀节点执⾏ `turbot --version` 命令，可确认已部署的单集群巡检工具版本。
  > - 巡检工具版本和 SMTX OS 软件版本不存在绑定关系，当发布新版本巡检工具后，建议将集群中旧版本巡检工具升级至最新版。
  > - SMTX OS 4.0.5 ~ 4.0.10 版本的集群默认包含旧版本巡检工具，建议参考部署步骤升级至最新版本。
- 若集群在部署单集群巡检工具后进行了扩容，由于新节点不会自动部署单集群巡检工具，此时需对集群重新进行部署操作，以确保所有节点部署了单集群巡检工具。

  重新部署前，请在集群任一节点执行 `cat /etc/zbs/zbs.conf` 命令，查看 cluster 信息是否已包含新添加节点。

  - 若 cluster 信息已包含新添加节点，请直接参考[部署步骤](/clustercheck/1.18/cluster_check_ug/cluster_check_user_guide_12#%E9%83%A8%E7%BD%B2%E6%AD%A5%E9%AA%A4)执行部署。
  - 若 cluster 信息未包含新添加节点，请在当前节点执行 `zbs-node sync_cluster_ips` 命令，然后参考[部署步骤](/clustercheck/1.18/cluster_check_ug/cluster_check_user_guide_12#%E9%83%A8%E7%BD%B2%E6%AD%A5%E9%AA%A4)执行部署。

## 在节点上通过命令行执行巡检 > 单集群巡检 > 部署单集群巡检工具 > 部署步骤

# 部署步骤

**前提条件**

- 已将 `tar.gz` 格式的单集群巡检工具安装包下载到本地电脑中。
- 本地电脑能够访问集群的管理 IP。

**操作步骤**

1. 在本地电脑上向集群中任一节点上传单集群巡检工具安装包。

   - Linux/macOS 系统

     在本地电脑执行如下命令，将单集群巡检工具安装包上传至节点。其中，`turbot-deploy-*.tar.gz` 为安装包的名称，`management_ip` 为节点的管理 IP。

     `scp turbot-deploy-*.tar.gz smartx@management_ip:/home/smartx/`
   - Windows 系统

     可使用 WinSCP 界面工具将安装包上传至节点。
2. 在节点中执行以下命令解压安装包。

   `cd /home/smartx`

   `tar -xzvf turbot-deploy-*.tar.gz`
3. 依次执行以下两条命令部署单集群巡检工具。

   `cd turbot_deploy`

   `./deploy.sh`

   若终端显示 `Deploy success!`，表明巡检工具已经在集群中成功部署。

## 在节点上通过命令行执行巡检 > 单集群巡检 > 执行单集群巡检

# 执行单集群巡检

部署单集群巡检工具后，在当前节点执行以下命令对集群发起巡检：

`turbot check all`

巡检过程将按照如下步骤自动进行：

1. 对集群进行前置条件检查，确认集群的网络、存储服务和监控报警系统是否正常运行。若有检查项未通过，则集群可能存在严重问题，此时巡检工具不会执行后续步骤，并在终端进行相应提示。
2. 通过前置条件检查后，自动执行巡检项的检查，并实时打印所有巡检项的巡检结果。
3. 生成巡检报告。

`scp smartx@management_ip:/home/smartx/filename ./`

其中 `filename` 是巡检报告文件的名称，`management_ip` 为节点的管理 IP。

# 下载并查看集群数据

单集群巡检工具在巡检过程中可收集集群数据。⽤⼾可以将集群数据下载⾄本地进⾏[查看](/clustercheck/1.18/cluster_check_ug/cluster_check_user_guide_24#%E6%9F%A5%E7%9C%8B%E5%8D%95%E4%B8%AA%E9%9B%86%E7%BE%A4%E7%9A%84%E9%9B%86%E7%BE%A4%E6%95%B0%E6%8D%AE)。

执行如下命令，将集群数据文件夹下载至本地电脑。

`scp smartx@management_ip:/home/smartx/turbot_deploy/filename ./`

其中 `filename` 是以 `<date>-<uuid>` 格式命名的集群数据所在文件夹的名称，`management_ip` 为节点的管理 IP。

# 在 CloudTower 上通过界面执行巡检

SmartX 开发了集群巡检工具的 Alpha 插件，该插件在 CloudTower 上为集群巡检功能提供了一个可视化的入口，可以帮助您快速完成集群的健康巡检。

## 在 CloudTower 上通过界面执行巡检 > 安装 Alpha 插件

# 安装 Alpha 插件

本小节将介绍在 Tower VM 上离线安装 Alpha 插件的操作步骤，如 CloudTower 上已经安装了 Alpha 插件，请直接参考[执行集群巡检](/clustercheck/1.18/cluster_check_ug/cluster_check_user_guide_15)进行巡检。

**操作步骤**

1. 下载 Alpha 插件的安装包 **alpha.tar.gz** 与安装脚本 **install.sh** 至本地。
2. 执行如下命令将 **alpha.tar.gz** 与 **install.sh** 复制到 Tower VM。其中 `filename` 是安装包和安装脚本的文件名称， `management_ip` 为节点的管理 IP。

   `scp filename smartx@management_ip:/home/smartx/`
3. 登录 Tower VM 并切换为 root 用户，执行如下命令确认安装环境。

   1. 在控制台输入以下命令，使不同版本 CloudTower 的环境变量保持一致。

      ```
      export PATH=$PATH:/usr/local/bin
      ```
   2. 在控制台输入 `ctr -v` 命令，确认可输出如下格式信息：

      ```
      ctr containerd.io 1.6.21
      ```
   3. 在控制台输入 `helm version` 命令，确认可输出如下格式信息：

      ```
      version.BuildInfo{Version:"v3.9.3", GitCommit:"414ff28d4029ae8c8b05d62aa06c7fe3dee2bc58", GitTreeState:"clean", GoVersion:"go1.17.13"}
      ```

   表示当前环境可用于安装 Alpha 插件。
4. 依次执行以下命令安装脚本 **install.sh**。

   `chmod +x ./install.sh`

   `./install.sh`

   安装时长约 2 ~ 3 分钟。
5. 登录 CloudTower，单击 CloudTower 页面右下角 ![](https://cdn.smartx.com/internal-docs/assets/70c13d32/cluster_check_ug_1.jpg) 或通过快捷键 **Ctrl + K** 打开列表，若**巡检工具**出现在插件列表里，表示安装成功。

## 在 CloudTower 上通过界面执行巡检 > 执行集群巡检

# 执行集群巡检

**操作步骤**

1. 通过 https 网址登录 CloudTower。
2. 单击 CloudTower 页面右下角 ![](https://cdn.smartx.com/internal-docs/assets/70c13d32/cluster_check_ug_1.jpg) 或通过快捷键 **Ctrl + K** 打开列表，在插件列表里单击**巡检工具**。
3. 在弹出的对话框中填写 **Tower VM 信息**。

   | 字段 | 说明 |
   | --- | --- |
   | ip 地址 | Tower VM 的 IP 地址，默认会填入当前浏览器 URL 中的 IP 地址 |
   | 端口 | Tower VM 的 SSH 端口，默认值为 22 |
   | 用户名 | Tower VM 的 SSH 用户名 |
   | 密码 | Tower VM 的 SSH 用户密码 |
4. 单击**连接**，将 Alpha 插件通过 SSH 的方式连接 Tower VM。

   连接按钮变为不可点击，同时执行按钮变为可点击则表示连接成功。
5. 勾选需要执行巡检的集群，填写所勾选集群的 SSH 用户名和密码，建议使用 smartx 账户的用户名和密码。

   > **注意**：
   >
   > - 至少勾选一个集群。
   > - 仅支持勾选连接状态为 **CONNECTED** 的集群。
   > - 确保用户名密码正确，否则无法成功执行巡检。
6. （可选）填写客户信息、服务提供方信息，此信息将会同步到每个集群的巡检报告中。
7. 单击**执行**按钮，执行巡检任务。任务状态由**无任务执行中**变更为**任务执行中**。

   > **注意**：
   >
   > - 同一时间仅支持执行一个巡检任务，如果需要更改巡检的集群，必须等待当前任务结束，才能执行下一个巡检任务。
   > - 任务开始后 1~5 s 内任务状态和日志将会更新。若未更新，请联系 SmartX 研发人员协助定位问题。
   > - 等待任务结束的过程中，**Tower VM 信息**页面可以关闭，巡检任务将在后端持续进行。但重新打开页面时，需重新连接 Tower VM。
8. 当任务状态变更为**任务执行结束**，确认巡检任务成功。

   有以下两种方式确认：

   - 如巡检结果文件名称内的时间后缀是任务结束的时间，则表示任务成功。
   - 查看日志，日志中无 `check failed` 字样，则表示任务成功。
