# visualization_cn/1.4.3/可观测性平台用户指南
## 关于本文档

# 关于本文档

本文档介绍了如何通过 CloudTower 在 SMTX OS（ELF）集群、SMTX ZBS 集群和 SMTX ELF 集群部署可观测性服务，并为关联的集群和系统服务提供观测服务，包括为集群提供网络流量可视化功能、为集群和系统服务提供监控报警功能、为集群提供日志管理功能等。

## 概述

# 概述

IT 系统不断增加的复杂性使排障的难度也随之增加，为了提升对系统内部状态的观测能力，降低运维人员定位问题的难度，北京志凌海纳科技股份有限公司（以下简称 “SmartX”）推出了可观测性平台，依托该平台可提供更加丰富且高效的监控、报警、日志、追踪、分析功能。

可观测性平台具备以下能力：

- 提供网络流量可视化能力，利用图形和图表的形式直观展现集群中的数据流信息，使得流量数据更容易理解和分析，同时支持对典型网络问题进行自动识别和报警。
- 提供监控能力，能够对集群内的虚拟机、主机、系统服务等多种资源对象的指标进行监控。
- 提供报警能力，能够对集群以及 CloudTower、可观测性自身等系统服务的异常状态进行报警，并支持配置邮件、SNMP 陷阱等报警通知。
- 提供日志管理能力，支持在线检索，支持通过 syslog 将日志传输至外部服务器。

可观测性平台在运行过程中会收集并存储网络流量可视化、监控、报警和日志数据，数据保留策略如下：

| 数据类型 | 保留策略 |
| --- | --- |
| 网络流量可视化 | 保留 14 天 |
| 监控 | 保留 365 天 |
| 报警 | 若未解决和已解决的报警总数超过 1000 条，第 1000 条之后（按触发时间从近到远排序）的已解决的报警将被删除。 |
| 日志 | 保留 31 天 |

# 部署可观测性服务

可观测性服务可以虚拟机形式部署在 SMTX OS（ELF）集群、SMTX ZBS 集群和 SMTX ELF 集群中。

# 确认版本配套要求

## 部署可观测性服务 > 部署前规划

# 部署前规划

## 规划可观测性服务的数量

由于可观测性服务需要利用管理网络收集被关联集群的网络流量信息，为保证服务运行良好，建议在每个数据中心中部署一个可观测性服务。

## 规划可观测性服务的配置

可观测性虚拟机在运行过程中会收集并储存网络流量、日志、报警和监控等数据，因此需要占用一定的集群资源。在部署可观测性服务时，虚拟机拥有的系统资源决定了可观测性服务的管理能力，您可根据是否需要开启网络流量可视化和待管理的节点数，配置相应的 vCPU、内存和存储空间（逻辑）。

推荐配置如下：

| 是否开启网络流量可视化 | 管理节点数 | 推荐配置 | | |
| --- | --- | --- | --- | --- |
| vCPU | 内存 | 存储空间 |
| 否 | 8 节点以下 | 4 | 8 GiB | 80 GiB × 节点数 |
| 8 ~ 32 节点 | 8 | 16 GiB |
| 32 ~ 64 节点 | 16 | 32 GiB |
| 是 | 8 节点以下 | 8 | 17 GiB |
| 8 ~ 32 节点 | 13 | 34 GiB |
| 32 ~ 64 节点 | 26 | 68 GiB |

## 规划可观测性服务的网络

在部署前，需规划端口和可观测性服务的 IP 地址。

### 规划端口

为了确保可观测性服务的正常进行，当源端访问目标端时，目标端需开放相应端口，开放的端口及端口说明如下。如未特殊说明端口类型（例如：UDP 端口），则所提及的所有端口均默认为 TCP 端口。

| 源端 | 目标端 | 目标端需开放的端口 | 端口说明 |
| --- | --- | --- | --- |
| CloudTower 虚拟机 | 可观测性虚拟机 | 8686 | CloudTower 虚拟机通过该端口对可观测性虚拟机进行健康检查。 |
| CloudTower 虚拟机 | 可观测性虚拟机 | 80、443 | CloudTower 虚拟机通过该端口向可观测性虚拟机发送 API 请求，可观测性虚拟机中的所有组件通过该端口提供服务。 |
| CloudTower 虚拟机 | 可观测性虚拟机 | 10022 | CloudTower 虚拟机通过该端口导入镜像和提供启动服务等。 **注意：**若由低版本升级至此版本可观测性服务，会沿用 22 端口。 |
| 用户端 | CloudTower 虚拟机 | 80 | 用户端通过该端口访问 CloudTower 虚拟机。 |
| 用户端 | 可观测性虚拟机 | 10022 | 用户端通过该端口登录可观测性虚拟机。 **注意：**若由低版本升级至此版本可观测性服务，会沿用 22 端口。 |
| 被关联集群的节点[[1]](#user-content-f1) | 可观测性虚拟机 | 443 | 被关联集群的节点上的数据采集器通过该端口向可观测性服务虚拟机上报数据。 |
| 被关联集群的节点[[1]](#user-content-f1) | 可观测性虚拟机 | 10022 | 数据迁移过程中，迁移数据采集器通过该端口向可观测性服务虚拟机上报数据。 **注意：**若由低版本升级至此版本可观测性服务，会沿用 22 端口。 |
| 可观测性虚拟机 | CloudTower 虚拟机 | 443 | 可观测性虚拟机通过该端口从 CloudTower 虚拟机获取安全策略信息和虚拟机信息。 |
| 可观测性虚拟机 | 被关联集群的节点[[1]](#user-content-f1)、被关联集群的管理虚拟 IP | 80、443 | 可观测性虚拟机通过该端口从集群中获取配置信息。 |
| CloudTower 虚拟机 | 被关联集群的节点[[1]](#user-content-f1) | 10000 | CloudTower 虚拟机通过该端口在被关联集群的节点上部署分布式网络数据采集器。 |
| CloudTower 虚拟机 | 被关联集群的节点[[1]](#user-content-f1) | 3260、3261 | 在部署可观测性服务时，CloudTower 虚拟机通过该端口上传镜像。 |

> **说明**：
>
> [1]：被关联集群的节点包括 SMTX OS（ELF）、SMTX OS（VMware ESXi）、SMTX ELF 或 SMTX ZBS 节点，具体取决于关联集群的类型。

### 规划 IP 地址

可观测性虚拟机需通过虚拟机网络连通 CloudTower 以及关联集群的管理网络。因此，在部署可观测性服务时，需提前在部署集群中创建一个可连通集群管理网络的虚拟机网络。

每个可观测性虚拟机需要规划一个 IP 地址、子网掩码和网关，并确保此 IP 地址与 CloudTower 的 IP 以及被观测集群的管理 IP 连通，请提前向网络管理员申请 IP 地址。

> **注意**：
>
> IP 地址不能与以下 CIDR 范围内的任何 IP 地址重叠：
>
> - 169.254.0.0/16
> - 240.0.0.0/4
> - 224.0.0.0/4
> - 198.18.0.0/15
> - 127.0.0.0/8
> - 10.4.0.0/24
> - 0.0.0.0/8

## 部署可观测性服务 > 执行部署

# 执行部署

## 在 CloudTower 上传安装包

在部署前请根据部署集群的 CPU 架构，在 CloudTower 中上传对应架构的安装包（.tar.gz 文件）。当部署多个可观测性服务时，建议部署相同版本号。

**操作步骤**

1. 登录 CloudTower，进入**系统配置**界面，在左侧导航栏选择**可观测性**，单击右上角的**设置**。
2. 在弹出的管理窗口中，单击**安装包管理**，再单击右上角的**上传安装包**，上传 `.tar.gz` 格式的安装包。
3. 单击**上传**。

## 在集群中部署可观测性服务

可观测性服务以虚拟机的形式部署在集群中，部署可观测性服务需要设置服务的基本信息、资源配置、运行虚拟机的节点和网络信息。

**前提条件**

- 部署可观测性服务的集群已经被 CloudTower 关联。
- 部署可观测性服务的集群及主机，需满足[规划可观测性服务的配置](/visualization_cn/1.4.3/visualization_user_guide/visualization_user_guide_04#%E8%A7%84%E5%88%92%E5%8F%AF%E8%A7%82%E6%B5%8B%E6%80%A7%E6%9C%8D%E5%8A%A1%E7%9A%84%E9%85%8D%E7%BD%AE)中的计算及存储资源要求。

**注意事项**

部署完成后不可删除安装包，仅在可观测性服务升级到新版本或被卸载后才可删除。

**操作步骤**

1. 登录 CloudTower，进入**系统配置**界面，在左侧导航栏选择**可观测性**，单击右上角的**设置**。
2. 在弹出的管理窗口中最下方，单击**部署可观测性服务**。
3. 在**部署可观测性服务**界面中，设置服务的名称和描述，并选择可观测性服务的版本。
4. 选择部署可观测性虚拟机的集群和主机。

   | 配置项 | 说明 |
   | --- | --- |
   | 集群 | 选择部署可观测性服务的集群。 |
   | 主机 | 选择运行可观测性服务的主机，默认为**自动调度至合适的主机**。 |
5. 根据[规划可观测性服务的配置](/visualization_cn/1.4.3/visualization_user_guide/visualization_user_guide_04#%E8%A7%84%E5%88%92%E5%8F%AF%E8%A7%82%E6%B5%8B%E6%80%A7%E6%9C%8D%E5%8A%A1%E7%9A%84%E9%85%8D%E7%BD%AE)中的管理范围和资源要求，设置可观测性服务的 vCPU、内存和存储空间配置。
6. 设置可观测性服务的网络信息。

   | 配置项 | 说明 |
   | --- | --- |
   | 虚拟机网络 | 选择可观测性服务使用的虚拟机网络，选择的虚拟机网络需要能连通集群的管理网络。 |
   | IP 地址 | 根据网络管理员提供的 IP 地址分配表，输入分配的 IP 地址。 |
   | 网关 | 输入可观测性虚拟机的网关地址。 |
   | 子网掩码 | 输入 IP 地址的子网掩码。 |
7. 单击**部署**，系统将自动在选择的集群和主机上完成可观测性服务的部署。

## 关联集群

# 关联集群

完成可观测性服务的部署后，您需将可观测性服务和待观测的集群进行关联。

**关联规则**

- 一个可观测性服务可以关联多个集群，但一个集群仅能被一个可观测性服务关联。
- 一个可观测性服务关联的所有集群的物理节点总数以及虚拟机总数，不超过[规划可观测性服务的配置](/visualization_cn/1.4.3/visualization_user_guide/visualization_user_guide_04#%E8%A7%84%E5%88%92%E5%8F%AF%E8%A7%82%E6%B5%8B%E6%80%A7%E6%9C%8D%E5%8A%A1%E7%9A%84%E9%85%8D%E7%BD%AE)中的要求。
- 可观测性服务仅可关联与 CloudTower 的连接状态为**已连接**的集群。

**操作步骤**

1. 登录 CloudTower，进入**系统配置**界面，在左侧导航栏选择**可观测性**，单击右上角的**设置**。
2. 在弹出的管理窗口中选择一个已部署的可观测性服务，选择**集群关联**。
3. 单击**编辑集群关联**，在弹出的操作界面中，添加一个或多个集群。
4. 单击**保存**，完成可观测性服务和集群的关联操作。

完成关联后，在**集群关联**界面，可查看可观测性服务已关联的集群列表。

| 参数 | 说明 |
| --- | --- |
| 集群名称 | 可观测性服务关联的集群名称。 |
| 集群类型 | 集群的类型，可能为以下类型：  - SMTX OS（ELF） - SMTX OS（VMware ESXi） - SMTX ELF - SMTX ZBS |
| 与可观测性服务的关联状态 | 集群与可观测性服务的连接状态。  - 已关联 - 关联中 - 取消关联中 - 关联异常：集群中至少存在一个代理处于异常状态。 - 更新中 |
| 网络流量可视化 | 是否已启用网络流量可视化功能。仅 SMTX OS（ELF）集群和 SMTX ELF 集群支持启用。 |
| 集群 IP | 集群的管理虚拟 IP 地址。 |
| 所属数据中心 | 集群所属的数据中心。 |
| 工作负载集群 | 集群上运行的已启用 EIC CNI 插件的工作负载集群。   - 当仅有 1 个工作负载集群时，将显示工作负载集群的名称。 - 当工作负载集群数量大于 1 个时，将显示工作负载集群的数量。鼠标悬浮在数量后的图标上，然后单击**详情**，可查看所有工作负载集群的名称。   单击工作负载集群的名称链接，可跳转至该工作负载集群的概览页面。 |

若可观测性服务需增加关联的集群或移除已关联的集群，可单击**编辑集群关联**，在弹出的界面中进行相应操作。当可观测性服务状态或集群状态异常时，如需移除关联的集群，您可以在集群右侧单击 **...** > **强制解除关联**并进行相应操作。

## 关联系统服务

# 关联系统服务

完成可观测性服务的部署后，若您需要为系统服务提供监控报警功能，请将系统服务和可观测性服务进行关联。

## 关联规则

一个可观测性服务可以关联多个系统服务，但一个系统服务仅能被一个可观测性服务关联。

## 关联 SMTX Kubernetes 服务

您需要在 CloudTower 的 **Kubernetes 服务**主界面关联具体对象。

- 若需要关联 SKS 系统服务，详细操作可参考《SMTX Kubernetes 服务管理指南》的**管理 SKS 系统服务 > 为 SKS 系统服务启用监控、报警、日志、事件和审计功能**小节。
- 若需要关联 SKS 工作负载集群，详细操作可参考《SMTX Kubernetes 服务管理指南》的**管理工作负载集群 > 管理插件 > 管理可观测性插件**小节。

## 关联其他系统服务

1. 在 CloudTower 的**系统配置**主界面的左侧导航栏单击**可观测性**，再在右侧界面右上角单击**设置**。
2. 在弹出的窗口中单击目标可观测性服务，再单击**系统服务关联**。
3. 在系统服务关联界面中单击**编辑系统服务关联**，然后在弹出的操作界面中单击选择框，选择要关联的一个或多个系统服务。
4. 单击**保存**。

若可观测性服务需增加关联的系统服务或取消已关联的系统服务，可单击**编辑系统服务关联**，在弹出的界面中进行相应操作。

## 查看关联的系统服务

1. 在 CloudTower 的**系统配置**主界面的左侧导航栏单击**可观测性**，再在右侧界面右上角单击**设置**。
2. 在弹出的窗口中单击目标可观测性服务，再单击**系统服务关联**，可查看所选可观测性服务已关联的系统服务的名称、类型、版本，以及系统服务与可观测性服务的关联状态。

   > **说明**：
   >
   > 查看关联的 SMTX Kubernetes 服务时，您需要在 SKS 右侧单击 **...** > **查看关联详情**以查看可观测性服务关联的具体对象（包含关联的 SKS 系统服务和 SKS 工作负载集群）。若需要查看这些对象与可观测性服务的关联状态，您需要在 CloudTower 的 **Kubernetes 服务**主界面的具体关联界面中查看。

## 管理网络流量可视化 > 典型使用场景

# 典型使用场景

## 简化日常运维和巡检工作

网络流量可视化可以自动生成虚拟网络上的数据流拓扑图，展示虚拟机间的流量路径、数据流数量、流量大小、网络带宽、网络时延等，帮助网络管理员以简单的操作实现全方位实时监控网络流量状态。

网络流量可视化是集群观测、洞察、预测功能的一个重要组成部分，由每一个虚拟机的网络通信繁忙程度可以客观反映其使用量，使得管理员对集群运行状况的了解更加全面。管理员通过对虚拟机流量的排序，可以定位集群中那些占用网络带宽较大的虚拟机，也可以基于管理规则对它们进行速率限制或设置 QoS。

## 快速发现并定位网络性能问题

网络流量可视化可提供网络性能报警信息，并自带典型网络异常的报警触发阈值，用户也可自定义流量异常报警的触发条件，可实现典型网络性能问题的实时报警和实时定位，提高实时排障效率，更好地适应业务连续性的要求。特别是通过对存储网络进行性能监控和报警，可以提升存储网络的运维效率。

## 安全审计与威胁识别

网络流量可视化配合 Everoute 分布式防火墙进行安全策略的观测与审计。当 Everoute 安全策略生效后，可以监控策略的执行效果是否符合预期（例如，该放行的流量是否已放行，该丢弃的流量是否被准确丢弃）以满足安全审计的要求。同时，图形化的拓扑图能够清晰展示哪些未被允许访问的终端尝试连接被保护的对象，从而辅助管理员快速识别网络中存在的攻击或异常。

## 应用拓扑发现

通过自动生成的虚拟机之间流量关系拓扑，虚拟机之间活跃的数据会话将成为它们之间存在应用层互访关系的证明。管理员可以此为依据划分“应用组”，并设置适当的组内、组间安全策略。

由于开源代码在应用开发中被广泛使用，开发的应用使用了哪些协议和端口号也不透明。网络流量可视化功能可以将这些应用有关的网络通信进行完全呈现，消除网络和安全管理中的盲区。

## 安全策略维护

在长期运行的系统中，会积累大量由不同管理员在不同时期、为不同应用设置的安全策略。这些累积的陈旧策略常常会在管理员不知情的情况下导致网络访问异常。通过网络流量可视化功能对安全策略进行监控，可以帮助管理员发现那些长期没有被命中的安全策略（即系统中的“配置垃圾”）。管理员可以有选择地清理这些配置，以降低系统的运行风险。

# 为集群启用网络流量可视化

在可观测性服务部署成功后，您可以按需为集群启用网络流量可视化功能。

启用后，您可以以图形和图表的形式查看集群中主机和虚拟机的数据流信息。若集群运行了 SMTX Kubernetes 服务，对于启用了 EIC CNI 插件的工作负载集群，您还可以查看相应 Pod 的流量。此外，系统还支持对典型网络问题进行自动识别和报警。

**前提条件**

- 当前 CloudTower 中不存在低于 1.1.2 版本的 Everoute 服务。
- 集群类型为 SMTX OS（ELF） 集群或 SMTX ELF 集群。
- 集群关联的可观测性服务的服务虚拟机的计算资源配置满足启用网络流量可视化的要求，可参考[规划可观测性服务的配置](/visualization_cn/1.4.3/visualization_user_guide/visualization_user_guide_04#%E8%A7%84%E5%88%92%E5%8F%AF%E8%A7%82%E6%B5%8B%E6%80%A7%E6%9C%8D%E5%8A%A1%E7%9A%84%E9%85%8D%E7%BD%AE)进行确认，若不满足要求，请对服务虚拟机进行扩容。
- 集群关联的可观测性服务处于正常运行状态。
- 集群中不存在任何异常的代理。

**注意事项**

启用过程中，在集群所关联可观测性服务的**集群关联**列表中，集群的**与可观测性服务的关联状态**参数将显示为`更新中`，此时无法进行卸载可观测性服务、编辑服务基本信息、编辑集群和系统服务关联等操作。

**操作步骤**

1. 登录 CloudTower，进入**系统配置**界面，在左侧导航栏选择**可观测性**，该页面展示了所有已部署的可观测性服务。
2. 单击目标集群关联的可观测性服务，在弹出的窗口中单击**集群关联**，然后在集群列表中的目标集群所在行右侧单击 **...** > **编辑功能启用状态**。
3. 在弹出的**编辑功能启用状态**对话框中，启用**网络流量可视化**。

   启用后，可在 CloudTower 的**网络流量可视化**主界面查看集群的网络流量情况。

# 筛选条件保留规则说明

设置筛选条件后，当您进行页面切换时，系统支持对已配置的筛选条件进行保留。

具体规则如下：

- 从**概览页**切换至**详情页**：

  - 仅选择按**集群**、**服务**或**时间范围**筛选，未使用高级搜索：

    - 切换至详情页时，系统将提示您当前筛选条件下匹配的数据流数量。
    - 您可以单击**添加筛选条件**，进一步添加其他筛选条件来限定数据流范围、提高页面加载速度；也可以直接单击**加载全部**以显示所有数据，但数据流较多时可能会导致加载时间较长。
  - 使用了高级搜索中的筛选条件：切换至详情页将保留已设置的所有筛选条件，并直接展示筛选出的数据流信息。
- 详情页中**拓扑视图**和**表格视图**互相切换：保留已设置的所有筛选条件，切换视图不会影响筛选结果。
- 从**详情页**切换至**概览页**：由于部分筛选条件仅在详情页支持，切换后将仅保留概览页中可支持的筛选条件，并展示筛选出的节点有关的数据流信息。

## 连线

连线代表了端点和端点之间的数据流。连线的指向代表数据流发送方向，若两个端点间存在双向流量，则连线指向为双向。连线的宽度与两个端点间传输的数据量成正比。

对于应用了 Everoute 分布式防火墙被观测集群，拓扑视图使用**未保护、已保护、已拒绝、默认允许**四种连线类型来表示 Everoute 对集群中数据流的保护情况。您可以通过勾选不同的连线类型过滤拓扑视图和表格视图中展示的数据流。具体描述请参考[数据流类型及处理原因](/visualization_cn/1.4.3/visualization_user_guide/visualization_user_guide_25)。

当预期预览效果开关打开时，拓扑视图将对预期类型与当前类型不符的数据流进行高亮显示，高亮颜色为数据流预期类型的颜色。

| 图例 | 说明 |
| --- | --- |
|  | 已保护，代表当前数据流符合策略白名单或全局白名单，已被 Everoute 允许通信。 |
|  | 已拒绝，代表当前数据流不符合放行策略的白名单规则或者符合拒绝策略的黑名单规则，已被 Everoute 服务拒绝。 |
|  | 在虚拟机网络中表示**默认允许**，代表当前数据流不在自定义放行策略的白名单或全局白名单内，但 Everoute 全局策略默认允许通信;在系统网络中，数据流不存在类型属性，全部数据流均展示为此连线。 |
|  | 未保护，代表当前数据流的源端或目标端虚拟机均不在 Everoute 管理范围内。 |
|  | 当前类型为已拒绝，预期类型为已保护，代表数据流所属的所有安全策略均生效后，该数据流由已拒绝变为已保护。 |
|  | 当前类型为默认允许，预期类型为已保护，代表数据流所属的所有安全策略均生效后，该数据流由默认允许变为已保护。 |
|  | 当前类型为默认允许，预期类型为已拒绝，代表数据流所属的所有安全策略均生效后，该数据流由默认允许变为已拒绝。 |
|  | 当前类型为已保护，预期类型为已拒绝，代表数据流所属的所有安全策略均生效后，该数据流由已保护变为已拒绝。 |

  | 虚拟机连接数过大 | 虚拟机在 2 分钟内的连接数超过报警阈值。 **报警阈值：**  - 信息：200000 - 注意：500000 **默认启用状态：**未启用 | 可能原因：虚拟机中毒或者被劫持。 | 虚拟机连接数过高，存在异常攻击的可能。 | 排查虚拟机的连接情况，排查期间可将虚拟机隔离。 |
  | 虚拟机网卡丢包数过大 | 虚拟机的网卡在 2 分钟内丢包数量超过报警阈值。 **报警阈值：**  - 信息：50000 - 注意：100000 **默认启用状态：**未启用 | 网络流量太高，虚拟机任务繁忙处理不过来，导致虚拟网卡队列满丢包。 | 网络异常，业务访问变慢或者质量下降。 | 检查虚拟机资源使用情况，CPU/内存占用，虚拟机中的任务运行情况等。 |

  | TCP SYN 泛洪攻击 | 主机 IP 地址在 2 分钟内接收及发送的 TCP 半连接请求数量超过报警阈值。 **报警阈值：**  - 信息：5000 - 注意：10000 **默认启用状态：**启用 | 主机连接数过高，存在异常攻击的可能，或者系统异常存在未释放连接。 | 系统资源消耗增加，正常网络响应变慢。 | 分析连接对端信息情况，判断是否为正常业务需要还是存在异常攻击。 |

# 与 Everoute、SKS 共同使用

当被观察 SMTX OS 集群使用了 Everoute 提供网络安全策略时，网络流量可视化可以配合 Everoute 共同使用，分组展示虚拟机在组内和组间的数据流信息。使用前需已部署至少一个符合版本要求的 Everoute 服务，且 Everoute 服务所属的 SMTX OS 集群需关联可观测性服务并启用网络流量可视化功能。

## 管理网络流量可视化 > 与 Everoute、SKS 共同使用 > 数据流类型及处理原因

# 数据流类型及处理原因

网络流量可视化按照虚拟机或 Pod 在 Everoute 服务中被隔离的情况、应用的全局白名单、自定安全策略以及全局默认拒绝通信规则，判断当前数据流的类型及并进行相应处理，并以不同的连线类型展示处于不同安全策略保护下的数据流。

| 数据流类型 | 数据流处理原因 |
| --- | --- |
| 已保护 | 目标虚拟机或 Pod 已开启隔离，当前数据流符合诊断隔离白名单，已被 Everoute 允许通信。 |
| 当前数据流符合全局白名单，已被 Everoute 允许通信。 |
| 当前数据流符合自定义放行策略的白名单，已被 Everoute 允许通信。 |
| 已拒绝 | 目标虚拟机或 Pod 已开启隔离，当前数据流不符合隔离白名单，已被 Everoute 拒绝。 |
| 当前数据流符合自定义拒绝策略的黑名单，已被 Everoute 拒绝。 |
| 当前数据流不符合自定义放行策略的白名单，已被 Everoute 拒绝。 |
| 默认拒绝通信已启用，当前数据流已被 Everoute 拒绝。 |
| 默认允许 | 代表默认拒绝通信未启用，当前数据流已被 Everoute 默认允许。 |
| 未保护 | 代表当前数据流的源端或目标端的虚拟机或 Pod 均不在 Everoute 管理范围内。 |

Everoute 对不同数据流详细的处理流程和原因如下图所示：

![](https://cdn.smartx.com/internal-docs/assets/3a42c517/user_guide_32.png)

> **说明**：
>
> - 数据流类型按照数据流在 Everoute 服务中应用的自定义安全策略和全局安全策略进行统计，如果在通信期间安全策略规则发生变化，可能导致数据流前半段和后半段的处理方式不同。例如前半段未匹配到自定义安全策略，为`已保护`类型，后半段被自定义安全策略保护但是未匹配到允许流量通过的自定义安全策略或匹配到自定义拒绝策略，为`已拒绝`类型，则此流量被视为`已拒绝`类型。
> - SKS 为 Pod 默认配置了系统服务白名单，使得 Pod 之间默认可以正常互通，行为和**默认允许**一致。

## 管理监控功能 > 监控方式说明

# 监控方式说明

当前，集群支持三种监控方式：**基础监控**、**高级监控**和**可观测性服务**。

- **基础监控**

  基础监控是 SMTX OS、SMTX ELF 以及 SMTX ZBS 集群自带的本地监控系统，运行时会受到预留的 CPU、内存和存储资源的限制。基础监控可以对集群内的多种资源对象指标进行监控，并以视图和图表的方式呈现数据。

  对于 SMTX OS、SMTX ELF 集群而言，当采集的数据量占用的系统内存即将达到 1 GiB 时，监控系统将对监控能力采取降级处理，以及去除某些优先级相对较低的监控数据，以最大限度保证对关键指标的监控。对 SMTX ZBS 集群而言，因为预留资源容量足够大，所以不会发生监控降级的问题。
- **高级监控**

  高级监控以虚拟机的形式独立部署在集群中，可以提供和基础监控相同的监控能力，并支持保存更长时间的数据，同时还支持自定义 CPU、内存和存储资源。

  SMTX OS 集群可以通过部署高级监控，确保监控数据不会因为系统资源的限制而被删除。
- **可观测性服务**

  可观测性服务是 SmartX 推出的更为全面和完善的监控报警解决方案，支持对多集群的监控数据进行统一处理。相较于高级监控，可观测性服务提供了更加稳定的数据采集和存储能力，更加丰富的监控报警能力，能够根据需要随时增加 CPU、内存和存储资源，确保在监控数据量显著增长的情况下监控性能不受影响，且可以对收集的监控数据进行更深入的分析和处理。

由于可观测性服务可提供更加全面和稳定的监控能力，对于新部署集群，强烈建议您关联可观测性服务。

> **说明**：
>
> - 对于 SMTX OS (ELF) 6.0.0 及以上的 6.x.x、SMTX ELF 6.0.0 及以上的 6.x.x 版本的集群，不支持新部署高级监控功能，请将集群关联可观测性服务以获得更完善的监控功能。
> - 对于仅采用基础监控或是已部署了高级监控的集群，建议将集群和可观测性服务关联，并[将监控数据迁移至可观测性服务](/visualization_cn/1.4.3/visualization_user_guide/visualization_user_guide_56)，由可观测性服务替代高级监控，为集群提供更加全面和稳定的监控能力。

## 管理监控功能 > 将监控数据迁移至可观测性服务

# 将监控数据迁移至可观测性服务

**前提条件**

集群已关联可观测性服务。

**注意事项**

- 当低于 6.0.0 版本的 SMTX OS（ELF）集群计划升级到 6.0.0 及以上的 6.x.x 版本，且欲使用可观测性并迁移数据，流程建议如下：
  - 若集群已部署高级监控，建议先升级集群再迁移数据，以节省迁移本地监控数据的时间，不阻塞升级流程。
  - 若集群未部署高级监控，请先将集群和可观测性服务关联并执行数据迁移，然后升级集群，以避免原有的监控数据丢失。
- 数据迁移过程中具有如下操作限制：
  - 禁止对高级监控执行停用、升级和编辑操作。
  - 禁止取消集群和可观测性服务的关联。
  - 禁止对可观测性服务进行升级、卸载、提升资源配置，禁止更改网络流量可视化的启用状态。
- 数据迁移可能耗费较长时间，平均每小时迁移 6 ~ 7 GB 数据。传输速度上限为 4 MB/s，防止占用过多管理网络带宽。

**操作步骤**

1. 登录 CloudTower，进入**系统配置**界面，在左侧导航栏选择**可观测性**，单击右上角的**设置**。
2. 在弹出的管理窗口中选择集群已关联的可观测性服务，选择**集群关联**。
3. 在已关联集群的列表中，选择需要执行数据迁移的集群，单击右侧 **...**，选择**数据迁移**。
4. 确认界面提示，单击**迁移**，开始进行数据迁移。

> **说明**：
>
> - 若集群部署了高级监控，数据迁移完成后将自动卸载高级监控服务。
> - 若迁移任务失败，请根据界面提示解决相关问题后重新执行数据迁移。

## 管理监控功能 > 创建 SNMP 传输

# 创建 SNMP 传输

支持为已关联可观测性服务的集群统一配置 SNMP 传输。完成配置后，集群产生的监控信息将通过可观测性服务发送至第三方 SNMP 客户端。

**前提条件**

已在第三方监控工具中配置了 SNMP。

**操作步骤**

1. 在 CloudTower **系统配置**主界面左侧的导航树选择 **SNMP 传输**，单击右上角**创建 SNMP 传输**，弹出**创建 SNMP 传输**对话框。
2. 填写如下信息：

   - **基本信息**

     | 参数 | 描述 |
     | --- | --- |
     | 名称 | 所创建的 SNMP 传输的名称。 |
     | 传输对象 | 所创建的 SNMP 传输的传输对象。 |
     | 协议 | 目前仅支持 UDP 协议。 |
     | 端口 | 传输端口号，默认为 161。不能为已被其他传输所占用的端口。 |
     | 版本 | v2c/v3。  **说明**: 由于 zabbix 对 v3 版本的 SNMP 传输支持能力不稳定，因此建议选择 v2c 版本。 |
   - **版本配置信息**

     请根据基本信息中选择的协议版本进行配置。

     | 版本 | 参数 | 描述 |
     | --- | --- | --- |
     | v2c | 群组 | 指定用于访问远程 SNMP 管理器的群组名称。 |
     | v3 | 用户名 | 新增 SNMP 用户的用户名。 |
     | 用户密码 | 用于认证的用户密码。 |
     | 身份认证协议 | MD5/SHA，加密用户密码的协议，用于确保用户身份的安全性。 |
     | 隐私协议 | DES/AES，对 SNMP 消息进行加密的方式。 |
     | 私有密钥 | 加密时使用的密钥。 |
3. 单击**创建**，完成创建操作。

   完成创建后，在 **SNMP 传输**页面将展示所有配置的 SNMP 传输。

## 管理报警功能 > 设置自定义报警规则

# 设置自定义报警规则

可观测性服务支持您为关联集群中的虚拟机自定义报警规则，由此类规则触发的报警将在不满足触发条件时自动解决。

## 创建自定义报警规则

**前提条件**

需要配置自定义报警规则的虚拟机所在的集群已关联可观测性服务。

**注意事项**

- 由自定义报警规则触发的报警仅支持通过可观测性服务发送报警通知。
- 在某报警规则中，若对报警对象中的某虚拟机执行了跨集群迁移，并且不保留源虚拟机，则报警对象将根据迁移的目标集群的具体情况发生更改：
  - 当目标集群符合本功能的版本要求，并且已关联可观测性服务时，报警对象中的该虚拟机将更新为迁移后的新虚拟机，并且旧虚拟机的报警将自动解决。
  - 当目标集群不符合本功能的版本要求，或者未关联可观测性服务时，报警对象中的该虚拟机将无法应用当前报警规则。
- 若报警对象为虚拟机组，则当对应虚拟机组的虚拟机数量有变化时，该规则先前已触发的报警将会自动解决并重新触发。

**操作步骤**

1. 在 CloudTower 的**报警**主界面左侧的导航树单击**报警规则**，然后在报警规则界面单击**自定义规则**。
2. 在自定义规则界面右上角单击**创建自定义规则**，将弹出**创建自定义报警规则**对话框。
3. 设置基本信息。请设置规则的名称，并可选择为规则添加描述。
4. 选择报警对象。您可以通过选择集群或虚拟机组筛选虚拟机，也可以直接选择特定的虚拟机。
   > **说明**：
   >
   > 由于系统服务虚拟机已默认应用一些内置报警规则，因此为避免重复报警，界面默认勾选**排除系统服务虚拟机**。您可在**内置规则**列表通过筛选查看系统服务虚拟机应用的报警规则，若无需排除系统服务虚拟机，可取消勾选。
5. 设置规则内容。您可以针对同一个指标添加最多 3 个触发条件，分别触发三种等级（严重警告、注意、信息）的报警。确定指标后将自动生成报警文案，您还可以选择为报警文案添加报警说明。
6. 设置是否启用报警（默认启用）。启用后该规则将在创建完成后立即生效。
7. 单击**创建**。

## 编辑或删除自定义报警规则

**注意事项**

修改触发条件（**持续时间**除外）或报警对象后，该规则先前已触发的报警将会自动解决并在满足触发条件时重新触发。

**操作步骤**

1. 在 CloudTower 的**报警**主界面左侧的导航树单击**报警规则**，然后在报警规则界面单击**自定义规则**。
2. 在自定义规则列表中，单击目标规则，再在弹出的详情面板中单击**编辑**或**删除**，并完成相应操作。

## 管理报警功能 > 设置报警通知

# 设置报警通知

可观测性服务支持以下三种报警通知渠道，您可以自定义触发报警通知的报警等级，并将匹配的报警通过指定的报警渠道发送。

- **邮件通知配置**

  配置邮件报警通知后，当有报警被触发，或报警被自动或手动解决时，系统会自动向您指定的邮箱发送邮件。
- **SNMP 陷阱配置**

  系统支持使用简单网络管理协议（SNMP）和第三方监控报警平台（如 Zabbix）集成。配置 SNMP 陷阱后，当有报警被触发，或报警被自动或手动解决时，系统会自动向 SNMP 客户端发送报警信息。
- **Webhook 通知配置**

  配置 webhook 通知后，当有报警被触发，或者报警被自动或手动解决时，系统会自动向指定的 webhook 地址发送报警信息。

## 管理报警功能 > 设置报警通知 > 创建邮件通知配置

# 创建邮件通知配置

本小节仅介绍如何使用可观测性服务创建邮件通知配置。为确保通知的连续性，建议也为集群创建集群内部的邮件通知配置，以作为可观测性的应急报警通知渠道，当可观测性服务无法正常工作时，将由集群内部监控发送报警通知，详细操作指导可参考《CloudTower 使用指南》的**管理报警 > 设置报警通知 > 设置 SMTP 邮件报警通知 > 创建邮件通知配置**小节。

  该策略启用后，在所设置的结束时间之前，与静默对象相关的报警通知将不再以邮件、SNMP 陷阱或 webhook 渠道发送。
- **通知聚合策略**

  该策略启用后，在所设置的周期内，相同对象、相同规则的报警条目会聚合为一条报警通知，再以邮件、SNMP 陷阱或和 webhook 渠道发送。

# 设置通知静默策略

## 创建通知静默策略

**前提条件**

需要配置通知静默策略的[集群](/visualization_cn/1.4.3/visualization_user_guide/visualization_user_guide_07)和[系统服务](/visualization_cn/1.4.3/visualization_user_guide/visualization_user_guide_49)已关联可观测性服务。

**注意事项**

- 只有通过可观测性服务发送的报警通知才支持静默，由集群内部发送的邮件通知不受静默策略影响。
- 不支持将可观测性服务作为静默对象，因此网络流量、可观测性服务自身相关的报警通知不支持静默。

**操作步骤**

1. 在 CloudTower 的**报警**主界面左侧的导航树单击**通知策略**，然后在通知策略界面右上角单击**创建策略** > **创建通知静默策略**，将弹出**创建通知静默策略**对话框。
2. 设置策略名称。
3. 选择静默对象。您可以同时选择多个不同类型的静默对象。
4. 设置静默的结束时间。
   - **永不**：选择该项时，报警通知将永远保持静默状态。
   - **设置时间点**：选择该项时，报警通知将在所设置的时间点前保持静默状态。
5. 设置是否启用策略（默认启用）。启用后该策略将在创建完成后立即生效。
6. 单击**创建**。

创建成功后，若静默策略列表中已有启用的策略，则 CloudTower **集群**主界面的数据中心管理界面顶部将提示哪些对象已开启通知静默策略，单击**去关闭**可跳转回静默策略列表编辑策略。

## 编辑或删除通知静默策略

1. 在 CloudTower 的**报警**主界面左侧的导航树单击**通知策略**。
2. 在静默策略列表中，单击目标策略，再在弹出的详情面板中单击**编辑**或**删除**，并完成相应操作。

- 在一个聚合周期内，若同时发生了报警的触发和解决，则接收端将收到一条报警触发通知和一条报警解决通知。

## 管理日志功能 > 日志检索 > 设置查询条件

# 设置查询条件

系统支持设置查询条件进行日志筛选，查询条件包括标签、文本筛选条件和选项。

在添加新的查询条件时，可对查询条件进行如下操作：

- **编辑名称**：查询条件默认以“A、B、C...”命名，单击查询框左上角的名称，可重新编辑。
- **复制**：单击查询框右上角的复制按钮，将生成一组相同的查询条件。
- **关闭/开启**：单击查询框右上角的关闭/开启按钮，日志数量面板和日志详情面板将同步隐藏由该组查询条件查询所得的结果。再次单击按钮，对应结果将重新展示。
- **删除**：单击查询框右上角的删除按钮删除该查询条件，日志数量面板和日志详情面板将同步删除由该组查询条件查询所得的结果。

**注意事项**

- 当查询条件过多时，可能引发查询超时或网页卡顿，可以减少查询条件、缩短查询时间范围或者调低日志数量上限后重新尝试运行查询。
- 当搜索特殊字符(如 | )时，可能引发网页卡顿，可以增加精确的筛选条件后重新尝试运行查询。

**操作步骤**

1. 单击**标签**栏中的 **+** 按钮，添加一条新的查询标签，选择标签类型、匹配符号和对象。

   > **说明**：
   >
   > - 系统默认筛选当前集群的日志，因此系统会自动生成一条标签类型为 cluster\_uuid 的查询标签，该查询标签无法更改或删除。
   > - 当添加一条以上的查询标签时，多条查询标签之间为逻辑与的关系。后添加的标签可选对象，是满足前置标签条件后的剩余选项，因此可能出现无可选对象的情况。

   - **标签类型**

     | 字段 | 说明 |
     | --- | --- |
     | instance | 筛选特定节点产生的日志，可选对象为节点存储 IP。 |
     | service | 筛选特定服务产生的日志，可选对象为服务组件的名称。若当前集群已关联 Everoute 分布式防火墙，可以选择 `EverouteDFW` 以检索分布式防火墙相关的策略日志。 |
     | severity | 筛选特定等级的日志，可选对象为日志等级。 |
     | everoute\_policy\_id | 仅当当前集群已关联 Everoute 分布式防火墙时展示该选项。筛选特定策略相关的流量日志，可选对象为策略 ID。 |
   - **匹配符号**

     | 字段 | 说明 |
     | --- | --- |
     | = | 完全等于。筛选所选对象对应的日志。 |
     | =~ | 正则表达式匹配。筛选所选对象对应的日志，可选择一个或多个对象。 |
     | != | 不等于。筛选排除所选对象后的所有对象对应的日志。 |
     | !~ | 排除正则表达式匹配。筛选排除所选对象后的所有对象对应的日志，可选择一个或多个对象。 |
2. 单击 **+ 目标文本筛选**，添加一条目标文本筛选条件，当添加一条以上的文本筛选条件时，多条文本筛选条件之间为逻辑与的关系。

   | 字段 | 说明 |
   | --- | --- |
   | 包含目标文本的日志 | 填写目标文本，筛选包含目标文本的日志。 |
   | 排除目标文本的日志 | 填写目标文本，筛选不包含目标文本的日志。 |
   | 匹配正则表达式的日志 | 填写正则表达式，筛选匹配正则表达式的日志。 例如：在文本筛选框内输入正则表达式`^info`，则系统将筛选出以`info`为开头的日志。 |
   | 不匹配正则表达式的日志 | 填写正则表达式，筛选不匹配正则表达式的日志。 |

   添加查询标签和文本筛选条件之后，相关信息将同步展示在文本查询条件下方区域。
3. 单击**选项**，展开数量上限和分辨率选项。

   | 字段 | 说明 |
   | --- | --- |
   | 数量上限 | 指定当前查询能够返回的日志数量上限，最高可填 5000。 |
   | 分辨率 | 分辨率用于调整日志数量面板中的数据点聚合度，影响数据点间的时间间隔。提供 1/1、1/2、1/3、1/4、1/5、1/10 六种分辨率。 当选择不同的分辨率时，随着分母的增大，数据点间的时间间隔也越大，两个数据点间被忽略的数据点也越多，从而降低统计精度。因此，面板上最终呈现的结果更侧重于展示数据的分布情况，而不是准确的数据量统计。 |
4. （可选）单击 **+ 添加查询**，在新的查询条件中设置查询标签、文本筛选条件和选项。

## 管理日志功能 > 日志检索 > 执行日志检索

  单击**运行查询**右侧的箭头，出现 Off（关闭自动刷新）以及 5s、10s、30s、1m、5m 的时间间隔选项。当前检索页面将根据所选时间间隔自动刷新日志查询。

## 管理日志功能 > 日志传输

# 日志传输

## 管理日志功能 > 日志传输 > 配置 Syslog 服务器

# 配置 Syslog 服务器

**前提条件**

- 您已架设好可用的 Syslog 服务器。
- 确保集群中所有节点的管理 IP 地址可以访问 Syslog 服务器的 IP 地址。

**操作步骤**

1. 进入 CloudTower **系统配置**主界面，在左侧的导航树选择 **Syslog 服务器**，单击右上角的 **+ 配置 Syslog 服务器** > **用于日志传输**，弹出**配置 Syslog 服务器**的对话框。

   也可以在 CloudTower 主界⾯左侧导航树中，选择已关联本版本可观测性服务的任一集群，在右侧管理界⾯选择**全部**>**日志**，在日志界面的顶部按钮中选择**传输**，并单击**配置 Syslog 服务器**跳转至 Syslog 服务器界面。然后单击右上角的 **+ 配置 Syslog 服务器** > **用于日志传输**，弹出**配置 Syslog 服务器**的对话框。
2. 填写如下信息：

   | 字段 | 说明 | 取值/示例 |
   | --- | --- | --- |
   | 名称 | Syslog 服务器配置的名称。 | `Syslog-1` |
   | 可观测性服务 | 为 Syslog 服务器配置所属的可观测性服务，该配置只能应用给已关联该可观测性服务的集群。 | `obs-1` |
   | 应用集群 | 应用该 Syslog 服务器配置传输日志的集群：  - 全部集群：已关联所选可观测性服务的全部集群。 - 部分集群：选择已关联所选可观测性服务的一个或多个集群。 | `全部集群` |
   | 服务器地址 | Syslog 服务器的地址，可填写 IP 或域名。 | `192.168.1.1` |
   | 端口 | Syslog 服务器的接收端口。 | `514` |
   | 协议 | 进行日志传输的协议类型，可选 UDP 和 TCP。 | `UDP` |
   | 启用状态 | Syslog 服务器的启用状态，分为已启用和未启用。 | `已启用` |
3. 单击**保存**按钮，完成 Syslog 服务器的配置。若 Syslog 服务器的启用状态为`已启用`，则集群将开始向 Syslog 服务器传输日志。

## 管理可观测性操作权限 > 角色权限说明

# 角色权限说明

在 CloudTower 中，用户的权限由其角色决定，每个用户只能对应一个角色。不同角色拥有不同的操作权限，用户仅在具备相应的权限时才可对资源进行操作，在不具备相应的权限时，仅拥有基本的可读权限。

CloudTower 提供两种类型的角色：默认角色和自定义角色。其中，默认角色的权限是固定的，不可更改。自定义角色可根据需要自行创建，并为其自定义操作权限。

默认角色和自定义角色的可观测性权限如下：

- **默认角色的可观测性权限**

  CloudTower 提供六种默认角色。每种角色的可观测性权限说明如下所示：

  | 角色名称 | 操作权限 |
  | --- | --- |
  | 超级管理员 | 超级管理员拥有 CloudTower 的最高权限，可查看可观测性事件并进行所有可观测性操作。 |
  | 运维管理员 | 可查看可观测性事件并进行所有可观测性操作。 |
  | 只读用户 | 仅可查看可观测性配置信息，不可查看日志检索信息，无可观测性操作权限。 |
  | 安全审计员 | 仅可查看 CloudTower 全局事件，无可观测性的查看和操作权限。 |
  | 用户管理员 | 仅可查看可观测性配置信息，不可查看可观测性事件和日志检索信息，且无任何操作权限。 |
  | 虚拟机使用者 | 无可观测性的查看和操作权限。 |
- **自定义角色的可观测性权限**

  所有自定义角色都拥有可观测性的**基本可读权限**，基本可读权限不包含可观测性的事件信息和日志检索信息的可读权限。用户可以在创建自定义角色时，为该角色指定可以进行的可观测性操作。

6. 单击**创建**，完成配置。

## 可观测性服务运维 > 管理网络流量可视化许可

# 管理网络流量可视化许可

可观测性平台针对网络流量可视化功能提供了独立的许可，使用该功能时需单独购买许可。

## 查看许可

1. 进入 CloudTower 的**系统配置**界面，在左侧导航栏中单击**可观测性**，然后在右上角单击**设置**。
2. 在弹出的窗口中，单击**网络流量可视化**，可以查看网络流量可视化的许可信息。

   | 参数 | 描述 |
   | --- | --- |
   | 序列号 | 网络流量可视化的序列号。 |
   | 版本 | 网络流量可视化的软件版本，目前只有一种软件版本，即标准版。 |
   | 许可类型 | 网络流量可视化有以下三种许可类型： - 试用许可：首次启用网络流量可视化后的许可类型，默认有效期为 90 天，且不限制许可总额度。 - 订阅许可：根据需要选择订阅时长，更为灵活。 - 永久许可：有效期为永不过期。 |
   | 过期时间 | 当许可类型为**试用许可**或**订阅许可**时，展示许可的过期时间。 |
   | 许可使用 | 许可的总额度以及已使用的额度。网络流量可视化以可观测性服务所关联集群的 CPU 插槽数为单元提供许可额度，仅当关联集群已启用网络流量可视化时，才会消耗许可额度。 |

当许可过期，或为新的集群开启网络流量可视化功能、为已开启功能的集群添加节点后导致许可使用额度超限，此时无法继续使用网络流量可视化，将停止展示所有网络流量可视化视图中的流量数据，在此期间流量数据仍会被采集和保存，许可更新后可继续查看。

## 更新许可

在如下场景中，需要更新许可码：

- 更新许可类型：
  - 试用许可更新为订阅许可或永久许可。
  - 订阅许可变更为永久许可。
- 增加订阅许可或永久许可的许可总额度。
- 延长试用许可或订阅许可的有效期。

更新许可时，需将当前软件许可信息及可观测性平台版本号提供给 SmartX 客户经理，在完成商务流程后，客户经理将提供新的许可码。

**操作步骤**

1. 进入 CloudTower 的**系统配置**界面，在左侧导航栏中单击**可观测性**，然后在右上角单击**设置**。
2. 在弹出的窗口中，单击**网络流量可视化**，将许可码粘贴在右侧界面底部的**新许可码**输入框中。
3. 单击输入框外部的空白区域，载入新许可信息。
4. 确认新许可信息无误后，单击**更新**，完成许可更新。

## 可观测性服务运维 > 配置 NTP 服务器

# 配置 NTP 服务器

可观测性服务支持配置 NTP 服务器进行周期性时间同步，以使可观测性服务虚拟机的时间与您的目标校准时间保持同步。由于当可观测性服务的时间与其所关联集群的时间以及 CloudTower 的时间产生偏差时，将导致可观测性服务记录的指标时间和集群实际产生指标的时间不一致，进而导致 CloudTower 上监控图表展示不准确。因此，建议为每个可观测性服务均配置和 CloudTower 相同的 NTP 服务器。

**前提条件**

您已架设好 NTP 服务器。

**操作步骤**

1. 登录 CloudTower，进入**系统配置**界面，在左侧导航栏选择**可观测性**，单击右上角的**设置**。
2. 在弹出的管理窗口中选择一个已部署的可观测性服务，选择**服务时间**，该页面展示了可观测性服务的当前时间，并提供了 NTP 服务器的配置框。
3. 在配置框中填写 NTP 服务器的地址，建议填写至少三个服务器地址，并使用英文逗号 `,` 隔开。

   > **说明**：
   >
   > 建议配置与 CloudTower 相同的 NTP 服务器，以使 CloudTower 的监控图表能够展示准确的监控数据。
   >
   > - 若 CloudTower 已配置 NTP 服务器，配置框下方将显示 CloudTower 配置的 NTP 服务器地址，单击右侧的**填入**，将 NTP 服务器地址填入配置框。
   > - 若 CloudTower 未配置 NTP 服务器，配置框下方将提供跳转按钮。单击**配置**跳转至 CloudTower 的 NTP 服务器配置界面，填写 NTP 服务器地址以完成 CloudTower 的 NTP 服务器配置。
4. 单击**保存**，完成 NTP 服务器的配置。

   完成配置后，当可观测性服务与 NTP 服务器连接异常或时间不同步时，系统将在**报警**界面进行报警。您可根据提供的解决方法解决该报警。

## 可观测性服务运维 > 提升可观测性服务的资源配置

# 提升可观测性服务的资源配置

在可观测性服务部署成功后，可以在可观测性的**服务虚拟机**页面查看资源使用情况。当资源使用率过高时，系统将生成报警信息。您可按需提升可观测性虚拟机的资源配置。

> **说明**：
>
> 当资源使用率过高而产生报警信息时，**服务虚拟机**页面展示的 CPU 使用率和内存使用率可能和报警阈值存在偏差，请以报警为准。

**操作步骤**

1. 登录 CloudTower，进入**系统配置**界面，在左侧导航栏选择**可观测性**，单击右上角的**设置**。
2. 在弹出的管理窗口中选择待操作的可观测性服务，单击**服务虚拟机**，单击**资源配置**右上角的**提高配置**，弹出**提高可观测性服务的资源配置**对话框。
3. 编辑 vCPU 分配、内存分配和存储空间（逻辑）分配字段，并单击**保存**。

   > **说明**：
   >
   > 仅允许提升配置，且配置的资源量不能超过当前可观测性服务所属集群的可分配资源量。

## 可观测性服务运维 > 升级服务

# 升级服务

可观测性服务支持一键在线升级。如果 CloudTower 中部署了多个可观测性服务，建议升级到相同的版本。

**前提条件**

- 当前 CloudTower 满足版本配套要求。
- 参考[规划端口](/visualization_cn/1.4.3/visualization_user_guide/visualization_user_guide_04#%E8%A7%84%E5%88%92%E7%AB%AF%E5%8F%A3)，确认目标端已开放相应端口。
  > **注意**：
  >
  > 1.4.2 版本的可观测性服务调整了访问集群和 CloudTower 的方式，新增了开放 443 端口的要求。若升级前的可观测性服务低于 1.4.2 版本，请确保在相关目标端中开放 443 端口。
- 可观测性服务处于正常运行或升级异常状态（升级失败后服务会进入升级异常状态，此时可再次尝试升级）。
- 可观测性服务关联的所有集群均处于已关联状态，且不存在任何异常的代理。
- 提前获取计划升级到的版本安装包（.tar.gz 文件）。
- 对于 1.3.0 以下版本的可观测性服务，若升级前已启用网络流量可视化功能，请提前联系 SmartX 售后工程师协助更新网络流量可视化许可。

**注意事项**

- 上传升级安装包前，需确保当前 CloudTower 已升级到与目标可观测性服务版本兼容的版本。
- 升级过程中无法卸载可观测性服务、编辑服务基本信息、编辑集群和系统服务关联。
- 升级过程中不允许删除原本使用的安装包。升级完成后，如果原本使用的安装包没有关联的服务，允许删除。
- 升级完成后不允许删除升级时使用的安装包，直到服务升级到下一个新版本或被卸载。
- 1.3.0 及以上版本的可观测性服务不再为 SMTX ZBS 集群提供网络流量可视化功能。若升级前的可观测性服务已启用网络流量可视化功能并关联了 SMTX ZBS 集群，则升级至本版本后，SMTX ZBS 集群将自动关闭网络流量可视化功能。

## 上传升级安装包

1. 登录 CloudTower，进入**系统配置**界面，在左侧导航栏选择**可观测性**，单击右上角的**设置**。
2. 在弹出的管理窗口中，单击**安装包管理**，再单击右上角的 **+ 上传安装包**，上传 `.tar.gz` 格式的安装包。
3. 单击上传。

## 执行升级

1. 登录 CloudTower，进入**系统配置**界面，在左侧导航栏选择**可观测性**，单击右上角的**设置**。
2. 在弹出的管理窗口中选择待升级的可观测性服务，单击**服务升级**，可查看服务的基本信息及可升级版本。若 CloudTower 中存在多个适用 CPU 架构相同、版本号更高的安装包，将展示多个可升级版本。
3. 选择升级的版本，单击**升级**。
4. 在弹出的对话框中单击**升级**。

## 可观测性服务运维 > 卸载可观测性服务

# 卸载可观测性服务

当可观测性服务部署失败、或不再需要可观测性服务继续运行时，允许卸载可观测性服务。

服务卸载后，保留在服务中的网络流量及日志数据将被删除，请谨慎操作。

**前提条件**

- 可观测性服务处于正常运行或异常状态（部署失败后服务会进入异常状态，此时可卸载服务）。
- 待卸载的可观测性服务服务没有关联的集群和系统服务。

**注意事项**

- 卸载过程中无法升级可观测性服务、编辑服务基本信息、编辑集群关联。
- 卸载过程中不允许删除原本使用的安装包。卸载完成后，如果原本使用的安装包没有关联的服务，允许删除。

**操作步骤**

1. 登录 CloudTower，进入**系统配置**界面，在左侧导航栏选择**可观测性**，单击右上角的**设置**。
2. 在弹出的管理窗口中选择待卸载的可观测性服务，单击下方**卸载服务**。
3. 在弹出的对话框中单击**卸载**。
