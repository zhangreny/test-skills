# CLOUDTOWER_COMMANDER/1.0.0/CloudTower Commander 用户指南
## 关于本文档

# 关于本文档

本文档描述了如何在 SMTX OS（ELF）集群中部署 CloudTower Commander（简称“Commander”），并在关联 CloudTower 后使用 Commander 查看 CloudTower 的资源情况以及报警情况。

---

## 更新信息

# 更新信息

**2024-05-31**：**配合 CloudTower Commander 1.0.0 正式发布**

---

## 概述

# 概述

CloudTower 可以统一管理数据中心内和跨数据中心的多个 SmartX 集群。当用户使用多个 CloudTower 管理不同集群时，可能会产生对多个 CloudTower 统一管理的需求。北京志凌海纳科技股份有限公司（以下简称 “SmartX”）推出了一个全新的产品 CloudTower Commander，用以连接多个 CloudTower，并实现统一监控和管理。

CloudTower Commander 当前提供的管理规格为 150 个 CloudTower 和 300 个 SmartX 集群。其功能包括：

- 汇总展示多个 CloudTower 管理的资源情况，提供全面的概览和详细信息，便于用户整体了解各个 CloudTower 管理的资源情况；
- 汇总展示多个 CloudTower 收集的报警情况，便于用户及时了解各个 CloudTower 中出现的问题。

---

## 部署 Commander

# 部署 Commander

Commander 以虚拟机的形式部署在 SMTX OS（ELF）集群中。请先根据部署前的准备确认相关软件和浏览器版本，并提前规划待部署集群和网络、获取安装文件后再执行部署。

---

## 部署 Commander > 部署前的准备

# 部署前的准备

为了保证安装部署 Commander 顺利进行，请参考下面要求，完成部署前的准备工作。

---

## 部署 Commander > 部署前的准备 > 确认软件版本

# 确认软件版本配套

Commander 通过 CloudTower 在 SMTX OS（ELF）集群中进行安装部署，并通过关联多个 CloudTower 以为其提供管理服务。

- **与 SMTX OS 的版本配套**

  Commander 支持部署在 5.0.5 及以上版本的 SMTX OS（ELF）集群中，且要求服务器架构为 Intel x86\_64 或 Hygon x86\_64。
- **与 CloudTower 的版本配套**

  Commander 支持关联 3.4.2 及以上版本的 CloudTower。

---

## 部署 Commander > 部署前的准备 > 确认浏览器版本

# 确认浏览器版本

部署完成后，您需要使用 Web 浏览器访问并登录 Commander，为保证兼容性，我们建议您使用以下版本的浏览器，同时开启 Cookie 和 JavaScript。

| **浏览器** | **版本要求** |
| --- | --- |
| Google Chrome | 91 及以上版本 |
| Microsoft Edge | 107 及以上版本 |
| Mozilla Firefox | 107.0 及以上版本 |
| Apple Safari | 14.1 及以上版本 |

---

## 部署 Commander > 部署前的准备 > 规划待部署集群

# 规划待部署集群

Commander 以虚拟机的形式部署在 SMTX OS（ELF）集群中，为确保 Commander 能成功部署并正常运行，请提前规划待部署的集群，以确保该集群有足够的资源分配给 Commander 虚拟机。

Commander 支持管理 150 个 CloudTower 或 300 个 SmartX 集群。占用的系统资源如下：

- 4 核 vCPU
- 8 GiB 内存
- 40 GiB 存储

---

## 部署 Commander > 部署前的准备 > 规划网络

# 规划网络

为了确保 Commander 与被管理的 CloudTower 相互通信，在部署 Commander 前，需要规划相应 TCP 端口和 IP 地址。

## 规划端口

为了确保 Commander 的正常运行，当源端访问目标端时，目标端需开放相应端口，开放的端口及端口说明如下：

| 源端 | 目标端 | 目标端需要开放的端口 | 端口说明 |
| --- | --- | --- | --- |
| Commander 虚拟机 | CloudTower 虚拟机 | - 80 - 443 | Commander 虚拟机通过该端口访问 CloudTower 虚拟机的 API |
| 用户端 | Commander 虚拟机 | - 80 - 443 | 用户端通过该端口登录 Commander 虚拟机 |

## 规划 IP 地址

Commander 需要通过虚拟机网络连通待管理的 CloudTower。因此，在部署 Commander 时，请提前为 Commander 虚拟机规划 IP 地址、子网掩码和网关，并确保此 IP 地址与相应的 CloudTower 的 IP 地址连通，请提前向网络管理员申请 IP 地址。

> **注意**：
>
> IP 地址不能与以下 CIDR 范围内的任何 IP 地址重叠：
>
> - 169.254.0.0/16
> - 240.0.0.0/4
> - 224.0.0.0/4
> - 198.18.0.0/15
> - 127.0.0.0/8
> - 0.0.0.0/8
> - 172.17.0.1/16
> - 172.19.100.1/24

## 部署 Commander > 部署前的准备 > 获取安装文件

# 获取安装文件

Commander 仅支持在 SMTX OS（ELF）集群内部署，在部署前请提前获取 Commander 的安装文件，包括 OVF 格式和 VMDK 格式的文件。

## 部署 Commander > 执行部署

# 执行部署

完成部署前的准备后，即可执行部署。请参考以下步骤进行部署。

**前提条件**

待部署集群已和 CloudTower 关联。

**操作步骤**

1. 登录 CloudTower，在**集群**主界面单击右上角 **+ 创建**，选择**导入虚拟机**，弹出**导入虚拟机**对话框。
2. 选择待部署 Commander 的集群和主机，并上传提前获取的 OVF 格式和 VMDK 格式的文件。
3. 分别设置基本信息、计算资源、磁盘、虚拟机网卡和其他配置信息。
4. 单击**确定**，即可完成 Commander 虚拟机的导入。
5. 导入虚拟机后，打开虚拟机控制台，用默认账户 `SmartX` 登录 Commander 的操作系统。
6. 配置 Commander 的 IP 地址、子网掩码和网关，使 Commander 连通待管理的 CloudTower。

   完成部署后，即可使用浏览器访问 Commander 的 IP 地址并进行初始化。

---

## 初始化 Commander（首次登录）

# 初始化 Commander（首次登录）

在浏览器中输入 Commander 虚拟机的 IP 地址，进入 Commander 的初始化界面，并设置账户密码以完成初始化。

当前 Commander 仅支持使用超级管理员的账户进行登录。您需要在首次登录 Commander 时设置 root 的密码以完成初始化。完成初始化后，后续即可使用 root 账户进行登录。

---

## 关联 CloudTower

# 关联 CloudTower

初始化完成后，您需要将待管理的 CloudTower 和 Commander 进行关联，从而实现多个 CloudTower 的统一集中管理。

**前提条件**

待关联的 CloudTower 中至少有一个本地创建的用户，并提前获取该用户的用户名和密码。

**操作步骤**

1. 在 Commander 的 **CloudTower 管理**界面，单击右上角的 **+ 关联 CloudTower**，弹出**关联 CloudTower** 对话框。
2. 填写以下信息：

   | 参数 | 描述 |
   | --- | --- |
   | CloudTower 名称 | 自定义待关联的 CloudTower 的名称 |
   | IP 地址 | 填写待关联 CloudTower 的 IP 地址。 |
   | CloudTower 用户名 | 输入 CloudTower 中任意一个本地创建的用户的用户名。 |
   | CloudTower 密码 | 输入该用户的密码。 |
   | 描述（选填） | 输入待关联的 CloudTower 的描述信息。 |
3. 单击**关联**，完成 CloudTower 的关联。

   完成关联后，Commander 将统计已关联 CloudTower 的资源情况和报警情况并进行展示。当 CloudTower 连接异常、CloudTower 服务异常或 CloudTower 中存在集群连接异常时，可能导致资源统计出现偏差。您可根据界面顶部的提示单击**查看**进行问题定位。

   完成关联后，CloudTower 的相关信息将展示在 **CloudTower 管理**界面的列表中。您可进行以下操作：

   - **编辑**：

     当某一 CloudTower 的关联信息中的 **IP 地址**、**CloudTower 用户名**、**CloudTower 密码**发生了变更，将导致 CloudTower 与 Commander 连接异常，并在界面顶部进行提示。

     您可单击**查看**，查看导致无法连接的原因，并编辑 CloudTower 关联信息的相应字段（IP 地址、CloudTower 用户名或 CloudTower 密码）以进行重新关联。若关联失败，请根据提示信息解决问题后再次执行编辑操作。关联成功后，**关联更新时间**字段将自动更新。
   - **解除关联**：

     当您不需要统计某一 CloudTower 的资源信息和报警信息时，可单击 CloudTower 右侧的 **...**，并选择**解除关联**以解除 CloudTower 的关联。

## 资源盘点

# 资源盘点

Commander 支持对关联的 CloudTower 进行统一管理，并在**资源盘点**页面汇总所有已关联 CloudTower 的资源对象数量以及资源使用情况，以便您快速了解 CloudTower 的整体情况。

---

## 资源盘点 > 查看资源使用情况

# 查看资源使用情况

在 Commander 的**资源盘点**界面的**资源使用情况**页签下展示了关联的所有 CloudTower 中集群的 CPU、内存和存储使用情况。

- **CPU**（仅统计虚拟化平台是 ELF 或 VMware ESXi 的集群）

  展示 Commander 管理的所有集群与 CPU 资源相关的汇总信息。

  - 该区域的右上角展示以下 CPU 信息：
    - **总频率**：集群中所有主机的 CPU 频率之和，例如 `91.6 THz`。
    - **总核数**：集群中所有主机的物理核数之和，例如 `40390 核`。
  - `使用率`：集群中 CPU 的使用率。
- **内存**（仅统计虚拟化平台是 ELF 或 VMware ESXi 的集群）

  展示 Commander 管理的所有集群的内存使用状态，包括**总内存**、**已使用内存**和**当前可分配内存**。
- **存储**

  展示 Commander 管理的所有集群的存储**总容量**，以及**已使用**、**失效**和**空闲**的容量。

  其中，**失效容量**是指由不健康且在线的物理盘、存储失效的节点和正常卸载的物理盘提供的数据空间容量之和。

---

## 资源盘点 > 查看资源汇总

# 查看资源汇总

在 Commander 的**资源盘点**界面的**资源汇总**页签下展示了关联的所有 CloudTower 包括的资源对象的汇总信息。

- **CloudTower**

  展示关联的 CloudTower 的总数及无法连接的 CloudTower 的数量。
- **数据中心**

  展示 Commander 管理的数据中心数量。
- **集群**

  展示 Commander 管理的集群总数及处于各种状态的集群数量。
- **主机**

  展示 Commander 管理的集群中的主机总数、以及处于各种状态的主机数量。
- **虚拟机**（仅统计虚拟化平台为 ELF 的集群）

  展示 Commander 管理的集群中的虚拟机总数，以及处于各种状态的虚拟机数量。

---

## 查看 CloudTower 的相关信息

# 查看 CloudTower 的相关信息

在 **CloudTower 管理**界面展示了所有已关联的 CloudTower 的相关信息，包括基本信息、资源对象数量、计算和存储资源的总量及使用情况。单击某一 CloudTower 的名称，可查看其详细信息。

## 查看 CloudTower 总览信息

在**总览**页签下展示了 CloudTower 信息、资源使用情况及包含的资源对象数量。

- **CloudTower 信息**

  - 基本信息：CloudTower 版本、组织、关联更新时间
  - 网络信息：IP 地址
  - 软件许可：序列号、版本、许可类型、许可过期时间、最大集群数、最大虚拟机数
- **资源使用情况**

  - CPU（仅统计虚拟化平台是 ELF 或 VMware ESXi 的集群）：集群的 CPU 的总核数、总频率和使用率。
  - 内存（仅统计虚拟化平台是 ELF 或 VMware ESXi 的集群）：集群的总内存、已使用内存和可分配内存。
  - 存储：集群的存储总容量，以及已使用、失效和空闲的容量。
- **资源对象数量**

  - 集群数量
  - 数据中心数量
  - 主机数量
  - 虚拟机数量

## 查看资源对象信息

在**数据中心**、**集群**、**主机**、**虚拟机**页签下，以列表的形式展示了 CloudTower 中包括的所有数据中心、集群、主机和虚拟机的相关信息。

- **数据中心**：展示每个数据中心的基本信息、资源对象数量、计算和存储资源的总量及使用情况。
- **集群**：展示每个集群的基本信息、资源对象总量、资源使用情况以及软件许可信息。
- **主机**：展示 CloudTower 管理的所有主机的信息，包括基本信息、虚拟机数量、资源使用情况和物理盘信息。
- **虚拟机**：展示 CloudTower 管理的所有虚拟机的信息，包括基本信息、资源使用情况和网络信息。

## 查看 CloudTower 的报警信息

# 查看 CloudTower 的报警信息

Commander 支持对 CloudTower 收集的报警信息进行汇总展示，方便您进行运维监控，以快速了解各个 CloudTower 的情况。

在 Commander 的**监控**页面的**全部**、**未解决**和**已解决**页签分别展示了 CloudTower 收集的所有、未解决和已解决的报警信息。其中，报警信息包括：所属 CloudTower、产生集群、报警等级、触发时间和持续时间。

## 信息筛选

# 信息筛选

Commander 支持通过关键字搜索或高级搜索的方式对资源信息和报警信息进行筛选，以快速获取目标信息。

## 关键字搜索

在信息列表界面单击搜索框，在搜索框中输入关键字以搜索当前页面的信息。

## 高级搜索

Commander 支持使用高级搜索功能以结合多个条件进行搜索，搜索结果为满足多个筛选条件的交集。

不同信息界面支持的高级搜索条件不同，搜索条件依据其信息列表中可展示的信息而定。

**搜索方法**

1. 进入信息列表界面，单击筛选框左侧的筛选图标，弹出筛选条件列表。
2. 选择筛选项并输入筛选条件。没有输入搜索条件的项默认搜索范围为全部。不同筛选项的搜索结果为交集关系，即搜索出的资源同时满足多个筛选项条件。
3. 单击**筛选**，在信息列表中查看筛选结果。

## 导出报表

# 导出报表

Commander 支持导出 .csv 格式的报表。您可以在相应资源信息和报警信息页面单击右上角的下载图标将相应的报表导出至本地，以对所有资源信息和报警信息进行统计，便于您实时掌握全局的资源情况和报警情况。

## 设置账号安全

# 设置账号安全

在**个人设置**的**账号安全**模块，支持修改 root 账号的密码。

**操作步骤**

1. 单击 Commander 主界面右上角的用户图标，选择**个人设置**，进入**账号安全**页面。
2. 该页面展示了已登录账号的基本信息和个人安全信息，单击密码右侧的**更新**，弹出更新密码的下拉框。
3. 在下拉框中输入当前密码和新密码，并再次确认密码。
4. 单击**更新**，完成密码更新。更新密码之后，账户将自动退出登录，您需要输入新密码进行重新登录。
