## 版本更新说明

# 版本更新说明

## 新增

- 支持迁移虚拟机后自动卸载目标端虚拟机的 VMware Tools。
- 支持设置在迁移增量数据后自动关闭源端虚拟机。
- 当目标集群为 6.3.0 及以上版本的 SMTX OS 或 SMTX ELF 集群时，创建迁移任务时支持为启用静态数据加密的虚拟机选择加密算法。

## 改进

- 当目标集群为 6.3.0 及以上版本的 SMTX OS 或 SMTX ELF 集群时，迁移后目标端虚拟机的虚拟卷条带数默认设置为 8，以提升存储性能。
- 优化了创建迁移任务时目标端站点剩余数据空间与内存资源展示：
  - 增加当前未完成的迁移任务预计将会占用的存储与内存资源展示；
  - 若目标端站点在迁移完成后，因资源不足可能导致单台主机故障后 HA 失败时，系统将进行提示。
- 创建源端站点为 VMware vCenter 站点时的迁移任务时，增加源端虚拟机版本检查。

## 修复问题

- 修复了 Linux 虚拟机迁移后，通过 CloudTower 界面配置网络可能失效的问题。
- 修复了 VDDK 版本为 7.0 及以上时，可能导致带有 RDM 磁盘的虚拟机迁移失败的问题。

## 版本配套说明

# 版本配套说明

## 与服务器架构配套说明

通过本版本的 SMTX 迁移工具执行迁移时，源端和目标端集群的节点支持的服务器架构如下。

| 源端集群节点 | 目标端集群节点 |
| --- | --- |
| Intel x86\_64 架构 | Intel x86\_64 架构 |
| Intel x86\_64 架构 | Hygon x86\_64 架构 |

## 与 SMTX OS 及 SMTX ELF 版本配套说明

本版本的 SMTX 迁移工具支持以下版本的 SMTX OS（ELF）集群或 SMTX ELF 集群作为迁移的源端或目标端集群。

- **源端**：

  **SMTX OS（ELF） 集群**：5.0.3 及以上的 5.x.x 和 6.x.x 版本
- **目标端**：

  - **SMTX OS（ELF） 集群**：5.0.3 及以上的 5.x.x 和 6.x.x 版本
  - **SMTX ELF 集群**：6.0.0、6.2.0、6.3.0

## 与 VMware vSphere 版本配套说明

本版本的 SMTX 迁移工具支持从以下版本的 VMware ESXi 平台迁移虚拟机至 ELF 平台，并支持以下 vCenter 版本。

- ESXi 组件版本：5.0, 5.1, 5.5, 6.0, 6.5, 6.7, 7.0, 8.0
- vCenter 组件版本：6.0, 6.5, 6.7, 7.0, 8.0

## 与虚拟机操作系统版本配套说明

本版本的 SMTX 迁移工具不支持迁移 32 位操作系统的虚拟机。

当源端集群为使用 VMware ESXi 平台的集群时，本版本的 SMTX 迁移工具支持迁移操作系统版本和固件类型在《SMTX 迁移工具兼容性列表》上的虚拟机。

对于操作系统版本和固件类型不在上述兼容性列表，但在《 SMTX 虚拟机服务兼容性指南》中的 Windows 和 Linux 虚拟机也支持通过 SMTX 迁移工具迁移，但在迁移后可能将以 IDE 总线启动，需要手动安装 Virtio 驱动。

## 版本的已知问题

# 版本的已知问题

- vCenter 7.0 只读账号可以关联成功。
- 若源端虚拟机的系统盘使用了硬 RAID 或软 RAID，则 SMTX 迁移工具无法在迁移后为虚拟机的系统盘安装磁盘驱动。
- 若源端集群使用的是免费的 license，则不支持迁移。
- 不支持从使用免费许可的 VMware ESXi 主机迁移虚拟机。
