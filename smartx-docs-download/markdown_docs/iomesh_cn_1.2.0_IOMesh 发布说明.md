## 版本更新说明

# 版本更新说明

## 新增

### 存储功能

- IOMesh 集群的部署模式为混闪配置模式时，支持使用纠删码作为 PV 的冗余策略。
- IOMesh 集群的部署模式为混闪配置模式时，支持为 PV 启用卷常驻缓存功能。启用后，PV 中的数据将保留在缓存层中，从而提供更稳定的高性能。
- 支持将 PV 的副本数从 2 提升至 3。

### 运维管理

- UI 支持简体中文语言选项。
- UI 支持展示更多物理盘健康信息，帮助判断物理盘状态。
- UI 支持展示 Meta Pod 的 Leader 角色信息，提升更换物理盘时的效率。
- IOMesh Dashboard 支持展示物理盘亚健康信息。

## 改进

- 优化了多 IOMesh 集群场景下，单个 IOMesh 集群的 chunk 或 meta 异常后的处理，避免其他 IOMesh 集群的运维操作受到影响。
- 优化了对未启用 WWN 的物理盘的处理方式：

  - 全闪配置模式下可正常挂载未启用 WWN 的物理盘。
  - 混闪配置模式下自动拒绝挂载未启用 WWN 的物理盘，对存量已挂载物理盘无影响。
- 限制了 PV 容量需为 2Gi 的整倍数，优化物理盘大于 4 块时的读写性能。
- 限制了各组件可使用的 CPU 和内存上限，避免内存泄漏影响到用户环境。

## 修复问题

- 修复了 Kubernetes 集群 Node 名称和 IOMesh 节点的 hostname 不一致时，因 IOMesh Dashboard 无法正常展示各个节点物理盘状态，导致监控报警信息异常的问题。
- 修复了全闪配置模式下，卸载未启用 WWN 的物理盘后 blockdevice 对象有残留的问题。
- 修复了 IOMesh 集群被删除后，metrics 未被清理的问题。
- 修复了 IOMesh 节点不断上报 udev device add/remove 事件时的 NDM 内存泄漏问题。

## 版本配套说明

# 版本配套说明

## 与服务器类型配套说明

IOMesh 支持运行在 Intel x86\_64、Hygon x86\_64、鲲鹏 AArch64 架构的服务器上。

## 与 Kubernetes 和 Linux 操作系统的配套说明

| 软件/操作系统名称 | 适配版本 |
| --- | --- |
| Kubernetes | - Kubernetes v1.17-v1.27 - Openshift v3.11-v4.14 |
| Linux 操作系统 | - CentOS 7 - CentOS 8 - CoreOS - RHEL 7 - RHEL 8 - OpenEluer 22.03 LTS |

> **说明**：
>
> IOMesh 对 Linux 操作系统的版本没有依赖，表格中仅列出经过测试的版本。
