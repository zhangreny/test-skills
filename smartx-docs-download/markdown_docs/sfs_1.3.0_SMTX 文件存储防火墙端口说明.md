# sfs/1.3.0/SMTX 文件存储防火墙端口说明
## 关于本文档

# 关于本文档

本文档描述了与 SMTX 文件存储（SMTX File Storage，简称“SFS”）相关的防火墙端口规则，具体涵盖以下内容：

- 如何配置文件控制器所在主机网络之间的防火墙端口、文件存储集群和外部客户端之间的防火墙端口，以确保集群能正常对外提供服务。
- 文件存储内部服务占用的本地端口，以及内部服务依赖的其他服务占用的端口。

> **注意**：
>
> - 请勿修改文件控制器所在主机网络防火墙的默认配置和默认状态(关闭)，否则会引起系统异常。
> - 若未作特殊说明，本文所提及端口均为 TCP 端口。

# 文档更新信息

**2026-02-06：文档随 SMTX 文件存储 1.3.0 正式发布**

相较于 1.2.1，本版本更新如下：

- 多个服务的部分端口使用的网络从文件管理网络修改为文件存储网络。

## 外部防火墙端口开放规则

# 外部防火墙端口开放规则

若文件控制器所在物理主机的网络之间存在防火墙（如交换机上），或文件存储集群与外部客户端之间存在防火墙，为确保文件存储集群能够正常对外提供服务，防火墙均需开放如下端口。

以下开放规则仅讨论默认端口号。

## SSH

| 端口 | 用途 | 说明 |
| --- | --- | --- |
| 22 | SSH 远程登录文件控制器。 | SSH 协议的默认端口号。 |

## NFS

外部客户端使用文件存储集群的 NFS 存储服务时，请确保客户端和集群之间的防火墙开启了如下端口：

| 协议版本 | 端口 | 用途 | 说明 |
| --- | --- | --- | --- |
| NFSv3 | 111、2049、2050 | 通过 NFSv3 协议访问文件存储集群的分布式文件存储。 | 标准 NFS 服务端口。 |
| NFSv4.1 | 2049 | 通过 NFSv4.1 协议访问文件存储集群的分布式文件存储。 | 标准 NFS 服务端口。 |

## HDFS

外部客户端使用文件存储集群的 HDFS 存储服务时，请确保客户端和集群之间的防火墙开启了如下端口：

| 端口 | 用途 | 说明 |
| --- | --- | --- |
| 20010 | 访问文件系统的元数据。 | SFS 元数据服务端口。 |
| 20020 | 访问文件系统的数据。 | SFS 数据服务端口。 |
| 6443 | 访问 SFS API server 以获取文件存储集群控制信息。 | SFS API server 服务端口。 |
| 8429 | 通过文件控制器将 Hadoop 客户端的性能监控数据上传到监控平台。 | SFS OBS agent 监控端口。 |
| 20005 | 访问 SFS cloud provider 以获取文件存储集群的授权信息。 | SFS cloud provider 端口。 |

## 文件存储服务防火墙端口配置要求

# 文件存储服务防火墙端口配置要求

为确保文件存储集群正常提供服务，源端和目标端都需要开放如下表所示的端口及协议。如没有特殊说明，则表示端口的传输端协议为 TCP。

| 源端 | 目标端 | 需开放的端口和协议 | 用途 |
| --- | --- | --- | --- |
| 文件控制器文件管理 IP 和文件存储集群文件管理虚拟 IP | SMTX OS（ELF）集群的管理虚拟 IP | 10206 | SFS 的 sfs-cloud-provider 服务依赖 SMTX OS（ELF）集群的 zbs-chunkd 提供的 meta proxy 服务。 |
| 文件控制器文件管理 IP | CloudTower IP | 80 | 文件控制器通过此端口调用 CloudTower API 上传 island。 |
| 文件控制器文件管理 IP | CloudTower 中配置的 NTP 服务器 IP | UDP：123 | 文件控制器通过此端口访问 NTP 服务器进行时间同步。 |
| 文件控制器文件管理 IP | 可观测性虚拟机管理 IP | 80 | 文件控制器内的 obs-agent 通过此端口调用 OVM API 上传采集的数据。 |
| 443 | 配套 1.4.2 及以上版本可观测性平台使用时需开放。  文件控制器内的 obs-agent 通过此端口调用 OVM API 上传采集的数据。 |
| CloudTower IP | 文件控制器文件管理 IP 和文件存储集群文件管理虚拟 IP | 6443 | CloudTower 和其上的 sfs-operator 服务通过此端口访问文件存储集群中的 API server 服务。 |
| ICMP | 部署在 6.2.0 以下版本的 SMTX OS（ELF）集群中需开放。  CloudTower 上的 sfs-operator 服务通过此协议检测文件控制器文件管理 IP 和文件管理虚拟 IP 是否已被占用。 |
| CloudTower IP | 文件控制器文件管理 IP | 22 | CloudTower 上的 CAPP 服务通过此端口建立 SSH 隧道代理文件控制器内 containerd 服务的 socket。 |
| 20003 | CloudTower 上的 sfs-operator 服务通过此端口访问文件控制器内的 NTPM 服务。 |
| 8685 | CloudTower 上的 bundle-application-operator 服务通过此端口访问文件控制器内的 obs-agent 服务。 |

## 文件存储内部服务占用端口

# 文件存储内部服务占用端口

为了保证 SFS 的服务或功能正常使用，请您确保文件控制器的以下端口处于可用或者开放状态，并确保服务使用的端口未被其他服务所占用。

| 服务名 | 使用网络 | 使用端口 |
| --- | --- | --- |
| sfs-agent | 文件存储网络 | 20045 |
| sfs-meta | 文件存储网络 | 20010、20015 |
| sfs-data | 文件存储网络 | 20020、20025 |
| sfs-ganesha | 文件接入网络 | 111、2049、2050 |
| 文件存储网络 | 9587、20030、20031、20035 |
| sfs-manager | 文件存储网络 | 20002、20050 |
| sfs-cloud-provider | 文件存储网络 | 20005、20006 |
| ntpm | 文件管理网络 | 20003 |
| obs-agent | 文件管理网络 | 8681、8682、8685、8429 |
| etcd-operator | 文件存储网络 | 20007 |
| traefik | 文件存储网络 | 20001 |
| kube-etcd | 文件管理网络  文件存储网络 | 2379、2380 |
| kube-apiserver | 文件管理网络  文件存储网络 | 6443 |
| node-exporter | 文件存储网络 | 9100 |
| cadvisor | 文件存储网络 | 8080 |
| sshd | 文件管理网络  文件存储网络 | 22 |
| rpcbind | 文件接入网络 | 111 |
| rebpf-exporter | 文件存储网络 | 9435 |
