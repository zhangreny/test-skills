## 版本更新说明

# 版本更新说明

## 新增

- 支持在原生 Kubernetes 集群类型中部署高可用数据平面。
- 支持模型缓存，提升模型加载速度。
- 支持将在线下载的模型分发到统一模型仓库。
- 支持暂停推理实例。
- 支持推理引擎版本包管理，以快速验证新发布的模型。
- 支持用户配置 RBAC。
- 支持 API 密钥认证。
- 支持敏感字段自动过滤，减少安全风险。
- 支持资源名称规范校验和资源非空参数校验。
- 支持控制器层面的强制删除资源，优雅处理异常资源清理场景。
- 新增许可管理，默认提供 90 天的使用许可。
- 支持 OpenShift 4.20 集群。

## 改进

- Kubernetes 集群 vLLM 版本升级至 0.11.2，支持 DeepSeek-V3.2-Exp、Qwen3-VL、Qwen3-Next 等模型，提升性能和兼容性。
- 优化了集群和推理实例的状态管理和错误展示。
- 为资源添加依赖关系检查，防止误删除。
- 优化安装部署流程，支持将离线镜像一键导入至镜像仓库。
- 优化模型上传处理，使用流式上传机制避免占用临时磁盘空间。
- 移除 NFS 挂载版本参数，提升兼容性，当前支持 NFSv3 和 NFSv4.x 版本。
- 固定 Ray 端口配置，避免端口冲突。
- 减少控制平面导出端口数量。
- 更新 token 使用的统计维度为推理实例级别。

## 修复问题

- 修复了 tensor parallel > 1 时 vLLM GPU P2P 检查失败的问题。
- 修复了静态节点集群异常无法恢复的问题。
- 修复了允许上传 latest 版本模型的问题。
- 修复了获取模型时模型大小返回为空的问题。
- 修复了镜像仓库认证配置不生效的问题。
- 修复了 Hugging Face token 验证逻辑错误的问题。
- 修复 API 密钥使用量不准确的问题。
- 修复了重复创建同名静态节点集群无法初始化的问题。
- 修复了默认工作空间可被删除的安全问题。
- 修复了推理实例代理访问路径冲突的问题。
- 修复了静态节点集群 llama-cpp 初始化时未检查配置的问题。

## 版本兼容说明

# 版本兼容说明

## 与 CPU 架构的兼容性

服务器的 CPU 需为如下架构：

- Intel x86\_64
- Hygon x86\_64
- AMD x86\_64
- 鲲鹏 AArch64
- 飞腾 AArch64

## 与 Kubernetes 版本的兼容性

榫卯 AI 平台兼容 Kubernetes v1.26 及以上版本。

已经过充分测试验证的版本如下：

- Kubernetes v1.26
- Kubernetes v1.30

## 与操作系统的兼容性

榫卯 AI 平台兼容的操作系统与版本如下：

- RHEL/Rocky Linux 8.10 或更高版本
- Ubuntu 22.04 LTS
- OpenEuler 22.03 LTS
