## 版本更新说明

# 版本更新说明

## 新增功能

### 备份与恢复

- 当原备份服务不可用时，可通过恢复备份存储库功能重新关联新的备份服务，并自动恢复备份存储库中已有的备份计划及恢复点。
- 创建备份存储库时，支持使用仅包含系统文件的非空路径，并在指定路径不存在且父目录存在时自动创建子目录作为备份存储库文件夹。
- 在原备份服务不可用的情况下，通过命令行从恢复点恢复虚拟机时，可交互式选择恢复选项。

### 复制与恢复

- 虚拟机新增同步复制计划类型，支持副本虚拟机与原虚拟机 RPO=0 的实时同步。

### 运维管理

- 备份与复制服务在虚拟机跨集群迁移后，支持自动继续保护迁移后的虚拟机，并可使用迁移前的恢复点进行重建恢复或故障处理操作。
- 虚拟机快照支持显示关联的备份计划或复制计划。

## 改进

- 优化了备份与复制服务在增量备份或复制时对改动数据块的处理逻辑。在卷扩容场景下，仅读取扩容后实际有数据的块，跳过全零数据块。

## 修复问题

- 修复了复制服务在网络映射时，将源端配置静态 IP 的 VLAN 网卡映射到目标端 VPC 网卡后可能导致目标网卡失效的问题。
- 修复了备份存储库异常可能导致关联的备份服务异常重启的问题。
- 修复了备份服务在执行增量备份任务期间，可能因自动触发恢复点清理而导致任务失败的问题。
- 修复了因备份存储库连接异常导致增量备份任务失败后，后续的增量备份被错误转为全量备份的问题。
- 修复了故障处理生成的恢复点被复制服务错误自动清理的问题。
- 修复了复制对象在进行故障转移测试期间，复制服务未按保留策略自动清理恢复点的问题。
- 修复了任务报表通知中，由故障处理触发的复制任务不显示关联复制计划名称的问题。
- 修复了隐藏的 RPC 服务安全问题。
- 修复了安全配置问题，减少不必要的端口暴露。

## 版本配套说明

# 版本配套说明

## 与集群版本配套说明

SMTX 备份与容灾适用于以下版本的 SMTX OS（ELF）集群和 SMTX ZBS 集群。同一备份服务或复制服务支持关联多种 CPU 架构的集群。

| 集群类型 | 配套版本 |
| --- | --- |
| SMTX OS（ELF） | - 5.0.3 及以上的 5.0.x 版本 - 5.1.0 及以上的 5.1.x 版本 - 6.0.0 及以上的 6.x.x 版本  **注意**：使用同步复制保护虚拟机时，待保护虚拟机所在的集群需为 6.3.0 及以上的 6.x.x 版本。 |
| SMTX ZBS | - 5.6.0 及以上的 5.6.x 版本 - 5.7.0 及以上版本的 5.7.x 版本 |

## 与 CloudTower 版本配套说明

本版本 SMTX 备份与容灾只支持使用 CloudTower 4.8.0 及以上版本管理。

## 版本使用限制

# 版本使用限制

- 本版本备份服务和复制服务可支持的备份和复制数据量如下：

  - 单个备份服务可对实际数据量不超过 100 TiB 的单个虚拟机进行备份，也可对总实际数据量之和不超过 100 TiB 的多个虚拟机进行备份。若多个虚拟机的实际数据量之和超过 100 TiB，建议部署多个备份服务以完成操作。
  - 单个复制服务可对实际数据量不超过 320 TiB 的单个虚拟机或存储卷进行复制，也可对总实际数据量不超过 320 TiB 的多个虚拟机或存储卷进行复制。若需复制的实际数据量超出 320 TiB，建议部署多个复制服务以完成操作。
  > **说明**：
  >
  > 实际数据量指虚拟卷的逻辑已使用容量。对于精简置备虚拟卷，为实际已使用的容量；对于厚置备虚拟卷，为分配的容量。
- 本版本仅支持使用集群的存储网络对虚拟机进行备份和恢复。如需备份多个集群，且集群间的存储网络不连通，需为每个集群单独部署备份服务。
- 本版本仅支持使用集群的存储网络对虚拟机进行复制和恢复。如需复制多个集群，且集群间的存储网络不连通，需为每个集群单独部署复制服务。
- IP 映射规则仅在源集群和目标集群为以下版本时可生效。

  - SMTX OS 5.0.6 及以上的 5.0.x 版本。
  - SMTX OS 5.1.0 及以上的 5.1.x 版本。
  - SMTX OS 6.0.0 及以上的 6.x.x 版本。
