# sfs/1.3.0/SMTX 文件存储故障场景说明
## 关于本文档

# 关于本文档

本文档介绍了 SMTX 文件存储（SMTX File Storage，简称“SFS”）在实际运行过程中可能遇到的故障场景和故障影响。

阅读本文档需了解 SFS 分布式文件存储软件、SMTX OS 超融合软件，了解软件定义网络、分布式存储等相关技术背景知识，并具备丰富的数据中心操作经验。

## 文件控制器故障

# 文件控制器故障

## 文件控制器类型

文件控制器根据其上运行的服务分为主节点和存储节点。详细信息请参考《SMTX 文件存储内部服务说明》的[服务功能描述](/sfs/1.3.0/sfs_service_list/sfs_service_list_01)章节。

### 主节点

每个文件存储集群包含 3 个主节点。主节点上运行着文件存储集群的[管控服务](/sfs/1.3.0/sfs_failure_scenario_guide/sfs_failure_scenario_guide_01#%E7%AE%A1%E6%8E%A7%E6%9C%8D%E5%8A%A1)和[存储服务](/sfs/1.3.0/sfs_failure_scenario_guide/sfs_failure_scenario_guide_01#%E5%AD%98%E5%82%A8%E6%9C%8D%E5%8A%A1)。

### 存储节点

存储节点只运行文件存储集群的[存储服务](/sfs/1.3.0/sfs_failure_scenario_guide/sfs_failure_scenario_guide_01#%E5%AD%98%E5%82%A8%E6%9C%8D%E5%8A%A1)。

## 服务类型

服务主要分为管控服务和存储服务。

### 管控服务

管控服务主要包括 kube-etcd、kube-apiserver、sfs-manager、sfs-cloud-provider 等，负责管理和控制文件存储集群。

- kube-etcd 是一个分布式键值数据库，提供存、取 CRD（CustomResourceDefinition）对象的服务。
- kube-apiserver 提供管理文件存储集群的 REST API 接口，包括认证授权、数据校验以及集群状态变更等，同时还是其他模块之间的数据交互和通信的枢纽。
- sfs-manager 提供文件存储集群的管理和控制服务。
- sfs-cloud-provider 负责接入底层块存储。

kube-etcd、sfs-manager 和 sfs-cloud-provider 都是以集群的形式提供服务，集群中包含 leader 和 follower。同一时刻，集群中只有唯一的 leader 对外提供服务。leader 发生故障后，将自动从 follower 中选出新的 leader 对外提供服务。

### 存储服务

存储服务主要包括 sfs-meta、sfs-data、sfs-ganesha 等。

- sfs-meta 提供文件系统的元数据服务，sfs-data 提供文件系统的数据服务。

  shard 是文件的数据和元数据的载体，根据 shard 中承载的数据类型将其分为 meta shard 和 data shard 两种类型，文件系统的元数据存放在 meta shard 中，文件系统的数据存放在 data shard 中。每个 shard 都会挂载到两个文件控制器上，其中一个作为 shard leader，另一个为 shard follower。同一时间，一个 shard 中只有 shard leader 提供服务。

  为了保证文件系统的高可用，shard leader 所在的文件控制器故障时，该 shard 中位于其他文件控制器上的 shard follower 会被选为 shard leader 继续提供服务。
- sfs-ganesha 提供 NFS 接入服务。

  sfs-ganesha 通过文件接入 IP 对计算端提供 NFS 服务，每一个文件控制器上的 sfs-ganesha 服务对应一个文件接入 IP，计算端使用文件接入 IP 挂载文件系统。

  为了保证文件接入服务的高可用，sfs-ganesha 所在的文件控制器故障时，SFS 将自动触发文件接入 IP 迁移，将故障文件控制器上的文件接入 IP 迁移至健康的文件控制器，以保证计算端可正常访问使用该文件接入 IP 挂载的文件系统。当原文件控制器恢复健康后，该文件接入 IP 将会迁移回原文件控制器，以保证文件接入 IP 的均衡和性能的稳定。

## 故障影响

### 主节点故障影响

单个主节点故障对 SFS 的影响如下：

- **文件存储集群异常**

  如果 kube-etcd、sfs-cloud-provider 和 sfs-manager 服务的 leader 位于发生故障的文件控制器，kube-etcd 通过投票机制自动重新选主，etcd 完成选主后，自动触发 sfs-cloud-provider 和 sfs-manager 服务选主。选主期间，文件存储集群将异常，您无法通过 CloudTower 在文件存储集群中创建、上线、下线和删除文件系统，但计算端仍然可以访问已挂载的文件系统。
- **I/O 将被暂停和重试**

  - 如果发生故障的文件控制器挂载了文件系统的 meta shard 或者 data shard 的 shard leader，故障文件控制器上的 shard leader 将无法提供服务，SFS 会自动将原本的 shard follower 切换为 shard leader 继续提供服务。shard 切换过程中，shard 对应的文件系统的 I/O 将被暂停和重试。
  - 如果故障文件控制器的故障后重启完成时间大于阈值，SFS 将自动触发文件接入 IP 迁移，将故障文件控制器上的文件接入 IP 迁移至健康的文件控制器。文件接入 IP 迁移过程中，通过对应文件接入 IP 挂载文件系统的计算端的 I/O 将被暂停和重试。

多个主节点故障对 SFS 影响如下：文件存储集群无法正常工作，计算端停止 I/O 。

### 存储节点故障影响

单个存储节点故障对 SFS 的影响如下：

- 您可以在 CloudTower 上查看文件存储集群的信息，但无法创建、上线、下线和删除文件系统，计算端仍然可以访问已挂载的文件系统。
- 如果发生故障的文件控制器挂载了文件系统的 meta shard 或者 data shard 的 shard leader，故障文件控制器上的 shard leader 将无法提供服务，SFS 会自动将原本的 shard follower 切换为 shard leader 继续提供服务。shard 切换过程中，shard 对应的文件系统的 I/O 将被暂停和重试。
- 如果故障文件控制器的故障后重启完成时间大于阈值，SFS 将自动触发文件接入 IP 迁移，将故障文件控制器上的文件接入 IP 迁移至健康的文件控制器。文件接入 IP 迁移过程中，通过对应文件接入 IP 挂载文件系统的计算端的 I/O 将被暂停和重试。

多个存储节点故障对 SFS 的影响如下：

- 您可以在 CloudTower 上查看文件存储集群的信息，但无法创建、上线、下线和删除文件系统，计算端仍然可以访问已挂载的文件系统。
- 如果文件系统的 shard leader 和 shard follower 所在的文件控制器均故障，则该文件系统的 I/O 将一直重试，直至超时。

## 物理节点故障

# 物理节点故障

SMTX OS（ELF）集群的节点故障对文件存储集群的影响包含两个方面：

- **文件控制器**

  文件控制器实质是运行在 SMTX OS（ELF）集群的节点上的虚拟机。SMTX OS（ELF）集群的节点发生故障，可能会导致节点上的文件控制器发生故障。
- **文件存储服务**

  文件系统的冗余策略中的副本数依赖于 SMTX OS（ELF）集群中的块存储服务。SMTX OS（ELF）集群的节点发生故障，可能会影响 SMTX OS（ELF）集群中的块存储服务，进而影响文件存储集群提供的文件存储服务。

SMTX OS（ELF）集群节点故障对虚拟机和块存储服务的影响，您可以查看对应版本的《SMTX OS 故障场景说明》的**节点故障**章节。

## 网络故障

# 网络故障

文件存储集群内包含文件管理网络、文件存储网络和文件接入网络。SMTX OS（ELF）集群中的三个虚拟机网络作为上述三个网络的实际载体，并且每个虚拟机网络都需要关联虚拟分布式交换机。同时，文件存储集群中每个文件控制器将通过三个虚拟网卡分别关联这三个虚拟机网络。

如 SMTX OS（ELF）集群中文件存储集群网络对应的虚拟机网络发生故障，故障的虚拟机网络上关联的文件控制器将无法正常进行网络通信。虚拟机网络发生故障的原因一般是关联的虚拟分布式交换机上对应的物理网口发生故障。本小节讨论的网络故障，是文件控制器所在的物理节点上，文件存储集群网络对应的物理网口及该网口绑定的物理网口均发生了故障。

## 文件管理网络故障影响

CloudTower 通过文件管理网络访问文件存储集群的 API server，从而对文件存储集群的各个资源进行创建、读取、更新和删除等操作。

文件控制器的文件管理网卡对应的物理网口故障时，文件存储集群仍然可以提供正常 I/O。

文件管理网络所属的虚拟分布式交换机故障时，无法通过 CloudTower 管理文件存储集群，此时已创建的文件系统可以正常被挂载和使用。

## 文件存储网络故障影响

文件存储网络是文件存储集群内部服务之间通信的网络。

单个文件控制器的文件存储网卡对应的物理网口故障时，该文件控制器上的内部服务的通信将受到影响，该文件控制器与其他文件控制器将无法通信，对文件存储集群的影响等同于单个文件控制器故障，请参考[文件控制器故障](/sfs/1.3.0/sfs_failure_scenario_guide/sfs_failure_scenario_guide_01)。

所有文件控制器的文件存储网卡对应的物理网口均故障，或者文件存储网络所属的虚拟分布式交换机故障时，对文件存储集群的影响等同于所有文件控制器故障，请参考[文件控制器故障](/sfs/1.3.0/sfs_failure_scenario_guide/sfs_failure_scenario_guide_01)。

## 文件接入网络故障影响

文件接入网络是文件存储集群对外提供文件存储服务的网络，计算端通过此网络挂载文件系统访问文件存储服务。

单个文件控制器的文件接入网卡对应的物理网口故障时，文件存储集群将自动触发文件接入 IP 迁移，将文件接入 IP 迁移至健康的文件控制器。在文件接入 IP 迁移过程中，I/O 将被暂停和重试，直至文件接入 IP 迁移完成。

所有文件控制器的文件接入网卡对应的物理网口均故障，或者文件接入网络所属的虚拟分布式交换机故障时，I/O 将一直被暂停，直至物理网口故障恢复。
