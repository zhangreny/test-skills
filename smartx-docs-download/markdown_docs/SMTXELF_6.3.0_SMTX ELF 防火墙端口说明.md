# SMTXELF/6.3.0/SMTX ELF 防火墙端口说明
## 关于本文档

# 关于本文档

本文档描述了与 SMTX ELF 相关的防火墙端口规则，具体涵盖以下内容：

- 如何配置集群主机网络之间的防火墙端口、集群和外部客户端之间的防火墙端口，以确保集群能正常对外提供服务。
- 虚拟机跨集群迁移的端口开放规则。
- SMTX ELF 内部服务占用的本地端口。

> **注意：**
>
> - 请勿修改 SMTX ELF 集群主机防火墙的默认配置和默认状态(关闭)，否则会引起系统异常。
> - 若未作特殊说明，本文所提及端口均为 TCP 端口。

## 外部防火墙端口开放规则

# 外部防火墙端口开放规则

若集群物理主机的网络之间存在防火墙（如交换机上），或集群与外部客户端之间存在防火墙，为确保 SMTX ELF 集群能够正常对外提供服务，防火墙均需开放如下端口。

以下开放规则仅讨论默认端口号。

## Web 控制台

| 端口 | 用途 | 说明 |
| --- | --- | --- |
| 443 | 网页浏览端口，用于 HTTPS 服务 | - |
| 80 | 网页浏览端口，用于 HTTP 服务 | - |

如需要从 SMTX Web 控制台操作界面打开虚拟机控制台，除开放以上端口外，还需要开放 VNC 相关端口。

## SSH

| 端口 | 用途 | 说明 |
| --- | --- | --- |
| 22 | SSH 远程登录物理主机/SCVM/客户虚拟机 | SSH 协议的默认端口号 |

## IPMI

| 端口 | 用途 | 说明 |
| --- | --- | --- |
| 623（UDP） | 访问 IPMI 管理台 | 适用于戴尔、超微服务器。其他厂商产品请参见其详细要求。 |

## SNMP

| 端口 | 用途 | 说明 |
| --- | --- | --- |
| 1705 | 对接 SMTX ELF 监控报警服务 octopus，用于从 octopus 获取监控数据。 | 若未添加 SNMP 传输，则无需开放此端口。 |

## 虚拟机跨集群迁移的端口开放规则

# 虚拟机跨集群迁移的端口开放规则

当虚拟机进行跨集群迁移时，根据待迁移虚拟机所属的源端集群和目标端集群的类型，需要采用不同的端口开放规则，下面进行详细说明。

## 源端集群部署在 SMTX ELF 集群，目标端集群部署在 SMTX OS (ELF) 集群或 SMTX ELF 集群

支持使用 CloudTower 的跨集群迁移功能完成迁移任务，源端集群节点和目标端集群节点需开放的端口及端口要求如下表。

| **迁移方式** | **端口要求** |
| --- | --- |
| **热迁移** | 源端集群节点与目标端集群节点迁移时使用的网络需双向开通以下 TCP 端口：   - 16509 - 16514 - 49152～49215   源端集群节点与目标端集群节点的管理网络需双向开通以下 TCP 端口：   - 80 - 443 |

## 源端集群部署在 VMware ESXi 平台，目标端集群部署在 SMTX ELF 集群

支持使用迁移工具完成迁移任务，相应的端口开放规则请参考《SMTX 迁移工具用户指南》。

## SMTX ELF 内部服务占用端口

# SMTX ELF 内部服务占用端口

SMTX ELF 默认不启用本地防火墙。为了保证 SMTX ELF 的内部服务或功能正常使用，用户必须要确保集群主机网络之间的防火墙开放以下使用的 TCP 或 UDP 端口。

## 随机端口

在实际场景中，NFS 服务和客户端本地也可能使用一些随机端口，具体说明如下：

- NFS 服务除了使用 `2049` 和 `111` 两个固定端口外，还会启动其他进程（如 status、nlockmgr 等），这些进程的端口由 RPC 服务进行随机分配，每次启动 NFS 服务时使用的端口均不相同。

  若需查看 NFS 当前使用的所有端口，或查看某一未知的端口对应的进程，可采用如下方法：

  - 查看 NFS 当前使用的随机端口：在主机执行 `rpcinfo -p` 命令查看。
  - 查看某一端口对应的服务进程：在主机执行 `netstat -nlp` 命令查看。
- 客户端本地也可能启用一些随机端口对接集群服务。若扫描到此类未知的本地端口，可在主机执行 `netstat -nlp` 命令，查看该端口对应的源端口或目的端口，并根据 SMTX ELF 服务使用的固定端口，确定本地端口对应的集群服务。
