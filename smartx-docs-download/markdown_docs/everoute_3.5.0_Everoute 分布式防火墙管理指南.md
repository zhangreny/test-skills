# everoute/3.5.0/Everoute 分布式防火墙管理指南
## 关于本文档

# 关于本文档

Everoute 是由北京志凌海纳科技股份有限公司（以下简称 “SmartX”）自主研发的网络与安全产品，为 SmartX 超融合和企业云基础设施提供了软件定义的网络和安全系列功能。Everoute 可提供分布式防火墙、网络安全导流、负载均衡、虚拟专有云网络、容器网络等功能，同时支撑虚拟化和容器化应用，从而形成与虚拟机和容器一体化管理的网络和安全解决方案。

本文档介绍了如何使用 Everoute 分布式防火墙的功能特性。阅读本文档的对象需了解 SMTX OS 超融合软件、SMTX ELF 虚拟化软件以及 Everoute 服务的相关技术背景知识，并具备丰富的数据中心操作经验。

> **声明**：
>
> 由于 Everoute 和 CloudTower 软件对外提供多个版本，且每个版本的功能和服务均有所不同。为了完整描述 Everoute 软件的功能和特性，本⽂档中所有功能均基于 Everoute 3.5.0 和 CloudTower 4.8.0。若您所购买的软件版本不包含文档中描述的部分功能和特性，请忽略并以实际使用的软件功能为准。

## 概述

# 概述

Everoute 分布式防火墙（Distributed Firewall）实现了基于零信任网络模型的微分段技术，解决数据中心虚拟网络安全面临的挑战，为业务提供更加灵活精细的网络安全保障。

Everoute 分布式防火墙支持虚拟机隔离、自定义安全策略、全局安全策略三种形态的分布式防火墙策略。Everoute 为虚拟化和 Kubernetes 所承载的应用负载提供分布式安全策略的管理和执行能力，符合“零信任架构”对网络安全的要求。

---

## 概述 > 产品功能

# 产品功能

- **全局安全策略**

  支持在全局设置是否启用默认拒绝通信，并且支持设置全局白名单，确保在数据中心之外部署的服务（例如堡垒机）与数据中心内的虚拟机能够正常通信。
- **自定义安全策略**

  支持将携带指定标签的虚拟机，或者包含在指定安全组内的虚拟机、IP 地址或 Pod 作为策略对象，为其配置放行策略或拒绝策略。其中，Pod 必须是已启用 EIC CNI 插件（即 SKS 1.3.2 及以下版本的 ECP CNI 插件）的 SKS 工作负载集群中的 Pod。
- **虚拟机隔离**

  虚拟机隔离策略的应用对象为单个虚拟机。支持在有虚拟机中毒或需要进行调试时，将其设置为完全隔离，禁止指定虚拟机接收或发出任意流量；或者设置为诊断隔离，为指定虚拟机设置出、入流量白名单。

## 概述 > 应用场景

# 应用场景

- **数据中心服务间的零信任安全**

  支持为业务资源提供精细化的策略，有效地消除了数据中心横向移动引发的内部威胁，大大减少了总攻击面，为数据中心东西向业务流量提供了坚实的安全保护。
- **动态隔离域**

  可以针对单个业务进行安全隔离和控制，通过微分段动态灵活地创建隔离域，满足动态业务隔离域的访问需求。
- **异常虚拟机隔离**

  网络攻击手段种类多样，潜伏期也越来越长，例如高级可持续威胁（APT）入侵与攻击，可以绕过基于代码的传统安全方案（如防病毒软件、防火墙和 IPS 等）长时间潜伏于系统中。

  Everoute 分布式防火墙可以针对异常虚拟机提供隔离策略。
- **业务部门分组安全**

  企业数据中心承载着不同业务部门的 IT 需求，而每个业务部门的权限与安全策略都不同，业务部门之间的访问策略与规则也各有区别。

  利用 Everoute，可以对不同部门管理和维护的应用虚拟机和 Pod 进行灵活分组，制定并执行不同部门组内和组间的访问安全策略。
- **虚拟机与容器之间的访问控制**

  在虚拟化平台上运行容器时，需要对容器与虚拟机之间的网络流量进行精细的访问控制，以保障平台的安全性。容器与虚拟机之间可能存在紧密的业务关联，需要根据具体的业务场景配置面向虚拟机和容器的统一访问策略，允许必要的通信，同时阻止未经授权的访问。

  Everoute 不仅支持在虚拟机之间、Pod 之间设置安全策略，还支持在 Pod 与虚拟机之间设置安全策略。
- **灵活定义内外部服务访问策略**

  在某些场景下，数据中心内部的某些应用可能需要访问外部的服务，但不应该访问数据中心内部某些服务。这些不应该被访问到的内部服务可能与外部服务使用相同的 IP。

  此时可以通过设置自定义安全策略的 IP 地址白名单来允许内部应用访问数中心外的服务，同时防止应用访问内部服务。

## 快速入门

# 快速入门

在部署 Everoute 服务时，您可以启用并配置分布式防火墙功能；如未在部署阶段启用，也可单独[启用分布式防火墙](/everoute/3.5.0/er_administration_guide_dfw/administration_guide_dfw_37)功能。

启用分布式防火墙功能之后，即可按照如下步骤说明开始使用分布式防火墙。

1. [关联集群](/everoute/3.5.0/er_administration_guide_dfw/administration_guide_dfw_38)：您需要先关联 SMTX OS（ELF）集群或 SMTX ELF 集群，以确定微分段管控的范围。
2. 为虚拟机、IP 或 Pod 分组，以便之后在设置虚拟机隔离策略或自定义安全策略时，可批量添加具有相同特征或要求的虚拟机、IP 或 Pod。

   - 为虚拟机分组：可参考[创建虚拟机安全组](/everoute/3.5.0/er_administration_guide_dfw/administration_guide_dfw_07)。
   - 为 IP 分组：可参考[创建 IP 安全组](/everoute/3.5.0/er_administration_guide_dfw/administration_guide_dfw_15)。
   - 为 Pod 分组：可参考[创建 Pod 安全组](/everoute/3.5.0/er_administration_guide_dfw/administration_guide_dfw_11)。
3. [创建服务](/everoute/3.5.0/er_administration_guide_dfw/administration_guide_dfw_09)：您可以事先创建服务，之后在设置自定义安全策略、全局安全策略或虚拟机隔离策略时，可选择该服务来统一设置协议和端口。
4. 按需设置分布式防火墙，包括如下三个功能：

   - [设置全局安全策略](/everoute/3.5.0/er_administration_guide_dfw/administration_guide_dfw_14)：

     - 如您在部署 Everoute 服务时勾选了分布式防火墙，则部署时已完成的全局安全策略的初始设置，您可以按需变更设置。
     - 如您在部署 Everoute 服务时未勾选分布式防火墙，后续单独启用了分布式防火墙功能，则需要按本操作完成全局安全策略设置。
   - [设置自定义安全策略](/everoute/3.5.0/er_administration_guide_dfw/administration_guide_dfw_113)：为携带指定标签的虚拟机，或者包含在指定安全组内的虚拟机、IP 或 Pod 设置放行策略或拒绝策略。
   - [设置虚拟机隔离策略](/everoute/3.5.0/er_administration_guide_dfw/administration_guide_dfw_16)：在有虚拟机中毒或需要进行调试时，禁止指定虚拟机接受或发出任意流量，或者为指定虚拟机设置出、入流量白名单。
5. 通过查看[策略命中的数据流个数](/everoute/3.5.0/er_administration_guide_dfw/administration_guide_dfw_106)以及[安全策略日志](/everoute/3.5.0/er_administration_guide_dfw/administration_guide_dfw_105)，了解分布式防火墙的运行效果。

## 启用分布式防火墙

# 启用分布式防火墙

**前提条件**

- 确认已开启相应的防火墙端口，请参考《Everoute 安装与升级指南》的[规划端口](/everoute/3.5.0/er_installation_guide/installation_guide_08)章节。
- Everoute Controller 节点数量为 `3 个`或 `5 个`。如不满足，请参考《Everoute 安装与升级指南》的[增加 Everoute Controller 数量](/everoute/3.5.0/er_installation_guide/installation_guide_31#%E5%A2%9E%E5%8A%A0-everoute-controller-%E6%95%B0%E9%87%8F)章节 。
- 如当前 Everoute 服务已启用网络安全导流之外的其他功能，则 Everoute Controller 配置类型至少为`中配`。如不满足，请参考《Everoute 安装与升级指南》的[提高 Everoute Controller 配置](/everoute/3.5.0/er_installation_guide/installation_guide_31#%E6%8F%90%E9%AB%98-everoute-controller-%E9%85%8D%E7%BD%AE)章节。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**设置**，选择要启用分布式防火墙的 Everoute 服务。
2. 在**服务设置**里单击**基本信息**。
3. 在**分布式防火墙**功能右侧单击**启用功能**。
4. 确认启用功能后，单击**启用**。

**后续操作**

1. 启用后，需为分布式防火墙[关联集群](/everoute/3.5.0/er_administration_guide_dfw/administration_guide_dfw_38)。
2. 关联集群和虚拟分布式交换机后，需参考以下步骤配置分布式防火墙全局安全策略。

   1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏的**分布式防火墙**中单击**安全策略**。
   2. 单击**全局安全策略**。
   3. 在列表中选择启用分布式防火墙的 Everoute 服务，单击其右侧的 **...**，选择**编辑**。
   4. 在弹出的**编辑全局安全策略**对话框中参考《Everoute 安装与升级指南》的[配置分布式防火墙](/everoute/3.5.0/er_installation_guide/installation_guide_15)章节填写配置信息。

## 查看 Everoute 服务概览

进入 CloudTower 的**网络与安全**界面，您可以在概览页查看当前所有已部署的 Everoute 服务的信息。

每个 Everoute 服务概览卡片包括如下内容：

- **服务名称**：卡片标题即为 Everoute 服务的名称。
- **版本**：展示当前 Everoute 服务的版本。当存在可升级的版本时，单击**可升级**，可进入**服务升级**页面，选择版本完成升级操作。
- **Controller 状态**：展示 Everoute Controller 的数量及当前状态。单击 **Controller 状态**可查看 Everoute Controller 信息。

  > **说明**：
  >
  > - Controller 状态包括正常运行、异常、正在部署、正在卸载。
  > - 绿色表示正常状态的 Controller 个数，红色表示异常状态（包括异常、正在部署、正在卸载）的 Controller 个数，黄色表示 vCPU、内存、存储容量使用率大于 80% 的 Controller 个数。
- **分布式防火墙**：若当前 Everoute 服务已开启分布式防火墙功能，将展示已关联的集群数量，以及 Everoute 代理异常的数量；否则将隐藏该项。

  - 当 Everoute 代理异常时，可显示异常代理数量，单击**查看详情**可查看具体异常的主机信息。
  - 若当前服务未关联集群，单击**关联集群**，可添加集群关联。
  - 单击**分布式防火墙**可进入**集群关联**页面。

## 查看分布式防火墙概览

当存在 Everoute 服务启用了分布式防火墙功能时，可查看分布式防火墙的信息。

分布式防火墙概览包含如下卡片：

- **自定义安全策略**：展示当前自定义安全策略总数、以及生效中和未生效的自定义安全策略数量。
- **隔离中的虚拟机**：展示当前隔离中虚拟机的总数、以及处于完全隔离和诊断隔离的虚拟机数量。
- **安全组**：展示当前已有的安全组总数，以及虚拟机安全组、IP 安全组和 Pod 安全组各自的数量。
- **24 小时内命中的数据流**：仅当关联了分布式防火墙的集群也同时关联了 1.3.0 或以上版本的可观测性服务时展示如下数据流信息：

  - 命中数据流总数
  - 已保护数据流个数
  - 默认允许数据流个数
  - 已拒绝数据流个数

## 管理关联集群 > 关联集群

# 关联集群

启用分布式防火墙后，需要关联 SMTX OS（ELF）集群或 SMTX ELF 集群及业务所属的虚拟分布式交换机，以确定分布式防火墙管控的范围。

**注意事项**

关联集群可能对集群网络产生影响，导致网络丢包，建议在业务空闲时或业务下线后谨慎操作。

**Everoute 服务关联规则**

- 一个 Everoute 服务可关联多个相同或不同 CPU 架构的集群，但一个集群仅能关联一个 Everoute 服务。
- 一个 Everoute 服务关联的所有集群的物理节点总数不超过 512 个。
- Everoute 服务只能关联可以添加虚拟机网络的虚拟分布式交换机。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**设置**，选择待关联集群的 Everoute 服务。
2. 在**服务设置**中单击**分布式防火墙**，在右侧页面单击**编辑集群关联**。
3. 在弹出的**编辑集群关联**窗口中，单击 **+ 添加集群**，选择待关联的**集群**和**虚拟分布式交换机**。
4. 单击**保存**。

   完成关联操作后，在**集群关联**列表中，可查看已关联的集群和虚拟分布式交换机。

## 管理关联集群 > 编辑或删除关联集群

# 编辑或删除关联集群

您可以按需更换关联的虚拟分布式交换机，或者解除 Everoute 服务和集群的关联。

**风险提示**

- 取消关联集群可能对集群网络产生影响，导致网络丢包，建议在业务空闲时或业务下线后谨慎操作。
- 取消关联集群后，Everoute 服务对该集群内虚拟机的网络流量控制将会失效。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**设置**，选择待编辑或删除关联集群的 Everoute 服务。
2. 在**服务设置**中单击**分布式防火墙**，在右侧页面单击**编辑集群关联**。
3. 在弹出的**编辑集群关联**对话框中：

   - 重新选择**虚拟分布式交换机**
   - 单击删除按钮（x）解除集群关联
4. 单击**保存**。

## 管理安全组

# 管理安全组

在设置自定义安全策略的策略对象、设置出/入流量白/黑名单、以及筛选要隔离的虚拟机等场景时，可能需要添加多个虚拟机或 Pod，为方便批量添加具有相同特征或要求的虚拟机或 Pod，可以通过安全组来实现批量添加。

Everoute 支持创建以下三种安全组：

- **虚拟机安全组**

  您可以将在某一 Everoute 服务关联的集群中，具有相同网络安全要求的虚拟机作为一个虚拟机安全组统一管理。您可以筛选关联了多个标签的虚拟机或者手动选择虚拟机添加至安全组，此分组方式相较于关联标签的方式更灵活。
- **IP 安全组**

  您可以将在同一 Everoute 服务中具有相同网络安全要求的 IP 地址、CIDR 或 IP 地址范围作为一个 IP 安全组统一管理。此分组方式相较于直接使用 IP 地址配置策略更高效，且支持动态调整成员 IP，确保相关安全策略自动生效。
- **Pod 安全组**

  对于某一 Everoute 服务关联的 SMTX OS 集群，若运行了 SKS 工作负载集群，您可以按需将这些集群上具有相同网络安全要求的 Pod 作为一个 Pod 安全组统一管理。您可以通过 SKS 工作负载集群、名字空间和 Kubernetes 标签属性筛选 Pod。

## 管理安全组 > 创建虚拟机安全组

# 创建虚拟机安全组

虚拟机安全组成员包括动态成员和静态成员。

- 动态成员：所有由标签组筛选出的虚拟机。单个标签组筛选出的虚拟机为关联了该标签组中所有标签的虚拟机。若要添加动态成员，可先参考《CloudTower 使用指南》的**管理标签**章节为虚拟机关联标签。
- 静态成员：所有手动选择的虚拟机。

**注意事项**

- 仅当虚拟机存在至少一个虚拟机网络所属的虚拟分布式交换机，关联了虚拟机安全组所属的 Everoute 服务时，该虚拟机才可以作为虚拟机安全组成员。
- 当许可方式为**按虚拟机数许可**时，仅支持为虚拟机安全组添加静态成员。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，单击**安全组**，再在界面右上角单击**创建安全组 > 虚拟机安全组**。
2. 在弹出的**创建虚拟机安全组**窗口中，填写安全组的名称及描述，并选择安全组所属的 Everoute 服务。
3. 添加动态成员。在**通过标签选择虚拟机**字段后，单击 **+ 添加标签组**，选择要添加的标签。

   - 若要在同一标签组内添加多个标签，可单击已添加标签后的 **+** 进行添加。
   - 若要添加多个标签组，可再次单击 **+ 添加标签组**进行添加。
4. 添加静态成员。在**选择虚拟机**字段后，输入虚拟机名称搜索，并选择要添加的一个或多个虚拟机。

   在**安全组**界面，可查看已创建的安全组。

## 管理安全组 > 创建 IP 安全组

# 创建 IP 安全组

IP 安全组允许将一组 IP 地址（包括单个 IP、CIDR 块或 IP 地址范围）作为成员进行统一管理，以便更灵活地定义策略对象或黑白名单规则，满足精细化的网络访问控制需求。

   - 支持 IPv4、IPv6，可输入一个或多个 IP 地址、CIDR 或 IP 地址范围，多条配置之间需使用英文逗号 `,` 分隔。
   - 单击**排序**，可按照 IPv4 地址升序、IPv6 地址升序排列。

   在**安全组**界面，可查看已创建的安全组。

## 管理安全组 > 创建 Pod 安全组

# 创建 Pod 安全组

Pod 安全组成员包括所有由筛选组筛选出的 Pod。单个筛选组可以筛选出在所选 SKS 工作负载集群和任一所选名字空间下，并且已附加所有所选 Kubernetes 标签的 Pod。

**前提条件**

- 分布式防火墙功能的许可方式为按 **CPU 插槽数**许可。

- 仅当 SKS 工作负载集群已启用 EIC CNI 插件，并且集群节点的 EIC 专用网卡所使用的虚拟机网络所属的虚拟分布式交换机已关联 Pod 安全组所属的 Everoute 服务时，该工作负载集群上的 Pod 才可以作为 Pod 安全组成员。
- 创建完 Pod 安全组后，如果需要通过 Everoute 安全策略控制 Pod 安全组的出、入流量，您还需要确保 Everoute Controller 虚拟机 IP 和 SKS 工作负载集群的 Control Plane 虚拟 IP 可以三层互通，若存在防火墙，则工作负载集群的 Control Plane 虚拟 IP 需对 Everoute Controller 虚拟机 IP 开放用于访问 kube-apiserver 的端口（TCP：6443）。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，单击**安全组**，再在界面右上角单击**创建安全组 > Pod 安全组**。
2. 在弹出的**创建 Pod 安全组**窗口中，填写安全组的名称及描述，并选择安全组所属的 Everoute 服务。
3. 添加安全组成员。选择要添加的 Pod 所在的 SKS 工作负载集群和名字空间，以及附加的 Kubernetes 标签。

   若需要添加多个筛选组，可在窗口左下角单击**添加筛选组**。
4. 单击**创建**，完成 Pod 安全组的创建。

   在**安全组**界面，可查看已创建的安全组。

## 设置安全策略

# 设置安全策略

Everoute 支持全局安全策略、自定义安全策略和虚拟机隔离三种分布式防火墙策略。三种策略都可以控制虚拟机的出、入流量，其中自定义安全策略还可以控制 IP 地址和 Pod 的出、入流量。

> **说明**：
>
> - 使用任一分布式防火墙策略控制虚拟机的出、入流量时，仅当虚拟机所在的集群、以及所有虚拟机网络所属的虚拟分布式交换机全部关联同一 Everoute 服务时，防火墙策略才可对该虚拟机生效。
> - Everoute 自动放通 IGMP 和 MLD 协议的报文，无需手动配置，您可按需创建自定义安全策略以放通组播业务流量。
> - 虚拟机隔离策略中的**完全隔离**策略会阻断所有 L2 和 L3 流量，其余分布式防火墙策略仅对 L3 流量（基于 IP 地址）进行放通或拦截。

- **全局安全策略**

  全局安全策略包括**默认拒绝通信**和**全局白名单**。**默认拒绝通信**不允许虚拟机接受和发出任何流量，**全局白名单**可全局设置虚拟机的出、入流量白名单。
- **自定义安全策略**

  您可以将携带指定标签的虚拟机，或者包含在指定安全组内的虚拟机、IP 地址或 Pod 作为策略对象，为其自定义**放行策略**和**拒绝策略**。
- **虚拟机隔离策略**

  可在有虚拟机中毒或需要进行调试时，禁止指定虚拟机接受或发出任意流量，或者为指定虚拟机设置出、入流量白名单。

## 设置安全策略 > 相关说明 > 优先级说明

# 优先级说明

以上策略应用于虚拟机和 Pod 的优先级，按照从高到低的顺序依次为：**虚拟机隔离策略** > **自定义安全策略（拒绝策略）** > **自定义安全策略（放行策略）** = **全局白名单** > **默认拒绝通信**。

> **说明**：
>
> - Pod 所在的虚拟机应用的分布式防火墙策略一般不影响该 Pod 的出、入流量。但当该 Pod 的出、入流量通过 NodePort Service 或 LoadBalancer Service 并由 Pod 所在节点的业务网卡 IP 转发时，这些流量将受虚拟机应用的分布式防火墙策略的控制。
> - SKS 工作负载集群的 Pod 与 Pod 之间、节点与 Pod 之间的流量默认放通，不受**默认拒绝通信**策略的影响，但这些流量能被**虚拟机隔离策略**、**自定义安全策略**控制。

您也可以在**安全策略**界面右上角单击 **...**，查看和下载策略生效优先级示意图。

## 设置安全策略 > 相关说明 > 对称策略说明

# 对称策略说明

由于网络通信是双向的，源端和目标端能否正常通信，取决于两端的安全规则。例如，为实现虚拟机 A 访问虚拟机 B，需要在 A 上设置允许向 B 发送流量的出规则，同时在 B 上设置允许从 A 接收流量的入规则，它们之间才能实现正常通信。这种情况下，虚拟机 A 的出流量规则和虚拟机 B 的入流量规则被称为一组对称策略。

为确保应用了分布式防火墙的虚拟机和 Pod 能够按照预期对流量进行处理，Everoute 在不同的安全策略下设置了不同的对称策略，具体说明如下：

- **自定义安全策略（放行策略）**

  在自定义安全策略（放行策略）中，若为策略对象设置了出、入流量白名单，Everoute 会为策略对象针对入流量白名单设置入流量规则，允许接受所有白名单中虚拟机和 Pod 的流量；同时针对出流量白名单设置出流量规则，允许向出流量白名单发送流量。同时，为保证出、入流量白名单生效，还需为白名单中虚拟机和 Pod 配置允许出/入站流量规则。根据不同的白名单设置方法，配置方式也有所不同，具体说明如下：

  - **使用虚拟机标签、虚拟机安全组和 Pod 安全组添加白名单**

    该方式下，由于虚拟机标签、虚拟机安全组和 Pod 安全组关联的全部为 Everoute 服务管理范围内的虚拟机和 Pod，Everoute 服务自动为白名单中虚拟机和 Pod 配置允许出/入站流量规则，确保白名单可生效。
  - **使用 IP 安全组或 IP 地址添加白名单**

    使用 IP 安全组或 IP 地址添加白名单时，由于添加的 IP 地址可能为 Everoute 服务管理范围内的虚拟机和 Pod，也可能为 Everoute 服务管理范围外的虚拟机和 Pod，上述不同类型的 IP 地址可能会有不同的访问需求。用户可根据实际场景，选择是否为白名单配置允许出/入站流量规则。

    - **启用配置**

      启用配置后，Everoute 会检查其服务范围内的每个虚拟机和每个 Pod 所使用的 IP 地址，如果这些 IP 地址在配置的 IP 地址白名单内，则自动为这些 IP 地址配置允许出/入站流量规则；若白名单中的 IP 地址不在 Everoute 服务管理范围内，则 Everoute 无需也不会为这些 IP 地址配置允许出/入站流量规则。

      该设置适用于不区分 IP 范围是否处于 Everoute 管理范围内，可允许所有的 IP 地址访问策略对象（或允许策略对象访问所有 IP 地址）的场景。例如，若希望策略对象的通信不受任何限制，将白名单配置为 `0.0.0.0/0,::/0`（即允许所有内部和外部地址都可以访问策略对象），则需启用该配置。
    - **禁用配置**

      禁用配置后，无论白名单内 IP 地址是否在 Everoute 服务管理范围内，都不会为其配置允许出/入站流量规则。

      该设置适用于白名单 IP 范围只需对 Everoute 管理范围外地址生效，只允许外部对应的 IP 访问策略对象的场景。例如，若仅希望策略对象连通互联网，允许外部所有 IP 地址访问，而集群内部 IP 需遵循其他特定的安全规则，则需禁用该配置。
- **自定义安全策略（拒绝策略）**

  在自定义安全策略（拒绝策略）中，只需要限制策略对象不接收或不发出相关流量，不会为入流量、出流量黑名单中的虚拟机和 Pod 配置流量规则。
- **全局白名单**

  由于全局白名单中设置的 IP 地址白名单，预期均为 Everoute 服务管理范围外的 IP 地址，因此默认不会配置允许出/入站流量规则。若全局白名单的 IP 地址中包含了 Everoute 服务管理范围内的虚拟机或 Pod 使用的 IP 地址，则全局白名单无法保证该虚拟机或 Pod 可与其他虚拟机正常通信，需要在自定义安全策略中增加更加精确的白名单来确保正常通信。
- **虚拟机隔离策略**

  完全隔离下，Everoute 将拒绝被隔离虚拟机的所有出、入流量，不会设置任何的允许出/入站流量规则。

  在诊断隔离情况下，由于用于诊断的白名单通常为集群内部虚拟机，Everoute 服务将为所有诊断隔离虚拟机的出、入白名单自动配置允许出/入站流量规则。

## 设置安全策略 > 相关说明 > 安全策略监控模式

# 安全策略监控模式

设置**全局白名单**、以及将虚拟机作为策略对象设置**自定义安全策略**时，为确保策略运行效果符合预期，建议您搭配使用可观测性 1.2.0 及以上版本的网络流量可视化功能，在启用安全策略和全局白名单前，打开流量可视化的**预览预期效果**开关，观察当前的数据流和策略生效后的预期数据流。若发现数据流的预期类型和期望不符，您可以再次编辑自定义安全策略和全局白名单，直到满足期望为止。通过这种方式，您可以在策略实际生效前直观了解其影响，提前发现和解决潜在的安全问题，且不会对正常运行的业务流量产生任何实际影响。

**操作步骤**

1. 创建全局白名单或自定义安全策略时，选择不启用策略。具体如下：

   - **全局白名单**：在部署 Everoute 服务阶段，在[配置分布式防火墙](/everoute/3.5.0/er_installation_guide/installation_guide_15)时配置全局白名单，并选择**不启用**全局白名单。
   - **自定义安全策略**：在[创建自定义安全策略](/everoute/3.5.0/er_administration_guide_dfw/administration_guide_dfw_113)时创建相关策略，选择**不启用**策略状态。
2. 正常运行一段时间相关业务。此时由于安全策略未生效，因此不会对业务流量产生实际影响。
3. 在全局白名单或自定义安全策略详情页单击`查看相关数据流`，将自动跳转至**网络流量可视化**中与该安全策略相关的视图。
4. 打开**预览预期效果**开关。

   - 在拓扑视图中，可根据数据流连线的颜色及样式查看数据流的当前类型及预期类型。
   - 在表格视图中，可在**数据流类型**列查看数据流的当前类型及预期类型。
   > **说明**：
   >
   > - 数据流类型包含已保护、已拒绝、默认允许、未保护四类，具体参见《可观测性平台用户指南》的**数据流类型及处理原因**章节。预期数据流类型为数据流所属的所有安全策略均生效后的类型。
   > - 如该策略在 24 小时内无相关数据流，则跳转后此视图默认为空，您可以手动调整时间范围，查看更长时间内的数据流情况。
5. 观察预期数据流类型是否与期望相符。例如业务相关的正常流量，预期类型应为**已保护**，其他不希望放行的风险数据流，预期类型应为**已拒绝**。
6. 如观察到的效果与期望类型不符，可通过编辑自定义安全策略和全局白名单来调整策略，继续运行一段时间的相关业务，并重复上述步骤，直至所有流量的预期类型都符合期望。

   > **注意**：
   >
   > 在调整安全策略后，需按照时间段筛选出策略调整后新生成的数据流，并观察其预期类型是否符合期望。
7. 所有流量的预期类型均确认无误后，通过编辑操作启用自定义安全策略或全局白名单。

## 设置安全策略 > 设置全局安全策略

# 设置全局安全策略

全局安全策略包括**默认拒绝通信**和**全局白名单**。用户在部署 Everoute 服务时即已对**默认拒绝通信**和**全局白名单**做了初始设置，并默认不启用**日志**。若需要变更该设置，或者手动导入、导出全局安全策略，可参考如下介绍。

## 编辑全局安全策略

1. 进入 CloudTower 的**网络与安全**界面，单击**安全策略**，在**全局安全策略**选项卡中找到要变更全局策略的 Everoute 服务。

## 设置安全策略 > 设置自定义安全策略 > 自定义安全策略类型说明

# 自定义安全策略类型说明

自定义安全策略分为**放行策略**和**拒绝策略**两种类型。在分布式防火墙中，可以通过设置放行策略和拒绝策略，有选择性地允许或拒绝特定网络流量，为系统和业务提供安全保护。自定义安全策略的策略对象可以是虚拟机、IP 和 Pod。

## 放行策略

放行策略是一种只允许信任流量通过防火墙的策略。为策略对象配置了放行策略后，除了策略中明确指定的流量，其他流量都不允许通过。这种严格的控制机制确保了只有经过授权流量才被允许进入系统，从而极大地提高了系统的安全性。

放行策略典型应用场景如下：

- **满足高安全性要求**

  在涉及敏感数据和关键业务的场景中，如金融机构、政府部门和医疗系统，放行策略可以确保只有授权的访问可以通过，防止未经许可的访问和潜在的安全威胁。

  例如，银行的核心交易系统可能只允许来自特定应用服务器的访问，其他流量一律被拒绝。
- **控制内部访问权限**

  在企业内部，放行策略可以严格管理不同部门或应用之间的访问权限。例如，销售部门的系统只能访问其业务相关的应用，而无法访问财务系统。这种策略可以有效地防止内部数据泄露和未经授权的访问。

## 拒绝策略

拒绝策略是一种阻止特定流量的策略。与放行策略不同的是，拒绝策略只阻止在策略中明确指定的流量，其余流量能否通行则取决于系统中配置的其他安全策略。拒绝策略提供了一种简单、灵活的管理机制，可以高效控制和响应已知的安全威胁，在保护关键资源的同时保持网络的开放性。

拒绝策略典型应用场景如下：

- **限制已知的风险流量**

  通过拒绝策略，可以限制外部通过风险端口（例如 22、135、136、445 等端口）访问生产环境中的关键业务，以降低策略对象受到外部攻击的风险。
- **隔离生产与测试环境**

  拒绝策略可以将测试环境与生产环境进行严格隔离，确保测试过程中不会意外影响到生产系统的稳定性和安全性。
- **快速屏蔽新的安全威胁**

  通过拒绝策略，可以迅速屏蔽已知的恶意 IP 地址或端口，以防止潜在的网络攻击。例如，当检测到某个 IP 地址频繁尝试登录系统且被识别为恶意行为时，可以为其设置拒绝策略，立即阻止其访问。

## 放行策略和拒绝策略的配合使用

放行策略和拒绝策略在网络安全管理中各有其独特的优势。通过结合使用这些策略，能够在不同的安全需求和业务场景下，为企业提供更全面和灵活的网络流量控制，并降低策略管理的复杂度。

以下是几种常见的配合使用场景：

- **核心业务的保护**

  对于关键业务，放行策略可以严格控制允许的访问来源和流量类型，同时使用拒绝策略屏蔽已知的高风险端口和未经授权的外部流量。
- **灵活应对网络变化**

  在动态变化的网络中，放行策略可以维护日常业务的正常持续运行，拒绝策略可以快速响应和阻止新发现的安全威胁。
- **根据需求灵活调整策略**

  在执行特定任务时，需要临时允许或阻止某些特定流量，可以通过快速配置放行策略或者拒绝策略以适应业务临时需求，且不会对系统的全局安全性产生负面影响。

## 安全策略配置建议

部署 Everoute 服务后，如各类应用间的访问关系尚不明确，建议按照如下流程进行安全策略配置：

1. 关闭全局安全策略中的**默认拒绝通信**，保证未配置自定义策略的指定流量均可通行。如有明确的风险端口和环境隔离需求，可为策略对象配置拒绝策略。
2. 对业务的访问关系进行梳理，并根据梳理结果逐步为相关业务添加对应的放行策略，拒绝所有未授权流量。
3. 当系统中所有业务均已配置了放行策略后，可启用全局安全策略中的**默认拒绝通信**，这样其他未知流量将被拒绝，从而保护环境中的策略对象免受外界攻击。

## 设置安全策略 > 设置自定义安全策略 > 创建自定义安全策略

# 创建自定义安全策略

**注意事项**

- 当虚拟机的虚拟机网络所属的虚拟分布式交换机全部未关联该策略所属的 Everoute 服务时，该虚拟机无法作为策略对象。
- 如果虚拟机包含多个虚拟网卡，其中仅部分网卡所属的虚拟分布式交换机已关联 Everoute 服务，则安全策略仅对这部分网卡中的流量生效。
- 仅当虚拟机存在至少一个虚拟机网络所属的虚拟分布式交换机，关联了该策略所属的 Everoute 服务时，该虚拟机才可以加入该策略的白名单。
- 控制 Pod 的出、入流量时，Everoute 的自定义安全策略与 Kubernetes 原生的 NetworkPolicy 不可混用，建议仅使用 Everoute 的自定义安全策略。
- 控制组播流量时，需将包含组播 IP 的 IP 安全组作为策略对象。
- 当许可方式为**按虚拟机数许可**时，不支持通过虚拟机标签、IP 安全组、Pod 安全组添加策略对象，不支持通过虚拟机标签、Pod 安全组添加放行策略的白名单和拒绝策略的黑名单。
- 当未部署符合版本要求的 SKS 服务时，不支持通过 Pod 安全组添加策略对象、放行策略的白名单和拒绝策略的黑名单。版本要求可参考[《Everoute 发布说明》](/everoute/3.5.0/er_release_notes/release_notes_02)进行确认。
- 由于 EIC CNI 暂不支持 IPv6，运行着 EIC CNI 的工作负载集群中的 Pod 无 IPv6 地址，因此策略中 IP 地址和 IP 安全组里如有 IPv6 对 Pod 无效。

## 创建放行策略

1. 进入 CloudTower 的**网络与安全**界面，单击**安全策略**，在界面右上角单击**添加安全策略** > **创建自定义安全策略**。
2. 在弹出的**创建自定义安全策略**窗口中，填写策略的名称，并选择所属 Everoute 服务。
3. 安全策略类型选择**放行策略**。
4. 选择是否启用**日志**，完成后单击**下一步**。
5. 单击 **+ 添加策略对象**，添加要应用该策略的策略对象，完成后单击**添加**。

   - **通过虚拟机安全组添加**：搜索并选择要应用该策略的虚拟机安全组。或者在下拉框底部单击**新建安全组**，参考[创建虚拟机安全组](/everoute/3.5.0/er_administration_guide_dfw/administration_guide_dfw_07)创建要应用该策略的虚拟机安全组。
   - **通过 IP 安全组添加**：搜索并选择要应用该策略的 IP 安全组。或者在下拉框底部单击**新建 IP 安全组**，参考[创建 IP 安全组](/everoute/3.5.0/er_administration_guide_dfw/administration_guide_dfw_15)创建要应用该策略的 IP 安全组。
   - **通过 Pod 安全组添加**：搜索并选择要应用该策略的 Pod 安全组。或者在下拉框底部单击**新建 Pod 安全组**，参考[创建 Pod 安全组](/everoute/3.5.0/er_administration_guide_dfw/administration_guide_dfw_11)创建要应用该策略的 Pod 安全组。
   - **通过虚拟机标签添加**：通过标签名或标签值搜索并添加一个或多个虚拟机标签，筛选出要应用该策略的虚拟机。
   - **策略对象互相通信**（仅虚拟机安全组、虚拟机标签支持配置）：选择应用该策略的虚拟机之间是否可以互相通信。

   单击已添加的策略对象右侧 **...**，可以编辑或删除该策略对象。

   > **说明**：
   >
   > - 通过虚拟机安全组或虚拟机标签添加策略对象后，可以通过单击策略对象内包含的虚拟机，进入[安全策略查询界面](/everoute/3.5.0/er_administration_guide_dfw/administration_guide_dfw_107#%E5%AE%89%E5%85%A8%E7%AD%96%E7%95%A5%E6%9F%A5%E8%AF%A2%E7%95%8C%E9%9D%A2)，查看该虚拟机当前已配置的安全策略。
   > - 通过虚拟机标签添加策略对象时，可选择一个或多个标签。当选择多个标签时，虚拟机的筛选逻辑为**逻辑与**，即筛选出的虚拟机为同时关联了所有标签的虚拟机。
   > - 支持同时通过虚拟机标签、虚拟机安全组、IP 安全组和 Pod 安全组的方式添加多组策略对象。多组策略对象之间为**逻辑或**的关系，即包含在任一组策略对象中的虚拟机、IP 或 Pod 都作为此放行策略的策略对象，并应用相关的安全规则。
6. 单击**入流量**字段后的下拉框，设置策略对象的入流量。

   - **全部允许**：允许策略对象接收所有流量。
   - **仅允许白名单**：指定可接收的流量，若白名单留空则禁止全部入流量。

     单击 **+ 添加**，并选择添加白名单方式：

     - **按虚拟机安全组**：选择或创建可访问策略对象的虚拟机安全组，并可通过输入协议、端口号，或选择特定服务指定允许的协议和端口，若无需限制协议可选择**任意协议**。
     - **按 IP 安全组**：选择或创建可访问策略对象的 IP 安全组，并可通过输入协议、端口号，或选择特定服务指定允许的协议和端口，若无需限制协议可选择**任意协议**。可选择是否**为源端虚拟机配置允许出站流量规则**，系统默认勾选，如无需配置可取消勾选。
     - **按 Pod 安全组**：选择或创建可访问策略对象的 Pod 安全组，并可通过输入协议、端口号，或选择特定服务指定允许的协议和端口，若无需限制协议可选择**任意协议**。
     - **按虚拟机标签**：通过标签筛选出可访问策略对象的虚拟机，并可通过输入协议、端口号，或选择特定服务指定允许的协议和端口，若无需限制协议可选择**任意协议**。
     - **按 IP 地址**：支持 IPv4、IPv6，可输入一个或多个 IP 地址、IP 地址范围或 CIDR 块指定白名单，并可通过输入协议、端口号，或选择特定服务指定允许的协议和端口。您还可以单击输入框右侧 **...**，手动从白名单中排除特定 IP 地址、IP 地址范围或 CIDR 块，若无需限制协议可选择**任意协议**。可选择是否**为源端虚拟机配置允许出站流量规则**，系统默认勾选，如无需配置可取消勾选。
     > **说明**：
     >
     > - 若希望 IP 地址白名单仅对外部 IP 地址生效，不允许内部虚拟机或 Pod 有访问权限，请勿勾选**为源端虚拟机配置允许出站流量规则**。例如，若只希望集群外部互联网中的 IP 可以访问策略对象，不希望放行集群内部所有的虚拟机或 Pod 流量，可以将白名单设置为 `0.0.0.0/0,::/0`，并取消勾选**为源端虚拟机配置允许出站流量规则**。
     > - 通过虚拟机标签添加白名单时，可选择一个或多个标签。当选择多个标签时，虚拟机的筛选逻辑为**逻辑与**，即筛选出的虚拟机为同时关联了所有标签的虚拟机。
     > - 支持同时通过虚拟机标签、虚拟机安全组、IP 安全组、Pod 安全组和 IP 地址的方式添加多组白名单。多组白名单之间为**逻辑或**的关系，即包含在任一组白名单中的虚拟机、Pod 或 IP 地址都可应用相关的安全规则，与策略对象通信。
     > - 通过虚拟机标签或虚拟机安全组添加白名单时，可以通过单击白名单中包含的虚拟机，进入[安全策略查询界面](/everoute/3.5.0/er_administration_guide_dfw/administration_guide_dfw_107#%E5%AE%89%E5%85%A8%E7%AD%96%E7%95%A5%E6%9F%A5%E8%AF%A2%E7%95%8C%E9%9D%A2)，查看该虚拟机当前已配置的安全策略。
7. 参考上一步骤设置策略对象的出流量白名单，完成后单击**下一步**。
8. 设置该放行策略创建后是否立即生效。若不启用**状态**，后续也可以编辑策略状态，使策略生效。
9. 单击**创建**。

   > - 支持同时通过虚拟机标签、虚拟机安全组、IP 安全组和 Pod 安全组的方式添加多组策略对象。多组策略对象之间为**逻辑或**的关系，即包含在任一组策略对象中的虚拟机、IP 或 Pod 都作为此拒绝策略的策略对象，并应用相关的安全规则。
6. 单击**入流量黑名单**字段后的下拉框，设置策略对象的入流量。单击 **+ 添加**，并选择添加入流量黑名单方式：

   - **按虚拟机安全组**：选择或创建禁止访问策略对象的虚拟机安全组，并可通过输入协议、端口号，或选择特定服务指定拒绝的协议和端口，若需限制所有协议可选择**任意协议**。
   - **按 IP 安全组**：选择或创建禁止访问策略对象的 IP 安全组，并可通过输入协议、端口号，或选择特定服务指定拒绝的协议和端口，若需限制所有协议可选择**任意协议**。
   - **按 Pod 安全组**：选择或创建禁止访问策略对象的 Pod 安全组，并可通过输入协议、端口号，或选择特定服务指定拒绝的协议和端口，若需限制所有协议可选择**任意协议**。
   - **按虚拟机标签**：通过标签筛选出禁止访问策略对象的虚拟机，并可通过输入协议、端口号，或选择特定服务指定拒绝的协议和端口，若需限制所有协议可选择**任意协议**。
   - **按 IP 地址**：支持 IPv4、IPv6，可输入一个或多个 IP 地址、IP 地址范围或 CIDR 块指定黑名单，并可通过输入协议、端口号，或选择特定服务指定拒绝的协议和端口。您还可以单击输入框右侧 **...**，手动从黑名单中排除特定 IP 地址、IP 地址范围或 CIDR 块，若需限制所有协议可选择**任意协议**。
   > **说明**：
   >
   > - 通过虚拟机标签添加黑名单时，可选择一个或多个标签。当选择多个标签时，虚拟机的筛选逻辑为**逻辑与**，即筛选出的虚拟机为同时关联了所有标签的虚拟机。
   > - 支持同时通过虚拟机标签、虚拟机安全组、IP 安全组、Pod 安全组和 IP 地址的方式添加多组黑名单。多组黑名单之间为**逻辑或**的关系，即包含在任一组黑名单中的虚拟机、Pod 或 IP 地址都可应用相关的安全规则，限制与策略对象通信。
   > - 通过虚拟机标签或虚拟机安全组添加黑名单时，可以通过单击黑名单中包含的虚拟机，进入[安全策略查询界面](/everoute/3.5.0/er_administration_guide_dfw/administration_guide_dfw_107#%E5%AE%89%E5%85%A8%E7%AD%96%E7%95%A5%E6%9F%A5%E8%AF%A2%E7%95%8C%E9%9D%A2)，查看该虚拟机当前已配置的安全策略。
7. 参考上一步骤设置策略对象的出流量黑名单，完成后单击**下一步**。
8. 设置该拒绝策略创建后是否立即生效。若不启用**状态**，后续也可以编辑策略状态，使策略生效。
9. 单击**创建**。

## 设置安全策略 > 设置自定义安全策略 > 导出、导入自定义安全策略

# 导出、导入自定义安全策略

若要为不同的 Everoute 服务添加相同的自定义安全策略，可以先将一个 Everoute 服务中已有的策略导出，再将导出的策略文件导入另一个 Everoute 服务。另外，为避免误删重要的自定义安全策略，也可以将这些策略导出做备份，在需要恢复策略的时候再做导入操作。

## 导出自定义安全策略

**注意事项**

从本版本的 Everoute 服务中导出的自定义安全策略不支持导入低于本版本的 Everoute 服务。

**操作步骤**

进入 CloudTower 的**网络与安全**界面，单击**安全策略**，在**自定义安全策略**选项卡下，找到需要导出的安全策略，然后选择进行以下操作：

- **导出单个策略**

  在该策略最右侧单击 **...**，选择**导出**；或单击该策略，进入策略详情界面后，在界面右上角单击**导出**。
- **导出多个策略**

  勾选要导出的策略，在界面右上角单击**导出**。

所选安全策略将自动导出为 `.json` 格式的策略文件。

## 导入自定义安全策略

导入自定义安全策略是添加安全策略的方式之一，导入的策略不会影响目标 Everoute 服务已有的策略。

**导入规则**

- 若目标 Everoute 服务中有安全组的名称、成员类型与导入策略中的安全组相同，则策略导入后将直接使用 Everoute 服务中对应的安全组。
- 若目标 Everoute 服务中有安全组的名称与导入策略中的安全组相同，但成员类型不同，则策略中的这些安全组及其配置规则将无法被导入。
- 若目标 Everoute 服务中不存在与导入策略中的安全组名称相同的安全组，则策略导入后这些安全组将被自动创建，并被导入的策略使用。
  - 对于新创建的虚拟机安全组，静态成员将不被保留。
  - 对于新创建的 Pod 安全组，Everoute 服务不会校验相关的 SKS 部署情况、SKS 工作负载集群和 SKS 名字空间。
- 若当前 CloudTower 中已包含导入策略中的虚拟机标签、服务，则策略导入后将直接使用 CloudTower 中对应的虚拟机标签及服务。
- 当前 CloudTower 中不存在导入策略中的虚拟机标签、服务，则策略导入后这些虚拟机标签及服务将被自动创建，并被导入的策略使用。
- 当许可方式为按**虚拟机数**许可时，若导入策略中的策略对象包含虚拟机安全组的动态成员、IP 安全组、Pod 安全组或标签，或者策略的出入流量黑白名单中包含虚拟机安全组的动态成员、Pod 安全组或标签，则策略无法被导入。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，选择**安全策略**，在界面右上角单击**添加安全策略**，选择**导入自定义安全策略**。
2. 在弹出的**导入自定义安全策略**窗口中，选择要导入安全策略的 Everoute 服务。
3. 上传 `.json` 格式的策略文件。
4. 文件上传后，选择此次需要导入的策略，完成后单击**下一步**。
5. 设置策略导入后是否立即生效。

   - 若不启用**状态**，则该安全策略导入后将无法生效，但后续也可以编辑策略状态，使策略生效。
   - 当许可方式为**按虚拟机数许可**时，在导入时无法启用**状态**，策略默认不生效。
6. 为导入的策略填写新名称，可以选择为新名称设置统一的后缀，完成后单击**下一步**。
7. 确认导入后的资源变更，完成后单击**导入**。

## 设置安全策略 > 设置虚拟机隔离策略

# 设置虚拟机隔离策略

虚拟机的隔离策略分为**完全隔离**和**诊断隔离**。

- **完全隔离**：该隔离策略下，虚拟机将拒绝所有的出、入流量（包括 L2 与 L3 报文）。此隔离模式适用于虚拟机中毒场景，可避免病毒在内部环境中传播，减少对业务的影响。
- **诊断隔离**：该隔离策略下，虚拟机仅可以和白名单通信。可以分别设置虚拟机的出流量白名单和入流量白名单。

**注意事项**

- 仅当虚拟机的所有虚拟机网络所属的虚拟分布式交换机，都关联了 Everoute 服务时，该虚拟机才可以作为隔离对象。
- 仅当虚拟机存在至少一个虚拟机网络所属的虚拟分布式交换机，关联了隔离对象对应的 Everoute 服务时，该虚拟机才可以加入隔离白名单。
- 当许可方式为**按虚拟机数许可**时，不支持通过虚拟机标签、Pod 安全组添加诊断隔离策略的出、入流量白名单。
- 当未部署符合版本要求的 SKS 服务时，不支持通过 Pod 安全组添加诊断隔离策略的出、入流量白名单。版本要求可参考[《Everoute 发布说明》](/everoute/3.5.0/er_release_notes/release_notes_02)进行确认。

## 隔离单个虚拟机

1. 在集群的虚拟机列表中，选择需要隔离的虚拟机，在弹出的虚拟机详情面板中，单击**网络安全**选项卡，再在界面右上角单击**隔离**。
2. 在弹出的**隔离虚拟机**窗口中，为该虚拟机设置隔离策略。若选择**诊断隔离**，可为隔离对象设置出、入流量白名单。
3. 完成设置后，单击**隔离**。

## 批量隔离虚拟机

1. 进入 CloudTower 的**网络与安全**界面，单击**隔离中的虚拟机**，在界面右上角单击**隔离虚拟机**。
2. 在弹出的**隔离虚拟机**窗口中，选择隔离对象对应的 Everoute 服务，并选择是否为隔离对象启用**日志**。
3. 选择按**虚拟机名称**、**虚拟机安全组**、或**关联标签**筛选虚拟机，并从筛选结果中选择一个或多个虚拟机作为隔离对象。完成后单击**下一步**。

   > **注意**：
   >
   > 若所选虚拟机已被隔离，则此次配置的隔离策略将会覆盖已有的隔离策略。
4. 为隔离的虚拟机设置隔离策略。若选择**诊断隔离**，可为隔离对象设置出、入流量白名单。
5. 完成设置后，单击**隔离**。

1. 进入**网络与安全**界面，单击**隔离中的虚拟机**，在右侧列表中找到要编辑隔离策略的虚拟机。
2. 单击该虚拟机，在弹出的虚拟机详情面板中的**网络安全**选项卡下，单击**编辑隔离策略**。

1. 进入**网络与安全**界面，单击**隔离中的虚拟机**，在右侧列表中找到要编辑日志启用状态的虚拟机。
2. 单击该虚拟机，在弹出的虚拟机详情面板中单击 **···**，选择**隔离**>**编辑日志启用状态**进行编辑。

   如有多个虚拟机需要编辑，也可以在列表中勾选多个虚拟机，单击**编辑日志启用状态**，批量修改虚拟机的日志启用状态。

## 停止隔离虚拟机

1. 进入**网络与安全**界面，单击**隔离中的虚拟机**，在右侧列表中找到要停止隔离的虚拟机。

## 设置安全策略 > 查看虚拟机的安全策略

# 查看虚拟机的安全策略

Everoute 支持通过查看虚拟机已配置的安全策略，来帮助运维管理人员全面掌握虚拟机的所有相关安全策略和具体规则。这一功能不仅可以帮助运维管理人员更有效地配置新的安全策略，还能在虚拟机出现网络问题时，快速查看所有可能影响该虚拟机的安全策略，方便进行问题排查和策略调整。

## 查看方式

根据不同的功能使用场景，您可以在多个入口进入虚拟机策略查询页面;

- **入口 1**

  当您在创建、编辑或查看安全策略时，可以通过单击策略对象、白名单和黑名单中包含的某个虚拟机的名称，进入[安全策略查询界面](#%E5%AE%89%E5%85%A8%E7%AD%96%E7%95%A5%E6%9F%A5%E8%AF%A2%E7%95%8C%E9%9D%A2)，来查看该虚拟机已配置的安全策略。
- **入口 2**

  您可以进入**网络与安全** > **分布式防火墙** > **安全策略**页面，单击页面右上角的**策略查询**，进入[安全策略查询界面](#%E5%AE%89%E5%85%A8%E7%AD%96%E7%95%A5%E6%9F%A5%E8%AF%A2%E7%95%8C%E9%9D%A2)，在搜索框中输入虚拟机的名称或 IP 地址，单击**查询**，来查看该虚拟机已配置的安全策略。
- **入口 3**

  您可以在 CloudTower 的**虚拟机**列表中，单击虚拟机的名称进入虚拟机详情页面，在**网络安全**页签内，查看该虚拟机已关联的所有安全策略。单击策略名称，即可跳转查看该策略的详细信息。
- **入口 4**

  若虚拟机所在集群已关联了可观测性服务，且已启用了网络流量可视化功能，在通过拓扑视图查看虚拟机相关流量时，可单击虚拟机图例，在弹出的数据流侧边栏中，单击**查看策略**，即可查看该虚拟机已配置的安全策略。

## 安全策略查询界面

### 搜索相关虚拟机

在安全策略查询界面，您可以通过在搜索框中输入虚拟机的名称、关键字或 IP 地址（仅支持已安装了 VMTools 的虚拟机），选择目标虚拟机，单击**查询**，搜索虚拟机已配置的安全策略。

**前提条件**

- 虚拟机所在集群已关联 Everoute 分布式防火墙。
- 虚拟机任意虚拟网卡所在 VDS 已关联了 Everoute 分布式防火墙。

**注意事项**

如需通过 IPv6 地址搜索虚拟机，请确保已安装 3.2.0 及以上版本的 VMTools。

### 查看虚拟机相关安全策略

您可以通过单击**全部**、**作为源端**、**作为目标端**，来查询虚拟机作为源端、目标端和所有的安全策略信息。

- **虚拟机隔离**

  展示该虚拟机是否处于隔离状态，或是否作为其他诊断隔离虚拟机的出、入流量白名单。

  - 若虚拟机处于完全隔离或诊断隔离状态，其他安全策略将不会生效；
  - 若虚拟机作为其他诊断隔离虚拟机的出、入流量白名单，其他策略将继续生效。
- **黑名单**

  黑名单中包含了**相关自定义安全策略**和**影响全局流量的自定义安全策略**。

  - **相关自定义安全策略**

    相关自定义安全策略是指当虚拟机符合以下任一条件时，以标签或安全组的方式，展示与该虚拟机相关的拒绝策略：

    - 虚拟机作为某一拒绝策略的策略对象；
    - 虚拟机被包含在某一拒绝策略的入流量黑名单中；
    - 虚拟机被包含在某一拒绝策略的出流量黑名单中。
  - **影响全局流量的自定义安全策略**

    影响全局的自定义安全策略是指影响当前 Everoute 服务中所有虚拟机的拒绝策略。

    当入流量黑名单或出流量黑名单的范围为 `0.0.0.0/0,::/0`，且没有被排除的 IP 地址时，可查看相关的流量规则。
- **白名单**

  白名单中包含了**相关自定义安全策略**、**影响全局流量的自定义安全策略**和**全局白名单**。

  - **相关自定义安全策略**

    相关自定义安全策略是指当虚拟机符合以下任一条件时，以标签或安全组的方式，展示与该虚拟机相关的放行策略：

    - 虚拟机作为某一放行策略的策略对象；
    - 虚拟机被包含在某一放行策略的入流量白名单中；
    - 虚拟机被包含在某一放行策略的出流量白名单中。
  - **影响全局流量的自定义安全策略**

    影响全局的自定义安全策略是指影响当前 Everoute 服务中所有虚拟机的放行策略。

    当存在符合以下任一条件的放行策略时，可查看相关的流量规则：

    - 入流量白名单或出流量白名单的范围为 `0.0.0.0/0,::/0`，且没有被排除的 IP 地址。
    - 入流量白名单或出流量白名单为`全允许`。
  - **全局白名单**

    若虚拟机所属的 Everoute 服务包含任一全局白名单，可查看其入流量和出流量白名单。
- **缺省策略**

  若当前虚拟机作为**任一自定义放行策略的策略对象**，此处将展示该策略对象缺省规则（即`默认拒绝`）以及该虚拟机所属的放行策略。

  若当前虚拟机**不属于任一自定义放行策略的策略对象**，此处将展示全局默认拒绝通信的处理规则。若全局启用了默认拒绝通信，则展示`默认拒绝`；若未启用默认拒绝通信，则展示`默认允许`。

## 安全策略命中计数

# 安全策略命中计数

Everoute 分布式防火墙**默认启用**安全策略命中计数功能，将采集已关联至 1.3.0 或以上版本可观测性服务的集群中的流量信息。

安全策略命中计数功能将记录最近 24 小时之内，每条**自定义安全策略**、每个**隔离中的虚拟机**以及**全局安全策略**的命中计数，并展示命中的数据流个数、已保护的数据流个数、已拒绝的数据流个数和默认允许的数据流个数。

您可以通过查看安全策略拦截的数据流个数，识别虚拟机是否正在受到攻击，对数据中心整体的网络安全情况进行排查；结合[安全策略日志](/everoute/3.5.0/er_administration_guide_dfw/administration_guide_dfw_105)，判断是否存在预期内的流量未被添加至白名单，及时调整安全策略。

> **说明**：
>
> - 如 Everoute 分布式防火墙已关联的集群中，部分集群未关联可观测性服务，或所有集群未关联至同一个可观测性服务，则无法对此类集群中的虚拟机提供安全策略命中计数功能。
> - 若隔离中的虚拟机采用**完全隔离**策略，L2 流量也将被阻断，但安全策略命中计数不会包含 L2 数据流计数。

## 安全策略命中计数 > 启用安全策略命中统计通知

# 启用安全策略命中统计通知

启用安全策略命中统计通知后，将每天在特定时间为特定地址发送最近 24 小时内所有安全策略命中统计的邮件通知，告知当前的保护和拒绝的数据流情况，方便您及时了解业务变更或系统收到的攻击行为。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**设置**，选择待启用安全策略命中统计通知的 Everoute 服务。
2. 在**服务设置**中单击**分布式防火墙**，在右侧页面单击**启用通知**。
3. 在弹出的**安全策略命中统计通知**对话框中填写如下配置信息：

   - **SMTP 服务器**：选择 CloudTower 中已有的一个 SMTP 服务器。如当前 CloudTower 中没有 SMTP 服务器，请参考对应版本的《CloudTower 使用指南》的**配置 SMTP 服务器**章节进行配置。
   - **发件地址**：设置发出邮件的地址。
   - **收件地址**：设置接受邮件的地址。如需设置多个收件地址，请用英文逗号 `,` 隔开。
   - **通知时间**：设置发出邮件的时间。
   - **通知语言**：选择邮件的语言为简体中文或英文。
4. 单击**保存**。

   启用通知后，单击**编辑**可以按需修改配置信息；如果不需要邮件通知，可以单击**停用通知**，但配置将被清空，如后续再次启用邮件通知，需要重新填写配置信息。

# 更新许可

分布式防火墙功能单独对应一个许可码，在如下场景中需更新许可码：

- 许可额度达到上限，希望提升许可额度。
- 转换许可方式。若当前 Everoute 服务中存在 Pod 安全组，则不支持转换许可方式。
- 升级许可类型。例如，某一功能的**试用许可**需升级为**订阅许可**或**永久许可**。
- 延长订阅时长。

更新许可时，请将所有当前软件许可信息提供给 SmartX 客户经理，在完成商务流程后，客户经理将提供新的许可码。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，选择**设置**，单击**软件许可**，将许可码粘贴在**新许可码**处。
2. 单击输入框外部的空白区域，载入新许可信息。
3. 确认新许可信息无误后，单击**更新**，完成许可更新。

## 卸载分布式防火墙

# 卸载分布式防火墙

当您不再需要某个 Everoute 服务中的分布式防火墙功能，想要卸载该功能时，请在知晓卸载功能可能导致的风险并确认满足卸载的要求后，参照操作步骤进行卸载。

**风险提示**

卸载分布式防火墙后，将造成如下影响：

- 已有的全局安全策略、自定义安全策略、虚拟机隔离及安全组将被删除。
- 如后续再次开启分布式防火墙功能时，需要重新初始化所有功能。

**卸载要求**

Everoute 服务的分布式防火墙功能中没有关联的集群。

**操作步骤**

1. 登录 CloudTower，进入**网络与安全**界面，选择**设置**，选择待卸载功能的 Everoute 服务。
2. 在**服务设置**中单击**基本信息**，进入基本信息界面。
3. 在待卸载功能名称右侧单击**卸载功能**。
4. 确认卸载功能后，单击**卸载**。
