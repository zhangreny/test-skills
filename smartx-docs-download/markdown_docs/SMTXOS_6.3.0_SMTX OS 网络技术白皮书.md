## 基本概念

# 基本概念

在了解 SMTX OS 网络架构之前，需先理解以下基本的网络概念。

---

## 基本概念 > 物理网络

# 物理网络

物理网络是指不同的物理网络设备，如主机、交换机、路由器等通过网线连接起来组成的网络。SMTX OS 的物理网络中的物理设备主要包含物理网卡、物理网口和物理交换机。

## 物理网卡

物理网卡是一种硬件设备，简称 NIC（Network Interface Card），用于将物理主机和其他主机或网络设备进行连接。物理网卡可以接收和发送数据，从而让主机可以和其他设备进行网络通信。

根据支持的速率划分，物理网卡通常可以划分为 1 Gbps（千兆）、10 Gbps（万兆）、25 Gbps、100 Gbps 网卡等，不同速率体现了网卡的能力等级。

主流物理网卡厂商有 Intel、Mellanox、Broadcom、华为等。

## 物理网口

物理网口是指物理网卡提供的网络接口，用于连接其他物理主机或网络设备。根据网卡规格的不同，一张物理网卡可以提供一个或多个物理网口，用于连接多个设备，从而提高整个网络的可用性。例如，一个总带宽 50 Gbps 的物理网卡，可以提供一个 50 Gbps 的网络接口，也可以提供 2 个 25 Gpbs 的网络接口。

## 物理交换机

物理交换机是一种用于连接多个设备（如计算机、无线接入点、打印机和服务器等）的网络设备。交换机可以接收来自每台设备的网络数据，并将数据正确转发至目标设备，从而实现互联设备的信息共享和正常通信。

按照功能和连接位置来划分，交换机可以划分为接入层交换机、汇聚层交换机和核心层交换机。

- 接入层交换机

  接入层交换机通常用于直连用户主机，用于为用户主机提供网络接入点。接入层交换机一般用于二层网络连通。
- 汇聚层交换机

  汇聚层交换机通常用于连接接入层交换机和核心层交换机。汇聚层交换机主要用于将来自接入层交换机的数据汇聚到核心层交换机。
- 核心层交换机

  核心层交换机是网络的枢纽，用于处理大量来自汇聚层交换机的网络流量，并对整个网络的通信提供保障。

本文描述的交换机均属于接入层交换机。物理主机之间通过连接到交换机实现了相互通信。

按照不同类型交换机的能力，服务器与交换机之间支持多种类型的连接方式。最常见的是服务器与交换机点到点直联。若交换机支持虚拟化技术，还可以支持堆叠或 M-LAG（Multichassis Link Aggregation Group）等高可用连接方式。

- 堆叠

  堆叠指将多台交换机通过堆叠线缆连接在一起，使多台交换机在逻辑上变成一台交换机，作为一个整体参与数据转发。
- M-LAG

  M-LAG，即跨设备链路聚合组，是一种实现跨设备链路聚合的机制。通过将一台设备与另外两台设备进行跨设备链路聚合，从而把链路可靠性从单板级提高到了设备级。

常见的物理交换机厂商有华为，新华三，锐捷，思科，Mellanox 等，不同厂商的物理交换机能力及配置不同，同厂商不同型号的物理交换机间能力也有差异，具体介绍可参考各厂商物理交换机产品手册。

## 基本概念 > 网络虚拟化

# 网络虚拟化

网络虚拟化是一种虚拟化技术，可以将原本基于硬件设备实现的网络功能转变为通过软件实现。
网络虚拟化的目标是在物理硬件和利用该硬件的能力之间引入一个抽象层，实现网络相关的功能。网络虚拟化允许独立于硬件来交付软件资源和网络功能（即虚拟网络），可以用来合并多物理网络，将虚拟网络进一步分割，以及实现虚拟机和虚拟化服务器（Hypervisor）之间的网络通信。

## 虚拟网络

虚拟网络是指通过网络虚拟化技术，使用软件来模拟物理网络。虚拟网络复用了物理网络的工具和概念（如交换机、网卡等），并使用软件来扩展网络功能，从而可完成更简单有效的网络配置。

虚拟网络能够支持不同地点和数据中心的多台计算机、虚拟机、物理主机或其他设备，以和物理网络相同的方式进行互联。相比于基于硬件设备的物理网络，虚拟网络由软件定义，网络管理员无需切换或购买更多硬件设备，即可远程对网络的不同组成部分进行更新和测试，从而使网络管理变得更加集中、便捷和简单。此外，在为特定需求和应用提供网络时，虚拟网络具有更大的灵活性，并能够在不影响服务、安全性和可用性的情况下提供跨网络基础架构移动应用的能力。

虚拟网络的常见组成部分包括：虚拟网卡、虚拟接口、虚拟交换机（Virtual Switch）、虚拟局域网（VLAN）、虚拟路由器（Virtual Router）和虚拟网关等。
在 SMTX OS（ELF）平台中，虚拟网络主要由虚拟分布式交换机、虚拟网卡、系统网络、虚拟机网络等组成，总体框架如下图所示：

## 虚拟分布式交换机

虚拟分布式交换机（Virtual Distributed Switch，以下简称 VDS）是一种通过软件实现的交换机。VDS 和物理交换机具有类似的功能，都可以为它们所连接的设备提供网络通信功能。不同之处在于，物理交换机用于连接物理设备，实现硬件之间的网络连通；而 VDS 主要运行在虚拟化环境，虚拟机可以在 VDS 上建立网络连接，并可通过 VDS 与其他虚拟机或者外部网络进行通信。

VDS 采用了分布式架构，可以提供统一的网络控制平面，可管理所有主机上的虚拟网络的连接。这样一方面可以对多台主机的虚拟交换机进行统一配置和管理，另一方面可以保证虚拟机在不同主机之间迁移时的网络一致性。

VDS 与主机的物理端口形成关联，关联后会每个主机上创建一个 VDS 数据面实例（即 OVS 网桥，基于开源的 Open vSwitch 实现），用来转发主机上的网络流量、封装/解封装 VLAN，以及处理安全规则和 QoS 策略等。

VDS 也可以视为物理交换机在服务器主机上的延伸。VDS 会对主机的物理网口和虚拟机的虚拟网口进行管理，虚拟机的一个虚拟网卡对应虚拟交换机的一个虚拟端口（即图中的 Port），主机的一个或者多个物理网口作为虚拟交换机的上联端口（Uplink-Port）。

VDS 在每个主机上都可以关联一个或者多个物理网口。若虚拟交换机只关联一个物理网口，可能存在单点故障和带宽限制等问题，为解决这些问题，可以通过关联 2 个或 2 个以上的物理网口来提高网络带宽并实现高可用。

![](https://cdn.smartx.com/internal-docs/assets/2706883a/network_whitepaper_04.png)

## 虚拟网卡

虚拟网卡（Virtual Network Interface Card，也称虚拟网络接口适配器）是通过软件模拟物理网卡的功能，用于实现虚拟机与外部网络互联的关键组件。虚拟网卡可以像物理网卡一样配置 IP 地址和 MAC 地址等。

SMTX OS 虚拟机支持 E1000 和 VirtIO 两种类型的虚拟网卡，两种虚拟网卡后端均由软件实现的 TUN 设备提供支持。其中，E1000 是通过模拟 Intel E1000 物理网卡功能而实现的一种虚拟网卡类型，而 VirtIO 是使用 VirtIO 标准虚拟 IO 框架实现的虚拟网卡类型。由于实现方式不同，VirtIO 虚拟网卡相比 E1000 虚拟网卡性能更优。

SMTX OS 还支持提供系统网络使用的虚拟网卡，此类虚拟网卡通过 OVS 网桥创建出来的内部接口进行支持。

## 系统网络

系统网络是主机自身服务使用的虚拟网络。根据用途划分，系统网络可以分为管理网络、存储网络、迁移网络和接入网络。系统网络是 VDS 上的一个虚拟端口，需要创建在某个具体的 VDS 上，通过给系统网络配置 IP 地址、子网掩码和网关等，用户的系统网络流量可以在经过主机的网络协议栈处理后，进入 VDS 或者对应的应用。

系统网络支持配置 MTU 和 QoS 策略，允许设置预留带宽、最大带宽、服务优先级和突发通信量限制。

系统网络均支持修改 VLAN ID 和 IP 地址，但修改后会导致相应系统网络中断，请根据实际情况进行修改。

### 管理网络

管理网络是主机自身管理服务使用的系统网络，用于用户访问和管理集群，以及对接 CloudTower、Everoute、SMTX 备份与容灾等配套产品。

一个 SMTX OS 节点只允许有一个管理网络。在安装部署集群阶段，必须创建管理网络，并设置管理 IP、子网掩码、缺省网关和 VLAN ID 等。管理网络创建后不允许被删除，但可以根据实际需要修改管理网络的配置。

### 存储网络

存储网络是主机上分布式块存储服务所使用的网络。后端的分布式块存储服务在不同主机上通过存储网络进行正常工作。

一个 SMTX OS 节点只允许有一个存储网络。在安装部署阶段，必须创建存储网络，并设置存储 IP、子网掩码和 VLAN ID 等。存储网络创建后不允许删除，支持修改 VLAN ID、MTU 和 QoS 策略。

> **说明**：
>
> 启用 RDMA 功能后，存储网络不支持修改 VLAN ID 和配置 QoS 策略。

### 迁移网络

迁移网络是虚拟机进行集群内迁移和跨集群热迁移时使用的网络。迁移网络可以按需创建，并支持编辑和删除。

在未创建迁移网络的情况下：

- 集群内迁移或跨集群热迁移使用存储网络进行；若存储网络不可达，则使用管理网络迁移。
- 跨集群冷迁移和分段迁移直接使用管理网络进行迁移。

### 接入网络

接入网络用于为其他计算端提供存储接入服务。若需要将 SMTX OS（ELF）集群的分布式块存储服务通过 iSCSI 协议提供给其他计算端使用，需要创建接入网络。

### 镜像出口网络

镜像出口网络专门负责将封装后的镜像流量发送至目标端。在流量镜像功能中，需要由一个系统网络负责发送封装后的镜像流量，目前支持使用**管理网络**或者**镜像出口网络**。

当存在以下任一情况或需求时，推荐创建专用的**镜像出口网络**：

- 为实现流量隔离，希望通过专用的物理网卡发送镜像流量。
- 用于接收镜像流量的目标端与管理网络不通，或无法通过管理网络的默认网关通信。
- 希望通过 QoS 避免带宽争抢，为镜像流量和管理网络流量分别设置特定的预留带宽或带宽使用上限。
- 需要更灵活的静态路由配置，例如为不同目标端指定不同的下一跳路由。

## 虚拟机网络

虚拟机网络是虚拟机运行使用的网络。根据业务需求，集群中可以存在多个虚拟机网络。一台虚拟机可以通过不同的虚拟网卡连接不同的虚拟机网络。当虚拟机在集群内不同主机间进行迁移时，虚拟机网络可保证虚拟机提供的业务在迁移过程中不会中断，也无需重新配置网络数据库。
虚拟机网络可配置 VLAN 和 QoS 等属性。关于 VLAN 和 QoS 详细介绍，可参考 [VLAN](/smtxos/6.3.0/network_whitepaper/network_whitepaper_14) 和 [QoS](/smtxos/6.3.0/network_whitepaper/network_whitepaper_15)。

## SMTX OS 网络架构 > ELF 平台 > 网络分层

# 网络分层

SMTX OS 的网络由物理网络和虚拟网络组成。

## 物理网络

SMTX OS 的物理网络由物理设备通过网线互联组成，包括物理网口、物理交换机、路由器等物理设备。

## 虚拟网络

SMTX OS 的虚拟网络由管理平面和数据平面组成。

### 管理平面

管理平面主要用于管理 VDS、系统网络和虚拟机网络。其管理功能主要由 CloudTower 虚拟网络管理模块提供。

- **VDS 管理**

  用户可在 CloudTower 界面上通过关联物理主机的一个或多个网口来创建 VDS。当选择了多个物理网口时，需选择网口绑定模式以保证网口的高可用。用户也可通过 CloudTower 修改 VDS 名称、关联的网口以及网口绑定模式。

  VDS 创建后，关联 VDS 的物理主机上会生成对应的 VDS 数据面实例（OVS 网桥），用于在各物理主机之间转发网络流量。
- **系统网络管理**

  用户可通过 CloudTower 界面，在 VDS 上创建系统网络，并可设置系统网络的 IP 地址、子网掩码、VLAN ID、MTU、缺省路由、静态路由和 QoS 等。系统网络所在的 VDS 必须关联集群中所有物理主机。
- **虚拟机网络管理**

  用户可通过 CloudTower 界面，在 VDS 上创建虚拟机网络。创建虚拟机网络时，可以为虚拟机网络配置 VLAN ID 和 QoS 策略。当为虚拟网卡分配某个虚拟机网络时，虚拟网卡将继承该虚拟机网络的 VLAN ID 和 QoS 策略。

### 数据平面

数据平面是执行流量转发、转发策略匹配和 QoS 限速功能的实体。数据平面分布式存在于每个 SMTX OS 节点上。

- **VDS 数据面实例**

  VDS 在每一个关联的物理节点上都会创建一个数据面实例（即 OVS 网桥），用于执行 VDS 的转发逻辑。OVS 网桥一方面提供该节点上的虚拟机间的通信能力，另一方面通过将关联主机的物理网口添加到 OVS 网桥的虚拟端口上，实现 SMTX OS 节点与外部物理网络的连通。
- **系统网络实例**

  系统网络属于主机系统业务所使用的系统虚拟网络接口，每种系统网络接口都是在 OVS 网桥上创建的一个 OVS 内部端口（Internal Port），通过给对应的内部端口配置 IP 地址，子网掩码等参数，系统网络可使用系统网络协议栈与其他节点或业务通信。

  系统网络支持配置参数如下：

  | 系统网络类型 | IP | 子网掩码 | VLAN ID | QoS | MTU | 网关 | 静态路由 |
  | --- | --- | --- | --- | --- | --- | --- | --- |
  | 管理网络 | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ | X |
  | 存储网络 | ✓ | ✓ | ✓ | ✓ | ✓ | X | X |
  | 接入网络 | ✓ | ✓ | ✓ | ✓ | ✓ | X | ✓ |
  | 迁移网络 | ✓ | ✓ | ✓ | ✓ | ✓ | X | X |
  | 镜像出口网络 | ✓ | ✓ | ✓ | ✓ | ✓ | X | ✓ |
- **虚拟机网络实例**

  虚拟机网络是一个逻辑概念，在数据平面对应的是虚拟交换机上配置的一个 VLAN 端口组。当虚拟机的虚拟网卡关联到对应的虚拟机网络中时，虚拟网卡将继承虚拟机网络配置的 VLAN ID，在 OVS 网桥上创建虚拟端口时将对应的 VLAN ID 设置到对应的虚拟端口中（Access 类型）。当给虚拟机网络设置 QoS 时，加入到该虚拟机网络中的所有虚拟网卡将共享该虚拟网络 QoS 的带宽设置。

## SMTX OS 网络架构 > ELF 平台 > 网络数据流

# 网络数据流

SMTX OS 的网络数据流包括系统网络数据流和虚拟机网络数据流。

## 系统网络数据流

SMTX OS 提供了管理服务、存储服务、迁移服务和接入服务，每种服务均使用各自的系统网络进行数据流的传输。

系统网络的流量包括出流量和入流量，通过 SMTX OS 主机网络协议栈（tcp/ip stack）与其他节点或者 client 间互通。

![](https://cdn.smartx.com/internal-docs/assets/2706883a/network_whitepaper_06.png)

此处以管理服务为例说明系统网络出入流量的转发过程，其他类型的系统网络处理流程与管理网络类似。

用户对 SMTX OS 进行管理操作时，会将访问请求经过管理网络发往 SMTX OS 的管理服务，管理服务在收到访问请求后，会通过管理网络响应访问请求。在此过程中，管理网络的入流量和出流量的路径和处理方式如下：

- **管理网络的出流量**

  管理服务发出的报文通过管理网络发往主机的网络协议栈（TCP/IP Stack）进行处理，然后通过为管理网络创建的虚拟端口（port-mgt）发送至主机的 OVS 网桥。OVS 网桥根据报文的 VLAN ID 和目标 MAC 地址等信息，将报文发送至对应的主机物理网口，然后转发至物理交换机进行进一步的转发。
- **管理网络的入流量**

  用户访问管理服务时，报文首先经过物理网络的转发，通过物理交换机进入主机的物理网口。物理网口将报文转发至主机上的 OVS 网桥，OVS 网桥根据报文的 VLAN ID 和目标 MAC 地址等信息，将报文转发至管理网络的虚拟端口（port-mgt），经 port-mgt 转发至主机的网络协议栈（TCP/IP Stack）进行处理，最终将处理后的报文通过管理网络发送至主机中的管理服务。

## 虚拟机网络数据流

虚拟机网络主要用于处理虚拟机业务流量。虚拟机可以通过添加虚拟网卡，并将虚拟网卡和某个虚拟机网络进行关联。虚拟网卡将继承所在虚拟机网络的 VLAN ID 和 QoS 策略。例如，若虚拟机网络设置的 VLAN ID 为 100，则该虚拟机网络中的虚拟网卡将只转发 VLAN 100 的流量。

![](https://cdn.smartx.com/internal-docs/assets/2706883a/network_whitepaper_07.png)

- **虚拟机网络的出流量**

  虚拟机中运行的业务提供服务时，发起的数据流量先由虚拟机 Guest OS 中的网络协议栈进行处理，并通过虚拟网卡驱动发送至虚拟网卡。虚拟网卡将流量通过关联的虚拟机网络通过虚拟端口转发至主机的 OVS 网桥，OVS 网桥根据该虚拟机网络所在端口的 VLAN 配置和转发策略转发流量。根据流量不同的目的地，有不同的转发路径。

  - 流量目的地为同一主机上的虚拟机

    OVS 网桥将按照 ⓵>⓶>⓷>⓸>⓻>⓼ 的路径，将虚拟机的流量转发至同主机上的另一台虚拟机。
  - 流量目的地在所在主机之外

    OVS 网桥将按照 ⓵>⓶>⓷>⓹ 的路径，将虚拟机的流量通过主机的物理网口转发至物理网络中。
- **虚拟机网络的入流量**

  主机外部流量通过物理网口进入 OVS 网桥，OVS 网桥根据报文的 VLAN ID 和目的 MAC 地址等信息，通过对应的虚拟网口将流量转发至虚拟机网络，虚拟机网络进一步将流量转发至目的虚拟机的虚拟网卡。虚拟机的虚拟网卡驱动将接收到流量送至虚拟机 Guest OS 的网络协议栈进行处理，处理后发往 Guest OS 的应用中。

  完整的流量转发路径为 ⓺>⓸>⓻>⓼。

## SMTX OS 网络架构 > ELF 平台 > 系统网络部署模式

# 系统网络部署模式

部署系统网络时，可选择分离部署模式或融合融合部署模式。二者的主要区别为多个系统网络是否共用同一个 VDS。

SMTX OS 5.0.4 及之前版本仅支持分离部署，从 SMTX OS 5.0.5 开始，同时支持分离部署和融合部署。

## 分离部署

分离部署指的是 SMTX OS 的每个系统网络均需独占一个 VDS。在部署 SMTX OS 阶段，管理员需分别为管理网络和存储网络各创建一个 VDS。部署后，若需使用接入网络和迁移网络，也需为它们创建各自的 VDS。

分离式部署模式下，不同类型的系统网络不能共用物理网口，因此在为不同系统网络创建 VDS 时，需绑定不同的物理网口。

![](https://cdn.smartx.com/internal-docs/assets/2706883a/network_whitepaper_08.png)

## 融合部署

融合部署指的是 SMTX OS 多个系统网络可以使用同一个 VDS，且虚拟机网络也可以使用系统网络的 VDS，多个网络可以共享 VDS 上关联的一个或多个物理网口的带宽。
使用融合部署模式时，对于使用相同物理网口的系统网络和虚拟机网络，可通过 QoS 策略控制不同网络的出方向流量，包括设置该网络流量在 VDS 转发时的优先级和权重、为该网络预留的带宽和最大限制带宽。

若需要对同一 VDS 上的不同网络进行隔离，可以为不同网络分别划分不同 VLAN。同时，为保证物理网络能够正常转发 VDS 上所有网络的流量，需将与物理网口相连的交换机端口设置为 VLAN Trunk 模式，并保证 VLAN Trunk 中包含了所有系统网络和虚拟机网络设置的 VLAN。

# VMware ESXi 平台

SMTX OS 部署在 VMware ESXi 平台时的网络架构如下：

![](https://cdn.smartx.com/internal-docs/assets/2706883a/network_whitepaper_09.png)

SMTX OS 集成在 VMware ESXi 平台时，运行在 ESXi 主机的 SCVM 虚拟机上，并复用了 VMware vSphere 本身的虚拟网络功能。

SCVM 上需创建 3 张虚拟网卡，并分别配置到 vSphere 虚拟交换机（vSwitch）对应的端口组中，分别给管理网络、存储网络和 NFS 网络使用。

以下主要对 SMTX OS 部署在 VMware ESXi 平台上时，重要的网络组成部分进行介绍，包括 SCVM 使用的虚拟网络和虚拟交换机。VMware vSphere 网络更多详细的介绍，可参考 [VMware 文档](https://docs.vmware.com/cn/VMware-vSphere/7.0/com.vmware.vsphere.networking.doc/GUID-B15C6A13-797E-4BCB-B9D9-5CBC5A60C3A6.html)。

## 虚拟交换机

在 VMware ESXi 平台中，可通过在每台 ESXi 主机中创建标准虚拟交换机，分别实现管理网络、存储网络和 NFS 网络的通信。包含的虚拟交换机和要求如下：

- **管理网络虚拟交换机 —— vSwitch0**

  ESXi 安装完成后，会默认创建一个用于 ESXi 管理和 SCVM 管理的虚拟交换机，默认名称为 vSwitch0。vSwitch0 关联的 vmnic 为 ESXi 管理网络对应的物理网口。vSwitch0 中已默认创建了一个网络标签（即网络名称）为 Management Network 的 VMkernel 和一个名称为 VM Network 的虚拟机端口组，分别用于 ESXi 管理网络和 SCVM 管理网络。
- **存储网络虚拟交换机 —— vSwitch1**

  默认情况下，需要为存储网络单独创建一个虚拟交换机（ vSwitch1）。若存储网络与管理网络共用相同的物理网卡，可复用管理网络虚拟交换机 vSwitch0。存储网络虚拟交换机需关联至少一个带宽为 10 Gbps 及以上的物理网口。

  存储网络虚拟机交换机上需创建一个网络标签固定为 **ZBS** 的虚拟机端口组，用于为 SCVM 提供存储网络。
- **NFS 网络虚拟交换机**

  NFS 网络虚拟交换机用于创建 NFS 网络和 NFS VMkernel。由于 NFS 网络仅供 ESXi 主机与本节点上的 SCVM 虚拟机进行通信，所以 NFS 网络虚拟交换机不需要关联任何物理网卡。

  NFS 网络虚拟机交换机上需创建一个网络标签固定为 **NFS** 的虚拟机端口组，用于为 SCVM 提供 NFS 网络。

## 虚拟网络

在 VMware ESXi 平台中，需要创建一系列的 VMkernel 或虚拟机端口组作为系统网络，用于 ESXi 和 SCVM 之间的通信。

- **ESXi 管理网络 —— Management Network（VMkernel）**

  ESXi 管理网络对应的是 vSwitch0 中名为 Management Network 的 VMkernel，用于 ESXi 主机之间的管理流量通信。ESXi 管理网络在 ESXi 的安装过程中自动创建，用户可配置该网络的 IP 地址、子网掩码、网关和 VLAN ID 信息。
- **SCVM 管理网络 —— VM Network（虚拟机端口组）**

  SCVM 管理网络对应的是 vSwitch0 中名为 VM Network 的虚拟机端口组，用于给 SCVM 虚拟机和 CloudTower 虚拟机使用，并处理 SCVM 虚拟机之间的管理流量。

  在每个 SCVM 虚拟机中，需创建一个网络适配器并关联 SCVM 管理网络。后续可在此网络中配置 SCVM 的管理网络 IP、子网掩码、网关和 VLAN ID 信息。

  在 CloudTower 虚拟机中，需创建一个网络适配器并关联 SCVM 管理网络。后续可在此网络中配置 CloudTower IP、子网掩码、网关和 VLAN ID 信息。
- **SCVM 存储网络 —— ZBS（虚拟机端口组）**

  SCVM 存储网络对应存储网络虚拟交换机中，网络标签为 **ZBS** 的虚拟机端口组。

  在每个 SCVM 虚拟机中，需创建一个网络适配器并关联 SCVM 存储网络。后续可在此网络中配置 SCVM 的存储网络 IP、子网掩码、网关和 VLAN ID 信息。
- **IO 重定向网络（连接 ZBS 网络的 VMkernel）**

  IO 重定向网络对应一个 VMkernel，用于 ESXi 主机中的 IO 请求重定向。IO 重定向网络 需和 SCVM 存储网络使用同一个存储网络虚拟交换机。后续可在此网络中配置 IO 重定向网络 IP 及子网掩码。
- **NFS 网络（虚拟机端口组）**

  NFS 网络对应 NFS 网络虚拟交换机中名为 **NFS** 的虚拟机端口组，用于 ESXi 主机和本主机上的 SCVM 虚拟机进行通信。

  在每个 SCVM 虚拟机中，需创建一个网络适配器并关联 NFS 网络。NFS 网络的 IP 地址固定为 192.168.33.2，子网掩码为 255.255.255.0，不允许更改。
- **NFS VMkernel（连接 NFS 网络的 VMkernel）**

  NFS VMkernel 对应一个 VMKernel 网络适配器，用于 SCVM 连接 NFS 网络。NFS Kernel 的 IP 地址固定为 192.168.33.1，子网掩码为 255.255.255.0，不允许更改。
- **业务网络（虚拟机端口组）**

  对应一个业务虚拟机上的虚拟机端口组，可以供集群中的业务虚拟机使用。

## 网络特性 > 网口绑定

# 网口绑定

网口绑定指通过将多个物理网口绑定在一起形成一个逻辑端口，网口绑定既可以提高物理网口的高可靠性，也可以提高物理网口的总带宽。

SMTX OS 在创建 VDS 时，可以关联物理主机一个或多个网口。当 VDS 关联了多个网口时，支持设置网口绑定模式。SMTX OS 支持两种绑定类型：**OVS Bond** 和 **Linux Bond**。

在编辑 VDS 时，若 VDS 关联了主机的多个非隔离状态的物理网口，且网口绑定模式为 **active-backup**，则支持将正常状态的网口指定活动网口，原活动网口自动切换为备用网口。

## OVS Bond

若虚拟分布式交换机中未包含存储网络，或虚拟分布式交换机中包含存储网络但存储网络未启用 RDMA 功能，则使用 **OVS Bond** 绑定类型。

OVS Bond 支持以下绑定模式：

- **active-backup**

  该模式下，只有一个网口处于活跃状态，用于发送和接收所有的流量，其余网口作为备用。若活跃网口发生故障，将立即切换至备用网口。
- **balance-slb**

  虚拟分布式交换机根据报文的源 MAC 地址和 VLAN Tag 计算哈希值，根据不同的哈希值，为不同流量选择不同的物理网口进行转发，实现网口的负载均衡。用户可配置 rebalance interval 参数，设置网口的负载平衡周期。

  该模式对物理网口所连物理交换机无配置要求。
- **balance-tcp**

  VDS 使用了 LACP（Link Aggregation Control Protocol，由 802.3ad 定义）协议，可同时使用多个网口进行流量转发。若某一网口发生故障，其他网口可自动接替该故障网口，分担流量。该模式可充分利用网络带宽，实现负载均衡和保障网络传输的可靠性。

  该模式下，VDS 支持和单物理交换机、堆叠交换机和 M-LAG 交换机组网，但物理交换机必须已启用 LACP 动态链路聚合。

## Linux Bond

若虚拟分布式交换机中包含存储网络且存储网络已启用 RDMA 功能，则使用 **Linux Bond** 绑定类型。

Linux Bond 支持以下绑定模式：

- **active-backup**

  该模式下，只有一个网口处于活跃状态，用于发送和接收所有的流量，其余网口作为备用。若活跃网口发生故障，将立即切换至备用网口。
- **balance-xor**

  虚拟分布式交换机根据报文的源 MAC 地址和目的 MAC 地址计算哈希值，根据不同的哈希值，为不同流量选择不同的物理网口进行转发，实现网口的负载均衡。

  该模式下，与 VDS 相连的物理交换机需配置静态 EtherChannel，不然会存在 MAC 地址漂移的问题。
- **802.3ad**

  VDS 使用了 LACP（Link Aggregation Control Protocol，由 802.3ad 定义）协议，可同时使用多个网口进行流量转发。若某一网口发生故障，其他网口可自动接替该故障网口，分担流量。该模式可充分利用网络带宽，实现负载均衡和保障网络传输的可靠性。

  该模式下，VDS 支持和单物理交换机、堆叠交换机和 M-LAG 交换机组网，但物理交换机必须已启用 LACP 动态链路聚合。

## 网络特性 > VLAN

# VLAN

VLAN（Virtual LAN）是一种将二层网络在逻辑上划分为多个不同的虚拟局域网，从而实现二层网络隔离的技术。划分到同一个 VLAN 中的网络设备可以互相通信，而不同 VLAN 之间的网络设备则不能直接通信。

SMTX OS 的系统网络和虚拟机网络均支持设置 VLAN。通过为不同虚拟网络设置不同的 VLAN ID（范围为 0-4095。0 表示未设置 VLAN ID；4095 通常为保留 ID，配置后需根据物理交换机的行为确定最终生效状态），可以将各虚拟网络划分进不同的 VLAN，从而进行二层隔离。

在实际业务场景中，若需要将不同服务进行网络隔离，可以对提供相应服务的虚拟机网络配置不同的 VLAN ID。

## 虚拟网络层面

SMTX OS 虚拟网络支持两种类型的 VLAN：Access 和 Trunk。

### Access

SMTX OS 的系统网络和虚拟机网络支持 Access 类型的 VLAN。Access 类型的虚拟网络只能配置一个 VLAN ID，在 Access 虚拟机网络中的虚拟机端口也只能属于一个 VLAN，一般用于连接计算机的端口。Access 虚拟机网络的流量进出虚拟机端口时不带任何的 VLAN Tag。

### Trunk

SMTX OS 的虚拟机网络支持 Trunk VLAN，可配置一个或多个 VLAN（例如 10，20-30），并允许这些 VLAN 的数据报文通过。在 Trunk 类型的虚拟机网络中，虚拟机端口发送的网络报文带有 VLAN Tag，用于标识其所属的 VLAN，且虚拟机端口能够接收和识别其他网络报文中的 VLAN Tag。Trunk 类型的虚拟机端口常用于交换机之间的连接，用于转发来自不同 VLAN 的网络流量。若要配置虚拟机网络为 Trunk 类型，需要在虚拟机中配置相应的 VLAN 子接口。

## 物理网络层面

- 对于 Access 类型的系统网络和虚拟机网络，对应的虚拟分布式交换机上联网卡连接的物理交换机端口可配置为 Access 模式，也可配置为 Trunk 模式。
- 对于 Trunk 类型的虚拟机网络，与该虚拟机网络对应的虚拟分布式交换机上联网卡连接的物理交换机端口需配置为 Trunk 模式，并设置相应的 VLAN ID 范围。

为了保证流量能够通过物理网络转发，需确保物理交换机端口中设置的 VLAN ID 范围包含相关虚拟分布式交换机中所有虚拟网络的 VLAN ID。

## 网络特性 > QoS

# QoS

QoS（Quality of Service，服务质量）是一种可针对不同业务流量设置不同带宽和优先级，从而保证关键服务能够在网络资源有限的情况下依然能够提供较好性能的技术。

例如，在使用网络融合模式部署时，多个系统网络或虚拟机网络共用一个 VDS，并共享 VDS 关联的物理网口的带宽。在没有设置 QoS 策略的情况下，所有网络的流量将被无差别进行转发。若一种业务流量过高，可能会耗尽所有物理网口的所有带宽，导致其他业务无法正常提供服务。此时，设置 QoS 策略能够保障关键业务的服务质量。

QoS 可以为 VDS 上的虚拟网络预留带宽，设置流量优先级和权重和限制带宽。“预留带宽”是指为虚拟网络设置的保证带宽，且一个 VDS 中所有网络的预留带宽之和不允许超过物理网口上限；“限制带宽”是指虚拟网络能够达到的上限，可以限制虚拟网络的最大带宽和突发通信量上限。

为虚拟网络设置 QoS 能够优化业务性能。例如管理网络的流量带宽要求较低，可以为其设置较小的带宽；存储网络带宽延时要求都比较高，可以设置较高的带宽和较高的共享带宽优先级。例如，VDS 关联了两张带宽为 10 Gbps 的物理网口，并使用了 Balance-slb 绑定模式，使得 VDS 最大带宽达到 20 Gbps。通过设置 QoS，可以为存储网络预留 10 Gbps 带宽，为管理网络预留 1 Gbps，为虚拟机网络预留 5 Gbps，剩余 4 Gbps 带宽可以由不同业务根据优先级共享。

创建 VDS 时，将多个物理网口绑定可以提升出口总带宽，但是在不同网口绑定模式下， QoS 设置带宽策略时有一些限制。以两物理网口绑定为例，在不同的绑定模式下，VDS 可用的总带宽不同：

- **active-backup**

  一个物理网口状态为 Active，另外一个物理网口为 Backup，总带宽由 Active 网口的带宽决定。
- **balance-slb**

  两个物理网口都是 Active ，但由于 balance-slb 根据源 MAC 地址 + VLAN 的方式分流，同一个虚拟机的虚拟网卡由于 MAC 地址固定的，因此流量只能从一个物理网口转发，因此总带宽虽然是两个物理网口的带宽之和，但是这个虚拟机在某一时刻只能用到其中一个物理网口的带宽。

  例如：两张 10 Gbps 的物理网口，总带宽 20 Gbps，给管理网络设置 1 Gbps 带宽，存储网络设置 12 Gbps，虚拟机网络设置 4 Gbps，由于管理/存储/虚拟机网络的某一时刻 MAC 地址固定，因此流量在某一时刻只能使用一个物理网口，各虚拟网络的分配总带宽由两个物理网口共同提供。比如虚拟机网络的 4 Gbps 带宽，是由两张物理网口各提供 2 Gbps。
- **balance-tcp**

  两个物理网口都为 Active，但是 balance-tcp 根据一个网络连接的五元组信息分流，因此对于一个确定的连接只能使用到其中一个物理网口。balance-tcp 的带宽分配方式与 balance-slb 相同。该绑定模式下，多个不同连接可以充分利用物理带宽。

## 虚拟网卡 QoS

SMTX OS 从 5.1.0 版本开始，利用 kernel 提供的 TC（Traffic Control）功能组件，通过在端口处建立一个队列，实现了对虚拟网卡出、入流量的 QoS 设置，包括设置最大带宽和突发通信量上限。

为保证网络的稳定性并平衡突发流量，建议将突发通信量上限设置为最大带宽的 20%。例如，如果要将某个虚拟机端口的出流量最大带宽设置为 1000 Mbit/s，突发通信量上限可以设置为 200 Mbit。

需要注意的是，由于速率限制涉及多个因素，实际速率限制可能与设置的值存在一定误差，通常约为设置值的 5% 左右。

## 网络特性 > MTU

# MTU

MTU（Maximum Transmission Unit，最大传输单元），是指网络能够传输的最大数据包大小，以字节为单位。MTU 决定了发送端一次性能够发送报文的最大字节数。如果 MTU 过大，超过了接收端所能够承受的最大值，或者是超过了发送路径上途经的某台设备所能够承受的最大值，就会造成报文分片甚至丢弃，加重网络传输的负担。如果 MTU 过小，那实际传送的数据量就会过小，影响传输效率。

SMTX OS 支持设置虚拟网络（包括系统网络、虚拟机网络）和物理网口的 MTU。

当设置虚拟网络的 MTU 时，若对应的任一物理网口的 MTU 小于虚拟网络的 MTU，则必须在设置时勾选同时调高关联的物理网口 MTU 至 ...，系统自动将物理网口的 MTU 调整至与虚拟网络 MTU 一致；若对应的任一物理网口的 MTU 高于虚拟网络的 MTU，则不会同时调低关联的物理网口 MTU。

修改虚拟机网络的 MTU 后，若有 VLAN 类型的虚拟网卡使用了该虚拟机网络，虚拟网卡会自动应用相同的 MTU。如为 SR-IOV 直通网卡，则不会自动应用虚拟机网络的 MTU；如为 E1000 类型的虚拟网卡，则仅支持 MTU 为 1500。

修改 MTU 同时需要保证链路中所有途经节点的 MTU 一致，因此物理网口所连接的物理交换机也需要对 MTU 进行相应的调整。

## 网络特性 > IGMP/MLD 侦听

# IGMP/MLD 侦听

IGMP/MLD 侦听（IGMP/MLD Snooping）运行在二层设备上，通过侦听三层设备与主机之间的 IGMP 或 MLD 报文来生成二层组播转发表（MDB，Multicast Database Table），从而管理和控制组播数据报文的转发，实现组播数据报文在二层的按需分发。

SMTX OS 支持在虚拟分布式交换机（VDS）粒度启用 IGMP/MLD 侦听。启用后，VDS 对应的 OVS bridge 将侦听连接的虚拟机或外部设备发送的 IGMP/MLD 报文，并生成 MDB 表用于控制组播报文转发。

- 对于目的组播地址在 MDB 表中的报文，仅转发给已加入相应组播组的虚拟机。
- 对于目的组播地址不在 MDB 表中的报文，将丢弃而不转发给本节点任何虚拟机。

若虚拟分布式交换机中的业务流量为组播流量，但未使用 IGMP/MLD 报文进行组成员管理，请勿启用此功能，否则该类组播报文将无法被转发。

在启用 IGMP/MLD 侦听后，为防止 MDB 表项过期，SMTX OS 节点上的 `mcast-agent` 组件会定期向本节点虚拟机发送 IGMP/MLD Query 报文，虚拟机通过 Report 报文响应以刷新对应表项。

## 网络特性 > LLDP

# LLDP

LLDP（Link Layer Discovery Protocol，链路层发现协议）是定义在 802.1ab 中的一个二层协议。网络中的设备可以通过其将管理地址、设备标识、接口标识等信息发送给同一个局域网络中的其它设备。通过 LLDP 协议可以非常方便的检查物理链路的连接情况，用户可以实时查看主机物理网口所连接的物理交换机的端口，名称和 VLAN 设置等信息。

SMTX OS 从 5.0.5 版本开始支持 LLDP 协议，且默认开启，无需手动设置。

## 网络特性 > SR-IOV

# SR-IOV

SR-IOV (Single Root - I/O Virtualization) 是一种基于硬件的虚拟化解决方案，可提高网卡性能和可伸缩性。SR-IOV 具有如下特性：

- SR-IOV 允许在虚拟机之间高效共享 PCIe（Peripheral Component Interconnect Express，快速外设组件互连）设备，并且由于 SR-IOV 是在硬件中实现的，因此可以获得能够与主机性能媲美的 I/O 性能。
- 启用了 SR-IOV 并且具有适当的硬件和操作系统支持的 PCIe 设备可以显示为多个单独的物理设备，可达到单个 I/O 资源被许多虚拟机共享的目的。与物理网口直通相比，SR-IOV 具备更高的可伸缩性。

SR-IOV 有两种功能类型：

- 物理功能 (Physical Function，PF)

  PF 是全功能的 PCIe 功能，用于支持和管理 SR-IOV 功能，一个 PF 可以切割出多个 VF。
- 虚拟功能 (Virtual Function，VF)

  VF 是一种轻量级的 PCIe 功能，与 PF 以及与同一 PF 关联的其他 VF 共享一个或多个物理资源。

SMTX OS 支持 SR-IOV 直通功能，可将一个支持 SR-IOV 的物理网口虚拟化出多个 SR-IOV 直通网卡（即 VF），并直接挂载给虚拟机使用，大幅降低虚拟机的网络延时。

在数据库、高频交易等场景，业务虚拟机往往需要较高的网络性能。通过使用 SR-IOV 功能及低延时网卡（如 Solarflare，Mellanox 等），SMTX OS 可用于低延时高频交易场景，为用户提供可媲美物理机的性能和虚拟化的更高资源利用率。

与传统虚拟网卡相比，SR-IOV 直通网卡有以下优势：

- 可绕过虚拟化层，缩短数据传输的路径，在网络延时上可获得与传统物理网卡接近的网络 I/O 性能。
- 结合低延时网卡的加速方案，可以绕过内核协议栈，缩短网络包的传输路径，虚拟机的网络 I/O 延时可以超过物理机直连的性能。

![](https://cdn.smartx.com/internal-docs/assets/2706883a/network_whitepaper_11.png)

通过将支持 SR-IOV 的物理网口进行切割，可以提供多个 VF，每个 VF 可以作为一个 SR-IOV 直通网卡，分配给挂载该网口作为 SR-IOV 直通网卡的虚拟机使用，开启 SR-IOV 之后的网络架构如下：

## 网络特性 > RDMA

# RDMA

RDMA （Remote Direct Memory Access）是一种可以绕过双方操作系统内核，通过网络直接访问其内存数据的技术。由于不经过操作系统，使用 RDMA 协议可以节省大量CPU资源，提高系统吞吐量，并降低系统的网络通信延时。

SMTX OS 集群支持对存储网络开启 RDMA，远程副本传输可越过多个 OS 缓冲区，降低延时，提高集群的整体性能，同时改进延时和吞吐能力两个指标。

![](https://cdn.smartx.com/internal-docs/assets/2706883a/network_whitepaper_13.png)

使用 RDMA 技术可以带来如下优势：

- 高带宽压力下，对 CPU 占用极低，在使用 25 GbE 以上（25 GbE、 100 GbE、 200 GbE）的高速网络时可以充分发挥其带宽优势；
- 避免了操作系统的缓冲区开销，实现了不同节点之间的分布式块存储服务直接通讯，有效降低延时。

## ELF 平台部署架构

与非 RDMA 场景下的部署架构相同，SMTX OS 直接控制物理网口，配置形态为物理网口加入虚拟分布式交换机，存储网络 IP 配置在虚拟分布式交换机的虚拟接口上。

![](https://cdn.smartx.com/internal-docs/assets/2706883a/network_whitepaper_14.png)

## VMware ESXi 平台部署架构

VMware ESXi 平台下，由于 SCVM 中 SMTX OS 无法直接控制网口，因此开启 RDMA 之后，需要对物理网口开启 SR-IOV。ESXi 使用网口的物理功能 PF，同时提供虚拟功能 VF 给 SCVM 使用。当物理网口支持 RDMA 时，这个 VF 也具备了 RDMA 能力。此模式接近透传，性能也与整张网卡直接透传接近。

如果存储网络有多个物理网口，会分配出多个 VF 给 SCVM 使用，需要在 SCVM 内部进行多链路处理。因此在 SCVM 内增加了一层 OVS，为了 OVS 可以和外界通信，需要单独增加一个 vSwitch 并开启混杂模式。

在 SCVM 中，SR-IOV 的 VF 网络作为存储网络，负责节点间的数据传输，原有 `vmxnet3` 类型的 ZBS 网络作为 VMware 接入网络，负责 SCVM 和本地 ESXi 的通信。

启用 RDMA 后，网络架构变更如下：

![](https://cdn.smartx.com/internal-docs/assets/2706883a/network_whitepaper_15.png)

- **管理网络（Management）**

  与原有模式相同，连接 1 GB 物理网口。
- **NFS 网络**

  与原有模式相同，不连接物理网口，仅用于 SCVM 和 ESXi 之间的通信。
- **VMware 接入网络（ZBS）**

  用于 SCVM 与本地 ESXi 通信，流量经过 vSwitch 直接转发，不经过物理网口。
- **存储网络（RDMA）**

  用于 SCVM 与外部（其他 ESXi 或者 SCVM）通信。

  此网络中的 vSwitch 不连接物理网口，只用于 SR-IOV 配置（需开启混杂模式）：将 SR-IOV 的 VF 直接配置在该 vSwitch 下的 RDMA portgroup 中。流量不会经过 vSwitch（虚线表示），会通过 VF 做收发，直接经过物理交换机。

  网络架构调整后，RDMA 网络使用原有的存储 IP，需要为 ZBS 网络新增 VMware 接入网络 IP。当 SCVM 对外通信时，如果目的端是本地 ESXi，将通过 VMware 接入网络 IP 发送；其他情况下，使用存储网络 IP 发送。

## 网络特性 > 流量镜像

# 流量镜像

流量镜像（Traffic Mirror）会将监控设备上符合指定条件的数据包（例如指定端口或 VLAN 流量）复制一份副本，并将其转发至观察端口。当观察端口和监控设备非直连，而是通过二层网络或三层网络相连时称为**远程镜像**。

- 当观察端口通过二层网络与监控设备相连时，称为**二层远程镜像（RSPAN，Remote Switched Port Analyzer）**。
- 当观察端口通过三层网络与监控设备相连时，称为**三层远程镜像（ERSPAN，Encapsulated Remote Switched Port Analyzer）**。

ERSPAN 已逐渐发展为一套标准协议，基于 GRE 隧道进行封装。

SMTX OS 集群支持 ERSPAN 封装的三层远程镜像。允许将符合镜像配置规则的流量通过**管理网络**或者专用的**镜像出口网络**发送至远端，如下图所示。
