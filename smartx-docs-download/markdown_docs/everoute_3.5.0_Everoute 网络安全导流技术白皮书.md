# 关于本文档

## 概述

# 概述

在虚拟化与云原生技术成为主流的当下，传统边界防火墙模型已无法适应动态、东西向流量为主的数据中心内部环境。Everoute 分布式防火墙提供了基础的 L2 至 L4 微隔离能力，对于应用层（L7）的高级威胁，如漏洞利用、恶意软件通信、命令与控制（Command and Conrtol，简称 “C2”）攻击等，我们通过与安全厂商深度合作，共同构建深度安全防护的生态系统来应对。

基于上述背景，Everoute 网络安全导流解决方案应运而生。该方案将深度包检测（Deep Packet Inspection，简称 “DPI”）与入侵防御系统（Intrusion Prevention System，简称 “IPS”）能力无缝集成到虚拟化网络中，作为对 Everoute 分布式防火墙的关键补充。在此基础上，它利用虚拟化网络架构，实现了对虚拟机工作负载流量的无损、自动化导流与高级安全检测，为云内工作负载提供了一套零信任、非侵入式的无代理 L7 高级威胁防护体系。

Everoute 网络安全导流具有如下特点：

- **无代理部署，简化运维**：无需在业务虚拟机内安装任何代理程序，避免了代理与业务应用的兼容性问题，降低了资源开销与运维复杂度。
- **安全与基础设施解耦**：通过标准化的导流机制，将安全能力（DPI/IPS）从业务基础设施中解耦，实现了安全策略的独立管理与动态更新。
- **业务无感知的弹性防护**：支持虚拟机的动态迁移。当受保护的虚拟机迁移至新主机时，导流策略自动跟随，无需人工干预，确保安全防护的连续性与一致性。
- **集中化管理**：安全管理员可统一在的 CloudTower 管理平台上便捷定义导流策略，并在安全管理平台上配置精细化的 IPS 防护规则，实现安全策略的集中化、精细化管控。

通过分布式防火墙与网络安全导流能力，Everoute 帮助企业构建适应虚拟化环境的安全体系，同时显著降低高级威胁防护的部署与运维复杂度。

## 产品架构

# 产品架构

Everoute 网络安全导流架构主要由管理平面、控制平面和数据平面组成。各平面组件相互独立，通过 API 与声明式配置协同工作，将虚拟机工作负载流量导入安全防护虚拟机（Security Virtual Machine，简称 “SVM”）中。

## 产品架构 > 管理平面

# 管理平面

管理平面主要由 everoute-operator、CloudTower 和 API 组成，用于管理所有配置相关的 API 请求，以及所有 Everoute 网络安全导流相关的运维操作。

- **CloudTower**：用于提供 UI 并与 Everoute 控制平面和运维管理平面进行交互，负责将业务配置和运维管理配置通过相关 API 进行下发。
- **everoute-operator**：用于 Everoute 服务的运维管理，负责接收来自 CloudTower 的请求并且触发相应操作。

管理平面主要提供以下功能：

- 管理用于部署、升级 Everoute 服务的文件。
- 多 Everoute 服务管理，部署、卸载、一键热升级 Everoute 服务，关联 SMTX OS（ELF）集群或 SMTX ELF 集群，Everoute Controller 虚拟机生命周期管理。
- 提供 REST API，将用户的配置进行持久化存储和下发。
- 管理所有增、删、改、查任务的生命周期，为后端服务器预先处理高并发的请求。
- 过滤所有用户配置，校验相关参数，减轻后端服务器系统压力。
- 提供监控、告警、认证、数据采集等基本能力，简化系统运维的复杂性。

## 产品架构 > 控制平面

# 控制平面

控制平面采用集中控制和分布式代理的架构，通过一组由 1、3 或 5 个控制器节点所组成的 Everoute 控制器集群，对 SMTX OS（ELF）集群或 SMTX ELF 集群进行集中式管理，并将控制代理 (everoute-agent) 分布在每个节点上部署，以负责下发导流规则到数据面。

- **Everoute 控制器集群**
  - 由 1、3 或 5 个控制器节点组成，每个控制器（Controller）占用少量资源，无单点故障（1 节点除外）。
  - 控制器集群利用 etcd 提供主节点选举和实时监控。当有节点发生变化或者掉线后，集群可以第一时间检测到节点的健康状态，并实现主从切换，以满足业务连续性。同时 etcd 还为控制平面提供数据的持久化。
  - 通过 k8s-apiserver 提供声明式 API。
  - 通过 tr-controller 监听 CloudTower 虚拟机网卡的导流配置，并转换为 Everoute 导流规则通过声明式 API 存储到 etcd 中。
  - 通过 tr-controller 实现基于声明式 API webhook 的 API 参数校验。
- **everoute-agent**
  - 部署在 SMTX OS（ELF）集群或 SMTX ELF 集群的每个节点上。
  - 监听 Everoute 导流规则下发到数据面。
  - 通过 vm-security-controller 服务监听 SVM DPI 模块的健康状态。在 SVM DPI 不健康时控制数据面不再导流，并在 SVM DPI 状态健康时恢复导流。

## 产品架构 > 数据平面

# 数据平面

数据平面负责网络流量的转发与导流执行，基于 Open vSwitch（OVS）实现数据包的转发控制。

每个计算节点上部署的 SVM 配置有两张导流网卡（如 NIC 1 与 NIC 2），均连接至同一 VDS-Bridge。当受保护的虚拟机发送或接收流量时，其出入向数据包会被 VDS-Bridge 通过导流网卡送入本节点的 SVM 中。

SVM 对数据包进行深度检测与安全处理后，再通过导流网卡将其送回 VDS-Bridge。随后，VDS-Bridge 会按原有的转发逻辑，将处理后的流量送达最终目的地，该目的地可能是同一节点上的其他虚拟机、其他节点的虚拟机、或外部网络。

## 技术实现

# 技术实现

Everoute 网络安全导流通过与虚拟化网络深度集成，为云内虚拟机提供非侵入式的无代理高级安全防护。安全管理员无需在虚拟机中安装任何代理程序，可直接在 CloudTower 管理平台上进行配置，定义哪些虚拟机的流量需被导流至深度安全服务。例如，管理员可以一键将所有 Web 服务器虚拟机的出站和入站流量纳入导流范围。

Everoute 网络安全导流的运作架构与数据流如下图所示，其工作流程构成一个完整的闭环：

![](https://cdn.smartx.com/internal-docs/assets/2fbdb675/whitepaper_sts_06_01.png)

1. **流量重定向**：系统通过 VDS 内置的导流模块，将指定的业务虚拟网卡的全部出入向流量，精确地重定向至其所在宿主机上的 SVM。
2. **深度包检测**：SVM 内置的 DPI 模块（具备防火墙及 IPS 能力）负责接收并检测此部分流量。该模块依据安全管理平台下发的安全策略，对流量进行深度分析，识别并处置潜在的安全威胁。
3. **流量回注**：经过 DPI 模块安全处理后的流量将被重新送回到 VDS 中。随后，VDS 会将这些“洁净”的流量按其原始目的进行正常转发，其目的地可能是同一虚拟化环境内的其他虚拟机，也可能是外部网络。
