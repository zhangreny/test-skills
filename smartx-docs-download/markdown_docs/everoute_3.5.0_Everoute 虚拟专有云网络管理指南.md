# everoute/3.5.0/Everoute 虚拟专有云网络管理指南
## 关于本文档

# 关于本文档

Everoute 是由北京志凌海纳科技股份有限公司（以下简称 “SmartX”）自主研发的网络与安全产品，为 SmartX 超融合和企业云基础设施提供了软件定义的网络和安全系列功能。Everoute 可提供分布式防火墙、网络安全导流、负载均衡、虚拟专有云网络、容器网络等功能，同时支撑虚拟化和容器化应用，从而形成与虚拟机和容器一体化管理的网络和安全解决方案。

本文档介绍了如何使用 Everoute 虚拟专有云网络的功能特性。阅读本文档的对象需了解 SMTX OS 超融合软件、SMTX ELF 虚拟化软件以及 Everoute 服务的相关技术背景知识，并具备丰富的数据中心操作经验。

## 概述

# 概述

Everoute 虚拟专有云网络（ Virtual Private Cloud，VPC）是 SmartX 的网络虚拟化产品，它采用 GENEVE 作为覆盖（Overlay）网络协议，可以在由多个数据中心/集群组成的 SmartX 企业云环境中提供统一的虚拟化网络。

Everoute VPC 为 ELF 虚拟机提供安全隔离的网络空间和动态地址分配、浮动 IP、负载均衡等多种虚拟化网络功能（VNF），并通过 VPC 对等连接、L3 路由、地址转换或 L2 桥接方式，实现了 VPC 内部、VPC 之间、以及 VPC 与外部网络的互通。Everoute VPC 支持虚拟机在多个数据中心的高可用、负载均衡和容灾恢复，实现更短的业务恢复时间（RTO）。

SmartX 的 CloudTower 管理中心实现了在直观的图形化界面上对 SmartX 虚拟化计算（ELF）、虚拟化存储（ZBS）与虚拟化网络和安全（Everoute）的统一化管理、配置、监控和运维。

通过开放的 API，Everoute VPC 可以与云平台无缝对接，为企业提供自动化和灵活的网络配置选项，以更好地支持敏态云应用。

## 概述 > 产品功能

# 产品功能

- **支持自定义逻辑隔离的 VPC**

  支持自定义地构建多个逻辑隔离的 VPC。用户可以在其中创建专属的 VPC 资源、管理自己的子网结构、按需分配 IP 地址、以及通过网关与安全服务等功能实现对流量的自主控制。
- **支持为虚拟机分配浮动 IP**

  支持分配浮动 IP，虚拟机动态关联浮动 IP 后即可与外部网络进行通信。
- **提供丰富的网关功能**

  - 支持配置路由网关，使 VPC 内的虚拟机能够使用私有地址与 VPC 外部的三层网络进行通信。
  - 支持配置 NAT 网关，提供 DNAT 和 SNAT 两种能力，用户可以按需将 VPC 内的私有地址转换为外部子网的地址。
  - 支持配置 L2 网关，能够将 VPC 子网和某个 VLAN 虚拟机网络连接并形成一个二层子网，从而使 VPC 内部的虚拟机可以直接访问 VLAN 网络上的服务。
  - 支持配置 VPC 对等连接，使两个 VPC 之间的虚拟机使用私有地址进行通信。
- **支持规划出入 VPC 的流量**

  支持为 VPC 子网关联路由表，并设置多条路由规则，将通向目标地址的流量路由至指定的下一跳网关服务。
- **支持为 VPC 设置分布式防火墙**

  支持为每个 VPC 设置安全组、安全策略及隔离策略，将包含在安全组中的虚拟机作为策略对象，为虚拟机配置入流量白名单及出流量白名单。在有虚拟机中毒或需要进行调试时，将其设置为完全隔离，禁止指定虚拟机接收或发出任意流量；或者设置为诊断隔离，为指定虚拟机设置出、入流量白名单。

## 概述 > 应用场景

# 应用场景

- **降低维护物理网络的成本**

  VPC 基于 GENEVE 协议的 Overlay 虚拟化网络技术实现了与物理网络的解耦，在 VPC 内配置网络无需变更物理网络。只需满足物理网络三层连通，即可灵活地对 VPC 网络进行扩缩容、配置不同类型的网关、跨集群/机架/机房迁移虚拟机。
- **建立业务网络的高可用**

  支持跨集群/机架/机房/数据中心构建 VPC，可以将其业务应用无缝地扩展到不同站点，并且快速实现业务在异地站点的高可用、负载均衡或灾备，从而减少单点故障的风险、提高业务连续性。
- **业务部门相互隔离**

  不同 VPC 内的网络配置操作互不影响，不同业务部门可以在专属的网络空间内自定义地配置网关、路由、安全策略等，不会对其他 VPC 环境产生资源冲突或影响，也无需依赖网络管理员的统一规划。
- **适应云环境**

  企业云环境中，不同类型的用户需要在共享基础架构的同时，享有自主管理计算、存储、网络资源的能力。虚拟专有云网络能够良好地满足这类需求，支持划分成百上千个相互隔离的 VPC，每一个 VPC 中可独立管理 IP、网关等资源。

## 快速入门

# 快速入门

在部署 Everoute 服务时，您可以启用并配置虚拟专有云网络功能；如未在部署阶段启用，也可单独[启用虚拟专有云网络](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_46)功能。启用虚拟专有云网络功能之后，即可按照如下步骤说明开始使用虚拟专有云网络。

下图为一个 VPC 的功能总览。实际使用过程中，请参考 [VPC 规划示例](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_98)结合具体业务场景按需规划 VPC。

![](https://cdn.smartx.com/internal-docs/assets/83f40769/administration_guide_vpc_96.png)

1. [关联集群](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_48)

   即确定 VPC 的服务范围。当您希望集群上的虚拟机能够使用 VPC 网络时，请为该集群关联虚拟专有云网络。

   关联集群上的所有节点的 Underlay 网络之间需要二层或三层互通（包含跨集群节点间的互通），以使所有节点上的 TEP IP 能够相互连通。

   - 如果节点的 Underlay 网络三层互通，则在为节点配置 TEP IP 后，需在三层网络上配置相应的路由，以确保 TEP IP 能够三层互通。
   - 如果节点的 Underlay 网络二层互通，则无需为 TEP IP 额外配置路由。
2. 当您希望 VPC 能够与外部网络通信时，请[创建边缘网关或边缘网关组](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_49)，并[配置外部子网或外部子网组](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_55)，为浮动 IP、路由网关、NAT 网关提供外部可路由的 IP 地址块。

   根据不同业务场景，有如下两种使用方式：

   - **使用边缘网关**：您可以创建一个或多个边缘网关，并为每个边缘网关创建一个或多个外部子网。多边缘网关适用于需要网关流量负载分担的场景。
   - **使用边缘网关组**：您可以创建一个或多个边缘网关组，并为每个边缘网关组创建一个或多个外部子网组。适用于需要高可用性和冗余连接的场景。
     - 每个边缘网关组包含两个配置类型相同的边缘网关，分别作为主边缘网关和备边缘网关。
     - 每个外部子网组包含主、备两个外部子网，主外部子网属于主边缘网关、备外部子网属于备边缘网关。
3. 创建 VPC

   根据业务是否需要通过 VPC 进行隔离，您可以按需[创建一个或多个 VPC](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_60)。

   每个 VPC 中均可以创建 VPC 子网、路由表、浮动 IP、网关服务和安全服务。不同 VPC 之间网络相互隔离、互不影响。
4. 创建 VPC 中的子网

   为便于对同一个 VPC 中不同的业务或者服务进行差异化管理，您可以按需[创建多个 VPC 子网](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_65)并关联指定的[路由表](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_68)，以控制 VPC 子网内网络流量的流向。
5. 为集群中的虚拟机配置 VPC 网络

   参考《SMTX OS 管理指南》或《SMTX ELF 管理指南》中的**管理虚拟机**章节，为集群中的虚拟机配置 VPC 网络。您可以按需将不同集群中的虚拟机规划至同一个 VPC 网络，以实现跨集群、机架、机房、数据中心构建 VPC。
6. 按需配置浮动 IP 或网关服务，使 VPC 中的虚拟机能够通过边缘网关或边缘网关组与外部网络通信。

   - [管理浮动 IP](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_71)
   - [设置路由网关](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_75)
   - [设置 NAT 网关](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_76)
   - [设置 L2 网关](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_77)
   - [设置 VPC 对等连接](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_05)
7. 为 VPC 配置安全服务，以保障 VPC 内各类业务的安全运行。

   - [设置 VPC 安全组](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_79)
   - [设置 VPC 安全策略](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_80)
   - [设置 VPC 隔离虚拟机](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_81)

## VPC 规划示例

# VPC 规划示例

## 开发/测试环境

创建用于日常开发或者测试使用的 VPC。

这种场景的主要特征如下：

- 允许从公司内网访问。
- 允许开发、测试虚拟机间互相访问。
- 允许开发、测试虚拟机主动访问外网。
- 不允许外部访问。

这类场景下，可以参考下图规划：

![](https://cdn.smartx.com/internal-docs/assets/83f40769/administration_guide_vpc_98_01.png)

在一个 VPC 中创建两类 VPC 子网，分别用于创建堡垒虚拟机以及开发和测试使用的虚拟机：

- **堡垒虚拟机**：用于安全访问管理。

  - 绑定浮动 IP，提供从公司内网登录 VPC 内虚拟机的能力。
  - 配置安全组、安全策略，实现访问控制。
  - 关联路由表，实现流量路径。
- **开发/测试虚拟机**：用于执行日常的开发、测试任务。

  - 配置 NAT 网关，实现公网访问的需求。
  - 配置安全组、安全策略，实现访问控制。
  - 关联路由表，实现流量路径。

## 生产环境

创建用于生产服务的 VPC。

这种场景的主要特征是：

- 允许从公司内网访问。
- 允许外部网络访问部分生产服务，比如：Web 服务。
- 部分生产服务只允许其依赖的生产服务访问，比如：DB 服务只允许 Web 服务访问。
- 不需要主动访问外网。

这类场景下，可以参考下图规划：

![](https://cdn.smartx.com/internal-docs/assets/83f40769/administration_guide_vpc_98_02.png)

在一个 VPC 中创建三类 VPC 子网，分别用于创建堡垒虚拟机、public 服务虚拟机、private 服务虚拟机。

- **堡垒虚拟机**：用于安全访问管理。

  - 绑定浮动 IP，提供从公司内网登录 VPC 内虚拟机的能力。
  - 配置安全组、安全策略，实现访问控制。
  - 关联路由表，实现流量路径。
- **public 服务虚拟机**：用来部署允许外部访问的服务。

  - 配置 NAT 网关，实现公网访问的需求。
  - 配置安全组、安全策略，实现访问控制。
  - 关联路由表，实现流量路径。
- **private 服务虚拟机**：用来部署不允许外部访问的服务。

  - 配置安全组、安全策略，实现访问控制。
  - 关联路由表，实现流量路径。

## 启用虚拟专有云网络

# 启用虚拟专有云网络

**前提条件**

- 确认已开启相应的防火墙端口，请参考《Everoute 安装与升级指南》的[规划端口](/everoute/3.5.0/er_installation_guide/installation_guide_08)章节。
- 规划虚拟专有云网络的 IP 地址，请参考《Everoute 安装与升级指南》的[规划虚拟专有云网络](/everoute/3.5.0/er_installation_guide/installation_guide_36)章节。
- Everoute Controller 节点数量为 `3 个`或 `5 个`。如不满足，请参考《Everoute 安装与升级指南》的[增加 Everoute Controller 数量](/everoute/3.5.0/er_installation_guide/installation_guide_31#%E5%A2%9E%E5%8A%A0-everoute-controller-%E6%95%B0%E9%87%8F)章节。
- 如当前 Everoute 服务已启用网络安全导流之外的其他功能，则 Everoute Controller 配置类型至少为`中配`。如不满足，请参考《Everoute 安装与升级指南》的[提高 Everoute Controller 配置](/everoute/3.5.0/er_installation_guide/installation_guide_31#%E6%8F%90%E9%AB%98-everoute-controller-%E9%85%8D%E7%BD%AE)章节。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**设置**，选择要启用虚拟专有云网络的 Everoute 服务。
2. 在**服务设置**里单击**基本信息**。
3. 在**虚拟专有云网络**功能右侧单击**启用功能**。
4. 进入**启用虚拟专有云网络功能**页面，参考《Everoute 安装与升级指南》的[配置虚拟专有云网络](/everoute/3.5.0/er_installation_guide/installation_guide_35)章节填写配置信息。
5. 填写配置信息后，单击**启用**。

## 查看 Everoute 服务概览

进入 CloudTower 的**网络与安全**界面，您可以在概览页查看当前所有已部署的 Everoute 服务的信息。

每个 Everoute 服务概览卡片包括如下内容：

- **服务名称**：卡片标题即为 Everoute 服务的名称。
- **版本**：展示当前 Everoute 服务的版本。当存在可升级的版本时，单击**可升级**，可进入**服务升级**页面，选择版本完成升级操作。

  > **说明**：
  >
  > - Controller 状态包括正常运行、异常、正在部署、正在卸载。
  > - 绿色表示正常状态的 Controller 个数，红色表示异常状态（包括异常、正在部署、正在卸载）的 Controller 个数，黄色表示 vCPU、内存、存储容量使用率大于 80% 的 Controller 个数。
- **虚拟专有云网络**：若当前 Everoute 服务已开启虚拟专有云网络功能，将展示已关联的集群数量，否则将隐藏该项。单击**虚拟专有云网络**可进入**关联集群**页面。

虚拟专有云网络概览包含如下卡片：

- **外部子网**：分别展示当前外部子网和外部子网组的总数。

## 设置 TEP IP 池

# 设置 TEP IP 池

TEP IP 池的剩余可用 IP 不足导致无法创建边缘网关、关联集群或为 LB 实例配置 TEP IP 时，您可以增加 TEP IP 池也可以扩容已有的 TEP IP 池。

**注意事项**

- 增加 TEP IP 池时，请确保所有 TEP IP 池可以互通。
- 已被使用的 TEP IP 池，只支持修改名称、子网掩码和网关，只允许扩容。
- 已被使用的 TEP IP 池不允许删除。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**设置**，选择更新 TEP IP 池的 Everoute 服务。
2. 在**服务设置**中单击**虚拟专有云网络**，在右侧页面单击**预留 IP** 页签。
3. 单击 TEP IP 池右侧的的**编辑**，在弹出的**编辑 TEP IP 池**的对话框中：

   - 单击 **+ 添加 TEP IP 池**，并设置新 TEP IP 池的名称、起始 IP、结束 IP、子网掩码和网关。
   - 编辑已有 TEP IP 池的起始 IP 、结束 IP、子网掩码和网关。
   - 单击 **x**，删除 TEP IP 池。
4. 单击**保存**。

## 管理关联集群 > 关联集群

# 关联集群

启用虚拟专有云网络后，需要关联 SMTX OS（ELF）集群或 SMTX ELF 集群，使集群上的虚拟机可以使用 VPC 网络。

一个 Everoute 服务可关联多个相同或不同 CPU 架构的集群，但一个集群仅能关联一个 Everoute 服务。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**设置**，选择待关联集群的 Everoute 服务。
2. 在**服务设置**中单击**虚拟专有云网络**，在右侧页面单击**关联集群**页签，单击 **+ 添加关联集群**。
3. 在弹出的**关联集群**对话框中选择待关联的**集群**、**虚拟分布式交换机**和虚拟专有云系统网络相关信息，包括 **TEP IP 池**、**VLAN ID** 和 **MTU**。

   - **集群**：仅允许关联 6.0.0 及以上版本的的 SMTX OS（ELF）集群和 SMTX ELF 集群。请确保集群中的每台主机的管理 IP 地址与 Everoute Controller 管理 IP 地址连通。
   - **虚拟分布式交换机**：可以选择已有的虚拟分布式交换机，也可以新建一个满足要求的虚拟分布式交换机。虚拟分布式交换机必须关联集群中每一台主机上至少一个网口，且没有关联开启 RDMA 的存储网络。
   - **TEP IP 池**：选择 TEP IP 池。请确保所选择的 TEP IP 池中可用的 TEP IP 数量多于所关联的集群中的主机数量。
   - **VLAN ID**：填写虚拟专有云系统网络的 VLAN ID。请确保当前 Everoute 服务中关联的所有集群的虚拟专有云系统网络互相连通，并且与边缘网关的 TEP 虚拟机网络互相连通。
   - **MTU**：在该集群上自动创建的虚拟专有云系统网络的 MTU。若虚拟机分布式交换机关联的物理网口 MTU 小于该值，将自动调高物理网口 MTU。
4. （可选）从所选择 TEP IP 池中，为所关联的集群中的每台主机分配 TEP IP，留空则由系统自动分配。您可以填写**起始 TEP IP**，系统将根据此 IP 地址为所有主机自动分配连续的 TEP IP 地址。
5. 单击**关联**。

   完成关联操作后，在**关联集群**页面可以查看上述配置信息以及所关联集群的 CPU 架构。

   当关联集群状态异常时，页面上方会出现异常提示，此时您可以在异常提示信息右侧单击**恢复**，以重新同步关联集群的状态。

## 管理边缘网关和边缘网关组

# 管理边缘网关和边缘网关组

如果虚拟专有云网络中的 VPC 需要与数据中心、公网、本地 VLAN 等外部网络通信，请在虚拟专有云网络中创建边缘网关或组建边缘网关组。

**使用场景**

- 当 VPC 与外部通信的流量较多，或需要对不同 VPC 的南北向流量进行完全隔离时，可部署多个边缘网关，对网关流量进行负载分担及隔离。您可以灵活选择每个 VPC 使用单独的边缘网关，或每个 VPC 子网使用单独的边缘网关。
- 如需进一步提升边缘网关的高可用性，可在两个集群或两个数据中心分别部署边缘网关，并组成边缘网关组。两个边缘网关形成主、备关系，当主边缘网关所在集群或所在数据中心整体故障后，可以由备边缘网关接管网关流量，以保证业务通信不受影响。

## 管理边缘网关和边缘网关组 > 管理边缘网关 > 创建边缘网关

# 创建边缘网关

一个 Everoute 服务中能创建多个边缘网关，多个边缘网关可位于相同或不同的集群。

每个边缘网关中默认包含两个边缘网关虚拟机，采用一主一备模式放置在不同主机。一个 Everoute 服务关联多个集群时，单个边缘网关的主备虚拟机只支持放置在同一个集群。

**前提条件**

边缘网关虚拟机默认开启 CPU 独占，请提前确认集群中至少存在两个主机能够为边缘网关虚拟机提供足够 vCPU 和内存。

**注意事项**

- 当多个边缘网关位于不同集群时，请确保集群之间的管理网络和 TEP 网络均保持连通。
- 如您后续计划将创建的边缘网关用于组建边缘网关组，则组内两个边缘网关的**配置类型**需保持一致。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**设置**，选择待创建边缘网关的 Everoute 服务。
2. 在**服务设置**中单击**虚拟专有云网络**，在右侧页面单击**边缘网关**页签，默认进入**边缘网关**页面。
3. 单击 **+ 创建边缘网关**，弹出**创建边缘网关**对话框。
4. 配置基本信息。填写**名称**、**描述**，选择**配置类型**后，单击**下一步**。

   边缘网关不同的配置规格，能支持的网关实例上限不同，可以根据下表选择所需边缘网关的配置。

   | 配置类型 | 边缘网关管理能力 | 边缘网关虚拟机占用的系统资源 |
   | --- | --- | --- |
   | **低配** | - 1000 个路由网关 - 500 个 NAT 网关 - 2000 个浮动 IP - 500 个 L2 网关 | - 2 vCPU - 2 GiB 内存 |
   | **中配** | - 2000 个路由网关 - 1000 个 NAT 网关 - 5000 个浮动 IP - 1000 个 L2 网关 | - 4 vCPU - 4 GiB 内存 |
   | **高配** | - 4000 个路由网关 - 2000 个 NAT 网关 - 10000 个浮动 IP - 3000 个 L2 网关 | - 8 vCPU - 8 GiB 内存 |
5. 配置网络。

   1. 选择部署边缘网关的**集群**。
   2. 选择边缘网关的**虚拟分布式交换机**。

      可以选择多个虚拟分布式交换机，后续可以在这些虚拟分布式交换机上配置外部子网。
   3. 批量设置边缘网关虚拟机**管理 IP 地址**、**子网掩码**和**网关**。

      填写**起始 IP 地址**后，系统将根据此 IP 地址为所有边缘网关虚拟机自动分配连续的管理 IP 地址。如留空，则需在下一步骤中手动填写每台边缘网关虚拟机的管理 IP 地址。
   4. 为每一个边缘网关虚拟机配置管理网络：

      - **虚拟机网络**：用于与集群的管理网络连通，需保证该虚拟机网络能够与 Everoute Controller 连通。
      - **IP 地址**：边缘网关虚拟机的管理 IP 地址。
   5. 为每一个边缘网关虚拟机配置 TEP 网络。

      - **虚拟机网络**：用于与虚拟专有云网络关联集群的主机的 TEP IP 连通。默认由系统自动创建一个新的 VLAN 网络，也可以选择已有的 VLAN 网络。
      - **TEP IP 池**：为边缘网关虚拟机选择一个 TEP IP 池。
      - **TEP IP 地址**：填写 TEP IP 地址，留空则由系统从所选择的 TEP IP 池中自动分配。
6. 单击**创建**。

## 管理边缘网关和边缘网关组 > 管理边缘网关组 > 创建边缘网关组

# 创建边缘网关组

您可以将两个**配置类型**相同的边缘网关关联为边缘网关组，并指定其中一个边缘网关作为主边缘网关、另一个作为备边缘网关。两个边缘网关可以位于不同的集群或位于同一集群的不同物理节点。

**前提条件**

已[创建所需边缘网关](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_50)，且边缘网关未关联外部子网和 L2 网关。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**设置**，选择待创建边缘网关组的 Everoute 服务。
2. 在**服务设置**中单击**虚拟专有云网络**，在右侧页面单击**边缘网关**页签，默认进入**边缘网关**页面。
3. 单击**边缘网关组**，进入边缘网关组页面。
4. 单击 **+ 创建边缘网关组**，填写边缘网关组的**名称**和**描述**。
5. 选择满足要求的**主边缘网关**和**备边缘网关**。

   创建后默认主边缘网关负责实际承载业务流量，备边缘网关则作为主网关的冗余，负责在主边缘网关发生故障时接管业务流量，以确保网络的高可用性。
6. 单击**创建**。

## 管理边缘网关和边缘网关组 > 管理边缘网关组 > 切换活动网关

# 切换活动网关

在边缘网关组中，活动网关负责承载业务流量，非活动网关不承载流量。当主、备边缘网关健康状态均为**正常**时，默认主边缘网关为**活动**网关。

- **自动切换**：当系统检测到主边缘网关健康状态为**异常**时，如果备边缘网关健康状态为**正常**，则系统自动将备边缘网关切换为**活动**网关，并将所有网关实例及其流量自动切换至备边缘网关。当主边缘网关恢复**正常**状态后，系统会自动切换活动网关至主边缘网关。
- **手动切换**：当系统未自动检测到边缘网关异常时，您也可以按需手动切换活动网关。

**风险提示**

切换活动网关时，使用边缘网关组的网关服务会出现短暂的网络中断，建议在业务空闲时或业务下线后谨慎操作。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**设置**，选择待切换边缘网关组活动网关的 Everoute 服务。

3. 单击**边缘网关组**，进入边缘网关组页面。
4. 单击某一边缘网关组右侧的 **...** > **切换活动网关**，切换主、备边缘网关的活动状态。
5. 单击**确认**。

## 管理外部子网和外部子网组

# 管理外部子网和外部子网组

创建外部子网或外部子网组，相当于为边缘网关虚拟机指定了数据平面与外部网络通信的接口，并可以为路由网关、NAT 网关和浮动 IP 服务分配可使用的地址块。

您可以在边缘网关中创建一个或多个外部子网，或在边缘网关组中创建外部子网组。外部子网组由主、备外部子网组成，分别属于主、备边缘网关，当主边缘网关故障时，由备边缘网关和备外部子网接管网关流量。

**使用场景**

- 使用外部子网时，可在边缘网关可连通外部环境的 VLAN 下创建外部子网，为所需的不同网关服务分别分配地址块。
- 使用外部子网组时：
  - 如主、备边缘网关属于同一集群，且关联了相同的虚拟分布式交换机，则可以选择主、备外部子网共用网络配置，即两个外部子网所属的虚拟机网络、CIDR 块以及为不同网关服务分配的地址块均完全相同。配置后当主、备边缘网关发生切换时，对外提供服务的 IP 地址保持不变。
  - 如主、备边缘网关属于不同集群，或关联的虚拟分布式交换机不同，则无法共用网络配置，需分别指定主、备外部子网所属的虚拟机网络、CIDR，并分别为不同的网关服务分配主、备地址块。配置后，发生主、备边缘网关切换时，对外提供服务的 IP 将从主外部子网切换至备外部子网。

3. 单击 **+ 创建外部子网**，根据如下参数说明完成配置。

   | 参数 | 描述 |
   | --- | --- |
   | 名称 | 外部子网的名称。 |
   | 描述（选填） | 外部子网的描述。 |
   | 属于边缘网关 | 外部子网所属的边缘网关。 |
   | 虚拟机网络 | 边缘网关虚拟机关联的虚拟机网络。 |
   | VPC 共享类型 | - **共享型**：表示该外部子网可以被所有 VPC 使用，无需关联。 - **非共享型**：表示为该外部子网关联一个 VPC，关联后外部子网只能被该 VPC 使用。可在创建时指定具体的 VPC，也可以留空，待使用时关联。如留空且并未关联 VPC，则效果等同于共享型。 |
   | CIDR 块 | 外部子网的地址块。 |
   | 网关 | 外部子网的网关。 |
   | 分配地址块 | 您可以为 **NAT 网关**、**路由网关**和**浮动 IP** 服务分配一个或多个可使用的地址块。 |

   创建外部子网后，在**外部子网** > **属于边缘网关**页面，除了上述创建时的配置信息，您还可以查看各种用途下已分配的地址块数量。

## 修改已有虚拟机的 MTU

下面以某 Linux 操作系统为例，介绍修改虚拟机网卡 MTU 的操作步骤。不同操作系统及版本对应的修改方式和命令可能有所不同，具体请参考相应操作系统的官方文档。

1. 使用命令 `netstat -i` 查看虚拟机网卡当前的 MTU。

   ```
   netstat -i
   Kernel Interface table
   Iface             MTU    RX-OK RX-ERR RX-DRP RX-OVR    TX-OK TX-ERR TX-DRP TX-OVR Flg
   ens4             1400 51878953      0 140518 0        400336      0      0      0 BMRU
   ```
2. 编辑 VPC 的 MTU 完成后，使用如下任意一种方式修改虚拟机网卡 MTU。

   - 重启虚拟机，以重新发送 DHCP 请求。
   - 使用 `service network restart` 命令重启网络，以重新发送 DHCP 请求。
   - 使用如下命令，手动发送 DHCP 请求。

     ```
     dhclient -x
     dhclient ens4
     ```
   - 使用 `ifconfig ens4 mtu VPC_MTU` 命令将网卡的 MTU 直接设置为修改后 VPC 的 MTU。其中，`VPC_MTU` 需替换为 VPC 实际设置的 MTU 值。
3. 使用命令 `netstat -i` 验证虚拟机网卡 MTU 修改结果。

## 管理 VPC 子网 > 创建 VPC 子网

# 创建 VPC 子网

一个 VPC 内可以创建多个 VPC 子网，同一 VPC 子网内可以通信，同一 VPC 内的 VPC 子网间默认可以通信。

**前提条件**

已[创建 VPC](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_60)。

**注意事项**

每个 VPC 子网必须关联一张路由表，默认关联 VPC 创建完成后自动生成的默认路由表。如需关联其他路由表，需先[创建路由表](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_68)，之后即可在创建 VPC 子网时选择该路由表。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击 **VPC 子网**。
2. 单击 **+ 创建 VPC 子网**，选择 VPC 子网所属的 Everoute 服务和 VPC。
3. 根据如下参数说明完成 VPC 子网配置。

   | 参数 | 描述 |
   | --- | --- |
   | 名称 | VPC 子网的名称，同一 VPC 内的 VPC 子网不可重名。 |
   | 描述（可选） | VPC 子网的描述信息，您可以按需填写。 |
   | CIDR 块 | 设置 VPC 子网的 CIDR 块，用于定义 VPC 子网的网络范围。  仅支持 IPv4 CIDR 块，格式示例为 `192.168.0.0/24`。  **注意**：该值不能与 VPC 内其他 VPC 子网以及内部服务子网重叠，且网络掩码长度必须介于 `/8` 和 `/30` 之间，推荐使用 `/24`。 |
   | 默认网关 | 设置 VPC 子网的默认网关，用于与其他 VPC 子网通信。 |
   | 分配 IP 池（可选） | 从 CIDR 块中设置一段用于 DHCP 服务的 IP 范围，分配 IP 池中的 IP 地址将被自动分配给关联该 VPC 子网的虚拟机网卡使用。留空则默认 CIDR 块内的所有 IP 均被用作分配 IP 池。  分配 IP 池之外的 IP 地址，可以被手动指定给虚拟机，且不会被 DHCP 服务使用。  **注意**：网络地址、广播地址不会用于 VPC 子网内虚拟机网卡的 IP 分配。例如 192.168.0.0/24，其中 192.168.0.0、192.168.0.255 分别为网络地址和广播地址，不会被分配给 VPC 子网中的虚拟机网卡。 |
   | 关联路由表 | 为 VPC 子网选择一个路由表，路由表中的路由规则用于决定 VPC 子网内网络流量的流向。  默认关联默认路由表 `default`，您也可以选择已创建的其他路由表。 |
4. 单击 **+ 添加 VPC 子网**，可在 VPC 内批量创建多个 VPC 子网。
5. 单击**创建**。

   创建完成后，您可以在 **VPC 子网**列表中查看已创建的所有 VPC 子网信息。

   - 单击**所属 VPC** 的名称链接，可跳转至 VPC 详情页面。
   - 单击某一 VPC 子网可查看该 VPC 子网的详情，以及关联的虚拟机信息。单击**关联路由表**的名称链接，可跳转至路由表详情页面。
   - 单击 **VPC 子网**列表右上方的导出报表图标，可将当前的 VPC 子网列表导出成 `.csv` 文件至本地保存。

## 管理路由表

# 管理路由表

路由表由一系列路由规则组成，用于控制 VPC 子网内网络流量的流向。您可以按需将去往目标地址的流量路由至指定的下一跳网关服务，支持路由网关和 NAT 网关。

一个 VPC 子网只能关联一个路由表并使用该路由表中的路由规则，但一个路由表可以被多个 VPC 子网关联。

- **默认路由表**：创建 VPC 时，系统会自动为其生成一个默认路由表 `default`。创建 VPC 子网时，将默认为该子网关联默认路由表。该路由表中路由规则默认为空，您可以按需添加、删除或修改路由规则，但不能删除默认路由表。
- **自定义路由表**：当某些 VPC 子网需要使用不同于默认路由表的路由规则时，您可以为其单独创建一个新的路由表并配置路由规则，然后将该路由表与 VPC 子网关联。新建的自定义路由表与默认路由表以及其他自定义路由表间相互独立。自定义的路由表支持删除。

**注意事项**

VPC 内网络流量匹配的优先级，按照从高到低的顺序依次为：**路由规则** > **浮动 IP** > **默认路由（0.0.0.0）**。

## 管理路由表 > 创建路由表

# 创建路由表

**前提条件**

- 已[创建 VPC](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_60)。
- 为路由表添加路由规则前，请先[创建路由网关](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_105)或[创建 NAT 网关](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_107)。

**注意事项**

- 不同路由规则中的目标地址不能完全相同。
- 当不同路由规则中的目标地址存在重叠时，流量优先以最长前缀匹配。
- 当路由规则中的目标地址和当前 VPC 的子网重叠时，流量将转发至 VPC 子网。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**路由表**。
2. 单击 **+ 创建路由表**，设置路由表的名称、描述，并选择路由表所属的 Everoute 服务和所属 VPC。
3. （可选）单击 **+ 添加路由规则**，根据如下参数说明完成路由规则配置。

   - 单击 **+ 添加路由规则**，继续添加路由规则。
   - 单击**目标地址（CIDR）** 右侧的 **X**，可移除路由规则。

   | 参数 | 描述 |
   | --- | --- |
   | 下一跳类型 | 选择下一跳类型，支持选择 **NAT 网关**、**路由网关**或 **VPC 对等连接**。 |
   | 下一跳 | 选择已配置的任一 NAT 网关、路由网关或 VPC 对等连接。 |
   | 目标地址（CIDR 块） | 设置路由的目标地址。 - 若下一跳类型为 **NAT 网关**或**路由网关**，请填写 IPv4 CIDR 块。 - 若下一跳类型为 **VPC 对等连接**，请选择对端 VPC 的一个 VPC 子网。 |
4. 单击**创建**。

   创建完成后，您可以在**路由表**列表中查看已创建的所有路由表信息。

   - 单击**所属 VPC** 的名称链接，可跳转至 VPC 详情页面。
   - 单击某一路由表可查看该路由表的详情，以及已关联的 VPC 子网。单击已关联的 VPC 子网名称链接，可跳转至 VPC 子网详情页面。
   - 单击**路由表**列表右上方的导出报表图标，可将当前的路由表列表导出成 `.csv` 文件至本地保存。

## 管理浮动 IP

# 管理浮动 IP

浮动 IP 即从外部子网或外部子网组的浮动 IP 地址池中分配的 IP，是一种 1:1 NAT。虚拟机可以动态关联浮动 IP，之后即可通过浮动 IP 访问外部网络。

**使用场景**

- 如果 VPC 网络内的特定虚拟机需要使用固定 IP 被外部网络访问，则可以为该虚拟机配置浮动 IP。
- 如果需要保障特定对外业务的高可用性，则可以使用主备虚拟机运行相同业务，并为其中的主虚拟机配置浮动 IP，令外部网络通过浮动 IP 访问该业务。当主虚拟机发生故障，可以快速将浮动 IP 关联至备用虚拟机，从而确保外部网络对业务的访问不中断。

**注意事项**

VPC 内网络流量匹配的优先级，按照从高到低的顺序依次为：**路由规则** > **浮动 IP** > **默认路由（0.0.0.0）**。

---

## 管理浮动 IP > 分配浮动 IP

# 分配浮动 IP

为了有节制地合理使用 IP 资源，在使用浮动 IP 前需先分配浮动 IP，虚拟机只能与已分配的浮动 IP 进行关联。

**前提条件**

- 已[创建 VPC](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_60)。
- 已[创建关联至 VPC 的外部子网](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_56)，且该外部子网已分配浮动 IP 地址块；或已[创建关联至 VPC 的外部子网组](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_35)，且该外部子网组已分配浮动 IP 地址块。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**浮动 IP**。
2. 单击 **+ 分配浮动 IP**，选择浮动 IP 所属的 Everoute 服务和所属 VPC。
3. 选择**所属边缘网关类型**。

   - 浮动 IP 属于**边缘网关**时，需选择**所属边缘网关**及相应的**外部子网**。
   - 浮动 IP 属于**边缘网关组**时，需选择**所属边缘网关组**及相应的**外部子网组**。
4. 设置浮动 IP 分配方式。

   - 浮动 IP 属于边缘网关；或浮动 IP 属于边缘网关组，且外部子网组的网络配置类型为**共用型**：

     | 参数 | 描述 |
     | --- | --- |
     | 自动分配 | 需设置**分配 IP 数**，即该 VPC 所选外部子网或外部子网组中可用于自动分配浮动 IP 的地址数量。  默认为 `1`，最大不能超过外部子网或外部子网组浮动 IP 地址块中剩余的可用 IP 数量。 |
     | 手动分配 | 需从外部子网或外部子网组浮动 IP 地址块中至少指定一个浮动 IP 地址。  单击 **+ 添加 IP**，可添加多个 IP 地址。  IP 地址需包含在外部子网或外部子网组浮动 IP 地址块中，未被分配且不能重复。 |
   - 浮动 IP 属于边缘网关组，且外部子网组的网络配置类型为**非共用型**：

     | 参数 | 描述 |
     | --- | --- |
     | 自动分配 | 需设置**分配 IP 数**，即该 VPC 所选外部子网组的主、备外部子网中分别可用于分配浮动 IP 的地址数量。  默认为 `1`，最大不能超过主、备外部子网浮动 IP 地址块中剩余的可用 IP 数量。 |
     | 手动分配 | 需同时指定主 IP 和备用 IP： - **IP**：从外部子网组的主外部子网浮动 IP 地址块中至少指定一个主浮动 IP 地址。 - **备用 IP**：从外部子网组的备外部子网浮动 IP 地址块中至少指定一个备用浮动 IP 地址。 单击 **+ 添加 IP**，可添加多个主、备 IP 地址。IP 地址需分别包含在主、备外部子网浮动 IP 地址块中，未被分配且不能重复。 |
5. 单击**分配**。

   分配完成后，您可以分别查看**属于边缘网关**和**属于边缘网关组**的浮动 IP 列表。

   列表中展示已分配的所有浮动 IP 信息以及该浮动 IP 的分配对象。其中，分配对象将包括**虚拟机名称**、**虚拟机网卡名称**和**虚拟机网卡从 VPC 子网分配的 IP 地址**，单击**虚拟机名称**链接可跳转至虚拟机详情页面。

   单击**浮动 IP** 列表右上方的导出报表图标，可分别将**属于边缘网关**和**属于边缘网关组**的浮动 IP 列表导出成 `.csv` 文件至本地保存。

## 管理浮动 IP > 释放浮动 IP

# 释放浮动 IP

当已分配的浮动 IP 未被虚拟机使用时，您可以按需释放浮动 IP。如有备用 IP，也会同时释放。释放后 IP 将回收至外部子网或外部子网组的浮动 IP 地址块中。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**浮动 IP**。
2. 单击**属于边缘网关**或**属于边缘网关组**页签，找到待释放的浮动 IP。
3. 单击 **...** 后选择**释放浮动 IP**。
4. 单击**释放**。

## 管理网关服务 > 设置路由网关

# 设置路由网关

路由网关提供了 VPC 子网与外部网络三层互通的功能。配置路由网关后，VPC 内的虚拟机可直接使用私有地址与外部进行通信。

**使用场景**

如果 VPC 内的虚拟机需要访问某个部署在本地数据中心上的服务，且服务需要记录访问来源的真实 IP，则可以使用路由网关，使 VPC 内部使用私有地址与外部通信。

---

## 管理网关服务 > 设置路由网关 > 创建路由网关

# 创建路由网关

**前提条件**

- 已[创建 VPC](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_60) 和 [VPC 子网](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_65)。
- 已[创建关联至 VPC 的外部子网](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_56)，且该外部子网已分配路由网关地址块；或已[创建关联至 VPC 的外部子网组](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_35)，且该外部子网组已分配路由网关地址块。

**注意事项**

创建路由网关后，还需完成以下两个步骤，才能使 VPC 子网内的虚拟机通过路由网关与外部网络通信。

1. 在相应 VPC 子网关联的路由表中添加路由规则，目标地址填写为外部地址，将下一跳指定为该路由网关。
2. 在物理网络中配置相应的路由规则，目标地址填写为 VPC 子网，将下一跳指定为该 VPC 路由网关的外部接口。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**网关服务**。
2. 单击 **+ 创建 > 路由网关**。
3. 设置路由网关的**名称**、**所属 Everoute 服务**和**所属 VPC**。
4. 选择**所属边缘网关类型**，并根据类型设置如下信息。

   - 路由网关属于边缘网关：

     | 参数 | 描述 |
     | --- | --- |
     | 所属边缘网关 | 选择路由网关所属的边缘网关。 |
     | 外部子网 | 选择所属 VPC 中已关联的外部子网，且该外部子网需已分配路由网关地址块。 |
     | 外部接口（可选） | 从外部子网路由网关地址块中选择一个未使用的 IP 地址作为路由网关的外部接口，留空则将从路由网关地址块中自动分配。 |
     | 关联 VPC 子网 | 选择所属 VPC 内的一个或多个 VPC 子网。关联 VPC 子网后，该子网内的虚拟机即可通过路由网关与外部通信。 |
   - 路由网关属于边缘网关组：

     | 参数 | 描述 |
     | --- | --- |
     | 所属边缘网关组 | 选择路由网关所属的边缘网关组。 |
     | 外部子网组 | 选择所属 VPC 中已关联的外部子网组，且该外部子网组需已分配路由网关地址块。 |
     | 外部接口和备用外部接口（可选） | - 外部子网组的网络配置类型为**共用型**：从外部子网组的路由网关地址块中选择一个未使用的 IP 地址作为路由网关的外部接口，留空则将从路由网关地址块中自动分配。 - 外部子网组的网络配置类型为**非共用型**：从外部子网组的主外部子网路由网关地址块中选择一个未使用的 IP 地址作为路由网关的外部接口，从备外部子网路由网关地址块中选择一个未使用的 IP 地址作为路由网关的备用外部接口。留空则将从相应路由网关主、备地址块中自动分配。 |
     | 关联 VPC 子网 | 选择所属 VPC 内的一个或多个 VPC 子网。关联 VPC 子网后，该子网内的虚拟机即可通过路由网关与外部通信。 |
5. 设置外部子网与外部网络通信的路由规则。单击 **+ 添加规则**，可添加多条路由规则。

   - 路由网关属于边缘网关；或路由网关属于边缘网关组，且外部子网组的网络配置类型为**共用型**：

     | 参数 | 描述 |
     | --- | --- |
     | 目标地址（CIDR 块） | 设置目标地址，格式为 IP 或 CIDR 块。 |
     | 下一跳地址 | 从外部子网或外部子网组路由网关地址块中选择一个未使用的 IP 作为下一跳地址，留空则自动分配外部子网或外部子网组的默认网关作为下一跳地址。 |
   - 路由网关属于边缘网关组且外部子网组的网络配置类型为**非共用型**：

     | 参数 | 描述 |
     | --- | --- |
     | 目标地址（CIDR 块） | 设置目标地址，格式为 IP 或 CIDR 块。 |
     | 下一跳地址 | 从外部子网组的主外部子网路由网关地址块中选择一个未使用的 IP 作为下一跳地址，留空则自动分配主外部子网的默认网关作为下一跳地址。 |
     | 备用下一跳地址 | 从外部子网组的备外部子网路由网关地址块中选择一个未使用的 IP 作为下一跳地址，留空则自动分配备外部子网的默认网关作为下一跳地址。 |
6. 单击**创建**。

   创建完成后，您可以分别查看已创建的所有**属于边缘网关**和**属于边缘网关组**的路由网关信息。

   - 单击某一路由网关可查看该路由网关的详情，以及该路由网关关联的路由表信息。
   - 单击**所属 VPC**、**所属边缘网关**、**所属边缘网关组**、**外部子网**、**外部子网组**、**关联 VPC 子网**、**关联路由表**的名称链接，可跳转至相应功能的详情页面。
   - 单击**路由网关**列表右上方的导出报表图标，可分别将**属于边缘网关**和**属于边缘网关组**的路由网关列表导出成 `.csv` 文件至本地保存。

## 管理网关服务 > 设置 NAT 网关

# 设置 NAT 网关

NAT 网关提供了 VPC 内的虚拟机共享 NAT IP 与外部网络通信的功能。配置 NAT 网关后，VPC 子网中的所有虚拟机可通过 NAT 网关将虚拟机的私网地址转换为外部子网的地址。

**使用场景**

- VPC 内的虚拟机承载了 Web 服务，则可以配置 NAT 网关，使其能够被公网访问。
- VPC 内的虚拟机需要访问本地数据中心的数据库，可以配置 NAT 网关，使 VPC 内部共享转换地址与数据库通信。

---

## 管理网关服务 > 设置 NAT 网关 > 创建 NAT 网关

# 创建 NAT 网关

**前提条件**

- 已[创建 VPC](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_60) 和 [VPC 子网](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_65)。
- 已[创建关联至 VPC 的外部子网](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_56)，且该外部子网已分配 NAT 网关地址块；或已[创建关联至 VPC 的外部子网组](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_35)，且该外部子网组已分配 NAT 网关地址块。

**注意事项**

创建 NAT 网关后，还需在相应 VPC 子网关联的路由表中添加路由规则，目标地址填写为外部地址，将下一跳指定为该 NAT 网关，之后 VPC 子网内的虚拟机即可通过 NAT 网关与外部通信。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**网关服务**。
2. 单击 **+ 创建 > NAT 网关**。
3. 设置 NAT 网关的**名称**、**所属 Everoute 服务**和**所属 VPC**。
4. 选择**所属边缘网关类型**，并根据类型设置如下信息。

   - NAT 网关属于边缘网关：

     | 参数 | 描述 |
     | --- | --- |
     | 所属边缘网关 | 选择 NAT 网关所属的边缘网关。 |
     | 外部子网 | 选择所属 VPC 中已关联的外部子网，且该外部子网需已分配 NAT 网关地址块。 |
     | 转换地址（可选） | 从外部子网 NAT 网关地址块中选择一个未使用的 IP 地址作为 NAT 网关的转换地址，留空则将从 NAT 网关地址块中自动分配。 |
   - NAT 网关属于边缘网关组：

     | 参数 | 描述 |
     | --- | --- |
     | 所属边缘网关组 | 选择 NAT 网关所属的边缘网关组。 |
     | 外部子网组 | 选择所属 VPC 中已关联的外部子网组，且该外部子网组需已分配 NAT 网关地址块。 |
     | 转换地址和备用转换地址（可选） | - 外部子网组的网络配置类型为**共用型**：从外部子网组的 NAT 网关地址块中选择一个未使用的 IP 地址作为 NAT 网关的转换地址，留空则将从 NAT 网关地址块中自动分配。 - 外部子网组的网络配置类型为**非共用型**：从外部子网组的主外部子网 NAT 网关地址块中选择一个未使用的 IP 地址作为 NAT 网关的转换地址，从备外部子网 NAT 网关地址块中选择一个未使用的 IP 地址作为 NAT 网关的备用转换地址。留空则将从相应 NAT 网关主、备地址块中自动分配。 |
5. 设置至少一个转换规则，您可以按需启用 SNAT 或 DNAT 转换。

   - **SNAT**：源地址转换。当 VPC 内的虚拟机需要访问外部网络时，内部地址发起主动连接，SNAT 会将虚拟机的内部私有 IP 地址转换为**转换地址**，用于与外部网络通信。主要用于 VPC 内共享 IP 地址访问外部网络。
   - **DNAT**：目标地址转换。当 VPC 内的虚拟机需要对外提供服务时，外部地址发起主动连接，DNAT 会将外部地址转换为**转换地址**，用于与内部虚拟机通信。主要用于内部共享 IP 对外提供业务服务。

     启用 DNAT 后，需根据如下参数说明设置一条或多条 DNAT 规则。

     | 参数 | 描述 |
     | --- | --- |
     | 内部目标地址 | 设置 VPC 内部需要提供业务服务的虚拟机的 IP 地址，或已在 LB 中使用的 VIP 地址或 LIP 地址。 |
     | 内部目标地址端口号 | 设置内部目标地址的端口号，支持 1 ～ 65535。 |
     | 转换地址端口号 | 设置转换地址的端口号，支持 1 ～ 32768。 |
6. 单击**创建**。

   创建完成后，您可以分别查看已创建的所有**属于边缘网关**和**属于边缘网关组**的 NAT 网关信息。

   - 单击某一 NAT 网关可查看该 NAT 网关的详情，以及该 NAT 网关关联的路由表信息。
   - 单击**所属 VPC**、**所属边缘网关**、**所属边缘网关组**、**外部子网**、**外部子网组**、**关联路由表**的名称链接，可跳转至相应功能的详情页面。
   - 单击 **NAT 网关**列表右上方的导出报表图标，可分别将**属于边缘网关**和**属于边缘网关组**的 NAT 网关列表导出成 `.csv` 文件至本地保存。

## 管理网关服务 > 设置 L2 网关

# 设置 L2 网关

L2 网关提供了 VPC 子网与外部网络二层互通的功能。配置 L2 网关后，可以连接传统 VLAN 网络中的子网与 VPC 中的子网，使得二者逻辑上处于一个二层子网。

**使用场景**

- 如果需要将虚拟机从物理网络迁至 VPC，或从 VPC 迁至物理网络，则可以先创建一个 L2 网关连接两个子网，使两方子网上的虚拟机能够在迁移期间保持通信，再参考《SMTX OS 管理指南》或《SMTX ELF 管理指南》中的**迁移虚拟机**章节进行操作。
- 如果需要允许 VPC 子网内任何地址无障碍访问物理数据库服务器，则可以创建一个 L2 网关使其与物理网络连通。

---

## 管理网关服务 > 设置 L2 网关 > 创建 L2 网关

# 创建 L2 网关

**前提条件**

- 已[创建 VPC](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_60) 和 [VPC 子网](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_65)。
- 已[创建边缘网关](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_50)或[创建边缘网关组](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_38)。

**注意事项**

- 同一个 VPC 子网只能关联一个 L2 网关。
- VPC 子网的子网范围与虚拟机网络的子网范围必须相同，且 VPC 子网与外部网络中不能使用相同的 IP。
- 同一个虚拟机网络关联的多个 VPC 之间的 VPC 子网地址不能重叠。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**网关服务**。
2. 单击 **+ 创建 > L2 网关**。
3. 设置 L2 网关的**名称**和**所属 Everoute 服务**。
4. 选择**所属边缘网关类型**。

   - L2 网关属于**边缘网关**时，需选择**所属边缘网关**。
   - L2 网关属于**边缘网关组**时，需选择**所属边缘网关组**。
5. 设置 L2 网关的**所属 VPC**，并选择所属 VPC 内的一个或多个 **VPC 子网**。关联 VPC 子网后，该子网内的虚拟机即可通过 L2 网关与外部网络建立二层通信。
6. 选择**虚拟机网络**。

   - L2 网关属于**边缘网关**时，选择一个当前 Everoute 服务的边缘网关已关联的虚拟分布式交换机内的 VLAN 网络（Access 类型）。
   - L2 网关属于**边缘网关组**时，选择一个当前 Everoute 服务的边缘网关组中，主、备边缘网关均已关联的虚拟分布式交换机内的 VLAN 网络（Access 类型）。
7. 单击**创建**。

   创建完成后，您可以分别查看已创建的所有**属于边缘网关**和**属于边缘网关组**的 L2 网关信息。

   - 单击某一 L2 网关可查看该 L2 网关的详情。
   - 单击**所属 VPC**、**所属边缘网关**、**所属边缘网关组**、**VPC 子网**和**虚拟机网络**的名称链接，可跳转至相应功能的详情页面。
   - 单击 **L2 网关**列表右上方的导出报表图标，可分别将**属于边缘网关**和**属于边缘网关组**的 L2 网关列表导出成 `.csv` 文件至本地保存。

## 管理网关服务 > 设置 VPC 对等连接

# 设置 VPC 对等连接

VPC 对等连接提供了两个 VPC 子网之间三层互通的功能。配置 VPC 对等连接后，两个 VPC 可直接使用私有地址相互访问。

**使用场景**

- 如果需要在两个不同的 VPC 之间共享资源，例如跨 VPC 的应用程序或数据库，则可以使用 VPC 对等连接实现安全的私有网络通信。
- 如果一个 VPC 中的虚拟机需要访问另一个 VPC 中的服务或应用，但不希望通过公网访问，可以使用 VPC 对等连接实现私有地址的直接通信。

**注意事项**

- 两个 VPC 必须属于同一个 Everoute 服务。
- 对于建立了对等连接的两个 VPC，若两者的 VPC 子网 IP 有重叠，则重叠 IP 之间的通信将优先在同一个 VPC 内完成，不使用对等连接。
- 当您需要使用 VPC 对等连接时，请先创建 VPC 对等连接，然后分别在两个对应的 VPC 子网关联的路由表中添加路由规则，将下一跳指定为该 VPC 对等连接，目标地址为对端 VPC 的 VPC 子网，之后两个 VPC 即可互相通信。

# 创建 VPC 对等连接

**前提条件**

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**网关服务**。
2. 单击 **+ 创建** > **VPC 对等连接**。
3. 根据如下参数说明，设置 VPC 对等连接的基本信息。

   | 参数 | 描述 |
   | --- | --- |
   | 名称 | VPC 对等连接的名称。 |
   | 所属 Everoute 服务 | VPC 对等连接所属的 Everoute 服务。  仅支持选择已启用 VPC 功能的 Everoute 服务。 |
   | VPC 1 | 选择当前 Everoute 服务内的一个 VPC 作为对等连接的其中一端。 |
   | VPC 2 | 选择当前 Everoute 服务内的另一个 VPC 作为对等连接的另一端。 |
4. 单击**创建**。

   创建完成后，您可以在 **VPC 对等连接**列表中查看已创建的所有 VPC 对等连接信息。

   - 单击某一 VPC 对等连接可查看该 VPC 对等连接的详情，以及该 VPC 对等连接关联的路由表信息。
   - 单击 **VPC 1**、**VPC 2**的名称链接，可跳转至相应 VPC 的详情页面。
   - 单击 **VPC 对等连接**列表右上方的导出报表图标，可将当前的 VPC 对等连接列表导出成 `.csv` 文件至本地保存。

## 管理安全服务

# 管理安全服务

安全服务用于 VPC 内虚拟机的访问控制。VPC 支持设置安全策略和虚拟机隔离两种访问控制策略，虚拟机隔离策略的优先级高于安全策略。

- **安全策略**：可为包含在指定 VPC 安全组内的虚拟机设置出、入流量白名单。
- **虚拟机隔离策略**：可在 VPC 内有虚拟机中毒或需要进行调试时，禁止指定虚拟机接受或发出任意流量，或者为指定虚拟机设置出、入流量白名单。

---

## 管理安全服务 > 设置 VPC 安全组

# 设置 VPC 安全组

VPC 安全组是一系列虚拟机的集合。如需对特定业务的虚拟机应用相同的安全策略，可以将其添加至同一个安全组中，并在各类所需策略中使用安全组作为策略对象。

VPC 安全组仅针对关联了该 VPC 网络的虚拟机有效。一个虚拟机可以加入一个或多个安全组，一个安全组也可以包含多个虚拟机。

- **默认安全组**：创建 VPC 时，系统会自动为其生成一个默认安全组 `All`，包含 VPC 内的所有虚拟机，且随着 VPC 内虚拟机的增减动态更新。默认安全组可用于配置 VPC 全局的安全策略，不支持编辑或删除。
- **自定义安全组**：如需对特定业务的虚拟机应用相同的安全策略，可将其添加至同一个自定义安全组中，并在相应安全策略中使用该安全组作为策略对象。

---

## 管理安全服务 > 设置 VPC 安全组 > 创建 VPC 安全组

# 创建 VPC 安全组

安全组的虚拟机成员包括如下两类：

- **动态成员**：所有由标签组筛选出的关联了所属 VPC 的虚拟机。通过标签组筛选虚拟机时，需提前参考《CloudTower 使用指南》的**管理标签**章节为虚拟机关联标签。
- **静态成员**：所有手动选择的关联了所属 VPC 的虚拟机。

**前提条件**

已[创建 VPC](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_60) 且已存在使用该 VPC 网络的虚拟机。

**注意事项**

- 关联静态成员，实际上关联了所选虚拟机的 UUID。当虚拟机由于跨集群分段迁移或跨集群冷迁移导致虚拟机 UUID 发生变化时，需重新将其添加为安全组的静态成员。建议优先使用动态成员。
- 如果虚拟机删除了所有关联该 VPC 的网卡，则安全组的静态成员中将自动删除该虚拟机。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**安全服务**。
2. 单击 **+ 操作 > 创建安全组**。
3. 根据如下参数说明，设置安全组的基本信息。

   | 参数 | 描述 |
   | --- | --- |
   | 名称 | 安全组的名称，同一 VPC 内不允许重名。 |
   | 描述（可选） | 安全组的描述信息，您可以按需填写。 |
   | 所属 Everoute 服务 | 安全组所属的 Everoute 服务。  仅支持选择已启用 VPC 功能的 Everoute 服务。 |
   | 所属 VPC | 安全组所属的 VPC。  仅支持选择当前 Everoute 服务中已创建的 VPC。 |
4. 设置安全组的虚拟机成员。

   - **设置动态成员**：即通过标签选择虚拟机。

     - 单击 **+ 添加标签组**，选择要添加的标签。单击 **+** 可添加多个标签。

       添加完成后您可以查看当前标签组筛选出的虚拟机总数，同一个标签组筛选出的虚拟机为关联了该标签组中所有标签的虚拟机。
     - 单击 **X** 可移除标签或移除标签组。
     - 若要添加多个标签组，可再次单击 **+ 添加标签组**。
   - **设置静态成员**：支持选择关联了所属 VPC 的虚拟机。您可以输入虚拟机名称快速搜索，并选择要添加的一个或多个虚拟机。
5. 单击**创建**。

   创建完成后，您可以在 **VPC 安全组**列表中可以查看所有安全组的信息。

   - 单击某一 VPC 安全组可查看该安全组的详情、该安全组应用的安全策略、以及该安全组处于隔离中的虚拟机信息。
   - 单击**所属 VPC**的名称链接，可跳转至 VPC 详情页面。
   - 单击 **VPC 安全组**列表右上方的导出报表图标，可将当前的 VPC 安全组列表导出成 `.csv` 文件至本地保存。

## 管理安全服务 > 设置 VPC 安全策略

# 设置 VPC 安全策略

VPC 安全策略以包含在对应安全组中的虚拟机为策略的目标对象，控制目标对象在出方向和入方向上的流量。

VPC 安全策略仅针对关联当前 VPC 的网卡有效。若一台虚拟机同时关联了多个 VPC，则关联其他 VPC 的网卡不受当前 VPC 安全策略的管理。

**安全策略规则说明**

- 当 VPC 内部的虚拟机未加入任何安全策略的策略对象时，虚拟机默认能够与 VPC 外部通信。当虚拟机加入任一安全策略时，则只能与设置的出入流量白名单通信。
- 当虚拟机应用了多条 VPC 安全策略时，多条策略取并集，共同控制虚拟机流量，即只要有一条策略允许流量通过该虚拟机，流量即被允许通过。
- 若 VPC 安全策略的出、入流量任意一侧选择了**全允许**通过，或任意一侧白名单中存在某一策略对象中的虚拟机，则即使在该策略对象组内配置了**不允许策略对象互相通信**，该虚拟机与组内的其他虚拟机仍然可以单方向或双方向通信。

---

## 管理安全服务 > 设置 VPC 安全策略 > 创建 VPC 安全策略

# 创建 VPC 安全策略

**前提条件**

已[创建 VPC](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_60)。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**安全服务**。
2. 单击 **+ 操作 > 创建安全策略**。
3. 根据如下参数说明，设置安全策略的基本信息，完成后单击**下一步**。

   | 参数 | 描述 |
   | --- | --- |
   | 名称 | 安全策略的名称，同一 VPC 内不允许重名。 |
   | 描述（可选） | 安全策略的描述信息，您可以按需填写。 |
   | 所属 Everoute 服务 | 安全策略所属的 Everoute 服务。  仅支持选择已启用 VPC 功能的 Everoute 服务。 |
   | 所属 VPC | 安全策略所属的 VPC。  仅支持选择当前 Everoute 服务中已创建的 VPC。 |
4. 添加策略对象，即通过安全组添加要应用安全策略的虚拟机。

   1. 单击 **+ 添加策略对象**。
   2. 搜索并选择已创建的 VPC 安全组，您也可以单击 **+ 新建安全组**，快速[创建 VPC 安全组](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_111)。
   3. 设置所选安全组内的虚拟机之间是否允许互相通信。
   4. 单击**添加**。

      添加完成后，您可以按需继续添加多个策略对象。多个策略对象之间为**逻辑或**的关系，即包含在任一个策略对象中的虚拟机都将作为此安全策略的策略对象，并应用相关的安全规则。

      单击已添加的策略对象右侧的 **...**，可以选择编辑或删除该策略对象。
5. 单击**入流量**后的下拉框，设置策略对象的入流量。

   - **全允许**：允许策略对象接收所有入流量。
   - **仅白名单**：仅允许策略对象接收来自白名单的入流量。

     单击 **+ 添加白名单**，在下拉框中可选择添加白名单的方式。

     > **说明**：
     >
     > - 支持同时通过标签、安全组和 IP 地址的方式添加多组白名单，例如一个白名单通过安全组添加，另一个白名单通过标签添加。
     > - 多组白名单之间为**逻辑或**的关系，即包含在任一组白名单中的虚拟机或 IP 地址都可应用相关的安全规则，与策略对象通信。
     > - 白名单留空则表示禁止全部入流量。

     - **按标签添加**：通过标签筛选出可访问策略对象的虚拟机，请提前参考《CloudTower 使用指南》的**管理标签**章节为虚拟机关联标签。选择多个标签时，虚拟机的筛选逻辑为**逻辑与**，即筛选出的虚拟机为同时关联了所有标签的虚拟机。您还可以指定允许的协议及端口，若无需限制协议则可选择任意协议。
     - **按安全组添加**：选择已创建的可访问策略对象的 VPC 安全组，也可以直接[创建 VPC 安全组](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_111)。您还可以指定允许的协议及端口，若无需限制协议则可选择任意协议。
     - **IP 地址**：输入 IP 地址或 CIDR 块指定白名单。输入 CIDR 块后，还可以单击输入框右侧的 **...**，手动从白名单中排除特定 IP 地址或 CIDR 块，并可指定允许的协议和端口，如无需限制协议可选择任意协议。

     单击已添加的白名单右侧的 **...**，可以选择编辑或删除该白名单。
6. 参考上一步的操作说明，设置策略对象的出流量，完成后单击**下一步**。
7. 设置策略状态。

   若启用**策略状态**，则安全策略在创建后立即生效；若不启用**策略状态**，则安全策略创建后将无法立即生效，后续您可通过编辑策略状态，使策略生效。
8. 单击**创建**。

   创建完成后，您可以在 **VPC 安全策略**列表中可以查看所有安全策略的信息。

   - 单击某一 VPC 安全策略的名称链接，可查看该安全策略的详情。
   - 单击 **Everoute 服务**的名称链接，可查看该 Everoute 服务的基本信息。
   - 单击 **VPC 安全策略**列表右上方的导出报表图标，可将当前的 VPC 安全策略列表导出成 `.csv` 文件至本地保存。

## 管理安全服务 > 设置 VPC 安全策略 > 编辑或删除 VPC 安全策略

# 编辑或删除 VPC 安全策略

**注意事项**

不支持编辑安全策略的**所属 Everoute 服务**和**所属 VPC**。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**安全服务**，默认进入 **VPC 安全组**页面。
2. 单击 **VPC 安全策略**页签，在列表中找到待编辑的安全策略，单击 **...** 后按需编辑如下内容。

   - **编辑策略**：编辑安全策略的策略对象以及出入流量白名单。
   - **编辑策略状态**：编辑安全策略的生效模式，按需启用或禁用策略。
   - **编辑名称和描述**：编辑安全策略的名称和描述信息。

   您也可以在列表中单击某个安全策略，在弹出的详情界面中单击**编辑策略**，或单击 **...** 后选择**编辑策略状态**和**编辑名称和描述**。
3. 在列表中找到待删除的安全策略，单击 **...** 后选择**删除**；或者单击待删除的安全策略，在详情页面单击 **...** 后选择**删除**。

## 管理安全服务 > 设置 VPC 隔离虚拟机

# 设置 VPC 隔离虚拟机

VPC 隔离虚拟机用于对 VPC 内的虚拟机进行隔离，仅针对关联当前 VPC 的网卡有效。若一台虚拟机同时关联了多个 VPC，则关联其他 VPC 的网卡不受当前 VPC 隔离策略的管理。

隔离策略分为**完全隔离**和**诊断隔离**。

- **完全隔离**：虚拟机将拒绝所有的出、入流量。此隔离模式适用于虚拟机中毒场景，可避免病毒在内部环境中传播，减少对业务的影响。
- **诊断隔离**：虚拟机仅可以和白名单通信，支持分别设置虚拟机的出流量白名单和入流量白名单。

## 管理安全服务 > 设置 VPC 隔离虚拟机 > 隔离虚拟机

# 隔离虚拟机

您可以批量隔离虚拟机，或按需单独隔离某个虚拟机。

## 批量隔离虚拟机

**前提条件**

已[创建 VPC](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_60) 且已存在使用该 VPC 网络的虚拟机。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**安全服务**。
2. 单击 **+ 操作 > 隔离虚拟机**。
3. 选择待隔离的虚拟机所属的 Everoute 服务和 VPC。
4. 设置隔离对象。若所选虚拟机已被隔离，则此次配置的隔离策略将会覆盖已有的隔离策略。

   - 支持按**虚拟机名称**、**所属安全组**或**关联标签**快速筛选虚拟机。
   - 单击 **+**，可从筛选结果中选择一个或多个虚拟机作为隔离对象。
   - 单击**全选**，可将筛选出的虚拟机全部作为隔离对象。
5. 单击**下一步**，设置隔离策略。

   当您选择**诊断隔离**时，可设置隔离对象的出入流量。具体操作方式与[创建 VPC 安全策略](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_113)时为策略对象设置出入流量白名单相同。
6. 单击**隔离**。

   隔离成功后，您可以在 **VPC 隔离虚拟机**列表中可以查看所有隔离虚拟机的信息。

   - 单击某一 VPC 隔离虚拟机，可查看该虚拟机的 VPC 隔离信息。
   - 在 **VPC 隔离虚拟机**列表中单击 **...** 后选择**跳转到虚拟机详情**，可查看该虚拟机的详细信息；您也可以单击某一 VPC 隔离虚拟机，在详情页面单击**跳转到虚拟机详情**。
   - 单击**所属安全组**的名称链接，可跳转至 VPC 安全组详情页面。
   - 单击 **VPC 隔离虚拟机**列表右上方的导出报表图标，可将当前的 VPC 隔离虚拟机列表导出成 `.csv` 文件至本地保存。

3. 在**隔离状态**区域，单击**隔离**，设置虚拟机完全隔离或诊断隔离。
4. 对于已隔离的虚拟机，可**编辑隔离策略**或**停止隔离**。

## 管理许可 > 更新许可

# 更新许可

虚拟专有云网络功能单独对应一个许可码，在如下场景中需更新许可码：

- 计划为集群和主机扩容，但当前授权数量不满足需求。
- 升级许可类型。例如，某一功能的**试用许可**需升级为**订阅许可**或**永久许可**。
- 延长订阅时长。

更新许可时，请将所有当前软件许可信息提供给 SmartX 客户经理，在完成商务流程后，客户经理将提供新的许可码。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，选择**设置**，单击**软件许可**，将许可码粘贴在**新许可码**处。
2. 单击输入框外部的空白区域，载入新许可信息。
3. 确认新许可信息无误后，单击**更新**，完成许可更新。

## 卸载虚拟专有云网络

# 卸载虚拟专有云网络

当您不再需要某个 Everoute 服务中的虚拟专有云网络功能，想要卸载该功能时，请在知晓卸载功能可能导致的风险并确认满足卸载的要求后，参照操作步骤进行卸载。

**风险提示**

卸载虚拟专有云网络后，将造成如下影响：

- 所有 TEP IP 和内部服务 IP 将被删除。
- 后续再次开启虚拟专有云网络功能时，需要重新初始化所有功能。

**卸载要求**

- Everoute 服务的虚拟专有云功能中没有关联的集群，且不包含任何 VPC 和边缘网关虚拟机。
- Everoute 服务的网络负载均衡器功能中不存在启用了**支持 VPC** 功能的 LB 实例或 LB 实例组。

**操作步骤**

1. 登录 CloudTower，进入**网络与安全**界面，选择**设置**，选择待卸载功能的 Everoute 服务。
2. 在**服务设置**中单击**基本信息**，进入基本信息界面。
3. 在待卸载功能名称右侧单击**卸载功能**。
4. 确认卸载功能后，单击**卸载**。
