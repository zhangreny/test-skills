## A-E

# A-E

Boost 模式
:   一种高性能模式，通过 vhost 共享协议实现 Guest OS、QEMU、ZBS 三者之间内存共享，从而提升 ELF 虚拟机性能，降低 I/O 延时。

常驻缓存
:   将热数据固定在高速的缓存层的策略。启用常驻缓存后，可提升访问性能并避免数据下沉。

CloudTower
:   SmartX 多集群资源集中管理平台，可统一管理多种类型的 SmartX 集群及其资源。

存储节点
:   集群中仅运行存储服务的文件控制器。

ELF
:   SmartX 基于 KVM 开发的原生虚拟化计算平台，提供虚拟机生命周期管理、电源操作、高可用、冷热迁移等基本功能，并能与 SmartX 设计研发的分布式块存储服务（ZBS）结合，提供完整的超融合虚拟化解决方案。

## F-J

# F-J

副本
:   副本是分布式存储系统为数据创建的数据冗余拷贝，同一份数据会在多个节点上保存多个副本，确保数据高可用性和容错能力。

挂载路径
:   计算端访问 NFS 服务的路径。

HDFS
:   HDFS（Hadoop Distributed File System，分布式文件系统）是一个易于扩展的分布式文件系统，主要用于存储和管理大型数据集，并提供对应用程序数据的高吞吐量访问。

精简置备
:   按需动态分配空间，仅在实际写入时占用存储空间的置备方式。

纠删码
:   一种数据冗余机制。它通过特定的算法对 K 个原始数据块（Data Block）进行处理，生成 M 个校验块（Parity Block）。在损坏数据块的数量不超过 M 的情况下，可通过纠删码算法，利用 K 个可用的数据块和校验块即可重建出损坏的数据块，实现数据的恢复和容错能力。

---

## K-O

# K-O

NFS
:   NFS（Network File System，网络文件系统）是一种用于文件共享的网络协议，允许网络中的计算机之间共享资源。

NFS 身份验证
:   用于验证用户身份以及控制文件系统内资源的访问权限，使计算端用户可以通过存储端映射后的 UID、GID 访问文件系统。

NFS Squash
:   一种 NFS 服务器配置选项，用于限制计算端的用户，特别是 root 用户访问共享文件系统的权限。

匿名用户
:   NFS 自动创建的程序用户账号，不可用于登陆系统。

匿名用户组
:   NFS 自动创建的用户组账户。

---

## P-T

# P-T

SMTX OS 集群
:   SMTX OS 集群属于逻辑概念。在实际生产环境中，一个 SMTX OS 集群由至少 3 个运行了 SMTX OS 软件的节点通过网络互连组成。

SMTX 文件存储（SFS）
:   SMTX 文件存储（SMTX File Storage，简称“SFS”）是分布式文件存储产品，能够帮助用户高效地存储和管理非结构化数据。

---

## U-Z

# U-Z

文件存储
:   一种将数据组织为文件的形式进行存储的存储方式。文件存储使用树状的目录结构，在根目录下有若干子目录和文件。

文件存储集群
:   用于提供文件服务，由至少三个文件控制器组成。

文件存储网络
:   用于文件存储集群内部数据交换的网络。

文件管理网络
:   用于管理文件存储集群的网络。

文件接入网络
:   用于向外部的计算端提供文件存储服务的网络。

文件控制器
:   运行 SFS 服务组件的系统服务虚拟机。

文件系统
:   可挂载的共享文件系统单元。

虚拟机网络
:   用于虚拟机业务通信和对外提供服务的网络，可以实现不同虚拟机之间的逻辑隔离，以提高网络安全性。

主节点
:   集群中运行存储服务和管控服务的文件控制器。
