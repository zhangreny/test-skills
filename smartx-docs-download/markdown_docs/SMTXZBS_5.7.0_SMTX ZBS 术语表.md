## A-E

# A-E

Boost 模式
:   一种高性能模式，通过 vhost 共享协议实现 Guest OS、QEMU、ZBS 三者之间内存共享，降低 I/O 延时。

白名单
:   一种用于指定允许访问块存储的客户端的访问控制机制。

CHAP 验证
:   基于 CHAP 协议（Challenge-Handshake Authentication Protocol，质询握手身份验证协议）的身份验证机制，可用于确保 iSCSI Initiator 与 iSCSI Target 之间的连接安全。

常驻缓存
:   将热数据固定在高速缓存层的策略。启用常驻缓存后，可避免数据下沉，提升访问性能并。

重建
:   通过快照重建存储卷，新建的存储卷与快照创建时的状态一致

次级可用域
:   在双活集群中，当优先可用域发生故障且无法恢复时，业务将在另外一个可用域内恢复，该可用域即为次级可用域。

存储节点
:   在块存储集群中，指仅运行了存储服务的节点；在文件存储集群中，指仅运行了存储服务的文件控制器。

存储网络
:   用于块存储集群内各节点间数据交换的网络。

存储策略
:   块存储集群的存储策略包括冗余策略（多副本、纠删码）、置备方式、条带化和常驻缓存模式。

存储分层模式
:   将块存储集群内的存储设备分为缓存层和容量层，其中容量层是所有节点数据分区构成的空间，用于存放冷数据；缓存层可分为写缓存和读缓存，分别用于存放新写入的数据和频繁访问的已下沉数据。

存储不分层模式
:   不设置缓存层。除了含有系统分区的物理盘，剩余的所有物理盘都作为数据盘使用。

COW
:   COW（Copy-on-write，写时复制）一种延迟数据复制的机制，在数据进行修改前不立即创建副本，仅在实际修改时才创建副本。该技术常用于快照场景，可显著节省空间并提高写入效率。

读缓存
:   用于缓存被频繁访问的冷数据，保障数据的读性能。

独占容量
:   被单个存储卷或快照独占使用的存储容量。

多副本
:   一种存储策略，指在多个节点上存放相同数据的多个副本，以保障数据的高可用性。

## F-J

# F-J

分布式块存储
:   一种将块格式数据（LUN/Namespace）拆分为更小的数据单元，并分布在多个节点上来实现高性能和高可靠的存储方式。

分离部署模式
:   一种网络部署模式。在该模式下，集群的每个系统网络分别独占一台虚拟机分布式交换机。

共享卷
:   多个客户端尽量使用不同的接入点连接存储，适用于多个客户端同时访问同一存储资源的场景。

共享容量
:   被多个存储卷或快照共同使用的存储容量，在存储卷发生克隆或快照操作后，不会立即进行完整的数据拷贝，而是多个存储对象共享实际存储的数据。

挂载路径
:   计算端访问 NFS 服务的路径。

HDFS
:   HDFS（Hadoop Distributed File System，分布式文件系统）是一个易于扩展的分布式文件系统，主要用于对海量文件信息和大数据的存储和管理。

I/O 突发
:   在指定的时间段内，系统的 I/O 操作可以短暂超过预设的 IOPS 上限或带宽上限的一种机制。

IP 白名单
:   通过 IP 地址标识的允许访问块存储的客户端。

IQN 白名单
:   通过 IQN 标识的允许访问块存储的客户端。

Initiator CHAP
:   由 iSCSI Target 对 iSCSI Initiator 发起的单向 CHAP 认证，适用于控制客户端访问权限。

管理网络
:   用于管理块存储集群的网络。

厚置备
:   在创建存储卷时立即为其分配全部空间，即使存储卷尚未写入，也占用全部分配空间的置备方式。

缓存层
:   所有节点缓存分区构成的空间，通常采用低延迟、高吞吐的存储介质，用于放置热数据。缓存层分为写缓存和读缓存。

回滚
:   通过快照将存储卷回滚到创建快照时的状态。

iSCSI
:   iSCSI（Internet Small Computer System Interface）是一种基于 IP 网络的存储协议，用于将 SCSI 命令封装在 IP 数据包中进行传输。它允许客户端通过网络访问远程存储设备，具备部署简单、成本低、可扩展性强等优势。

iSCSI Initiator
:   客户端主机上的软件或硬件组件，用于发起与远程 iSCSI Target 的连接并进行数据传输。它使客户端能够通过 IP 网络像使用本地磁盘一样访问远程存储资源。

iSCSI Target
:   用于响应 iSCSI Initiator 请求的存储端组件，负责提供 iSCSI LUN 供客户端访问。

iSCSI LUN
:   在 iSCSI 存储服务中用于唯一标识存储设备上逻辑单元的编号。一个 LUN 通常对应于一个存储卷。

接入点分配策略
:   NVMe Namespace 的接入点分配方式，包含继承策略和均衡策略两种。

均衡策略
:   通过 Namespace Group 平衡接入路径，使多个 Namespace 分散在不同接入点的介入策略，可提高 I/O 并发性能。

继承策略
:   同一客户端使用同一接入点，不同客户端尽量分散接入的接入点分配策略，适用于共享卷场景。

接入网络
:   用于为为集群外部的计算端提供存储服务的网络。

精简置备
:   按需动态分配空间，仅在实际写入时占用存储空间的置备方式。

纠删码
:   一种数据冗余机制。它通过特定的算法对 K 个原始数据块（Data Block）进行处理，生成 M 个校验块（Parity Block）。在损坏数据块的数量不超过 M 的情况下，可通过纠删码算法，利用 K 个可用的数据块和校验块即可重建出损坏的数据块，实现数据的恢复和容错能力。

## K-O

# K-O

LACP
:   LACP（Link Aggregation Control Protocol，链路聚合控制协议）是一种基于 IEEE802.3ad 标准的实现链路动态聚合的协议，术语在链路聚合中常用的协议之一。链路聚合组中启用了 LACP 协议的成员端口通过发送 LACP PDU 报文进行交互，双方对能够发送和接收报文的端口明确并达成一致，确定承担业务流量的链路。

NFS
:   NFS（Network File System，网络文件系统）是一种用于文件共享的网络协议，允许网络中的计算机之间共享资源。

NFS 身份验证
:   用于验证用户身份以及控制文件系统内资源的访问权限，使计算端用户可以通过存储端映射后的 UID、GID 访问文件系统。

NFS Squash
:   一种 NFS 服务器配置选项，用于限制计算端的用户，特别是 root 用户访问共享文件系统的权限。

NQN 白名单
:   通过 NQN 标识的允许访问 NVMe Namespace 的客户端范围。

NVMe
:   一种用于服务器访问内部或直连存储的协议，具有高带宽、低延时和低 CPU 使用率的特性。

NVMe Namespace
:   NVMe Namespace 类似于存储设备或 LUN，客户端在发现 Namespace 后，将在存储设备列表中显示该 Namespace 的存储设备。

NVMe Subsystem
:   NVMe Subsystem 是逻辑上的存储集合，通过 NVMe 限定名称 (NQN) 标识，一个 NVMe Subsystem 可以包含一个或多个 NVMe Namespace。

NVMe-oF
:   NVMe-oF（NVMe over Fabrics，NVMe 远程访问扩展协议）支持通过 RDMA 或 TCP 等网络传输访问存储。

NVMe over RDMA
:   NVMe-oF 协议的一种，指通过 RDMA 协议栈直接访问内存，降低延迟和 CPU 占用，提升吞吐性能。

NVMe over TCP
:   NVMe-oF 协议的一种，指通过 TCP/IP 协议直接访问内存。 NVMe over RCP 可在普通以太网环境中部署，具备良好的兼容性和性能。

可用域
:   双活集群的两个可用域一般位于同一个城市且相互间有一定距离，分别属于不同的数据中心。每个可用域至少包含 3 个节点，并且可以完全独立地为双活集群提供计算和存储服务。

块存储集群
:   块存储集群是由多个物理节点组成的存储资源池，通过分布式软件协同工作，为上层应用提供高性能、高可靠、可横向扩展的块级存储服务。

快照
:   对存储卷在某个时间点的状态进行记录和保存的功能，可用于数据回滚。

快照对象
:   受快照计划保护的资源。

快照计划
:   对快照的创建和保留进行定期自动化管理的策略。

快照组
:   执行快照计划后生成的一组快照的集合。

匿名用户
:   NFS 自动创建的程序用户账号，不可用于登陆系统。

## P-T

# P-T

RDMA
:   RDMA（Remote Direct Memory Access，远程直接内存访问）是一种直接内存访问技术，节点间进行数据传输时，可以绕过双方操作系统内核，通过网络直接访问其内存数据。

ROW
:   ROW（Redirect-On-Write，写时重定向），一种在写操作时不复制原始数据，而是将新数据写入新位置并更新元数据指向的机制。该方式有利于保护原始数据、支持快速回滚，并提高存储性能。

SDR
:   Sensor Data Record，传感器数据记录。

SEL
:   System Event Log，系统事件日志。

SMTX ZBS 集群
:   SMTX ZBS 集群属于逻辑概念，在实际生产环境中，一个 SMTX ZBS 集群包含两个集群，分别是运行块存储文件的集群和运行文件存储软件的集群。

SMTX ZBS 块存储集群
:   块存储集群是由多个物理节点组成的存储资源池，通过分布式软件协同工作，为上层应用提供高性能、高可靠、可横向扩展的块级存储服务。

SMTX ZBS 文件存储集群
:   用于提供文件服务，由至少三个文件控制器组成。

SMTX ZBS 双活集群
:   以拉伸形态部署的 SMTX ZBS 集群，由两个可用域和一个仲裁节点组成。两个可用域与仲裁节点间通过网络连接通信。SMTX ZBS 双活集群也简称为双活集群。

Target CHAP
:   iSCSI Initiator 对 iSCSI Target 发起的 CHAP 认证，通常与 Initiator CHAP 配合使用，构成双向验证。

日志重放
:   文件系统和存储引擎保障数据一致性的机制，通过重做或撤销日志中记录的未完成操作，以将数据还原至一致状态。

融合部署模式
:   一种网络部署模式。在该模式下，集群中两个或两个以上的系统网络共享一台虚拟分布式交换机。

容量层
:   所有节点数据分区构成的空间，用于放置冷数据。

容量规格
:   按照节点物理盘容量上限区分的规格。

数据分片
:   在分布式存储系统中，根据冗余策略（如副本或纠删码）将数据块拆分为多个部分，并分别存储于不同物理节点。每个部分称为一个数据分片。

条带化
:   将数据切分后并行写入多个硬盘的策略，可用于提升存储性能。条带数越多，并行能力越强。

## U-Z

# U-Z

ZBS
:   分布式块存储软件，负责接管服务器的所有存储资源（包括所有的 SSD 和 HDD），将多个服务器的存储资源池化，构建统一的存储池并提供存储服务。

重建

文件存储
:   一种将数据组织为文件的形式进行存储的存储方式。文件存储使用树状的目录结构，在根目录下有若干子目录和文件。

文件存储集群
:   用于提供文件服务，由至少三个文件控制器组成。

文件存储网络
:   用于文件存储集群内部数据交换的网络。

文件管理网络
:   用于管理文件存储集群的网络。

文件接入网络
:   用于向外部的计算端提供文件存储服务的网络。

文件控制器
:   运行 SFS 服务组件的系统服务虚拟机。

文件系统
:   可挂载的共享文件系统单元。

性能层
:   即写缓存空间，用于新数据的写入，保障数据的写性能。

一致性组
:   将多个卷作为一组进行快照创建和快照回滚等相关操作，支持快照操作的一致性，确保多卷数据同步回滚。

一致性快照组
:   对一致性组内所有成员创建的一组快照。通过一致性快照组，可将所有一致性组成员同时回滚至同一时间点。

优先可用域
:   正常工作时业务主要运行的可用域。

置备方式
:   一种存储空间分配策略，包括厚置备和精简置备两种方式。

主节点
:   在块存储集群中，指运行了元数据服务的节点；在文件存储集群中，指运行了存储服务和管控服务的文件控制器。

仲裁节点
:   一般位于两个可用域所在数据中心以外的其他数据中心，主要为保障双活集群选主类服务正常运行，并使得业务在优先可用域或次级可用域中可以自动切换。
