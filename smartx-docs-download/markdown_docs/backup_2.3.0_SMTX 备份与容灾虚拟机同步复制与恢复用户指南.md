# backup/2.3.0/SMTX 备份与容灾虚拟机同步复制与恢复用户指南
## 关于本文档

# 关于本文档

本文档介绍了如何使用 SMTX 备份与容灾的复制服务对 SMTX OS（ELF）集群中的虚拟机进行同步复制和故障处理。

参考本文档使用虚拟机同步复制相关功能前，请您先参考《SMTX 备份与容灾-安装与运维指南》完成复制服务的部署。

## 概述

# 概述

同步复制是一种为关键业务设计的容灾保护方案。该功能首先为原虚拟机创建一个初始的副本虚拟机，随后将原虚拟机的数据变化同步到副本虚拟机。在原虚拟机和副本虚拟机进入同步状态后，系统将进入双写模式，所有数据写入 I/O 均在源端与目标端写入。通过这一机制，同步复制能够在灾难场景下实现零数据丢失（RPO = 0），为关键业务提供持续可靠的保护。

同步复制同时支持故障转移、故障转移测试、故障恢复与永久故障转移等恢复操作。

---

## 概述 > 功能简介

# 功能简介

- **虚拟机同步复制**

  将运行在 SMTX OS（ELF）集群中的虚拟机复制到另一个 SMTX OS（ELF）集群中，并在目标集群创建对应的副本虚拟机，通过数据同步和双写机制，实时保证副本虚拟机和原虚拟机的数据一致性。

  ![](https://cdn.smartx.com/internal-docs/assets/73b10dc3/vm_replication_user_guide_01.png)
- **故障转移**

  故障转移是指当原虚拟机发生故障时，可使用副本虚拟机接替原虚拟机继续提供服务。根据应用的场景不同，可以分为计划外故障转移和计划内故障转移。

  ![](https://cdn.smartx.com/internal-docs/assets/73b10dc3/vm_replication_user_guide_02.png)

  - **计划外故障转移**

    计划外故障转移是生产环境发生意外或故障时立即启动的应急措施，通常应用于如硬件故障或网络中断等紧急情况，为业务提供快速、安全、可靠的灾备保障。当原虚拟机发生故障时，复制服务可将副本虚拟机回滚至指定的恢复点状态，并将业务切换至副本虚拟机提供。
  - **计划内故障转移**

    计划内故障转移是根据预先设定的计划进行的有序转移。在进行灾备演练、计划中的数据中心维护或升级、或预计将发生灾难的情况下，复制服务可以执行计划内故障转移。在副本虚拟机已同步状态下，可立即将原虚拟机的业务切换至副本虚拟机，实现 RPO = 0 的数据保护。
- **故障转移测试**

  故障转移测试用于在不影响生产业务的情况下，验证故障转移流程是否符合预期。系统会基于选定的恢复点，在目标端测试环境中生成测试虚拟机，模拟真实的业务切换流程，且测试过程中产生的数据不会回写到源端。

  ![](https://cdn.smartx.com/internal-docs/assets/73b10dc3/vm_replication_user_guide_22.png)
- **故障恢复**

  故障恢复是在进行故障转移后，当原虚拟机恢复正常时，将故障转移期间副本虚拟机产生的数据增量同步回原虚拟机，并恢复至由原虚拟机提供服务，可确保原虚拟机在恢复后能够无缝切换回原始的生产环境，保障业务的连续性。

  ![](https://cdn.smartx.com/internal-docs/assets/73b10dc3/vm_replication_user_guide_03.png)
- **永久故障转移**

  永久故障转移是在进行故障转移后，当原虚拟机无法恢复时，将原虚拟机永久切换到副本虚拟机，并使用副本虚拟机持续提供服务。

  ![](https://cdn.smartx.com/internal-docs/assets/73b10dc3/vm_replication_user_guide_04.png)

## 概述 > 应用场景

# 应用场景

虚拟机同步复制与恢复可应用于以下场景：

- **灾难恢复**

  防止因自然或人为灾害、勒索软件攻击、恐怖事件等导致的大规模数据损坏和丢失。一旦灾难发生，可快速将业务切换至灾备集群，并迅速恢复业务运行。
- **灾备演练**

  同步复制与恢复可用于真实灾备演练和模拟灾备演练：

  - 真实灾备演练：通过故障转移与故障恢复功能进行，用于在接近真实灾难的条件下验证灾备接管能力。演练过程中存在业务停机时间，业务将由副本虚拟机接管；演练结束后，可将演练期间产生的数据同步回源端虚拟机，恢复正常生产运行。
  - 模拟灾备演练：通过故障转移测试功能进行，用于在不影响生产业务的情况下验证灾备方案。演练过程中，生产虚拟机保持正常运行，演练数据不会同步回源端虚拟机，不会对生产环境造成影响。
- **数据复用**

  利用集群原生的快照重建功能实现数据复用。通过从副本虚拟机的快照重建虚拟机进行开发、测试、故障排除等活动，避免对生产环境造成任何影响。数据复用功能能够在不影响主要业务的前提下，为开发人员和测试团队提供高效且隔离的环境，加快开发和测试过程，同时确保生产环境的稳定性和安全性。

## 概述 > 产品特性

# 产品特性

复制与恢复模块具有如下特性：

- 易用性：当同步异常发生时，系统会生成事件记录，帮助用户及时发现问题并进行处理，提高运维可视化和管理效率。
- 灵活性：提供灵活的恢复点管理，支持定期和异常恢复点创建，使用户可根据业务需求回滚至任意有效状态，满足不同场景下的恢复要求。
- 高可用性：当复制服务所在节点发生故障时，复制服务可自动在其他节点拉起并继续提供服务。
- 负载控制：支持自定义任务重试次数、故障恢复任务传输速率限制对复制服务进行负载控制。

## 相关概念

# 相关概念

本章解释了手册中使用的专业术语。在阅读过程中，如果遇到不熟悉的概念，可在此快速查找其定义和详细解释。

**复制服务**

负责管理同步复制计划、同步复制任务和状态、故障转移、故障转移测试、故障恢复和永久故障转移等任务调度。

**同步复制计划**

用于规划和安排数据的复制操作。通过同步复制计划，用户可以定义复制对象、复制方向、目标集群、复制服务、同步策略、副本虚拟机和测试虚拟机设置等信息。

**站点**

站点是指由单一 CloudTower 管理的一个故障域，可以包含一个或多个集群和数据中心。每个站点的资源都由其专属的 CloudTower 进行管理，且在地理位置、硬件设备和软件服务等方面完全隔离，不与其他站点共享任何资源。

站点可以分为以下两种类型：

- **源站点**：通常为实际运行业务的站点。复制服务将源站点的原虚拟机复制到目标站点的副本虚拟机，实现数据的同步。
- **目标站点**：通常为用于容灾的站点，用于存储从源站点复制过来的副本虚拟机。

**站内复制**

源集群和目标集群位于同一站点。当原虚拟机发生故障时，可切换使用站点内其他集群中的副本虚拟机，实现集群级别的容灾。若管理站点的 CloudTower 发生故障，则无法发起故障转移。

**跨站点复制**

源集群和目标集群位于不同站点。当源站点内的原虚拟机不可用时，无论管理源站点的 CloudTower 是否发生故障，均可切换至目标站点的副本虚拟机，实现站点级别的容灾。

跨站点复制包括以下两种类型：

- **入站复制**：目标站点为当前站点，源站点为其他关联站点。
- **出站复制**：源站点为当前站点，目标站点为其他关联站点。

**源集群**

通常为生产集群，承载着实际运行的业务应用和数据，是原虚拟机所属的集群，位于源站点。

**目标集群**

通常为用于容灾的集群，用于存储从源集群复制过来的副本虚拟机，位于目标站点。

**虚拟机**

通过虚拟化技术实现，运行在物理服务器虚拟化平台上的计算机系统，具有与物理计算机相同的特性。物理服务器上的所有虚拟机共享底层的物理硬件资源，但每个虚拟机拥有独立的操作系统、应用程序和资源配置。虚拟机之间相互隔离，运行互不影响，可用于测试、开发和生产环境。

对虚拟机进行复制时，涉及三种虚拟机角色：

- **原虚拟机**

  需要进行容灾保护的虚拟机，负责承载实际的业务应用和数据。
- **测试虚拟机**

  通过故障转移测试任务创建的原虚拟机的完整副本，具有和原虚拟机相同的数据和配置，通常用于灾备演练，验证故障转移流程的正确性以及数据一致性。
- **副本虚拟机**

  通过全量复制任务创建的原虚拟机的完整副本，具有和原虚拟机相同的数据和配置。副本虚拟机通常用于容灾，当原虚拟机不可用时，可以接替原虚拟机继续提供服务。

**复制资源**

指复制对象、同步复制计划和恢复点。

**复制对象**

一个原虚拟机及其对应的副本虚拟机称为一组复制对象。

**复制恢复点**

每次为虚拟机执行复制、故障转移测试（立即生成恢复点模式）、故障转移（立即生成恢复点模式）或故障恢复时生成的数据状态，以快照形式保存在副本虚拟机上。在已同步状态下，系统还会按照用户配置的时间间隔自动创建恢复点；当同步复制任务发生异常时，系统将生成对应的异常恢复点。

恢复点可以使虚拟机在需要时快速回滚到特定的数据状态，例如在灾难恢复场景中将虚拟机恢复到某个先前的时间点，以恢复虚拟机的正常运行。

**全量复制**

将虚拟机的所有数据，包括操作系统、应用程序、文件和配置等，复制到副本虚拟机，从而确保副本虚拟机与原虚拟机完全一致。全量复制通常发生在同步初始化阶段，用于建立虚拟机的初始副本，并为后续的数据同步提供基准。

**增量复制**

复制自上次复制后发生变化的数据，能够持续跟踪和同步数据源的变化，保障数据的准确性和一致性。

**故障处理**

故障发生后可进行的恢复操作，包括故障转移、故障转移测试、故障恢复和永久故障转移。

- **故障转移**

  应对虚拟机故障的操作，当原虚拟机发生故障时，可以将副本虚拟机的虚拟卷数据回滚到与指定恢复点一致的状态，并启动副本虚拟机继续提供服务。通过故障转移，可以在短时间内将服务快速恢复到故障发生前的状态，保障业务的连续性和稳定性。
- **故障转移测试**

  故障转移测试是一种模拟容灾演练的操作，在不中断源端生产业务的前提下，通过生成测试虚拟机，在测试虚拟机验证恢复机制的有效性。演练过程中的增量数据不会同步回被保护的虚拟机。
- **故障恢复**

  当原虚拟机恢复正常后，将副本虚拟机在故障转移期间变化的数据回传至原虚拟机，并由原虚拟机继续提供服务。故障恢复旨在将原虚拟机和副本虚拟机之间的数据同步，确保原虚拟机可以在故障解决后继续提供服务。
- **永久故障转移**

  当原虚拟机遭遇严重故障且无法从故障中恢复时采取的措施，此时副本虚拟机将正式接替原虚拟机的服务角色，以确保业务的持续运行。

**RPO** (Recovery Point Objective)

数据恢复点目标，指在故障转移场景中允许的数据损失的时间范围。RPO 以时间为度量单位，标示了从最近执行的复制时间点到发生故障时的时间跨度。较小的 RPO 值表示系统要求在发生故障时允许的数据损失较少。较大的 RPO 值意味着在故障发生时系统可以容忍较长时间范围内的数据丢失。

**RTO** (Recovery Time Objective)

数据恢复时间目标。指业务系统从崩溃到恢复运行所能容忍的时间。较短的 RTO 值表示业务系统要求在发生故障时能够迅速恢复，较长的 RTO 值则意味着业务系统可以容忍更长的恢复时间。

**VPC**（Virtual Private Cloud）

虚拟专有云，是一套为云资源构建的逻辑隔离的网络环境，用户可以在 VPC 中自主设置 VPC 子网、IP 地址、路由表、浮动 IP、网关服务和安全服务等，以实现网络隔离和安全控制。副本虚拟机和原虚拟机可以部署在相同或不同的 VPC 中，并支持灵活的网络映射。测试虚拟机需部署在和原虚拟机不同的 VPC 中。

**VPC 子网**

VPC 子网是组成 VPC 的基础网络模块，用于划分 VPC 的 IP 地址空间。位于同一 VPC 子网的虚拟机默认可以直接通信，跨 VPC 子网虚拟机在在路由可达的情况下也可以互相通信。

**浮动 IP**

浮动 IP 是一种网络地址转换（NAT）机制，通过一对一映射的方式，将外部子网中的 IP 地址动态绑定到 VPC 内部的虚拟机上。这使得外部网络可以主动访问 VPC 内的虚拟机。浮动 IP 为 VPC 内的虚拟机提供了灵活的外部访问能力，同时也保证了网络安全性。

**网络映射**

旨在解决原虚拟机和副本虚拟机/测试虚拟机之间所处的网络环境差异问题的一项配置规则。

由于副本虚拟机/测试虚拟机和原虚拟机往往处于不同的网络环境，若副本虚拟机/测试虚拟机完全复制原虚拟机的网络配置，可能无法在目标集群环境正常配置或连通网络。通过配置网络映射规则，可将原虚拟机所属的虚拟机网络映射到目标集群中对应的虚拟机网络，从而保证副本虚拟机/测试虚拟机能够正常与网络相连。

**IP 映射**

旨在解决原虚拟机和副本虚拟机或测试虚拟机在不同网络环境下的 IP 地址不匹配的问题的一项配置规则。

由于副本虚拟机或测试虚拟机和原虚拟机往往处于不同的网络环境，若副本虚拟机/测试虚拟机完全复制原虚拟机的 IP 地址，可能无法在目标集群正常联通网络。通过配置 IP 映射规则，在故障转移和故障转移测试时，复制服务可以自动为副本虚拟机或测试虚拟机设置正确的 IP 地址，确保副本虚拟机或测试虚拟机可以按照预期的网络配置提供服务，无需手动去更新虚拟机 IP。

**SMTX OS 集群**

SMTX OS 集群属于逻辑概念。在实际生产环境中，一个 SMTX OS 集群至少由 3 个节点通过网络互连组成。

**ELF**

SMTX OS 原生虚拟化计算平台，是当 SMTX OS 以超融合形态直接部署在 x86\_64 或 AArch64 架构的服务器上时，该超融合软件中基于 KVM 的虚拟化引擎组件。ELF 除提供虚拟机生命周期管理、电源操作、高可用、冷热迁移等基本功能外，通过与存储组件 ZBS 相结合，还能提供秒级快照、模板和克隆等高级的虚拟机服务。

ELF 简单易用，对 SMTX OS 超融合集群的所有操作均可通过 CloudTower 完成。

**CloudTower**

SmartX 多集群资源集中管理平台，旨在管理站点内数据中心及跨数据中心的多个 SmartX 集群和系统服务，并提供完整和标准的 RESTful API 和多种语言的 SDK。

## 使用前规划 > 规划复制计划类型

# 规划复制方向

使用复制服务对虚拟机进行容灾保护时，根据不同的业务需求和站点布局，可选择站内复制或跨站点复制。

- **站内复制**

  站内复制通常用于集群级别的容灾保护。原虚拟机和副本虚拟机位于同一站点内不同集群中，当原虚拟机不可用时，可以通过切换到站内其他集群中的副本虚拟机，以保持业务的持续运行。该复制策略适用于对数据保护的需求集中在单个站点内的场景。

  ![](https://cdn.smartx.com/internal-docs/assets/73b10dc3/vm_replication_user_guide_19.png)
- **跨站点复制**

  跨站点复制可提供站点级别的容灾保护，当一个站点不可用时，可以立即切换到位于其他地理位置的站点，确保业务的高可用性。该复制策略适用于在不同地理位置部署了多个数据中心，需确保跨地域容灾能力的场景。

  ![](https://cdn.smartx.com/internal-docs/assets/73b10dc3/vm_replication_user_guide_20.png)

## 使用前规划 > 规划站点

# 规划站点

使用跨站点复制策略时，请从如下几方面规划源站点和目标站点。

- **确定站点的定位**

  根据业务需求，明确源站点和目标站点的角色。通常将生产站点作为复制的源站点进行保护，将灾备站点作为目标站点。一个生产站点最多支持配置 5 个灾备站点，以提升数据冗余和灾备能力。
- **确保站点隔离**

  源站点和目标站点的分布应充分考虑地理隔离，以应对地震、火灾等自然灾害和其他潜在风险。同时还应确保各站点硬件、管理平台（CloudTower）和软件服务的独立性。通过多层次的隔离避免所有站点同时发生故障，以确保业务的高可用性和数据安全。
- **选择合适的复制架构**

  根据实际业务需要和数据保护策略选择合适的复制架构。

  - **单向的生产-灾备架构**

    此架构适合需要明确区分业务生产和灾备的场景。可将生产站点作为源站点，专注于正常的业务运行；将灾备站点作为目标站点，专注于数据恢复和灾难保护。
  - **双向的互为灾备架构**

    此架构下，两个站点相互备份对方数据，任一站点发生故障时，另一站点可以迅速接管业务。

---

## 使用前规划 > 规划集群

# 规划集群

规划源集群和目标集群时，请从如下几个方面综合考虑：

- **确定集群的定位**

  根据业务需求，将生产集群作为复制的源集群进行保护，将灾备集群作为复制的目标集群。
- **选择合适的复制架构**

  根据实际业务需要和数据保护策略选择合适的复制架构。

  - **单向的生产 - 灾备架构**

    若需要实现业务生产和灾备的隔离，可选择此架构，将生产集群作为源集群，灾备集群作为目标集群。
  - **双向的互为灾备架构**

    若两个集群需各自运行业务且互为灾备集群，可选择此架构，两集群之间互相复制，并各自使用部分资源为对方存储副本。
- **确认 CPU 架构的一致性**

  请确保源集群和目标集群的 CPU 架构保持一致，避免兼容性问题。

## 使用前规划 > 规划同步复制计划

# 规划同步复制计划

在创建同步复制计划前，请从以下几个方面规划同步复制计划：

- **计划保护的虚拟机**

  每个同步复制计划中可包含一个或多个虚拟机，这些虚拟机共享相同的复制策略。因此，建议根据不同的数据保护要求，对虚拟机进行分类，将具有相同目标集群和同步策略的虚拟机置于同一个复制计划中，避免重复定义相同的复制策略，提高运维和管理效率。
- **规划初始化时间**

  通过设定初始化时间，用户可控制首次全量复制的执行时点，实现灵活启动同步初始化。
- **恢复点策略**

  在复制对象进入同步状态后，设置恢复点的创建周期，以及恢复点保留策略。

## 使用前规划 > 规划网络

# 规划网络

原虚拟机和副本虚拟机的数据同步需通过源集群和目标集群的存储网络进行，需确保两端集群的存储网络在三层网络层面互通。

为保障同步复制的稳定性与性能，建议网络带宽不低于 10Gb，且端到端 RTT 小于 5 ms。

## 使用前规划 > 规划许可单元的分配

# 规划许可单元的分配

SMTX 备份与容灾的软件许可由备份与复制两个功能模块共享。许可单元根据所保护对象类型及所用功能进行消耗，具体规则如下：

- **SMTX OS（ELF）集群虚拟机**

  - 备份功能：每备份一个虚拟机，消耗 1​ 个许可单元。
  - 异步复制功能：每异步复制一个虚拟机，消耗 2​ 个许可单元。
  - 同步复制功能：每同步复制一个虚拟机，消耗 4​ 个许可单元。
- **SMTX ZBS 块存储集群存储卷**：每异步复制 1 TiB 的存储卷，消耗 2​ 个许可单元。

请您根据实际的业务需求，综合考虑要使用的功能模块，需要保护的虚拟机和存储卷数量、以及未来业务的扩展等，提前规划许可单元的分配。

## 管理站点（仅跨站点复制）

# 管理站点（仅跨站点复制）

一个站点使用一个 CloudTower 对站点内的集群进行管理。若您使用跨站点同步复制策略，需要通过将当前站点的 CloudTower 和其他站点的 CloudTower 进行关联，从而建立站点之间的关联关系，以支持站点之间的数据复制。若您使用站点内复制策略，则无需关联站点。

## 管理站点（仅跨站点复制） > 添加关联站点

# 添加关联站点

完成复制服务的部署后，请为当前站点添加关联站点。

**前提条件**

- 仅超级管理员和运维管理员有关联站点的操作权限，因此在当前站点请使用超级管理员或运维管理员账号登录。
- 提前获取待添加站点所属的 CloudTower 的 IP 地址、超级管理员或任一运维管理员账号及密码。关联站点时仅支持使用站点所属的 CloudTower 的本地账户，不支持 LDAP 账户。
- 每个站点最多可关联 5 个站点，因此请确保当前站点和待添加站点已关联的站点数均小于 5。
- 为避免兼容性问题，建议源站点和目标站点关联相同版本的 CloudTower。

**操作步骤**

1. 登录 CloudTower，在备份与容灾界面中，单击**设置** > **站点管理**进入站点管理界面。
2. 单击界面右上角 **+ 添加关联站点**，弹出**添加关联站点**对话框。
3. 输入以下信息：

   | 参数 | 描述 |
   | --- | --- |
   | 站点名称 | 自定义待关联站点的名称。 |
   | CloudTower IP | 输入待关联站点所属 CloudTower 的 IP 地址。 |
   | 用户名 | 输入待关联站点 CloudTower 中超级管理员或任意运维管理员账号的用户名。 |
   | 密码 | 输入 CloudTower 用户名对应的密码。 |
4. 单击**添加**。
5. （可选）若关联站点启用了双因子认证，您还需查看该站点用户信息中填写的邮箱，将 6 位数验证码填写至输入框中，并单击**验证**。验证通过后即可完成关联。

完成添加后，站点管理界面将展示已关联站点的信息，包括站点名称、和当前站点的连接状态和 CloudTower IP。

## 管理站点（仅跨站点复制） > 编辑关联站点

# 编辑关联站点

当关联站点的 CloudTower IP 发生变更时，站点的连接状态将显示连接异常。您可以通过编辑操作更新关联站点最新的 CloudTower IP 信息，以重新和站点进行关联。

**前提条件**

- 当前站点的用户角色为超级管理员或运维管理员。
- 待编辑站点相关的入站复制计划中不存在虚拟机正在进行复制、故障转移、故障恢复和永久故障转移任务。
- 待编辑站点相关的出站复制计划中不存在虚拟机正在进行复制任务。
- 待编辑站点和当前站点未处于`连接中`状态。

**操作步骤**

1. 登录 CloudTower，在备份与容灾界面中，单击**设置** > **站点管理**进入站点管理界面。
2. 单击某一站点右侧的 **...**，选择**编辑**，弹出**编辑关联站点**对话框。
3. 编辑站点名称或 CloudTower IP。
4. 单击**保存**。

## 管理站点（仅跨站点复制） > 删除关联站点

# 删除关联站点

若需解除当前站点与已关联站点的关联关系，可通过删除关联站点来解除关联关系。

**前提条件**

- 当前站点的用户角色为超级管理员或运维管理员。
- 待删除站点和当前站点处于`已连接`状态，当前站点与待删除站点之间需不存在出站复制计划、入站复制计划、出站复制对象和入站复制对象。
- 待删除站点和当前站点处于`连接异常`状态，当前站点与待删除站点之间需不存在处于`故障转移中`、`永久故障转移中`状态或正在执行故障转移任务的入站复制对象。
- 待删除站点和当前站点未处于`连接中`状态。

**操作步骤**

1. 登录 CloudTower，在备份与容灾界面中，单击**设置** > **站点管理**进入站点管理界面。
2. 单击某一站点右侧的 **...**，选择**删除**，弹出**删除关联站点**对话框。
3. 单击**删除**。

## 管理关联集群 > 添加关联集群

# 添加关联集群

一个复制服务可以关联多个 SMTX OS（ELF）集群，一个 SMTX OS（ELF）集群也可以关联多个复制服务。

**前提条件**

- 复制服务仅可关联与 CloudTower 的连接状态为`已连接`的集群。
- 请确保复制服务和待关联集群的存储网络连通。
- 进行站内同步复制时，源集群和目标集群需关联同一个复制服务。

**操作步骤**

1. 登录 CloudTower，在**备份与容灾**界面中，单击**设置**进入功能管理界面。
2. 选择一个已部署的复制服务，单击**集群关联**。
3. 单击**编辑集群关联**，在弹出的操作界面中，单击 **+ 添加集群**，添加一个或多个 SMTX OS（ELF）集群。
4. 单击**保存**。

完成关联后，在**集群关联**界面，可查看复制服务已关联的集群列表。

| 参数 | 说明 |
| --- | --- |
| 集群名称 | 复制服务关联的集群名称。 |
| 集群类型 | 复制服务关联的集群类型。 |
| 集群 IP | 集群关联至 CloudTower 时使用的 IP 地址。 |
| 所属数据中心 | 集群在 CloudTower 中所属的数据中心。 |

## 管理关联集群 > 删除关联集群

# 删除关联集群

若复制服务需取消和集群的关联，可单击**编辑集群关联**，在弹出的界面中进行相应操作。

**注意**：

- 若存在虚拟机将已关联集群作为目标集群，并使用当前复制服务时，无法取消关联。
- 若已关联集群中存在虚拟机已加入复制计划或虚拟机存在恢复点，且该集群只关联了该复制服务时，无法取消关联。
- 若当前复制服务仅关联了一个集群，则无法取消关联。
- 若已关联集群中包含副本虚拟机或测试虚拟机，则无法取消关联。

**操作步骤**

1. 登录 CloudTower，在**备份与容灾**界面中，单击**设置**进入功能管理界面。
2. 选择一个已部署的复制服务，单击**集群关联**。
3. 单击**编辑集群关联**，在弹出的操作界面中，单击相关集群旁边 `×` 以移除集群。
4. 单击**保存**。

## 管理同步复制计划 > 创建同步复制计划 > 设置基本信息

# 设置基本信息

1. 输入同步复制计划的名称和描述。
2. 选择复制方向。默认为**站内复制**，当本站点添加了其他站点且至少一个站点处于`已连接`状态，允许选择入站或出站。

   | 配置项 | 说明 |
   | --- | --- |
   | 站内 | 在本站点内进行虚拟机同步复制。 |
   | 入站 | 将外站点虚拟机复制至本站点。请选择复制的源站点。 |
   | 出站 | 将本站点虚拟机复制至外站点。请选择复制的目标站点。 |
3. 设置目标集群和复制服务。

   | 配置项 | 说明 |
   | --- | --- |
   | 目标集群 | 目标集群需满足下述条件： - 集群为 SMTX OS（ELF）6.3.0 及以上版本。 - 集群与 CloudTower 连接正常。 - 集群已关联了 2.3.0 及以上版本复制服务。 |
   | 复制服务 | 当复制方向为站内复制时，请选择一个站内的复制服务。 |
   | 目标端复制服务 | 当复制方向为入站或出站复制时，请选择一个与目标集群关联了的复制服务作为目标端复制服务。 |
4. 选择加入同步复制计划的虚拟机。支持按照虚拟机名称、所属集群名称、标签名和标签值筛选虚拟机，可多选。

   > **说明**:
   >
   > 不支持复制以下虚拟机：
   >
   > - 挂载了共享虚拟卷的虚拟机。
   > - 未挂载虚拟卷的虚拟机。
   > - 运行系统服务的虚拟机，如 Everoute Controller 虚拟机、CloudTower 虚拟机、可观测虚拟机、备份服务虚拟机、复制服务虚拟机、高级监控虚拟机、安全防护虚拟机（SVM）等。
   > - SKS 工作负载集群节点虚拟机。
   > - 容器镜像仓库虚拟机。
   > - 处于`操作中`或`未知`状态的虚拟机。
   > - 所在集群与 CloudTower 连接异常的虚拟机。
   > - 所在集群的许可类型为试用许可且许可已过期的虚拟机。
   > - 复制服务创建的副本虚拟机和测试虚拟机。
   > - 启用了随主机开机功能的虚拟机。
   > - 已加入其他同步复制计划的虚拟机。
   > - 已加入目标集群相同的其他异步复制计划的虚拟机。

## 管理同步复制计划 > 创建同步复制计划 > 设置同步策略

# 设置同步策略

设置首次全量数据同步的执行时间，以及恢复点策略。设置完成后单击**下一步**。

1. **设置同步的初始化时间**

   初始化阶段，复制服务将通过全量复制在目标集群创建副本虚拟机。可选择**立即开始**或**在指定时间开始**。

   - **立即开始**

     同步复制计划创建完成后，加入同步复制计划的复制对象会立即开始同步的初始化。
   - **在指定时间开始**

     复制计划中处于**待同步**状态的复制对象会在指定时间开始初始化。
2. **设置恢复点策略**

   - 设置恢复点创建周期
   - 设置恢复点保留份数

   恢复点保留策略默认自动保留最近的 3 份恢复点，支持保留 1 ~ 16 份，超出保留策略的将会被自动清理。

> **说明**：
>
> 恢复点保留策略仅适用于复制计划自动执行生成的恢复点，通过其他方式创建的恢复点将永久保留。
>
> 自动生成的恢复点包括：
>
> - 同步过程或异常状态下生成的恢复点：数据同步进行中或出现异常时会自动创建恢复点。
> - 已同步状态下的周期性恢复点：复制对象进入已同步状态后，按照设定周期自动创建恢复点。

## 管理同步复制计划 > 创建同步复制计划 > 设置副本虚拟机

# 设置副本虚拟机

1. **设置副本虚拟机的命名规则**

   支持通过添加前缀或后缀的方式为副本虚拟机命名，也可选择与原虚拟机名称保持一致。
2. **设置副本虚拟机存储策略**

   支持为副本虚拟机的虚拟卷配置与原虚拟机不同的存储策略。

   | 参数 | 说明 |
   | --- | --- |
   | 冗余策略 | 可选择副本或纠删码。 **说明**：  - 若目标集群为双活集群，选择副本策略时仅支持选择 3 副本。 - 选择纠删码策略时，目标集群需满足纠删码的使用条件。 |
   | 置备方式 | 可选择厚置备或精简置备。 |
   | 静态数据加密 | 启用后，对于原虚拟机已加密的虚拟卷，将在创建副本虚拟机后继续加密。 |
3. **设置网络映射规则**

   在复制原虚拟机时，为确保在目标集群中创建的副本虚拟机能够正确连接到所需的网络环境，需要设置网络映射规则。映射规则可以将原虚拟机使用的虚拟机网络映射到目标集群中指定的虚拟机网络，以确保副本虚拟机能够正常连通网络。

   复制服务支持 VPC 和 VLAN 网络之间的灵活映射。您可以根据实际的网络环境和业务需求自由设置。使用 VPC 网络的虚拟机能够在保持 IP 和路由等网络配置不变的情况下快速完成虚拟机故障转移切换，实现业务在多站点的网络容灾和高可用。例如，源端虚拟机使用 VPC 网络时，推荐副本端网络映射为相同 VPC 及同一子网，若源端虚拟机发生故障时，能够在保持 IP 和路由等网络配置不变的情况下将业务快速切换至副本虚拟机，确保业务连续性和实现网络高可用。

   > **说明**:
   >
   > - 若原虚拟机后续使用了新的虚拟机网络，请通过编辑复制计划重新设置网络映射规则，否则对应的副本虚拟机将不会创建对应的网络设备。
   > - 若原虚拟机后续不再使用某个虚拟机网络，且复制计划中剩余的虚拟机均不使用该虚拟机网络，可通过编辑复制计划来删除对应的网络映射规则。
4. **设置 IP 映射规则（可选）**

   由于原虚拟机和副本虚拟机可能处于不同的网络环境，为确保副本虚拟机能够正确连接到新的网络环境，可以设置 IP 映射规则，将原虚拟机的 IP 地址映射到可在目标集群使用的 IP 地址。

   在进行故障转移时，复制服务会尝试通过原虚拟机的 IP 和子网掩码查找匹配的 IP 映射规则，并根据规则设置副本虚拟机的 IP 地址，确保副本虚拟机在启动后能够使用正确的网络配置。

   > **说明**:
   >
   > - IP 映射规则被成功应用需要满足下述条件：
   >   - 若原虚拟机网络为 VLAN，原虚拟机需配置静态 IP 且已安装虚拟机工具。
   >   - 若原虚拟机网络为 VPC，副本虚拟机网络为 VLAN，原虚拟机需安装虚拟机工具。
   >   - 原虚拟机需使用特定版本的客户端操作系统。已通过验证的操作系统为 CentOS 7.4、CentOS 8.4、Windows 8.1 Enterprise、Windows Server 2012 R2 Standard、Windows Server 2016 Standard。
   > - 支持添加多条 IP 映射规则。如设置了多条映射规则，将按照添加顺序采用匹配的第一条规则。
   > - IP 地址支持以子网掩码或者 CIDR 块的形式输入。若原虚拟机 IP 为单个 IP 地址，映射的副本虚拟机 IP 也需要为单个 IP 地址；若原虚拟机 IP 为 IP 段，映射的副本虚拟机 IP 也需要设置 IP 段。设置 IP 段时，请使用 `*` 号标识 IP 地址的第四位，如 `192.168.0.*`，映射后的 IP 地址第四位和原 IP 地址第四位将保持相同。
   > - 若原虚拟机和副本虚拟机均使用相同的 VPC 子网，则副本虚拟机将直接使用原虚拟机的 IP 地址，无需设置 IP 映射规则。
   > - 若原虚拟机网络类型为 VPC 且配置了多个 IP：当副本虚拟机网络为 VPC 时，IP 映射规则可对 VPC 网卡上的所有 IP 生效；当副本虚拟机网络为 VLAN 时，仅对 VPC 网卡上的首个 IP 生效。
5. **设置副本虚拟机 MAC 地址**

   设置副本虚拟机的 MAC 地址选项，默认为随机分配，也可选择与原虚拟机一致。

   > **注意**：
   >
   > 若副本虚拟机与原虚拟机处于同一 VPC 子网，则不支持将 MAC 地址配置为与原虚拟机地址一致。

## 管理同步复制计划 > 创建同步复制计划 > （可选）设置测试虚拟机

# （可选）设置测试虚拟机

测试虚拟机用于故障转移测试，如您需要进行故障转移测试，请您设置测试虚拟机。

1. **设置测试虚拟机的命名规则**

   支持通过添加前缀或后缀的方式为测试虚拟机命名，也可选择与原虚拟机名称保持一致。请为测试虚拟机设置与副本虚拟机不同的命名规则。
2. **设置网络映射规则**

   在故障转移测试时，为确保在目标集群中创建的测试虚拟机能够正确连接到所需的网络环境，需要设置网络映射规则。映射规则可以将原虚拟机使用的虚拟机网络映射到目标集群中指定的虚拟机网络，以确保测试虚拟机能够正常连通网络。

   复制服务支持 VPC 和 VLAN 网络之间的灵活映射。您可以根据实际的网络环境和业务需求自由设置。进行灾备演练时，推荐为测试端网络配置 VPC 网络类型，以快速创建一个与生产环境完全相同的沙箱网络。无需手动进行复杂的网络配置即可真实模拟业务灾难场景，同时实现生产环境零干扰的灾备演练。

   > **说明**:
   >
   > - 由于同一 VPC 下的网络默认互通，请为测试虚拟机选择和原虚拟机不同的 VPC。
   > - 若原虚拟机后续使用了新的虚拟机网络，请通过编辑复制计划重新设置网络映射规则，否则对应的测试虚拟机将不会创建对应的网络设备。
   > - 若原虚拟机后续不再使用某个虚拟机网络，且复制计划中剩余的虚拟机均不使用该虚拟机网络，可通过编辑复制计划来删除对应的网络映射规则。
3. **设置 IP 映射规则（可选）**

   由于原虚拟机和测试虚拟机可能处于不同的网络环境，为确保测试虚拟机能够正确连接到新的网络环境，可以设置 IP 映射规则，将原虚拟机的 IP 地址映射到可在目标集群使用的 IP 地址。

   故障转移测试时，复制服务会尝试通过原虚拟机的 IP 查找匹配的 IP 映射规则，并根据规则设置测试虚拟机的 IP，确保测试虚拟机在启动后能够使用正确的网络配置。

   > **说明**:
   >
   > - IP 映射规则被成功应用需要满足下述条件：
   >   - 若原虚拟机网络为 VLAN，原虚拟机需配置静态 IP 且已安装虚拟机工具。
   >   - 若原虚拟机网络为 VPC，测试虚拟机网络为 VLAN，原虚拟机需安装虚拟机工具。
   >   - 原虚拟机需使用特定版本的客户端操作系统。已通过验证的操作系统为 CentOS 7.4、CentOS 8.4、Windows 8.1 Enterprise、Windows Server 2012 R2 Standard、Windows Server 2016 Standard。
   > - IP 地址支持以子网掩码或者 CIDR 块的形式输入。如设置了多条映射规则，将按照添加顺序采用匹配的第一条规则。
   > - IP 地址支持输入单个 IP 或 IP 段。若原虚拟机 IP 为单个 IP 地址，映射的测试虚拟机 IP 也需要为单个 IP地址；若原虚拟机 IP 为 IP 段，映射的测试虚拟机 IP 也需要设置 IP 段。设置 IP 段时，请使用 `*` 号标识 IP 地址的第四位，如 `192.168.0.*`，映射后的 IP 地址第四位和原 IP 地址第四位将保持相同。
   > - 若原虚拟机网络类型为 VPC 且配置了多个 IP：当测试虚拟机网络为 VPC 时，IP 映射规则可对 VPC 网卡上的所有 IP 生效；当测试虚拟机网络为 VLAN 时，仅对 VPC 网卡上的首个 IP 生效。
4. **设置测试虚拟机 MAC 地址**

   设置测试虚拟机的 MAC 地址选项，默认为随机分配，也可选择与原虚拟机一致。

   > **注意**：
   >
   > 当故障转移测试网络与故障转移网络映射到相同 VPC 及子网时，不支持测试虚拟机与副本虚拟机同时保留原虚拟机的 MAC 地址。若两者均配置为与原虚拟机一致的 MAC 地址，将发生 MAC 地址冲突，导致故障转移测试失败。

## 管理同步复制计划 > 删除同步复制计划

# 删除同步复制计划

1. 在**同步复制计划**列表选择要删除的同步复制计划，在列表右侧或同步复制计划详情页单击 **...**，选择**删除**。

## 暂停或恢复同步任务

# 暂停或恢复同步任务

您可以对复制对象手动暂停和恢复同步任务。

> **说明**：
>
> 若为跨站点类型的同步复制计划，暂停或恢复同步仅允许在目标站点进行操作。

## 暂停同步

您可以对处于**同步中**或 **已同步**状态的复制对象执行暂停操作。

- 对处于**同步中**的复制对象暂停同步后，将取消源端和目标端之间的同步复制任务。
- 对处于**已同步**状态的复制对象暂停同步后，将取消源端和目标端的数据双写。

**前提条件**

- 复制对象当前不存在正在进行的故障转移任务。
- 原虚拟机未被删除、迁移或转化为虚拟机模板。

**操作步骤**

1. 在**副本虚拟机**列表选择需暂停同步的虚拟机，单击 **...**，选择**暂停同步**。
2. 在弹出的对话框中单击**暂停同步**。

您也可以进入虚拟机所属的同步复制计划的详情页面，在**复制对象**中，选择需暂停同步的虚拟机，在右侧单击**暂停同步**。

## 恢复同步

您可以对处于**暂停**状态的复制对象执行恢复操作。

**前提条件**

原虚拟机未被删除、迁移或转化为虚拟机模板。

**操作步骤**

1. 在**副本虚拟机**列表选择需恢复同步的虚拟机，单击 **...**，选择**恢复同步**。
2. 在弹出的对话框中单击**恢复同步**。

您也可以进入虚拟机所属的同步复制计划的详情页面，在**复制对象**中，选择需恢复同步的虚拟机，在右侧单击**恢复同步**。

## 故障处理 > 故障转移测试

# 故障转移测试

您可以在生产环境业务正常运行的情况下，通过故障转移测试生成测试虚拟机来模拟真实故障转移流程，从而验证灾备方案的有效性，确保在灾难发生时能够快速恢复关键业务。测试过程中产生的增量数据不会被同步回源端虚拟机，源端虚拟机可正常运行业务。

故障转移测试对虚拟机的同步复制任务、故障转移任务和故障恢复任务没有影响。

您可以对多个虚拟机批量进行故障转移测试，也可以对单个虚拟机进行故障转移测试。

**注意事项**

待进行故障转移测试的虚拟机复制对象需满足以下条件：

- 虚拟机复制对象的故障转移状态为`复制中`或`就绪`。

  若复制状态为`复制中`，则选择恢复点时不允许选择**立即生成恢复点**。
- 虚拟机复制对象当前不存在其他测试虚拟机，即未在其他复制计划中存在执行中的故障转移测试任务。
- 虚拟机复制对象所属的复制计划[已配置故障转移测试网络](/backup/2.3.0/vm_sync_replication_user_guide/vm_sync_replication_user_guide_46)。

## 故障处理 > 故障转移测试 > 执行测试 > 虚拟机批量执行测试

# 虚拟机批量执行测试

执行以下步骤对多个虚拟机进行批量执行故障转移测试。

**操作步骤**

1. 登录 **CloudTower**，在**备份与容灾**界面选择同步复制计划，单击页面右上角**故障处理**，并选择**故障转移测试**。
2. 如果存在入站复制对象，需选择待进行故障转移测试的虚拟机所属的站点作为源站点。
3. 选择需进行故障转移测试的虚拟机，支持按照虚拟机名称、所属集群和所属复制计划进行搜索。
4. 为每个虚拟机选择恢复点及设置启动顺序。

   - **恢复点**

     您需要为每个虚拟机选择恢复点，用于将测试虚拟机恢复到对应时间点的状态。

     - **立即生成恢复点**

       若复制对象当前同步复制状态为`已同步`，允许选择**立即生成恢复点**。
     - **选择历史恢复点**

       若复制对象当前未处于`已同步`状态，或需要基于历史数据状态进行验证时，请选择一个已有的历史恢复点。在故障转移测试任务创建成功后，复制服务将根据选择的历史恢复点进行故障转移测试。
   - **启动顺序**

     若存在多个测试虚拟机关联了同一个复制服务，支持调整测试虚拟机的启动顺序，并为每个测试虚拟机设置启动延迟，故障转移测试任务完成之后，复制服务将按照启动顺序依次启动测试虚拟机。

     - 设置启动顺序：拖动测试虚拟机以调整启动顺序。
     - 设置启动延时：启动延迟支持设置为 0 ~ 300 秒，默认为 60 秒。设置为 0 表示该测试虚拟机和上一个测试虚拟机同时启动，设置为 60 表示该测试虚拟机会在上一个测试虚拟机开始启动 60 秒后启动。
5. （可选）若存在使用了 VPC 网络的测试虚拟机，可选择为测试虚拟机设置浮动 IP，以确保外部网络可以稳定访问测试虚拟机提供的业务。

   > **说明**：
   >
   > 若原虚拟机网络类型为 VLAN，需安装虚拟机工具且配置静态 IP。
6. 单击**创建**。

> **说明**：
>
> 每个虚拟机的故障转移测试任务相互独立，无依赖关系。如果某个虚拟机故障转移测试任务失败，可以单独为该虚拟机重新创建故障转移测试任务。

## 故障处理 > 故障转移

# 故障转移

当原虚拟机发生故障时，可通过故障转移将原虚拟机切换至目标集群的副本虚拟机，由副本虚拟机接替原虚拟机提供服务。

您可以对多个虚拟机批量进行故障转移，也可以对单个虚拟机进行故障转移。

**注意事项**

- 待进行故障转移的虚拟机需满足以下条件：

  - 当虚拟机存在多个副本虚拟机时，需选择其中一个副本虚拟机进行故障转移，请确保其对应的复制对象的故障转移状态为`就绪`，且其余复制对象不处于`故障转移中`、`故障恢复中`和`永久故障转移中`状态。
  - 虚拟机当前没有正在执行的故障转移任务。
- 故障转移期间，如果原虚拟机从故障状态中恢复，为避免发生 IP 冲突，不允许对原虚拟机执行开机操作，请通过故障恢复操作完成副本虚拟机到原虚拟机的切换。

## 故障处理 > 故障转移 > 虚拟机批量故障转移

# 虚拟机批量故障转移

执行以下步骤对多个虚拟机进行批量故障转移。

**操作步骤**

1. 登录 **CloudTower**，在**备份与容灾**界面，选择**同步复制计划**，单击页面右上角**故障处理**，并选择**故障转移**。
2. 如果存在入站复制对象，需选择待进行故障转移的虚拟机所属的站点作为源站点。
3. 选择需进行故障转移的虚拟机，支持按照虚拟机名称、所属集群和所属复制计划进行搜索。

   > **说明**：
   >
   > 若虚拟机存在多个副本虚拟机，仅支持选择其中一个副本虚拟机进行故障转移。
4. 为每个虚拟机选择恢复点及设置启动顺序。

   - **恢复点**

     您需要为每个虚拟机选择恢复点，用于将副本虚拟机恢复到对应时间点的状态。

     - **立即生成恢复点**

       若复制对象当前同步复制状态为`已同步`，允许选择**立即生成恢复点**。在故障转移任务创建成功后，业务将直接切换至由副本虚拟机提供。
     - **选择历史恢复点**

       若复制对象当前未处于`已同步`状态，请选择一个已有的历史恢复点。在故障转移任务创建成功后，复制服务将根据选择的历史恢复点进行故障转移。

       > **注意**：
       >
       > 选择历史恢复点进行故障转移后，若后续进行故障恢复，复制服务将会把副本虚拟机在故障转移期间产生的数据同步回原虚拟机。此过程中，原虚拟机自历史恢复点至故障转移时刻的所有数据将被覆盖，导致该时间段内的数据丢失。
   - **启动顺序**

     若存在多个副本虚拟机关联了同一个复制服务，支持调整副本虚拟机的启动顺序，并为每个副本虚拟机设置启动延迟，故障转移任务完成之后，复制服务将按照启动顺序依次启动副本虚拟机。

     - 设置启动顺序：拖动副本虚拟机以调整启动顺序。
     - 设置启动延时：启动延迟支持设置为 0 ~ 300 秒，默认为 60 秒。设置为 0 表示该副本虚拟机和上一个副本虚拟机同时启动，设置为 60 表示该副本虚拟机会在上一个副本虚拟机开始启动 60 秒后启动。
5. 设置原虚拟机关机选项，支持选择**关机**或**强制关机**。

   > **说明**：
   >
   > 仅当所选择的虚拟机中存在虚拟机满足下述条件时，需选择关机选项：
   >
   > - 虚拟机存在：虚拟机未被删除、虚拟机未被转换成虚拟机模板、虚拟机未被迁移到其他集群、虚拟机未被移入回收站、所属集群未从 CloudTower 移除；
   > - 虚拟机未处于`关机`或`未知`状态；
   > - 虚拟机所属集群与 CloudTower 连接正常；
   > - 源站点连接状态为`已连接`。
6. （可选）若存在使用了 VPC 网络的副本虚拟机，可选择为副本虚拟机设置浮动 IP，以确保外部网络可以稳定访问副本虚拟机提供的业务。

   > **说明**：
   >
   > 若原虚拟机和副本虚拟机处于相同的 VPC 子网，则副本虚拟机的浮动 IP 默认和原虚拟机保持相同。若原虚拟机未启用浮动 IP，也可以手动为副本虚拟机设置浮动 IP。若原虚拟机网络为 VLAN，需安装虚拟机工具且配置静态 IP。
7. 单击**创建**。

副本虚拟机启动成功后，其对应的复制对象的故障转移状态即变更为`故障转移中`。若对应的同步复制计划中配置了 IP 映射规则且副本虚拟机存在匹配的 IP 映射规则，则复制服务将对副本虚拟机进行 IP 映射。

> **说明**：
>
> - 发起故障转移后，原虚拟机的同步复制任务将暂停，同步复制任务状态将变为已暂停。
> - 每个虚拟机的故障转移任务相互独立，无依赖关系。如果某个虚拟机故障转移失败，可以单独为该虚拟机重新创建故障转移任务。

   > - 虚拟机未被删除、未被转换成虚拟机模板、未被迁移到其他集群、未被移入回收站、所属集群未从 CloudTower 移除；
   > - 虚拟机未处于`关机`或`未知`状态；
   > - 虚拟机所属集群与 CloudTower 连接正常；
   > - 源站点连接状态为`已连接`。
7. 单击**创建**。

副本虚拟机启动成功后，其对应的复制对象的故障转移状态即变更为`故障转移中`，且若该副本虚拟机已配置 IP 映射规则，则复制服务将对副本虚拟机进行 IP 映射。

> **说明**：
>
> 发起故障转移后，原虚拟机的同步复制任务将暂停，同步复制任务状态将变为`已暂停`。

## 故障处理 > 故障恢复

# 故障恢复

故障转移后，若原虚拟机恢复运行，可通过故障恢复将故障转移期间副本虚拟机产生的数据同步回原虚拟机，并恢复至由原虚拟机提供服务。

您可以对单个虚拟机进行故障恢复，也可以对多个虚拟机批量进行故障恢复。

**前提条件**

虚拟机进行故障恢复需满足以下条件：

- 复制对象的故障转移状态为`故障转移中`。
- 原虚拟机存在且处于`关机`状态。
- 原虚拟机恢复运行后未挂载共享虚拟卷。
- 对于入站复制对象，源站点需处于`已连接`状态。
- 源集群和目标集群连接正常。

## 故障处理 > 故障恢复 > 虚拟机批量故障恢复

# 虚拟机批量故障恢复

执行以下步骤为多个虚拟机进行批量故障恢复。

**操作步骤**

1. 登录 **CloudTower**，在**备份与容灾**界面，选择同步复制计划，单击页面右上角的**故障处理**，并选择**故障恢复**。

   > **说明**：
   >
   > 仅当同步复制计划中存在`故障转移中`状态的复制对象时展示**故障恢复**的操作入口。
2. 如果存在入站复制对象，需选择要故障恢复的虚拟机所属的站点作为源站点。
3. 选择需要进行故障恢复的原虚拟机。支持按照虚拟机名称、所属集群和所属复制计划进行搜索。
4. 为每个原虚拟机设置启动顺序。

   若存在多个原虚拟机关联了同一个复制服务，支持调整原虚拟机的启动顺序，并为每个原虚拟机设置启动延迟，故障恢复任务完成之后，复制服务将按照启动顺序和启动延迟依次启动原虚拟机。

   - 启动顺序：拖动原虚拟机以调整启动顺序。
   - 启动延迟：支持设置为 0 ~ 300 秒，默认为 60 秒。设置为 0 表示该原虚拟机和上一个原虚拟机同时启动，设置为 60 表示该原虚拟机会在上一个原虚拟机开始启动 60 秒后启动。
5. （可选）若原虚拟机网络类型为 VPC，可选择为原虚拟机设置浮动 IP。
6. 单击**创建**。

创建故障恢复任务后，复制对象即进入`故障恢复中`状态。各个复制服务会将副本虚拟机在故障转移期间产生的数据变化增量同步回原虚拟机，在所有数据传输完成后，复制对象的故障转移状态变更为`就绪`状态，并按照设定的启动顺序和启动延迟依次启动原虚拟机，原虚拟机的同步复制任务将重新启动。

启动成功后，若故障转移所选恢复点记录的原虚拟机网络类型为 VLAN 且已安装虚拟机工具，或原虚拟机网络类型为 VPC，则其 IP 配置将恢复为该恢复点中记录的配置。

> **说明**：
>
> - 每个虚拟机的故障恢复任务相互独立，无依赖关系。如果某个虚拟机故障恢复失败，可以单独为该虚拟机重新创建故障恢复任务。
> - 若故障恢复任务失败，则复制对象的故障转移状态将由`故障恢复中`变更为`故障转移中`。
> - 支持在任务中心手动停止故障恢复任务。
> - 故障恢复完成后，同步复制将重新建立源端与副本端之间的同步关系，状态变更为`同步中`。

## 管理恢复点 > 删除恢复点

# 删除恢复点

根据恢复点的执行类型，恢复点可被自动删除或由用户手动删除。

- 执行类型为初始化生成、同步中生成、已同步生成和同步状态异常生成的恢复点，会根据恢复点保留策略进行保存，超过保留范围的恢复点会被自动删除，用户也可手动删除。
- 其他情况下生成的恢复点默认永久保留，仅支持手动删除。

> **注意**：
>
> 执行复制任务依赖最新的恢复点，因此无法删除最新的恢复点。该操作不可撤回，请谨慎操作。

**前提条件**

手动删除恢复点时，需保证复制对象处于`就绪`状态，且没有正在进行的故障转移任务。

**操作步骤**

1. 在**备份与容灾**界面，选择**同步复制计划**，选择一个同步复制计划，单击复制计划名称，进入复制计划详情页。
2. 选择**复制对象**，单击虚拟机名称，在右侧页面通过以下方式删除恢复点。

   - 单击某一恢复点右侧 **...**，选择**删除**，可删除该恢复点。
   - 勾选恢复点左侧复选框，单击右上角**删除**，可删除一个或多个恢复点。
   - 单击页面右侧 **...**，选择**删除全部恢复点**，可删除该复制对象除最新恢复点外的所有恢复点。
3. 在弹出的页面中，确认提示信息，单击删除按钮，完成删除。

## 查看虚拟机同步复制概况

# 查看虚拟机同步复制概况

在**备份与容灾**的**概览**页面中可查看被保护虚拟机的总体情况、复制计划的数量及状态、复制计划的执行记录概况、副本虚拟机及复制对象的状态信息、目标集群的存储空间信息、复制服务的健康状况，以及关联站点等信息。执行记录、复制计划、副本虚拟机页签包含站内、入站、出站复制计划的信息。

- **被保护资源**

  选择**虚拟机**标签，可查看当前 CloudTower 管理的所有 SMTX OS（ELF）集群中同步复制保护的虚拟机数量。
- **复制计划**

  选择**按复制类型**标签，可查看站内、入站、出站的同步复制计划数量。

  选择**按对象类型**标签，可查看被保护的虚拟机的数量。
- **副本虚拟机**

  选择**按复制类型**标签，可查看站内、入站、出站的复制计划已创建的副本虚拟机数量；

  选择**按故障转移状态**标签，可查看各故障转移状态下的副本虚拟机数量；

  选择**按同步状态**标签，可查看各同步状态下的副本虚拟机数量。
- **集群存储空间**

  查看所有目标集群的总容量、容量使用率和已使用容量。
- **复制服务**

  查看所有复制服务的信息，包括复制服务名称、状态（包括正在部署、可升级、正在升级、正在卸载、异常，未展示表示状态正常）、关联的集群数量、复制计划数量。
- **关联站点**

  查看当前站点已关联的站点，关联站点中的复制计划数量，以及关联站点的连接状态。

  > **注意**：
  >
  > 当站点间连接状态异常时，将无法获取出站复制计划的相关信息。
