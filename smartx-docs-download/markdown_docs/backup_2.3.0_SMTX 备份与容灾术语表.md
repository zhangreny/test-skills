## A-E

# A-E

备份服务
:   负责管理备份计划并执行虚拟机备份与恢复的系统组件。

备份窗口
:   允许备份计划自动执行的时间范围。

备份对象
:   被指定进行备份的虚拟机。

备份计划
:   用于定义备份对象，备份服务、备份周期、备份窗口、备份存储库、恢复点保留策略等关键资源的执行方案。

备份恢复点
:   每次备份操作生成的虚拟机状态快照，可用于回滚恢复。

备份周期
:   备份计划自动执行的频率和时间。

备份资源
:   与备份相关的所有实体，包括备份计划、虚拟机、备份文件、备份恢复点和存储库。

CloudTower
:   SmartX 多集群资源集中管理平台，可统一管理多种类型的 SmartX 集群及其资源。

测试虚拟机
:   原虚拟机的完整副本，通常用于灾备演练，验证故障转移流程的正确性以及数据一致性。

重建
:   一种恢复模式，可创建一个新的和原虚拟机生成恢复点时状态一致的虚拟机。

出站复制
:   跨站点复制的一种，其中源站点为当前站点，目标站点为其他关联站点。

存储卷
:   SMTX ZBS 块存储集群对外提供的最基本的存储单元。复制服务支持对 iSCSI LUN 进行复制和故障处理。

ELF
:   SmartX 基于 KVM 开发的原生虚拟化计算平台，提供虚拟机生命周期管理、电源操作、高可用、冷热迁移等基本功能，并能与 SmartX 设计研发的分布式块存储服务（ZBS）结合，提供完整的超融合虚拟化解决方案。

## F-J

# F-J

复制服务
:   负责管理复制计划并执行复制与故障处理任务的系统组件。

复制对象
:   进行虚拟机复制时，一个原虚拟机及其副本虚拟机称为一组复制对象；进行卷复制时，一个原卷及其副本卷称为一组复制对象。

复制窗口
:   允许异步复制计划自动执行的时间范围。

复制恢复点
:   同步复制、异步复制和故障处理任务生成的数据快照，用于被保护对象的回滚或恢复。

复制周期
:   异步复制计划自动执行的频率和时间。

复制资源
:   与复制相关的所有实体，包括复制计划、复制对象和恢复点。

副本对象
:   通过复制生成的副本虚拟机或副本卷。

副本卷
:   原卷的完整副本，通常用于容灾。当原卷不可用时，可以接替原卷提供业务数据。

副本虚拟机
:   原虚拟机的完整副本，通常用于容灾。当原虚拟机不可用时，可以接替原虚拟机继续提供服务。

浮动 IP
:   一种 1:1 NAT，用于将外部子网中的 IP 绑定给需要被外部主动连接的 VPC 内部虚拟机，效果等同于公有云中的弹性公网 IP。

故障处理
:   故障发生后可进行的恢复操作，包括故障转移、故障转移测试、故障恢复和永久故障转移。

故障恢复
:   当原对象恢复正常后，将副本对象在故障转移期间变化的数据回传至源端，并由原对象继续提供服务的机制。

故障转移
:   原虚拟机或存储卷故障时，切换至副本对象接管业务的机制。

故障转移测试
:   一种模拟虚拟机容灾演练的操作，可以在不中断源端生产业务的前提下，验证故障转移机制的有效性。

IP 映射
:   在故障转移或故障转移测试中，通过设定 IP 地址的映射关系自动为副本虚拟机配置正确 IP 地址的机制，确保网络连通。

## K-O

# K-O

跨站点复制
:   源集群与目标集群位于不同站点的复制方式。

目标端复制服务
:   与目标集群关联的复制服务，主要负责执行复制计划、故障转移、故障转移测试、管理复制资源，并在故障恢复时从目标集群读取数据传输给源端复制服务。

目标集群
:   通常为用于容灾的集群，用于存储从源集群复制过来的副本虚拟机，位于目标站点。

目标站点
:   通常为用于容灾的站点，用于存储从源站点复制过来的复制对象。

## P-T

# P-T

RPO
:   RPO（Recovery Point Objective）是容灾场景中允许的数据最大丢失时间范围。

RTO
:   RTO（Recovery Time Objective）是容灾场景中，系统从发生故障到恢复运行可容忍的最长时间。

SMTX OS 集群
:   SMTX OS 集群属于逻辑概念。在实际生产环境中，一个 SMTX OS 集群由至少 3 个运行了 SMTX OS 软件的节点通过网络互连组成。

SMTX ZBS 块存储集群
:   SMTX ZBS 块存储集群是由多个物理节点组成的存储资源池，通过分布式软件协同工作，为上层应用提供高性能、高可靠、可横向扩展的块级存储服务。

全量备份
:   将虚拟机全部数据完整备份到存储库的方式，通常用于初始备份。

全量复制
:   将复制对象的全部数据完整复制到目标端，通常用于首次复制。

入站复制
:   跨站点复制的一种，其中目标站点为当前站点，源站点为其他关联站点。

同步复制计划
:   用于同步复制中定义复制对象、复制方向、目标集群、复制服务、同步策略、副本虚拟机和测试虚拟机设置等关键资源的执行方案。

## U-Z

# U-Z

VPC
:   VPC（Virtual Private Cloud，虚拟专有云）是一个用户自定义的逻辑隔离网络空间。您可以在 VPC 中管理自己的子网结构、IP 地址范围和分配方式、网关服务、安全服务等。

VPC 子网
:   组成 VPC 的基础网络模块。同一 VPC 的虚拟机在 VPC 子网内可以互相通信，在 VPC 子网间也可以通信。

网络映射
:   将原虚拟机网络配置映射到目标集群对应网络的配置规则，用于解决源端和目标端网络环境不一致问题。

虚拟机
:   通过虚拟化技术实现，运行在虚拟化平台上的计算机系统，具有独立操作系统和资源。

压缩
:   对虚拟卷数据先执行压缩后再传输到备份存储库并存储为备份文件。

异步复制
:   数据非实时地，而是周期性地从源端传输到目标端的复制方式。

异步复制计划
:   用于定义异步复制中的复制对象，复制服务、复制周期、复制窗口、恢复点保留策略等关键资源的执行方案。

永久故障转移
:   当原对象遭遇严重故障且无法从故障中恢复时，副本端将正式接替原对象的服务角色，以确保业务的持续运行。

源端复制服务
:   与源集群关联的复制服务，主要负责从源集群读取虚拟机或存储卷数据并传输到目标端复制服务，并在故障恢复时接收目标端复制服务传输的数据并将数据写入源集群。

原机恢复
:   一种恢复模式，可将虚拟机的虚拟卷数据恢复到和生成恢复点时一致。

源集群
:   通常为生产集群，承载着实际运行的业务应用和数据，是原虚拟或原卷机所属的集群，位于源站点。

原卷
:   需要进行容灾保护的存储卷，负责实际承载业务数据的读写操作。

原虚拟机
:   需要进行容灾保护的虚拟机，负责承载实际的业务应用和数据。

增量备份
:   备份自上次备份任务以来发生变化的数据。

增量复制
:   复制自上次复制任务以来发生变化的数据。

站点
:   由单一 CloudTower 管理的一个故障域，可包含一个或多个集群和数据中心。站点与站点之间在逻辑和物理上都互相隔离。

源站点
:   通常为实际运行业务的站点。复制服务将源站点的复制对象复制到目标站点，实现数据的同步。

站内复制
:   源集群与目标集群在同一站点内的复制方式。
