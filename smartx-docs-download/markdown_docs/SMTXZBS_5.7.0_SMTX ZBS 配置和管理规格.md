# SMTXZBS/5.7.0/SMTX ZBS 配置和管理规格
## 关于本文档

# 关于本文档

本文档介绍了 SMTX ZBS（读作 SmartX ZBS）块存储集群和文件存储集群相关资源支持的最大参数值。如果相关配置有最佳实践，则本文档会给出最佳实践中推荐的配置供项目实施人员参考，否则，相应位置会写为 `-`。

---

## 文档更新信息

# 文档更新信息

**2025-12-01**：**文档随 SMTX ZBS 5.7.0 正式发布**。

相较于 SMTX ZBS 5.6.4，本版本主要更新如下：

- 更新物理节点配置要求；
- 更新系统资源占用说明；
- 新增仲裁节点的配置要求。

## 块存储集群配置 > 集群规格

# 集群规格

| **条目** | **最低配置要求** | **最大规格** | **推荐配置** |
| --- | --- | --- | --- |
| 服务器节点数量 | 3 | 255 | 不超过 64 个节点。推荐通过多个中小型的集群组成一个大的数据中心，限制软硬件错误域。 |
| 裸存储容量 | - | 6 PiB | - |

> **说明**：
>
> SMTX ZBS 块存储集群的物理盘池数量上限为 255，若采用多物理盘池，则集群最大节点数将小于 255。

## 块存储集群配置 > 集群节点配置要求 > 物理节点配置要求

# 物理节点配置要求

建议先明确集群节点的容量规格，以及集群采用存储分层模式还是存储不分层模式，再请确认集群内每个物理节点的配置满足以下要求。

> **注意**：
>
> - 同一集群内所有物理节点的 CPU 架构、CPU 供应商和容量规格必须相同。
> - 如需使用纠删码的冗余策略，必须选择存储分层模式。

# 标准容量规格（存储分层模式）

## 将 SMTX ZBS 块存储安装在独立的硬 RAID 1 中

建议使用[推荐配置](/smtxzbs/5.7.0/zbs_installation_guide_INTERNAL_EXTERNAL/zbs_installation_guide/zbs_installation_guide_82)的服务器，以获得 RAID 监控报警能力，再参考下表规划集群的存储设备。

| **项目** | **混闪配置、多种类型 SSD 全闪配置** | **单一类型且多种属性 SSD 全闪配置** | **单一类型且单一属性 SSD 全闪配置** |
| --- | --- | --- | --- |
| 缓存盘 | 至少 1 块 960 GB 及以上（DWPD ≥ 3）的数据中心级 SSD。建议使用 2 块 960 GB 及以上（DWPD ≥ 3）的数据中心级 SSD，以保证缓存盘的冗余。  全闪配置下，建议使用高性能 SSD 作为缓存盘。如：节点包含 NVMe 和 SATA 两种类型 SSD 时，使用 NVMe SSD 作为缓存盘。 | 至少 1 块 960 GB 及以上（DWPD ≥ 3）的数据中心级 SSD。建议使用 2 块 960 GB 及以上（DWPD ≥ 3）的数据中心级 SSD，以保证缓存盘的冗余。  建议使用高性能 SSD 作为缓存盘。如：节点包含高 DWPD 和低 DWPD 两种属性 SSD 时，使用高 DWPD 的 SSD 作为缓存盘。 | - |
| 数据盘 | - 混闪配置：建议配置 1 TB 及以上的 HDD。 - 全闪配置：建议配置 960 GB 及以上（DWPD ≥ 1）的数据中心级 SSD。 | 建议配置 960 GB 及以上（DWPD ≥ 1）的数据中心级 SSD。 | 至少 1 块 960 GB 及以上（DWPD ≥ 1）的数据中心级 SSD。  建议使用同一品牌和型号且容量规格相同的 SSD。 |
| 存储控制器 | - 1 张支持 JBOD 模式的 RAID 卡或 SAS HBA 卡。  **注意**：该存储控制器用于管理除 SMTX 系统盘外的所有物理盘。若 U.2 NVMe 硬盘通过 retimer 或 Switch 或者直接使用 AIC NVMe 转接至 PCIe 接口，也可以不使用存储控制器。 - 1 个支持 RAID 1 的存储控制器（例如 M.2 RAID 卡或 SAS RAID 卡），用于构建硬件 RAID 1。 | | |
| SMTX 系统盘 | 2 块 480 GB 及以上（DWPD ≥ 1）的 SSD 和存储控制器构建硬 RAID 1。  建议使用 M.2 SSD 构建硬 RAID 1，以获得更好的系统稳定性和数据安全性。 | | |

> **说明**:
>
> 单节点限制如下：
>
> - **数据盘**总容量不超过 128 TB。
> - **缓存盘**总容量不超过 51 TB。
> - 混闪配置、多种类型 SSD 全闪配置和单一类型且多种属性 SSD 全闪配置时：
>   - **缓存盘**总容量与**数据盘**总容量之比高于 10%。若计划在集群中启用常驻缓存模式，则在扣除常驻缓存容量后，**缓存盘**剩余容量与**数据盘**总容量之比仍需高于 10%。
>   - **缓存盘**和**数据盘**总数量不超过 64。
>   - 对于多种类型 SSD 全闪配置和单一类型且多种属性 SSD 全闪配置，若计划预留常驻缓存比例高于 75%，需要在部署块存储集群的[第 4 步：配置存储](/smtxzbs/5.7.0/zbs_installation_guide_INTERNAL_EXTERNAL/zbs_installation_guide/zbs_installation_guide_38)阶段，将所有**数据盘**的用途修改为**缓存盘**。
> - 单一类型且单一属性 SSD 全闪配置时，**数据盘**总数量不超过 32。

# 系统对主机 CPU 和内存的占用

根据主机配置和物理盘池数量的不同，以及是否启用 **NVMe-oF**，SMTX ZBS 块存储对 CPU 和内存资源的占用会相应变化，具体请参考下方表格。

> **说明**：
>
> - 若节点未启用超线程，系统将使用 CPU 的物理核心。
> - 若节点启用了超线程，系统将使用 CPU 的逻辑核心。然而，由于同一物理核心上的超线程逻辑核心之间存在资源竞争，另一颗逻辑核心的实际性能会有显著降低，故系统对 CPU 的占用实际应按照物理核心进行计算。
> - 当节点的 CPU 架构为鲲鹏 AArch64 时，系统额外占用 2 GB 内存。
> - 当节点缓存盘容量超过容量规格上限的 20% 时，每超过 8 TB 需要增加 2 GB 的内存占用。
> - SMTX ZBS 块存储集群**启⽤ NVMe over TCP** 或**启⽤ NVMe over TCP 和 RDMA** 时，对系统资源的要求相同。

| 单个文件系统容量 | 1 PiB | - |
| 单个文件系统的文件数量 | 10 亿 | - |
| 文件系统内单个文件的逻辑容量 | 4 TiB | - |
| 单个目录下一级子目录数量和文件数量之和 | 100 万 | - |

> **说明**：
>
> 由于文件系统需要占据文件控制器的磁盘槽位，且占用的槽位数与文件系统的容量有关，因此，实际可创建的文件系统数量由文件系统的容量和当前空闲的磁盘槽位数量共同决定。具体说明如下：
>
> - 每个文件控制器可提供 32 个磁盘槽位，总共可提供 32\*n 个槽位，其中 n 为文件控制器数量。
> - 单个文件系统所需的磁盘槽位数量 required\_slot 与文件系统容量 capacity、文件控制器数量 n 的关系为：
>   - 当 n ≤ capacity / 32 TiB 时，required\_slot = (⌈ capacity / 32 TiB ⌉ + 1) \* 2，其中 `⌈⌉` 代表向上取整。
>   - 当 n > capacity / 32 TiB 时，required\_slot = (n + 1) \* 2。
> - 当文件存储集群的空闲磁盘槽位数量不足时，无法继续创建文件系统。

## 文件存储集群配置 > 文件存储系统资源占用

# 文件存储系统资源占用

文件存储服务组件以容器的形式运行在 SMTX ZBS 集群的系统服务虚拟机中，这些虚拟机在文件存储集群中被称为文件控制器，单个文件控制器占用的系统资源如下。

| **项目** | **资源占用** |
| --- | --- |
| CPU | 取决于部署文件存储集群时为文件控制器分配的 vCPU 数量，至少占用 16 vCPU，至多占用 240 vCPU。当主机 CPU 架构为 Hygon x86\_64 时，文件控制器将独占 CPU 资源。  您可参考[系统总 CPU 占用说明](/smtxzbs/5.7.0/zbs_installation_guide_INTERNAL_EXTERNAL/zbs_installation_guide_109)小节，了解同时部署块存储集群、文件存储集群和其他系统服务时系统总体的 CPU 占用情况，以便合理规划主机计算资源。 |
| 内存 | 取决于部署文件存储集群时为文件控制器分配的内存数量，至少占用 16 GiB 内存。 |
| 存储空间 | 部署文件存储集群时，每个文件控制器会自动配置两块虚拟磁盘作为系统盘，系统盘的冗余策略为 2 副本，需占用 240 GiB 存储空间。   - 块存储集群为混闪配置为混闪配置或多种类型 SSD 全闪配置时，请登录 CloudTower 进入目标 SMTX ZBS 集群的管理界面，单击**设置**，选择**常驻缓存**，启用**允许数据常驻缓存**，每个文件控制器将占用该 SMTX ZBS 集群中 240 GiB 的空闲常驻缓存容量； - 块存储集群为单一类型 SSD 全闪配置时无法开启常驻缓存，文件控制器将占用该 SMTX ZBS 集群中 240 GiB 的可用存储容量。 |
