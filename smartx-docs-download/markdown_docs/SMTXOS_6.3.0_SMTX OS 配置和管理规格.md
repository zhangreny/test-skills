# SMTXOS/6.3.0/SMTX OS 配置和管理规格
## 关于本文档

# 关于本文档

本文档介绍 SMTX OS（读作 SmartX OS）支持的存储、计算、网络、硬件等资源的最大参数值。在具体使用 SMTX OS 的过程中，请不要超过相关最大值，否则， SMTX OS 有可能运行异常。本文档中介绍的最大参数可能被其他因素影响，例如硬件具体配置。

如果相关配置有相关最佳实践，则本文档会给出最佳实践中推荐的配置，供项目实施人员参考，否则，相应位置会留空。

## 物理集群配置

# 物理集群配置

| **条目** | **最低配置要求** | **最大规格** | **推荐配置** |
| --- | --- | --- | --- |
| 服务器节点数量 | 3 | 255 | 不超过 64 个节点。推荐通过多个中小型的集群组成一个大的数据中心，限制软硬件错误域。 |
| 裸存储容量 | - | 6 PiB | - |

> **说明**：
>
> SMTX OS（ELF）集群的物理盘池数量上限为 255，若采用多物理盘池，则集群最大节点数将小于 255。

## 集群节点配置要求

# 集群节点配置要求

本章节介绍了集群中节点的配置要求。

- 对于未启用双活特性的 SMTX OS 集群节点，以及双活集群的优先可用域、次级可用域节点的配置要求，请参考[物理节点配置要求](/smtxos/6.3.0/config_specs/elf_installation_guide/elf_installation_guide_59)。
- 对于双活集群的仲裁节点的配置要求，请参考[仲裁节点配置要求](/smtxos/6.3.0/config_specs/elf_multiactive_installation/elf_multiactive_installation_20)。
- 如需启用其他高级特性，除满足本章节的配置要求外，请参考[《SMTX OS 特性说明》](/smtxos/6.3.0/os_property_notes/preface-generic)并根据相应特性的硬件要求进行准备。

## 集群节点配置要求 > 物理节点配置要求

# 物理节点配置要求

建议先明确集群节点使用的容量规格，以及集群采用存储分层模式还是不分层模式，再确认集群内每个主机的配置满足以下要求。

> **注意**：
>
> - 同一集群内所有物理节点的 CPU 架构、CPU 供应商和容量规格必须相同。
> - 如需使用纠删码的冗余策略，必须选择存储分层模式。

# 标准容量规格（存储分层模式）

## 将 SMTX OS 安装在独立的硬 RAID 1 中

建议使用[推荐配置](/smtxos/6.3.0/elf_installation_guide/elf_installation_guide_73)的服务器，以获得 RAID 监控报警能力，再参考下表规划集群的存储设备。

| **项目** | **混闪配置、多种类型 SSD 全闪配置** | **单一类型且多种属性 SSD 全闪配置** | **单一类型且单一属性 SSD 全闪配置** |
| --- | --- | --- | --- |
| 缓存盘 | 每个物理盘池至少 1 块 960 GB 及以上（DWPD ≥ 3）的数据中心级 SSD。建议每个物理盘池使用 2 块 960 GB 及以上（DWPD ≥ 3）的数据中心级 SSD，以保证缓存盘的冗余。  全闪配置下，建议使用高性能 SSD 作为缓存盘。如：节点包含 NVMe 和 SATA 两种类型 SSD 时，使用 NVMe SSD 作为缓存盘。 | 每个物理盘池至少 1 块 960 GB 及以上（DWPD ≥ 3）的数据中心级 SSD。建议每个物理盘池使用 2 块 960 GB 及以上（DWPD ≥ 3）的数据中心级 SSD，以保证缓存盘的冗余。  建议使用高性能 SSD 作为缓存盘。如：节点包含高 DWPD 和低 DWPD 两种属性 SSD 时，使用高 DWPD 的 SSD 作为缓存盘。 | - |
| 数据盘 | - 混闪配置：建议配置 1 TB 及以上的 HDD。 - 全闪配置：建议配置 960 GB 及以上（DWPD ≥ 1）的数据中心级 SSD。 | 建议配置 960 GB 及以上（DWPD ≥ 1）的数据中心级 SSD。 | 每个物理盘池至少 1 块 960 GB 及以上（DWPD ≥ 1）的数据中心级 SSD。  建议使用同一品牌和型号且容量规格相同的 SSD。 |
| 存储控制器 | - 1 张支持 JBOD 模式的 RAID 卡或 SAS HBA 卡。  **注意**：该存储控制器用于管理除 SMTX 系统盘外的所有物理盘。若 U.2 NVMe 硬盘通过 retimer 或 Switch 或者直接使用 AIC NVMe 转接至 PCIe 接口，也可以不使用存储控制器。 - 1 个支持 RAID 1 的存储控制器（例如 M.2 RAID 卡或 SAS RAID 卡），用于构建硬件 RAID 1。 | | |
| SMTX 系统盘 | 2 块 480 GB 及以上（DWPD ≥ 1）的 SSD 和存储控制器构建硬 RAID 1。  建议使用 M.2 SSD 构建硬 RAID 1，以获得更好的系统稳定性和数据安全性。 | | |

> **说明**:
>
> - 单节点限制如下：
>
>   - **数据盘**总容量不超过 128 TB。
>   - 混闪配置、多种类型 SSD 全闪配置和单一类型且多种属性 SSD 全闪配置时，**缓存盘**总容量与**数据盘**总容量之比高于 10%。若计划在集群中启用常驻缓存模式，则在扣除常驻缓存容量后，**缓存盘**剩余容量与**数据盘**总容量之比仍需高于 10%。
> - 单物理盘池限制如下：
>
>   - **缓存盘**总容量不超过 51 TB。
>   - 混闪配置、多种类型 SSD 全闪配置和单一类型且多种属性 SSD 全闪配置时，**缓存盘**和**数据盘**总数量不超过 64。
>   - 多种类型 SSD 全闪配置和单一类型且多种属性 SSD 全闪配置时，若计划预留常驻缓存比例高于 75%，需要在部署集群的[第 4 步：配置存储](/smtxos/6.3.0/elf_installation_guide/elf_installation_guide_36)阶段，将所有**数据盘**的用途修改为**缓存盘**。
>   - 单一类型且单一属性 SSD 全闪配置时，**数据盘**总数量不超过 32。

> **说明**:

>   - 混闪配置、多种类型 SSD 全闪配置和单一类型且多种属性 SSD 全闪配置时，**缓存盘**、**含元数据分区的缓存盘**和**数据盘**总数量不超过 64。

# SMTX OS 对主机 CPU 和内存的占用

SMTX OS 对主机的系统资源占用情况如下：

> **说明**：
>
> - 若服务器未启用超线程，系统将使用 CPU 的物理核心；若启用了超线程，系统将使用 CPU 的逻辑核心。
> - 当节点缓存盘容量超过挂载的数据盘容量的 20% 时，每超过 8 TiB 需要增加 2 GB 的内存占用。
> - 若通过 CLI 命令将网络服务的 CPU 模式设置为 isolate\_normal 模式，系统还将额外占用 1 个 CPU 核心。
> - 若该节点为主节点，且 CPU 架构不为飞腾 AArch64，系统将额外占用 1 个 CPU 核心。
> - 若启用了加密加速功能，在单物理盘池、2 物理盘池和 4 物理盘池配置下，系统分别需要额外占用 2、4、4 个 CPU 核心。
> - 若启用深度防护功能，系统将额外占用 800 MB 内存；对于 x86\_64 架构且未启用 Boost 模式的主机，即使未开启深度防护，也需预留 800 MB 内存。
> - 集群启用双活特性时，对应的数值表示 SMTX OS 运行在优先和次级可用域的主机中时，系统需占用主机的 CPU 和内存资源。
> - `⌈⌉`为向上取整符号，其运算结果为大于或等于原数的最小整数，例如 ⌈3.2⌉ = 4。

## 系统资源占用 > VMware ESXi 平台 > 系统对物理盘存储空间的占用

# 系统对物理盘存储空间的占用

对于同一种容量规格，系统对每个节点中特定用途的物理盘占用的存储空间是固定的，如下表。

| 容量规格 | 物理盘用途 | 系统占用的存储空间 | 说明 |
| --- | --- | --- | --- |
| 标准容量规格 | 含元数据分区的缓存盘或含元数据分区的数据盘 | 305 GiB | 系统分区、元数据分区、Journal 分区和预留空间固定共占用 305 GiB 空间。 |
| 缓存盘或数据盘 | 20 GiB | Journal 分区固定占用 20 GiB 空间。 |
| 大容量规格 | 含元数据分区的缓存盘或含元数据分区的数据盘 | 660 GiB | 系统分区、元数据分区、Journal 分区和预留空间固定共占用 660 GiB 空间。 |
| 缓存盘或数据盘 | 20 GiB | Journal 分区固定占用 20 GiB 空间。 |

您可以参考下表了解集群采用不同的存储配置时每个节点的存储空间分配情况和逻辑可用容量。

| 存储配置 | 物理盘用途 | 存储空间分配说明 | 逻辑可用容量说明 |
| --- | --- | --- | --- |
| 混闪配置或多种类型 SSD 全闪配置 | 含元数据分区的缓存盘 | 系统占用一定的存储空间（如上表），剩余空间为缓存分区。 | 逻辑可用容量 = 数据分区用于存储数据的容量 × 空间利用率  **说明：**   - 若集群使用纠删码的冗余策略，则空间利用率 = K / (K + M)。 - 若集群使用副本的冗余策略，则空间利用率 = 1 / 副本数。 |
| 缓存盘 |
| 数据盘 | 数据分区。  **说明：** 若集群采用混闪配置，则数据盘（HDD）数据分区的 1 / 17 将用于存放校验信息，剩余空间用于存放数据。 |
| 单一类型 SSD 全闪配置 | 含元数据分区的数据盘 | 系统占用一定的存储空间（如上表），若集群采用存储分层模式，则剩余空间由缓存与数据共享，缓存分区和数据分区的比例为 1：9；若集群采用存储不分层模式，则剩余空间为数据分区。 |
| 数据盘 |

下面以集群采用如下配置为例展示逻辑可用容量的计算过程：

- 存储配置：单一类型 SSD 全闪配置
- 存储分层/不分层模式：分层模式
- 冗余策略：纠删码（2 + 1 配比）
- 节点容量规格：标准容量规格
- 每节点的含元数据分区的数据盘：2 块 960 GB（约为 894 GiB）的数据中心级 SSD
- 每节点的数据盘：1 块 960 GB（约为 894 GiB）的数据中心级 SSD

每个节点的逻辑可用容量 = [(894 - 305) × 2 + (894 - 20)] × 9 / (1 + 9) × 2 / (2 + 1) = 1231.2 GiB。

## 系统资源占用 > VMware ESXi 平台 > 系统对 SCVM 的 CPU 和内存的占用

# SMTX OS 对 SCVM 的 CPU 和内存的占用

标准容量规格和大容量规格下，SMTX OS 对 SCVM 的 CPU 和内存的占用相同。

| **是否启用高级特性** | **CPU 占用** | **内存占用** |
| --- | --- | --- |
| 不启用任何高级特性 | 7 核 | 30 GB + ⌈节点数据盘总容量（TB）/ 10 TB × 1.5 GB⌉ |
| 启用双活特性 | 8 核 | 31 GB + ⌈节点数据盘总容量（TB）/ 10 TB × 1.5 GB⌉ |
| 启用 RDMA 功能 | 9 核 | 30 GB + 3 GB（大页内存）+ ⌈节点数据盘总容量（TB）/ 10 TB × 1.5 GB⌉ |

> **说明**：
>
> - 表格中的内存占用为 SMTX OS 对 SCVM 的需求，实际部署时需在此基础上额外预留 2 GB 内存供 SCVM 自身使用。
> - 集群启用双活特性时，对应的数值表示 SMTX OS 运行在优先和次级可用域的 ESXi 主机的 SCVM 虚拟机中时，系统需占用 SCVM 的 CPU 和内存资源。
> - 当节点缓存盘容量超过挂载的数据盘容量的 20% 时，每超过 8 TiB 需要增加 2 GB 的内存占用。
> - `⌈⌉`为向上取整符号，其运算结果为大于或等于原数的最小整数，例如 ⌈3.2⌉ = 4。
