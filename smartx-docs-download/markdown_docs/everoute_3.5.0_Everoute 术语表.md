## A-E

# A-E

安全策略
:   采用三段式架构设计，允许针对不同策略对象（如安全组、虚拟机或全局流量）灵活定义入向和出向的白名单黑名单，实现对东西向流量的细粒度管控与安全隔离。

安全防护虚拟机（SVM）
:   安全防护虚拟机（Security Virtual Machine，SVM）是一个为所在主机上的虚拟机提供防病毒及网络包检测能力的虚拟机。

安全组
:   用于将具有相同安全需求的虚拟机、IP 地址或 Pod 等资源进行逻辑分组，通过统一配置安全策略，实现对组内成员出、入流量的集中控制。

API
:   API（Application Programming Interface，应用程序编程接口）是一些预先定义的函数，应用将自身的服务能力封装成 API，并开放给用户调用。

边缘网关
:   为虚拟专有云网络提供服务的一组边缘网关虚拟机，用于实现 VPC 与外部网络（底层网络或覆盖网络）的三层互通。

CIP
:   Client IP，客户端地址。

调度算法
:   决定将请求分发给哪个后端服务器。

底层网络
:   在数据中心环境中，提供从物理设备（服务器、存储设备、路由器或交换机）到其他物理设备的 IP 连接。

DNAT
:   DNAT（Destination Network Address Translation，目标地址转换）。
:   在网络负载均衡器功能中，使用 VIP 地址与客户端通信，负载均衡器虚拟机接收请求后，自动将请求的目标地址从 VIP 地址转为后端服务器的实际 IP 地址。
:   在虚拟专有云网络功能中，主要用于对外发布 VPC 内部的服务。当 VPC 内部需要提供对外服务时，外部地址发起主动连接访问 DNAT IP，DNAT 接收后根据设置的 DNAT 规则将连接转换至 VPC 内部。

对称策略
:   在自定义安全策略配置中，为确保通信正常，需要在源端设置允许发送流量的出规则，同时在目标端设置允许接收流量的入规则的配置机制。

Everoute
:   为企业云平台提供软件定义的网络和安全系列功能，包括分布式防火墙、负载均衡、虚拟专有云网络、容器网络等，同时支撑虚拟化和容器化应用，从而形成与虚拟机和容器一体化管理的网络和安全解决方案。

Everoute Controller
:   Everoute 控制器，提供网络与安全核心控制逻辑的计算与处理。

Everoute operator
:   Everoute 管理器，负责 Everoute 服务的运维管理、微分段安全策略的配置等。

## F-J

# F-J

分布式防火墙
:   分布式防火墙实现了基于零信任网络模型的微分段技术，解决数据中心虚拟网络安全面临的挑战，为业务提供更加灵活精细的网络安全保障。

浮动 IP
:   一种 1:1 NAT，用于将外部子网中的 IP 绑定给需要被外部主动连接的 VPC 内部虚拟机，效果等同于公有云中的弹性公网 IP。

覆盖网络
:   通过软件创建网络抽象层，使得在物理网络之上能够运行多个独立、离散的虚拟化网络层。

服务器池
:   负责为应用提供服务的一组后端服务器，负责接收并响应负载均衡器分发的请求。

服务器池组
:   两个服务器池建立主备关系，组成一个服务器池组。当主服务器池中的成员全部故障时，可自动切换至使用备服务器池中的成员，以提供后端服务器层面的高可用。

后端服务器
:   处理真实请求的后端服务器。

会话保持
:   一段时间内来自同一用户的多个连接都要进入同一个后端服务主机，无需进行负载均衡，网络负载均衡器中会根据 Source IP 来进行会话保持，即来自同一客户端 IP 的会话都发送到同一个后端服务器。

健康检查
:   由​​健康检查监视器负责检查后端服务器的健康状态，只有健康状态符合要求的后端服务器可以对请求进行接收和响应。

## K-O

# K-O

L2 网关
:   用于连接底层网络中的子网与 VPC 中的子网，使得底层网络的 VLAN 和 VPC 子网在逻辑上形成一个二层子网。

LB 实例
:   一组用于部署 LB 服务的虚拟机。

LB 实例组
:   两个 LB 实例建立主备关系，组成一个 LB 实例组，用于提供实例的高可用。

LB 虚拟机
:   在负载均衡中用于实际数据转发的虚拟机。

LIP
:   LIP（Local IP）指用于虚拟服务和后端服务器通信的地址。

## P-T

# P-T

Pod
:   Kubernetes 中创建和管理的最小可部署计算单元，代表了一个独立的应用程序运行实例，该实例可能由单个容器或者几个紧耦合在一起的容器组成。
:   本产品文档中的 Pod，如无特殊说明，均指已启用 EIC CNI 插件（即 SKS 1.3.1 及以下版本的 ECP CNI 插件）的 SKS 工作负载集群中的 Pod。

RIP
:   RIP（Real Server IP）指后端服务器的地址，请求经过负载均衡器处理后，会根据调度结果发送至实际的后端服务器。

深度包检测模块（DPI）
:   深度包检测模块（Deep Packet Inspection，DPI）包含防火墙和入侵防御/检测(IPS/IDS)功能，针对网络流检测出入站流量并保护操作系统和应用程序免受攻击。

SNAT
:   SNAT（Source Network Address Translation，源地址转换）。
:   在网络负载均衡器功能中，接收请求后，自动将客户端地址转换为负载均衡器虚拟机的固定 IP 地址（LIP），与后端服务器通信，转换后服务端将不会感知真实的客户端地址。
:   在虚拟专有云网络功能中，主要用于 VPC 内部共享 IP 访问外部网络。当 VPC 内部需要访问外部网络时，内部地址发起主动连接，SNAT 会将内部地址的私有 IP 转换为 SNAT IP。

隧道端点（TEP）
:   隧道端点（Tunnel Endpoint，TEP）是两个节点之间建立 Overlay 隧道的起点和终点。在数据传输过程中，这些端点的IP 地址被用于定义隧道的两端，分别作为外层 IP 数据包头部的源地址和目标地址。

## U-Z

# U-Z

VIP
:   VIP（Virtual IP）指虚拟服务的地址。每个虚拟服务中配置一个，负责接收客户端请求。

VPC 对等连接
:   两个 VPC 之间的网络连接，可以实现两个 VPC 之间的私网连通。

VPC 子网
:   组成 VPC 的基础网络模块。同一 VPC 的虚拟机在 VPC 子网内可以互相通信，在 VPC 子网间也可以通信。

外部子网
:   一个外部可路由的 IP 资源池，用于 VPC 网络与物理网络之间的映射。外部子网中的 IP 地址可以用作 VPC 的 NAT IP、浮动 IP、路由 IP 等。

网络负载均衡器
:   又称四层负载均衡器，运行在 OSI 模型中的第四层，通过报文中的 IP 地址和端口，按照特定的负载均衡调度算法分发请求。

微分段
:   零信任模型下的网络隔离技术，每个 APP 或 VM 都可以被视作独立的分段，每个分段将分段以外所有设备、使用者、网络区域都视为不可信任的潜在威胁。

虚拟分布式交换机（VDS）
:   虚拟分布式交换机（Virtual Distributed Switch，VDS）是 SMTX OS 或 SMTX ELF 利用虚拟化技术将一组物理连通的物理网口转化为虚拟化层的分布式交换机，为主机之间提供网络连接，并为处理各类型系统流量及业务流量提供基础。

虚拟服务
:   每个虚拟服务对应一个应用，负载均衡器在虚拟服务中负责接收所有客户端连接请求并在后端服务器之间分发。

虚拟专有云（VPC）
:   虚拟专有云（Virtual Private Cloud，VPC）是一个用户自定义的逻辑隔离网络空间。您可以在 VPC 中管理自己的子网结构、IP 地址范围和分配方式、网关服务、安全服务等。

虚拟专有云系统网络
:   即主机 TEP 网络，用于实现 VPC 关联集群节点间的隧道连接和数据交换。
