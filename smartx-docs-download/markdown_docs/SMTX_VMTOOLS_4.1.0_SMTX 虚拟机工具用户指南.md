## SMTX 虚拟机工具简介

# SMTX 虚拟机工具简介

SMTX 虚拟机工具是一款安装在客户机操作系统中的工具，通过提供更丰富的虚拟机信息展示和强大的虚拟机管理功能，显著提升虚拟机的管理效率。

- **查看客户机操作系统信息**

  在虚拟机详情面板中展示操作系统名称、内核版本等系统信息，以及主机名、IPv4 地址、IPv6 地址、子网掩码、网关地址及 DNS 服务器等配置信息。
- **查看虚拟机的性能数据**

  支持在虚拟机详情面板中展示虚拟机内部的 CPU 使用率、内存使用率和文件系统使用情况（包括挂载点路径、分区容量、分区使用量和分区使用率），以便更准确地监控虚拟机运行状态、规划存储容量。
- **修改客户机操作系统的配置信息**

  支持在虚拟机详情面板中修改配置信息（包括主机名、IPv4 地址、IPv6 地址、子网掩码、网关地址、DNS 服务器），并支持修改账户密码、重置 NTP 服务器以及恢复时同步时间。
- **创建文件系统一致性快照**

  支持在为虚拟机创建快照的同时创建文件系统一致性快照，保障数据一致性和完整性。
- **为虚拟机提供驱动**

  SMTX 虚拟机工具内置两类驱动：**virtio-win 驱动**和 **DS 驱动**。

  - virtio-win 驱动：主要用于提升 Windows 虚拟机的性能。
  - DS 驱动：用于安全防护虚拟机，在虚拟机启用防病毒功能时由系统自动部署。

如需了解上述各项功能的具体使用方法，请参考《SMTX OS 管理指南》或《SMTX ELF 管理指南》的**管理虚拟机**章节。

## 安装 SMTX 虚拟机工具

# 安装 SMTX 虚拟机工具

本章节介绍如何在 Windows 和 Linux 虚拟机中安装 SMTX 虚拟机工具。

---

## 安装 SMTX 虚拟机工具 > 准备工作 > 确认版本配套关系

# 确认版本配套关系

请参考《SMTX 虚拟机工具发布说明》的[版本配套说明](/smtx-vmtools/4.1.0/release_notes/release_notes_04)确认集群版本和虚拟机操作系统版本与 SMTX 虚拟机工具兼容。

---

## 安装 SMTX 虚拟机工具 > 准备工作 > 上传 SMTX 虚拟机工具映像

# 上传 SMTX 虚拟机工具映像

您可以在集群的**设置**界面为集群单独上传 SMTX 虚拟机工具 ISO 映像，也可以在 CloudTower 的**系统配置**界面上传可供多个集群共享的虚拟机工具 ISO 映像

- 在集群的**设置**界面为集群单独上传映像

  1. 进入 SMTX OS 集群的管理界面，单击**设置**，选择**虚拟机工具**。
  2. 单击界面右上角的**上传虚拟机工具 ISO 映像**，弹出上传对话框。
  3. 选择需要上传的虚拟机工具 ISO 映像。
  4. 单击**上传**，开始上传文件。
- 在 CloudTower **系统配置**界面上传共享映像

  1. 进入 CloudTower 的**系统配置**界面，选择**虚拟机工具**。
  2. 单击界面右上角的**上传虚拟机工具 ISO 映像**，弹出上传对话框。
  3. 选择需上传的虚拟机工具 ISO 映像，并选择目标集群。
  4. 单击**上传**，开始上传文件。

## 安装 SMTX 虚拟机工具 > 准备工作 > 挂载 SMTX 虚拟机工具映像

# 挂载 SMTX 虚拟机工具映像

**前提条件**

- 虚拟机处于`运行中`状态。
- 虚拟机已安装操作系统，且该操作系统与 SMTX 虚拟机工具兼容。
- 集群中已[上传虚拟机工具映像](/smtx-vmtools/4.1.0/vmtools_user_guide/vmtools_user_guide_03)。

**操作步骤**

1. 单击虚拟机详情面板右上角的 **...**，选择**挂载虚拟机工具映像**。
2. 在弹出的挂载虚拟机工具对话框中单击**载入 ISO 映像**。

---

## 安装 SMTX 虚拟机工具 > 在 Windows 系统中安装

# 在 Windows 系统中安装

下面以 Windows 7 为例，介绍在 Windows 系统中安装 SMTX 虚拟机工具的详细步骤。

**前提条件**

- 安装 SMTX 虚拟机工具需要具有管理员权限。
- Windows 的系统变量 path 中必须包含：`%SystemRoot%\System32\WindowsPowerShell\v1.0\`，否则将无法使用虚拟机工具的功能。

**注意事项**

- 不支持在 Server Core 模式下安装 SMTX 虚拟机工具。
- 若系统内已安装杀毒软件，请将 SMTX 虚拟机工具的安装目录 `C:\Program Files\svt` 添加至杀毒软件的白名单中，避免该工具被杀毒软件误删。

**操作步骤**

1. 在虚拟机详情面板中单击**打开终端**，以管理员账户登录虚拟机的 Windows 操作系统。
2. 打开**我的电脑**，查看**文件管理器**。
3. 打开加载了虚拟机工具 ISO 映像的 CD-ROM，进入安装 ISO 映像所在的目录。
4. 运行 **SMTX\_VM\_TOOLS\_INSTALL.exe** 文件，开始安装虚拟机工具。
5. 打开任务管理器，选择**服务**页签，检查 SVT 的系统服务运行状态，若显示为**正在运行**，表明 SMTX 虚拟机工具安装成功。

安装完成后，可在 `C:\Program Files\svt\install.log` 路径查看安装日志。在虚拟机详情面板右上方点击 **...**，选择弹出虚拟机工具映像，虚拟机工具映像将被弹出。

## 安装 SMTX 虚拟机工具 > 在 Linux 系统中安装

# 在 Linux 系统中安装

下面以在 CentOS Linux 7.5 为例，介绍在 Linux 系统中安装 SMTX 虚拟机工具的步骤。

**前提条件**

- 安装 SMTX 虚拟机工具需要具有 root 权限。
- 虚拟机已具备 USB 驱动。

**操作步骤**

1. 以 **root** 用户身份 SSH 到虚拟机，或者使用普通用户 SSH 到虚拟机后，使用 **sudo** 执行后续的安装命令。

   `ssh root@<VM's IP address>`
2. 在客户机操作系统内挂载 ISO 映像。

   `mkdir /mnt/<dir_to_mount_iso>`

   `mount /dev/<vmtools_iso_cdrom> /mnt/<dir_to_mount_iso>`

   其中 **dir\_to\_mount\_iso** 表示计划用来挂载 iso 映像的目录名称，**vmtools\_iso\_cdrom** 指在[挂载虚拟机工具映像](/smtx-vmtools/4.1.0/vmtools_user_guide/vmtools_user_guide_04)时，挂载 ISO 映像的 CD-ROM 的设备名称（如无法确定设备名称，可以通过 **lsblk** 命令来查看此设备名称），例如：

   `mkdir /mnt/cdrom`

   `mount /dev/sr0 /mnt/cdrom`
3. 执行安装脚本。

   `bash /mnt/<dir_to_mount_iso>/SMTX_VM_TOOLS_INSTALL.sh`

   - 若安装成功，则会看到 `SVT is installed successfully!` 的提示;
   - 若安装失败，且出现错误提示：`Uncompressing SMTX VM ToolsExtraction failed.`，请安装 `tar` (例如：`yum -y install tar`)，然后重新执行安装脚本。
4. 运行 `systemctl status SVT` 命令，检查虚拟机工具的系统服务是否正在运行。
5. 如果看到如下提示，则表明成功运行：

   ```
   [root@scvm152 18:02:02 bin]$systemctl status SVT
   SVT.service
   Loaded: loaded (/usr/lib/systemd/system/SVT.service; enabled; vendor preset: disabled)
   Active: active (running) since Fri 2019-09-27 13:38:53 CST; 4h 25min ago
   ```

安装完成后，可在 `/opt/svt/install.log` 路径查看安装日志。在虚拟机详情面板右上方点击 **...**，选择弹出虚拟机工具映像，虚拟机工具映像将被弹出。

## 升级 SMTX 虚拟机工具 > 单独升级 > 一键升级

# 一键升级

当虚拟机所属集群满足以下版本，且关联的 CloudTwower 版本为 4.4.2 及以上或 4.6.0 及以上时，支持对 SMTX 虚拟机工具进行一键自动升级。

- SMTX OS 5.1.5 及以上的 5.1.x 版本
- SMTX OS 6.3.0 以及上的 6.3.x 版本
- SMTX ELF 6.3.0 及以上的 6.3.x 版本

**前提条件**

- 已获取新版本的 SMTX 虚拟机工具映像（ISO 映像文件）。
- SMTX 虚拟机工具处于`运行中`状态。

**操作步骤**

1. 参考[上传虚拟机工具映像](/smtx-vmtools/4.1.0/vmtools_user_guide/vmtools_user_guide_03)小节，上传目标版本的 SMTX 虚拟机工具映像至虚拟机所在集群。
2. 单击虚拟机详情面板右上方的 **...**，选择**升级虚拟机工具**。
3. 在弹出的对话框中单击**升级**，虚拟机将自动挂载 SMTX 虚拟机工具映像，并在虚拟机内自动执行升级操作。

升级完成后，虚拟机将自动弹出 SMTX 虚拟机工具映像。

---

## 升级 SMTX 虚拟机工具 > 单独升级 > 手动升级

# 手动升级

当虚拟机所在集群关联的 CloudTower 版本为 4.4.1 及以下、4.5.0 及以下或 3.x.x 版本时，不支持一键自动升级 SMTX 虚拟机工具，您可以参考以下步骤手动升级。

**前提条件**

- 已获取新版本的 SMTX 虚拟机工具映像（ISO 映像文件）。
- 手动升级 SMTX 虚拟机工具需要具有 root 权限或管理员权限。

**操作步骤**

1. 参考[上传虚拟机工具映像](/smtx-vmtools/4.1.0/vmtools_user_guide/vmtools_user_guide_03)小节，上传目标版本的 SMTX 虚拟机工具映像至虚拟机所在集群。
2. 单击虚拟机详情面板右上方的 **...**，选择**升级虚拟机工具**。
3. 在弹出的对话框中单击**载入 ISO 映像**，目标版本的 SMTX 虚拟机工具映像将载入虚拟机。
4. 参考[在 Windows 系统中安装](/smtx-vmtools/4.1.0/vmtools_user_guide/vmtools_user_guide_05)或[在 Linux 系统中安装](/smtx-vmtools/4.1.0/vmtools_user_guide/vmtools_user_guide_06)小节，在操作系统中安装目标版本的虚拟机工具。

升级完成后，在虚拟机详情面板右上方点击 **...**，选择弹出虚拟机工具映像，虚拟机工具映像将被弹出。

---

## 升级 SMTX 虚拟机工具 > 批量升级

# 批量升级

当虚拟机所属集群满足以下版本，并且关联的 CloudTower 为 4.8.0 及以上版本时，支持对 SMTX 虚拟机工具进行批量升级。

- SMTX OS 5.1.5 及以上的 5.1.x 版本
- SMTX OS 6.3.0 以及上的 6.3.x 版本
- SMTX ELF 6.3.0 及以上的 6.3.x 版本

**前提条件**

- 已获取新版 SMTX 虚拟机工具映像（ISO 映像文件）。
- 待升级的 SMTX 虚拟机工具均处于`运行中`状态。
- 待升级虚拟机中不存在由复制计划创建的副本虚拟机。

**操作步骤**

1. 参考[上传虚拟机工具映像](/smtx-vmtools/4.1.0/vmtools_user_guide/vmtools_user_guide_03)小节，上传目标版本的 SMTX 虚拟机工具映像至虚拟机所在集群。
2. 在虚拟机列表中勾选需要升级 SMTX 虚拟机工具的虚拟机。
3. 单击列表右上方的 **...**，选择**升级虚拟机工具**。
4. 在弹出的**升级虚拟机工具**对话框中确认是否执行批量升级操作：

   - **全部可升级**：单击**升级**，为全部选中虚拟机执行升级操作。
   - **部分可升级**：单击**部分升级**，系统将跳过不可升级的虚拟机，仅升级可升级的虚拟机；也可单击**取消**并在解决问题后重试。
   - **全部不可升级**：单击**关闭**，根据提示处理后重试。

升级开始后，系统会自动挂载 ISO 映像并执行升级。升级完成后，将自动弹出虚拟机工具映像。

## 卸载 SMTX 虚拟机工具

# 卸载 SMTX 虚拟机工具

本章节介绍如何在 Windows 和 Linux 虚拟机中卸载 SMTX 虚拟机工具。

---

## 卸载 SMTX 虚拟机工具 > 在 Windows 系统中卸载

# 在 Windows 系统中卸载

下面以在 Windows 7 为例，介绍在 Windows 系统中卸载 SMTX 虚拟机工具的步骤。

**前提条件**

卸载 SMTX 虚拟机工具需要具有管理员权限。

**操作步骤**

1. 单击 Windows **开始**按钮，搜索**控制面板**。
2. 打开**控制面板**后，双击**程序与功能**，在程序列表中选中 SMTX 虚拟机工具的相关选项并选择**卸载**。

卸载完成后，可在 `C:\Program Files\svt\uninstall.log` 路径查看卸载日志。

---

## 卸载 SMTX 虚拟机工具 > 在 Linux 系统中卸载

# 在 Linux 系统中卸载

下面以在 CentOS Linux 7.5 为例，介绍在 Linux 系统中卸载 SMTX 虚拟机工具的步骤。

**前提条件**

卸载 SMTX 虚拟机工具需要具有 root 权限。

**操作步骤**

1. 以 **root** 用户身份 SSH 到虚拟机，或者使用普通用户 SSH 到虚拟机后，使用 **sudo** 执行后续的安装命令。

   `ssh root@<VM's IP address>`
2. 执行卸载脚本：

   `/opt/svt/uninstall_svt.sh`
3. 如果界面输出 `SVT is uninstalled successfully` 的提示，则表示虚拟机工具已卸载。

卸载完成后，可在 `/opt/svt/uninstall.log` 路径查看卸载日志。
