## 版本更新说明

# 版本更新说明

## 新增

- 适配 CloudTower 4.8.0 的高可用功能。
- 支持在对内核升级、SCVM 扩容、系统分区扩容等场景下进行集群数据风险检查。
- 支持在内核升级子步骤失败时进行断点重试。
- 支持在扩容 SCVM 内存和 CPU 时，自动关闭 SCVM 的同步时钟与日志记录。

## 修复问题

- 修复了在集群内核版本不一致以及升级集群失败场景下，非预期显示**查看记录**的问题。
- 修复了升级中心 Webhook 通知发送成功，但界面显示消息发送失败的问题。
- 修复了在低版本的 Firefox 与 Safari 浏览器中打开升级中心弹窗时，时间显示异常的问题。
- 修复了集群概览页面中集群卡片错误显示**可升级**状态的问题。

## 版本配套说明

# 版本配套说明

## 与 CloudTower 版本的配套说明

升级中心是部署在 CloudTower 中的一项服务，CloudTower 4.8.0 默认携带本版本的升级中心。您也可以将 1.0.0 ～ 1.2.0 版本的升级中心升至 1.2.1，以使用本版本的升级中心服务。

## 与服务器架构的配套说明

本版本升级中心支持对部署在 Intel x86\_64、AMD x86\_64、Hygon x86\_64 和鲲鹏 AArch64 架构的服务器上的集群进行升级。

## 与集群版本的配套说明

升级中心支持对 SMTX OS、SMTX ZBS 和 SMTX ELF 集群进行升级和安装补丁操作。具体的升级路径请参考下表，其中：

- **`Y`** 表示通过升级 ISO 升级；
- **`Y(1)`** 表示通过安装补丁文件升级；
- **`-`** 表示不支持该路径。

当集群主版本存在多个补丁版本时，建议升级至最新的补丁版本，以获得全部修复和优化，提升系统稳定性。

## 版本使用注意事项

# 版本使用注意事项

- 升级中心仅支持集群使用适配升级中心的升级 ISO 文件进行升级，不可使用集群的安装 ISO 文件进行升级。
- 升级中心仅支持使用 ISO 格式的 vGPU 驱动文件升级 GPU 驱动，不可使用 RPM 格式的驱动文件。
- 升级中心不支持升级 NVIDIA L20 的 vGPU 驱动，需要手动升级驱动，具体步骤请参考对应版本的升级指南。
- 对于 SMTX OS 6.3.0 或 SMTX ELF 6.3.0 以下版本的集群，若当前 vGPU 驱动版本为 11.9 或 12.4，则不支持在升至 6.3.0 版本内核的同时升级 vGPU 驱动。请先将集群升级至 6.3.0 及以上版本，并将内核升级至对应的配套版本；随后，参考对应版本特性说明中的 **vGPU 功能**小节，卸载现有 vGPU 驱动并安装新版本驱动。
- 对于 SMTX OS 6.2.0 及以下版本集群，扩容系统分区后，新增容量将分配给 lsm2db。若扩容前系统分区使用率超过 78%，扩容后系统分区的使用率可能超过 90%，从而触发系统分区空间使用率超过阈值的报警。建议先清理文件释放系统分区空间；若无法进一步释放，可参考《CloudTower 使用指南》，手动调整系统分区使用率的报警阈值，以消除该报警。
