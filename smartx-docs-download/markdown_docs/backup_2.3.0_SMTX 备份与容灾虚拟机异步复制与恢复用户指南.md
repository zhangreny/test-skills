# backup/2.3.0/SMTX 备份与容灾虚拟机异步复制与恢复用户指南
## 关于本文档

# 关于本文档

本文档介绍了如何使用 SMTX 备份与容灾的复制服务对 SMTX OS（ELF）集群中的虚拟机进行异步复制和故障处理。

参考本文档使用虚拟机异步复制相关功能前，请您先参考《SMTX 备份与容灾-安装与运维指南》完成复制服务的部署。

## 概述

# 概述

异步复制是一种高效容灾的方案。该功能会在目标集群为原虚拟机创建副本，并按照预设的复制计划，利用快照技术周期性地将原虚拟机的增量数据同步至副本虚拟机。利用定期生成的恢复点，当原虚拟机发生故障时，可通过副本虚拟机快速恢复业务，实现分钟级恢复点目标（RPO）。

异步复制同时支持在原虚拟机发生故障时，进行故障转移、故障转移测试、故障恢复和永久故障转移。

## 概述 > 功能简介

# 功能简介

- **虚拟机异步复制**

  将运行在 SMTX OS（ELF）集群中的虚拟机复制到另一个 SMTX OS（ELF）集群中，并在目标集群创建对应的副本虚拟机。通过配置复制计划，系统可周期性地将原虚拟的增量数据同步给副本虚拟机，并生成恢复点。

  ![](https://cdn.smartx.com/internal-docs/assets/73b10dc3/vm_replication_user_guide_01.png)
- **故障转移**

  故障转移是指当原虚拟机发生故障时，可使用副本虚拟机接替原虚拟机继续提供服务。根据应用的场景不同，可以分为计划外故障转移和计划内故障转移。

  ![](https://cdn.smartx.com/internal-docs/assets/73b10dc3/vm_replication_user_guide_02.png)

  - **计划外故障转移**

    计划外故障转移是生产环境发生意外或故障时立即启动的应急措施，通常应用于突发性灾难，如硬件故障或网络中断等紧急情况，为业务提供快速、安全、可靠的灾备保障。当原虚拟机发生故障时，复制服务可将副本虚拟机回滚至指定的恢复点状态。这种故障转移方案能够在灾难发生时实现分钟级别的 RPO 和 RTO，使业务中断影响最小化。
  - **计划内故障转移**

    计划性故障转移是根据预先设定的计划进行的有序转移。在进行灾备演练、计划中的数据中心维护或升级、或预计将发生灾难的情况下，复制服务可以执行计划性故障转移。首先，系统将对原虚拟机进行一次复制，将最新的数据同步至副本虚拟机，然后将原虚拟机的服务切换至副本虚拟机。
- **故障转移测试**

  故障转移测试用于在不影响生产业务的情况下，验证故障转移流程是否符合预期。系统会基于选定的恢复点，在目标端测试环境中生成测试虚拟机，模拟真实的业务切换流程，且测试过程中产生的数据不会回写到源端。

  ![](https://cdn.smartx.com/internal-docs/assets/73b10dc3/vm_replication_user_guide_22.png)
- **故障恢复**

  故障恢复是在进行故障转移后，当原虚拟机恢复正常时，将故障转移期间副本虚拟机产生的数据增量同步回原虚拟机，并恢复至由原虚拟机提供服务，可确保原虚拟机在恢复后能够无缝切换回原始的生产环境，保障业务的连续性。

  ![](https://cdn.smartx.com/internal-docs/assets/73b10dc3/vm_replication_user_guide_03.png)
- **永久故障转移**

  永久故障转移是在进行故障转移后，当原虚拟机无法恢复时，将原虚拟机永久切换到副本虚拟机，并使用副本虚拟机持续提供服务。

  ![](https://cdn.smartx.com/internal-docs/assets/73b10dc3/vm_replication_user_guide_04.png)

## 概述 > 应用场景

# 应用场景

异步复制可应用于以下场景：

- **灾难恢复**

  防止因自然或人为灾害、勒索软件攻击、恐怖事件等导致的大规模数据损坏和丢失。一旦灾难发生，可快速将业务切换至灾备集群，并迅速恢复业务运行。
- **灾备演练**

  通过故障转移测试进行灾备演练，模拟灾难发生后的快速恢复过程，确保灾备方案的有效性和可靠性，验证恢复点的可恢复性。
- **数据中心迁移**

  结合复制和故障转移功能，将业务系统和数据迁移到灾备中心，实现零恢复点目标（RPO）的业务切换。完成切换后，即可对源数据中心硬件实施物理搬迁，确保业务系统与底层硬件设施解耦。
- **数据复用**

  利用集群原生的快照重建功能实现数据复用。通过从副本虚拟机的快照重建虚拟机进行开发、测试、故障排除等活动，避免对生产环境造成任何影响。数据复用功能能够在不影响主要业务的前提下，为开发人员和测试团队提供高效且隔离的环境，加快开发和测试过程，同时确保生产环境的稳定性和安全性。

## 概述 > 产品特性

# 产品特性

异步复制具有如下特性：

- 易用性：复制策略自动化和执行记录可视化，简化运维和管理。
- 灵活性：提供多样的复制选项，支持自定义复制周期、复制窗口、保留策略，满足不同类型业务的 SLA 要求。
- 高可用性：当复制服务所在节点发生故障时，复制服务可自动在其他节点拉起并继续提供服务。
- 负载控制：支持自定义同时执行的任务数量、任务重试次数、任务传输速率限制对复制服务进行负载控制。

## 相关概念

# 相关概念

本章解释了手册中使用的专业术语。在阅读过程中，如果遇到不熟悉的概念，可在此快速查找其定义和详细解释。

**复制服务**

复制服务负责管理异步复制计划，并执行虚拟机的异步复制、故障转移、故障转移测试、故障恢复和永久故障转移等任务调度。

根据关联的集群不同，复制服务分为目标端复制服务和源端复制服务。

- **目标端复制服务**

  与目标集群关联的复制服务，主要负责执行异步复制计划、故障转移、故障转移测试、管理复制资源，并在故障恢复时从目标集群读取数据传输给源端复制服务。目标端复制服务负责确保复制数据的同步性，保障副本虚拟机与原虚拟机的数据一致性。
- **源端复制服务**

  与源集群关联的复制服务，主要负责从源集群读取虚拟机数据并传输到目标端复制服务，并在故障恢复时接收目标端复制服务传输的数据并将数据写入源集群。

**异步复制计划**

用于规划和安排数据的复制操作。通过异步复制计划，用户可以定义复制对象、复制类型，复制站点、目标集群、复制服务、复制周期、复制窗口、恢复点保留策略、测试虚拟机和副本虚拟机设置等信息。

**站点**

站点是指由单一 CloudTower 管理的一个故障域，可以包含一个或多个集群和数据中心。每个站点的资源都由其专属的 CloudTower 进行管理，且在地理位置、硬件设备和软件服务等方面完全隔离，不与其他站点共享任何资源。

站点可以分为以下两种类型：

- **源站点**：通常为实际运行业务的站点。复制服务将源站点的原虚拟机复制到目标站点的副本虚拟机，实现数据的同步。
- **目标站点**：通常为用于容灾的站点，用于存储从源站点复制过来的副本虚拟机。

**站内复制**

源集群和目标集群位于同一站点。当原虚拟机发生故障时，可切换使用站点内其他集群中的副本虚拟机，实现集群级别的容灾。若管理站点的 CloudTower 发生故障，无法发起故障转移。

**跨站点复制**

源集群和目标集群位于不同站点。当源站点内的原虚拟机不可用时，无论管理源站点的 CloudTower 是否发生故障，均可切换至目标站点的副本虚拟机，实现站点级别的容灾。

跨站点复制包括以下两种类型：

- **入站复制**：目标站点为当前站点，源站点为其他关联站点。
- **出站复制**：源站点为当前站点，目标站点为其他关联站点。

**源集群**

通常为生产集群，承载着实际运行的业务应用和数据，是原虚拟机所属的集群，位于源站点。

**目标集群**

通常为用于容灾的集群，用于存储从源集群复制过来的副本虚拟机，位于目标站点。

**虚拟机**

通过虚拟化技术实现，运行在物理服务器虚拟化平台上的计算机系统，具有与物理计算机相同的特性。物理服务器上的所有虚拟机共享底层的物理硬件资源，但每个虚拟机拥有独立的操作系统、应用程序和资源配置。虚拟机之间相互隔离，运行互不影响，可用于测试、开发和生产环境。

对虚拟机进行复制时，涉及三种虚拟机角色：

- **原虚拟机**

  需要进行容灾保护的虚拟机，负责承载实际的业务应用和数据。
- **测试虚拟机**

  通过故障转移测试任务创建的原虚拟机的完整副本，具有和原虚拟机相同的数据和配置，通常用于灾备演练，验证故障转移流程的正确性以及数据一致性。
- **副本虚拟机**

  通过全量复制任务创建的原虚拟机的完整副本，具有和原虚拟机相同的数据和配置。副本虚拟机通常用于容灾，当原虚拟机不可用时，可以接替原虚拟机继续提供服务。

**复制资源**

指复制对象、异步复制计划和恢复点。

**复制对象**

一个原虚拟机及其对应的副本虚拟机称为一组复制对象。

**复制周期**

异步复制计划自动执行的频率和时间，用于指定复制服务对虚拟机进行数据复制的时间间隔。较短的复制周期可以实现更小的 RPO。例如，如果复制周期设置为 15 分钟，那么在发生灾难性事件时，最多可能丢失 15 分钟的数据。

**复制恢复点**

每次为虚拟机执行异步复制、故障转移测试（立即生成恢复点模式）、故障转移（立即生成恢复点模式）或故障恢复时生成的数据状态，以快照形式保存在副本虚拟机上。

恢复点可以使虚拟机在需要时快速回滚到特定的数据状态，例如在灾难恢复场景中将虚拟机恢复到某个先前的时间点，以恢复虚拟机的正常运行。通过定期生成恢复点，可以确保虚拟机的数据是最新的。

**复制窗口**

允许异步复制计划自动执行的时间范围。

**全量复制**

将虚拟机的所有数据，包括操作系统、应用程序、文件和配置等，复制到副本虚拟机，从而确保副本虚拟机与原虚拟机完全一致。全量复制通常发生在初次复制时，用于建立虚拟机的初始副本，并为后续的增量复制提供基准。

**增量复制**

复制自上次复制后发生变化的数据，能够持续跟踪和同步数据源的变化，保障数据的准确性和一致性。

**故障处理**

故障发生后可进行的恢复操作，包括故障转移、故障转移测试、故障恢复和永久故障转移。

- **故障转移**

  应对虚拟机故障的操作，当原虚拟机发生故障时，可以将副本虚拟机的虚拟卷数据回滚到与指定恢复点一致的状态，并启动副本虚拟机继续提供服务。通过故障转移，可以在短时间内将服务快速恢复到故障发生前的状态，保障业务的连续性和稳定性。
- **故障转移测试**

  故障转移测试是一种模拟容灾演练的操作，在不中断源端生产业务的前提下，通过生成测试虚拟机，在测试虚拟机验证恢复机制的有效性。演练过程中的增量数据不会同步回被保护的虚拟机。
- **故障恢复**

  当原虚拟机恢复正常后，将副本虚拟机在故障转移期间变化的数据回传至原虚拟机，并由原虚拟机继续提供服务。故障恢复旨在将原虚拟机和副本虚拟机之间的数据同步，确保原虚拟机可以在故障解决后继续提供服务。
- **永久故障转移**

  当原虚拟机遭遇严重故障且无法从故障中恢复时采取的措施，此时副本虚拟机将正式接替原虚拟机的服务角色，以确保业务的持续运行。

**RPO** (Recovery Point Objective)

数据恢复点目标，指在故障转移场景中允许的数据损失的时间范围。RPO 以时间为度量单位，标示了从最近执行的复制时间点到发生故障时的时间跨度。较小的 RPO 值表示系统要求在发生故障时允许的数据损失较少。较大的 RPO 值意味着在故障发生时系统可以容忍较长时间范围内的数据丢失。

**RTO** (Recovery Time Objective)

数据恢复时间目标。指业务系统从崩溃到恢复运行所能容忍的时间。较短的 RTO 值表示业务系统要求在发生故障时能够迅速恢复，较长的 RTO 值则意味着业务系统可以容忍更长的恢复时间。

**VPC**（Virtual Private Cloud）

虚拟专有云，是一套为云资源构建的逻辑隔离的网络环境，用户可以在 VPC 中自主设置 VPC 子网、IP 地址、路由表、浮动 IP、网关服务和安全服务等，以实现网络隔离和安全控制。副本虚拟机和原虚拟机可以部署在相同或不同的 VPC 中，并支持灵活的网络映射。测试虚拟机需部署在和原虚拟机不同的 VPC 中。

**VPC 子网**

VPC 子网是组成 VPC 的基础网络模块，用于划分 VPC 的 IP 地址空间。位于同一 VPC 子网的虚拟机默认可以直接通信，跨 VPC 子网虚拟机在在路由可达的情况下也可以互相通信。

**浮动 IP**

浮动 IP 是一种网络地址转换（NAT）机制，通过一对一映射的方式，将外部子网中的 IP 地址动态绑定到 VPC 内部的虚拟机上。这使得外部网络可以主动访问 VPC 内的虚拟机。浮动 IP 为 VPC 内的虚拟机提供了灵活的外部访问能力，同时也保证了网络安全性。

**网络映射**

旨在解决原虚拟机和副本虚拟机/测试虚拟机之间所处的网络环境差异问题的一项配置规则。

由于副本虚拟机/测试虚拟机和原虚拟机往往处于不同的网络环境，若副本虚拟机/测试虚拟机完全复制原虚拟机的网络配置，可能无法在目标集群环境正常配置或连通网络。通过配置网络映射规则，可将原虚拟机所属的虚拟机网络映射到目标集群中对应的虚拟机网络，从而保证副本虚拟机/测试虚拟机能够正常与网络相连。

**IP 映射**

旨在解决原虚拟机和副本虚拟机/测试虚拟机在不同网络环境下的 IP 地址不匹配的问题的一项配置规则。

由于副本虚拟机/测试虚拟机和原虚拟机往往处于不同的网络环境，若副本虚拟机/测试虚拟机完全复制原虚拟机的 IP 地址，可能无法在目标集群正常联通网络。通过配置 IP 映射规则，在故障转移和故障转移测试时，复制服务可以自动为副本虚拟机/测试虚拟机设置正确的 IP 地址，确保副本虚拟机/测试虚拟机可以按照预期的网络配置提供服务，无需手动去更新虚拟机 IP。

**SMTX OS 集群**

SMTX OS 集群属于逻辑概念。在实际生产环境中，一个 SMTX OS 集群至少由 3 个节点通过网络互连组成。

**ELF**

SMTX OS 原生虚拟化计算平台，是当 SMTX OS 以超融合形态直接部署在 x86\_64 或 AArch64 架构的服务器上时，该超融合软件中基于 KVM 的虚拟化引擎组件。ELF 除提供虚拟机生命周期管理、电源操作、高可用、冷热迁移等基本功能外，通过与存储组件 ZBS 相结合，还能提供秒级快照、模板和克隆等高级的虚拟机服务。

ELF 简单易用，对 SMTX OS 超融合集群的所有操作均可通过 CloudTower 完成。

**CloudTower**

SmartX 多集群资源集中管理平台，旨在管理站点内数据中心及跨数据中心的多个 SmartX 集群和系统服务，并提供完整和标准的 RESTful API 和多种语言的 SDK。

## 使用前规划 > 规划复制方向

# 规划复制方向

使用复制服务对虚拟机进行容灾保护时，根据不同的业务需求和站点布局，可选择站内复制或跨站点复制两种类型。

- **站内复制**

  站内复制通常用于集群级别的容灾保护。原虚拟机和副本虚拟机位于同一站点内不同集群中，当原虚拟机不可用时，可以通过切换到站内其他集群中的副本虚拟机，以保持业务的持续运行。该复制策略适用于对数据保护的需求集中在单个站点内的场景。

  ![](https://cdn.smartx.com/internal-docs/assets/73b10dc3/vm_replication_user_guide_19.png)
- **跨站点复制**

  跨站点复制可提供站点级别的容灾保护，当一个站点不可用时，可以立即切换到位于其他地理位置的站点，确保业务的高可用性。该复制策略适用于在不同地理位置部署了多个数据中心，需确保跨地域容灾能力的场景。

  ![](https://cdn.smartx.com/internal-docs/assets/73b10dc3/vm_replication_user_guide_20.png)

## 使用前规划 > 规划站点

# 规划站点

使用跨站点复制策略时，请从如下几方面规划源站点和目标站点。

- **确定站点的定位**

  根据业务需求，明确源站点和目标站点的角色。通常将生产站点作为复制的源站点进行保护，将灾备站点作为目标站点。一个生产站点最多支持配置 5 个灾备站点，以提升数据冗余和灾备能力。
- **确保站点隔离**

  源站点和目标站点的分布应充分考虑地理隔离，以应对地震、火灾等自然灾害和其他潜在风险。同时还应确保各站点硬件、管理平台（CloudTower）和软件服务的独立性。通过多层次的隔离避免所有站点同时发生故障，以确保业务的高可用性和数据安全。
- **选择合适的复制架构**

  根据实际业务需要和数据保护策略选择合适的复制架构。

  - **单向的生产-灾备架构**

    此架构适合需要明确区分业务生产和灾备的场景。可将生产站点作为源站点，专注于正常的业务运行；将灾备站点作为目标站点，专注于数据恢复和灾难保护。
  - **双向的互为灾备架构**

    此架构下，两个站点相互备份对方数据，任一站点发生故障时，另一站点可以迅速接管业务。

## 使用前规划 > 规划集群

# 规划集群

规划源集群和目标集群时，请从如下几个方面综合考虑：

- **确定集群的定位**

  根据业务需求，将生产集群作为复制的源集群进行保护，将灾备集群作为复制的目标集群。
- **选择合适的复制架构**

  根据实际业务需要和数据保护策略选择合适的复制架构。

  - **单向的生产 - 灾备架构**

    若需要实现业务生产和灾备的隔离，可选择此架构，将生产集群作为源集群，灾备集群作为目标集群。
  - **双向的互为灾备架构**

    若两个集群需各自运行业务且互为灾备集群，可选择此架构，两集群之间互相复制，并各自使用部分资源为对方存储副本。
- **确认 CPU 架构的一致性**

  请确保源集群和目标集群的 CPU 架构保持一致，避免兼容性问题。

## 使用前规划 > 规划异步复制计划

# 规划异步复制计划

在创建异步复制计划前，请从以下几个方面规划：

- **计划保护的虚拟机**

  每个异步复制计划中可包含一个或多个虚拟机，这些虚拟机共享相同的复制策略。因此，建议根据不同的数据保护要求，对虚拟机进行分类，将具有相同目标集群、复制周期和保留策略的虚拟机置于同一个异步复制计划中，避免重复定义相同的复制策略，提高运维和管理效率。
- **复制周期**

  通过设定复制周期，复制服务可以对虚拟机自动化进行周期性的复制。用户可以根据业务系统的 RPO（Recovery Point Objective）目标，正确设置数据复制的时间间隔。最小 RPO 为 15 分钟，因此可根据业务需求合理选择复制周期，以确保数据的及时恢复能力。
- **恢复点保留策略**

  每次执行完异步复制计划后，复制服务会为中的每个虚拟机创建一个恢复点，恢复点包含了复制时间点的所有虚拟机数据，在需要时，可通过恢复点将原虚拟机故障转移到对应时间点状态的副本虚拟机。

  恢复点保留策略可以定义保留的恢复点数量，复制服务将根据恢复点保留策略，自动清理超出范围的恢复点。

## 使用前规划 > 规划许可单元的分配

# 规划许可单元的分配

SMTX 备份与容灾的软件许可由备份与复制两个功能模块共享。许可单元根据所保护对象类型及所用功能进行消耗，具体规则如下：

- **SMTX OS（ELF）集群虚拟机**

  - 备份功能：每备份一个虚拟机，消耗 1​ 个许可单元。
  - 异步复制功能：每异步复制一个虚拟机，消耗 2​ 个许可单元。
  - 同步复制功能：每同步复制一个虚拟机，消耗 4​ 个许可单元。
- **SMTX ZBS 块存储集群存储卷**：每异步复制 1 TiB 的存储卷，消耗 2​ 个许可单元。

请您根据实际的业务需求，综合考虑要使用的功能模块，需要保护的虚拟机和存储卷数量、以及未来业务的扩展等，提前规划许可单元的分配。

# 管理站点（仅跨站点复制）

一个站点使用一个 CloudTower 对站点内的集群进行管理。若您使用跨站点复制策略，需要通过将当前站点的 CloudTower 和其他站点的 CloudTower 进行关联，从而建立站点之间的关联关系，以支持站点之间的数据复制。若您使用站点内复制策略，则无需关联站点。

## 管理站点（仅跨站点复制） > 添加关联站点

# 添加关联站点

完成复制服务的部署后，请为当前站点添加关联站点。

**前提条件**

- 仅超级管理员和运维管理员有关联站点的操作权限，因此在当前站点请使用超级管理员或运维管理员账号登录。
- 提前获取待添加站点所属的 CloudTower 的 IP 地址、超级管理员或任一运维管理员账号及密码。关联站点时仅支持使用站点所属的 CloudTower 的本地账户，不支持 LDAP 账户。
- 每个站点最多可关联 5 个站点，因此请确保当前站点和待添加站点已关联的站点数均小于 5。
- 为避免兼容性问题，建议源站点和目标站点关联相同版本的 CloudTower。

**操作步骤**

1. 登录 CloudTower，在备份与容灾界面中，单击**设置** > **站点管理**进入站点管理界面。
2. 单击界面右上角 **+ 添加关联站点**，弹出**添加关联站点**对话框。
3. 输入以下信息：

   | 参数 | 描述 |
   | --- | --- |
   | 站点名称 | 自定义待关联站点的名称。 |
   | CloudTower IP | 输入待关联站点所属 CloudTower 的 IP 地址。 |
   | 用户名 | 输入待关联站点 CloudTower 中超级管理员或任意运维管理员账号的用户名。 |
   | 密码 | 输入 CloudTower 用户名对应的密码。 |
4. 单击**添加**。
5. （可选）若关联站点启用了双因子认证，您还需查看该站点用户信息中填写的邮箱，将 6 位数验证码填写至输入框中，并单击**验证**。验证通过后即可完成关联。

完成添加后，站点管理界面将展示已关联站点的信息，包括站点名称、和当前站点的连接状态和 CloudTower IP。

## 管理站点（仅跨站点复制） > 编辑关联站点

# 编辑关联站点

当关联站点的 CloudTower IP 发生变更时，站点的连接状态将显示连接异常。您可以通过编辑操作更新关联站点最新的 CloudTower IP 信息，以重新和站点进行关联。

**前提条件**

- 当前站点的用户角色为超级管理员或运维管理员。
- 待编辑站点相关的入站复制计划中不存在虚拟机正在进行复制、故障转移、故障恢复和永久故障转移任务。
- 待编辑站点相关的出站复制计划中不存在虚拟机正在进行复制任务。
- 待编辑站点和当前站点未处于`连接中`状态。

**操作步骤**

1. 登录 CloudTower，在备份与容灾界面中，单击**设置** > **站点管理**进入站点管理界面。
2. 单击某一站点右侧的 **...**，选择**编辑**，弹出**编辑关联站点**对话框。
3. 编辑站点名称或 CloudTower IP。
4. 单击**保存**。

## 管理站点（仅跨站点复制） > 删除关联站点

# 删除关联站点

若需解除当前站点与已关联站点的关联关系，可通过删除关联站点来解除关联关系。

**前提条件**

- 当前站点的用户角色为超级管理员或运维管理员。
- 待删除站点和当前站点处于`已连接`状态，当前站点与待删除站点之间需不存在出站复制计划、入站复制计划、出站复制对象和入站复制对象。
- 待删除站点和当前站点处于`连接异常`状态，当前站点与待删除站点之间需不存在处于`故障转移中`、`永久故障转移中`状态或正在执行故障转移任务的入站复制对象。
- 待删除站点和当前站点未处于`连接中`状态。

**操作步骤**

1. 登录 CloudTower，在备份与容灾界面中，单击**设置** > **站点管理**进入站点管理界面。
2. 单击某一站点右侧的 **...**，选择**删除**，弹出**删除关联站点**对话框。
3. 单击**删除**。

## 管理关联集群 > 添加关联集群

# 添加关联集群

一个复制服务可以关联多个 SMTX OS（ELF）集群，一个 SMTX OS（ELF）集群也可以关联多个复制服务。

**前提条件**

- 复制服务仅可关联与 CloudTower 的连接状态为`已连接`的集群。
- 请确保复制服务和待关联集群的存储网络连通。

**操作步骤**

1. 登录 CloudTower，在**备份与容灾**界面中，单击**设置**进入功能管理界面。
2. 选择一个已部署的复制服务，单击**集群关联**。
3. 单击**编辑集群关联**，在弹出的操作界面中，单击 **+ 添加集群**，添加一个或多个 SMTX OS（ELF）集群。
4. 单击**保存**。

完成关联后，在**集群关联**界面，可查看复制服务已关联的集群列表。

| 参数 | 说明 |
| --- | --- |
| 集群名称 | 复制服务关联的集群名称。 |
| 集群类型 | 复制服务关联的集群类型。 |
| 集群 IP | 集群关联至 CloudTower 时使用的 IP 地址。 |
| 所属数据中心 | 集群在 CloudTower 中所属的数据中心。 |

## 管理关联集群 > 删除关联集群

# 删除关联集群

若复制服务需取消和集群的关联，可单击**编辑集群关联**，在弹出的界面中进行相应操作。

**注意**：

- 若存在虚拟机将已关联集群作为目标集群，并使用当前复制服务时，无法取消关联。
- 若已关联集群中存在虚拟机已加入复制计划或虚拟机存在恢复点，且该集群只关联了该复制服务时，无法取消关联。
- 若当前复制服务仅关联了一个集群，则无法取消关联。
- 若已关联集群中包含副本虚拟机或测试虚拟机，则无法取消关联。

**操作步骤**

1. 登录 CloudTower，在**备份与容灾**界面中，单击**设置**进入功能管理界面。
2. 选择一个已部署的复制服务，单击**集群关联**。
3. 单击**编辑集群关联**，在弹出的操作界面中，单击相关集群旁边 `×` 以移除集群。
4. 单击**保存**。

# 创建异步复制计划

异步复制计划用于定义具体的复制对象和执行策略，如复制的虚拟机、复制计划类型、目标集群、复制周期、恢复点保留策略，以及副本虚拟机和测试虚拟机的设置等。

登录 CloudTower，在**备份与容灾**界面，选择**异步复制计划**，单击**创建异步复制计划**，弹出创建界面，设置**基本信息**、**复制周期**、**保留策略**、**副本虚拟机**和**测试虚拟机**。

## 管理异步复制计划 > 创建异步复制计划 > 设置基本信息

# 设置基本信息

1. 输入异步复制计划的名称和描述。
2. 选择复制对象类型为**虚拟机**，并选择复制方向。默认为站内复制，当本站点添加了其他站点且至少一个站点处于`已连接`状态，允许选择入站或出站。

   | 配置项 | 说明 |
   | --- | --- |
   | 站内 | 在本站点内进行虚拟机异步复制。 |
   | 入站 | 将外站点虚拟机异步复制至本站点。请选择复制的源站点。 |
   | 出站 | 将本站点虚拟机异步复制至外站点。请选择复制的目标站点。 |
3. 设置目标集群、目标端复制服务和文件系统一致性。

   | 配置项 | 说明 |
   | --- | --- |
   | 目标集群 | 目标集群需满足下述条件： - 集群类型为 SMTX OS（ELF）。 - 集群与 CloudTower 连接正常。 - 集群已关联了复制服务。 |
   | 目标端复制服务 | 选择一个与目标集群关联了的复制服务作为目标端复制服务。 |
   | 文件系统一致性 | 默认启用，对于已安装虚拟机工具且处于运行中状态的虚拟机，通过静默或冻结文件系统，确保快照捕获时所有磁盘的 I/O 已完成，且数据处于可恢复的完整状态，从而实现文件系统一致性复制；其他虚拟机将进行崩溃一致性复制，快照中不会包含虚拟机文件系统的内存数据。 关闭后将对所有虚拟机进行崩溃一致性复制，相关快照中不包含虚拟机文件系统的内存数据。  **注意**：虚拟机创建文件系统一致性快照期间将暂时无法使用。 |
4. 选择加入异步复制计划的虚拟机。支持按照虚拟机名称、所属集群名称、标签名和标签值筛选虚拟机，可多选。

   > **说明**:
   >
   > - 不支持复制以下虚拟机：
   >
   >   - 挂载了共享虚拟卷的虚拟机。
   >   - 未挂载虚拟卷的虚拟机。
   >   - 运行系统服务的虚拟机，如 Everoute Controller 虚拟机、CloudTower 虚拟机、可观测虚拟机、备份服务虚拟机、复制服务虚拟机、高级监控虚拟机、安全防护虚拟机（SVM）等。
   >   - SKS 工作负载集群节点虚拟机。
   >   - 容器镜像仓库虚拟机。
   >   - 处于`操作中`或`未知`状态的虚拟机。
   >   - 所在集群与 CloudTower 连接异常的虚拟机。
   >   - 所在集群的许可类型为试用许可且许可已过期的虚拟机。
   >   - 复制服务创建的副本虚拟机和测试虚拟机。
   >   - 启用了随主机开机功能的虚拟机。
   >   - 已加入其他同步复制计划或异步复制计划的虚拟机。
   > - 对于已加入异步复制计划的虚拟机，在进行跨集群热迁移，以及不保留源虚拟机的跨集群冷迁移和分段迁移时：
   >
   >   - 若迁移的目标集群与复制的目标集群相同，迁移完成后该虚拟机将被移出复制计划;
   >   - 若迁移的目标集群与复制的目标集群不同，且未关联复制服务，虚拟机迁移后将被移出复制划；
   >   - 若迁移的目标集群与复制的目标集群不同，且已关联了复制服务，在源端与目标端复制服务的复制网络连通时，复制计划将继续保护该虚拟机；若不连通，则需在目标集群手动关联新的复制服务后方可继续保护；
   >   - 对于通过冷迁移或分段迁移迁移至低于 5.1.3 的 5.x.x 集群或 6.0.x 集群的虚拟机，迁移完成后将被移出复制计划。
   > - 对于已加入异步复制计划的虚拟机，若进行保留源虚拟机的跨集群冷迁移和分段迁移，将继续保护源虚拟机而非迁移后的虚拟机。

## 管理异步复制计划 > 创建异步复制计划 > 设置复制周期和时间窗口

# 设置复制周期和时间窗口

设置异步复制计划自动执行的周期，包括设置复制周期和复制时间窗口。设置完成后单击**下一步**。

1. **设置复制周期**

   | 执行类型 | 默认周期 | 支持设置周期 | 说明 |
   | --- | --- | --- | --- |
   | 分钟 | 15 分钟 | 15 ~ 59 分钟 | 系统将从每日 00:00 开始计时，按照设置的时间周期执行复制任务。 |
   | 小时 | 1 小时 | 1 ~ 23 小时 |
   | 天 | 1 天 | 1 ~ 30 天 | 需要为执行当天设置一个或多个执行时间，两个执行时间最小间隔为 15 分钟。 |
   | 星期 | 每星期一 | 星期一 ~ 星期日 | 可选择在每周中的一天或几天执行，并需要为执行当天设置一个或多个执行时间，两个执行时间最小间隔为 15 分钟。 |
2. **设置复制时间窗口**

   选择是否启用仅允许在时间窗口内执行复制任务，默认不启用。启用后需设置时间窗口，要求如下：

   - 复制时间窗口开始和结束的间隔需不少于 15 分钟。
   - 复制周期按天和星期设置时，复制时间窗口的范围必须包含复制周期设定的执行时间，且窗口结束时间不能为自动执行的时间点。
   - 默认情况下，若复制在超出复制窗口的结束时间后仍未完成，系统将停止复制任务，并删除对应虚拟机的快照和恢复点信息，若为首次复制，还将删除副本虚拟机。勾选“**若复制不能在时间窗口内完成，继续完成当前复制**” 可取消此限制。
   > **说明**:
   >
   > - 在复制周期的自动执行时间点，如果当前异步复制计划正在执行，本次自动执行将跳过。
   > - 复制周期按分钟和小时设置时，若自动执行时间点刚好在时间窗口的结束时间，则不会执行复制任务。
   > - 在复制周期的自动执行时间点，如果异步复制计划中存在虚拟机当前正在执行故障转移任务、故障恢复或永久故障转移任务或处于`故障转移中`状态，对应虚拟机的本次复制将跳过执行。处于`未就绪`状态的虚拟机本次复制将会失败。

## 管理异步复制计划 > 创建异步复制计划 > 设置副本虚拟机

# 设置副本虚拟机

1. **设置副本虚拟机的命名规则**

   支持通过添加前缀或后缀的方式为副本虚拟机命名，也可选择与原虚拟机名称保持一致。
2. **设置副本虚拟机存储策略**

   支持为副本虚拟机的虚拟卷配置与原虚拟机不同的存储策略。

   | 参数 | 说明 |
   | --- | --- |
   | 冗余策略 | 可选择副本或纠删码。 **说明**：  - 若目标集群为双活集群，选择副本策略时仅支持选择 3 副本。 - 选择纠删码策略时，目标集群需满足纠删码的使用条件。 |
   | 置备方式 | 可选择厚置备或精简置备。 |
   | 数据加密 | 启用后，对于原虚拟机已加密的虚拟卷，将在创建副本虚拟机后继续加密。 |
3. **设置网络映射规则**

   在复制原虚拟机时，为确保在目标集群中创建的副本虚拟机能够正确连接到所需的网络环境，需要设置网络映射规则。映射规则可以将原虚拟机使用的虚拟机网络映射到目标集群中指定的虚拟机网络，以确保副本虚拟机能够正常连通网络。

   复制服务支持 VPC 和 VLAN 网络之间的灵活映射。您可以根据实际的网络环境和业务需求自由设置。使用 VPC 网络的虚拟机能够在保持 IP 和路由等网络配置不变的情况下快速完成虚拟机故障转移切换，实现业务在多站点的网络容灾和高可用。例如，源端虚拟机使用 VPC 网络时，推荐副本端网络映射为相同 VPC 及同一子网，若源端虚拟机发生故障时，能够在保持 IP 和路由等网络配置不变的情况下将业务快速切换至副本虚拟机，确保业务连续性和实现网络高可用。

   > **说明**:
   >
   > - 若原虚拟机后续使用了新的虚拟机网络，请通过编辑复制计划重新设置网络映射规则，否则对应的副本虚拟机将不会创建对应的网络设备。
   > - 若原虚拟机后续不再使用某个虚拟机网络，且复制计划中剩余的虚拟机均不使用该虚拟机网络，可通过编辑复制计划来删除对应的网络映射规则。
4. **设置 IP 映射规则（可选）**

   由于原虚拟机和副本虚拟机可能处于不同的网络环境，为确保副本虚拟机能够正确连接到新的网络环境，可以设置 IP 映射规则，将原虚拟机的 IP 地址映射到可在目标集群使用的 IP 地址。

   在进行故障转移时，复制服务会尝试通过原虚拟机的 IP 和子网掩码查找匹配的 IP 映射规则，并根据规则设置副本虚拟机的 IP 地址，确保副本虚拟机在启动后能够使用正确的网络配置。

   > **说明**:
   >
   > - IP 映射规则被成功应用需要满足下述条件：
   >   - 源集群和目标集群需为 SMTX OS 5.0.6 及以上版本、5.1.0 及以上版本、6.0.0 及以上版本、6.1.0 及以上版本和 6.2.0 及以上版本。
   >   - 若原虚拟机网络为 VLAN，原虚拟机需配置静态 IP 且已安装虚拟机工具。
   >   - 若原虚拟机网络为 VPC，副本虚拟机网络为 VLAN，原虚拟机需安装虚拟机工具。
   >   - 原虚拟机需使用特定版本的客户端操作系统。已通过验证的操作系统为 CentOS 7.4、CentOS 8.4、Windows 8.1 Enterprise、Windows Server 2012 R2 Standard、Windows Server 2016 Standard。
   > - 支持添加多条 IP 映射规则。如设置了多条映射规则，将按照添加顺序采用匹配的第一条规则。
   > - IP 地址支持以子网掩码或者 CIDR 块的形式输入。若原虚拟机 IP 为单个 IP 地址，映射的副本虚拟机 IP 也需要为单个 IP 地址；若原虚拟机 IP 为 IP 段，映射的副本虚拟机 IP 也需要设置 IP 段。设置 IP 段时，请使用 `*` 号标识 IP 地址的第四位，如 `192.168.0.*`，映射后的 IP 地址第四位和原 IP 地址第四位将保持相同。
   > - 若原虚拟机和副本虚拟机均使用相同的 VPC 子网，则副本虚拟机将直接使用原虚拟机的 IP 地址，无需设置 IP 映射规则。
   > - 若原虚拟机网络类型为 VPC 且配置了多个 IP：当副本虚拟机网络为 VPC 时，IP 映射规则可对 VPC 网卡上的所有 IP 生效；当副本虚拟机网络为 VLAN 时，仅对 VPC 网卡上的首个 IP 生效。
5. **设置副本虚拟机 MAC 地址**

   设置副本虚拟机的 MAC 地址选项，默认为随机分配，也可选择与原虚拟机一致。

   > **注意**：
   >
   > 若副本虚拟机与原虚拟机处于同一 VPC 子网，则不支持将 MAC 地址配置为与原虚拟机地址一致。

   > **说明**:

## 管理异步复制计划 > 手动执行异步复制计划

# 手动执行异步复制计划

已创建的异步复制计划默认将按照设定的复制周期自动执行，也可由用户手动执行。

异步复制计划将由计划中指定的目标端复制服务执行，在执行时，如果原虚拟机所在集群关联了多个复制服务，目标端复制服务将随机选择一个连通的复制服务作为源端复制服务，以实现复制时的数据传输。

**前提条件**

- 异步复制计划当前没有在执行的复制任务。
- 当前没有对异步复制计划进行编辑操作。

**操作步骤**

1. 在**异步复制计划**列表选择待执行的复制计划，在列表右侧单击 **...**，选择**手动执行**。您也可以单击复制计划名称进入详情页，选择**手动执行**。
2. 在弹出的页面中单击**手动执行**。复制虚拟机的任务创建后支持在任务中心停止。

   - 对于等待创建副本虚拟机的虚拟机，将执行全量复制。
   - 对于已创建副本虚拟机的虚拟机，将执行增量复制。

> **说明**：
>
> - 在对虚拟机进行增量复制时，如果副本虚拟机除磁盘外的配置信息更新失败，复制服务仍然会继续同步数据，并在数据同步成功后创建一个恢复点，该恢复点和原虚拟机的部分配置项不一致。复制服务将在下次执行复制任务时再次同步最新的配置和虚拟卷数据以保证一致性。
> - 若复制计划中存在虚拟机处于以下状态，则本次复制任务将跳过该虚拟机的复制：
>   - 虚拟机正在执行故障转移、故障恢复、永久故障转移或处于`故障转移中`状态。
>   - 虚拟机被删除、转化为虚拟机模板、迁移至另一集群或所属集群从 CloudTower 中移除。
>   - 处于`未就绪`状态和移至回收站的虚拟机将会复制失败。

# 暂停或继续异步复制计划

已创建的异步复制计划默认将按照设定的复制周期自动执行。您也可暂停异步复制计划，使当前复制周期暂时失效。暂停后的复制计划不会自动执行。复制计划暂停后可以通过**继续复制计划**恢复生效。

在**异步复制计划**列表选择要暂停或继续的异步复制计划，在列表右侧或异步复制计划详情页单击 **...**，选择**暂停复制计划**或**继续复制计划**，可暂停或继续复制计划。

  > 当被保护虚拟机变更过网络设备导致有新增虚拟机网络时，该页签将提示**故障转移网络映射规则待补充**和**故障转移测试网络映射规则待补充**，您需要编辑副本虚拟机和测试虚拟机以重新设置网络映射规则。
- **复制时间**

  展示异步复制计划下次自动执行时间、最近一次执行时间（包括手动执行和自动执行）、最近一次执行结果（包括手动执行和自动执行）和复制窗口。
- **执行记录**

  展示当前异步复制计划的执行情况，不同的颜色表示不同的状况。

  - 绿色表示当日执行过异步复制计划且除跳过的复制任务外其余复制任务均执行成功。
  - 红色表示当日执行过异步复制计划且除跳过的复制任务外存在复制失败的复制任务，悬浮鼠标可查看异步复制计划执行失败的时间和复制失败的虚拟机数量。
  - 蓝色表示当日有待执行异步复制计划但仍未执行，悬浮鼠标可查看异步复制计划将要执行的时间。
  - 灰色表示当日没有计划要执行的异步复制计划。
- **复制对象**

  - 展示当前异步复制计划包含的所有虚拟机和虚拟机对应的副本虚拟机、测试虚拟机，以及每个虚拟机所有的恢复点。
  - 展示从当前异步复制计划中移除但是保留恢复点的虚拟机和对应的副本虚拟机、测试虚拟机，以及每个虚拟机所有的恢复点。
- **事件**

  展示当前异步复制计划相关的用户事件和系统事件。在跨站点场景下，系统事件仅记录在目标端站点所关联的 CloudTower 界面。

## 故障处理 > 故障转移测试

# 故障转移测试

您可以在生产环境业务正常运行的情况下，通过故障转移测试生成测试虚拟机来模拟真实故障转移流程，从而验证灾备方案的有效性，确保在灾难发生时能够快速恢复关键业务。测试过程中产生的增量数据不会被同步回源端虚拟机，源端虚拟机可正常运行业务。

故障转移测试对虚拟机的复制任务、故障转移任务和故障恢复任务没有影响。

您可以对多个虚拟机批量进行故障转移测试，也可以对单个虚拟机进行故障转移测试。

**注意事项**

待进行故障转移测试的虚拟机复制对象需满足以下条件：

- 虚拟机复制对象的故障转移状态为`复制中`或`就绪`。

  若复制状态为`复制中`，选择恢复点时不允许选择**立即生成恢复点**。
- 虚拟机复制对象当前不存在其他测试虚拟机，即未在其他异步复制计划中存在执行中的故障转移测试任务。
- 虚拟机复制对象所属的异步复制计划[已配置故障转移测试网络](/backup/2.3.0/vm_replication_user_guide/vm_replication_user_guide_46)。

## 故障处理 > 故障转移测试 > 执行测试 > 虚拟机批量执行测试

# 虚拟机批量执行测试

执行以下步骤对多个虚拟机进行批量执行故障转移测试。

**操作步骤**

1. 登录 **CloudTower**，在**备份与容灾**界面选择**异步复制计划**，单击页面右上角**故障处理**，并选择**故障转移测试**。
2. 如果存在入站复制对象，需选择待进行故障转移测试的虚拟机所属的站点作为源站点。
3. 选择需进行故障转移测试的虚拟机，支持按照虚拟机名称、所属集群和所属复制计划进行搜索。
4. 为每个虚拟机选择恢复点及设置启动顺序。

   - **恢复点**

     您需要为每个虚拟机选择恢复点，用于将测试虚拟机恢复到对应时间点的状态。

     - **立即生成恢复点**

       在故障转移测试任务创建成功后，将先对原虚拟机执行复制，再基于最新的恢复点进行故障转移测试。  
       当原虚拟机满足下述条件时，允许选择**立即生成恢复点**。

       - 源集群与 CloudTower 连接正常；
       - 原虚拟机存在且不处于`未知`状态；
       - 原虚拟机所属复制计划未被删除；
       - 源站点连接状态为`已连接`。
       - 虚拟机复制对象的故障转移状态为`就绪`。
     - **选择历史恢复点**

       当原虚拟机不允许选择**立即生成恢复点**时，请选择一个已创建的历史恢复点。在故障转移测试任务创建成功后，复制服务将根据选择的历史恢复点进行故障转移测试。
   - **启动顺序**

     若存在多个测试虚拟机关联了同一个复制服务，支持调整测试虚拟机的启动顺序，并为每个测试虚拟机设置启动延迟，故障转移测试任务完成之后，复制服务将按照启动顺序依次启动测试虚拟机。

     - 设置启动顺序：拖动测试虚拟机以调整启动顺序。
     - 设置启动延时：启动延迟支持设置为 0 ~ 300 秒，默认为 60 秒。设置为 0 表示该测试虚拟机和上一个测试虚拟机同时启动，设置为 60 表示该测试虚拟机会在上一个测试虚拟机开始启动 60 秒后启动。
5. （可选）若存在使用了 VPC 网络的测试虚拟机，可选择为测试虚拟机设置浮动 IP，以确保外部网络可以稳定访问测试虚拟机提供的业务。

   > **说明**：
   >
   > 若原虚拟机网络类型为 VLAN，需安装虚拟机工具且配置静态 IP。
6. 单击**创建**。

> **说明**：
>
> 每个虚拟机的故障转移测试任务相互独立，无依赖关系。如果某个虚拟机故障转移测试任务失败，可以单独为该虚拟机重新创建故障转移测试任务。

     - 原虚拟机所属复制计划未被删除；
     - 源站点连接状态为`已连接`。
     - 虚拟机复制对象的故障转移状态为`就绪`。
   - **选择历史恢复点**

     当原虚拟机不允许选择**立即生成恢复点**时，请选择一个已创建的历史恢复点。在故障转移测试任务创建成功后，复制服务将根据选择的历史恢复点进行故障转移测试。
5. （可选）若测试虚拟机使用了 VPC 网络，可选择为测试虚拟机设置浮动 IP，以确保外部网络可以稳定访问测试虚拟机提供的业务。

   > **说明**：
   >
   > 若原虚拟机网络类型为 VLAN，需安装虚拟机工具且配置静态 IP。

## 故障处理 > 故障转移测试 > 停止测试 > 停止单个虚拟机测试

# 停止单个虚拟机测试

故障转移测试结束后，您可以执行以下步骤为单个正在进行故障转移测试的虚拟机停止测试。

**操作步骤**

1. 登录 **CloudTower**，进入**备份与容灾**界面。
2. 选择**异步复制计划**，进入异步复制计划列表。当选择入站复制计划时，可在**入站**页签右上角单击源站点下拉框切换源站点。
3. 进入异步复制计划详情页面，在**复制对象**页签下选择正在进行故障转移测试的虚拟机，单击页签右上角的**停止故障转移测试**。

   您也可以在恢复点列表中，选择正在进行故障转移测试的恢复点，单击右侧的 **...**，选择**停止故障转移测试**。

   您还可以在**副本虚拟机**页面，选择正在进行故障转移测试的复制对象，单击右侧的 **...**，选择**停止故障转移测试**。
4. 在弹出的**创建停止故障转移测试任务**对话框中选择保留或删除测试虚拟机，然后单击**创建**。

## 故障处理 > 故障转移

# 故障转移

当原虚拟机发生故障时，可通过故障转移将原虚拟机切换至目标集群的副本虚拟机，由副本虚拟机接替原虚拟机提供服务。

您可以对多个虚拟机批量进行故障转移，也可以对单个虚拟机进行故障转移。

**注意事项**

- 待进行故障转移的虚拟机需满足以下条件：

  - 当虚拟机存在多个副本虚拟机时，需选择其中一个副本虚拟机进行故障转移，请确保其对应的复制对象的故障转移状态为`就绪`，且其余复制对象不处于`故障转移中`、`故障恢复中`和`永久故障转移中`状态。
  - 虚拟机当前没有正在执行的故障转移任务。
- 故障转移期间，如果原虚拟机从故障状态中恢复，为避免发生 IP 冲突，不允许对原虚拟机执行开机操作，请通过故障恢复操作完成副本虚拟机到原虚拟机的切换。

## 故障处理 > 故障转移 > 虚拟机批量故障转移

# 虚拟机批量故障转移

执行以下步骤对多个虚拟机进行批量故障转移。

**操作步骤**

1. 登录 **CloudTower**，在**备份与容灾**界面，选择**异步复制计划**，单击页面右上角**故障处理**，并选择**故障转移**。
2. 选择复制对象类型为**虚拟机**。
3. 如果存在入站复制对象，需选择待进行故障转移的虚拟机所属的站点作为源站点。
4. 选择需进行故障转移的虚拟机，支持按照虚拟机名称、所属集群和所属复制计划进行搜索。

   > **说明**：
   >
   > 若虚拟机存在多个副本虚拟机，仅支持选择其中一个副本虚拟机进行故障转移。
5. 为每个虚拟机选择恢复点及设置启动顺序。

   - **恢复点**

     您需要为每个虚拟机选择恢复点，用于将副本虚拟机恢复到对应时间点的状态。

     - **立即生成恢复点**

       在故障转移任务创建成功后，将先对原虚拟机执行复制，再基于最新的恢复点进行故障转移。  
       当原虚拟机满足下述条件时，允许选择**立即生成恢复点**。

       - 源集群与 CloudTower 连接正常；
       - 原虚拟机存在且不处于`未知`状态；
       - 原虚拟机所属复制计划未被删除；
       - 源站点连接状态为`已连接`。
     - **选择历史恢复点**

       当原虚拟机不允许选择**立即生成恢复点**时，请选择一个已创建的历史恢复点。在故障转移任务创建成功后，复制服务将根据选择的历史恢复点进行故障转移。

       > **注意**：
       >
       > 选择历史恢复点进行故障转移后，若后续进行故障恢复，复制服务将会把副本虚拟机在故障转移期间产生的数据同步回原虚拟机。此过程中，原虚拟机自历史恢复点至故障转移时刻的所有数据将被覆盖，导致该时间段内的数据丢失。
   - **启动顺序**

     若存在多个副本虚拟机关联了同一个复制服务，支持调整副本虚拟机的启动顺序，并为每个副本虚拟机设置启动延迟，故障转移任务完成之后，复制服务将按照启动顺序依次启动副本虚拟机。

     - 设置启动顺序：拖动副本虚拟机以调整启动顺序。
     - 设置启动延时：启动延迟支持设置为 0 ~ 300 秒，默认为 60 秒。设置为 0 表示该副本虚拟机和上一个副本虚拟机同时启动，设置为 60 表示该副本虚拟机会在上一个副本虚拟机开始启动 60 秒后启动。
6. 设置原虚拟机关机选项，支持选择**关机**或**强制关机**。

   > **说明**：
   >
   > 仅当所选择的虚拟机中存在虚拟机满足下述条件时，需选择关机选项：
   >
   > - 虚拟机存在：虚拟机未被删除、虚拟机未被转换成虚拟机模板、虚拟机未被迁移到其他集群、虚拟机未被移入回收站、所属集群未从 CloudTower 移除；
   > - 虚拟机未处于`关机`或`未知`状态；
   > - 虚拟机所属集群与 CloudTower 连接正常；
   > - 源站点连接状态为`已连接`。
7. （可选）若存在使用了 VPC 网络的副本虚拟机，可选择为副本虚拟机设置浮动 IP，以确保外部网络可以稳定访问副本虚拟机提供的业务。

   > **说明**：
   >
   > 若原虚拟机和副本虚拟机处于相同的 VPC 子网，则副本虚拟机的浮动 IP 默认和原虚拟机保持相同。若原虚拟机未启用浮动 IP，也可以手动为副本虚拟机设置浮动 IP。
   > 若原虚拟机网络类型为 VLAN，需安装虚拟机工具且配置静态 IP。
8. 单击**创建**。

副本虚拟机启动成功后，其对应的复制对象的故障转移状态即变更为`故障转移中`。若对应的复制计划中配置了 IP 映射规则且副本虚拟机存在匹配的 IP 映射规则，则复制服务将对副本虚拟机进行 IP 映射。

> **说明**：
>
> - 故障转移期间，如果原虚拟机关联的复制计划自动执行，该虚拟机的复制任务将会跳过。
> - 故障转移期间，若手动执行该虚拟机所属的复制计划，该虚拟机的复制任务将跳过。
> - 每个虚拟机的故障转移任务相互独立，无依赖关系。如果某个虚拟机故障转移失败，可以单独为该虚拟机重新创建故障转移任务。

## 故障处理 > 故障恢复

# 故障恢复

故障转移后，若原虚拟机恢复运行，可通过故障恢复将故障转移期间副本虚拟机产生的数据同步回原虚拟机，并恢复至由原虚拟机提供服务。

您可以对单个虚拟机进行故障恢复，也可以对多个虚拟机批量进行故障恢复。

**前提条件**

虚拟机进行故障恢复需满足以下条件：

- 复制对象的故障转移状态为`故障转移中`。
- 原虚拟机存在且处于`关机`状态。
- 原虚拟机恢复运行后未挂载共享虚拟卷。
- 对于入站复制对象，源站点需处于`已连接`状态。
- 源集群和目标集群连接正常。

## 故障处理 > 故障恢复 > 虚拟机批量故障恢复

# 虚拟机批量故障恢复

执行以下步骤为多个虚拟机进行批量故障恢复。

**操作步骤**

4. 选择需要进行故障恢复的原虚拟机。支持按照虚拟机名称、所属集群和所属复制计划进行搜索。

   - 启动顺序：拖动原虚拟机以调整启动顺序。

> - 在虚拟机故障恢复期间，如果其关联的复制计划自动执行或手动执行，该虚拟机的复制任务将会跳过。

   > **说明**：
   >
   > 仅当异步复制计划中存在`故障转移中`状态的复制对象时展示**永久故障转移**的操作入口。
2. 选择复制对象类型为**虚拟机**。
3. 如果存在入站复制对象，需选择要永久故障转移的虚拟机所属的站点作为源站点。
4. 选择需进行永久故障转移的虚拟机。支持按照虚拟机名称、所属集群和所属复制计划进行搜索。
5. 选择是否保留副本虚拟机的快照，默认保留。
6. 如果存在测试虚拟机，需选择是否保留测试虚拟机，默认保留。
7. 单击**创建**。

创建永久故障转移任务后，复制对象即进入`永久故障转移中`状态。复制服务会将原虚拟机从复制计划中移除，将副本虚拟机转化为普通虚拟机，并按照设定保留或删除副本虚拟机的快照和测试虚拟机。

> **说明**：
>
> - 在虚拟机永久故障转移期间，如果其关联的异步复制计划自动执行或手动执行，该虚拟机的复制任务将会跳过。
> - 每个虚拟机的永久故障转移任务相互独立，无依赖关系，执行失败的永久故障转移任务将自动重试。
> - 若永久故障转移任务失败，则复制对象的故障转移状态将由`永久故障转移中`变更为`故障转移中`。

## 故障处理 > 永久故障转移 > 单个虚拟机永久故障转移

# 单个虚拟机永久故障转移

执行以下步骤对单个虚拟机进行永久故障转移。

**操作步骤**

1. 登录 **CloudTower**，进入**备份与容灾**界面。
2. 选择**异步复制计划**，异步复制计划页面分别展示了站内、入站和出站的复制计划。其中，仅支持对站内和入站类型的复制计划中的虚拟机进行永久故障转移。在入站页签右上角单击源站点下拉框可切换源站点。
3. 单击某一异步复制计划名称，进入详情页面，在**复制对象**页签下选择一个待进行永久故障转移的虚拟机，单击页签右上角的**故障处理**，选择**永久故障转移**，弹出**创建永久故障转移任务**对话框。

   您也可以在**副本虚拟机**页面，选择一个待进行永久故障转移的复制对象，单击右侧的 **...**，选择**永久故障转移**，弹出**创建永久故障转移任务**对话框。
4. 选择是否保留副本虚拟机的快照，默认保留。
5. 如果存在测试虚拟机，需选择是否保留测试虚拟机，默认保留。
6. 单击**创建**。

创建永久故障转移任务后，复制对象即进入`永久故障转移中`状态。复制服务会将原虚拟机从复制计划中移除，将副本虚拟机转化为普通虚拟机，并按照设定保留或删除副本虚拟机的快照和测试虚拟机。

> **说明**：
>
> - 在虚拟机永久故障转移期间，如果其关联的异步复制计划自动执行或手动执行，该虚拟机的复制任务将会跳过。
> - 若永久故障转移任务失败，则复制对象的故障转移状态将由`永久故障转移中`变更为`故障转移中`。

## 管理复制对象 > 删除虚拟机复制对象

# 删除虚拟机复制对象

如果复制对象不属于任何异步复制计划，删除复制对象将删除复制对象所有恢复点、测试虚拟机、副本虚拟机及相关的快照。

如果复制对象属于某一异步复制计划，删除复制对象会将复制对象从复制计划移除，并删除复制对象所有恢复点、测试虚拟机、副本虚拟机及相关的快照。

> **注意**：
>
> 该操作不可撤回，请谨慎操作。

**前提条件**

- 复制对象所在集群关联的复制服务未处于`升级中`或`异常`状态。
- 复制对象处于`就绪`状态，且当前没有正在执行的故障转移任务。

**操作步骤**

1. 在**备份与容灾**界面，进入**副本虚拟机**页面，在复制对象列表选择待删除的复制对象，单击右侧 **...**，选择**删除**。
2. 在弹出的对话框确认提示信息后，单击**删除**，完成操作。

## 管理恢复点 > 删除恢复点

# 删除恢复点

根据恢复点的执行类型，恢复点可被自动删除或由用户手动删除。

- 自动执行异步复制计划生成的恢复点会根据恢复点保留策略进行保存，超过保留范围的恢复点会被自动删除，用户也可手动删除。
- 其他情况下生成的恢复点默认永久保留，仅支持手动删除。

> **注意**：
>
> 执行异步复制任务依赖最新的恢复点，因此无法删除最新的恢复点。该操作不可撤回，请谨慎操作。

**前提条件**

手动删除恢复点时，需保证复制对象处于`就绪`状态，且没有正在进行的故障转移任务。

**操作步骤**

1. 在**备份与容灾**界面，选择**异步复制计划**，选择一个复制计划，单击计划名称详情页。
2. 选择**复制对象**，单击虚拟机名称，在右侧页面通过以下方式删除恢复点。

   - 单击某一恢复点右侧 **...**，选择**删除**，可删除该恢复点。
   - 勾选恢复点左侧复选框，单击右上角**删除**，可删除一个或多个恢复点。
   - 单击页面右侧 **...**，选择**删除全部恢复点**，可删除该复制对象除最新恢复点外的所有恢复点。
3. 在弹出的页面中，确认提示信息，单击删除按钮，完成删除。

  选择**虚拟机**标签，可查看当前 CloudTower 管理的所有 SMTX OS（ELF）集群中异步复制保护的虚拟机数量。

  选择**虚拟机**标签，可查看所有虚拟机异步复制计划的执行情况。绿色表示任务执行成功，红色表示任务执行失败。
