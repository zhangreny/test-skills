## A-E

# A-E

安全审计员
:   CloudTower 提供的默认角色之一，拥有查看事件审计信息的权限。

CloudTower
:   SmartX 多集群资源集中管理平台，可统一管理多种类型的 SmartX 集群及其资源。

CloudTower 代理
:   部署在虚拟机中，用于提升跨集群数据传输时的传输效率。

超融合
:   一种 IT 基础架构，通过将计算虚拟化、分布式存储、网络及各类运维管理等服务融合部署在通用的 x86\_64 或 AArch64 架构的服务器上，以简化基础设施类资源的交付、上线和运维。

超融合集群
:   指以超融合形态部署的物理节点集群，由至少 3 个运行 SMTX OS 软件的节点组成，通常包括未启用双活特性的 SMTX OS 集群和启用双活特性的 SMTX OS 集群（简称“SMTX OS 双活集群”）。

超级管理员
:   CloudTower 提供的默认角色之一，拥有系统的所有权限。

次级可用域
:   指 SMTX OS 双活集群正常工作时业务主要运行的可用域。

ELF
:   SmartX 基于 KVM 开发的原生虚拟化计算平台，提供虚拟机生命周期管理、电源操作、高可用、冷热迁移等基本功能，并能与 SmartX 设计研发的分布式块存储服务（ZBS）结合，提供完整的超融合虚拟化解决方案。

## F-J

# F-J

分布式存储集群
:   基于 SMTX ZBS 软件构建的物理节点集群，由至少 3 个节点构成。通常指 SMTX ZBS 集群，包括未启用双活特性的 SMTX ZBS 集群和启用双活特性的 SMTX ZBS 双活集群（简称“SMTX ZBS 双活集群”）。

ISO 映像
:   一种光盘的存档文件，用于为虚拟机安装操作系统，安装应用程序，传输数据或修复软件。

工作负载集群使用者
:   CloudTower 提供的默认角色之一，拥有查看所属工作负载集群列表的权限。

---

## K-O

# K-O

内容库
:   CloudTower 提供的用于管理虚拟机模板和 ISO 映像的功能。

可用域
:   SMTX OS 双活集群的两个可用域一般位于同一个城市且相互间有一定距离，分别属于不同的数据中心。每个可用域至少包含 3 个节点，并且可以完全独立地为双活集群提供计算和存储服务。

---

## P-T

# P-T

SmartX 集群
:   一般位于实体数据中心机房内，包含至少 3 台主机。可分为三类：超融合集群、分布式存储集群和虚拟化集群。

SMTX OS 集群
:   属于逻辑概念。在实际生产环境中，一个 SMTX OS 集群由至少 3 个运行了 SMTX OS 软件的节点通过网络互连组成。

SMTX OS (ELF) 集群
:   虚拟化平台为 ELF 的 SMTX OS 集群的简称。

SMTX OS (VMware ESXi) 集群
:   虚拟化平台为 VMware ESXi 的 SMTX OS 集群的简称。

SMTX OS 双活集群
:   以拉伸形态部署的集群，由两个可用域和一个仲裁节点组成。两个可用域与仲裁节点间通过网络连接通信。SMTX OS 双活集群也简称为双活集群。

SMTX ZBS 集群
:   属于逻辑概念，在实际生产环境中，一个 SMTX ZBS 集群包含两个集群，分别是运行块存储文件的集群和运行文件存储软件的集群。

SMTX ZBS 双活集群
:   以拉伸形态部署的集群，由两个可用域和一个仲裁节点组成。每个可用域的节点和仲裁节点上均运行了 SMTX ZBS 块存储软件，两个可用域与仲裁节点间通过网络连接通信。SMTX ZBS 双活集群也简称为双活集群。

SMTX ELF 集群
:   属于逻辑概念。在实际生产环境中，一个 SMTX ELF 集群至少由 3 个运行 SMTX ELF 软件的节点通过网络互连组成。

## U-Z

# U-Z

VMware ESXi 平台
:   ESXi 是 vSphere 的裸机虚拟机管理程序，可独立安装和运行在物理服务器上。VMware ESXi 平台特指运行着 ESXi 程序的虚拟化平台。

虚拟化集群
:   基于 SMTX ELF 虚拟化软件构建的物理节点集群，由至少 3 个节点组成，通常指 SMTX ELF 集群。

虚拟机使用者
:   CloudTower 提供的默认角色之一，拥有通过虚拟机工具编辑虚拟机信息、打开控制台和对虚拟机进行电源操作的权限。

运维管理员
:   CloudTower 提供的默认角色之一，拥有和超级管理员相同的权限。

用户管理员
:   CloudTower 提供的默认角色之一，拥有基础设施、运维管理和设置的基本可读权限；用户与角色管理的操作权限及用户信息的可读权限。

优先可用域
:   指 SMTX OS 双活集群正常工作时业务主要运行的可用域。

仲裁节点
:   一般位于两个可用域所在数据中心以外的其他数据中心，主要为保障 SMTX OS 双活集群选主类服务正常运行，并使得业务在优先可用域或次级可用域中可以自动切换。

只读用户
:   CloudTower 提供的默认角色之一，拥有基础设施、运维管理和设置的基本可读权限。
