## 概述

# 概述

SMTX 备份与容灾的备份服务将虚拟机整机备份至集群外部的存储库（如 NAS 服务器），并可通过备份在存储库中的数据文件和配置文件，将虚拟机恢复至备份时的状态。

![](https://cdn.smartx.com/internal-docs/assets/d668feb7/installation_guide_003.png)

备份与恢复可应用于以下场景：

- **数据保护**：防止人为误操作、病毒感染、逻辑错误等因素导致的数据损坏和丢失。
- **数据复用**：利用备份数据进行开发、测试、故障排除等，避免影响生产环境。
- **数据安全**：防止因勒索软件攻击导致的大规模数据丢失。

## 相关概念

# 相关概念

本章解释了手册中使用的专业术语。在阅读过程中，如果遇到不熟悉的概念，可在此快速查找其定义和详细解释。

**备份服务**

备份服务负责管理备份计划，并执行虚拟机的备份和恢复任务。

**备份计划**

用于规划和安排数据的备份操作。通过备份计划，用户可以定义备份对象、备份服务、备份存储库、备份周期、备份窗口、恢复点保留策略等关键信息。

**备份对象**

备份的虚拟机。

**备份周期**

备份计划自动执行的频率和时间。用于指定备份服务对虚拟机进行备份的时间间隔。较短的备份周期可以实现更小的 RPO（Recovery Point Objective，故障恢复点目标）。例如，如果备份周期设置为 15 分钟，那么在发生灾难性事件时，最多可能丢失 15 分钟的数据。

**备份窗口**

允许备份计划自动执行的时间范围。

**全量备份**

将虚拟机的所有数据，包括操作系统、应用程序、文件和配置等备份到存储库。全量备份通常发生在初次备份时，并为后续的增量备份提供基准。

**增量备份**

备份上次备份后发生变化的数据，能够持续跟踪和同步数据源的变化，保障数据的准确性和一致性。

**备份恢复点**

每次执行备份操作时生成的虚拟机数据状态，以备份文件的形式存储在备份存储库中。恢复点完整记录了虚拟机的数据和配置信息，反映了虚拟机在备份操作执行时的特定状态。

恢复点可以使虚拟机在需要时能够迅速恢复到特定的数据状态。例如在灾难恢复场景中将虚拟机恢复到某个先前的时间点，以恢复虚拟机的正常运行。通过定期生成恢复点，可以确保虚拟机的数据是最新的，保障业务的连续性和可用性；也可以使用恢复点对应的备份文件重建一个对应时间点状态的虚拟机，用于开发测试等。

**压缩**

对虚拟卷数据先执行压缩后再传输到备份存储库并存储为备份文件。

**原机恢复**

一种恢复模式，可将虚拟机的虚拟卷数据恢复到和生成恢复点时一致。

**重建**

一种恢复模式，可创建一个新的和生成恢复点时一致的虚拟机。

**RPO (Recovery Point Objective)**

数据恢复点目标，主要指在故障恢复场景中允许的数据损失的时间范围。RPO 以时间为度量单位，标示了从最近执行的备份或恢复时间点到发生故障时的时间跨度。较小的 RPO 值表示系统要求在发生故障时允许的数据损失较少。较大的 RPO 值意味着在故障发生时系统可以容忍较长时间范围内的数据丢失。

**VPC**（Virtual Private Cloud）

虚拟专有云，是一套为云资源构建的逻辑隔离的网络环境，用户可以在 VPC 中自主设置 VPC 子网、IP 地址、路由表、浮动 IP、网关服务和安全服务等，以实现网络隔离和安全控制。副本虚拟机和原虚拟机可以部署在相同或不同的 VPC 中，并支持灵活的网络映射。

**VPC 子网**

VPC 子网是组成 VPC 的基础网络模块，用于划分 VPC 的 IP 地址空间。位于同一 VPC 子网的虚拟机默认可以直接通信，跨 VPC 子网虚拟机在在路由可达的情况下也可以互相通信。

**浮动 IP**

浮动 IP 是一种网络地址转换（NAT）机制，通过一对一映射的方式，将外部子网中的 IP 地址动态绑定到 VPC 内部的虚拟机上。这使得外部网络可以主动访问 VPC 内的虚拟机。浮动 IP 为 VPC 内的虚拟机提供了灵活的外部访问能力，同时也保证了网络安全性。

**SMTX OS 集群**

SMTX OS 集群属于逻辑概念。在实际生产环境中，一个 SMTX OS 集群至少由 3 个节点通过网络互连组成。

**ELF**

ELF 平台即指 SMTX OS 原生虚拟化计算，是当 SMTX OS 以超融合形态直接部署在 x86\_64/AArch64 架构的服务器上时，该超融合软件中基于 KVM 的虚拟化引擎组件。ELF 除提供虚拟机生命周期管理、电源操作、高可用、冷热迁移等基本功能外，通过与存储组件 ZBS 相结合，还能提供秒级快照、模板和克隆等高级的虚拟机服务。

ELF 简单易用，对 SMTX OS 超融合集群的所有操作均可通过 CloudTower 完成。

**CloudTower**

SmartX 多集群资源集中管理平台，旨在管理站点内数据中心及跨数据中心的多个 SmartX 集群和系统服务，并提供完整和标准的 RESTful API 和多种语言的 SDK。

## 使用前规划 > 规划备份计划

# 规划备份计划

备份服务使用备份计划来管理备份策略。备份计划定义了备份服务、备份对象、备份存储库、备份周期和恢复点保留策略等。在创建备份计划前，请从以下几个方面规划备份计划：

- **备份对象**

  每个备份计划中可包含一个或多个虚拟机，这些虚拟机共享相同的备份策略。因此，建议根据不同的数据保护要求，对虚拟机进行分类，将具有相同备份周期、备份窗口和保留策略的虚拟机置于同一个备份计划中，避免重复定义相同的备份策略，提高运维和管理效率。
- **备份类型**

  备份类型包括全量备份和增量备份，且在首次备份时会执行全量备份。您可以采取全量备份和增量备份的组合，创建最符合实际需求的备份策略。

  - 全量备份：在执行备份的时间点对虚拟机的所有数据进行完全拷贝。
  - 增量备份：在上一次备份（全量备份或增量备份）的基础上，备份所有发生变化的数据。
- **备份周期**

  通过设定备份周期，备份服务可以对虚拟机自动化进行周期性的备份。您可以根据业务系统的 RPO（Recovery Point Objective）目标，正确设置数据备份的时间间隔。备份支持的最小 RPO 为 15 分钟，因此可根据业务需求合理选择备份周期，以确保数据的及时备份和恢复能力。
- **恢复点保留策略**

  每次执行完备份计划后，备份服务会为备份计划中的每个虚拟机创建一个恢复点，恢复点包含了备份时间点的所有虚拟机数据，在需要时，可通过恢复点将虚拟机恢复至备份时间点的状态。

  恢复点保留策略可以定义保留的恢复点数量或时间范围，当备份数据不需要长期保存时，可通过恢复点保留策略控制备份存储库中的恢复点数量。

## 使用前规划 > 规划备份存储库容量

# 规划备份存储库容量

备份存储库是用于存储虚拟机备份文件的逻辑存储库，如 NAS 服务器。

请根据虚拟机的数量、分配的逻辑容量、当前使用容量、每天实际变化的数据量等信息，并结合虚拟机的备份周期和恢复点保留策略，预估当前容量情况和容量增长趋势，规划备份存储库的容量需求。

## 使用前规划 > 规划许可单元的分配

# 规划许可单元的分配

SMTX 备份与容灾的软件许可由备份与复制两个功能模块共享。许可单元根据所保护对象类型及所用功能进行消耗，具体规则如下：

- **SMTX OS（ELF）集群虚拟机**

  - 备份功能：每备份一个虚拟机，消耗 1​ 个许可单元。
  - 异步复制功能：每异步复制一个虚拟机，消耗 2​ 个许可单元。
  - 同步复制功能：每同步复制一个虚拟机，消耗 4​ 个许可单元。
- **SMTX ZBS 块存储集群存储卷**：每异步复制 1 TiB 的存储卷，消耗 2​ 个许可单元。

请您根据实际的业务需求，综合考虑要使用的功能模块，需要保护的虚拟机和存储卷数量、以及未来业务的扩展等，提前规划许可单元的分配。

## 管理关联集群 > 添加关联集群

# 添加关联集群

完成备份服务的部署后，请将备份服务和待保护的虚拟机所在的 SMTX OS（ELF）集群关联。

- 一个备份服务可以关联多个 SMTX OS（ELF）集群，一个 SMTX OS（ELF）集群也可以关联到多个备份服务。
- 备份服务仅可关联与 CloudTower 的连接状态为`已连接`的集群。

**操作步骤**

1. 登录 CloudTower，在**备份与容灾**界面中，单击**设置**。
2. 在弹出的窗口中，选择一个已部署的备份服务，单击 **SMTX OS 集群关联**。
3. 单击**编辑集群关联**，在弹出的操作界面中，添加一个或多个 SMTX OS（ELF）集群。
4. 单击**保存**，完成备份服务和集群的关联操作。

完成关联后，在 **SMTX OS 集群关联**界面，可查看备份服务已关联的集群列表。

| 参数 | 说明 |
| --- | --- |
| 集群名称 | 备份服务关联的集群名称。 |
| 与 CloudTower 的连接状态 | 集群与 CloudTower 的连接状态。 |
| 集群 IP | 集群关联至 CloudTower 时使用的 IP 地址。 |
| 所属数据中心 | 集群在 CloudTower 中所属的数据中心。 |

## 管理备份存储库 > 创建备份存储库

# 创建备份存储库

备份存储库用来存放备份虚拟机时生成的备份文件，目前支持使用 `NFS 3` 和 `NFS 4` 协议的 NAS 服务器。备份存储库由关联的备份服务进行管理。一个备份服务可以关联多个备份存储库，但一个备份存储库只能关联一个备份服务。

**前提条件**

备份存储库目标路径需对备份服务开放读写权限。

**操作步骤**

1. 登录 **CloudTower**，在**备份与容灾**界面，选择**备份存储库**，单击**创建备份存储库**，弹出创建界面。
2. 设置备份存储库的名称和描述。
3. 选择备份存储库关联的备份服务。建议选择和备份存储库位于同一个数据中心的备份服务。
4. 选择备份存储库类型，并配置存储库的服务器地址和存储文件夹路径。

   - **备份存储库类型**

     选择 NAS 服务器支持的协议版本，若同时支持 `NFS 3` 和 `NFS 4` 两种协议，推荐选择 `NFS 4`。
   - **服务器**

     输入备份存储库的服务器的 IP 地址。
   - **文件夹**

     输入存储备份数据的目标路径。需确保该路径为空目录（允许存在以 `.` 开头的系统隐藏文件），且路径不得与 CloudTower 中已有的其他备份存储库重复。若输入的路径不存在，但其父目录存在并且具备读写权限，备份服务将在父目录下自动创建该路径。

## 管理备份存储库 > 编辑备份存储库

# 编辑备份存储库

创建备份存储库后，备份服务将和备份存储库进行连接和初始化配置。若初始化失败，您可以修改备份存储库名称、描述、备份存储库类型、服务器地址和文件夹路径，但不可修改关联的备份服务，完成修改后备份存储库将尝试进行重新连接。在其他状态下，备份存储库仅支持修改名称和描述，不支持修改关联的备份服务、备份存储库的类型、服务器地址和文件夹路径。

**操作步骤**

1. 在**备份存储库**列表，选择一个备份存储库，单击 **...**，选择**编辑**；或单击某个备份存储库进入备份存储库详情面板，选择**编辑**。
2. 在弹出的**编辑备份存储库**界面中编辑相关信息。
3. 单击**保存**。

## 管理备份存储库 > 恢复备份存储库

# 恢复备份存储库

当备份服务出现异常且不可恢复，或者您在新的环境中重新部署了备份服务时，可以通过**恢复备份存储库**功能，使用新的备份服务接管原有的备份存储库，利用存储库中原有的数据和元数据，快速重建备份计划并关联历史恢复点。

**适用场景**

恢复备份存储库适用于以下场景：

- 备份服务发生故障：备份服务虚拟机因硬件故障、数据损坏等原因无法恢复，需要部署新的备份服务。
- CloudTower 发生故障：CloudTower 发生故障需要重新部署，同时需要恢复原有的备份管理功能。

**前提条件**

- 待恢复备份存储库之前关联的备份服务虚拟机已关机或不可用，避免多个备份服务同时访问同一个存储库造成数据冲突；
- 新的备份服务已经部署完成，且服务状态正常；
- 备份存储库所关联的原备份服务版本必须为 2.3.0 及以上，新关联的备份服务的版本必须不低于原备份服务版本；
- 新备份服务的备份网络必须能够访问待恢复的备份存储库。

**操作步骤**

1. 登录 **CloudTower**，在**备份与容灾**界面，选择**备份存储库**，单击界面右上角 **...**，选择**恢复备份存储库**。
2. 在弹出的窗口中，输入恢复后的存储库名称和描述。名称和描述支持自定义，可与原名称和描述不同。
3. 选择存储库新关联的备份服务，版本需不低于 2.3.0，且不低于原关联的备份服务版本。
4. 选择备份存储库类型，并输入存储库的服务器地址和存储文件夹路径。

   - **备份存储库类型**

     选择存储库使用的 NFS 协议版本（`NFS 3` 或 `NFS 4`）。可根据当前环境选择，无需与原配置保持一致。
   - **服务器**

     输入备份存储库的服务器的 IP 地址，**必须与原存储库地址完全一致**。
   - **文件夹**

     输入存储备份数据的目标路径，**必须与原存储库路径完全一致**。
5. 单击**恢复**，开始恢复操作。

完成恢复后，可进入**备份计划**页面，查看恢复的备份计划。

## 管理备份计划 > 创建备份计划

# 创建备份计划

备份计划用于定义具体的备份对象和执行策略，如备份的虚拟机、备份存储库，以及备份周期和恢复点保留策略等。

登录 CloudTower，在**备份与容灾**界面，选择**备份计划**，单击**创建备份计划**，弹出创建界面，设置备份计划的**基本信息**、**备份周期**和**保留策略**。

## 管理备份计划 > 创建备份计划 > 设置基本信息

# 设置基本信息

1. 输入备份计划的名称和描述。
2. 设置备份计划使用的备份服务和备份存储库，并选择是否启用文件系统一致性和压缩。

   | 配置项 | 说明 |
   | --- | --- |
   | 备份服务 | 选择备份服务。建议选择和待备份的虚拟机位于同一个数据中心的备份服务。 |
   | 备份存储库 | 选择备份存储库。仅可选择与备份服务关联的备份存储库。 |
   | 文件系统一致性 | 默认启用。对于已安装虚拟机工具且处于运行中状态的虚拟机，通过静默或冻结文件系统，确保快照捕获时所有磁盘的 I/O 已完成，且数据处于可恢复的完整状态，从而实现文件系统一致性备份；其他虚拟机将进行崩溃一致性备份，快照中不会包含虚拟机文件系统的内存数据。 关闭后将对所有虚拟机进行崩溃一致性备份，相关快照中不包含虚拟机文件系统的内存数据。  **注意**：虚拟机创建文件系统一致性快照期间将暂时无法使用。 |
   | 压缩 | 选择是否启用备份数据压缩，默认启用。虚拟机的虚拟卷数据传输到备份存储库后会先执行压缩，然后存储为备份文件。 |
3. 选择加入备份计划的虚拟机，可选择一个或多个虚拟机。支持按照虚拟机名称、所属集群名称、标签名和标签值筛选虚拟机。

   > **说明**:
   >
   > - 不支持备份以下虚拟机：
   >
   >   - 挂载了共享虚拟卷的虚拟机。
   >   - 未挂载虚拟卷的虚拟机。
   >   - 运行系统服务的虚拟机，如 Everoute Controller 虚拟机、CloudTower 虚拟机、备份服务虚拟机、复制服务虚拟机、可观测性虚拟机、高级监控虚拟机、安全防护虚拟机（SVM）等。
   >   - SKS 工作负载集群节点虚拟机。
   >   - 容器镜像仓库虚拟机。
   >   - 处于`操作中`或`未知`状态的虚拟机。
   >   - 所在集群与 CloudTower 连接异常的虚拟机。
   >   - 所在集群的许可类型为试用许可且许可已过期的虚拟机。
   >   - 复制服务创建的测试虚拟机和副本虚拟机。
   > - 对于已加入备份计划的虚拟机，在进行不保留源虚拟机的跨集群冷迁移和分段迁移时：
   >
   >   - 若迁移的目标集群关联了当前备份服务，备份计划将继续保护该虚拟机；
   >   - 若迁移的目标集群未关联当前备份服务，虚拟机迁移后将从该备份计划删除；
   >   - 对于通过冷迁移或分段迁移迁移至低于 5.1.3 的 5.x.x 集群或 6.0.x 集群的虚拟机，迁移完成后将被移出备份计划。
   > - 对于已加入备份计划的虚拟机，若进行保留源虚拟机的跨集群冷迁移和分段迁移，将继续保护源虚拟机而非迁移后的虚拟机。

## 管理备份计划 > 创建备份计划 > 设置备份周期和时间窗口

# 设置备份周期和时间窗口

设置备份计划自动执行的周期，包括设置备份周期、指定全量备份周期、设置备份窗口。

1. **设置备份周期**

   | 执行类型 | 默认周期 | 支持设置周期 | 说明 |
   | --- | --- | --- | --- |
   | 分钟 | 15 分钟 | 15 ~ 59 分钟 | 系统将从每日 00:00 开始计时，按照设置的时间周期执行备份任务。 |
   | 小时 | 1 小时 | 1 ~ 23 小时 |
   | 天 | 1 天 | 1 ~ 30 天 | 需要为执行当天设置一个或多个执行时间，两个执行时间最小间隔为 15 分钟。 |
   | 星期 | 每星期一 | 星期一 ~ 星期日 | 可选择在每周中的一天或几天执行，并需要为执行当天设置一个或多个执行时间，两个执行时间最小间隔为 15 分钟。 |

   > **说明**：
   >
   > 在备份周期的自动执行时间点，如果上一次自动执行任务尚未结束，本次执行将跳过。
2. **设置全量备份周期**

   在所有备份的执行时间中指定全量备份的执行时间，未被指定的时间点将执行增量备份。

   - 当备份周期选择按**分钟**、**小时**或**天**执行时，可将全量备份周期设置为按**天**、**星期**或**月**执行。设置完成后，需要在全量备份的执行周期内，从备份的所有执行时间中选择一个作为全量备份的执行时间。
   - 当备份周期选择按**星期**执行时，全量备份可选择按**星期**、**月**执行。设置完成后，需要在全量备份的执行周期内，从备份的所有执行时间中选择一个作为全量备份的执行时间。
3. **设置备份窗口**

   选择是否启用**仅允许在备份窗口内执行备份任务**，默认不启用。启用后：

   - 备份窗口开始和结束的间隔需不少于 15 分钟。
   - 备份周期中设定的执行时间必须在备份窗口的时间范围内。若执行时间和备份窗口结束时间相同，将不会发起备份任务。
   - 未在时间窗口内执行完成的备份任务在超过备份窗口的结束时间后将默认被停止，且对应虚拟机的快照和备份文件将会被删除。如果不希望停止这类备份任务，请勾选**若备份不能在时间窗口内完成，继续完成当前备份**。

## 管理备份计划 > 创建备份计划 > 设置恢复点保留策略

# 设置恢复点保留策略

设置恢复点的保留策略，保留策略仅适用于备份计划自动执行生成的恢复点，手动执行备份计划创建的恢复点将永久保留。

| 配置项 | 说明 |
| --- | --- |
| 按时间保留 | 保留最近 n 天内的恢复点。 |
| 按份数保留 | 保留最近的 n 份恢复点。 |
| 永久保留 | 永久保留恢复点 |

> **说明**:
>
> 备份服务采取了全量+增量的组合备份方式，在进行全量备份后，后续的增量备份皆在前一次备份的基础上进行，当进行虚拟机恢复时，需依赖完整的全量备份恢复点和所有的增量恢复点。因此，为保证可以正常恢复虚拟机，备份服务将保留全量恢复点和所有增量恢复点，即使恢复点数量已超过保留范围。当下一次全量备份及关联的增量备份恢复点满足相关保留策略后，上一次全量备份及其相关的增量备份恢复点将被删除。
>
> 例如：备份服务每天 0 点执行一次全量备份，每 1 个小时执行一次增量备份，设置的恢复点保留策略为 7 份。那么，当日 5 点时将保留前一天 0 点至当日 5 点产生的所有全量恢复点和增量恢复点，共计 30 份；6 点之后将删除前一天产生的所有恢复点，保留当日 0 点及之后产生的恢复点，确保最近的 7 份恢复点被保留。

## 管理备份计划 > 手动执行备份计划

# 手动执行备份计划

已创建的备份计划默认将按照设定的备份周期自动执行，也可手动执行。

**前提条件**

备份计划当前没有其他手动执行的任务。

**操作步骤**

1. 在**备份计划**列表选择待执行的备份计划，在列表右侧单击 **...**，或在备份计划详情页，选择**手动执行**。

   - 若备份计划中存在虚拟机不包含手动执行生成的恢复点，仅支持选择全量备份。
   - 若备份计划中所有虚拟机均有手动执行生成的恢复点，支持选择全量备份或增量备份。
3. 单击**手动执行**，提交手动执行备份计划的任务并自动创建备份虚拟机的任务，备份虚拟机的任务创建后可在任务中心停止备份任务。

## 从恢复点恢复虚拟机

# 从恢复点恢复虚拟机

每次执行备份计划，会为备份计划中包含的每个虚拟机创建一个恢复点。您可以通过恢复点对虚拟机进行恢复。

虚拟机的支持以两种方式恢复：原机恢复和重建虚拟机。

- **原机恢复**

  原机恢复指将原虚拟机的虚拟卷数据恢复到和生成恢复点时一致，且不改变原虚拟机的配置信息。原机恢复会覆盖原虚拟机，请谨慎操作。

  > **注意**：
  >
  > 以下场景中，无法进行原机恢复：
  >
  > - 原虚拟机处于`暂停`、`操作中`或`未知`状态；
  > - 原虚拟机已被删除、被移入回收站、被迁移到其他集群或被转化为虚拟机模板；
  > - 原虚拟机在生成恢复点后对虚拟卷进行了添加、删除或扩容操作；
  > - 原虚拟机所在集群已从 CloudTower 中移除。
- **重建虚拟机**

  重建虚拟机指使用恢复点创建一个新的虚拟机，原虚拟机将被保留。重建后的虚拟机的配置信息和虚拟卷数据与原虚拟机生成恢复点时一致。

**注意事项**

若恢复的目标集群的许可类型为试用许可且许可已过期，则虚拟机恢复会失败。

**操作步骤**

1. 登录 **CloudTower**，在**备份与容灾**界面，单击**备份计划**，选择一个备份计划，单击备份计划名称，进入备份计划详情页。
2. 选择要恢复的虚拟机，单击虚拟机名称，根据时间点选择一个恢复点。
3. 单击**恢复**，在弹出的**恢复虚拟机**界面中选择恢复模式并根据需要输入配置信息。

   - **原机恢复**

     将原虚拟机的虚拟卷数据恢复到和生成恢复点时一致，原机恢复不会改变虚拟机的配置信息。

     > **说明**：
     >
     > 原机恢复时，若虚拟机处于运行状态，会先对虚拟机进行关机操作。
   - **重建虚拟机**

     使用恢复点创建一个新的虚拟机，虚拟机的配置信息和虚拟卷数据与原虚拟机生成恢复点时一致。

     - **设置重建虚拟机的基本信息**

       | 配置项 | 说明 |
       | --- | --- |
       | 目标集群 | 选择重建虚拟机的目标集群。支持选择相同 CPU 架构类型的 SMTX OS（ELF）集群。 |
       | 目标主机 | 选择重建虚拟机的目标主机。可选择目标集群中一个特定的主机，默认为**自动调度至合适的主机**。 |
       | 重建虚拟机名称 | 输入重建虚拟机的名称，需保证和目标集群中已有虚拟机的名称不相同。默认为生成恢复点时虚拟机的名称。 |

       > **说明**：
       >
       > 若重建虚拟机选择的目标集群非原虚拟机所在集群，需保证备份服务的存储网络和目标集群的存储网络连通。
     - **设置重建虚拟机存储策略**

       选择重建虚拟机的存储策略。可选择与原虚拟保持一致，或自定义新的存储策略。

       > **说明**：
       >
       > 若原虚拟机的存储策略为纠删码，但目标集群不满足使用纠删码的前提条件，则重建虚拟机的存储策略无法选择和原虚拟保持一致。在此情况下，请为重建虚拟机自定义存储策略。
     - **设置网络映射**

       重建虚拟机时，为确保业务可以继续正常运行，需为重建后的虚拟机配置虚拟机网络映射。

       - 若目标集群已关联 VPC 网络，映射的网络类型可选择 VPC 或 VLAN。
       - 若目标集群不支持 VPC 功能或未关联 VPC 网络，映射的网络类型只能选择 VLAN。此时建议选择与源端 VLAN 类型（Access 或 Trunk）相同的虚拟机网络。
4. 选择是否**恢复完成后自动开机**，默认开启。
5. 单击**恢复**，提交恢复虚拟机任务。任务创建后可在任务中心停止恢复任务。

## 管理恢复点 > 删除恢复点

# 删除恢复点

根据恢复点的执行类型，恢复点可被自动删除或手动删除。

- 自动执行备份计划生成的恢复点会根据恢复点保留策略进行保存，超过保留范围的恢复点会被自动删除，您也可手动删除。
- 手动执行备份计划生成的恢复点默认永久保留，仅支持手动删除。

> **注意**：
>
> 根据恢复点的类型，手动删除恢复点时：
>
> - 若当前恢复点的备份类型是全量备份，将同时删除所有依赖该恢复点的增量恢复点；
> - 若当前恢复点的备份类型是增量备份，将同时删除该恢复点依赖的所有全量和增量恢复点，以及所有依赖该恢复点的增量恢复点。

**前提条件**

手动删除恢复点时，需保证当前没有依赖这些恢复点的备份任务或恢复任务。

**操作步骤**

1. 在**备份**界面，选择**备份计划**，选择一个备份计划，单击备份计划名称，进入备份计划详情页。
2. 选择**虚拟机恢复点**，单击虚拟机名称，在右侧页面通过以下方式删除恢复点。

   - 单击某一恢复点右侧 **...**，选择**删除**，可删除该恢复点。
   - 勾选恢复点左侧复选框，单击右上角**删除**，可删除一个或多个恢复点。
   - 单击页面右侧 **...**，选择**删除全部恢复点**，可删除该虚拟机所有恢复点。
3. 在弹出的页面中，确认提示信息，单击删除按钮，完成删除。

## 查看备份的概况

# 查看备份的概况

在**备份与容灾**的**概览**页面中可查看被保护虚拟机的总体情况、备份计划的执行记录概况、备份计划的状态信息、备份存储库的容量使用情况和备份服务的健康状况等信息。

- **被保护资源**

  选择**虚拟机**标签，可查看当前 CloudTower 管理的所有 SMTX OS（ELF）集群中未保护的虚拟机总数和被保护的虚拟机数量，以及被保护的虚拟机数量总和占虚拟机总数的比例。
- **执行记录**

  选择**虚拟机**标签，可查看所有虚拟机备份计划和复制计划的执行情况。绿色表示任务执行成功，红色表示任务执行失败。
- **备份计划**

  可查看已创建的备份计划总数、生效中的备份计划数量和已暂停的备份计划数量。
- **备份文件压缩比**

  可查看备份全局的压缩比，仅统计开启压缩的备份计划中成功生成的恢复点的压缩情况。压缩比 = 备份有效容量 / 备份文件总容量。

  - 备份有效容量：所有开启压缩的备份计划中成功生成的恢复点的备份有效容量之和。有效容量为备份文件压缩前的容量大小。
  - 备份文件总容量：所有开启压缩的备份计划中成功生成的恢复点的备份文件大小之和。
- **备份存储库总容量**

  可查看所有备份存储库的总容量、容量使用率、已使用容量和未使用容量。
- **容量使用率**

  可查看所有备份存储库容量使用率。
- **备份服务**

  可查看所有备份服务的信息，包括备份服务名称、版本号、状态（包括正在部署、可升级、正在升级、正在卸载、异常，未展示表示状态正常）、关联的 SMTX OS 集群数量、备份存储库数量和备份计划数量。
