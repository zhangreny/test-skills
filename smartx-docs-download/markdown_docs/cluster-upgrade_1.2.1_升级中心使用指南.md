# cluster-upgrade/1.2.1/升级中心使用指南
## 关于本文档

# 关于本文档

本文档描述了如何使用升级中心对 SMTX OS、SMTX ZBS 和 SMTX ELF 集群进行自动化升级、扩容 SCVM 和系统分区，以及安装补丁文件的操作。

## 概述

# 概述

为助力企业建立和完善简单、可靠、开放的现代化 IT 基础设施，SmartX 持续发布了功能更加丰富、性能更加稳定的产品线和版本。但在进行升级操作时，传统的手动升级和命令行升级方式操作较为繁琐，需要大量人力投入，效率较低，且易出现误操作，已无法有效应对日益复杂的版本管理和高效升级的需求。

为了应对这一挑战，我们在 CloudTower 管理平台中推出了升级中心，通过将分散的升级操作统一到一个界面化的流程中，实现了一站式完成集群升级、内核升级、虚拟机迁移等操作，可极大减少所需人为干预，简化升级流程，提升升级的效率、可靠性和成功率。

升级中心包含如下具体功能：

- **通过界面一站式升级集群和内核**

  为集群提供一站式升级界面，自动完成集群升级、内核升级、虚拟机迁移等操作，极大降低操作难度，减少所需人为干预，有效提升升级效率和成功率。
- **通过界面安装补丁文件、扩容系统分区和 SCVM**

  提供界面化的补丁安装、系统分区扩容和 SCVM 扩容方式，可查看安装进度，减少手动操作，提高效率和可靠性。
- **集群升级的统一管理**

  统一管理各集群升级，可结合当前环境中已上传的升级文件，为各版本提供合理的升级建议，并优化组件升级顺序。支持升级文件的统一管理和多集群分发，方便不同集群快速获取和灵活使用升级文件。
- **升级预检查**

  支持在正式执行升级或扩容操作前对当前环境进行全面检查，识别并预警可能存在的问题，帮助运维人员提前发现潜在的升级障碍，从源头规避升级风险，避免错过升级窗口。
- **升级历史记录与管理**

  自动记录并保存升级过程中所有集群和内核的升级日志，形成完整的升级历史方便后续追溯。
- **升级中心独立发布和升级**

  升级中心不依赖其他产品组件，可独立发布和升级新版本，及时支持新功能。同时，可以通过单独更新升级元数据文件支持新版本升级，版本更新更加灵活高效。

## 升级中心配置要求 > CloudTower 用户角色权限要求

# CloudTower 用户角色权限要求

为确保您能正常在 CloudTower 中使用升级中心执行升级和安装补丁操作，以及对升级中心进行运维管理，请进入 CloudTower **系统配置**主界面，选择**角色管理**，确认当前 CloudTower 用户角色已开通**升级中心管理**的操作权限。如未启用，请联系您的系统管理员开通权限。

## 升级中心配置要求 > 升级中心版本元数据文件要求

# 升级中心版本元数据文件要求

升级中心版本元数据文件包含了升级中心支持的所有集群版本的元数据信息、vGPU 驱动文件信息、补丁文件版本信息、升级路径以及内核升级建议。

在升级集群或安装补丁文件前，请先进入升级中心的**设置**页面，确认升级中心当前的版本元数据文件为最新版本，若不是最新版本，请您上传最新的版本元数据文件，并参考[更新升级中心版本元数据文件](/cluster-upgrade/1.2.1/cluster_upgrade_user_guide/cluster_upgrade_user_guide_25)，手动更新元数据文件。

## 升级集群和内核 > 升级前准备 > 确认可观测性服务

# 确认可观测性服务

为集群关联可观测性服务可实现网络流量可视化、监控报警、日志管理等功能，增强集群的运维管理能力。因此，建议为 SMTX OS（ELF）集群、SMTX ZBS 集群和 SMTX ELF 集群关联可观测性服务，以实现统一的指标采集、存储、处理，提升监控效率。

在升级集群前请参考下方表格，根据待升级集群是否已关联可观测性服务，确定升级顺序。如需关联或升级可观测性服务，请参考《可观测性平台用户指南》进行操作。

**注意事项**

由于迁移本地监控数据耗时较长，如果需要进行监控数据迁移操作，请在制定升级计划时预留足够的时间，避免影响整体的升级进度。

## SMTX OS（ELF）集群

| **源版本** | **目标版本** | **是否已关联可观测性服务** | **是否已部署高级监控** | **升级顺序** |
| --- | --- | --- | --- | --- |
| 5.0.5 及以上 | 5.0.x 或 5.1.x | 是 | - | 1. 升级 SMTX OS 集群 2. 升级可观测性服务 |
| 否 | - | 1. 升级 SMTX OS 集群 2. 关联可观测性服务并迁移本地监控数据 |
| 6.x | 是 | - | 1. 升级 SMTX OS 集群 2. 升级可观测性服务 |
| 否 | 是 | 1. 升级 SMTX OS 集群 2. 关联可观测性服务并迁移本地监控数据 |
| 否 | 1. 关联可观测性服务并迁移本地监控数据 2. 升级 SMTX OS 集群 |
| 5.0.5 以下 | 5.0.x 或 5.1.x | 否 | - | 1. 升级 SMTX OS 集群 2. 关联可观测性服务并迁移本地监控数据 |
| 6.x | 是 | 1. 升级 SMTX OS 集群 2. 关联可观测性服务并迁移本地监控数据 |
| 否 | 1. 升级 SMTX OS 集群 2. 关联可观测性服务并联系 SmartX 售后工程师迁移本地监控数据 `[1]` |

## SMTX ZBS 集群

| **源版本** | **目标版本** | **是否已关联可观测性服务** | **升级顺序** |
| --- | --- | --- | --- |
| 5.6.0 及以上 | 5.6.1 及以上 | 是 | 1. 升级 SMTX ZBS 集群 2. 升级可观测性服务 |
| 否 | 1. 升级 SMTX ZBS 集群 2. 关联可观测性服务并迁移本地监控数据 |
| 5.6.0 以下 | 5.6.1 及以上 | 是 | 1. 升级可观测性服务并迁移本地监控数据 `[2]` 2. 升级 SMTX ZBS 集群 |
| 否 | 1. 升级 SMTX ZBS 集群 2. 关联可观测性服务并联系 SmartX 售后工程师迁移本地监控数据 `[1]` |

## SMTX ELF 集群

SMTX ELF 集群的升级顺序与版本无关，请先升级 SMTX ELF 集群，然后再升级可观测性服务，或关联可观测性服务并迁移本地监控数据。

> **说明**:
>
> 集群从低版本升级至 SMTX OS 6.x 版本或 SMTX ZBS 5.6.1 及以上版本后，系统将只保留 1 GB 的本地监控数据。
>
> - **`[1]`**：由于升级前的 SMTX OS 或 SMTX ZBS 版本不支持关联可观测性服务，需在完成升级后才能进行关联。关联可观测性服务后，系统需要保留超过 1 GB 的监控数据，需要联系 SmartX 售后工程师处理。
> - **`[2]`**：集群在升级前已关联可观测性服务，为了在集群升级后能够保留原先的监控数据，需要在集群升级前迁移本地监控数据。

当待升级集群节点的系统分区容量或 SCVM 的 CPU 和内存资源不满足目标版本要求时，您可以参考[扩容系统分区](/cluster-upgrade/1.2.1/cluster_upgrade_user_guide/cluster_upgrade_user_guide_43)和[扩容 SCVM](/cluster-upgrade/1.2.1/cluster_upgrade_user_guide/cluster_upgrade_user_guide_46) 章节完成扩容后再执行升级操作。

> 升级中心仅支持使用目标版本适配升级中心的升级 ISO 文件进行升级，不可使用目标版本的安装 ISO 文件进行升级。

## 升级集群和内核 > 升级前准备 > 获取 vGPU 驱动文件

# 获取 vGPU 驱动文件

在升级 SMTX OS 集群或 SMTX ELF 集群时，若集群满足以下版本要求，并且存在用途为 vGPU 的 GPU 设备，支持在升级内核的同时升级 vGPU 驱动。请根据待升级集群的版本、内核版本和主机操作系统，提前获取 ISO 格式的 vGPU 驱动文件。

- **SMTX OS 集群**：5.1.5 及以上的 5.1.x 版本、6.2.0 及以上的 6.x 版本。
- **SMTX ELF 集群**：6.2.0 及以上版本。

> **说明**：
>
> 升级中心仅支持使用 ISO 格式的 vGPU 驱动文件，不支持使用 RPM 格式的驱动文件。

## 升级集群和内核 > 升级说明 > 升级限制操作

# 升级限制操作

在升级集群和升级内核的过程中，禁止执行如下操作：

- 为集群添加或移除主机。
- 将集群从 CloudTower 中移除。
- 手动将主机进入或退出维护模式。
- 手动关机或重启主机。
- 对主机执行角色转换操作。

## 升级集群和内核 > 升级说明 > 升级工具兼容性要求

# 升级工具兼容性要求

升级工具是包含在集群安装 ISO 文件或升级 ISO 文件中的一个组件，具体提供如下功能：

- **升级前环境检查**：对待升级集群进行全面检查，识别并预警可能存在的问题，确保当前环境满足升级条件。
- **执行升级操作**：升级检查通过后，完成升级所需的全流程操作。

集群在安装部署时，会将升级工具一并自动部署。同时，每个升级 ISO 文件本身也包含了对应版本的升级工具。执行升级时，要求**升级 ISO 文件中包含的升级工具版本必须不低于集群已有的升级工具版本**，以确保升级操作的兼容性和稳定性。因此，在正式执行升级操作前，系统会对比升级工具的版本。具体说明如下：

- 如果升级 ISO 文件中升级工具的版本与集群中升级工具的版本相同，无需重新安装升级工具，系统将直接进入检查环境阶段。
- 如果升级 ISO 文件中升级工具的版本高于集群中升级工具的版本，系统会自动将集群中的升级工具更新为升级文件中的版本，然后进入检查环境阶段。
- 如果升级 ISO 文件中升级的工具版本低于集群中升级工具的版本，界面将显示“升级工具版本不兼容”，请参考附录的[回滚升级工具版本](/cluster-upgrade/1.2.1/cluster_upgrade_user_guide/cluster_upgrade_user_guide_28)，手动回滚升级工具版本，然后重新执行环境检查或升级操作。

## 升级集群和内核 > 升级策略

# 升级策略

在进行升级操作时，根据是否选择升级内核，有以下三种升级策略：

- **仅升级集群**

  该策略仅升级集群，不升级内核，适用于无需要引入新内核特性或新版本内核更新对当前集群不适用的场景。
- **同时升级集群和内核**

  该策略是在集群升级完成后自动升级内核，适用于新版本内存在重大更新或修复了安全漏洞，且有足够升级窗口同时完成集群和内核升级的场景。通过升级内核，可以整体提升系统的稳定性、安全性和性能表现。
- **集群升级完成后单独升级内核**

  该策略是在集群升级完成后，选择合适的时间窗口单独升级内核。

若当前内核版本过低，升级界面将显示**建议**升级，请您根据实际情况选择升级策略。

## 升级集群和内核 > 升级步骤 > 第 1 步：上传升级文件

# 第 1 步：上传升级文件

升级中心使用专门的升级 ISO 文件和元数据文件进行集群升级和内核升级。升级 ISO 文件和元数据文件仅用于升级，不可用于集群的安装部署，需要在升级前上传或分发至待升级集群。若 CloudTower 已上传了升级 ISO 文件，可以通过编辑升级 ISO 文件的所属集群直接将文件分发到当前集群。

1. 进入 CloudTower 的升级中心主界面，单击界面右上角的**上传**按钮，选择**上传升级文件**。
2. 在上传升级文件对话框中选择**从本地选择**或**从 URL 上传**。

   - **从本地选择**：拖拽需要上传的升级 ISO 文件和元数据文件到虚线框中，或者单击**选择文件**并选择待上传的升级文件和元数据文件。
   - **从 URL 上传**：输入需要上传的升级 ISO 文件和元数据文件的 URL，然后单击**解析 URL**。
   > **注意**：
   >
   > - 上传的元数据文件须与升级 ISO 文件相匹配，否则不允许上传。
   > - 升级 ISO 文件 URL 应以 `.iso` 结尾，元数据文件 URL 应以 `.json` 结尾，示例如下：
   >   - 升级 ISO 文件：`http://192.168.xx.xx/xx/hygon-6.2.0-oe2003-250213003751-x86_64-upgrade.iso`
   >   - 元数据文件：`http://192.168.xx.xx/xx/hygon-6.2.0-oe2003-250213003751-x86_64-metadata.json`
3. 选择所要上传的目标集群后单击**上传**。

   上传过程中可在任务中心查看上传进度，可单击任务右上角的 **…**，选择**停止**，将停止上传升级文件至剩余未上传集群。

   > **注意**：
   >
   > 升级文件上传过程中，若因网络异常或刷新浏览器页面导致任务停滞，请在任务列表中手动停止该任务并重新上传。
4. 上传完成后可在任务中心选择对应任务，单击**详情**，查看上传详情。

升级文件上传完毕后，若需要修改升级文件所属集群，或删除升级文件，可在升级中心的**升级文件**页签，单击对应升级文件右侧的 **…**，选择**编辑所属集群**或**删除**升级文件，然后进行相应操作。

> **说明**：
>
> 若在编辑所属集群时清空了所属集群，升级文件将被彻底删除。

## 升级集群和内核 > 升级步骤 > 第 2 步：上传 vGPU 驱动文件

# 第 2 步：上传 vGPU 驱动文件

当集群中存在用途为 vGPU 的 GPU 设备时，在升级内核的同时需要更新 vGPU 驱动以适配内核版本，请在升级前上传 vGPU 驱动文件至待升级集群。若 CloudTower 已上传了 vGPU 驱动文件，可以通过编辑驱动文件所属集群的方式直接将文件分发到当前集群。

1. 单击升级中心主界面右上角的**上传**按钮，选择**上传 vGPU 驱动文件**。
2. 在上传 vGPU 驱动文件对话框中选择**从本地选择**或**从 URL 上传**。

   - **从本地选择**：拖拽需要上传的 vGPU 驱动文件到虚线框中，或者单击**选择文件**并选择待上传的 vGPU 驱动文件。
   - **从 URL 上传**：输入需要上传的 vGPU 驱动文件的 URL，然后单击**解析 URL**。
   > **注意**：
   >
   > vGPU 驱动文件 URL 应以 `.iso` 结尾，示例如下：
   >
   > `http://192.168.xx.xx/xx/VGPU-4.19.90-2307.3.0.el7.v79-el7-250812153202-x86_64.iso`
3. 选择所要上传的目标集群后单击**上传**。

## 升级集群和内核 > 升级步骤 > 第 3 步：（可选）检查升级环境

# 第 3 步：（可选）检查升级环境

建议在正式执行集群升级或内核升级前单独对集群进行全面检查，提前识别潜在问题，以免在正式升级过程中发现问题而错过有限的升级窗口期，提高升级效率。

1. 单击升级中心主界面的**集群**页签，在集群列表中选中待升级集群，弹出集群详情面板。
2. 在集群详情面板的**升级文件**一栏中，单击目标版本升级文件右侧的**检查环境**按钮。

   - 若待升级集群版本低于 SMTX OS 6.0.0 或 SMTX ZBS 5.6.0，且目标版本为 SMTX OS 6.x 版本或 SMTX ZBS 5.6.0 及以上版本，系统会根据集群是否已关联可观测性服务以及是否已部署高级监控进行提示，建议根据提示完成相应操作；您也可以单击**继续检查环境**直接执行预检查。
   - 若待升级集群版本为 SMTX OS 6.x 或 SMTX ZBS 5.6.0 及以上版本，或目标版本为 SMTX OS 6.0.0 以下版本或 SMTX ZBS 5.6.0 以下版本，建议您按照[确认可观测性服务](/cluster-upgrade/1.2.1/cluster_upgrade_user_guide/cluster_upgrade_user_guide_12)中的升级顺序进行升级。
3. 在检查升级环境界面中确认集群当前版本和目标版本，确认无误后，单击**检查**。

   当集群已升至以下目标版本，单独检查内核升级环境时，若主机上需迁出的虚拟机包含“必须”类型且已启用的放置组规则，需要选择迁出虚拟机时是否允许忽略放置组规则。

   - SMTX OS：5.1.5 及以上的 5.1.x 版本或 6.2.0 及以上的 6.x 版本
   - SMTX ELF：6.2.0 及以上版本
   - SMTX ZBS：5.7.0 及以上版本

   系统支持两种策略，**必须遵守规则**和**允许忽略规则**。

   - **必须遵守规则**：系统将遵守放置组规则迁出虚拟机，但“必须”放置组规则可能导致虚拟机无法迁出。
   - **允许忽略规则**：系统将忽略放置组规则迁出虚拟机，在主机退出维护模式后，迁回虚拟机也将忽略放置组规则。

若当前环境通过检查，可进行集群和内核升级。若未通过检查，请根据提示信息解决相关问题后进行升级。

## 升级集群和内核 > 升级步骤 > 第 4 步：升级集群和内核

# 第 4 步：升级集群和内核

1. 在升级中心的集群列表中选中待升级集群，弹出集群详情面板。在升级文件一栏中，单击目标版本升级文件右侧的**升级**按钮。
2. 选择升级范围。请参考[升级策略](/cluster-upgrade/1.2.1/cluster_upgrade_user_guide/cluster_upgrade_user_guide_16)，根据您的实际需求选择升级范围。

   - **升级集群**：若集群未曾执行过升级操作，默认勾选该选项且不可取消。若集群已升级至最新版本，则无法勾选。
   - **升级内核**：根据界面提示确认是否选择升级内核。若您选择升级内核，内核升级将在集群升级完成后自动进行。若所选主机上存在 vGPU 用途的 GPU 设备，在升级内核的同时将自动升级 vGPU 驱动。

     - **升级全部主机的内核**：当集群内所有节点均满足如下要求时，默认选择该项：

       - 当前内核版本低于目标内核版本
       - 节点的管理 IP 未用于关联 CloudTower
     - **升级部分主机的内核**：当集群内存在不满足如上要求的节点时，默认选择该选项，您可以选择升级内核的节点。
3. 选择是否指定升级期间的数据恢复速率。

   - 若不勾选该选项，默认为集群自身设置的数据恢复模式，系统会根据当前节点的硬件配置和业务 I/O 负载，自动调节升级期间的数据恢复速率。
   - 若勾选该选项，则可以指定升级期间的数据恢复速率，默认为 400 MiB/s。
4. （可选）当选择了升级内核时，集群在升级内核过程中会进入主机维护模式，并将节点上的虚拟机迁移或关机，因此需要配置以下选项：

   - **迁出虚拟机时是否允许忽略放置组规则**

     若集群已升至以下版本，且主机上需迁出的虚拟机包含“必须”类型且已启用的放置组规则，需要选择迁出虚拟机时是否允许忽略放置组规则。

     - SMTX OS：5.1.5 及以上的 5.1.x 版本或 6.2.0 及以上的 6.x 版本
     - SMTX ELF：6.2.0 及以上版本
     - SMTX ZBS：5.7.0 及以上版本

     系统支持两种策略，**必须遵守规则**和**允许忽略规则**。

     - **必须遵守规则**：系统将遵守放置组策略迁出虚拟机，但“必须”放置组规则可能导致虚拟机无法迁出。
     - **允许忽略规则**：系统将忽略放置组策略迁出虚拟机，在退出维护模式迁回虚拟机时，也将忽略虚拟机放置组规则。
   - **虚拟机迁回的执行方式**

     在集群退出主机维护模式后，需要将此前迁出的虚拟机迁回，因此还需要选择虚拟机开机或迁回的方式，支持**立即自动执行**和**手动确认执行**两种方式。

     - **立即自动执行**：集群内节点退出主机维护模式后，系统会自动将在进入维护模式过程中关机或迁移的虚拟机开机或迁回虚拟机。
     - **手动确认执行**：集群内节点退出主机维护模式后，需要手动确认是否将在进入维护模式过程中关机或迁移的虚拟机开机或迁回原虚拟机。如需在退出维护模式后，先手动迁回部分虚拟机验证业务是否可以正常运行，可选择此方式，确认无误后再手动点击**执行**按钮，自动迁回其余虚拟机。
5. （可选）设置任务通知。

   - 若升级中心已配置任务通知，任务通知默认启用，可设置任务通知的启用状态和定时通知的间隔时间。
   - 若升级中心未配置任务通知，任务通知默认禁用，可单击**配置**跳转到任务通知配置界面，参考[配置任务通知](/cluster-upgrade/1.2.1/cluster_upgrade_user_guide/cluster_upgrade_user_guide_39)章节进行设置。
6. 单击**升级**，进入自动升级流程。您可以在升级界面查看具体的升级进度。

   如果在升级集群或升级内核的过程中失败，可单击**查看日志**查看失败原因，或在集群详情面板中的**版本变更记录**页签，通过单击升级记录右侧的**详情**查看失败原因，解决问题后单击**重试**，将从失败处继续升级。查看最近一次升级失败记录的升级日志时，您可以在上方搜索框中进行搜索，也可以单击**下载日志**，将日志文件下载至本地进行问题分析。

> **说明**：
>
> - 升级内核过程中，如需中止升级，可单击**停止升级**按钮，系统将立即停止升级。后续如需继续升级，可单击**重试**，将从上次停止的节点开始继续升级。
> - 当选择了升级内核时，节点进入维护模式时可能对部分虚拟机执行了迁移或关机操作，升级完成后，系统将自动检查此类虚拟机是否已恢复至升级前的状态及所属主机，如存在未开机或未迁回的虚拟机，可查看提示，并通过单击**详情**按钮查看虚拟机的状态及所属主机信息对比。

## 安装补丁文件 > 补丁文件介绍

# 补丁文件介绍

根据补丁文件中包含的修复问题组件的数量，补丁文件分为如下两种：

- **Hotfix Package**：文件名以 `hotfix-package` 开头的补丁文件，包含多个组件的修复。

  - 上传至升级中心后：**补丁文件**页签的**版本**一列将显示补丁版本及该文件包含的所有服务组件信息。
  - 安装完成后：**集群**页签的**集群版本**一列将显示补丁版本及已安装的所有服务组件信息。
- **Hotfix Patch**：文件名以 `hotfix-patch` 开头的补丁文件，包含单个组件的修复。

  - 上传至升级中心后：**补丁文件**页签的**版本**一列将显示该文件包含的单一服务组件信息。
  - 安装完成后：**集群**页签的**集群版本**一列将显示已安装的单一服务组件信息。

## 安装补丁文件 > 安装步骤 > 第 1 步：上传补丁文件

# 第 1 步：上传补丁文件

1. 进入 CloudTower 的升级中心主界面，单击界面右上角的**上传**按钮，选择**上传补丁文件**。
2. 在上传补丁文件对话框中选择**从本地选择**或**从 URL 上传**。

   - **从本地选择**：拖拽需要上传的补丁文件到虚线框中，或者单击**选择文件**并选择待上传的补丁文件。
   - **从 URL 上传**：输入需要上传的补丁文件的 URL，然后单击**解析 URL**。
   > **注意**：
   >
   > 补丁文件 URL 应以 `.run` 结尾，示例如下：
   >
   > `http://192.168.xx.xx/xx/hotfix-package-smtxos-6.2.0-P1-oe2003-2503260345-hygon.run`
3. 确认补丁文件预览信息后，单击**上传**。

   上传过程中可在任务中心查看上传进度，可单击任务右上角的 **…**，选择**停止**，将停止上传补丁文件至升级中心。

   > **注意**：
   >
   > 补丁文件上传过程中若因网络异常或刷新浏览器页面导致任务停滞，请在任务列表中手动停止该任务并重新上传。
4. 上传完成后可在任务中心选择对应任务，单击**详情**，查看上传详情。

补丁文件上传完毕后，若需要删除补丁文件，可在升级中心的**补丁文件**页签，单击对应补丁文件右侧的 **…**，选择**删除**。

## 安装补丁文件 > 安装步骤 > 第 2 步：检查集群补丁文件

# 第 2 步：检查集群补丁文件

升级中心会对已上传的补丁文件进行校验，如果补丁文件适用于当前集群，则会在**补丁文件**一栏中显示该文件。

1. 单击升级中心主界面的**集群**页签，在集群列表中选中待安装补丁文件的集群，弹出集群详情面板。
2. 在**补丁文件**一栏中查看是否存在已上传的补丁文件。如果未显示已上传的补丁文件，可能是以下原因导致，请获取适配的补丁文件后重新上传。

   - 集群类型、版本、CPU 架构、主机操作系统或虚拟化平台与补丁文件不匹配。
   - 集群中已安装的服务组件版本与当前补丁文件不兼容。
   - 集群中所有主机已安装的补丁文件均包含与已上传补丁文件相同的服务组件版本。

## 安装补丁文件 > 安装步骤 > 第 3 步：（可选）检查补丁安装环境

# 第 3 步：（可选）检查补丁安装环境

建议在安装补丁文件前对集群进行全面检查，提前识别潜在问题，以免在安装补丁文件的过程中发现问题而错过有限的窗口期，提高安装效率。

1. 在集群详情面板的**补丁**一栏中，找到目标版本的补丁文件，单击右侧的**检查环境**按钮。
2. 在检查补丁安装环境界面中确认集群当前版本和目标版本，确认无误后，单击**检查**。

如果当前环境通过检查，可为集群安装补丁文件。如果未通过检查，请根据提示信息解决相关问题后再进行安装。

# 第 4 步：安装补丁文件

1. 在补丁文件一栏中，单击目标版本补丁文件右侧的**安装**按钮。
2. 在补丁安装界面中确认集群当前版本、目标版本和安装范围。
3. （可选）设置任务通知。

   - 若升级中心已配置任务通知，任务通知默认启用，可设置任务通知的启用状态和定时通知的间隔时间。
   - 若升级中心未配置任务通知，任务通知默认禁用，可单击**配置**跳转到任务通知配置界面，参考[配置任务通知](/cluster-upgrade/1.2.1/cluster_upgrade_user_guide/cluster_upgrade_user_guide_39)章节设置。
4. 单击**安装**，进入自动安装流程。您可以在安装界面查看具体的安装进度。

如果在安装补丁文件的过程中失败了，可单击**查看日志**查看失败原因，或在集群详情面板中**版本变更记录**页签，通过单击补丁安装记录右侧的**详情**查看失败原因，解决问题后单击重试，将从失败处继续安装。查看最近一次安装失败记录的日志时，您可以在上方搜索框中进行搜索，也可以单击**下载日志**，将日志文件下载至本地进行问题分析。

## 升级中心运维管理 > 更新升级中心版本元数据文件

# 更新升级中心版本元数据文件

升级中心版本元数据文件包含了升级中心支持的所有升级文件的元数据信息、vGPU 驱动文件信息、补丁文件的版本信息、升级路径以及内核升级建议。

在安装 CloudTower 时会自动安装升级中心并携带一个配合该升级中心版本的元数据文件。当发布新的 SMTX 集群软件版本或补丁文件版本后，为使升级中心能够支持集群升级到最新版本，可通过更新升级中心版本元数据文件，扩展支持的升级路径、vGPU 驱动版本和补丁文件版本。

升级中心版本元数据文件可以随 CloudTower 或升级中心的升级而自动更新，也可手动上传新版本进行更新。

---

## 升级中心运维管理 > 更新升级中心版本元数据文件 > 自动更新

# 自动更新

当升级 CloudTower 或升级中心时，系统将自动检查新的升级中心所携带的元数据文件版本是否高于当前版本。

- 如果新的元数据文件版本不高于当前版本，则不允许删除。
- 如果新的元数据文件版本高于当前版本，则会使用新文件替换原有文件，并更新文件名称、版本号和上传日期。

---

## 升级中心运维管理 > 更新升级中心版本元数据文件 > 手动更新

# 手动更新

当不更新 CloudTower 和升级中心时，可通过手动更新的方式升级版本元数据文件。

**前提条件**

- 待上传的元数据文件版本高于当前的元数据文件版本。
- 升级中心未处于升级中且状态正常。

**操作步骤**

1. 登录 CloudTower，在升级中心主界面单击右上角的**设置**按钮，选择**版本元数据文件**。
2. 在版本元数据文件界面选择**从本地选择**或**从 URL 上传**。

   - **从本地选择**：拖拽需要上传的元数据文件到虚线框中，或者单击**选择文件**后选择目标文件。
   - **从 URL 上传**：输入需要上传的元数据文件的服务器 URL，再单击**解析 URL**。
   > **注意**：
   >
   > 元数据文件 URL 应以 `.json` 结尾，示例如下：
   >
   > `http://192.168.xx.xx/xx/v1.1.0-rc.11-2505090926.json`
3. 单击**更新**，上传新的元数据文件。

   > **注意**：
   >
   > 如果升级中心存在正在上传的文件，则不允许上传，需等待当前文件完成上传后再执行上传操作。

## 升级中心运维管理 > 配置任务通知

# 配置任务通知

升级中心支持配置任务通知，提供**邮件通知**和 **Webhook 通知**两种方式。启用后，任务通知将在创建升级集群、升级内核和安装补丁文件任务时默认启用，您可以手动调整启用状态或修改定时通知的时间间隔。系统将在任务的关键节点（如需手动处理、重要步骤、执行失败、执行成功等）通过指定渠道发送通知。

## 升级中心运维管理 > 配置任务通知 > 配置邮件通知

# 配置邮件通知

1. 在升级中心主界面单击右上角的**设置**按钮，选择**任务通知**。
2. 在任务通知配置界面，启用邮件通知。
3. 配置邮件通知的相关参数。

   | **参数** | **描述** |
   | --- | --- |
   | SMTP 服务器地址 | 填写 SMTP 服务器的地址，可填写 IP 地址或域名。  若 SMTP 服务器地址填写为域名，需要为使用该 SMTP 服务器的集群配置 DNS 服务器以将域名解析为 IP 地址。  **说明**：升级中心需单独配置 SMTP 服务器，不可复用 CloudTower 的 SMTP 服务器。 |
   | 安全模式 | 选择与 SMTP 服务器连接和进行消息通知的加密方式。包括**无**、**SSL** 和 **STARTTLS** 三种加密方式。 |
   | 端口 | 填写 SMTP 服务器的端口。  默认 SMTP 端口为 25（未加密）、465（SSL）和 587（STARTTLS）。 |
   | 用户名/密码 | 勾选“使用用户名和密码验证”后填写，用于 SMTP 服务器发送邮件的账号密码。  当安全模式为**无**时，该选项为选填项，否则为必填项。 |
   | 发件地址 | 指定一个电子邮件地址，作为任务通知的发件人。 |
   | 收件地址 | 指定一个或多个电子邮件地址，作为任务通知的收件人。如有多个收件人，请用英文半角逗号 `,` 隔开多个邮件地址。 |
   | 语言 | 邮件通知的语言，支持**简体中文**和**英语**。 |
   | 发送测试邮件 | 单击该按钮，系统将发送测试邮件至填写的邮箱地址。  **说明**：测试邮件发送成功后，才允许保存邮件通知的配置。 |
4. 单击**保存**完成配置

---

## 升级中心运维管理 > 配置任务通知 > 配置 Webhook 通知

# 配置 Webhook 通知

1. 在升级中心主界面单击右上角的**设置**按钮，选择**任务通知**。
2. 在任务通知配置界面，启用 Webhook 通知。
3. 配置 Webhook 通知的相关参数。

   | **参数** | **描述** |
   | --- | --- |
   | 接收端 | 任务通知的接收端。支持选择**企业微信**、**飞书**、**钉钉**和**其他**。 |
   | Webhook 地址 | 填写 Webhook 地址。  **说明**：当 Webhook 通知的接收端不同时，获取 Webhook 地址的方式也不同，因此具体获取方式需要参考对应接收端的官方文档。以下仅列出三种常用接收端的官方文档链接。  - [企业微信](https://developer.work.weixin.qq.com/document/path/91770) - [钉钉](https://open.dingtalk.com/document/robots/custom-robot-access) - [飞书](https://www.feishu.cn/hc/zh-CN/articles/360024984973-%E5%9C%A8%E7%BE%A4%E7%BB%84%E4%B8%AD%E4%BD%BF%E7%94%A8%E6%9C%BA%E5%99%A8%E4%BA%BA) |
   | 请求头 | 默认填写名称为 `Content-Type` 的请求头。您可以手动添加其他请求头，但请确保请求头名称不重复。  **说明**：当接收端为**企业微信**、**飞书**或**钉钉**时，`Content-Type` 请求头不可修改或移除。 |
   | 语言 | 发送通知内容的语言，支持**简体中文**和**英语**。 |
   | 消息模板 | 接收端用于展示通知内容的消息模板。    - 当接收端为**企业微信**、**飞书**或**钉钉**时，系统默认配置对应的消息模板，不可修改。 - 当接收端为**其他**时，需要手动配置消息模板。 |
   | 发送测试消息 | 单击该按钮，系统将发送测试消息至接收端。  **说明**：测试消息发送成功后，才允许保存 Webhook 通知的配置。 |
4. 单击**保存**完成配置。

## 升级中心运维管理 > 升级中心自升级

# 升级中心自升级

升级中心包含在 CloudTower 中，无需单独安装。安装或升级 CloudTower 至指定版本后，将自动安装升级中心。

升级中心不允许卸载，但支持对自身进行升级。升级中心支持随 CloudTower 升级和单独升级两种自升级方式。

---

## 升级中心运维管理 > 升级中心自升级 > 随 CloudTower 升级

# 随 CloudTower 升级

当升级中心随 CloudTower 版本发布时，可在升级集群前先升级 CloudTower，此时升级中心会随 CloudTower 一并升级，可参考目标 CloudTower 版本的《CloudTower 安装与升级指南》中的**升级 CloudTower** 小节进行升级。

---

## 升级中心运维管理 > 升级中心自升级 > 单独升级

# 单独升级

当升级中心单独发布时，可为升级中心进行单独升级。

**前提条件**

- 已获取 `tar.gz` 格式的升级文件。
- 升级中心内没有正在执行的集群预检查或升级任务。
- 升级中心未进行 ISO 的上传和分发。
- 升级中心未处于升级中状态且升级中心服务正常。

**操作步骤**

1. 在升级中心主界面单击右上角的**设置**按钮，选择**服务升级**。
2. 在服务升级界面选择**从本地选择**或**从 URL 上传**，并选择和当前 CloudTower 相同 CPU 架构的升级文件。

   - **从本地选择**：拖拽需要上传的升级文件到虚线框中，或者单击**选择文件**并选择目标文件。
   - **从 URL 上传**：输入需要上传的升级文件的服务器 URL，再单击**解析 URL**。
   > **注意**：
   >
   > 升级文件 URL 应以 `.tar.gz` 结尾，示例如下：
   >
   > `http://192.168.xx.xx/xx/upgrade-lcm-charts-smartx-upgrade-center-v1.1.1-rc.4-amd64-20250422113639.tar.gz`
3. 单击**升级**，在弹出的确认对话框中单击**升级**。升级过程中可在任务中心查看升级进度。
4. 升级完成后，请刷新浏览器页面。

## 附录 > 回滚升级工具版本

# 回滚升级工具版本

请参考如下步骤回滚升级工具版本：

1. 使用 root 账号登录到对应集群内任一节点。
2. 上传包含所要回滚到的升级工具版本的升级 ISO 文件或集群当前版本 ISO 文件到此节点的 `/home/smartx/` 路径下。
3. 以上传名称为 `test.iso` 的 ISO 文件为例，执行以下命令进行升级工具的回滚操作：

   `nohup cluster-upgrade downgrade_tool --iso_path /home/smartx/test.iso &`
4. 在脚本执行过程中，可以通过下述命令查看集群升级工具回滚操作的实时日志输出：

   `tail -f nohup.out`

   - 若升级工具回滚成功，则日志中将输出 `downgrade tool success.`。
   - 若升级工具回滚失败，请根据日志信息定位失败原因并解决问题，然后重新执行回滚操作。
