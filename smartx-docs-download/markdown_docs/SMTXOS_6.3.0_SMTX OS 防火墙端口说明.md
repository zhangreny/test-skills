# SMTXOS/6.3.0/SMTX OS 防火墙端口说明
## 关于本文档

# 关于本文档

本文档描述了与 SMTX OS 相关的防火墙端口规则，包含防火墙端口规则和主机安全配置，具体涵盖以下内容：

- 如何配置集群主机网络之间的防火墙端口、集群和外部客户端之间的防火墙端口，以确保集群能正常对外提供服务。
- 集成 VMware ESXi 平台时防火墙需开放的端口。
- 虚拟机跨集群迁移的端口开放规则。
- SMTX OS 内部服务占用的本地端口。

> **注意：**
>
> - 请勿修改 SMTX OS 集群主机防火墙的默认配置和默认状态(关闭)，否则会引起系统异常。
> - 若未作特殊说明，本文所提及端口均为 TCP 端口。

## 外部防火墙端口开放规则

# 外部防火墙端口开放规则

若集群物理主机的网络之间存在防火墙（如交换机上），或集群与外部客户端之间存在防火墙，为确保 SMTX OS 集群能够正常对外提供服务，防火墙均需开放如下端口。

以下开放规则仅讨论默认端口号。

## Web 控制台

| 端口 | 用途 | 说明 |
| --- | --- | --- |
| 443 | 网页浏览端口，用于 HTTPS 服务 | - |
| 80 | 网页浏览端口，用于 HTTP 服务 | - |

如需要从 SMTX Web 控制台操作界面打开虚拟机控制台，除开放以上端口外，还需要开放 VNC 相关端口。

## VNC

从 SMTX Web 控制台打开虚拟机控制台时，必须开放 VNC 代理服务端口：

| 端口 | 用途 | 说明 |
| --- | --- | --- |
| 8000 | SMTX OS VNC 代理服务 | - |

## SSH

| 端口 | 用途 | 说明 |
| --- | --- | --- |
| 22 | SSH 远程登录物理主机/SCVM/客户虚拟机 | SSH 协议的默认端口号 |

## IPMI

| 端口 | 用途 | 说明 |
| --- | --- | --- |
| 623（UDP） | 访问 IPMI 管理台 | 适用于戴尔、超微服务器。其他厂商产品请参见其详细要求。 |

## SNMP

| 端口 | 用途 | 说明 |
| --- | --- | --- |
| 1705 | 对接 SMTX OS 监控报警服务 octopus，用于从 octopus 获取监控数据。 | 若未添加 SNMP 传输，则无需开放此端口。 |

## iSCSI

在 SMTX OS (ELF) 集群，用户可创建 iSCSI 数据存储供外部客户端使用。如果外部 iSCSI 客户端通过防火墙访问存储，请确保以下端口处于开启状态：

| 端口 | 用途 | 说明 |
| --- | --- | --- |
| 3260、3261 | 用于通过 iSCSI 协议访问 SMTX OS (ELF) 集群的存储 | 标准 iSCSI 服务端口。 |

## 集成 VMware ESXi 平台的端口开放规则

# 集成 VMware ESXi 平台的端口开放规则

下文描述了 SMTX OS 集成 VMware ESXi 平台时，如何配置 ESXi 和 vCenter 的防火墙端口开放规则，以确保 SMTX OS 功能正常工作。

- 为确保 vCenter Server 正常运行，需要开放如下端口：

  | 端口 | 用途 | 说明 |
  | --- | --- | --- |
  | 443 | vCenter Server API 访问获取监控数据 | 请在 vCenter 防火墙上打开此端口。 |
- 为确保 ESXi 和 SCVM 之间的通信正常，需要开放如下端口：

  | 端口 | 用途 | 说明 |
  | --- | --- | --- |
  | 443 | 用于 SCVM 访问 ESXi SDK，获取监控数据 | 请在 ESXi 防火墙上打开此端口。 |
  | 22 | 用于 I/O 重路由脚本通信 | 请在 ESXi 防火墙上打开此端口。 |

## 虚拟机跨集群迁移的端口开放规则

# 虚拟机跨集群迁移的端口开放规则

当虚拟机进行跨集群迁移时，根据待迁移虚拟机所属的源端集群和目标端集群的类型，需要采用不同的端口开放规则，下面进行详细说明。

## 源端集群部署在 SMTX OS (ELF) 集群，目标端集群部署在 SMTX OS (ELF) 集群或 SMTX ELF 集群

支持使用 CloudTower 的跨集群迁移功能完成迁移任务，源端集群节点和目标端集群节点需开放的端口及端口要求如下表所示。

> **说明**：
>
> 源端集群部署在 SMTX OS (ELF) 集群，目标端集群部署在 SMTX ELF 集群时，仅支持使用热迁移方式。

| **迁移方式** | **端口要求** |
| --- | --- |
| **热迁移** | 源端集群节点与目标端集群节点迁移时使用的网络需双向开通以下 TCP 端口：   - 16509 - 16514 - 49152～49215   源端集群节点与目标端集群节点的管理网络需双向开通以下 TCP 端口：   - 80 - 443 |
| **冷迁移/分段迁移** | 源端集群节点和目标端集群节点需向 CloudTower 虚拟机开放以下 TCP 端口：   - 10100 - 10201 - 10206 - 10207 |

## 源端集群部署在 VMware ESXi 平台，目标端集群部署在 SMTX OS (ELF) 集群

支持使用迁移工具完成迁移任务，相应端口开放规则请参考《SMTX 迁移工具用户指南》。

## SMTX OS 内部服务占用端口

# SMTX OS 内部服务占用端口

SMTX OS 默认不启用本地防火墙。为了保证 SMTX OS 的内部服务或功能正常使用，用户必须要确保集群主机网络之间的防火墙开放以下使用的 TCP 或 UDP 端口。

## 随机端口

在实际场景中，NFS 服务和客户端本地也可能使用一些随机端口，具体说明如下：

- NFS 服务除了使用 `2049` 和 `111` 两个固定端口外，还会启动其他进程（如 status、nlockmgr 等），这些进程的端口由 RPC 服务进行随机分配，每次启动 NFS 服务时使用的端口均不相同。

  若需查看 NFS 当前使用的所有端口，或查看某一未知的端口对应的进程，可采用如下方法：

  - 查看 NFS 当前使用的随机端口：在主机执行 `rpcinfo -p` 命令查看。
  - 查看某一端口对应的服务进程：在主机执行 `netstat -nlp` 命令查看。
- 客户端本地也可能启用一些随机端口对接集群服务。若扫描到此类未知的本地端口，可在主机执行 `netstat -nlp` 命令，查看该端口对应的源端口或目的端口，并根据 SMTX OS 服务使用的固定端口，确定本地端口对应的集群服务。
