## 概述

# 概述

Everoute 支持虚拟专有云网络功能，可以在 SmartX 企业云或超融合环境中提供安全隔离的网络空间，并通过边缘网关和防火墙等虚拟化网络功能实现虚拟网络内部和外部的安全互联互通，使用户可以快速灵活地在多个数据中心部署统一的企业云网络。

- **自定义的虚拟网络环境**

  - 基于 GENEVE 协议的覆盖（Overlay ）虚拟化网络技术实现了虚拟专有云网络与底层物理网络（Underlay）之间的隔离和解耦合，允许用户在标准服务器和通用网络设备上创建 VPC，降低对网络硬件的依赖，网络构建、维护和变更的总拥有成本（TCO）更低。
  - 支持构建多个逻辑隔离、独立管理的 VPC，每个 VPC 内部自主地设置 IP 地址和各种虚拟化网络服务，而无需对物理设备进行任何变更。
- **丰富的网关功能**

  将 VPC 与传统的数据中心、公网等连通，以满足不同业务下的流量出入与转化需求：

  - 支持配置路由网关，使 VPC 内的虚拟机能够使用私有地址与 VPC 外部的三层网络进行通信。
  - 支持配置 NAT 网关，提供 DNAT 和 SNAT 两种能力，用户可以按需将 VPC 内的私有地址转换为外部子网的地址。
  - 支持配置 L2 网关，能够将 VPC 子网和某个 VLAN 虚拟机网络连接并形成一个二层子网，从而使 VPC 内部的虚拟机可以直接访问 VLAN 网络上的服务。
  - 支持配置 VPC 对等连接，使两个 VPC 之间的虚拟机使用私有地址进行通信。
- **虚拟机安全防护**

  - 支持跨越本地数据中心边界，可扩展至更大的地理区域上的多个数据中心。业务应用可以发布到不同数据中心，快速实现业务在多个数据中心的高可用、负载均衡或容灾恢复。
  - 支持为每个 VPC 设置安全组、安全策略及隔离策略，为 VPC 内的虚拟机资源提供安全防护。
- **云网统一管理**

  - 支持以图形化方式（GUI）对企业云或超融合上的虚拟机、分布式存储、虚拟化网络的统一管理、配置、监控和运维。
  - 通过开放的 API 与各种云平台无缝对接，为企业提供自动化和灵活的网络配置选项，以更好地支持敏态云应用。

## 产品架构

# 产品架构

Everoute 虚拟专有云网络采用分层架构的方式实现，包含管理平面、控制平面和数据平面，如下图所示。

- **管理平面**：安装和管理虚拟专有云网络功能。
- **控制平面**：配置 VPC、VPC 子网、安全组以及网关等网络产品。
- **数据平面**：计算并转发具体数据流，匹配安全规则。

## 产品架构 > 管理平面

# 管理平面

管理平面主要由 Everoute operator、CloudTower 和 API 组成，用于管理所有配置相关的 API 请求，以及所有 Everoute 虚拟专有云网络功能相关的运维操作。

- **CloudTower**：用于提供 UI 界面并与 Everoute 控制平面以及运维管理平面进行交互，负责将业务配置以及运维管理配置通过相关 RESTful API 进行下发。
- **Everoute operator**：用于 Everoute 服务的运维管理、许可管理，负责接收来自 CloudTower 的请求并且触发相应操作。

管理平面具有以下主要功能：

- 查看或更新 Everoute 软件许可。
- 管理用于部署、升级 Everoute 服务的安装包文件。
- 提供监控告警、认证、数据采集等基本能力，简化系统运维的复杂性。
- 管理所有增、删、改、查任务的生命周期，过滤所有用户配置，校验相关参数。
- 通过北向提供的 REST API，将用户的配置进行持久化存储和下发。

## 产品架构 > 控制平面

# 控制平面

控制平面实现了 VPC 逻辑数据的计算和管理，可将逻辑数据翻译为对应的数据流规则，下发至数据平面。控制平面由多个节点构成，主要包括控制器集群和 agent，具有高可用且易扩展的特点，能够支持灵活地按需扩容，使 VPC 网络的管理具有高稳定性。

![](https://cdn.smartx.com/internal-docs/assets/bc6d64f3/whitepaper_vpc_03.png)

## 控制器集群

通过中心化地部署控制器集群实现 HA 容灾能力，控制器集群主要提供以下功能：

- 通过 `VPC-proxy` 提供 RESTful API。
- 通过 `apiserver` 提供声明式 API。
- 通过 `VPC-controller` 和 `gateway-controller` 校验同步 API 的参数、处理异步 API 的任务，从而管理 VPC 和网关服务等功能的逻辑数据。
- 通过 `VPC-controller` 监听 CloudTower 中虚拟机标签的变更，管理 VPC 安全组的动态成员。
- 通过 `etcd` 实现资源变更事件的异步消息传递。

## agent

agent 包括 `VPC-agent` 和 `gateway-agent`：

- `VPC-agent` 部署在 SMTX OS（ELF）集群或 SMTX ELF 集群的计算节点上，主要提供以下功能：

  - 监听 VPC 中逻辑数据的变更，并将其翻译成数据平面规则，下发至 VPC 数据平面。
  - 监听数据平面虚拟机网络变更事件，并为其下发对应的网络规则。
  - 为 SMTX OS（ELF）集群和 SMTX ELF 集群中的虚拟机提供 DHCP 地址管理服务。
- `gateway-agent` 部署在边缘网关虚拟机上，负责监听网关中逻辑数据的变更，并将其翻译成数据平面规则，下发至网关数据平面。

## 产品架构 > 数据平面

# 数据平面

数据平面包含 **VPC 数据平面**和**网关数据平面**。

![](https://cdn.smartx.com/internal-docs/assets/bc6d64f3/whitepaper_vpc_04.png)

## VPC 数据平面

VPC 数据平面实现 VPC 的基础功能，包括 VPC 内互通、VPC 间隔离、VPC 子网、路由表、安全策略等。

![](https://cdn.smartx.com/internal-docs/assets/bc6d64f3/whitepaper_vpc_05.png)

VPC 数据平面包含 VPC bridge 和 VDS-TEP：

- **VPC bridge**（VPC 网桥）：基于 OVS bridge 创建的虚拟交换机，实现 VPC 网络流量转发功能，是 VPC 数据平面的核心部分。

  - **VRF**：在关联了虚拟专有云网络服务的集群中的每一个节点上，每个 VPC 对应一个 VRF（Virtual Routing Forwarding，虚拟路由转发表），从而实现相同节点上，不同 VPC 之间的隔离。
  - **安全策略**：VPC 中的安全策略对出、入虚拟机的报文分别进行出、入向安全策略的过滤，并且支持配置优先级，以实现隔离策略优先过滤。
  - **VDS**：虚拟分布式交换机对 VPC 子网内的报文进行二层转发，将出 VPC 子网的报文转发到 VDR（Virtual Distributed Router，虚拟分布式路由器）；如果报文的目标地址在其他节点，则通过 TEP 端口转发到目的地址所在节点的 VDS 或 VDR 上。
  - **VDR**：虚拟分布式路由器实现出 VPC 子网报文的三层路由功能，并将报文转发到对应的 VDS 或者边缘网关上；如果目标地址在其他节点，则通过 TEP 端口转发到目标地址所在节点的 VDS。
  - **TEP 端口**：GENEVE 类型的隧道端口，对跨节点通信的 GENEVE 隧道报文进行封装和解封装，不同 VPC 对应不同的 VNI（Virtual Network Identifier，虚拟网络标识符），从而实现跨节点通信时，不同 VPC 间以及 VPC 与 Underlay 网络之间的隔离。
- **VDS-TEP 接口**：

  实现 VPC TEP IP 间的三层互联能力的系统接口。包含 **Port-VPC 端口**，该端口是 internal 类型的 OVS 端口，虚拟专有云网络关联的集群中节点的 TEP IP 也配置在这个端口上。

## 网关数据平面

网关数据平面主要实现边缘网关的网关功能，如浮动 IP、L2 网关、路由网关和 NAT 网关，这些功能是通过 OVS 的流表功能实现的。

![](https://cdn.smartx.com/internal-docs/assets/bc6d64f3/whitepaper_vpc_06.png)

网关数据平面部署在虚拟机中，2 台边缘网关虚拟机组成 1 个边缘网关集群，边缘网关虚拟机之间通过心跳机制实现高可用的主备容灾，并通过 conntrackd 会话同步实现热备。每台边缘网关虚拟机包含 VPC gateway bridge、External bridge 和虚拟机网卡：

- **VPC gateway bridge**：VPC 网关虚拟交换机，是 VPC 网关的核心部分，包含如下几部分：

  - **VRF**：在边缘网关虚拟机中，每个 VPC 对应一个 VRF，从而实现相同边缘网关虚拟机上不同 VPC 之间的隔离：

    - **浮动 IP**：实现 VPC 内部的 IP 地址通过浮动 IP 与外部网络三层互通。
    - **L2 网关**：实现 VPC 子网与外部网络中同 CIDR 的 IP 二层互通。
    - **路由网关**：实现 VPC 子网与外部网络的三层互通。
    - **NAT 网关**：实现 VPC 网络访问外部网络的 SNAT，以及外部网络访问 VPC 网络的 DNAT。
  - **TEP 端口**：GENEVE 类型的 OVS 端口，实现与 VPC 网络通信时 GENEVE 隧道报文的封装、解封装。
  - **Peer 端口**：VPC gateway bridge 与 External bridge 连接的端口。
- **External bridge**：

  - **Peer 端口**：External bridge 与 VPC gateway bridge 连接的端口。
  - **External 端口**：External bridge 与 External 网卡连接的端口。
- **虚拟机网卡**：

  - **VPC 网卡**：连接 VPC TEP 网络的网卡，边缘网关虚拟机的 TEP IP 也配置在此网卡上。
  - **External 网卡**：连接外部网络的网卡。
  - **MGT 网卡**：连接管理网络的网卡。

## 技术实现 > 网关服务

# 网关服务

不同的网关提供了 VPC 与外部网络通信的不同访问方式，边缘网关上外部子网中配置的地址块决定了边缘网关可以访问的网络空间：

- 浮动 IP 地址块：虚拟机绑定浮动 IP 后，虚拟机可互通的外部网络为该浮动 IP 地址块可互通的外部网络。
- NAT 网关地址块：为 VPC 子网配置 NAT 网关后，VPC 子网中的虚拟机通过 SNAT 可访问的外部网络为 NAT 网关地址块可访问的外部网络，可访问 NAT 网关地址块的外部网络，可以通过 DNAT 访问 VPC 内服务。
- 路由网关地址块：为 VPC 子网配置路由网关后，可以互通的外部网络为路由网关地址块可互通的外部网络的子集，因为还依赖于外部网络中对 VPC 子网 CIDR 的路由配置。

## 技术实现 > 网关服务 > 浮动 IP

# 浮动 IP

浮动 IP 是从外部子网的浮动 IP 地址池中分配给虚拟机网卡的 IP 地址。您可以为虚拟机添加虚拟机网卡来动态关联浮动 IP，并通过浮动 IP 访问外部网络。

虚拟机通过关联的浮动 IP 访问外部网络时，边缘网关对报文的操作如下图所示：

![](https://cdn.smartx.com/internal-docs/assets/bc6d64f3/whitepaper_vpc_10.png)

- 对于来自 VPC 的报文，根据浮动 IP 的绑定关系，将来源 IP（VM\_IP）变更为浮动 IP（F\_IP），并转发到外部网络。
- 对于来自外部网络的报文，根据浮动 IP 的绑定关系，将目的 IP（F\_IP）变更为虚拟机 IP（VM\_IP），并转发到对应 VPC 的虚拟机。

> **说明**：
>
> 浮动 IP 的优先级高于默认路由（0.0.0.0），低于其他路由规则。即如果虚拟机关联了浮动 IP，且虚拟机所在 VPC 子网关联的路由表中也配置了默认路由（0.0.0.0），则优先使用浮动 IP。

## 技术实现 > 网关服务 > NAT 网关

# NAT 网关

NAT 网关为 VPC 内的虚拟机提供了共享 NAT IP 与外部网络通信的功能。NAT IP 是从外部子网的 NAT 网关地址块中分配的IP地址。

在实现 VPC 虚拟机和外部网络通信时，NAT 网关将虚拟机的私网地址转换为外部子网的地址。即当 VPC 子网关联了 NAT 网关后，该子网中的所有虚拟机都可以通过 NAT IP 与外部网络进行通信。

NAT 网关支持 SNAT 和 DNAT 两种配置方式：

- **SNAT**：VPC 内的虚拟机可通过 NAT 网关，主动访问外部网络。

  - 支持协议：ICMP、TCP、UDP
  - 端口范围：32768-65535，TCP 或 UDP 协议 SNAT 之后，报文的源端口范围。
- **DNAT**：外部网络可通过 NAT 网关，主动访问 VPC 内服务。

  - 支持协议：TCP、UDP
  - 端口范围：1-32768，外部网络可以主动访问的 TCP 或 UDP 协议的目的端口范围。

以 TCP 或 UDP 协议为例，NAT 网关对报文的操作如下图所示：

![](https://cdn.smartx.com/internal-docs/assets/bc6d64f3/whitepaper_vpc_11.png)

- **SNAT**：NAT 网关将首包的来源 IP（VM\_IP）转换为 NAT\_IP，来源 Port（VM\_PORT）转换为 NAT\_PORT，并记录该 TCP 或 UDP 流的映射关系。后续收到的该 TCP 或 UDP 流的报文时，根据记录的映射关系将 NAT\_IP 转换为 VM\_IP，NAT\_PORT 转换为 VM\_PORT。
- **DNAT**：NAT 网关根据配置的 DNAT 规则，将首包的目的 IP（NAT\_IP）转换为 VM\_IP，目的 Port（NAT\_PORT）转换为 VM\_PORT，并记录该 TCP 或 UDP 流的映射关系。后续收到该 TCP 或 UDP 流的报文时，根据记录的映射关系将 VM\_IP 转换为 NAT\_IP，VM\_PORT 转换为 NAT\_PORT。

# L2 网关

L2 网关打通了传统 VLAN 网络中的子网与 VPC 子网，提供了 VPC 子网与外部 VLAN 网络二层互通的功能，使 VPC 子网与外部 VLAN 网络逻辑上属于相同的子网。

如下图所示，L2 网关只是对报文进行转发，不会修改报文的任何内容。

# 对等连接

对等连接提供了 VPC 间三层互通的功能，无须依赖边缘网关集群，通过 VPC 关联集群节点上的分布式路由器实现数据转发。

## 技术实现 > 网关高可用

# 网关高可用

## 多网关集群

如图所示，VPC 服务支持创建多个边缘网关集群。每个边缘网关集群均独立部署，并具有独立的外部子网配置。

VPC 网关服务可按需部署在指定的边缘网关集群。当某个边缘网关集群发生异常时，可将流量切换至其他边缘网关集群，从而提高服务的可用性。

![](https://cdn.smartx.com/internal-docs/assets/bc6d64f3/whitepaper_vpc_20.png)

## 主备网关集群组

支持将两个独立的边缘网关集群关联，实现主备容灾。边缘网关集群组通过 TEP 网络与 VPC 内的虚拟机通信，通过外部子网组与 VPC 外的网络通信。

![](https://cdn.smartx.com/internal-docs/assets/bc6d64f3/whitepaper_vpc_21.png)

具体机制如下：

1. 优先使用主边缘网关集群提供服务，当主边缘网关集群异常时，流量自动切换至备边缘网关集群。
2. 主、备边缘网关集群之间通过 UDP 进行心跳检测。当连续 3 次未收到对端集群所有网关虚拟机的心跳时，认为对端集群异常，触发流量切换；当主边缘网关集群恢复后，流量将自动切换至主集群。

   为了防止颠簸的情况发生，切换频率限制如下：

   - 3 分钟内最多自动切换一次
   - 服务重启（包含升级）10 分钟内不会触发切换
   - 手动触发切换无频率限制
3. 边缘网关集群组触发流量切换后，VPC 侧的流量会自动切换至正常的边缘网关集群；外部网络访问 VPC 的流量，则需要依赖外部服务的高可用策略进行切换。

## 技术实现 > VPC

# VPC

VPC 是逻辑上隔离的网络空间，您可以创建自己的专属 VPC，为每个 VPC 添加 VPC 子网、路由表、安全组、MTU、边缘网关等网络配置，且不影响其他网络空间，比如其他 VPC、物理网络等。

同一 VPC 内的 VPC 子网间可以互通，与其他 VPC 或其他网络空间（如公网、传统 VLAN 网络或 IDC 等）通信则需要通过边缘网关。

系统会为每个 VPC 自动创建：

- **保留子网**：用于创建网关实例，CIDR 为启用虚拟专有云网络时配置的内部服务 IP 池，网关实例从这个子网分配 IP 获取 VPC 内的身份。不允许编辑和删除。
- **默认路由表**：VPC 子网默认关联的路由表；您可以编辑其路由规则，但不允许删除。
- **默认安全组**：包含 VPC 内所有虚拟机的安全组，不允许编辑和删除。

---

## 技术实现 > VPC 子网

# VPC 子网

VPC 子网是组成 VPC 的基础网络模块，VPC 子网是 VPC 中的一段 IP 地址空间（CIDR），使用 Overlay 网络技术定义，与底层物理网络解耦。同一 VPC 子网内可以通信，同一 VPC 内的 VPC 子网间默认可以通信。

VPC 子网提供 IPAM、DHCP 功能。

## IPAM

IPAM（IP Address Management）功能用于在 VPC 子网中创建或释放虚拟机网卡时，为虚拟机网卡分配或回收 IP 地址。

- 分配 IP 地址时：

  - 如果您为虚拟机网卡指定了未被分配的、非网络地址、非广播地址的 IP 地址（即可用的 IP 地址），则使用指定的 IP 地址，该 IP 地址可以不在 VPC 子网的分配 IP 池中。如果用户没有指定 IP 地址，则自动为虚拟机网卡分配一个属于分配 IP 池的 IP 地址。
  - 如果您为虚拟机网卡选择的 VPC 子网配置了分配 IP 池，则从分配 IP 池中随机分配一个可用的 IP 地址给虚拟机网卡，否则，从 CIDR 中为其随机分配一个可用的 IP 地址，并标记该 IP 已分配。
- 回收 IP 地址就是在释放虚拟机网卡时删除 IP 地址已分配的标记，使该 IP 地址可以重新分配。

## DHCP

DHCP（Dynamic Host Configuration Protocol）功能用于响应 VPC 子网中虚拟机网卡的 DHCP 请求，并返回对应的 DHCP 选项配置，例如创建包含虚拟机网卡的虚拟机后，当虚拟机通过该虚拟机网卡发送 DHCP 请求时，DHCP 服务会响应该请求，响应信息包含：

- 虚拟机网卡 IP 地址：创建虚拟机网卡时 IPAM 分配的 IP 地址，如果虚拟机网卡有多个 IP 地址，则响应为其分配的第一个 IP 地址。
- DHCP options:
  - Lease time: 无限租约
  - MTU：优先使用 VPC 指定的 MTU，未指定则返回 1400
  - Classless static routes: 0.0.0.0/0, $subnet\_gateway
  - DNS: 8.8.8.8
  - Server id: $subnet\_gateway
  - Router: $subnet\_gateway
  - Subnet mask: 根据 VPC 子网 CIDR 计算的子网掩码
  - Subnet broadcast: 根据 VPC 子网 CIDR 计算的广播地址

## 技术实现 > 路由表

# 路由表

路由表包含了一系列路由规则，路由规则决定了 VPC 子网的流量将被路由到哪里。路由表可以将目标地址的流量路由至指定的下一跳网关服务，实现 VPC 中网络通信路径的规划。

如出 VPC 子网流量的目的 IP 地址属于 VPC 内其他 VPC 子网 CIDR，则路由到对应 VPC 子网的默认网关；

如目的 IP 地址属于路由表中的规则，则路由到对应规则指定的下一跳，如果匹配路由表内的多条规则，则路由到匹配前缀最长的规则指定的下一跳；

如都没有匹配到，则将报文丢弃。

## 技术实现 > 安全策略

# 安全策略

安全策略是以安全组内的虚拟机作为目标对象，指定目标对象在出方向和入方向上的流量规则，可以针对不同协议、不同应用端口设置安全规则。

![](https://cdn.smartx.com/internal-docs/assets/bc6d64f3/whitepaper_vpc_09.png)

## 策略对象

安全策略的策略对象是一个或多个**安全组**，并需要分别为每个安全组配置是否组内互通。

![](https://cdn.smartx.com/internal-docs/assets/bc6d64f3/whitepaper_vpc_08.png)

安全组是一系列虚拟机的集合。如需对特定业务的虚拟机应用相同的安全策略，可以将其添加至同一个安全组中，并在各类所需策略中使用安全组作为策略对象。

安全组的虚拟机成员，支持静态和动态两种配置方式：

- **静态配置**：将指定虚拟机添加到安全组中。
- **动态配置**：将关联了指定标签的虚拟机添加到安全组中。

> **说明**：
>
> - 安全组实际是作用在虚拟机网卡上，如果一个虚拟机包含属于不同 VPC 的多张虚拟机网卡，则只有与安全组同属一个 VPC 的虚拟机网卡，才会被添加到安全组中。
> - 如为一个安全组选择了多个标签时，虚拟机的筛选逻辑为**逻辑与**，即筛选出的虚拟机为同时关联了所有标签的虚拟机。
> - 如同时通过静态配置和动态配置为安全策略添加了多个安全组，多个安全组之间为**逻辑或**的关系，例如，一个安全组通过静态配置添加，另一个安全组通过动态配置添加，那么包含在这两个安全组中的所有虚拟机都作为此安全策略的策略对象。

## 白名单规则

可以为 `CIDR 地址段`、`安全组`以及`由标签定义的策略对象`设置**入向白名单**和**出向白名单**。

系统只允许白名单内的网络流量流入或流出，而白名单之外的网络流量默认不允许通过。安全规则支持 TCP、UDP、ICMP 协议，其中 TCP 协议和 UDP 协议支持指定端口和端口范围。

## 策略逻辑

安全策略的报文过滤逻辑如下图所示：

![](https://cdn.smartx.com/internal-docs/assets/bc6d64f3/whitepaper_vpc_19.png)

首先判断是否匹配到组内互通规则：是，则允许通信；否，则判断报文方向。

- 入向报文，是否匹配到入向白名单，是，则允许通信， 否，则不允许通信。
- 出向报文，是否匹配到出向白名单，是，则允许通信， 否，则不允许通信。

安全组中的虚拟机一旦匹配到允许通信的安全策略，就允许通信，即虚拟机所匹配到的所有安全策略都不允许通信的时候，才不允许通信。

## 技术实现 > 隔离策略

# 隔离策略

隔离策略通过隔离虚拟机来隔离虚拟机的 VPC 流量。当虚拟机受到攻击或者用户想对虚拟机进行隔离调试时，可以对虚拟机设置隔离策略。支持对虚拟机进行**完全隔离**和允许特定策略的**诊断隔离**。

- **完全隔离**：应用此策略后，所有出入虚拟机的流量将全部禁止。虚拟机与外界完全隔离，并禁止连接网络。用户只能通过管理控制器登录虚拟机。
- **诊断隔离**：如果需要对隔离的虚拟机放行部分流量，可以对虚拟机设置诊断隔离策略，用户指定访问虚拟机（入流量）的白名单，以及虚拟机访问对象（出流量）的白名单，以帮助排查和分析此虚拟机的故障。

> **说明**：
>
> 隔离策略优先级**高于**安全策略，即当虚拟机被完全隔离时，任何出入虚拟机相同 VPC 网卡的流量都会被拒绝，即使是安全策略允许通信的流量也会被拒绝。

## 虚拟计算支持

# 虚拟计算支持

SMTX OS（ELF）、SMTX ELF 集群中的虚拟机通过虚拟机网卡来关联 VPC 网络，从而使用虚拟专有云网络。

- 虚拟专有云网络对 ELF 平台的支持，主要包括：

  - 创建、更新、删除虚拟机网卡的资源
  - 下发、更新、清理 VPC 网桥上虚拟机网卡的网络配置
- ELF 平台对 VPC 网络的支持，主要包括：

  - 为虚拟机创建、更新、删除虚拟机网卡
  - 将虚拟机网卡插入到 VPC 网桥上，或者从 VPC 网桥拔出

下面以 SMTX OS（ELF）集群为例，介绍 VPC 网络下创建、删除、迁移虚拟机的过程。

## 虚拟计算支持 > 创建虚拟机

# 创建虚拟机

![](https://cdn.smartx.com/internal-docs/assets/bc6d64f3/whitepaper_vpc_14.png)

如上图所示，创建虚拟机的主要流程为：

1. 在 CloudTower 界面创建虚拟机，并配置虚拟机网卡。
2. CloudTower 调用 ELF 创建虚拟机接口，ELF 创建任务。
3. ELF 异步处理创建虚拟机的任务：

   1. ELF 调用 VPC controller 创建虚拟机网卡。
   2. 在 VPC 的 OVS 网桥上创建虚拟机网卡的端口，包含 vNIC ID 等信息。
4. VPC agent 监听到 VPC 网桥上端口的新增事件，下发虚拟机网卡相关配置，包括：

   1. 根据端口的 vNIC ID，VPC ID 信息，增加虚拟机网卡的所属主机、TEP IP 等信息。
   2. 获取端口的 VPC、VPC 子网、安全策略、路由表等本地配置信息以及所属 VPC 内其他虚拟机网卡信息。
   3. 根据获取的信息，生成数据平面规则，下发到 VPC 的 OVS 网桥上。

## 虚拟计算支持 > 删除虚拟机

# 删除虚拟机

![](https://cdn.smartx.com/internal-docs/assets/bc6d64f3/whitepaper_vpc_15.png)

如上图所示，创建虚拟机的主要流程为：

1. 在 CloudTower 界面删除虚拟机。
2. CloudTower 调用 ELF 接口删除虚拟机，ELF 创建任务。
3. ELF 异步处理删除虚拟机的任务：

   1. ELF 清理虚拟机在主机上的资源，从 VPC 的 OVS 网桥上拔出虚拟机网卡；
   2. ELF 清理虚拟机元数据：

      1. 调用 VPC controller 的接口，根据 vNIC ID 删除虚拟机网卡；
      2. VPC controller 删除虚拟机网卡，并释放 IP 等相关资源。
4. VPC agent 监听到 VPC 网桥上端口的拔出事件，清理虚拟机网卡相关配置，包括：

   1. 清理和端口相关的本地配置，比如 VPC、VPC 子网、安全策略、路由表等；
   2. 根据端口的 vNIC ID，VPC ID 信息，删除虚拟机网卡的所属主机，TEP IP 等信息。

## 虚拟计算支持 > 迁移虚拟机

# 迁移虚拟机

![](https://cdn.smartx.com/internal-docs/assets/bc6d64f3/whitepaper_vpc_16.png)

如上图所示，迁移虚拟机的主要流程为：

1. 在 CloudTower 集群界面操作迁移虚拟机，包括热迁移和冷迁移。
2. CloudTower 调用 ELF 虚拟机迁移接口，ELF 创建任务。
3. ELF 异步处理迁移虚拟机的任务：

   1. 创建目标虚拟机：将虚拟机网卡插入到 VPC 网桥上。
   2. 清理源虚拟机：从 OVS 网桥上拔出虚拟机网卡。
4. 目标主机 VPC agent 监听到 VPC 网桥上端口的插入事件，下发虚拟机网卡相关配置，包括：

   1. 根据端口的 vNIC ID，VPC ID 信息，增加虚拟机网卡的目标主机、TEP IP 信息；
   2. 获取端口的 VPC、VPC 子网、安全策略、路由表等本地配置信息以及所属 VPC 内其他虚拟机网卡信息；
   3. 根据获取的信息，生成数据平面规则，下发到 VPC 的 OVS 网桥上。
5. 源主机 VPC agent 监听到 VPC 网桥上端口的拔出事件，清理虚拟机网卡相关配置，包括：

   1. 清理和端口相关的本地配置，比如 VPC、VPC 子网、安全策略、路由表等；
   2. 根据端口的 vNIC ID，VPC ID 信息，删除虚拟机网卡的源主机、TEP IP 等信息。

## 监控报警

# 监控报警

整体架构如图所示，主要包含：

- **CloudTower UI 报警模块**：通知用户报警信息。
- **可观测性平台 OVM**：根据配置的报警规则和关联的 `obs-agent` 上报的监控数据，生成报警事件。
- **obs-agent**：采集 metrics 或日志，并推送到可观测性平台 OVM。
- **VPC 相关服务**：包括 `VPC-controller`、`VPC-proxy`、`VPC-agent`、`gateway-agent` 等，提供 metrics 接口、采集 VPC 相关的指标信息。

![](https://cdn.smartx.com/internal-docs/assets/bc6d64f3/whitepaper_vpc_22.png)

具体采集指标如下：

- VPC 各类资源数量：包括 VPC、VPC 子网、安全策略、路由表、网关服务等。
- VPC 网关服务转发数据：包括出、入向报文数量，出、入向流量等。
