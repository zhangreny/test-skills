# sfs/1.3.0/SMTX 文件存储用户指南
## 关于本文档

# 关于本文档

本文档介绍了在 SMTX OS（ELF）集群中安装部署文件存储集群的要求和操作流程，以及在文件存储集群中创建和管理文件系统的操作。

SMTX 文件存储可以配套多个版本的 SMTX OS 和 CloudTower ，每个版本的功能和服务均有所不同。本文档中所有功能均基于 SMTX OS 6.3.0 和 CloudTower 4.8.0 进行描述，若您所购买的软件版本不包含文档中描述的部分功能和特性，请忽略并以实际使用的软件功能为准。

阅读本文档的对象需了解 SMTX OS 超融合软件、CloudTower 管理平台以及文件存储的相关技术背景知识，并具备丰富的数据中心操作经验。

## 概述

# 概述

SMTX 文件存储（SMTX File Storage，简称“SFS”）是由北京志凌海纳科技股份有限公司（以下简称 “SmartX”）自主研发的分布式文件存储产品，能够帮助用户高效地存储和管理非结构化数据，如媒体、图像、音频、传感器数据、文本数据等，响应不同使用场景下的工作负载要求。

---

## 概述 > 产品简介

# 产品介绍

SFS 服务部署在 SMTX OS（ELF）集群中，您可以在一个文件存储集群中创建多个文件系统供计算端使用。通过一套硬件同时满足计算和存储的需求，适用于小规模使用文件存储的场景，如 VDI、文件共享等。

SFS 服务以容器形式运行在文件控制器中，文件控制器部署在 SMTX OS（ELF）集群中，通过 CloudTower 统一管理，对外提供一套完整的文件存储服务。

![](https://cdn.smartx.com/internal-docs/assets/34a9b77a/sfs_user_guide_4.png)

**文件存储集群**

- 文件控制器：文件控制器组成了文件存储集群的控制平面，负责管理文件存储内部组件的健康状态和高可用策略，管理数据与元数据，并负责对外输出 NFS 协议和 HDFS 协议。至少 3 个文件控制器构成一个文件存储集群。
- 文件系统：文件系统是 SFS 提供的分布式文件共享单元，可在文件存储集群中创建，并在计算端挂载使用。
- 网络：包含文件管理网络、文件存储网络和文件接入网络。

**SMTX OS（ELF）集群**：为 SFS 服务提供底层持久化的存储资源。

**CloudTower 管理平台**：支持所有文件存储集群的部署、扩容、升级、删除等全生命周期管理操作。

## 概述 > 配置与管理规格

# 配置与管理规格

## 文件存储集群配置

| 条目 | 最大配置 |
| --- | --- |
| 文件控制器数量 | 16 |

## 文件存储集群管理规格

| 条目 | 最大规格 | 注意事项 |
| --- | --- | --- |
| 文件系统数量 | 16\*n/(n+1)，结果向下取整  n 为文件控制器数量 | 实际可创建的文件系统数量由文件系统的容量和当前空闲的磁盘槽位数量共同决定，详见表格下方说明。 |
| 单个文件系统容量 | 1 PiB |
| 单个文件系统的文件数量 | 10 亿 | - |
| 文件系统内单个文件的逻辑容量 | 4 TiB | - |
| 单个目录下一级子目录数量和文件数量之和 | 100 万 | - |

> **说明**：
>
> 由于文件系统需要占据文件控制器的磁盘槽位，且占用的槽位数与文件系统的容量有关，因此，实际可创建的文件系统数量由文件系统的容量和当前空闲的磁盘槽位数量共同决定。具体说明如下：
>
> - 每个文件控制器可提供 32 个磁盘槽位，总共可提供 32\*n 个槽位，其中 n 为文件控制器数量。
> - 单个文件系统所需的磁盘槽位数量 required\_slot 与文件系统容量 capacity、文件控制器数量 n 的关系为：
>   - 当 n ≤ capacity / 32 TiB 时，required\_slot = (⌈ capacity / 32 TiB ⌉ + 1) \* 2，其中 “⌈⌉” 代表向上取整。
>   - 当 n > capacity / 32 TiB 时，required\_slot = (n + 1) \* 2
> - 当文件存储集群的空闲磁盘槽位数量不足时，无法继续创建文件系统，请您另外部署一个文件存储集群，在其上创建所需要的文件系统。

## 部署 SFS > 部署前准备 > 规划网络

# 规划网络

以一个 SMTX OS（ELF）集群上部署一个文件存储集群，且文件存储网络复用 SMTX OS（ELF）集群存储网络所属虚拟分布式交换机、文件管理网络复用 SMTX OS（ELF）集群管理网络所属虚拟分布式交换机的场景为例，文件存储集群的网络拓扑如下图所示：

![](https://cdn.smartx.com/internal-docs/assets/34a9b77a/sfs-01.png)

- **文件管理网络**：文件存储集群通过此网络与 CloudTower 通信。
- **文件存储网络**：文件存储集群内部的存储网络，文件控制器通过此网络进行文件存储集群内部的数据交换。
- **文件接入网络**：文件存储集群通过此网络向外部的计算端提供文件存储服务。

## 网络要求

部署前请确认 SMTX OS（ELF）集群已存在满足下表要求的虚拟机网络。如不存在，请参考《SMTX OS 管理指南》创建虚拟机网络章节创建。

| 文件存储集群网络 | 网络带宽要求 | 虚拟机网络要求 | 虚拟机网络所属的虚拟分布式交换机要求 |
| --- | --- | --- | --- |
| 文件管理网络 | 1 Gbps 及以上。 | 与 SMTX OS（ELF）集群的管理网络连通。 | 关联了 SMTX OS（ELF）集群每一个主机上至少一个网口。 |
| 文件存储网络 | 10 Gbps 及以上。 | 未关联其他虚拟机，可被文件控制器独占。 |
| 文件接入网络 | 10 Gbps 及以上。 | 与计算端网络连通。 |

## 规划文件控制器 IP 地址

在部署前，根据所规划的文件控制器数量向网络管理员申请 IP 地址分配表。

| 网络类型 | IP 地址 | 子网掩码 | 网关 |
| --- | --- | --- | --- |
| 文件管理网络 | 文件控制器 1：xx.xx.xx.xx  文件控制器 2：xx.xx.xx.xx  ···  文件控制器 n：xx.xx.xx.xx | xx.xx.xx.xx | xx.xx.xx.xx |
| 文件存储网络 | 文件控制器 1：xx.xx.xx.xx  文件控制器 2：xx.xx.xx.xx  ···  文件控制器 n：xx.xx.xx.xx | xx.xx.xx.xx | - |
| 文件接入网络 | 文件控制器 1：xx.xx.xx.xx  文件控制器 2：xx.xx.xx.xx  ···  文件控制器 n：xx.xx.xx.xx | xx.xx.xx.xx | - |

如使用文件存储服务的计算端的 IP 地址和文件存储集群的文件接入 IP 地址不在同一网段，需要启用静态路由配置。启用后，一条静态路由对应一个计算端，每条静态路由需要配置如下信息：

- 计算端 IP 地址：xx.xx.xx.xx
- 计算端子网掩码：xx.xx.xx.xx
- 路由网关：xx.xx.xx.xx

> **注意**：
>
> - 为文件管理网络与文件接入网络选择的虚拟机网络不同时，文件管理网络、文件存储网络和文件接入网络的 IP 必须属于不同网段。
> - 为文件管理网络与文件接入网络选择的虚拟机网络相同时，文件存储网络的 IP 必须与文件管理网络及文件接入网络的 IP 属于不同网段，但文件管理网络 IP 与文件接入网络 IP 可以属于同一网段。
> - 启用静态路由时，路由网关必须与文件接入 IP 位于同一网段。
> - 所有 IP 地址不能与以下 CIDR 范围内的任何 IP 地址重叠：
>
>   - 169.254.0.0/16
>   - 240.0.0.0/4
>   - 224.0.0.0/4
>   - 198.18.0.0/15
>   - 127.0.0.0/8
>   - 0.0.0.0/8

## 规划集群文件管理虚拟 IP 地址

SFS 通过文件管理虚拟 IP 保证集群文件管理网络的高可用，当文件管理虚拟 IP 所在文件控制器故障后，会自动切换到其他健康的文件控制器继续对外提供服务。

因此，请参考如下要求，提前规划文件存储集群的文件管理虚拟 IP：

- 确保 CloudTower 可以通过文件管理虚拟 IP 访问文件存储集群。
- 文件管理虚拟 IP 与文件控制器的文件管理 IP 在同一个子网内。
- 文件管理虚拟 IP 不能与以下 CIDR 范围内的任何 IP 地址重叠：

  - 169.254.0.0/16
  - 240.0.0.0/4
  - 224.0.0.0/4
  - 198.18.0.0/15
  - 127.0.0.0/8
  - 0.0.0.0/8

## 部署 SFS > 部署前准备 > 规划文件控制器

# 规划文件控制器

请根据实际的业务需求规划文件控制器的配置，以及是否启用最大带宽限制。

## 文件控制器资源占用及配置

文件控制器的配置与所属集群中主机可提供的资源量有关，且根据高可用原则，文件控制器将分布在 SMTX OS（ELF）集群的不同主机上，所以规划文件控制器前请您提前了解 SMTX OS（ELF）集群的可用存储容量、主机数量、每个主机上可独占的 vCPU 数量和内存容量。

**vCPU**：

文件控制器默认开启 CPU 独占。

- x86\_64 架构下，每个文件控制器至少需要独占 4 vCPU。
- AArch64 架构下，每个文件控制器至少需要独占 8 vCPU。

**内存**：

- x86\_64 架构下，每个文件控制器至少需要 8 GiB 内存。
- AArch64 架构下，每个文件控制器至少需要 16 GiB 内存。

**存储空间**：

- SMTX OS（ELF）集群为混闪配置或多种类型 SSD 全闪配置时，请登录 CloudTower 进入目标 SMTX OS（ELF）集群的管理界面，单击**设置**，选择**常驻缓存**，启用**允许数据常驻缓存**，每个文件控制器将占用该 SMTX OS（ELF）集群中 240 GiB 空闲常驻缓存容量。
- SMTX OS（ELF）集群为单一类型 SSD 全闪配置时无法开启常驻缓存，每个文件控制器将占用该 SMTX OS（ELF）集群中 240 GiB 可用存储容量。

**数量**：

每个文件存储集群至少需要 3 个文件控制器，至多允许 16 个文件控制器，且文件控制器的数量应当不超过 SMTX OS（ELF）集群中能够为单个文件控制器提供足够 vCPU 和内存的主机数量，因此文件控制器数量限制如下：

- SMTX OS（ELF）集群中满足要求的主机数量 ≤ 16 时，3 ≤ 文件控制器的数量 ≤ SMTX OS（ELF）集群中满足要求的主机数量
- SMTX OS（ELF）集群中满足要求的主机数量 > 16 时，3 ≤ 文件控制器的数量 ≤ 16

更多的文件控制器可提供更大的带宽，承载更多的并发访问量，文件控制器的数量还可以在部署后按需增加，请根据实际的业务需求规划文件控制器的数量。

## 最大带宽

文件存储网络和文件接入网络可能会有较大的吞吐量，当两个网络与其他网络共用一个虚拟分布式交换机时，请您为文件控制器设置最大带宽，防止两个网络对应的虚拟机网络挤占所属虚拟分布式交换机上其他网络的带宽。

## 部署 SFS > 配置 NTP 服务器

# 配置 NTP 服务器

为避免 SFS 功能异常，请务必为 CloudTower 配置 NTP 服务器，SFS 将自动与该 NTP 服务器同步系统时间。

请您前往 CloudTower 的**系统配置** > **CloudTower 时间**页面，确认 CloudTower 是否已配置了 NTP 服务器。若未配置，请参考《CloudTower 使用指南》的**配置 NTP 服务器**章节进行配置。

## 部署 SFS > 上传安装包

# 上传安装包

**准备工作**

获取 SFS 安装包（tar.gz 文件）。

**操作步骤**

1. 登录 CloudTower，进入**文件存储**页面，单击**设置**，选择**安装包管理**。
2. 单击右上角的 **+ 上传安装包**，上传 `tar.gz` 格式的安装包。
3. 单击**上传**。

**相关操作**

如需删除安装包，请确认 CloudTower 中不存在使用该安装包部署文件存储集群。

## 部署 SFS > 部署文件存储集群

# 部署文件存储集群

文件存储集群由文件控制器组成，部署在 SMTX OS（ELF）集群中。一个 SMTX OS（ELF）集群中可以部署多个文件存储集群。

> **说明**：
>
> 为提升文件存储集群中文件控制器的性能、降低 I/O 延迟，建议您部署文件存储集群前，为文件存储集群所在 SMTX OS（ELF）集群开启 Boost 模式。

**操作步骤**

1. 在 CloudTower 的**文件存储**页面，在左侧导航栏中单击**文件存储集群**。
2. 在右侧**文件存储集群**页面单击 **+ 部署文件存储集群**。
3. 请在弹出的**部署文件存储集群**对话框中完成配置。

## 创建文件系统 > 填写 NFS 访问信息

# 填写 NFS 访问信息

## 导出路径

文件系统默认的导出路径为 `/文件系统名称`，您可以在输入框内填入除 "/" 外的自定义路径。

> **说明**：
>
> 自定义路径要求如下：
>
> - 仅支持大小写字母、数字、空格和如下特殊字符： `'` ，`` ` `` ，`~` ，`!` ，`@` ，`#` ，`$` ，`%` ，`^` ，`&` ，`(` ，`)` ，`-` ，`=` ，`+` ，`_` ，`,` ，`.` ，`;` ，`[` ，`]` ，`{` ，`}` 。Windows 客户端挂载文件系统时，导出路径不允许为 `/!`，请务必添加其他字符。
> - 除 "/" 外，支持的字符长度为 1-255。当 Windows 客户端挂载文件系统时，字符长度请控制为 1-127。

## 身份验证

| 参数 | | 描述 | 注意事项 |
| --- | --- | --- | --- |
| NFS 身份验证 | UNIX 系统验证 | 计算端提供本地的 UID 和 GID 来验证身份，存储端信任计算端提供的 UID 和 GID。 | 该方式下文件系统对不同用户默认提供不同的文件访问权限，数据安全性高。 |
| None | 计算端的所有用户都会被映射到存储端的匿名用户。 | 该方式下所有用户权限相同，一般用于只读场景。  选择该方式时，无论 NFS Squash 选择哪一种方式，计算端的所有用户都会被映射为存储端的匿名用户。  **注意：**   - Windows 客户端无法访问该方式的文件系统； - Linux 客户端挂载文件系统时需要配置 `sec=none`。 |
| NFS Squash | Root Squash | 当计算端以 root 用户访问时，映射为存储端的匿名用户。 | 该方式下关闭了 root 用户的超级访问权限，数据安全性高。 |
| No Root Squash | 当计算端以 root 用户访问时，映射为存储端的 root 用户。 | 该方式下 root 用户拥有超级访问权限，包括所有数据的读、写、删权限。请谨慎使用。 |
| All Squash | 所有计算端用户均被映射为存储端的匿名用户。 | 该方式下所有用户权限相同，一般用于只读场景。 |
| 匿名用户 UID | | 存储端的匿名用户 UID，默认值为 -2，可自行修改。 | UID 在计算端展示为非负整数，例如 -2 会在计算端展示为 4294967294。 |
| 匿名用户组 GID | | 存储端的匿名用户 GID，默认值为 -2，可自行修改。 | GID 在计算端展示为非负整数，例如 -2 会在计算端展示为 4294967294。 |

## 文件权限管理

| 参数 | | 描述 |
| --- | --- | --- |
| 默认访问权限 | 读写 | 计算端可以挂载文件系统并读写数据。 |
| 只读 | 计算端可以挂载文件系统并读取数据，但无法写入。 |
| 禁止访问 | 计算端无法挂载文件系统也无法读写。 |
| 特例规则 | | 按需勾选，可以通过计算端的 IP 地址或指定子网的 CIDR 添加与**默认访问权限**不同的特例规则。如需设置多个特例规则，以 `,` 分隔。 |

**使用示例**

如您希望仅 IP 地址为 10.2.234.197 和 192.168.31.69 的客户端具有读写权限，10.0.0.11/8 和 172.20.180.115/28 的客户端具有只读权限，其余客户端均无法访问该文件系统时，您可以在**默认访问权限**中选择**禁止访问**，勾选**特例规则**，并在**特例规则**下的**只读**后的输入框中填写 `10.0.0.11/25,172.20.180.115/28`，在**读写**后的输入框中填写 `10.2.234.197,192.168.31.69`。

## 管理文件系统 > 编辑文件系统 > 编辑 NFS 访问信息

# 编辑 NFS 访问信息

**前提条件**

- 当前文件存储集群状态为`正常`或`维护中`。
- 已有计算端挂载当前文件系统时，如需修改 NFS 身份验证，请在计算端中卸载该文件系统。
- 已有 Windows 客户端挂载当前文件系统时，如需修改导出路径，请在 Windows 客户端[卸载](/sfs/1.3.0/sfs_configuration/sfs_configuration_14)当前挂载的**所有**文件系统，并在命令提示符执行如下命令重启 Windows NFS 客户端。

  ```
  nfsadmin client stop
  nfsadmin client start
  ```

**操作步骤**

1. 在文件系统列表中单击目标文件系统右侧的 **...**，或进入文件系统详情页面，选择[编辑 NFS 访问](/sfs/1.3.0/sfs_user_guide/sfs_user_guide_10)。
2. 在弹出的**编辑 NFS 访问**对话框中修改，然后单击**保存**。
3. 使用新的 NFS 访问信息将文件系统重新挂载至原计算端。

## 管理文件存储集群 > 编辑文件存储集群配置

# 编辑文件存储集群配置

文件存储集群部署后，您可以按需编辑其配置，包括**名称**、**描述**、**文件接入网络的网络 IP**、**文件接入网络的静态路由**和 **Windows NFS 访问**。编辑文件控制器的数量和资源配置请参考管理文件控制器章节。

## 编辑基本信息

**前提条件**

当前文件存储集群状态为`正常`。

**操作步骤**

1. 文件存储集群列表中单击 **...**，或在文件存储集群的**设置**页面单击**集群信息**。
2. 单击**编辑基本信息**，在弹出的**编辑基本信息**对话框内，按需编辑文件存储集群的名称或描述。
3. 单击**保存**。

## 编辑文件接入网络 IP

**前提条件**

- 当前文件存储集群状态为`正常`。
- 参考[规划网络](/sfs/1.3.0/sfs_user_guide/sfs_user_guide_07#%E8%A7%84%E5%88%92%E6%96%87%E4%BB%B6%E6%8E%A7%E5%88%B6%E5%99%A8-ip-%E5%9C%B0%E5%9D%80)确认新的文件接入网络 IP 地址可用。

**风险提示**

编辑文件接入网络的 IP 会导致使用了该 IP 地址挂载的文件系统无法访问。如需编辑，请在编辑前先卸载使用了该 IP 地址挂载的文件系统，待编辑完成后再重新挂载。

**操作步骤**

1. 文件存储集群列表中单击 **...**，或在文件存储集群的**设置**页面单击**网络配置**。
2. 单击**编辑网络配置**，在弹出的**编辑网络配置**对话框中编辑文件接入网络的网络 IP。
3. 单击**保存**。

## 编辑文件接入网络静态路由

**前提条件**

当前文件存储集群状态为`正常`。

**风险提示**

编辑文件接入网络的静态路由时，可能发生短暂的网络中断，建议在业务空闲时或业务下线后进行操作。

**操作步骤**

1. 文件存储集群列表中单击 **...**，或在文件存储集群的**设置**页面单击**网络配置**。
2. 单击**编辑网络配置**，在弹出的**编辑网络配置**对话框中编辑文件接入网络的静态路由。
3. 单击**保存**。

## 编辑 Windows NFS 访问

为了保障 Windows 客户端 I/O 高可用，使用 Windows 客户端访问 SFS 前，请您务必启用 Windows NFS 访问。当文件存储集群不再需要 Windows NFS 访问时，请您禁用该项。

**前提条件**

当前文件存储集群状态为`正常`。

**操作步骤**

1. 文件存储集群列表中单击 **...**，或在文件存储集群的**设置**页面单击**访问配置**。
2. 单击**编辑访问配置**，在弹出的**编辑访问配置**对话框中启用或禁用 **Windows NFS 访问**。
3. 单击**保存**。

**前提条件**

- 确认 SMTX OS 和 CloudTower 版本符合要求，请参考[《SMTX 文件存储发布说明》](/sfs/1.3.0/release_notes/release-notes-03)；
- 已[上传安装包](/sfs/1.3.0/sfs_user_guide/sfs_user_guide_11)。
- 文件存储集群状态为`正常`。
- 文件存储集群里不存在状态为`创建中`、`更新中`的文件系统。
- 从 1.2.0 以下版本升级至本版本时，如文件控制器所属的 SMTX OS（ELF）集群已启用常驻缓存，请确保该 SMTX OS（ELF）集群中空闲常驻缓存容量不少于`当前文件存储集群内所有文件控制器中未开启常驻缓存的系统盘容量的总和 * 2`。

**注意事项**

升级文件存储集群过程中，请不要为 SMTX OS（ELF）集群中其他虚拟卷启用常驻缓存。

**操作步骤**

1. 在文件存储集群列表中，单击文件存储集群版本信息右侧的**可升级**，或在**文件存储集群**页面中单击**设置**页签，选择**集群升级**。
2. 在**集群升级**页面单击目标版本右侧的**升级**。
3. 在弹出的**升级文件存储集群**对话框中单击**升级**。

## 管理文件存储集群 > 删除文件存储集群

# 删除文件存储集群

**前提条件**

文件存储集群状态为`异常 部署失败`或`正常`。

**风险提示**

删除文件存储集群后，该文件存储集群中所有配置、文件系统均会被删除，绑定的许可码会在删除后作废。操作无法被撤回，请谨慎操作。

**操作步骤**

1. 在文件存储集群列表中，单击文件存储集群右侧的 **...** 后，选择**删除**。
2. 在弹出的**删除文件存储集群**对话框中进行二次确认，阅读风险提示后，输入文件存储集群的名称。
3. 单击**删除**。

## 管理文件控制器 > 增加文件控制器

# 增加文件控制器

增加文件控制器可以提供更大的带宽、更多的并发访问量，承载更多的文件系统，提高文件存储集群的性能。

**前提条件**

- 当前所属 SMTX OS（ELF）集群中未部署文件控制器的主机中，能够提供与现有文件控制器相同计算资源的主机数量不少于即将增加的文件控制器数量。
- 请参考[规划文件控制器](/sfs/1.3.0/sfs_user_guide/sfs_user_guide_08)确认当前所属 SMTX OS（ELF）集群中空闲的存储空间足够。
- 当前文件存储集群状态为`正常`。

**注意事项**

增加文件控制器过程中，请不要为 SMTX OS（ELF）集群中其他虚拟卷启用常驻缓存。

**操作步骤**

1. 文件存储集群列表中单击 **...**，或在文件存储集群的**设置**页面单击**文件控制器**。
2. 单击**编辑资源配置**，选择**增加文件控制器**。
3. 在弹出的**增加文件控制**对话框内设置文件控制器数量，单击**下一步**。
4. 为新增的文件控制器[配置网络](/sfs/1.3.0/sfs_user_guide/sfs_user_guide_15)。
5. 单击**保存**。

---

## 管理文件控制器 > 编辑计算资源

# 编辑计算资源

编辑文件控制器的 vCPU 数量和内存容量，可更灵活地调配集群内的资源。

**前提条件**

- 当前已部署文件控制器的所有主机中空闲的计算资源不少于即将为文件控制器增加的计算资源。
- 当前文件存储集群状态为`正常`。

**操作步骤**

1. 文件存储集群列表中单击 **...**，或在文件存储集群的**设置**页面单击**文件控制器**。
2. 单击**编辑资源配置**，选择**编辑计算资源**。
3. 在弹出的**编辑计算资源**对话框内，按需编辑文件控制器的 vCPU 数量和内存容量。
4. 单击**保存**。

## 管理文件控制器 > 编辑最大带宽

# 编辑最大带宽

文件存储网络和文件接入网络可能会有较大的吞吐量，当两个网络与其他网络共用一个虚拟分布式交换机时，请您为文件控制器设置最大带宽，防止两个网络对应的虚拟机网络挤占所属虚拟分布式交换机上其他网络的带宽。

**前提条件**

当前文件存储集群状态为`正常`。

**操作步骤**

1. 文件存储集群列表中单击 **...**，或在文件存储集群的**设置**页面单击**文件控制器**。
2. 单击**编辑资源配置**，选择**编辑最大带宽**。
3. 在弹出的**编辑最大带宽**对话框内，按需启用或禁用最大带宽。
4. 单击**保存**。

## 管理文件控制器 > 下线文件控制器

# 下线文件控制器

**前提条件**

- 当前文件存储集群状态为`正常`。
- 当前文件存储集群中所有文件控制器的服务状态为`正常`。
- 当前文件存储集群中所有文件系统的状态为`已上线`或`已下线`。

**风险提示**

下线文件控制器可能会导致使用该文件控制器的文件接入 IP 挂载的文件系统发生短暂的 I/O 暂停，请在业务空闲期操作。

**操作步骤**

1. 您可以在 CloudTower **文件存储**页面左侧的导航栏中选择**文件存储集群**，在文件存储集群列表中单击目标集群的名称，再单击**设置**页签，选择**文件控制器**。
2. 单击目标文件控制器右侧的 **...**，选择**下线**。
3. 在弹出的对话框中阅读风险提示后单击**下线**。

> **说明**：
>
> 文件控制器下线后系统将自动触发该文件控制器异常的告警。

---

## 管理文件控制器 > 上线文件控制器

# 上线文件控制器

**前提条件**

- 当前文件存储集群状态为`维护中`。
- 当前文件控制器的服务状态为`已下线`。
- 当前文件控制器所属主机未处于维护模式中。

**操作步骤**

1. 您可以在 CloudTower **文件存储**页面左侧的导航栏中选择**文件存储集群**，在文件存储集群列表中单击目标集群的名称，再单击**设置**页签，选择**文件控制器**。
2. 单击目标文件控制器右侧的 **...**，选择**上线**。

   上线成功后，文件控制器的状态将更新为`正常`。如果文件控制器状态一直保持`更新中`，请联系售后工程师协助解决。

## 管理权限

# 管理权限

SFS 通过 CloudTower 进行运维管理。CloudTower 提供了两种类型的角色：**默认角色**和**自定义角色**，并对不同角色赋予不同的操作权限。其中默认角色的权限不可更改，自定义角色可根据用户需要自行创建。

您可以在创建自定义角色时，按照下表所示可分配权限项的细分粒度，赋予角色不同的操作权限。详细步骤请参见《CloudTower 使用指南》的**创建自定义角色**小节。

| 类别 | 可分配权限项 | 对应的操作权限 |
| --- | --- | --- |
| 软件许可 | 许可管理 | 更新许可 |
| 文件存储安装包 | 安装包管理 | - 上传安装包 - 删除安装包 |
| 文件存储集群 | 文件存储集群管理 | - 部署文件存储集群 - 编辑文件存储集群基本信息 - 增加文件控制器 - 编辑计算资源 - 编辑访问管理 - 编辑最大带宽 - 编辑文件存储集群网络配置 - 升级文件存储集群 - 删除文件存储集群 - 下载文件存储集群的 Kubeconfig 文件 |
| 文件控制器 | 文件控制器管理 | - 上线文件控制器 - 下线文件控制器 |
| 文件系统 | 文件系统配置管理 | - 创建文件系统 - 编辑文件系统 - 删除文件系统 |
| 文件系统可访问性管理 | - 上线文件系统 - 下线文件系统 |

## 管理许可

# 管理许可

每一个部署在 SMTX OS（ELF）集群上的文件存储集群使用一个独立的软件许可。

---

## 管理许可 > 查看许可

# 查看许可

您可以在 CloudTower **文件存储**页面左侧的导航栏中选择**文件存储集群**，在文件存储集群页面中单击**设置**页签，在**常规**导航栏中选择**软件许可**，即可查看当前文件存储集群的许可。

## 许可信息

| **许可信息** | **说明** |
| --- | --- |
| 序列号 | 文件存储集群的序列号。 |
| 版本 | SFS 的软件版本类型。当前仅提供标准版。 |
| 许可类型 | 试用许可：默认试用时长为 90 天，最长不超过 180 天。过期后所有文件系统将被下线。 |
| 订阅许可：可按需选择订阅时长，订阅时长 1 年起。过期后无法创建新的文件系统。 |
| 永久许可：使用时长永久有效。 |
| 许可容量 | 文件存储集群中所有文件系统可使用的最大容量。 |
| 过期时间 | 当前许可的过期时间。 |

## 许可使用

将展示文件存储集群**已使用容量**、**可用容量**和**许可容量**。

已使用容量 = 当前文件存储集群中所有文件系统已使用的逻辑容量。

---

## 管理许可 > 更新许可

# 更新许可

在如下场景中，需更新许可码：

- 增加许可容量。
- 试用许可升级为订阅许可或永久许可。
- 延长订阅时长。

更新许可时，请将所有当前软件许可信息提供给 SmartX 客户经理，在完成商务流程后，客户经理将提供新的许可码。

**操作步骤**

1. 将许可码粘贴在软件许可中的新许可码处。
2. 单击输入框外部的空白区域，以载入新许可信息。
3. 确认新许可信息无误后，单击**更新**，完成许可更新。
