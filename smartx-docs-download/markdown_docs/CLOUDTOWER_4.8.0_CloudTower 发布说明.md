## 版本更新说明

# 版本更新说明

## 新增

- 支持管理 SMTX OS 6.3.0 集群。
- 支持管理 SMTX ELF 6.3.0 集群。
- 支持管理 Everoute 3.5.0。
- 支持管理 SMTX 备份与容灾 2.3.0。
- 支持管理 SMTX 文件存储 1.3.0。
- 支持管理巡检中心 1.2.1。
- 支持管理升级中心 1.2.1。
- 支持 CloudTower 高可用。
- 新增要求用户首次登录修改密码的密码安全策略。
- LUN 和 Namespace 的监控图表支持选择“存储卷”层级，以查看卷级别的 IOPS 或 I/O 带宽性能数据。
- 支持通过界面收集磁盘信息。

## 改进

- 支持通过界面配置 NTP 服务器时间时，提供更清晰的 NTP 服务器错误提示信息。
- 支持在虚拟机列表和详情面板查看所属快照计划名称。
- 调整默认存储策略的配置：
  - 支持在将 SMTX OS（ELF）集群关联至 CloudTower 时，设置集群默认存储策略。
  - 支持在将 SMTX ELF 集群关联至 CloudTower 时，设置关联存储的默认存储策略。
  - 创建 NFS Export、iSCSI Target、NVMe Subsystem 时，默认副本数调整为空。
- 优化备份与复制计划的命名方式，支持使用除分号、反斜杠、双引号以外的任意字符进行命名。

## 修复问题

- 修复了编辑 VPC 服务的 TEP IP 池时，若先删除某个 TEP IP 池再编辑剩余池，可能会显示错误的校验信息的问题。
- 修复了在启用 Boost 模式的集群中，当虚拟机包含 IDE 总线的 CD-ROM 时，错误提示不支持跨集群热迁移的问题。
- 修复了复制计划存在未完成的编辑任务时，再次发起编辑任务可能导致数据异常的问题。
- 修复了高负载环境下删除虚拟机快照可能失败的问题。
- 修复了虚拟机创建、开机及热迁移任务，可能因挂载了存储在 IP 白名单和 IQN 白名单均为`全部禁止`的 iSCSI Target 或 iSCSI LUN 上的 ISO，从而导致失败的问题。
- 修复了导出的集群和主机中文报表中，**系统预留 CPU** 列未展示具体数值的问题。
- 修复了通过 API 创建虚拟机时可能报错的问题。
- 修复了在 SMTX ELF 集群中从模板创建的 Windows 虚拟机，其客户机操作系统类型及 Windows 优化配置可能与模板信息不一致的问题。
- 修复了通过 CloudTower 界面编辑虚拟机放置组时，无法将描述信息清空的问题。
- 修复了因 PCI 设备查询频率过高及主机接口返回非必要字段，导致 CloudTower 请求流量偏高，从而引发 CPU 或内存占用升高及访问变慢的问题。
- 修复了通过脚本完成 CloudTower 发行版迁移或重建后，原有实例仍被误判为系统服务虚拟机，从而触发非预期告警的问题。
- 修复了异步复制计划在包含大量复制对象时，可能因请求处理超时而无法创建复制任务的问题。
- 修复了网络安全策略查询页面显示异常的问题。
- 修复了在编辑资源优化预置规则范围时，可能因部分集群仅关联可观测性而未部署高级监控，导致非预期无法选择“所有符合条件的集群”选项的问题。
- 修复了备份与容灾任务报表通知在选择特定类型 SMTP 服务器时，无法正常发送邮件的兼容性问题。
- 修复了在创建虚拟机、编辑磁盘或编辑网络设备对话框中，滚动 ISO 映像或虚拟机网络下拉列表时，部分选项无法显示的问题。
- 修复了在启用 Boost 模式的集群中，动态资源调度系统可能因对主机内存使用情况评估不准确，导致无法在集群内主机内存资源使用不均衡时生成有效迁移建议的问题。

## 版本配套说明

# 版本配套说明

## 与服务器 CPU 架构配套说明

当前版本支持运行在 Intel x86\_64、AMD x86\_64、Hygon x86\_64、兆芯 x86 和鲲鹏 AArch64 架构的虚拟化平台或物理机环境中。

## 与 SmartX 集群版本配套说明

本版本的 CloudTower 支持部署在以下版本的 SMTX OS 集群、SMTX ZBS 集群和 SMTX ELF 集群中，并对集群进行管理。

| **集群名称** | **支持的软件版本** |
| --- | --- |
| SMTX OS | 5.0.3 ~ 5.0.7-HP2 |
| 5.1.0 ~ 5.1.5-HP2 |
| 6.0.0 ~ 6.3.0 |
| SMTX ZBS | 5.2.0 ~ 5.7.0 |
| SMTX ELF | 6.0.0 ~ 6.3.0 |

## 与 CloudTower 代理版本配套说明

本版本的 CloudTower 支持部署 1.3.0 及以上版本的 CloudTower 代理。

## 与其他 SmartX 产品配套说明

下列 SmartX 产品和平台可通过 CloudTower 进行部署或管理，但每个产品只有部分版本满足两个产品的兼容性要求，请查阅对应产品的发布说明文档获取两个产品的版本配套关系。

- Everoute
- SMTX Kubernetes 服务
- SMTX 备份与容灾
- SMTX 文件存储
- 可观测性平台
- 巡检中心
- 升级中心

## 安装部署注意事项

# 安装部署注意事项

在部分 SMTX OS 集群、SMTX ZBS 集群和 SMTX ELF 集群上通过 CloudTower Installer 安装部署 CloudTower 前，需先升级集群的 CloudTower Installer 相关 RPM 文件。集群的版本和服务器的 CPU 架构不同，对应升级的 CloudTower Installer 相关 RPM 文件也不同。
