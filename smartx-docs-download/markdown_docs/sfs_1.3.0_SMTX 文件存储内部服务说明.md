# sfs/1.3.0/SMTX 文件存储内部服务说明

# 关于本文档

## 服务功能描述

# 服务功能描述

内部服务是指保证 SFS 的存储和运维功能正常运行所需要的服务，包含了 SmartX 独立开发的服务和所依赖的第三方服务。

根据运行的服务类型，文件控制器分为两种节点：主节点、存储节点。

- **存储节点**运行 SFS 的存储服务。
- **主节点**除了存储服务外，还运行着文件存储集群的管控服务。

下表介绍了 SFS 内部服务的功能以及所部署的文件控制器。

## 文件存储相关服务

| 服务名 | 文件控制器节点 | 功能 |
| --- | --- | --- |
| sfs-agent | 主节点  存储节点 | 管理和监控文件控制器。 |
| sfs-meta | 主节点  存储节点 | 为文件系统提供元数据服务。 |
| sfs-data | 主节点  存储节点 | 为文件系统提供数据服务。 |
| sfs-ganesha | 主节点 存储节点 | 提供 NFS 接入服务。 |
| sfs-manager | 主节点 | 管理和控制文件存储集群。 |
| sfs-cloud-provider | 主节点 | 负责接入底层块存储。 |

## 运维相关服务

| 服务名 | 文件控制器节点 | 功能 |
| --- | --- | --- |
| ntpm | 主节点  存储节点 | 提供时间同步服务。 |
| obs-agent | 主节点  存储节点 | 提供可观测性服务。 |
| etcd-operator | 主节点 | 监控文件存储集群内部服务的健康状态。 |

## 第三方服务

| 服务名 | 文件控制器节点 | 功能 |
| --- | --- | --- |
| traefik | 主节点  存储节点 | 为文件存储集群内部服务提供 API Server 代理。 |
| kube-etcd | 主节点 | 提供分布式数据库服务。 |
| kube-apiserver | 主节点 | Kubernetes 的声明式 API Server。 |
| cadvisor | 主节点  存储节点 | 统计容器运行信息。 |
| node-exporter | 主节点  存储节点 | 采集文件控制器信息。 |
| sshd | 主节点  存储节点 | 提供远程登录服务。 |
| rpcbind | 主节点  存储节点 | 提供 RPC 动态端口分配服务。 |
| ebpf-exporter | 主节点  存储节点 | 采集 Linux 内核子系统信息。 |

## 服务失效影响

# 服务失效影响

## 文件存储相关服务

| 服务名 | 在单个文件控制器失效影响 | 在所有文件控制器失效影响 |
| --- | --- | --- |
| sfs-agent | docker 尝试自动拉起，若失败，则该文件控制器将在 20s 后被重启，I/O 短暂中断。 | docker 尝试自动拉起，若失败，则所有文件控制器将被重启，I/O 短暂中断。 |
| sfs-meta | docker 尝试自动拉起，若失败  - shard follower：无影响 - shard leader：如果文件系统的 meta shard 的leader 挂载在此文件控制器上，则该文件系统的元数据无法访问 | 所有文件系统无法访问。 |
| sfs-data | docker 尝试自动拉起，若失败  - shard follower：无影响 - shard leader：如果文件系统的某个或者某些 data shard 的 leader 挂载在此文件控制器上，则该文件系统分布在这些 data shard 上的数据无法访问 | 所有文件系统的数据无法访问。 |
| sfs-ganesha | docker 尝试自动拉起，若失败，则文件接入 IP 迁移到其他文件控制器，通过该文件接入 IP 挂载的文件系统的 I/O 短暂中断。 | 所有文件系统无法访问。 |
| sfs-manager | docker 尝试自动拉起，若失败  - follower：无影响 - leader：触发 leader 切换，短暂地无法通过 CloudTower 操作文件系统 | 文件存储集群异常，无法进行文件系统生命周期管理。 |
| sfs-cloud-provider | docker 尝试自动拉起，若失败  - follower：无影响 - leader：触发 leader 切换，短暂地无法通过 CloudTower 操作文件系统 | 文件存储集群异常，无法进行文件系统生命周期管理。 |

## 运维相关服务

| 服务名 | 在单个文件控制器失效影响 | 在所有文件控制器失效影响 |
| --- | --- | --- |
| ntpm | docker 尝试自动拉起，若失败，则该文件控制器的 NTP 配置无法被更新，可能导致该文件控制器时间不同步。 | 所有文件控制器的 NTP 配置无法被更新，可能导致所有文件控制器时间不同步。 |
| obs-agent | docker 尝试自动拉起，若失败，则该文件控制器的信息无法上报至可观测性平台，导致监控数据和告警丢失。 | 所有文件控制器的信息无法上报至可观测性平台，导致监控数据和告警丢失。 |
| etcd-operator | docker 尝试自动拉起，若失败  - kube-etcd 正常：无影响 - kube-etcd 异常：无法重启该文件控制器上的 kube-etcd 服务 | docker 尝试自动拉起，若失败  - kube-etcd 正常：无影响 - kube-etcd 异常：无法重启所有文件控制器上的 kube-etcd 服务 |

## 第三方服务

| 服务名 | 在单个文件控制器失效影响 | 在所有文件控制器失效影响 |
| --- | --- | --- |
| traefik | docker 尝试自动拉起，若失败，则等效于该文件控制器故障。 | 等效于所有文件控制器故障。 |
| kube-etcd | docker 尝试自动拉起，若失败  - follower：无影响 - leader：触发 leader 切换，CloudTower 短暂地无法访问文件存储集群 | - 管控面无法提供服务。 - 文件系统被下线，计算端无法访问文件系统。 - 文件接入 IP 将被从文件接入网卡上删除。 |
| kube-apiserver | docker 尝试自动拉起，若失败，则该文件控制器无法提供 API server 的服务，但计算端可以通过 traefik 选择其他健康的文件控制器提供服务。 | - 管控面无法提供服务。 - 文件系统被下线，计算端无法访问文件系统。 - 文件接入 IP 将被从文件接入网卡上删除。 |
| node-exporter | docker 尝试自动拉起，若失败，则该文件控制器的系统信息无法上报至可观测性平台，导致监控数据丢失。 | 所有文件控制器的系统信息无法上报至可观测性平台，导致监控数据丢失。 |
| cadvisor | docker 尝试自动拉起，若失败，则该文件控制器的容器信息无法上报至可观测性平台，导致监控数据丢失。 | 所有文件控制器的容器信息无法上报至可观测性平台，导致监控数据丢失。 |
| sshd | 无法通过 ssh 远程登录该文件控制器。 | 无法通过 ssh 远程登录所有文件控制器。 |
| rpcbind | 计算端无法使用 NFSv3 协议通过该文件控制器的文件接入 IP 挂载，但已通过 NFSv3 协议挂载的计算端的 I/O 不受影响。 | 计算端无法使用 NFSv3 协议通过所有文件控制器的文件接入 IP 挂载，但已通过 NFSv3 协议挂载的计算端的 I/O 不受影响。 |
| ebpf-exporter | docker 尝试自动拉起，若失败，则该文件控制器的 Linux 内核子系统信息无法上报至可观测性平台，导致相关监控数据丢失。 | 所有文件控制器的 Linux 内核子系统信息无法上报至可观测性平台，导致相关监控数据丢失。 |
