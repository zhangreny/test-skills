## 版本更新说明

# 版本更新说明

## 新增

### 块存储

- 支持在 Hygon x86\_64 和鲲鹏 AArch64 架构的服务器部署基于 TencentOS 操作系统的 SMTX ZBS 块存储集群。
- 支持使用第三方密钥管理服务对存储卷的数据进行静态加密。
- 支持单节点配置多个物理盘池，每个物理盘池对应一个 Chunk 实例，以提升单节点的存储性能上限。
- 新增加大容量规格，单节点数据分区总容量上限为 512TiB。
- 支持存储卷回收站。
- 快照计划支持 NVMe Namespace 和一致性组作为快照对象。
- 支持使用业务主机或业务主机组对存储卷进行访问控制。
- 支持将 NVMe Subsystem 的 NQN 白名单设置为自适应，自动保持为所包含 Namespace 的 NQN 白名单的并集。
- 支持展示存储超分比。
- 支持使用命令行将厚置备类型的卷转换为精简置备。

### 文件存储

- 支持 HDFS 协议，Hadoop 生态系统中的组件可以访问和使用文件系统。
- 支持为 NFS 协议的文件系统自定义导出路径。
- 支持为文件控制器增加计算资源。
- 支持文件管理网络和文件接入网络复用同一个虚拟机网络且 IP 地址位于同一网段。
- 支持展示文件系统的客户端活跃连接。
- 支持展示文件存储集群和文件系统的性能监控图表。
- 支持通过 CloudTower 配置文件存储集群是否允许 Windows 客户端通过 NFSv3 协议访问。
- 支持自动重启系统盘为只读状态的文件控制器。
- 支持文件预读功能。
- 支持在文件系统使用率过高时触发告警。

### 运维管理

- 分层模式下，所有节点的物理盘为同一类型 SSD（NVMe SSD 或 SATA/SAS SSD）时，可通过修改系统盘用途决定物理盘分区方式。

  - 系统盘用途为“含元数据分区的数据盘”：每块物理盘均包含缓存分区和数据分区，适用于物理盘是同一类型 SSD 且属性完全相同的场景。
  - 系统盘用途为“含元数据分区的缓存盘”：缓存盘和数据盘独立部署，适用于物理盘是同一类型 SSD 但属性不同的场景。
- 支持查看物理盘和网口的 NUMA 节点 ID。
- 支持通过 CloudTower 移除主机和转换主机角色。
- 支持通过命令行对集群进行上下线。
- L2 ping 支持维护模式。
- 支持将操作系统安装在独立物理盘中，通过硬件 RAID 1 保证系统盘冗余。
- 支持在 I/O 阻塞时主动对故障盘进行下线处理，防止单盘故障影响节点整体 I/O。
- 支持在集群部署和添加节点时修改操作系统的默认密码。
- 支持通过 CloudTower 查看集群和主机的当前系统时间。

### 网络

- 存储网络启用 RDMA 时，支持在所属的虚拟分布式交换机中选择跨网卡的多个网口进行绑定。
- 存储网络启用 RDMA 或接入网络启用 NVMe over RDMA 时，支持在重启节点 Chunk 服务后或对相关物理网口执行 ifdown 操作后自动进行流控配置。
- 支持网口连通性检测，并对异常网口提供告警。
- 支持对主机特定端口自定义访问控制，并配置允许访问此端口的 IP 白名单。
- 支持在部署集群前通过 TUI（终端用户界面）为节点配置管理网络、DNS、主机名等。
- 在虚拟分布式交换机的网口绑定模式为 Active-Backup 时，支持手动指定每个主机上的活动网口。
- 支持在 CloudTower 中修改未启用 RDMA 的存储网络的 VLAN ID，以及该存储网络所属虚拟分布式交换机所关联的物理网口。

### 内核

- 内核升级至 5.10.0-247.0.0。
- 支持 SSSRAID 控制卡。
- 支持网迅万兆网卡（txgbe 系列）。
- 升级 ps3stor 驱动至 2.6.0.23。
- 支持 Intel E610 网卡。
- 升级 mpt3sas 驱动至 55.00.00.00。
- 升级 i40e 驱动至 2.20.12。
- 升级 megaraid 驱动至 07.735.03.00-2。
- 支持海思 SP680 系列网卡。

## 改进

### 块存储

- 进行一系列改进以提升存储性能。

  - 增加节点间 Data channel 数量，在存储网络关联的 VDS 使用 balance-tcp 模式进行网口绑定时，可以充分利用存储网络关联的多个网口。
  - 对存储网络和接入网络对应的网口配置中断。
  - 优化 LSM 和 Access 的 I/O 处理逻辑。
  - 利用 Intel DSA 进行硬件加速。
  - 默认启用 IO Uring，降低 LSM I/O 资源消耗。
  - 在 AArch64 架构的集群中，将 Chunk 服务绑定至缓存盘存储控制器所连接的 NUMA 节点 CPU 核心。
  - 优化存储卷在创建快照后瞬时 I/O 的下降幅度。
- 优化数据迁移和恢复策略。

  - 当集群空间负载较高时，移除节点操作将自动停止。
  - 容量层和性能层分别生成迁移任务，提升待迁移数据量较大时的整体效率。
  - 移除节点期间若任一 Chunk 实例进入超高负载则优先进行容量均衡，以保证移除节点尽快完成。
  - 优化数据迁移和恢复时源节点的选择策略，以提升稀疏卷的迁移或恢复速率。
  - 卸载物理盘时，优先从其他包含数据副本的健康盘读取数据来加速数据迁移，从而缩短操作时间。
  - 允许通过配置文件调整每个节点的并行恢复或迁移任务的数量。
  - 细化数据下沉策略，在性能层负载较高时会提前下降相对冷的数据保证频繁访问的数据更有可能停留在性能层。
- 优化在性能层存在大量数据的虚拟卷执行快照或克隆操作时的限流机制，使其在性能层空间快速增长的情况下能够平滑进入限流并逐步恢复，避免出现突发的 I/O 陡降现象。
- 提升单节点的缓存分区总容量上限，单物理盘池缓存分区总容量上限为 51TiB。
- 提升单节点可为常驻缓存卷预留的缓存比例上限至 75%。
- 允许常驻缓存卷使用精简置备，采用精简置备时将不占用容量层空间。
- 写缓存容量较高时不允许关闭卷的常驻缓存。
- 接入网络启用 NVMe-oF 时，集群升级后新添加的节点与已有节点的 NVMe-oF 配置将保持一致。
- 调整集群试用许可和订阅许可的过期策略。许可过期后，已创建的资源仍可正常使用，I/O 不会中断，但无法新建存储卷或对存储卷进行扩容。
- 创建 iSCSI LUN 和 NVMe Namespace 时自动去除名称首尾空格，避免前端展示的名称和实际名称不一致。
- 部分存储资源的名称和描述字段新增支持中文，包括 iSCSI Target、NVMe Subsystem 的描述，LUN、Namespace 及其快照的名称，一致性组和一致性快照组的名称，业务主机和业务主机组的名称和描述。
- 双活集群的最低配置调整为每个可用域至少 2 个节点。
- 存储维护模式超时时间延长至 7 天。
- 允许在满足集群空间分配规则的情况下通过命令行人工调整指定数据块的位置。

### 文件存储

- 将新创建的文件系统的元数据盘冗余策略设置为 3 副本，保障了元数据性能并提升了元数据安全性。
- 减少了文件控制器上线、下线和 HA 文件控制器时的 I/O 暂停时间。

- 优化了文化存储集群升级流程，升级过程无需重启文件控制器，减少了升级期间的 I/O 暂停时间。
- 优化了将文件控制器的文件管理 IP 添加至 CloudTower 白名单的机制。
- 优化了存储引擎的块分配机制，提升块回收效率。

### 运维管理

- 调整不健康盘和亚健康盘的判断条件。
- 支持在安全的情况下自动隔离高延迟的 SSD（之前版本行为为告警）。
- 修改 UDEV 规则以确保物理盘 ID 链接的唯一性。
- 标准版不支持全闪配置的节点。
- 主机维护模式增加检测机制，运行文件存储控制器的节点需先下线文件存储控制器才可进入维护模式。
- 增加和调整部分报警规则。

  - 增加主机系统分区只读时的告警项。
  - 增加新的内存错误报警。
  - 增加双活集群仲裁节点管理 IP 连通性的检测和报警。
  - 调整 timemachine 相关告警规则，删除不适用于快照计划的规则并调整部分规则的描述。
- 增加部分监控指标。

  - 增加双活集群仲裁节点网络流量相关监控数据的采集。
  - 支持查看部署在块存储集群中的可观测性服务的 CPU 和内存使用情况。
- 减少主机上运行的服务在管理网络上占用的非必要端口。
- 统一主机上的容器引擎为 containerd。
- 支持在集群部署时检测 NTP 服务器的连通性。
- 支持在集群部署或添加节点时不挂载指定的物理盘。
- 优化网口状态检测周期，CloudTower 将在 15s 内展示网口的状态变化。
- 优化故障节点替换流程。

  - 集群可通过先移除节点，然后添加管理 IP、存储 IP 和接入 IP 完全相同的节点，完成数据同步和节点重建。
  - 支持在多个主节点异常的情况下进行角色转换和节点替换。
- 优化日志采集功能，支持采集硬件固件信息、安全审计信息及内存信息。
- 增加升级预检查项，阻止许可过期的集群进行升级。

## 修复问题

### 块存储

- 修复了存储网络启用 RDMA 时，网卡异常后存储服务无法正常启动的问题。
- 修复了集群使用 ZooKeeper v3.9.9-27 及更低版本时，可能出现 ZooKeeper 服务异常或主节点无法连接到 Meta Leader 的问题。
- 修复了 iSCSI 链路可能反复断开重连的问题。
- 修复了 `data_channel_bench.py` 脚本偶发部分测试无结果的问题。
- 通过动态调整全闪配置的集群中物理盘的 I/O 超时时间，降低物理盘偶然的慢速响应对业务 I/O 的影响。
- 修复了跨 iSCSI Target 执行 XCOPY 命令时，连接断开可能导致 Chunk 服务崩溃的问题。
- 修复了快照计划中包含大容量卷时，可能因为 timemachine 内存不足而无法正常工作的问题。
- 修复了 ESXi 将 iSCSI LUN 格式化为 VMFS 时，运行在其上的虚拟机使用 fstrim/blkdiscard 回收空间时可能会报错的问题。

### 文件存储

- 通过改进文件管理虚拟 IP 的高可用机制，修复了文件管理网络异常时文件管理虚拟 IP 未自动迁移的问题。
- 修复了 NTIRPC 组件泄露 socket 文件描述符导致 NFS 请求未被处理的问题。
- 修复了特定条件下 NFS 后端返回非预期的不可重试错误码的问题。
- 修复了 sfs-meta、sfs-data、sfs-ganesha 在 coredump 过程中被 sfs-agent 重启导致 coredump 中断的问题。

### 运维管理

- 修复了 Prometheus 查询所得的系统分区空间使用率和实际使用率存在差异的问题。
- 修复了集群重新部署的过程中，未参与重新部署的节点可能导致其余节点部署异常的问题。
- 修复了集群部署时节点服务信息注册失败，但未返回错误或触发重试的问题。
- 修复了由于内存中的物理盘信息未及时刷新而导致新盘错误触发故障隔离的问题。
- 修复了主机进入维护模式成功但显示任务失败的问题。
- 修复了网口的 RDMA 支持情况显示错误的问题。
- 修复了 ZooKeeper Leader 所在节点无法通过命令行重试角色转换的问题。
- 修复了主机 CPU 温度数值展示异常的问题。
- 修复了一系列漏洞。

  - 修复了 9100 端口的安全漏洞。
  - 修复了漏洞扫描时检测到目标 URL 存在 http host 头攻击漏洞的问题。
  - 修复了 libvirt 相关漏洞。
- 修复了卸载含元数据分区的物理盘时，可能因为 mdadm 进程卡住导致卸载任务长时间无法结束的问题。
- 修复了主机的 CPU 使用率监控数据可能出现中断的问题。
- 修复了未经授权可下载集群 Swagger YAML 文件的问题。
- 修复了节点在内核升级并重启后，consul 服务偶发未能自动启动的问题。
- 修复了配置文件中存在重复项时，通过命令行移除主机失败的问题。
- 修复了 AArch64 架构的集群中 zbs-chunkd 服务可能出现 OOM 的问题。
- 修复了 systemd 配置文件生成后或主机重启后，时间回退可能导致 ntpd 服务无法启动的问题。
- 修复了在硬件或 BIOS 配置变化后，节点重启时 cgconfig 服务启动失败的问题。
- 修复了集群扩容后，新节点升级工具版本与现有节点不一致，导致后续升级失败的问题。
- 修复了端口扫描时 ntpm 的 http 指标数量暴增，导致监控报警功能服务异常的问题，默认关闭相关指标的采集。
- 修复了浏览器存在缓存数据可能导致集群部署时出现非预期现象的问题。
- 修复了节点的部分风扇与其他风扇转速差异过大的问题。
- 修复了 UEFI 引导模式下因启动盘重建方式不准确，导致集群升级场景下新增的内核参数可能不生效的问题。
- 修复了在主机上安装 ISO 时无法选择 removable 类型的物理盘的问题。
- 修复了因 NTP 服务器最大往返延迟阈值设置过于敏感，导致误触发主机无法与 NTP 服务器时间同步告警的问题。
- 修复了日志轮询配置写入路径错误导致日志无法轮询的问题。
- 修复了双活集群重建仲裁节点后集群升级失败的问题。
- 修复了集群升级后历史监控图表数据无法显示的问题。

### 网络

- 修复了网口的隔离状态可能不符合预期的问题。
- 修复了无法手动从虚拟分布式交换机中移除隔离中状态的网口的问题。
- 修复了被隔离网口恢复后，无法自动重新加入网口绑定的问题。
- 修复了由于部署过程中为管理网络重复创建了多个相同 VLAN 子接口，导致部署完成后集群管理网络无法连通的问题。
- 修复了 LLDP 进程向系统服务虚拟机网口或系统网络虚拟端口发送 LLDPDU 报文时，由于对应端口不处理此类报文主动丢包，从而导致丢包计数增加的问题。
- 修复了因 OVS 对 IGMP 协议的流量处理异常，导致 OVS 用户态进程的 CPU 打满，进而引发网络性能持续下降的问题。
- 修复了管理网络或存储网络不通时，未及时触发报警的问题。

### 内核

- 解决 iouring 中重新提交 I/O 时，I/O 丢失的问题。
- 修复了为 QLogic 网卡开启 ptp 特性时， CPU 使用率突增的问题。
- 修复了安全漏洞 CVE-2019-3016。
- 修复了仅支持 32bit MSI 的 PCIe 设备无法工作的问题。
- 修复了数据不一致导致块层 I/O 卡死的问题。
- 修复了 ext4 文件系统在线重置容量可能导致数据被破坏的问题。
- 修复了内存子系统可能被破坏的问题。
- 修复了 buffer I/O 数据不一致的问题。
- 修复了 kvm 异步缺页异常导致卡死的问题。
- 修复了内存申请路径可能卡死的问题。
- 修复了因系统中残留旧版 RDMA 包导致内核升级失败的问题。

# 版本配套说明

## 与服务器类型配套说明

SMTX ZBS 块存储软件支持直接部署在 Intel x86\_64 服务器、AMD x86\_64 服务器、Hygon x86\_64 服务器和鲲鹏 AArch64 服务器上。

SMTX ZBS 文件存储软件支持部署在上述架构的 SMTX ZBS 块存储集群中。

## 与 CloudTower 版本配套说明

SMTX ZBS 集群的管理服务由 CloudTower 提供，本版本 SMTX ZBS 集群只支持由 CloudTower 4.7.1 及以上版本进行管理；部署文件存储集群时，CloudTower 需为高配，且版本为 4.8.0 及以上。

## 与其他 SmartX 产品配套说明

SMTX ZBS 块存储集群和双活集群可与其他 SmartX 产品配合使用，但每个产品只有部分版本支持与当前版本的 SMTX ZBS 块存储集群和双活集群配合使用。请查阅对应产品的发布说明文档获取两个产品的版本配套关系。
