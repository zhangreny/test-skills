# SMTXZBS/5.7.0/SMTX ZBS 防火墙端口说明
## 关于本文档

# 关于本文档

本文档描述了与 SMTX ZBS 块存储集群和文件存储集群相关的防火墙端口规则，具体涵盖以下内容：

- **块存储集群端口说明**

  - 如何配置块存储集群主机网络之间的防火墙端口、块存储集群和外部客户端之间的防火墙端口，以确保集群能正常对外提供服务。
  - 第三方计算平台使用块存储集群的块存储服务时，第三方计算平台应该开放的端口。
  - 块存储内部服务占用的本地端口。
- **文件存储集群端口说明**

  - 如何配置文件控制器所在主机网络之间的防火墙端口、文件存储集群和外部客户端之间的防火墙端口，以确保集群能正常对外提供服务。
  - 文件存储内部服务占用的本地端口。

> **注意：**
>
> - 请勿修改块存储集群主机防火墙以及文件控制器所在主机网络防火墙的默认配置和默认状态(关闭)，否则会引起系统异常。
> - 若未作特殊说明，本文所提及端口均为 TCP 端口。

## 块存储集群端口说明 > 外部防火墙端口开放规则

# 外部防火墙端口开放规则

若块存储集群物理主机的网络之间存在防火墙（如交换机上）、或块存储集群与外部客户端之间存在防火墙，为确保块存储集群能够正常对外提供服务，防火墙均需开放如下端口。

以下开放规则仅讨论默认端口号。

## Web 控制台

| 端口 | 用途 | 说明 |
| --- | --- | --- |
| 443 | 网页浏览端口，用于 HTTPS 服务 | - |
| 80 | 网页浏览端口，用于 HTTP 服务 | - |

## SSH

| 端口 | 用途 | 说明 |
| --- | --- | --- |
| 22 | SSH 远程登录物理主机/SCVM/客户虚拟机 | SSH 协议的默认端口号。 |

## IPMI

| 端口 | 用途 | 说明 |
| --- | --- | --- |
| 623（UDP） | 访问 IPMI 管理台 | 适用于戴尔、超微服务器。其他厂商产品请参见其详细要求。 |

## SNMP

| 端口 | 用途 | 说明 |
| --- | --- | --- |
| 1705 | 对接 SMTX ZBS 监控报警服务 octopus，用于从 octopus 获取监控数据。 | 若未添加 SNMP 传输，则无需开放此端口。 |

## iSCSI

用户可创建 iSCSI 数据存储供外部客户端使用。如果外部客户端使用块存储集群的 iSCSI 存储服务，请确保客户端和集群中间的防火墙开启了如下端口：

| 端口 | 用途 | 说明 |
| --- | --- | --- |
| 3260、3261 | 用于通过 iSCSI 协议访问块存储集群的存储 | 标准 iSCSI 服务端口。 |

## NVMe-oF

用户可创建 NVMe Subsystem 和 NVMe Namespace 供外部客户端使用。如果外部客户端使用块存储集群的 NVMe-oF 存储服务，请确保客户端和集群中间的防火墙开启了如下端口：

| 端口 | 用途 | 说明 |
| --- | --- | --- |
| 4420 | 通过 NVMe over RDMA 协议对外提供块存储服务 | 标准 NVMe 服务端口。 |
| 8009 | 通过 NVMe over TCP 协议对外提供块存储服务 | 标准 NVMe 服务端口。 |

## 块存储集群端口说明 > 第三方计算平台的端口开放规则

# 第三方计算平台的端口开放规则

第三方计算平台使用 SMTX ZBS 块存储集群提供的块存储服务时，应当开放如下端口。

按照使用的存储服务不同，端口开放规则如下：

## iSCSI

当第三方计算平台通过 iSCSI 协议访问 SMTX ZBS 块存储集群提供的分布式块存储时，需要开放如下端口：

| 端口 | 用途 | 说明 |
| --- | --- | --- |
| 3260、3261 | 通过 iSCSI 协议访问块存储集群的分布式块存储 | 标准 iSCSI 服务端口。 |

## NVMe-OF

当第三方计算平台通过 NVMe over RDMA 或 NVMe over TCP 协议访问块存储集群的分布式块存储时，需要开放如下端口：

| 端口 | 用途 | 说明 |
| --- | --- | --- |
| 4420 | 通过 NVMe over RDMA 协议访问块存储集群的分布式块存储 | 标准 NVMe 服务端口。 |
| 8009 | 通过 NVMe over TCP 协议访问块存储集群的分布式块存储 | 标准 NVMe 服务端口。 |

## 随机端口

在实际场景中，NFS 服务和客户端本地也可能使用一些随机端口，具体说明如下：

- NFS 服务除了使用 `2049` 和 `111` 两个固定端口外，还会启动其他进程（如 status、nlockmgr 等），这些进程的端口由 RPC 服务进行随机分配，每次启动 NFS 服务时使用的端口均不相同。

  若需查看 NFS 当前使用的所有端口，或查看某一未知的端口对应的进程，可采用如下方法：

  - 查看 NFS 当前使用的随机端口：在主机执行 `rpcinfo -p` 命令查看。
  - 查看某一端口对应的服务进程：在主机执行 `netstat -nlp` 命令查看。
- 客户端本地也可能启用一些随机端口对接集群服务。若扫描到此类未知的本地端口，可在主机执行 `netstat -nlp` 命令，查看该端口对应的源端口或目的端口，并根据块存储内部服务使用的固定端口，确定本地端口对应的集群服务。

## 文件存储集群端口说明 > 外部防火墙端口开放规则

# 外部防火墙端口开放规则

若文件控制器所在物理主机的网络之间存在防火墙（如交换机上），或文件存储集群与外部客户端之间存在防火墙，为确保文件存储集群能够正常对外提供服务，防火墙均需开放如下端口。

以下开放规则仅讨论默认端口号。

## SSH

| 端口 | 用途 | 说明 |
| --- | --- | --- |
| 22 | SSH 远程登录文件控制器。 | SSH 协议的默认端口号。 |

## NFS

外部客户端使用文件存储集群的 NFS 存储服务时，请确保客户端和集群之间的防火墙开启了如下端口。

| 协议版本 | 端口 | 用途 | 说明 |
| --- | --- | --- | --- |
| NFSv3 | 111、2049、2050 | 通过 NFSv3 协议访问文件存储集群的分布式文件存储。 | 标准 NFS 服务端口。 |
| NFSv4.1 | 2049 | 通过 NFSv4.1 协议访问文件存储集群的分布式文件存储。 | 标准 NFS 服务端口。 |

## HDFS

外部客户端使用文件存储集群的 HDFS 存储服务时，请确保客户端和集群之间的防火墙开启了如下端口：

| 端口 | 用途 | 说明 |
| --- | --- | --- |
| 20010 | 访问文件系统的元数据。 | SFS 元数据服务端口。 |
| 20020 | 访问文件系统的数据。 | SFS 数据服务端口。 |
| 6443 | 访问 SFS API server 以获取文件存储集群控制信息。 | SFS API server 服务端口。 |
| 8429 | 通过文件控制器将 Hadoop 客户端的性能监控数据上传到监控平台。 | SFS OBS agent 监控端口。 |
| 20005 | 访问 SFS cloud provider 以获取文件存储集群的授权信息。 | SFS cloud provider 端口。 |

## 文件存储集群端口说明 > 文件存储服务防火墙端口配置要求

# 文件存储服务防火墙端口配置要求

为确保文件存储集群正常提供服务，源端和目标端都需要开放如下表所示的端口及协议。如没有特殊说明，则表示端口的传输端协议为 TCP。

| 源端 | 目标端 | 需开放的端口和协议 | 用途 |
| --- | --- | --- | --- |
| 文件控制器文件管理 IP 和文件存储集群文件管理虚拟 IP | 块存储集群管理虚拟 IP | 10206 | 文件存储集群的 sfs-cloud-provider 服务依赖块存储集群的 zbs-chunkd 提供的 meta proxy 服务。 |
| 文件控制器文件管理 IP | CloudTower IP | 80 | 文件控制器通过此端口调用 CloudTower API 上传 island。 |
| 文件控制器文件管理 IP | CloudTower 中配置的 NTP 服务器 IP | UDP：123 | 文件控制器通过此端口访问 NTP 服务器进行时间同步。 |
| 文件控制器文件管理 IP | 可观测性虚拟机管理 IP | 80 | 文件控制器内的 obs-agent 通过此端口调用 OVM API 上传采集的数据。 |
| 443 | 配套 1.4.2 及以上版本可观测性平台使用时需开放。  文件控制器内的 obs-agent 通过此端口调用 OVM API 上传采集的数据。 |
| CloudTower IP | 文件控制器文件管理 IP 和文件存储集群文件管理虚拟 IP | 6443 | CloudTower 和其上的 sfs-operator 服务通过此端口访问文件存储集群中的 apiserver 服务。 |
| ICMP | CloudTower 上的 sfs-operator 服务通过此协议检测文件控制器文件管理 IP 和文件管理虚拟 IP 是否已被占用。 |
| CloudTower IP | 文件控制器文件管理 IP | 22 | CloudTower 上的 CAPP 服务通过此端口建立 SSH 隧道代理文件控制器内 containerd 服务的 socket。 |
| 20003 | CloudTower 上的 sfs-operator 服务通过此端口访问文件控制器内的 NTPM 服务。 |
| 8685 | CloudTower 上的 bundle-application-operator 服务通过此端口访问文件控制器内的 obs-agent 服务。 |
