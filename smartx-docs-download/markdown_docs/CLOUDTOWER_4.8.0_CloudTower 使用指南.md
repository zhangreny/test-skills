# CLOUDTOWER/4.8.0/CloudTower 使用指南
## 关于本文档

# 关于本文档

本文档描述了使用 CloudTower 如何管理多个数据中心和 SmartX 集群（SMTX OS 集群、SMTX ZBS 集群和 SMTX ELF 集群）。CloudTower 支持为不同类型的 SmartX 集群统一提供报警、资源搜索、报表导出、升级、账户安全设置等各种丰富功能，同时也支持为某类 SmartX 集群提供一些高级功能以提升管理效率，例如为 SMTX OS 或 SMTX ELF 集群提供内容库、CloudTower 代理等功能。

- CloudTower 提供多个软件版本（[CloudTower 功能规格](https://www.smartx.com/cloud-tower/spec/)），每个版本的功能和服务均有所不同。本文档中所有功能均基于 **CloudTower 4.8.0 企业版**进行描述，若您所购买的软件版本不包含文档中描述的部分功能和特性，请忽略并以实际使用的软件功能为准。
- 在 CloudTower 中管理单个 SMTX OS 集群、SMTX ZBS 集群或 SMTX ELF 集群，以及针对集群内各种资源的管理操作，请参考对应产品的管理指南。

阅读本文档需了解 CloudTower 管理平台和其管理的对象，了解虚拟化、软件定义网络、软件定义存储、分布式存储等相关技术背景知识，并具备丰富的数据中心操作经验。

## 了解 CloudTower > CloudTower 的管理层级

# CloudTower 的定位

CloudTower 作为 SmartX 统一的管理平台，使用分布式云架构，旨在管理数据中心内及跨数据中心的多个 SmartX 集群，以降低管理难度，提升运维效率。

- 将多个 SmartX 集群的计算、存储和网络资源统一管理，概览详情一应俱全。
- 共享跨集群的资源和配置，例如报警规则、虚拟机模板和 ISO 映像等，仅需维护一份便可用于所有 SmartX 集群。
- 集中管理 x86\_64 和 AArch64 资源池，免去多站点跳转负担。
- 提供多种外部管理接口：[标准 RESTful API](https://code.smartx.com)，以及支持配置 SNMP 对接 Zabbix。

## 了解 CloudTower > CloudTower 的软件版本与许可类型

# CloudTower 的软件版本与许可类型

本版本的 CloudTower 提供三种软件版本：社区版、基础版和企业版。每个版本可管理的最大集群数、最大主机数以及对商业服务的支持可参见 SmartX 官网（[CloudTower 功能规格](https://www.smartx.com/cloud-tower/spec/)）。

> **说明**：
>
> 新部署的 4.3.0 及以上版本的 CloudTower 将不再提供企业增强版；若 CloudTower 的版本为 4.3.0 以下且其软件版本为企业增强版，则升级至本版本后仍为企业增强版。

CloudTower 的管理能力除了与软件版本密切相关外，还取决于部署时选择的环境配置等级（如下表所示）。

| **环境配置等级** | **CloudTower 管理能力** | **CloudTower 虚拟机配置** |
| --- | --- | --- |
| **低配** | - 10 个集群 - 100 台主机 - 1000 个虚拟机 | - 4 核 vCPU - 12 GiB 内存 - 100 GiB 存储空间 |
| **高配** | - 100 个集群 - 1000 台主机 - 10000 个虚拟机 | - 8 核 vCPU - 19 GiB 内存 - 400 GiB 存储空间 |

CloudTower 提供试用、订阅和永久三种许可。

- **试用许可**

  首次安装后的许可类型，默认有效期为 90 天。
- **订阅许可**：

  根据需要选择订阅时长，更为灵活。
- **永久许可**：

  许可永久有效。

在 CloudTower **系统配置**主界面的左侧导航树单击**软件许可**，可查看软件版本与许可信息。单击 CloudTower 右上角的用户图标，选择**关于**菜单，可查看 CloudTower 的版本号。

## 登录 CloudTower

# 登录 CloudTower

当您在安装部署完 CloudTower，并在浏览器中输入 CloudTower 的 IP 地址首次登录系统后，还需对 CloudTower 完成初始化设置。

在后续登录时，您可以选择使用本地账户、LDAP/AD 账户或 SSO 账户登录 CloudTower。

## 登录 CloudTower > 初始化 CloudTower（首次登录）

# 初始化 CloudTower（首次登录）

首次登录 CloudTower，在初始化时您需要填写以下信息：

| 参数 | 描述 |
| --- | --- |
| 组织名称 | 指 CloudTower 所管理的最高层级的名称，详细解释请参考 [CloudTower 的管理层级](/cloudtower/4.8.0/cloudtower_user_guide/cloudtower_user_guide_003)。  组织的名称最长包含 55 个字符，如需修改，可在**系统配置**界面单击**组织名称**进行修改。 |
| 超级管理员密码 | 密码长度不少于 8 位字符，不超过 16 位字符，且同时包含至少 1 位大写字母、1 位小写字母、1 位数字，不能包含非 ASCII 字符。 |
| 确认密码 | 请再次输入超级管理员密码，以对密码进行确认。 |

## 登录 CloudTower > 登录 CloudTower 的方式

# 登录 CloudTower 的方式

CloudTower 支持以下三种登录方式，其中，**以 LDAP/AD 登录**和**以 SSO 登录**要求 CloudTower 已配置 [LDAP/AD 登录](/cloudtower/4.8.0/cloudtower_user_guide/cloudtower_user_guide_080)和 [SSO 登录](/cloudtower/4.8.0/cloudtower_user_guide/cloudtower_user_guide_122)。配置后，您可在 CloudTower **系统配置**界面选择**默认登录**方式，在弹出的界面中设置用户在登录页面默认采用的登录方式。

- 以本地账户登录
- 以 LDAP/AD 登录
- 以 SSO 登录

## 使用 LDAP/AD 账户或本地账户登录

**操作步骤**

1. 在浏览器中输入 CloudTower 的 IP 地址，进入登录界面。
2. 选择**以 LDAP/AD 登录**或**以本地账户**登录，并输入用户名和密码。

   > **说明**：
   >
   > - 若当前的 CloudTower 的 IP 地址为已废弃的 CloudTower 的 IP 地址，则系统将在登陆界面自动填充历史用户信息，请先单击**切换用户**后再执行上述步骤。
   > - 若 Cloudtower 已启用**首次登录修改初始密码**，且该账户为首次登录，则您需要根据界面提示修改密码后完成登录。
3. 单击**登录**。
4. （可选）若 CloudTower 启用了[双因子认证](/cloudtower/4.8.0/cloudtower_user_guide/cloudtower_user_guide_084)，则您还需查看用户信息中填写的邮箱或手机号码接收的验证信息，将 6 位数验证码填写至输入框中，并单击验证。

   > **说明**：
   >
   > 当您同时启用了邮件认证和短信认证时，需要选择其中一种方式进行验证。

   完成登录后，您可单击 CloudTower 界面右上角的用户图标，选择**登出**以退出登录。

## 使用 SSO 账户登录

**操作步骤**

1. 在浏览器中输入 CloudTower 的 IP 地址，进入登录界面。
2. 选择**以 SSO 登录**，并进行 IdP 身份认证。

   1. 单击**以 SSO 登录**后，界面将提示您选择使用已配置的 IdP 平台登录。
   2. 单击**使用 xxx 登录**，弹出身份认证窗口。
   3. 输入账号和密码之后单击**登录**。
   > **说明**：
   >
   > 若您在 IdP 平台更新了用户信息，则该用户重新登录 CloudTower 之后，该用户在 CloudTower 中保存的相关信息将被覆盖。
3. （可选）若 CloudTower 启用了[双因子认证](/cloudtower/4.8.0/cloudtower_user_guide/cloudtower_user_guide_084)，则您还需查看用户信息中填写的邮箱或短信，将 6 位数验证码填写至输入框中，并单击**验证**。

   > **说明**：
   >
   > 当您同时启用了邮件认证和短信认证时，需要选择其中一种方式进行验证。

   完成登录后，您可单击 CloudTower 界面右上角的用户图标，选择**登出**以退出登录。

## 关联 SmartX 集群

# 关联 SmartX 集群

CloudTower 支持对 SmartX 集群统一管理，您可以通过关联集群的操作将 CloudTower 和待关联集群进行关联，从而实现对多个集群的统一管理。

**前提条件**

- CloudTower 与待关联集群之间管理网络连通。
- 待关联集群已设置集群管理虚拟 IP。
- 已登录账户需具有**集群关联管理**的操作权限。

**操作步骤**

1. 在 CloudTower 的组织概览界面右上方单击 **+ 创建**，选择**关联集群**，弹出**关联集群**对话框。
2. 填写待关联集群的管理虚拟 IP、超级管理员账户的用户名和密码。完成后单击**下一步**，开始加载集群的数据。
3. 连接集群成功后，确认关联集群的信息，包括集群的名称、管理虚拟 IP、软件版本、以及使用的虚拟化平台（仅 SMTX OS 集群和 SMTX ELF 集群）等信息。
4. 选择是否将集群加入数据中心。

   > **说明**:
   >
   > - 对于 SMTX OS 集群、SMTX ZBS 块存储集群或 SMTX ELF 集群，支持选择已创建的数据中心，也可在**确认关联集群**阶段直接创建新的数据中心。
   > - 如果待关联的集群为 SMTX OS 双活集群或 SMTX ZBS 双活集群，则必须为双活集群的优先可用域和次级可用域分别选择其所属的数据中心。
5. 若关联集群为 SMTX OS（ELF）集群或已关联存储集群的 SMTX ELF 集群，则需为集群设置默认存储策略，包括**冗余策略**、**置备方式**和**静态数据加密**选项。您可在《SMTX OS 管理指南》的**管理虚拟卷（ELF 平台）**> **虚拟卷的配置选项**小节了解各个配置选项的详细信息。

   - **SMTX OS（ELF）集群**：建议选择 3 副本或 K+2 的纠删码策略，可容忍任意两块物理盘或两个节点同时故障。
   - **SMTX ELF 集群**：
     - 若关联存储仅有一个，且被当前 CloudTower 管理，则建议选择 3 副本或 K+2 的纠删码策略，可容忍任意两块物理盘或两个节点同时故障。
     - 若关联存储仅有一个，且未被当前 CloudTower 管理，则默认存储策略中的冗余策略将自动设置为 3 副本，置备方式为精简置备，不启用静态数据加密。您可将关联存储添加至当前 CloudTower 后，设置完整的默认存储策略。
     - 若关联存储有多个，则默认存储策略中的冗余策略将自动统一设置为 3 副本，置备方式为精简置备，不启用静态数据加密。关联集群后，您可为每个关联存储单独配置完整的默认存储策略。
6. 单击**关联集群**，将 CloudTower 与 SmartX 集群完成关联。

   关联成功后， 可在 CloudTower 中管理 SmartX 集群的主机、物理盘、网口、虚拟机和存储等资源。

## 查看 CloudTower 管理的所有集群信息 > 查看组织概况

# 查看组织概况

在 CloudTower 主界面的**概览**页签下展示了组织中包括的所有集群的相关信息。

- **报警**

  展示当前未解决报警的类型和数量，以及最近触发的报警信息。

  按照报警项的严重和紧急程度，报警分为**严重警告**、**注意**和**信息**三种类型。

  您可通过单击该模块，跳转至**报警**中心，查看每条报警的详细信息。
- **CPU**

  展示组织内两类 SmartX 集群（SMTX OS 集群和 SMTX ELF 集群）与 CPU 资源相关的汇总信息。下面各参数的统计信息只涉及 SMTX OS 集群和 SMTX ELF 集群。

  - 该区域的右上角展示以下 CPU 信息：
    - **总频率**：集群中所有主机的 CPU 频率之和，例如 `6.6 THz`。
    - **总核数**：集群中所有主机的物理核数之和，例如 `2896 核`。
  - `使用率`：集群中 CPU 的使用率。
- **内存**

  展示组织内两类集群（SMTX OS 集群和 SMTX ELF 集群）的内存使用状态，包括总内存、当前可分配内存、已使用内存。
- **存储资源**

  展示组织内两类集群（SMTX OS 集群和 SMTX ZBS 集群）的存储**总容量**，以及**已使用**、**失效**和**空闲**的容量，并根据副本数量展示相应的可分配容量。

  **失效容量**是指由不健康且在线的物理盘、存储失效的节点和正常卸载的物理盘提供的数据空间容量之和。
- **集群**

  展示 CloudTower 管理的集群总数，以及每种集群类型对应的集群数量。集群类型包括 SMTX OS 集群（ELF 平台和 VMware ESXi 平台）、SMTX ELF 集群和 SMTX ZBS 集群。

  单击集群卡片的任意位置，可跳转至**集群**页签，以查看组织内的所有集群。
- **主机**

  展示组织内的主机总数，以及处于各种状态的主机数量。
- **虚拟机**

  展示组织内 SMTX ELF 集群和 SMTX OS（ELF）集群的虚拟机总数，以及处于各种状态的虚拟机数量。
- **物理盘**

  展示组织内两类集群（SMTX OS 集群和 SMTX ZBS 集群）的 HDD 和 SSD 硬盘的总数以及各种状态下硬盘的数量。
- **数据恢复**

  展示组织内当前正在进行数据恢复的待恢复数据量，以及恢复速率。
- **回收站**

  展示系统回收站中以下资源对象的总数量，以及今天和总共可释放的空间信息。

  - SMTX ELF 集群和 SMTX OS（ELF）集群的虚拟机
  - SMTX ZBS 块存储集群和 SMTX OS（ELF）集群的 LUN
  - SMTX ZBS 块存储集群的 Namespace
  - SMTX OS（VMware ESXi）集群的 NFS 文件

## 管理报警

# 管理报警

报警是指系统在发生特定事件时自动生成的一种事件报告。出现报警信息表示系统检测到了异常情况，运维人员需采取相应措施解决报警，避免造成严重影响。

报警功能由集群内部的本地监控或可观测性服务提供。

- 本地监控仅针对集群进行监控报警。当集群产生报警时，将由本地监控输出报警信息。
- 可观测性服务除对集群进行监控报警外，还能够对其关联的系统服务及其自身进行监控报警，并提供更加丰富的报警项。使用可观测性服务提供报警时，需先将报警对象和 1.2.0 及以上版本的可观测性服务关联，具体操作请参考《可观测性平台用户指南》的[关联集群](/visualization_cn/1.2.0/visualization_user_guide/visualization_user_guide_07)和[关联系统服务](/visualization_cn/1.2.0/visualization_user_guide/visualization_user_guide_49)章节。

  关联完成后，可观测性服务可以作为多集群和多系统服务的统一报警输出端。

此外，CloudTower 还为集群和系统服务服务提供了完备的报警规则。当集群或系统服务所在虚拟机的资源使用达到设定阈值或者发生特定事件触发相应报警规则时，将生成报警信息。

您可以配置 SMTP 邮件报警通知、SNMP 陷阱或 webhook 通知，以将产生的报警信息发送至指定邮箱、第三方客户端或配置的 webhook 地址。

---

## 管理报警 > 报警信息的等级与分类

# 报警信息的等级与分类

**报警信息的等级**

系统的报警等级由低至高分为**信息**、**注意**和**严重警告**，不同等级的报警所需采取的措施不同。

- **信息**

  该问题造成的影响较小，需要用户知晓，并按需排查风险和异常。
- **注意**

  检测到异常问题，需要及时关注并尽快执行相应操作，否则可能会对集群产生影响。
- **严重警告**

  已检测到紧急情况，需要立刻采取行动进行处理。

**报警信息的分类**

报警信息可分为以下两类：

- **未解决**

  CloudTower 中产生的所有未解决的报警将会展示在未解决列表，单击报警信息可查看报警详情，包括触发原因、影响、解决方法、报警源、报警等级、来自集群、报警规则、触发时间和持续时间等。其中，报警源包括集群内部和可观测性服务。
- **已解决**

  报警被解决之后将被移至已解决的报警列表，单击报警信息可查看报警详情，包括触发原因、影响、解决方法、报警源、报警等级、来自集群、报警规则、触发时间和持续时间等。其中，报警源包括集群内部和可观测性服务。

# 管理报警规则

CloudTower 为集群和系统服务提供了完备的内置报警规则，涵盖了集群硬件、服务、计算、存储、网络等各方面。此外，当 CloudTower 已关联 1.4.0 及以上版本的可观测性服务时，您还可以创建报警对象为虚拟机的自定义报警规则，对虚拟机 CPU、磁盘、网卡相关指标进行监控，以便您及时获取目标虚拟机的异常情况。

## 管理报警 > 管理报警规则 > 编辑内置规则

# 编辑内置规则

内置规则由**全局规则**和**特例规则**组成。全局规则为系统默认设置，特例规则为用户自定义设置，默认不设置特例规则，您可根据需要添加一条或多条。

全局规则对报警描述和报警等级进行了定义，默认应用于所有报警对象。您可以选择是否启用全局规则并对报警等级进行设置。在设置特例规则时，支持选择一个或多个报警对象设置报警等级。加入特例规则的报警对象将不再遵循全局规则。

> **说明**：
>
> - 网络流量可视化的报警规则不支持禁用特定报警等级。
> - SEL 硬件报警不支持编辑阈值。
> - 若相关产品与服务对相关的报警规则的系统默认值（全局规则）进行了更新，则在 CloudTower 从低版本升级至本版本后，您需要在编辑内置报警规则页面单击**重置为默认值**，以应用最新的系统默认值。

**操作步骤**

1. 在 CloudTower 的**报警**界面，选择**报警规则**，在**内置规则**页面顶部搜索框搜索到需要编辑的规则，单击该规则，在弹出的规则详情界面选择**编辑**，进入**编辑内置报警规则**界面。
2. 选择是否开启全局报警规则，并设置报警级别和阈值。
3. （可选）：单击添加特例规则，选择集群，并为指定集群设置集群特例规则。
4. 单击**保存**，完成内置报警规则编辑。

但以下类型的报警，无法被检测到是否已解决。在事件处理完后，可以单击该报警进入详情界面，单击**标记为已解决**，手动将该报警移入**已解决**列表。

## 管理报警 > 设置报警通知

# 设置报警通知

CloudTower 支持为集群和系统服务设置报警通知，以将系统中产生的报警信息发送至用户指定的邮箱或第三方客户端。

- **设置 SMTP 邮件报警通知**

  设置 SMTP 邮件报警通知后，当有报警被触发或被解决时，系统会自动向用户指定的邮箱发送邮件。

  CloudTower 支持创建集群内部配置，以通过集群内部的本地监控发送报警通知。当集群或系统服务关联了 1.2.0 及以上版本的可观测性服务时，还支持创建可观测性配置，以通过可观测性服务发送报警通知。
- **配置 SNMP 陷阱**

  系统支持使用简单网络管理协议（SNMP）和第三方监控报警平台（如 Zabbix）集成，配置 SNMP 陷阱后，SNMP 客户端可以便捷地获取报警信息。

  CloudTower 支持创建集群内部配置，以通过集群内部的本地监控发送报警信息。当集群或系统服务关联了 1.2.0 及以上版本的可观测性服务时，还支持创建可观测性配置，以通过可观测性服务发送报警信息。
- **设置 webhook 通知配置**

  当 CloudTower 已部署 1.3.0 及以上版本的可观测性服务时，支持配置 webhook 通知。配置后，当有报警被触发或被解决时，系统会自动向指定的 webhook 地址发送报警信息。

> **说明**：
>
> 若您在低于 4.1.0 版本的 CloudTower 中为可观测性服务 1.1.0 配置了邮件报警通知和 SNMP 陷阱，在将 CloudTower 升级到 4.1.0 及以上版本后，系统将自动为可观测性服务 1.1.0 创建可观测性配置，并删除原有的配置。

# 设置 SMTP 邮件报警通知

2. 根据待创建的报警邮件通知类型，填写相应信息：

   | 参数 | 描述 |
   | --- | --- |
   | 报警源 | 对于未关联可观测性 1.2.0 及以上版本的集群，选择**集群内部**作为报警源。 |
   | 名称 | 邮件通知配置的名称。 |
   | 报警对象 | 仅支持选择未关联已启用的集群内部邮件通知配置的集群，可选择一个或多个。 |
   | SMTP 服务器 | 选择一个已有的 SMTP 服务器或新创建一个 SMTP 服务器，通过该 SMTP 服务器向指定邮箱发送报警通知邮件。  若要新创建 SMTP 服务器，您可在下拉列表中选择**+ 创建 SMTP 服务器**，并根据《CloudTower 使用指南》的[配置 SMTP 服务器](/cloudtower/4.8.0/cloudtower_user_guide/cloudtower_user_guide_074)章节完成配置。  **说明**:  选择以域名作为服务器地址的 SMTP 服务器时，请在所选集群的设置页面中配置 DNS 服务器，以便正常发送通知。 |
   | 发件地址 | 指定一个电子邮件地址，作为报警通知的发件人。 |
   | 收件地址 | 指定一个或多个电子邮件地址，作为报警通知的收件人。如有多个收件人，用英文逗号“,”隔开多个邮件地址。 |
   | 语言 | 邮件通知的语言，包括简体中文和英语。 |
   | 报警等级 | 当报警符合所设置的报警等级时，将发送报警通知邮件。 级别可多选，包括**严重警告**、**注意**、**信息**。 |
   | 启用配置 | 是否启用上述配置。 |
3. 单击**创建**，完成配置。

   创建邮件通知配置后，可以在**通知配置** > **邮件**界面查看所有邮件通知配置的信息，包括集群内部配置和可观测性配置。

## 配置 NTP 服务器

# 配置 NTP 服务器

当 CloudTower 的时间不准确时，将影响任务、监控等功能的时间显示，或造成备份与容灾、Kubernetes 等其他系统服务的功能异常。因此，建议您为 CloudTower 配置 NTP 服务器进行周期性时钟同步，以使 CloudTower 的时间与您的目标校准时间保持同步。

**前提条件**

用户已架设好可用的 NTP 服务器。

**操作步骤**

1. 在 CloudTower **系统配置**主界面左侧的导航树选择 **CloudTower 时间**，该页面展示了 CloudTower 的当前时间和配置的 NTP 服务器信息。

   - **当前时间**

     | 参数 | 描述 |
     | --- | --- |
     | CloudTower 时区 | 默认时区为 Asia/Shanghai（UTC+8:00） |
     | CloudTower 时间 | CloudTower 当前时间 |
     | NTP 服务器时间 | 配置的 NTP 服务器的时间 |
   - **NTP 服务器**

     | 参数 | 描述 |
     | --- | --- |
     | 服务器地址 | 外部 NTP 服务器的地址 |
     | 状态 | NTP 服务器的状态，包括： - 正常 - 地址解析失败 - 网络连通异常 - 服务端无响应 - 存在时间偏差 - 客户端检查未通过 |
2. 在 **NTP 服务器**下方单击 **+ 添加服务器**，并在配置框中填写 NTP 服务器地址。建议配置至少三个服务器地址，并确保 CloudTower 可访问，以保证时间同步。
3. 单击**检查有效性**，通过检查后单击**保存**，完成配置。

   完成配置后，您可单击 **NTP 服务器**右侧的**编辑**，修改服务器地址，或对服务器进行增删。

## 管理资源 > 全局搜索目标资源 > 高级搜索

# 高级搜索

CloudTower 支持使用高级搜索功能以结合多个条件进行搜索，搜索结果为满足多个筛选条件的交集。

不同资源类型的高级搜索条件不同，搜索条件依据其资源列表中可展示的信息而定。

**搜索方法**

1. 进入**资源列表**界面，单击搜索框，并单击搜索框右侧的筛选图标，弹出筛选条件列表。
2. 选择筛选项并输入筛选条件。没有输入搜索条件的项默认搜索范围为全部。

   - 同一筛选项内的多个条件搜索结果为并集关系，即只要满足其中一个条件的资源即可被搜索出来。
   - 不同筛选项的搜索结果为交集关系，即搜索出的资源同时满足多个筛选项条件。
3. 单击**筛选**，在资源列表中查看筛选结果。

ISO 映像可用于为虚拟机安装操作系统，安装应用程序，传输数据或修复软件，您可以在虚拟机的 CD-ROM 中载入对应用途的 ISO 映像。在内容库中，您可以将 ISO 映像上传至与 CloudTower 关联的一个或多个集群中，并通过内容库在多个集群之间共享 ISO 映像。

## 管理内容库 > 管理 ISO 映像 > 上传 ISO 映像

# 上传 ISO 映像

当内容库中没有需要使用的 ISO 映像时，可以在内容库中手动上传目标 ISO 映像至任一可选集群。如果已确认需要载入目标 ISO 映像的虚拟机，建议将 ISO 映像上传至虚拟机的所属集群，以便缩短载入耗时。当前仅支持上传单个 ISO 映像，有以下两种上传方式。

- **从本地上传**：如果本地已下载需要上传的 ISO 映像文件，可以从本地上传该文件至目标所属集群。
- **从 URL 上传**：如果已获得存放 ISO 映像文件的服务器 URL，可以直接在上传窗口输入该 URL，让系统自动连通存放文件的服务器与目标所属集群，实现文件上传。

**注意事项**

- 仅支持上传至 SMTX OS（ELF）集群和已关联 SMTX ELF 集群的 SMTX ZBS 块存储集群。
- 一次只能上传一个 ISO 映像。如果您在较短时段内陆续进行了多次上传，则在同一时段内最多只能有 5 个 ISO 映像进入传输进程，其余文件将按照执行上传操作的先后顺序排队等待上传。

**操作步骤**

1. 在 **ISO 映像**页面右上角单击**上传 ISO 映像**。
2. 在弹出的对话框中，选择从**本地选择文件**或**从 URL 上传**。

   - **本地选择文件**：拖拽需要上传的 ISO 映像文件到虚线框中，或者单击**选择文件**并选择目标文件。
   - **从 URL 上传**：输入需要上传的 ISO 映像文件的服务器 URL，再单击**解析 URL**。
3. 设置该 ISO 映像的名称。系统默认填写本地文件的名称或者通过 URL 解析出的名称。该名称支持手动修改，但不能与内容库中已有的 ISO 映像重名。
4. （可选）添加关于该 ISO 映像的描述。
5. 选择至少一个所属集群。
6. 单击**上传**。

   上传过程中，界面右上角将显示上传图标，单击该图标可以查看上传进度。上传结束后，您可以在 ISO 映像列表查看到刚刚上传的文件。

# 编辑 ISO 映像

在 ISO 映像列表中选中一个 ISO 映像后，在弹出的详情面板中即可进行编辑。编辑操作的注意事项和操作入口如下。

| 编辑操作 | 注意事项 | 操作入口 |
| --- | --- | --- |
| 编辑名称和描述 | 当 ISO 映像的任意一个所属集群连接异常时，不支持修改 ISO 映像的名称和描述。 | 单击**编辑** |
| 编辑所属集群 | - 仅 SMTX OS（ELF）集群和已关联 SMTX ELF 集群的 SMTX ZBS 块存储集群可作为所属集群。 - 当 ISO 映像已被挂载给虚拟机时，不支持移除该虚拟机所在的集群。 - 当 ISO 映像的某个所属集群连接异常时，不支持移除该集群。 - 当 ISO 映像仅有一个所属集群时，若移除该集群，则该映像也将被删除，且不可恢复。 | 单击**所属集群**页签，再单击**编辑所属集群** |

## 管理内容库 > 管理 ISO 映像 > 分发 ISO 映像

# 分发 ISO 映像

您可以将内容库中的一个或多个 ISO 映像同时分发给一个或多个集群。如果内容库中的 ISO 映像需要在多个集群中使用，建议您提前做好分发，以便在对应集群的虚拟机中直接载入这些 ISO 映像，缩短载入耗时。此外，如果 ISO 映像只有一个所属集群，为避免集群连接异常导致 ISO 映像无法使用，也建议您分发至更多集群，做好备份。

**注意事项**

- 为提升分发速度，建议先[部署 CloudTower 代理](/cloudtower/4.8.0/cloudtower_user_guide/cloudtower_user_guide_077)，再执行分发操作。若已部署 CloudTower 代理的版本低于 1.3.0，请尽快将 CloudTower 代理升级至 1.3.0 及以上，否则可能影响数据传输。

**操作步骤**

1. 在 ISO 映像列表中勾选一个或多个要分发的 ISO 映像，在列表上方单击**分发**。
2. 在弹出的对话框中，选择一个或多个集群作为分发的目标集群。
3. 单击**分发**。

   在任务中心可以查看分发任务的进度和详情，或者停止分发任务。

## 管理内容库 > 管理虚拟机模板 > 创建虚拟机模板 > 创建虚拟机模板的方式、要求和限制

# 创建虚拟机模板的方式、要求和限制

创建虚拟机模板有以下 3 种方式：

- 将虚拟机克隆为虚拟机模板：克隆虚拟机，并将其转化成为虚拟机模板，存放在目标集群中。
- 将虚拟机转化为虚拟机模板：将虚拟机直接转化为虚拟机模板，并存放在目标集群中。转化完成后源虚拟机将不存在。
- 将虚拟机 OVF 文件导入为虚拟机模板：将虚拟机的配置信息和磁盘文件以 OVF 文件的形式导入目标集群中作为虚拟机模板。如果需要添加在 CloudTower 关联集群之外的虚拟机模板，可以采用此方式。

以上 3 种创建方式分别有不同的限制与要求，如下：

| 创建方式 | 限制 | 要求 |
| --- | --- | --- |
| 将虚拟机克隆为虚拟机模板 | - 如果虚拟机已配置共享卷、USB 设备、SR-IOV 直通网卡、PCI 直通网卡、GPU 设备或 ISO 映像，则虚拟机模板将不包含相应设备。 - 虚拟机模板中的虚拟网卡不包含虚拟机网络配置信息和队列数量配置。 - 仅当虚拟机已安装 SMTX VMTools 时，虚拟机模板才包含“客户机操作系统”信息。 - 虚拟机模板不包含加密控制器。 - 虚拟机模板不包含 CPU QoS 设置和 CPU 独占设置。 | - 为虚拟机模板选择的所属集群必须为 SMTX OS（ELF）集群或者已关联 SMTX ELF 集群的 SMTX ZBS 块存储集群。 - 如果为虚拟机模板选择的所属集群是 SMTX OS（ELF）集群，则该集群与虚拟机的所属集群的 CPU 架构必须均为 x86\_64 或 AArch64。 |
| 将虚拟机转化为虚拟机模板 |
| 将虚拟机 OVF 文件导入为虚拟机模板 | 集群的服务器架构和虚拟化平台版本不同时，OVF 文件在虚拟化平台之间的导入导出也会有不同的限制。  - 如果模板所属集群为 SMTX OS （ELF）集群，可以参考对应集群版本的《SMTX OS 管理指南》中的 OVF 文件导入导出兼容性列表。 - 如果模板所属集群为已关联 SMTX ELF 集群的 SMTX ZBS 块存储集群，可以参考对应 SMTX ELF 集群版本的《SMTX ELF 管理指南》中的 OVF 文件导入导出兼容性列表。 | 为虚拟机模板选择的所属集群必须为 5.0.5 及以上的 5.0.x、5.1.x 或 6.x.x 版本的 SMTX OS（ELF）集群，或者已关联 SMTX ELF 集群的 SMTX ZBS 块存储集群。 |

## 管理内容库 > 管理虚拟机模板 > 创建虚拟机模板 > 将虚拟机克隆或转化为虚拟机模板

# 将虚拟机克隆或转化为虚拟机模板

**前提条件**

虚拟机处于`关机`状态。

**操作步骤**

1. 在 CloudTower 的**集群**主界面中，进入虚拟机列表。
2. 选中目标虚拟机，在弹出的虚拟机详情面板右上方单击 **...** > **创建虚拟机模板**，选择**克隆为虚拟机模板**或**转化为虚拟机模板**。
3. 在弹出的对话框中，设置虚拟机模板的名称。名称默认为`虚拟机名称-template`，支持手动修改。
4. （可选）如果在步骤 1 选择了**克隆为虚拟机模板**，可以为模板添加描述。
5. （可选）当需要使用该虚拟机模板创建使用 cloud-init 或 cloudbase-init 的虚拟机时，如果确认虚拟机中已安装 cloud-init 或 cloudbase-init，并已完成配置文件的修改，请开启**已安装虚拟机初始化工具**。

   > **注意**：
   >
   > - cloud-init 为 Linux 平台的一个开源工具，可以为 Linux 虚拟机完成一系列初始化配置，包括默认语言区域、主机名和 SSH 认证密钥等。由于 cloud-init 不支持 Windows 操作系统，因此 Windows 虚拟机可使用 cloudbase-init 作为替代。您可以参考《cloud-init 虚拟机初始化配置指南》或《cloudbase-init 虚拟机初始化配置指南》详细了解这两类工具的使用说明。
   > - 如果虚拟机中未安装 cloud-init 或 cloudbase-init，请勿开启**已安装虚拟机初始化工具**，否则后续通过该模板创建虚拟机时将无法成功配置初始化数据。
6. 选择至少一个所属集群。

   > **说明**：
   >
   > - 无论选择哪个版本或类型的所属集群，模板的存储策略均与源虚拟机保持一致。
   > - 当虚拟机的所属集群为 4.0.4 及以下版本的 SMTX OS（ELF）集群时，虚拟机模板的所属集群将固定为虚拟机的所属集群，不支持修改。
7. 单击**转化**或**克隆**。

   虚拟机模板创建完成后，可以在内容库的虚拟机模板列表中查看到该模板。

## 管理内容库 > 管理虚拟机模板 > 创建虚拟机模板 > 将虚拟机 OVF 文件导入为虚拟机模板

# 将虚拟机 OVF 文件导入为虚拟机模板

**前提条件**

已获取要导入的 OVF 文件，包含 `.ovf` 文件和 `.vmdk` 文件。

**操作步骤**

1. 在内容库的虚拟机模板列表右上方单击添加**添加虚拟机模板** > **导入虚拟机模板**。
2. 在弹出的**导入虚拟机模板**对话框中，填写配置信息。

   - **选择文件**

     | 配置参数 | 说明 |
     | --- | --- |
     | 所属集群 | 用于存放虚拟机模板的集群。 |
     | CPU 架构 | 当所属集群为 SMTX ZBS 块存储集群时，需选择虚拟机模板的 CPU 架构。如果已确认需要在与 SMTX ZBS 块存储集群关联的某 SMTX ELF 集群上创建虚拟机，可以选择与该 SMTX ELF 集群相同的 CPU 架构。 |
     | OVF 格式的文件 | 后缀名为 `.ovf` 的文件。最多支持选择一个文件。 |
     | VMDK 格式的文件 | 后缀名为 `.vmdk` 的文件。`.ovf` 文件不同时，需选择的 `.vmdk` 文件数量也不同，请根据提示选择所需的文件。 |
   - **基本信息**

     | 配置参数 | 说明 |
     | --- | --- |
     | 模板名称 | 不能与内容库内已有的虚拟机模板名称重复。 |
     | 描述（选填） | 一般用来记录对该模板的一些附加说明信息。 |
     | 作为安全防护虚拟机（SVM）模板 | 勾选后，使用该模板创建虚拟机时，将默认创建安全防护虚拟机。 |
   - **计算资源**

     | 配置参数 | 说明 |
     | --- | --- |
     | vCPU 分配 | 系统默认填写导入文件中的 vCPU 数值，您也可以手动修改，最多支持设置 240 vCPU。单击**高级**选项，您还可以设置插槽。 |
     | 内存分配 | 系统默认填写导入文件中的内存数值，您也可以选择 8 GiB 或 16 GiB，或者自定义内存，但所填数值必须为 0.5 的正整数倍，并且不能超过界面提示的最大可用内存。 |
   - **磁盘**

     | 配置参数 | 说明 |
     | --- | --- |
     | 磁盘 | 系统默认添加导入文件中具有的虚拟卷，不可删除，但其名称、总线和存储策略可重新设置。其中，存储策略默认为模板所属集群的默认存储策略。 |
     | CD-ROM | 系统默认添加导入文件中具有的 CD-ROM，您可按需添加、删除、启用或禁用。禁用的 CD-ROM 设备不会在虚拟机内部显示。  不支持载入 ISO 映像。 |
   - **虚拟网卡**

     虚拟网卡支持 VIRTIO 和 E1000 类型。系统默认添加导入文件中具有的网卡，可以添加新的网卡、删除已有的网卡。

     | 配置参数 | 说明 |
     | --- | --- |
     | 启用/禁用 | 启用或禁用虚拟网卡。 |
     | 模式 | 虚拟网卡的模式与部署的服务器的架构类型有关： - 当服务器为 Intel x86\_64 或者 Hygon x86\_64 时，网卡模式可设置为 **VIRTIO** 或 **E1000**。 - 当服务器为鲲鹏 AArch64 架构时，则网卡模式只能设置为 **VIRTIO**。 |
     | 镜像模式 | 启用镜像模式后，该网卡只能用于接收镜像流量，监听的范围是网卡连接的虚拟分布式交换机发往该主机的所有流量，而无法用于常规用途。 |
   - **其他配置**

     | 配置参数 | 说明 |
     | --- | --- |
     | 高可用（HA） | 启用该项后，当虚拟机存在可用性问题时，将会自动在正常的主机上重建。 |
     | 固件 | 系统默认选择导入文件中记录的固件类型，不可修改。 |
3. 单击**确定**。

   在任务中心可以查看虚拟机模板的导入状态。导入成功后，可以在虚拟机模板列表中查看到刚刚导入的模板。

| 编辑操作 | 注意事项 | 操作入口 |
| --- | --- | --- |
| 编辑名称、描述、已安装虚拟机初始化工具 | - 当虚拟机模板的任意一个所属集群连接异常时，不支持修改虚拟机模板的名称和描述，也不支持修改是否已安装虚拟机初始化工具选项。 - 如果需要开启**已安装虚拟机初始化工具**，请务必确认创建该虚拟机模板的源虚拟机已安装虚拟机初始化工具，并已完成配置文件的修改，否则后续通过该模板创建虚拟机时将无法成功配置初始化数据。 | 单击**编辑** |

## 管理内容库 > 管理虚拟机模板 > 分发虚拟机模板

# 分发虚拟机模板

您可以将内容库中的一个或多个虚拟机模板同时分发给一个或多个集群。如果内容库中的虚拟机模板需要在多个集群中使用，建议您提前做好分发，以便在对应集群中从模板创建虚拟机时直接使用虚拟机模板，缩短创建耗时。此外，如果虚拟机模板只有一个所属集群，为避免集群连接异常导致虚拟机模板无法使用，也建议您分发至更多集群，做好备份。

**注意事项**

- 仅支持分发至已关联 SMTX ELF 集群的 SMTX ZBS 集群，以及 CPU 架构（x86\_64 或 AArch64）与虚拟机模板相同的 SMTX OS（ELF）集群。
- 所属集群为 4.0.4 及以下版本的 SMTX OS（ELF）集群的虚拟机模板只能在集群内使用，无法分发至其他集群。其他集群的虚拟机模板也无法分发至 4.0.4 及以下版本的 SMTX OS（ELF）集群。
- 若分发的目标集群不支持虚拟机模板中某些磁盘的冗余策略，则在分发后，在未启用双活特性的集群的模板中，这些磁盘的冗余策略将变更为 **2 副本**，若目标集群是双活集群，则将变更为 **3 副本**。
- 分发后的虚拟机模板中的虚拟卷不启用静态数据加密。
- 为提升分发速度，建议先[部署 CloudTower 代理](/cloudtower/4.8.0/cloudtower_user_guide/cloudtower_user_guide_077)，再执行分发操作。若已部署 CloudTower 代理的版本低于 1.3.0，请尽快将 CloudTower 代理升级至 1.3.0 及以上，否则可能影响数据传输。

**操作步骤**

1. 在内容库的虚拟机模板列表中勾选一个或多个要分发的虚拟机模板，在列表上方单击**分发**。
2. 在弹出的对话框中，选择一个或多个集群作为分发的目标集群。
3. 单击**分发**。

   在任务中心可以查看分发任务的进度和详情，或者停止分发任务。

## 管理内容库 > 管理虚拟机模板 > 导出虚拟机模板

# 导出虚拟机模板

虚拟机模板可以以 OVF 文件的形式导出，导出的文件将包含以下信息：

- 基本信息：虚拟机模板名称
- 计算资源：vCPU 分配、内存分配
- 磁盘：虚拟卷（不包含存储策略的配置）、CD-ROM（不包含 ISO 映像、默认启用）
- 虚拟网卡（仅包含虚拟网卡的数量）
- 其他配置：固件

**注意事项**

集群的服务器架构和虚拟化平台版本不同时，OVF 文件在虚拟化平台之间的导入导出也会有不同的限制。

- 如果模板所属集群为 SMTX OS （ELF）集群，可以参考对应集群版本的《SMTX OS 管理指南》中的 OVF 文件导入导出兼容性列表。
- 如果模板所属集群为已关联 SMTX ELF 集群的 SMTX ZBS 块存储集群，可以参考对应 SMTX ELF 集群版本的《SMTX ELF 管理指南》中的 OVF 文件导入导出兼容性列表。

**前提条件**

- 虚拟机模板中包含虚拟卷。
- 虚拟机模板的所属集群包含集群版本为 5.0.5 及以上的 5.0.x、5.1.x 或 6.x.x 的 SMTX OS（ELF）集群或者已关联 SMTX ELF 集群的 SMTX ZBS 块存储集群。

**操作步骤**

1. 在内容库的虚拟机模板列表中选中要导出的模板，在弹出的虚拟机模板详情面板右上方单击**导出**。

   若需要导出多个模板，可以在**虚拟机模板**列表中勾选全部要导出的模板，再在列表右上方单击**导出**。
2. 在弹出的**导出虚拟机模板**对话框中，确认要导出的模板后名称后，单击**导出**。

   在任务中心可以查看虚拟机模板的导出状态。
3. 导出完成后，下载导出的 OVF 文件。有以下两种下载方式：

   - 在任务中心的本次导出任务条目下方，单击**下载**。
   - 在该虚拟机模板的详情面板中，在已生成 OVF 文件的提示信息右侧，单击**下载**。

   导出任务生成的 OVF 文件包含 .ovf 文件和 .vmdk 文件。导出文件在 CloudTower 管理平台只能保留 72 小时，请及时下载。

## 管理内容库 > 管理虚拟机模板 > 将虚拟机模板转化为虚拟机

# 将虚拟机模板转化为虚拟机

如果需要对虚拟机模板进行更新，可以先将虚拟机模板转化为虚拟机，以便更新配置信息，更新完成后可以再将虚拟机转化为虚拟机模板。

## 转化原则

- 仅支持选择虚拟机模板所属的其中一个 SMTX OS（ELF）集群或 SMTX ZBS 集群关联的 SMTX ELF 集群作为转化后的虚拟机的所属集群，并且要求集群与模板的 CPU 架构相同。若满足条件的集群只有一个，则不支持手动选择。
- 当选择 SMTX ELF 集群作为虚拟机的所属集群时，仅支持选择该集群中已存放当前模板的其中一个关联存储作为虚拟机使用的关联存储，若满足条件的关联存储只有一个，则不支持手动选择。
- 若虚拟机模板存放于多个集群中，转化完成后，在虚拟机模板的其他所属集群上存放的模板也将默认被删除，您可以手动取消该设置。
- 转化后的虚拟机的名称默认与模板的名称保持一致，您也可以手动修改，但不可与虚拟机的所属集群中的其他虚拟机重名。
- 若虚拟机模板的源虚拟机已配置 VPC 网卡，则转化后的虚拟机也将自动配置 VPC 网卡。转化得到的 VPC 网卡不设置浮动 IP，MAC 地址和 IP 地址均为自动分配，其余设置均与模板的源虚拟机保持一致。
- 若转化后的虚拟机和虚拟机模板不属于同一个集群，则转化后的虚拟机不启用静态数据加密。

## 转化步骤

1. 在内容库的虚拟机模板列表中选中一个虚拟机模板，在弹出的虚拟机模板详情面板的左上方单击**转化为虚拟机**。
2. 在弹出的**转化为虚拟机**对话框中，设置虚拟机的所属集群和名称，并确认是否自动删除存放在其它集群内的模板，可参考[转化原则](#%E8%BD%AC%E5%8C%96%E5%8E%9F%E5%88%99)进行设置。
3. 单击**转化**。

   在任务中心可以转化的进度和结果。转化成功后，在虚拟机所属集群的虚拟机列表中可以查看到转化完成的虚拟机。

## 管理用户与权限 > 用户、角色与权限的关系

# 用户、角色与权限的关系

每个用户只能对应一个角色，不同角色的用户拥有不同的操作权限。当用户需要执行某一操作时，系统会根据其角色权限来判断是否可以进行相应操作，在不具备相应的操作权限时，仅拥有基本的可读权限。

---

## 管理用户与权限 > 角色管理

# 角色管理

CloudTower 提供两种类型的角色：默认角色和自定义角色。默认角色及其权限说明如下：

| 角色类型 | 权限 |
| --- | --- |
| 超级管理员 | 超级管理员拥有系统的最高权限，包括：  - 创建、删除、修改其他任何类型角色的账号。 - 进行所有的读写操作。 |
| 运维管理员 | 运维管理员拥有和超级管理员相同的权限。 |
| 只读用户 | - 只读用户只拥有基础设施、运维管理和设置的基本可读权限，不能进行任何创建、修改操作。 - 当您部署了备份、复制、Everoute、SKS 或可观测性服务时，只读用户还拥有相应服务的基本可读权限。 |
| 安全审计员 | 安全审计员只拥有查看事件审计信息的权限。 |
| 用户管理员 | 用户管理员拥有以下权限：  - 拥有基础设施、运维管理和设置的基本可读权限。 - 用户与角色管理的操作权限。 - 用户信息的可读权限。 - 当您部署了备份、复制、Everoute、SKS 或可观测性服务时，用户管理员还拥有相应服务的基本可读权限。 |
| 虚拟机使用者 | 虚拟机使用者拥有编辑虚拟机信息、打开控制台、对虚拟机进行电源操作及对虚拟机进行快照管理的权限。 |
| 工作负载集群使用者 | 工作负载集群使用者拥有查看所属工作负载集群列表的权限，具体权限由集群项目内绑定的角色而定。 |

> **说明**：
>
> - 每个 CloudTower 中只能有一个超级管理员角色的用户，由系统在 CloudTower 部署完成后自动生成。其用户名为 root，不可修改；密码由用户在初始化时设置，支持修改。其余角色的用户可以创建多个。
> - 基本可读权限不包含审计信息、用户信息和日志检索信息的可读权限。

默认角色的操作权限不可更改，自定义角色的操作权限可以根据实际需要进行设置。您可以根据需要创建自定义角色，并在创建用户时为用户分配相应角色。

## 管理用户与权限 > 用户管理 > 创建用户

# 创建用户

CloudTower 支持创建本地用户，在创建用户时，您可以为用户分配默认角色或具备特定操作权限的自定义角色以创建具有不同操作权限的用户。

**注意事项**

- CloudTower 仅支持创建本地用户。
- 若您需要创建具备特定操作权限的用户，请先创建具备相应操作权限的自定义角色。
- 若 CloudTower 启用了双因子认证，则需在创建用户时填写相应信息：
  - 若启用邮件认证，需填写电子邮箱；
  - 若启用短信认证，需填写手机号码。为确保信息的准确性及顺利接受消息，建议您在填写手机号码时包含国家或地区代码。

以创建一个角色为**虚拟机使用者**的用户为例，以下为创建用户的操作步骤。

**操作步骤**

1. 在 CloudTower **系统配置**主界面左侧的导航树选择**用户管理**，进入用户管理界面。单击页面右上角的**创建用户**，弹出**创建用户**对话框。
2. 输入姓名、用户名和密码，并选择该用户的角色：虚拟机使用者。
3. 单击**创建**，完成配置。

   完成创建后，该用户可以通过 CloudTower 虚拟机使用者视图对已分配的虚拟机进行自主管理和操作。

## 管理用户与权限 > 用户管理 > 编辑或删除用户

# 编辑或删除用户

CloudTower 支持对已创建用户进行**编辑**或**删除**操作。

**注意事项**

- 不允许修改用户的用户名和创建来源。
- 对于创建来源为 LDAP/AD 和 SSO 的用户，不允许修改密码。
- 编辑 SSO 用户的用户名、邮件和手机号码信息后，SSO 用户下次登录 CloudTower 时，相应字段的信息会被 IdP 的信息覆盖。

**操作步骤**

在 CloudTower **系统配置**主界面左侧的导航树选择**用户管理**，进入用户管理页面。单击用户右侧的 **...**，可选择对用户进行编辑或删除操作。

- **编辑**：可根据需要调整用户的密码、姓名等信息，还可调整非超级管理员用户的角色。
- **删除**：删除该用户。

## 管理高级监控

# 管理高级监控

高级监控能力由单独部署的高级监控应用提供。高级监控应用以虚拟机的形式在 SMTX OS 集群中运行，并使用独立的安装文件进行安装。

由于高级监控应用拥有专属的 CPU、内存和存储资源，因此可以采集全量的监控指标和数据，并确保监控数据不会因为存储空间的限制而被删除。

> **说明**：
>
> - 对于 SMTX OS（ELF）集群，仅支持在 4.0.x、5.0.x 和 5.1.x 版本的集群中部署高级监控，6.0.0 及以上版本的集群不支持部署高级监控。
> - 对于已部署高级监控的集群，若在关联 1.2.0 及以上版本的可观测性服务后执行了数据迁移，高级监控将被自动卸载。
> - 对于 5.0.6 及以上版本的 SMTX OS（VMware ESXi）集群，若集群已关联了可观测性服务，则不支持部署高级监控。

## 管理高级监控 > 为 SMTX OS 集群部署高级监控

# 为 SMTX OS 集群部署高级监控

您可以为单个 SMTX OS 集群部署高级监控，也可选中多个兼容至相同高级监控版本的集群以批量部署。

**前提条件**

- 集群当前未处于初始化过程。
- 若集群使用的虚拟化平台为 VMware ESXi，则需要提前关联 vCenter Server。

**默认推荐配置**

根据集群的节点规模，系统会在高级监控部署界面自动填充推荐的资源配置，具体如下：

| 集群节点数（n） | vCPU | 内存容量 | 数据盘容量 |
| --- | --- | --- | --- |
| n ≤ 4 | 4 | 6 GiB | 60 GiB |
| 5 ≤ n ≤ 31 | n | 1.5 \* n GiB（向上取整） | 15 \* n GiB |
| n ≥ 32 | 32 | 48 GiB | 480 GiB |

**操作步骤**

1. 在 CloudTower **系统配置**主界面左侧的导航树选择**高级监控**。
2. 在**未部署的集群**中选择一个或多个需要部署高级监控的集群，单击**部署**或**批量部署**，弹出**部署高级监控应用**对话框。
3. 根据版本要求，上传相应版本的高级监控镜像文件。
4. 配置如下参数。

   | 字段 | 说明 | 要求 |
   | --- | --- | --- |
   | vCPU 分配 | 为高级监控应用虚拟机分配的 vCPU 数量。 | 每个版本的高级监控应用均有默认的 vCPU、内存及存储容量要求，如不单独修改，则默认使用此分配方式。如进行修改，则修改后的数量和容量不得低于默认要求。 |
   | 内存分配 | 为高级监控应用虚拟机分配的内存容量。 |
   | 数据盘容量 | 为高级监控应用虚拟机分配的数据盘总容量。 |
   | 虚拟机 IP | 为高级监控应用虚拟机设置 IP。 | 高级监控应用将关联虚拟网络，虚拟机 IP 必须处于存储网络的网段中，且不能与已分配的 IP 冲突。 |
5. 单击**部署**，开始部署任务。

2. 在**已部署的集群**中选择需要一个或多个需要升级高级监控的集群，单击**升级**或**批量升级**，弹出**升级高级监控应用**对话框。
3. 根据版本要求，上传相应版本的高级监控镜像文件。
4. 确认已选择集群的版本及当前高级监控版本。
5. 单击**升级**，开始升级任务。

## 管理高级监控 > 调整高级监控虚拟机的资源

# 调整高级监控应用虚拟机的资源

对于已部署高级监控的集群，支持根据实际需求调整部署高级监控应用虚拟机的资源配置。

**前提条件**

集群当前未处于初始化过程。

**注意事项**

为高级监控应用增加 vCPU 或内存需要重启虚拟机，高级监控功能可能会中断几分钟。

**操作步骤**

2. 设置 vCPU、内存、数据盘容量及应用 IP。具体字段要求与部署高级监控相同。
3. 单击**保存**，完成调整。

## 管理高级监控 > 停用高级监控

# 停用高级监控

若已部署高级监控的集群不再需要统计全量监控及报警数据，或者已关联了可观测性服务并执行了数据迁移，可停用高级监控应用。

**前提条件**

- 集群当前未处于初始化过程。
- 若集群关联了可观测性服务，请先迁移监控数据，然后停用高级监控。

**注意事项**

- 停用高级监控后，当基础监控能力不足时，部分数据采集将被停止。
- 停用高级监控操作无法撤回，一旦停用，系统将会清除已采集的数据，请谨慎操作。

**操作步骤**

1. 在**已部署的集群**中选择需要停用高级监控的集群，单击 **...**，选择**停用**。
2. 确认注意事项后，勾选**我确认停用集群的高级监控，并销毁其相关数据**。
3. 单击**停用**，停用高级监控。

## 配置 SMTP 服务器

# 配置 SMTP 服务器

您可在 CloudTower 中配置 SMTP 服务器。配置后的 SMTP 服务器可用于：

- 发送[邮件报警通知](/cloudtower/4.8.0/cloudtower_user_guide/cloudtower_user_guide_021)。
- 发送用户登录 CloudTower 时的[双因子认证邮件](/cloudtower/4.8.0/cloudtower_user_guide/cloudtower_user_guide_084)。
- 发送 Everoute 分布式防火墙的命中计数通知。
- 发送备份与容灾的任务报表通知。

**前提条件**

用户已架设好可用的 SMTP 服务器。

**注意事项**

若 CloudTower 新关联的集群原本配置有 SMTP 服务器，关联完成后其 SMTP 服务器配置会同步展示在 SMTP 服务器配置列表。但如果该配置使用了用户名和密码验证，则此时会提示该配置需要提供密码。可单击列表右侧的 **...**，选择**提供密码**完成此操作。

**操作步骤**

1. 在 CloudTower **系统配置**主界面左侧的导航树选择 **SMTP 服务器**，单击右上角 **+ 配置 SMTP 服务器**，弹出**配置 SMTP 服务器**对话框。
2. 填写以下信息：

   | 参数 | 描述 |
   | --- | --- |
   | 名称 | 填写 SMTP 服务器的名称。 |
   | 描述（选填） | 填写 SMTP 服务器的描述信息。 |
   | 服务器地址 | 填写 SMTP 服务器的地址，可填写 IP 地址或域名。  **说明**：  若 SMTP 服务器的地址填写为域名，则请根据不同使用场景为相应对象配置 DNS 服务器：  - **用于发送邮件报警通知：**   - 报警源为集群内部：请在选择的集群设置页面配置 DNS 服务器。   - 报警源为可观测性服务：请在选择的集群设置页面、可观测性服务和 CloudTower 服务中分别配置 DNS 服务器。 - **用于发送双因子邮件认证：**请在 CloudTower 服务中配置 DNS 服务器。 - **用于发送 Everoute 的安全策略命中统计通知：**请在 Everoute controller 中配置 DNS 服务器。 - **用于发送备份计划和复制计划的任务报表通知：**请在 CloudTower 服务中配置 DNS 服务器。 |
   | 安全模式 | 选择 CloudTower 与 SMTP 服务器连接和进行消息通信的加密方式。包括`无`、`SSL` 及 `STARTTLS` 三种加密方式。 |
   | 端口 | 填写 SMTP 服务器的端口。 默认 SMTP 端口为 25（未加密），587（STARTTLS）和 465（SSL）。 |
   | 用户名 / 密码 | 勾选“使用用户名和密码验证”后填写，用于 SMTP 服务器发送邮件的账号密码。  当安全模式选择`无`时，该项为选填项；否则为必填项。 |
3. 单击**配置**，完成 SMTP 服务器配置。

完成 SMTP 服务器配置后，若该服务器没有被应用于邮件报警通知、双因子认证、Everoute 分布式防火墙的命中计数通知或备份或复制计划的任务报表通知，则您可对其进行编辑和删除操作。

## 管理 CloudTower 高可用

# 管理 CloudTower 高可用

CloudTower 高可用（HA）集群通过主被架构，确保在硬件故障或网络中断时，CloudTower 对基础设施的管理服务能够持续在线，保障业务连续性。

启用 CloudTower 高可用之后，您可以在 **CloudTower 高可用**页面查看高可用集群的基本信息和主被切换记录，实时监控集群中各个节点的运行状况。

> **说明**：
>
> 如需了解 HA 集群的详细架构、部署流程及运维操作，请参考《CloudTower 安装与运维指南》。

## 查看基本信息

**操作步骤**

1. 在 CloudTower **系统配置**页面，单击左侧导航栏中的 **CloudTower 高可用**，进入 **CloudTower 高可用（HA）** 页面。
2. 在**基本信息**页签下展示了 CloudTower 管理 IP，HA 集群各个节点的运行状态和 HA 网络信息。

   | 基本信息 | 描述 |
   | --- | --- |
   | CloudTower 管理 IP | CloudTower 高可用集群对外提供服务的统一 IP。 |
   | 节点信息与运行状态 | 展示集群内所有节点（主动节点、被动节点和仲裁节点）的角色与运行状态，包括： - `正常`：节点服务健康，通信正常。 - `异常`：节点服务异常。 - `未知`：无法获取节点状态。 - `同步中`：（仅被动节点展示）被动节点正在从主动节点同步数据。 |
   | HA 网络信息 | 集群内部通信专用的 HA 网络配置，包括 IP 地址、子网掩码和默认网关。 |

## 查看主被切换记录

当主动节点或其所在环境异常，或主动节点重启时，将触发故障切换。您可以在**主被切换记录**页签下查看 CloudTower 的切换历史。

## 配置 CloudTower 代理

# 配置 CloudTower 代理

在虚拟机跨集群冷迁移或分段迁移，内容库分发虚拟机模板或 ISO 映像场景中，数据需要通过 CloudTower 虚拟机中转，导致数据传输路径变长，且 CloudTower 为数据传输的必经之地，网络带宽和计算资源使其容易成为数据传输的瓶颈，降低数据传输效率。

在上述场景中可以通过在集群中部署 CloudTower 代理提高数据传输效率。部署 CloudTower 代理之后，系统会自动选择最合适的代理用于数据传输，CloudTower 不再是数据传输的必经之地，从而确保 CloudTower 的性能不再成为数据传输的瓶颈。

## 配置 CloudTower 代理 > 上传代理安装包

# 上传代理安装包

**前提条件**

- CloudTower 代理支持部署在 CPU 架构为 Intel x86\_64、Hygon x86\_64 或鲲鹏 AArch64 的 SMTX OS 集群或 SMTX ELF 集群上，请根据实际情况提前获取相应的安装包。
- 本版本的 CloudTower 仅支持部署 1.3.0 及以上版本的 CloudTower 代理，请提前获取相应版本的安装包。

**操作步骤**

1. 在 CloudTower **系统配置**主界面左侧的导航树选择 **CloudTower 代理**，单击右上角**管理安装包**，弹出安装包管理对话框。
2. 单击右上角的 **+ 上传安装包**，弹出上传安装包弹窗，选择格式为 `.tar.gz` 的安装包文件，单击**上传**。

   > **说明**：
   >
   > - 安装包上传完成后，您可在 **CloudTower 代理**界面，单击右上角**管理安装包**，在**安装包管理**界面查看所有已经上传的安装包的信息，包括名称、版本和适用架构。
   > - 当使用该版本的安装包部署的代理均被删除或使用新的安装包升级到更高版本后，允许删除该安装包。

## 配置映射规则

完成服务器的配置之后，系统将测试 CloudTower 与服务器的连通性。若连接正常，则可进行映射规则的配置。通过设置服务器中用户信息与 CloudTower 中用户信息的映射规则，可将服务器中的用户信息导入到 CloudTower，或按设定规则为同步至 CloudTower 中的用户赋予角色。

关于 CloudTower 中的用户信息与角色，可参考[管理用户与权限](/cloudtower/4.8.0/cloudtower_user_guide/cloudtower_user_guide_059)。

**操作步骤**

1. **设置基本信息映射**

   分别填写 LDAP/AD 服务器中姓名、用户名、手机号码（若 CloudTower 已启用短信认证，则该选项为必填项）、电子邮箱（若 CloudTower 已启用邮件认证，则该选项为必填项）的属性，属性的取值将会被分别用作 LDAP/AD 用户在 CloudTower 中的姓名、用户名、手机号码和电子邮箱。

   例如，需将姓名为 Alice Smith 的用户的用户信息从 LDAP/AD 服务器导入 CloudTower，其属性和用户信息之间的对应关系及同步过程如下图所示：

   ![](https://cdn.smartx.com/internal-docs/assets/dc612fe9/cloudtower_user_guide_002.png)

   则基本信息映射配置为：

   ![](https://cdn.smartx.com/internal-docs/assets/dc612fe9/cloudtower_user_guide_003.png)
2. **设置角色映射**

   - **为同步到 CloudTower 的用户分配角色**

     单击**添加映射规则**，为 LDAP/AD 服务器中的用户设置匹配条件，并选择 CloudTower 中的角色。满足匹配条件的 LDAP/AD 用户被同步至 CloudTower 后将被赋予所选择的角色：

     - 默认角色：

       - 运维管理员
       - 只读用户
       - 安全审计员
       - 用户管理员
       - 虚拟机使用者
     - 自定义角色

       > **注意**:
       >
       > 对于两个不同角色，不能填写相同的匹配条件。

     由于 LDAP/AD 服务器可以通过 `ou` 或 `group` 的方式对用户进行分组，在填写匹配条件时，需要确认所筛选用户所属的组是通过 `ou` 还是 `group` 分组，以确定匹配条件的填写格式：

     - 若通过 `ou` 分组，需要获取该组的 RDN 值 rdn value，通过 `rdn:dn:=rdn value` 来进行筛选。

       > **说明**：
       >
       > 上述方法仅适用于 LDAP 服务器。

       例如：

       - 筛选 dev 组内的所有用户，该组的 DN 值为：OU=dev,DC=tower,DC=io。则可以知道，该组的 RDN 值 rdn value 即为 dev，则填写匹配条件：`ou:dn:= dev`。
       - 筛选 dev 组内的单个用户 Betty，该组的 RDN 值为该组所在的组织单位（OU）的 RDN 值，该用户的 DN 值为：CN=Betty,OU=dev,DC=tower,DC=io。则可以知道，该组的 RDN 值 rdn value 即为 dev，则填写匹配条件：`(&(ou:dn:=dev)(cn=Betty))`。
     - 若通过 `group` 分组，则获取该组的 DN 值 groupDn，通过 `memberOf=groupDn` 来筛选。

       > **说明**：
       >
       > 通过 `memberOf= groupDn` 筛选用户时，所筛选用户需具有 `memberOf` 的属性。

       例如：

       - 筛选 chengdu 组内的所有用户，该组的 DN 值为：CN=chengdu,OU=smartx,DC=smartx,DC=local；则填写匹配条件：`memberOf=CN=chengdu,OU=smartx,DC=smartx,DC=local`。
       - 筛选 chengdu 组内的单个用户 Alice，该用户的属性 memberOf 的值为：CN=Domin Admins,CN=chengdu,DC=tower,DC=smartx,DC=smartx,DC=com；则填写匹配条件：`(&(memberOf=CN=Domin Admins,CN=chengdu,DC=tower,DC=smartx,DC=smartx,DC=com)(cn=Alice))`。
   - **对于未匹配映射规则的用户的处理方式**

     对于不属于任何匹配规则，但在基本 DN 范围内的 LDAP/AD 用户，可选择以下其中一种处理方式：

     - **同步用户并分配默认角色**：这些用户会被同步至 CloudTower，并被赋予选择的默认角色。
     - **不同步用户**：这些用户不会被同步至 CloudTower。
3. 单击**保存**，将保存服务器配置并自动将指定范围中的用户同步至 CloudTower。可在**用户管理**界面查看被同步至 CloudTower 的用户，其创建来源为 LDAP/AD。

## 登录与安全管理 > 管理 SSO 登录

# 管理 SSO 登录

单点登录（Single Sign-On，SSO）是一种身份验证机制，通过在身份提供商 (Identity Provider，IdP) 和多个服务提供商（Service Provider，SP）之间建立信任关系，允许用户在登录 SP 时通过 IdP 进行身份验证，从而实现使用一个账户登录多个应用程序或系统，而无需在每个应用程序中重复输入用户名和密码。

CloudTower（SP）支持基于 SAML（Security Assertion Markup Language）协议实现的 SSO。SAML 是一种用于身份验证和授权的开放标准，使用 XML 格式传输用户身份信息，主要用于在不同域之间安全地交换用户身份信息。

当前版本的 CloudTower 仅支持使用 Thales 的 STA（SafeNet Trusted Access）作为 IdP，您可以参考如下步骤配置 Thales 的 SAML 应用来使用 Thales 作为 CloudTower 的单点登录门户。

## 登录与安全管理 > 管理 SSO 登录 > 配置 SSO 登录 > 获取 SP 元数据文件

# 获取 SP 元数据文件

您可以通过如下两种方式获取 SP 元数据文件 `metadata.xml`。该文件包含了 CloudTower 与 SAML 认证相关的信息，如 SP 的唯一标识符、SSO 端点 URL、认证信息、属性声明和绑定类型等信息。

- **通过链接获取**

  `https://tower_url/api/sso/saml/metadata`
- **使用命令行工具 **tacli** 获取**

  在 **tacli** 中输入如下命令获取 SP 元数据文件 `metadata.xml`。

  `./tacli -e https://tower_url -u "username" -p "password" saml download-metadata`

  > **说明**：
  >
  > - `https://tower_url`：需替换为 CloudTower 的实际网址。
  > - `username` 和 `password` 需替换为 CloudTower 超级管理员的用户名和密码。
  > - 执行上述命令后，元数据文件将自动下载至当前执行目录的 `tower-sp-metadata.xml` 中，您也可以通过 `-o/--output` 参数指定输出路径：
  >
  >   `./tacli -e https://tower_url saml download-metadata -o matadata.xml`

        ![](https://cdn.smartx.com/internal-docs/assets/dc612fe9/cloudtower_user_guide_008.png)

## 登录与安全管理 > 管理 SSO 登录 > 配置 SSO 登录 > 注册 IdP 至 CloudTower

# 注册 IdP 至 CloudTower

**操作步骤**

在 **tacli** 命令行工具中输入如下命令，将 IdP 注册至 CloudTower。

`./tacli -e https://example.com -u "username" -p "password" saml register-idp -n "name" -f ./metadata.xml`

> **参数说明**：
>
> - <`-e "https://example.com"`>：待配置的 CloudTower 的实际网址。
> - <`-u "username"`>和<`-p "password"`>：CloudTower 超级管理员的用户名和密码。
> - <`-n "name"`>：自定义 IdP 的名称，如 `Thales`。
> - <`-f metadata.xml`>： IdP 元数据文件路径，`metadata.xml` 需要替换为实际路径。

注册成功后，界面将返回 IdP 的 entityId。后续您将可以使用 SSO 账号登录 CloudTower，详情请参见[登录 CloudTower 的方式](/cloudtower/4.8.0/cloudtower_user_guide/cloudtower_user_guide_008)。

SSO 用户登录 CloudTower 后，该用户将被同步至 CloudTower，创建来源为 `协议名称（IdP 名称）`（如 `SAML(Thales)`），其分配的角色默认为**只读用户**。您可根据需要修改用户信息或删除该用户，详情请参考[编辑或删除用户](/cloudtower/4.8.0/cloudtower_user_guide/cloudtower_user_guide_065)。

# 配置双因子认证登录

双因子认证 (2FA) 是一种身份验证方法，要求用户提供密码和另一个认证因子或者至少提供两个认证因子（代替密码），才能访问网站、应用程序或网络。因此，2FA 可以提高帐户安全性，并更好地保护组织及其用户免遭未经授权的访问。

CloudTower 支持邮件类型和短信类型的双因子认证。开启后，用户在登录 CloudTower 时，除了使用用户名和密码进行身份验证，还需要进一步填写验证邮件或短信中的验证码，两步认证均通过后才能登录。

您可根据需要启用其中一种类型的双因子认证或同时启用。若同时启用，则用户在登录 CloudTower 时，可以任意选择通过邮件或短信的方式获取验证码。

## 登录与安全管理 > 配置双因子认证登录 > 配置短信认证 > 配置短信服务

# 配置短信服务

在启用短信认证前需要配置短信服务（Short Message Service，SMS），用于在登录时发送短信认证码。CloudTower 支持对接腾讯云短信服务，在启用短信认证前请参考如下步骤配置短信服务。

**前提条件**

- 请提前获取 CloudTower 超级管理员的账号和密码。
- 请提前下载并安装 AUTHN CLI 工具（简称 “tacli 工具”）。

**注意事项**

当前版本的 CloudTower 仅支持配置一个短信服务。

**操作步骤**

1. 请参考腾讯云官方文档提前完成[国内短信配置](https://cloud.tencent.com/document/product/382/37745)或[国际/港澳台短信配置](https://cloud.tencent.com/document/product/382/37797)，并获取以下参数：

   - **SDK AppID**：[创建应用](https://cloud.tencent.com/document/product/382/37808)并获取该应用的唯一标识码 **SDK AppID**。
   - **secretId** 和 **secretKey**: 在**访问管理** > **API 密钥管理**页面获取
   - **templateId**: 在**国内短信** > **正文模板管理**页面获取已配置的正文模板 **ID**。
   - **signName**: 在**国内短信** > **正文模板管理**页面获取已配置的**签名内容**。
2. 根据步骤 1 中获得的参数，在本地创建短信服务的元数据文件 `metadata.json`。参考模板如下：

   ```
   {
   "appId": "1400953869",
   "secretId": "<SecretId>",
   "secretKey": "<SecretKey>",
   "templateId": "1950598",
   "signName": "SmartX"
   }
   ```
3. 使用命令行工具 **tacli** 输入以下命令配置短信服务。

   `./tacli -e "https://example.com" -u "username" -p "password" sms create -t tencent -f metadata.json`

   其中：

   - <`-e "https://example.com"`>：待配置的 CloudTower 的实际网址。
   - <`-u "username"`>和<`-p "password"`>：CloudTower 超级管理员的账号和密码。
   - <`-t tencent`>：短信平台的类型，此处配置为腾讯云 tencent。
   - <`-f metadata.json`>：短信服务的元数据文件路径，`metadata.json`需要替换为实际路径。

若配置成功，界面将返回已配置的短信服务的 ID 及配置成功的提示。后续您可根据需要进行如下操作：

- 查看短信服务 ID：

  `./tacli -e "https://example.com" -u "username" -p "password" sms list`
- 更新短信服务：

  当您在腾讯云平台对配置的短信服务进行了更新时，如更新短信模板等，可执行如下命令进行同步更新。

  `./tacli -e "https://example.com" -u "username" -p "password" sms update -i ID -f metadata.json -t tencent`

  其中，<`-i ID`> 为短信服务的 ID，`ID`需要替换为实际 ID。
- 删除短信服务：

  `./tacli -e "https://example.com" -u "username" -p "password" sms delete -i ID`

## 登录与安全管理 > 安全访问管理 > 设置密码安全

# 设置密码安全

CloudTower 提供了密码安全功能以提高用户账号的安全性。您可以通过设置密码复杂度、密码重复限制、密码有效时间和登录次数限制，降低密码被恶意破解或泄漏的风险。

| 密码安全要素 | 设置选项 | 产生的影响 |
| --- | --- | --- |
| 密码复杂度 | - **低复杂度**：要求密码长度不少于 8 位字符 - **中复杂度（默认）**：要求密码长度不少于 8 位字符，且同时包含至少 1 位英文字母和 1 位数字 - **高复杂度**：要求密码长度不少于 12 位字符，且同时包含至少 1 位大写字母、1 位小写字母、1 位数字和 1 位特殊字符 | 在创建新用户或修改已有用户的密码时，需满足密码复杂度要求，否则密码无法设置成功 |
| 首次登录修改初始密码 | 启用/不启用 | 启用后，用户首次登录时，要求强制修改密码。 |
| 密码重复限制 | 启用/不启用 | 启用后，用户更换密码时，新密码不可与最近用过的 5 个密码重复。 |
| 密码有效时间 | 设置有效时长，默认为 90 天 | 为提高用户账号的安全性，建议定期对账号密码进行更新。若用户在密码有效期内未更新密码，则原密码将失效，失效后：  - 若账号处于登录状态，将自动退出登录。 - 若账号未处于登录状态，则下次在登录 CloudTower 输入原密码后，系统将提示密码已超过有效期，并提示设置新密码。  用户在更新密码后，密码有效期将重新开始计时 |
| 限制登录次数 | - **限定周期时间**：统计登录失败次数的时间周期，默认为 60 分钟 - **周期内最大失败次数**：周期内允许的最多登录失败次数，默认为 5 次 | 若用户在**限定周期时间**内输入错误密码的次数超过**最大失败次数**，账户将被锁定 30 分钟。锁定期间，用户将无法登录（即使输入正确的密码） 若需在锁定期内解锁，可通过重置密码方式解决：  - **自我重置**：通过密保问题验证身份，而后重置密码，解锁账户 - **管理员重置**：联系管理员帮助重置密码，解锁账户 |

**注意事项**

创建来源为 LDAP/AD 和 SSO 的用户不受密码安全相关配置的限制。

**操作步骤**

1. 在 CloudTower **系统配置**主界面左侧的导航树选择**密码安全**。
2. 在**密码复杂度**、**密码重复限制**、**密码有效时间**和**登录失败次数限制**配置项修改设置选项。
3. 单击**保存**。

## 登录与安全管理 > 配置密钥管理服务

# 配置密钥管理服务

密钥管理服务（Key Management Service，简称 “KMS”）是用于创建和管理密钥加密密钥（Key Encryption Key，简称 “KEK”）的集中化服务。KMS 通过一个或多个高可用的密钥管理服务器，对所有 KEK 进行统一维护和安全存储，防止密钥泄露，提供高效且安全的数据加密解决方案。

相较于集群内置密钥管理服务，配置的第三方密钥管理服务通过将密钥管理和存储基础设施解耦，从而提供更高层级的安全性与运营管控能力。

KMS 支持 `SM4` 或 `AES-256` 两种加密算法，在创建时需指定至少一种加密算法。若在为集群启用静态数据加密时选择了已创建的第三方 KMS，则在创建存储资源时即可选择 KMS 中指定的算法，用于保护数据的安全，以适应不同的合规性与安全性要求。

关于静态数据加密的详细介绍，请参考《SMTX OS 管理指南》或《SMTX ZBS 管理指南》的**管理静态数据加密**章节。

---

## 登录与安全管理 > 配置密钥管理服务 > 添加密钥管理服务

# 添加密钥管理服务

密钥管理服务支持添加一个或多个密钥管理服务器，请参考下述步骤进行操作。

**前提条件**

- 待添加的密钥管理服务器需为 KMIP 1.1 标准或更高版本。当前支持的密钥管理服务器品牌、型号和版本如下：

  - **Thales**

    - 型号：CipherTrust Manager k170v
    - 版本：2.11.1+9742\_LTS
  - **三未信安**

    - 型号：密钥管理系统(SYT1306)
    - 版本：4.1.0
- 已完成密钥管理服务器的部署和配置，记录密钥管理服务器的 IP 地址、端口信息以及 SM4 算法值。并确保 CloudTower 和使用 KMS 的集群可以访问到该 IP 地址。

**操作步骤**

1. 在 CloudTower **系统配置**主界面左侧的导航树选择**密钥管理服务**，进入密钥管理服务页面。
2. 单击页面右上角的 **+ 添加密钥管理服务**，弹出**添加密钥管理服务**对话框。
3. 在对话框输入以下信息。

   | 参数 | 描述 |
   | --- | --- |
   | 服务名称 | 密钥管理服务的名称 |
   | 加密算法 | 选择至少一种加密算法：`SM4` 或 `AES-256`。选择后，若集群启用静态数据加密时选择了创建的 KMS，则在创建存储资源时可选择指定的加密算法为数据进行加密。  **说明**：  选择 `SM4` 时，需要输入 SM4 算法值。 |
   | 密钥管理服务器 | 添加一个或多个密钥管理服务器，并输入密钥管理服务器的 IP 地址和端口。端口默认值为 5696，支持修改。 |
4. 单击**添加**。添加密钥管理服务后，还需要为该服务建立可信连接，才可以将其用于集群中的静态数据加密。

## 登录与安全管理 > 配置密钥管理服务 > 建立可信连接

# 建立可信连接

在完成添加 KMS 之后，需要通过以下两种方式为 KMS 建立可信连接，否则无法正常为集群提供加密服务。

- **上传证书和密钥**

  通过将密钥管理服务器的证书和密钥上传至 CloudTower 以建立可信连接。
- **创建证书签名请求（CSR）**

  通过创建证书签名请求（CSR）的方式建立可信连接。根据证书签名请求信息生成 CSR。将生成的 CSR 提供给密钥管理服务，让密钥管理服务签署证书，再将签署后的证书上传至 CloudTower 以建立可信连接。

## 上传证书和密钥

**前提条件**

请提前获取密钥管理服务器的证书和私钥文件。

**操作步骤**

1. 在密钥管理服务列表中，单击目标密钥管理服务右侧的 **...**，并选择**建立可信连接** > **上传证书和私钥**，弹出**上传证书和私钥**对话框。
2. 为密钥管理服务上传证书和私钥，请上传正确格式的证书和私钥，并确保证书在有效期内且内容正确。
3. 单击**保存**。

上传后集群在与密钥管理服务建立连接时将使用证书和私钥建立可信连接。

## 创建证书签名请求

**操作步骤**

1. 在密钥管理服务列表中，单击目标密钥管理服务右侧的 **...**，并选择**建立可信连接** > **生成 CSR**，弹出**生成 CSR**对话框。
2. 填写以下信息。

   | 参数 | 描述 | 示例 |
   | --- | --- | --- |
   | 组织 | 公司名称或部分名称等。 | `SmartX` |
   | 部门 | 组织内部的部门名称。 | `dev` |
   | 城市 | 组织所在的城市。 | `Shenzhen` |
   | 省份 | 组织所在的省份。 | `Guangdong` |
   | 国家 | 组织所在的国家。需填写 2 位 ISO 标准国家代码。 | `CN` |
   | 邮箱 | 联系人的电子邮箱。 | `xxx@smartx.com` |
   | 密钥算法 | CSR 请求的加密算法，可选 SM2 或 RSA。 | `SM2` |
3. 单击**生成 CSR**，系统将对信息进行校验，校验成功后将弹出**导出 CSR** 对话框。
4. 单击**复制**或**下载**，将生成的 CSR 上传至密钥管理服务，获取签名后的证书。
5. 在 CloudTower 的密钥管理服务列表中，单击目标密钥管理服务右侧的 **...**，选择**建立可信连接** > **上传已签名的证书**，弹出**上传密钥管理服务签名的证书**对话框。
6. 将步骤 4 中获取的签名后的证书上传至 CloudTower，并单击**保存**。

## 管理回收站

# 管理回收站

为了避免误删而导致数据的丢失，CloudTower 提供了回收站功能，支持将以下资源对象移动至回收站：

- SMTX OS (ELF) 集群和 SMTX ELF 集群中的虚拟机
- 6.3.0 及以上版本的 SMTX OS（VMware ESXi）集群的 NFS 文件
- 6.3.0 及以上版本的 SMTX OS（ELF）集群中的 LUN
- 5.7.0 及以上版本 SMTX ZBS 集群中的 LUN 和 Namespace

开启回收站的集群可以根据设置的回收策略对资源对象保留一定时间，但超过设定时间则会被永久删除。在回收站保留资源对象的期间，可以在回收站中恢复资源对象。

---

## 管理回收站 > 回收站与其规则设置

# 回收站与其规则设置

CloudTower 支持为以下集群启用回收站功能：

- SMTX OS (ELF) 集群
- SMTX ELF 集群
- 6.3.0 及以上版本的 SMTX OS（VMware ESXi）集群
- 5.7.0 及以上版本的 SMTX ZBS 集群

## 默认规则与集群特例规则

回收站规则分为**默认规则**和**集群特例规则**：

- **默认规则**：资源对象移至回收站时默认遵循的规则，对于未设置特例规则的集群，将遵循默认规则。
- **集群特例规则**：为集群单独设置的规则，对于已设置特例规则的集群，将优先遵循特例规则；当特例规则删除后，才会遵循默认规则。

## 规则设置与变更规则的影响

**设置默认规则**

1. 在 CloudTower **回收站**页面单击左侧的**回收站规则**进入回收站规则页面。
2. 单击**默认规则**右侧的**编辑**按钮，设置默认规则参数。
3. 单击**保存**，完成设置。

**设置集群特例规则**

1. 在 CloudTower **回收站**页面单击左侧的**回收站规则**进入回收站规则页面。
2. 设置特例规则参数。
   - 若要创建集群特例规则，请单击集群特例规则右侧的**创建特例规则**按钮。
   - 若需要编辑或删除已有集群特例规则，请单击目标规则右侧的 **...** 后选择**编辑**或**删除**。

> **注意**：
>
> 若删除了集群特例规则，集群中不满足默认规则的资源将被永久删除。

**回收站规则参数及变更影响**

| 规则项 | 描述 | 编辑操作 | 变更规则后的影响 |
| --- | --- | --- | --- |
| 启用回收站 | 回收站的启用状态。CloudTower 安装部署后，默认启用回收站。 | 从启用变更为不启用 | 回收站中的全部资源对象将被永久删除。 |
| 从不启用变更为启用 | - |
| 保留天数 | 资源对象在回收站中保留的天数。系统默认保留天数为 30 天。资源对象将在预期删除日的 24:00 被永久删除。 | 缩短保留天数 | 回收站中已超过修改后保留天数的资源对象将被永久删除。 |
| 延长保留天数 | - |

# 管理回收站内的资源

将集群中的虚拟机、LUN、Namespace 或 NFS 文件移至回收站之后，在设置的保留天数之内支持恢复回收站中的资源或永久删除回收站中的资源。

## 管理回收站 > 管理回收站内的资源 > 恢复回收站中的资源

# 恢复回收站中的资源

在设置的回收站保留时间内，可以在回收站中恢复资源，一旦超过设定时间，回收站中的资源将自动永久删除。

> **说明**：
>
> 仅支持恢复虚拟机、LUN 和 Namespace。

**前提条件**

资源被移至回收站的时长不超过集群设置的资源在回收站中的保留天数。

**操作步骤**

1. 在 CloudTower 的**回收站**页面左侧导航栏单击**虚拟机**、**LUN** 或 **Namespace**，分别查看回收站内的相应资源。
2. 在资源列表中勾选待恢复的一个或多个资源对象，单击列表上方的**恢复**按钮。
3. 在弹出的**恢复虚拟机**、**恢复 LUN** 或**恢复 Namespace**对话框中单击**恢复**按钮。

---

## 管理回收站 > 管理回收站内的资源 > 永久删除回收站中的资源

# 永久删除回收站中的资源

被移动至回收站的资源超过设置的保留期限后将被自动永久删除，您也可以提前进行手动清理。您可在**虚拟机**、**LUN**、**Namespace** 或 **NFS 文件**页面勾选一个或多个待删除的资源对象，并单击页面右上角的**永久删除**进行删除。

若您要清空回收站中的资源，您也可以通过单击**虚拟机**、**LUN**、**Namespace** 或 **NFS 文件**页面右上方的**一键清空**，将回收站中的相应资源对象一键永久删除。

## 管理事件 > 查看事件

# 查看事件

超级管理员、运维管理员、安全审计员及拥有**事件审计信息管理**的用户可以查看事件审计信息以便追溯集群故障原因或其他运维事件。

事件审计信息包括用户事件和系统事件，CloudTower 的**事件**界面展示了所有事件信息，您可在**用户事件**或**系统事件**页签下单击某个事件，查看该事件的详细信息。

- **用户事件**：展示所有用户的全部操作记录。
- **系统事件**：展示虚拟机高可用重建信息、虚拟机因操作系统故障重启信息、系统自动触发的任务信息、或未知用户发起的任务信息（例如在主机上插拔 USB）。

  > **说明**：
  >
  > 对于虚拟机因操作系统故障重启信息，您可单击该事件右侧的 **...**，并单击**下载**将记录操作系统报错信息的截图下载至本地。

事件信息只支持查看，不允许修改或删除。当需要备份事件记录时，可以将所有用户事件或系统事件导出到本地存储，导出的文件格式为 `.csv`。

## 管理事件 > 设置事件保留策略

# 设置事件保留策略

CloudTower 支持设置事件保留策略。事件保留策略默认关闭，关闭时，事件保存的时间和条目数量无上限，可能会对存储空间造成压力。启用后，您可以自定义事件的保留天数，超出该天数的事件将被删除，从而减少存储占用。

**前提条件**

仅**超级管理员**、**运维管理员**或分配了**事件审计配置管理**权限的自定义角色支持设置事件保留策略，请确保当前账户具有相应操作权限，若没有，请提前联系管理员获取。

**操作步骤**

1. 在 CloudTower 系统配置主界面左侧的导航树选择**事件**，单击**设置**进入设置界面。
2. 在**保留策略**字段下单击**启用保留策略**按钮，并设置事件保留范围，要求为 180 天及以上。
3. 单击**保存**。

## 管理事件 > 设置事件传输

# 设置事件传输

CloudTower 支持将审计事件通过 Syslog 服务器导出至第三方平台，从而实现事件的集中管理。您可配置一个或多个 Syslog 服务器以用于传输。

**前提条件**

- 您已架设好可用的 Syslog 服务器。
- 确保 CloudTower 虚拟机的管理 IP 地址可以访问 Syslog 服务器的 IP 地址。
- 请确保当前账户拥有**配置 Syslog** 的权限，若没有，请联系管理员进行获取。

**注意事项**

- 配置 Syslog 服务器之后，CloudTower 仅传输新生成的所有类型的事件。
- CloudTower 支持使用同一个 Syslog 服务器接收日志信息和事件信息，要求在分别配置 Syslog 服务器用于日志传输和事件传输时配置相同的服务器地址。

**操作步骤**

1. 在 CloudTower 系统配置主界面左侧的导航树选择**系统配置**，单击 **Syslog 服务器**，单击右上角的 **+ 配置 Syslog 服务器 > 用于事件传输**，弹出**配置 Syslog 服务器**的对话框。
2. 填写如下信息：

   | 参数 | 说明 | 示例 |
   | --- | --- | --- |
   | 名称 | 自定义 Syslog 服务器的名称。 | `Syslog-1` |
   | 服务器地址 | Syslog 服务器的地址，可填写 IP 或域名。 | `192.168.1.1` |
   | 端口 | Syslog 服务器的接收端口。 | `514` |
   | 协议 | 进行事件传输的协议类型，可选 `UDP` 和 `TCP`。 | `UDP` |
   | 语言 | 设置传输事件的语言。支持选择`中文`或`英文`。 | `中文` |
   | 启用状态 | Syslog 服务器的启用状态，分为已启用和未启用。 | `已启用` |
3. 单击**保存**按钮，完成 Syslog 服务器的配置。

完成配置之后，您可在 **Syslog 服务器**界面查看所有已配置的 Syslog 服务器的信息，并根据需要编辑 Syslog 服务器的信息或删除 Syslog 服务器。

若 Syslog 服务器的启用状态为`已启用`，则 CloudTower 将开始向 Syslog 服务器传输事件信息，已启用的 Syslog 服务器信息将展示在**事件 > 设置**页面的**事件传输**字段下。

> **说明**：
>
> 若因网络异常或 Syslog 服务器故障等原因导致事件传输中断，则故障恢复后，事件将从中断点继续传输。

## 移除 SmartX 集群

# 移除 SmartX 集群

将集群从 CloudTower 中移除后，集群将不再受 CloudTower 管理，但集群内的资源不会受到影响。

**前提条件**

当前用户需具有**集群关联管理**的操作权限。

**注意事项**

在以下场景中，不支持将集群从 CloudTower 中移除：

- 集群已关联 Everoute 服务或可观测性服务。
- 集群中存在 SKS 工作集群节点虚拟机。
- 集群中存在副本虚拟机或副本卷。
- 该集群为复制计划的目标集群。
- 集群中部署了文件存储集群。
- 集群中存在系统服务虚拟机：
  - CloudTower 虚拟机
  - CloudTower 代理虚拟机
  - 备份/复制服务虚拟机
  - Everoute Controller 虚拟机
  - registry 虚拟机
  - SKS 管控集群节点虚拟机
  - 可观测性服务虚拟机
  - 亚信安全防护服务虚拟机 SVM
- 集群类型为 SMTX ELF 集群，且该集群中存在虚拟机或虚拟卷。
- 待移除的集群为 SMTX ELF 集群的关联存储集群，且 SMTX ELF 集群中存在虚拟机或虚拟卷。

**操作步骤**

1. 在 CloudTower **集群**主页面单击**集群**页签，该页签下展示了 CloudTower 关联的所有集群。
2. 单击待移除集群右侧的 **...**，选择**移除集群**，弹出**移除集群**对话框。
3. 单击**确认**。
