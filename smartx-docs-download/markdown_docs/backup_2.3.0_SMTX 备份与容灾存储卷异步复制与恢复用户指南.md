# backup/2.3.0/SMTX 备份与容灾存储卷异步复制与恢复用户指南
## 关于本文档

# 关于本文档

本文档介绍了如何使用 SMTX 备份与容灾的复制服务对 SMTX ZBS 块存储集群中的存储卷进行异步复制和故障处理。

参考本文档使用存储卷异步复制相关功能前，请您先参考《SMTX 备份与容灾-安装与运维指南》完成复制服务的部署。

## 概述

# 概述

异步复制是一种高效容灾的方案。该功能可将 SMTX ZBS 块存储集群中的存储卷（iSCSI LUN ）异步复制到另一个 SMTX ZBS 块存储集群中，利用快照技术周期性地将原卷的增量数据同步至副本卷。利用定期生成的恢复点，当原卷发生故障时，可通过副本卷快速恢复业务，实现分钟级恢复点目标（RPO）。

异步复制同时支持在原卷发生故障时，进行故障转移、故障恢复和永久故障转移。

## 概述 > 功能简介

# 功能简介

复制服务可以对存储卷进行异步复制和故障处理，具体功能如下：

- **存储卷异步复制**

  将 SMTX ZBS 块存储集群中的存储卷异步复制到另一个 SMTX ZBS 块存储集群中，并在目标集群创建对应的副本卷。通过配置异步复制计划，系统可周期性地将原卷的增量数据同步给副本卷，并生成恢复点。

  ![](https://cdn.smartx.com/internal-docs/assets/7d3b1166/volume_replication_user_guide_01.png)
- **故障转移**

  故障转移是指当原卷发生故障时，可使用副本卷接替原卷继续提供服务。根据应用的场景不同，可以分为计划外故障转移和计划内故障转移。

  ![](https://cdn.smartx.com/internal-docs/assets/7d3b1166/volume_replication_user_guide_02.png)

  - **计划外故障转移**

    紧急故障转移是生产环境发生意外或故障时立即启动的应急措施，通常应用于突发性灾难，如硬件故障或网络中断等紧急情况，为业务提供快速、安全、可靠的灾备保障。当原存储卷发生故障时，复制服务可将副本卷回滚至指定的恢复点状态。这种故障转移方案能够在灾难发生时实现分钟级别的 RPO 和 RTO，使业务中断影响最小化。
  - **计划内故障转移**

    计划内故障转移是根据预先设定的计划进行的有序转移。在进行灾备演练、计划中的数据中心维护或升级、或预计将发生灾难的情况下，复制服务可以执行计划内故障转移。首先，系统将对原卷进行一次复制，将最新的数据同步至副本卷，然后将原卷切换至副本卷。
- **故障恢复**

  故障恢复是在进行故障转移后，当原卷恢复正常时，将故障转移期间副本卷产生的数据增量同步回原卷，可以确保故障转移期间产生的数据变化不会丢失。

  ![](https://cdn.smartx.com/internal-docs/assets/7d3b1166/volume_replication_user_guide_03.png)
- **永久故障转移**

  永久故障转移永久故障转移是在进行故障转移后，当原卷无法恢复时，将原卷永久切换到副本卷，并使用副本卷持续提供服务。

  ![](https://cdn.smartx.com/internal-docs/assets/7d3b1166/volume_replication_user_guide_04.png)

## 概述 > 应用场景

# 应用场景

存储卷异步复制是一种关键的数据保护策略，主要用于防止业务数据在生产集群或生产站点发生故障时丢失，影响业务的正常运行。

主要应用场景如下：

- **灾难恢复**

  防止因自然或人为灾害、勒索软件攻击、恐怖事件等导致的大规模数据损坏和丢失。一旦灾难发生，可切换至灾备集群的存储卷提供数据服务，并迅速恢复业务运行。
- **灾备演练**

  结合复制和故障转移功能，实现零恢复点目标（0 RPO）的业务切换。在进行灾备演练时，能够模拟灾难发生后的快速恢复过程，确保灾备方案的有效性和可靠性。
- **数据复用**

  利用集群原生的快照重建功能实现数据复用。通过从副本卷的快照重建存储卷进行开发、测试、故障排除等活动，避免对生产环境造成任何影响。数据复用功能能够在不影响主要业务的前提下，为开发人员和测试团队提供高效且隔离的环境，加快开发和测试过程，同时确保生产环境的稳定性和安全性。

## 概述 > 产品特性

# 产品特性

存储卷异步复制具有如下特性：

- 易用性：复制策略自动化和执行记录可视化，简化运维和管理。
- 灵活性：提供多样的异步复制选项，支持自定义复制周期、复制窗口、保留策略，满足不同类型业务的 SLA 要求。
- 高可用性：当复制服务所在节点发生故障时，复制服务可自动在其他节点拉起并继续提供服务。
- 负载控制：支持自定义同时执行的异步复制任务数量、任务重试次数、任务传输速率限制对复制服务进行负载控制。

## 相关概念

# 相关概念

本章解释了手册中使用的专业术语。在阅读过程中，如果遇到不熟悉的概念，可在此快速查找其定义和详细解释。

**复制服务**

复制服务负责管理异步复制计划，并执行存储卷的异步复制、故障转移、故障恢复和永久故障转移等任务。

根据关联的集群不同，复制服务分为目标端复制服务和源端复制服务。

- **目标端复制服务**

  与目标集群关联的复制服务，主要负责执行异步复制计划、故障转移和管理复制资源，在故障恢复时从目标集群读取数据并传输给源端复制服务。目标端复制服务负责确保复制数据的同步性，保障副本卷与原卷的数据一致性。
- **源端复制服务**

  与源集群关联的复制服务，主要负责从源集群读取数据并传输到目标端复制服务，并在故障恢复时接收目标端复制服务传输的数据并将数据写入源集群。

**站点**

站点是指由单一 CloudTower 管理的一个故障域，可以包含一个或多个集群和数据中心。每个站点的资源都由其专属的 CloudTower 进行管理，且在地理位置、硬件设备和软件服务等方面完全隔离，不与其他站点共享任何资源。

站点可以分为以下两种类型：

- **源站点**：通常为实际运行业务的站点。复制服务将源站点的原卷复制到目标站点的副本卷，实现数据的同步。
- **目标站点**：通常为用于容灾的站点，用于存储从源站点复制过来的副本卷。

**站内复制**

源集群和目标集群位于同一站点。当原卷发生故障时，可切换使用站点内其他集群中的副本卷，实现集群级别的容灾。若管理站点的 CloudTower 发生故障，故障转移将失败。

**跨站点复制**

源集群和目标集群位于不同站点。当源站点内的原卷不可用时，无论管理源站点的 CloudTower 是否发生故障，均可切换至目标站点的副本卷，实现站点级别的容灾。

跨站点复制包括以下两种类型：

- **入站复制**：目标站点为当前站点，源站点为其他关联站点。
- **出站复制**：源站点为当前站点，目标站点为其他关联站点。

**源集群**

通常为生产集群，承载着实际运行的业务应用和数据，是原存储卷所属的集群，位于源站点。

**目标集群**

通常为用于容灾的集群，用于存储从源集群复制过来的副本卷，位于目标站点。

**存储卷**

SMTX ZBS 块存储集群对外提供的最基本的存储单元。复制服务支持对 iSCSI LUN 进行复制和故障处理。

对存储卷进行复制时，涉及两种存储卷角色：

- **原卷**

  需要进行容灾保护的存储卷，负责实际承载业务数据的读写操作。
- **副本卷**

  通过复制计划创建的原卷的完整副本。副本卷通常用于容灾，当原卷不可用时，可以接替原卷提供业务数据。

**复制资源**

指复制对象、异步复制计划和恢复点。

**复制对象**

一个原卷及其对应的副本卷称为一组复制对象。

**异步复制计划**

用于规划和安排数据的异步复制操作。通过异步复制计划，用户可以定义复制对象、复制类型，复制站点、目标集群、复制服务、复制周期、复制窗口、恢复点保留策略和副本卷等信息。

**复制周期**

异步复制计划自动执行的频率和时间，用于指定数据复制的时间间隔。较短的复制周期可以实现更小的 RPO。例如，如果复制周期设置为 15 分钟，那么在发生灾难性事件时，最多可能丢失 15 分钟的数据。

**复制恢复点**

每次为存储卷执行异步复制、故障转移（立即生成恢复点模式）或故障恢复时生成的数据状态，以快照形式保存在副本卷上。

恢复点可以使存储卷在需要时快速回滚到特定的数据状态，例如在灾难恢复场景中将存储卷恢复到某个先前的时间点，以恢复业务运行。通过定期生成恢复点，可以确保副本卷的数据是最新的。

**复制窗口**

允许异步复制计划自动执行的时间范围。

**全量复制**

用于将存储卷的所有数据复制到副本卷，从而确保副本卷与原卷完全一致。全量复制通常发生在初次复制，用于建立存储卷的初始副本，并为后续的增量复制提供基准。

**增量复制**

复制自上次复制后发生变化的数据，能够持续跟踪和同步数据源的变化，保障数据的准确性和一致性。

**故障处理**

故障发生后可进行的恢复操作，包括故障转移、故障恢复和永久故障转移。

- **故障转移**

  应对存储卷不可用时的操作。当原卷不可用时，将副本卷回滚到与指定恢复点一致的状态，并切换至使用副本卷。通过故障转移，系统可以在短时间内将服务恢复到故障发生前的状态，保障业务的连续性和稳定性。
- **故障恢复**

  当原卷恢复正常使用后，将副本卷在故障转移期间变化的数据回传至原卷，然后切换回使用原卷。故障恢复旨在将原卷和副本卷之间的数据进行同步，确保原卷可以在故障解决后继续提供服务。
- **永久故障转移**

  当原卷无法从故障中恢复时，副本卷正式接替原卷的服务角色，以确保业务的持续运行。

**RPO** (Recovery Point Objective)

数据恢复点目标，指在故障转移场景中允许数据损失的时间范围。RPO 以时间为度量单位，标示了从最近执行的复制时间点到发生故障时的时间跨度。较小的 RPO 值表示系统要求在发生故障时允许的数据损失较少。较大的 RPO 值意味着在故障发生时系统可以容忍较长时间范围内的数据丢失。

**RTO** (Recovery Time Objective)

数据恢复时间目标，指业务系统从崩溃到恢复运行所能容忍的时间。较短的 RTO 值表示业务系统要求在发生故障时能够迅速恢复，较长的 RTO 值则意味着业务系统可以容忍更长的恢复时间。

**SMTX ZBS 集群**

SMTX ZBS 集群包含块存储集群和文件存储集群，分别提供块存储和文件存储服务。复制服务可以部署在 SMTX ZBS 块存储集群上，并为块存储集群提供存储卷复制和故障处理服务。

**CloudTower**

SmartX 多集群资源集中管理平台，旨在管理站点内数据中心及跨数据中心的多个 SmartX 集群和系统服务，并提供完整和标准的 RESTful API 和多种语言的 SDK。

## 使用前规划 > 规划复制方向

# 规划复制方向

使用复制服务对存储卷进行容灾保护时，根据不同的业务需求和站点布局，可选择站内复制或跨站点复制两种类型。

- **站内复制**

  站内复制通常用于集群级别的容灾保护。原卷和副本卷位于同一站点内不同集群中，当原卷不可用时，可以通过切换到站内其他集群中的副本卷，以保持业务的持续运行。该复制策略适用于对数据保护的需求集中在单个站点内的场景。

  ![](https://cdn.smartx.com/internal-docs/assets/7d3b1166/volume_replication_user_guide_05.png)
- **跨站点复制**

  跨站点复制可提供站点级别的容灾保护，当一个站点不可用时，可以立即切换到位于其他地理位置的站点，确保业务的高可用性。该复制策略适用于在不同地理位置部署了多个数据中心，需确保跨地域容灾能力的场景。

  ![](https://cdn.smartx.com/internal-docs/assets/7d3b1166/volume_replication_user_guide_06.png)

## 使用前规划 > 规划站点

# 规划站点

使用跨站点复制策略时，请从如下几方面规划源站点和目标站点。

- **确定站点的定位**

  根据业务需求，明确源站点和目标站点的角色。通常将生产站点作为复制的源站点进行保护，将灾备站点作为目标站点。一个生产站点最多支持配置 5 个灾备站点，以提升数据冗余和灾备能力。
- **确保站点隔离**

  源站点和目标站点的分布应充分考虑地理隔离，以应对地震、火灾等自然灾害和其他潜在风险。同时还应确保各站点硬件、管理平台（CloudTower）和软件服务的独立性。通过多层次的隔离避免所有站点同时发生故障，以确保业务的高可用性和数据安全。
- **选择合适的复制架构**

  根据实际业务需要和数据保护策略选择合适的复制架构。

  - **单向的生产-灾备架构**

    此架构适合需要明确区分业务生产和灾备的场景。可将生产站点作为源站点，专注于正常的业务运行；将灾备站点作为目标站点，专注于数据恢复和灾难保护。
  - **双向的互为灾备架构**

    此架构下，两个站点相互备份对方数据，任一站点发生故障时，另一站点可以迅速接管业务。

## 使用前规划 > 规划集群

# 规划集群

规划源集群和目标集群时，请从如下几个方面综合考虑：

- **确定集群的定位**

  根据业务需求，将生产集群作为复制的源集群进行保护，将灾备集群作为复制的目标集群。
- **选择合适的复制架构**

  根据实际业务需要和数据保护策略选择合适的复制架构。

  - **单向的生产 - 灾备架构**

    若需要实现业务生产和灾备的隔离，可选择此架构，将生产集群作为源集群，灾备集群作为目标集群。
  - **双向的互为灾备架构**

    若两个集群需各自运行业务且互为灾备集群，可选择此架构，两集群之间互相复制，并各自使用部分资源为对方存储副本。

## 使用前规划 > 规划异步复制计划

# 规划异步复制计划

在创建异步复制计划前，请从以下几个方面规划：

- **计划保护的存储卷**

  每个异步复制计划中可包含一个或多个存储卷，这些存储卷共享相同的复制策略。因此，建议根据不同的数据保护要求，对存储卷进行分类，将具有相同目标集群、复制周期和保留策略的存储卷置于同一个异步复制计划中，避免重复定义相同的复制策略，提高运维和管理效率。
- **复制周期**

  通过设定复制周期，复制服务可以对存储卷自动化进行周期性的复制。用户可以根据业务系统的 RPO（Recovery Point Objective）目标，正确设置数据复制的时间间隔。最小 RPO 为 15 分钟，因此可根据业务需求合理选择复制周期，以确保数据的及时恢复能力。
- **恢复点保留策略**

  每次执行完异步复制计划后，复制服务会为计划中的每个存储卷创建一个恢复点，在需要时，可通过恢复点将原卷故障转移到对应时间点状态的副本卷。

  恢复点保留策略可以定义保留的恢复点数量，复制服务将根据恢复点保留策略，自动清理超出范围的恢复点。

## 使用前规划 > 规划许可单元的分配

# 规划许可单元的分配

SMTX 备份与容灾的软件许可由备份与复制两个功能模块共享。许可单元根据所保护对象类型及所用功能进行消耗，具体规则如下：

- **SMTX OS（ELF）集群虚拟机**

  - 备份功能：每备份一个虚拟机，消耗 1​ 个许可单元。
  - 异步复制功能：每异步复制一个虚拟机，消耗 2​ 个许可单元。
  - 同步复制功能：每同步复制一个虚拟机，消耗 4​ 个许可单元。
- **SMTX ZBS 块存储集群存储卷**：每异步复制 1 TiB 的存储卷，消耗 2​ 个许可单元。

请您根据实际的业务需求，综合考虑要使用的功能模块，需要保护的虚拟机和存储卷数量、以及未来业务的扩展等，提前规划许可单元的分配。

## 管理站点（仅跨站点复制） > 添加关联站点

# 添加关联站点

完成复制服务的部署后，请为当前站点添加关联站点。

**前提条件**

- 仅超级管理员和运维管理员有关联站点的操作权限，因此在当前站点请使用超级管理员或运维管理员账号登录。
- 提前获取待添加站点所属的 CloudTower 的 IP 地址、超级管理员或任一运维管理员账号及密码。关联站点时仅支持使用站点所属的 CloudTower 的本地账户，不支持 LDAP 账户。
- 每个站点最多可关联 5 个站点，因此请确保当前站点和待添加站点已关联的站点数均小于 5。
- 为避免兼容性问题，建议源站点和目标站点关联相同版本的 CloudTower。

**操作步骤**

1. 登录 CloudTower，在备份与容灾界面中，单击**设置** > **站点管理**进入站点管理界面。
2. 单击界面右上角 **+ 添加关联站点**，弹出**添加关联站点**对话框。
3. 输入以下信息：

   | 参数 | 描述 |
   | --- | --- |
   | 站点名称 | 自定义待关联站点的名称。 |
   | CloudTower IP | 输入待关联站点所属 CloudTower 的 IP 地址。 |
   | 用户名 | 输入待关联站点 CloudTower 中超级管理员或任意运维管理员账号的用户名。 |
   | 密码 | 输入 CloudTower 用户名对应的密码。 |
4. 单击**添加**。
5. （可选）若关联站点启用了双因子认证，您还需查看该站点用户信息中填写的邮箱，将 6 位数验证码填写至输入框中，并单击**验证**。验证通过后即可完成关联。

   完成添加后，站点管理界面将展示已关联站点的信息，包括站点名称、和当前站点的连接状态和 CloudTower IP。

## 管理站点（仅跨站点复制） > 编辑关联站点

# 编辑关联站点

当关联站点的 CloudTower IP 发生变更时，站点的连接状态将显示连接异常。您可以通过编辑操作更新关联站点最新的 CloudTower IP 信息，以重新和站点进行关联。

- 待编辑站点相关的出站复制计划中不存在存储卷正在进行复制任务。
- 待编辑站点和当前站点未处于`连接中`状态。

**操作步骤**

1. 登录 CloudTower，在备份与容灾界面中，单击**设置** > **站点管理**进入站点管理界面。
2. 单击某一站点右侧的 **...**，选择**编辑**，弹出**编辑关联站点**对话框。
3. 编辑站点名称或 CloudTower IP。
4. 单击**保存**。

## 管理复制计划 > 创建异步复制计划 > 设置基本信息

# 设置基本信息

   | 入站 | 将外站点存储卷复制至本站点。请选择复制的源站点。 |
   | 出站 | 将本站点存储卷复制至外站点。请选择复制的目标站点。 |
3. 设置目标集群和目标端复制服务。

   | 配置项 | 说明 |

   > **说明**:
   >
   > 不支持复制以下存储卷：
   >
   > - 所属的 iSCSI Target 已被 SMTX ELF 集群使用的存储卷；
   > - 已加入所选目标集群的其他异步复制计划的存储卷；
   > - 所在集群与 CloudTower 连接异常的存储卷；
   > - 已加入一致性组的存储卷；
   > - 异步复制计划创建的副本卷。

## 管理复制计划 > 创建异步复制计划 > 设置复制周期和时间窗口

# 设置复制周期和时间窗口

设置异步复制计划自动执行的周期，包括设置复制周期和复制时间窗口。设置完成后单击**下一步**。

1. **设置复制周期**

   | 执行类型 | 默认周期 | 支持设置周期 | 说明 |
   | --- | --- | --- | --- |
   | 分钟 | 15 分钟 | 15 ~ 59 分钟 | 系统将从每日 00:00 开始计时，按照设置的时间周期执行复制任务。 |
   | 小时 | 1 小时 | 1 ~ 23 小时 |
   | 天 | 1 天 | 1 ~ 30 天 | 需要为执行当天设置一个或多个执行时间，两个执行时间最小间隔为 15 分钟。 |
   | 星期 | 每星期一 | 星期一 ~ 星期日 | 可选择在每周中的一天或几天执行，并需要为执行当天设置一个或多个执行时间，两个执行时间最小间隔为 15 分钟。 |
2. **设置复制时间窗口**

   选择是否启用仅允许在时间窗口内执行复制任务，默认不启用。启用后需设置时间窗口，要求如下：

   - 复制时间窗口开始和结束的间隔需不少于 15 分钟。
   - 复制周期按天和星期设置时，复制时间窗口的范围必须包含复制周期设定的执行时间，且窗口结束时间不能为自动执行的时间点。
   - 默认情况下，若复制在超出复制窗口的结束时间后仍未完成，系统将停止复制任务，并删除对应存储卷的快照和恢复点信息，若为首次复制，还将删除副本卷。勾选“**若复制不能在时间窗口内完成，继续完成当前复制**” 可取消此限制。
   > **说明**:
   >
   > - 在复制周期的自动执行时间点，如果当前复制计划正在执行，本次自动执行将跳过。
   > - 复制周期按分钟和小时设置时，若自动执行时间点刚好在时间窗口的结束时间，则不会执行复制任务。
   > - 在复制周期的自动执行时间点，如果异步复制计划中存在存储卷当前正在执行故障转移任务、故障恢复或永久故障转移任务或处于`故障转移中`状态，对应存储卷的本次复制将跳过执行。

# 设置副本卷

1. **设置副本卷的命名规则**

   支持通过添加前缀或后缀的方式为副本卷命名，也可选择与原卷名称保持一致。
2. **设置副本卷的冗余策略**

   支持为副本卷配置与原卷不同的冗余策略。

   | 配置项 | 说明 |
   | --- | --- |
   | 副本 | 可选择配置为 2 副本或 3 副本。若目标集群启用了双活特性，则副本数仅支持选择 3 副本。 |
   | 纠删码 | 若目标集群满足如下条件，可选择纠删码作为冗余策略，并选择合适的纠删码配比。 - 集群健康节点数不小于 4； - 集群为分层存储模式； - 集群未启用双活特性。 |
3. **设置副本卷的置备类型**

   可选择厚置备或精简置备。

## 管理复制计划 > 手动执行异步复制计划

# 手动执行异步复制计划

已创建的异步复制计划默认将按照设定的复制周期自动执行，也可由用户手动执行。

异步复制计划将由计划中指定的目标端复制服务执行，在执行时，如果原卷所在集群关联了多个复制服务，目标端复制服务将随机选择一个连通的复制服务作为源端复制服务，以实现复制时的数据传输。

**前提条件**

- 异步复制计划当前没有在执行的复制任务。
- 当前没有对异步复制计划进行编辑操作。

**操作步骤**

1. 在**异步复制计划**列表选择待执行的复制计划，在列表右侧单击 **...**，选择**手动执行**。您也可以单击复制计划名称，进入详情页，选择**手动执行**。
2. 在弹出的页面中单击**手动执行**。复制任务创建后支持在任务中心停止。

   - 对于等待创建副本卷的存储卷，将执行全量复制。
   - 对于已创建副本卷的存储卷，将执行增量复制。

> **说明**：
>
> - 在对存储卷进行增量复制时，如果副本卷 I/O 限制、白名单配置信息更新失败，复制服务仍然会继续同步数据，并在数据同步成功后创建一个恢复点，该恢复点和原卷的部分配置项不一致。复制服务将在下次执行复制任务时再次同步最新的配置和存储卷数据以保证一致性。
> - 若异步复制计划中存在存储卷处于以下状态，则本次复制任务将跳过该存储卷的复制：
>   - 存储卷正在执行故障转移、故障恢复、永久故障转移或处于`故障转移中`状态。
>   - 存储卷被删除或所属集群从 CloudTower 中移除。

## 管理复制计划 > 暂停或继续异步复制计划

# 暂停或继续异步复制计划

已创建的异步复制计划默认将按照设定的复制周期自动执行。您也可暂停异步复制计划，使当前复制周期暂时失效。暂停后的异步复制计划不会自动执行。复制计划暂停后可以通过**继续复制计划**恢复生效。

在**异步复制计划**列表选择要暂停或继续的复制计划，在列表右侧或异步复制计划详情页单击 **...**，选择**暂停复制计划**或**继续复制计划**，可暂停或继续异步复制计划。

- **基本信息**

  展示异步复制计划的描述、源站点（仅入站复制计划）、目标站点（仅出站复制计划）、目标集群、目标端复制服务、复制周期和恢复点保留策略，以及副本对象的命名规则、冗余策略和置备方式。
- **复制时间**

  展示异步复制计划下次自动执行时间、最近一次执行时间（包括手动执行和自动执行）、最近一次执行结果（包括手动执行和自动执行）和复制窗口。
- **执行记录**

  展示当前异步复制计划的执行情况，不同的颜色表示不同的状况。

  - 绿色表示当日执行过异步复制计划且除跳过的复制任务外其余复制任务均执行成功。
  - 红色表示当日执行过异步复制计划且除跳过的复制任务外存在复制失败的复制任务，悬浮鼠标可查看异步复制计划执行失败的时间和复制失败的存储卷数量。
  - 蓝色表示当日有待执行异步复制计划但仍未执行，悬浮鼠标可查看异步复制计划将要执行的时间。
  - 灰色表示当日没有计划要执行的异步复制计划。
- **复制对象**

  - 展示当前异步复制计划包含的所有存储卷和对应的副本卷，以及每个存储卷所有的恢复点。
  - 展示从当前异步复制计划中移除但是保留恢复点的存储卷和对应的副本卷，以及每个存储卷所有的恢复点。
- **事件**

  展示当前异步复制计划相关的用户事件和系统事件。在跨站点场景下，系统事件仅记录在目标端站点所关联的 CloudTower 界面。

## 故障处理

# 故障处理

故障处理包括**故障转移**、**故障恢复**和**永久故障转移**。

> **说明**：
>
> - 故障处理操作需在目标站点执行。
> - 仅支持对站内复制计划和入站复制计划的复制对象进行故障处理。

---

## 故障处理 > 故障转移

# 故障转移

当原卷不可用时，可通过故障转移将原卷切换至副本卷来继续提供服务。

您可以对多个存储卷批量进行故障转移，也可以对单个存储卷进行故障转移。

**前提条件**

待进行故障转移的存储卷需满足以下条件：

- 存储卷存在对应的副本卷。副本卷所属集群和 CloudTower 连接正常；
- 当存储卷存在多个副本卷时，需选择其中一个副本卷进行故障转移，请确保其对应的复制对象的故障转移状态为`就绪`，且其余复制对象不处于`故障转移中`、`故障恢复中`和`永久故障转移中`状态；
- 存储卷当前没有正在执行的故障转移任务。

---

## 故障处理 > 故障转移 > 存储卷批量故障转移

# 存储卷批量故障转移

执行以下步骤对多个存储卷进行批量故障转移。

**操作步骤**

1. 登录 **CloudTower**，在**备份与容灾**界面，选择**异步复制计划**，单击页面右上角**故障处理**，并选择**故障转移**。
2. 选择复制对象类型为**存储卷**。
3. 如果存在入站复制对象，需选择待进行故障转移的存储卷所属的站点作为源站点。
4. 选择需进行故障转移的存储卷。支持按照 LUN 的名称、所属 iSCSI Target、所属集群和所属异步复制计划进行搜索。

   > **说明**：
   >
   > 若存储卷存在多个副本卷，仅支持选择其中一个副本卷进行故障转移。
5. 为每个存储卷选择恢复点，用于将副本卷恢复到对应时间点的状态。

   - **立即生成恢复点**

     在故障转移任务创建成功后，将先对原卷执行复制，再基于最新的恢复点进行故障转移。选择立即生成恢复点时，请确保原卷在故障转移期间未进行新的 I/O。

     当原卷满足下述条件时，允许选择**立即生成恢复点**。

     - 源集群与 CloudTower 连接正常；
     - 原卷存在；
     - 原卷所属复制计划未被删除；
     - 源站点连接状态为`已连接`。
   - **选择历史恢复点**

     当原卷不允许选择**立即生成恢复点**时，请选择一个已创建的历史恢复点。在故障转移任务创建成功后，复制服务将根据选择的历史恢复点进行故障转移。

     > **注意**：
     >
     > 选择历史恢复点进行故障转移后，若后续进行故障恢复，复制服务将会把副本卷在故障转移期间产生的数据同步回原卷。此过程中，原卷自历史恢复点至故障转移时刻的所有数据将被覆盖，导致该时间段内的数据丢失。
6. 单击**创建**。

故障转移任务创建后，副本卷对应的复制对象的故障转移状态即变更为`故障转移中`。

> **说明**：
>
> - 故障转移期间，如果存储卷关联的复制计划自动执行，该存储卷的复制任务将会跳过。
> - 故障转移期间，若手动执行该存储卷所属的复制计划，该存储卷的复制任务将跳过。
> - 每个存储卷的故障转移任务相互独立，无依赖关系。如果某个存储卷故障转移失败，可以单独为该存储卷重新创建故障转移任务。

## 故障处理 > 故障恢复

# 故障恢复

故障转移期间，若原卷恢复正常，可通过故障恢复将故障转移期间副本卷产生的数据同步回原卷，并恢复至由原卷提供服务。

您可以对单个存储卷进行故障恢复，也可以对多个存储卷批量进行故障恢复。

**前提条件**

存储卷进行故障恢复需满足以下条件：

- 复制对象的故障转移状态为`故障转移中`。
- 原卷存在。
- 对于入站复制对象，源站点需处于`已连接`状态。
- 源集群和目标集群连接正常。

**注意事项**

进行故障恢复时，请确保副本卷在故障恢复期间未进行新的 I/O，以确保副本数据能完整同步回原卷。

# 存储卷批量故障恢复

   > **说明**：
   >
   > 仅当复制计划中存在`故障转移中`状态的复制对象时展示**故障恢复**的操作入口。
2. 选择复制对象类型为**存储卷**。
3. 如果存在入站复制对象，需选择要故障恢复的存储卷所属的站点作为源站点。

创建故障恢复任务后，复制对象即进入`故障恢复中`状态。各个复制服务会将副本卷在故障转移期间产生的数据变化增量同步回原卷，在所有数据传输完成后，复制对象的故障转移状态变更为`就绪`状态。

> **说明**：
>
> - 在存储卷故障恢复期间，若其关联的异步复制计划自动执行或手动执行，该存储卷的复制任务将会跳过。

> - 若故障恢复任务失败，则复制对象的故障转移状态将由`故障恢复中`变更为`故障转移中`。

3. 如果存在入站复制对象，需选择要永久故障转移的存储卷所属的站点作为源站点。
4. 选择需进行永久故障转移的存储卷。支持按照 LUN 的名称、所属 iSCSI Target、所属集群和所属异步复制计划进行搜索。
5. 选择是否保留副本卷的快照，默认保留。
6. 单击**创建**。

创建永久故障转移任务后，复制对象即进入`永久故障转移中`状态。复制服务会将原卷从异步复制计划中移除，将副本卷转化为普通存储卷，并按照设定保留或删除副本卷的快照。

> **说明**：
>
> - 在存储卷永久故障转移期间，如果其关联的异步复制计划自动执行或手动执行，该存储卷的复制任务将会跳过。
> - 每个存储卷的永久故障转移任务相互独立，无依赖关系，执行失败的永久故障转移任务将自动重试。
> - 若永久故障转移任务失败，则复制对象的故障转移状态将由`永久故障转移中`变更为`故障转移中`。

## 故障处理 > 永久故障转移 > 单个存储卷永久故障转移

   您也可以在**副本卷**页面，选择一个待进行永久故障转移的复制对象，单击右侧的 **...**，选择**永久故障转移**，弹出**创建永久故障转移任务**对话框。
4. 选择是否保留副本卷的快照，默认保留。
5. 单击**创建**。

创建永久故障转移任务后，复制对象即进入`永久故障转移中`状态。复制服务会将原卷从复制计划中移除，将副本卷转化为普通卷，并按照设定保留或删除副本卷的快照。

## 管理复制对象 > 删除存储卷复制对象

# 删除存储卷复制对象

如果复制对象不属于任何异步复制计划，删除复制对象将删除复制对象所有恢复点、副本卷及相关的快照。

如果复制对象属于某一异步复制计划，删除复制对象会将复制对象从复制计划移除，并删除复制对象所有恢复点、副本卷及相关的快照。

> **注意**：
>
> 该操作不可撤回，请谨慎操作。

**前提条件**

- 复制对象所在集群关联的复制服务未处于`升级中`或`异常`状态；
- 复制对象处于`就绪`状态，且当前没有正在执行的故障转移任务。

**操作步骤**

1. 在**备份与容灾**界面，进入**副本卷**页面，在复制对象列表选择待删除的复制对象，单击右侧 **...**，选择**删除**。
2. 在弹出的对话框确认提示信息后，单击**删除**，完成操作。

## 管理恢复点 > 删除恢复点

# 删除恢复点

根据恢复点的执行类型，恢复点可被自动删除或由用户手动删除。

- 自动执行异步复制计划生成的恢复点会根据恢复点保留策略进行保存，超过保留范围的恢复点会被自动删除，用户也可手动删除。
- 其他情况下生成的恢复点默认永久保留，仅支持手动删除。

> **注意**：
>
> 执行复制任务依赖最新的恢复点，因此无法删除最新的恢复点。该操作不可撤回，请谨慎操作。

**前提条件**

手动删除恢复点时，需保证复制对象处于`就绪`状态，且没有正在进行的故障转移任务。

**操作步骤**

1. 在**备份与容灾**界面，选择**异步复制计划**，单击某一复制计划名称，进入详情页。
2. 选择**复制对象**，单击存储卷名称，在右侧页面通过以下方式删除恢复点。

   - 单击某一恢复点右侧 **...**，选择**删除**，可删除该恢复点。
   - 勾选恢复点左侧复选框，单击右上角**删除**，可删除一个或多个恢复点。
   - 单击页面右侧 **...**，选择**删除全部恢复点**，可删除该复制对象除最新恢复点外的所有恢复点。
3. 在弹出的页面中，确认提示信息，单击删除按钮，完成删除。

## 查看存储卷复制概况

# 查看存储卷复制概况

在**备份与容灾**的**概览**页面中可查看被保护存储卷的总体情况、复制计划的数量及状态、复制计划的执行记录概况、副本卷及复制对象的状态信息、目标集群的存储空间信息、复制服务的健康状况，以及关联站点等信息。执行记录、复制计划、副本卷页签包含站内、入站、出站复制计划的信息。

- **被保护资源**

  选择**存储卷**标签，可查看 CloudTower 管理的所有 SMTX ZBS 集群中未保护的存储卷数量和被保护的存储卷数量，以及被保护存储卷数量总和占存储卷总数的比例。
- **执行记录**

  选择**存储卷**标签，可查看所有存储卷异步复制计划的执行记录。绿色表示任务执行成功，红色表示任务执行失败。
- **复制计划**

  选择**按复制类型**标签，可查看站内、入站、出站异步复制计划数量，以及生效中和已暂停的异步复制计划数量。

  选择**按对象类型**标签，可查看被保护的存储卷的数量。
- **副本卷**

  查看站内、入站和出站异步复制计划已创建的副本卷数量，以及各种故障转移状态下的副本卷数量。
- **集群存储空间**

  查看所有目标集群的总容量、容量使用率和已使用容量。
- **复制服务**

  查看所有复制服务的信息，包括复制服务名称、状态（包括正在部署、可升级、正在升级、正在卸载、异常，未展示表示状态正常）、关联的集群数量、复制计划数量。
- **关联站点**

  查看当前站点已关联的站点，关联站点中的复制计划数量，以及关联站点的连接状态。

  > **注意**：
  >
  > 当站点间连接状态异常时，将无法获取出站复制计划的相关信息。
