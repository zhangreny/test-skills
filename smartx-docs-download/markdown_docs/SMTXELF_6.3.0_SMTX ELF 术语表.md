# SMTXELF/6.3.0/SMTX ELF 术语表
## 关于本文档

# 关于本文档

本文档介绍了 SMTX ELF 相关文档中出现的专业术语。在阅读过程中，如果遇到不熟悉的术语，可在此快速查找定义和详细解释。

## 文档更新信息

# 文档更新信息

**2026-02-04：配合 SMTX ELF 6.3.0 正式发布**

相较于 SMTX ELF 6.2.0，本版本主要更新了以下内容：

- **A-E**：新增 ERSPAN。
- **F-J**：新增 IGMP、镜像出口网络。

## A-E

# A-E

CCP
:   CCP（Crypto Coprocessor，加密协处理器）用于硬件加速加密操作。

CloudTower
:   SmartX 多集群资源集中管理平台，可统一管理多种类型的 SmartX 集群及其资源。

COW
:   COW（Copy-On-Write，写时复制），一种延迟数据复制的机制，在数据进行修改前不立即创建副本，仅在实际修改时才创建副本。该技术常用于快照场景，可显著节省空间并提高写入效率。

常驻缓存模式
:   将虚拟卷中的数据保留在缓存层，从而获得更稳定的高性能。

超融合
:   一种 IT 基础架构，通过将计算虚拟化、分布式存储、网络及各类运维管理等服务融合部署在通用的 x86\_64 或 AArch64 架构的服务器上，以简化基础设施类资源的交付、上线和运维。

计算节点
:   集群中运行了虚拟化组件的节点。

存储接入网络
:   用于集群内各节点间数据交换，以及计算集群与存储集群之间数据交换的网络。

ELF
:   SmartX 基于 KVM 开发的原生虚拟化计算平台，提供虚拟机生命周期管理、电源操作、高可用、冷热迁移等基本功能，并能与 SmartX 设计研发的分布式块存储服务（ZBS）结合，提供完整的超融合虚拟化解决方案。

ERSPAN
:   ERSPAN（Encapsulated Remote Switch Port Analyzer）是一种支持跨网络、跨三层传输的远程流量镜像技术，能够在不同设备或网络之间远程复制和传输流量，适用于复杂网络环境中的流量监控与分析。

## F-j

# F-J

分离部署模式
:   一种网络部署模式。在该模式下，集群的每个系统网络分别独占一台虚拟机分布式交换机。

副本
:   分布式存储系统为数据创建的数据冗余拷贝，同一份数据会在多个节点上保存多个副本，确保数据高可用性和容错能力。

GPU 驱动
:   GPU 驱动部署在使用 GPU 的虚拟机中，使虚拟机可以使用 GPU 设备。

管理网络
:   用于管理集群的网络。

HCT
:   HCT（Hygon Cryptographic Technology，海光密码技术）是一套密码算法加速技术软件开发套件，能够提供软硬件结合的国密算法加密服务。

厚置备
:   在创建存储卷时立即为其分配全部空间，即使存储卷尚未写入，也占用全部分配空间的置备方式。

IGMP
:   ​IGMP（Internet Group Management Protocol，互联网组管理协议）​用于 IPv4 网络中主机与路由器之间的组播成员管理。

IOMMU
:   IOMMU（Input–Output Memory Management Unit，输入输出内存管理单元）是内存控制器中的一个组件，用于将设备的虚拟地址转换为物理地址。

IOPS
:   IOPS（Input/Output Operations Per Second，每秒输入/输出操作次数）表示存储设备每秒的读写次数。

iSCSI
:   iSCSI（Internet Small Computer System Interface）是一种基于 IP 网络的存储协议，用于将 SCSI 命令封装在 IP 数据包中进行传输。它允许客户端通过网络访问远程存储设备，具备部署简单、成本低、可扩展性强等优势。

镜像出口网络
:   当集群使用流量镜像功能时，专门用于将封装后的镜像流量发送至集群外部的网络。

精简置备
:   按需动态分配空间，仅在实际写入时占用存储空间的置备方式。

纠删码
:   一种数据冗余机制。它通过特定的算法对 K 个原始数据块（Data Block）进行处理，生成 M 个校验块（Parity Block）。在损坏数据块的数量不超过 M 的情况下，可通过纠删码算法，利用 K 个可用的数据块和校验块即可重建出损坏的数据块，实现数据的恢复和容错能力。

## K-O

# K-O

快照
:   对虚拟机在某个时间点的状态进行记录和保存的功能，可用于数据回滚。

快照对象
:   受快照计划保护的资源。

快照计划
:   对快照的创建和保留进行定期自动化管理的策略。

快照组
:   执行快照计划后生成的一组快照的集合。

LACP
:   LACP（Link Aggregation Control Protocol，链路聚合控制协议）是一种基于 IEEE802.3ad 标准的实现链路动态聚合的协议。链路聚合组中启用了 LACP 协议的成员端口通过发送 LACP PDU 报文进行交互，双方对能够发送和接收报文的端口明确并达成一致，确定承担业务流量的链路。

MDEV
:   MDEV（Mini udev）是一种在虚拟化环境中将虚拟设备直接传递给虚拟机的方法。

MLD
:   MLD（Multicast Listener Discovery，组播侦听者发现协议）​用于 IPv6 网络中主机与路由器之间的组播成员管理。

## P-T

# P-T

QoS
:   QoS（Quality of Service，服务质量）是一种根据业务重要性和性能需求，对计算、网络等资源进行分配、限制和优先调度的技术，在资源有限的情况下保障关键业务的服务质量。

迁移网络
:   为虚拟机迁移单独创建的网络。

SDR
:   Sensor Data Record，传感器数据记录。

SEL
:   System Event Log，系统事件日志。

SMTX ELF 集群
:   SMTX OS 集群属于逻辑概念。在实际生产环境中，一个 SMTX ELF 集群由至少 3 个运行了 SMTX ELF 软件的节点通过网络互连组成。

SMTX ELF 节点
:   指运行了 SMTX ELF 软件的单个物理服务器，是组成 SMTX ELF 集群的基本单位。

SR-IOV
:   SR-IOV (Single Root - I/O Virtualization，单根 I/O 虚拟化) 是一种基于硬件的虚拟化解决方案，将一个硬件设备虚拟化为多个独立的虚拟设备的技术，从而提高性能，降低硬件成本。

## U-Z

# U-Z

vGPU 驱动
:   vGPU 驱动随 SMTX OS 发布，是经过 SmartX 改造和适配的 NVIDIA Virtual GPU Manager。它部署在虚拟化平台，为虚拟机提供 vGPU 功能。

VLAN
:   一种将二层网络在逻辑上划分为多个不同的虚拟局域网，从而实现二层网络隔离的技术。

网口绑定
:   将多个物理网口绑定形成一个逻辑端口，以提高物理网口的高可靠性和总带宽。

物理功能（PF）
:   物理功能（Physical Function，PF）用于支持和管理 SR-IOV 功能，是全功能的 PCIe 功能，可像其他任何 PCIe 设备一样进行发现、管理和处理。

系统网络
:   主机自身服务使用的虚拟网络。根据用途划分，系统网络可以分为管理网络、存储接入网络和迁移网络。

虚拟分布式交换机（VDS）
:   虚拟分布式交换机（Virtual Distributed Switch，VDS）是 SMTX ELF 利用虚拟化技术将一组物理连通的物理网口转化为虚拟化层的分布式交换机，为主机之间提供网络连接，并为处理各类型系统流量及业务流量提供基础。

虚拟功能（VF）
:   虚拟功能（Virtual Function，VF）一种轻量级的 PCIe 功能，与 PF 以及与同一 PF 关联的其他 VF 共享一个或多个物理资源。

虚拟机放置组
:   是对虚拟机放置和迁移的约束和规则，以便虚拟机能够在适当的节点上运行。

虚拟机网络
:   用于虚拟机业务通信和对外提供服务的网络，可以实现不同虚拟机之间的逻辑隔离，以提高网络安全性。

虚拟机组
:   虚拟机组属于逻辑概念，将具有特定属性的虚拟机进行归类，便于统一查看和管理。

置备方式
:   一种存储空间分配策略，包括厚置备和精简置备两种方式。

主节点
:   集群中运行元数据服务的节点。
