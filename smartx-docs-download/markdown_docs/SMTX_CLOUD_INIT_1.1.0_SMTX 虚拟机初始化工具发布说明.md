# SMTX_CLOUD_INIT/1.1.0/SMTX 虚拟机初始化工具发布说明

# 关于本文档

本文档介绍了 SMTX 虚拟机初始化工具 1.1.0 的版本特性说明，版本配套信息、版本已知问题和产品配套文档。

## 版本更新说明

# 版本更新说明

## 新增

- 支持操作系统为统信 UOS 的虚拟机。
- 支持操作系统为 Fedora 的虚拟机。
- 支持 AArch64 架构下操作系统为 Rocky 9.0 的虚拟机。
- 支持操作系统为中标麒麟高级服务器的虚拟机。
- 支持操作系统为 TencentOS Sever的虚拟机。
- 支持操作系统为 OPENSUSE 和 SUSE Linux Enterprise Server 的虚拟机。
- 支持操作系统为 AnolisOS 的虚拟机。
- 支持清除 Linux 虚拟机的 Machine ID。

## 修复问题

- 修复了使用虚拟机初始化工具为 Windows 虚拟机配置静态 IP 可能不生效的问题。
- 修复了虚拟机初始化工具卸载后重装可能失败的问题。
- 修复了使用安装了虚拟机初始化工具的 Windows 虚拟机模板创建虚拟机时可能出现虚拟机启动超时的问题。
- 修复了在制作虚拟机初始化工具模板时，误将模板开机后，需要手动清理虚拟机初始化实例的问题。

## 版本配套关系

# 版本配套关系

## 与虚拟机操作系统版本配套说明

本版本的 SMTX 虚拟机初始化工具支持操作系统版本和固件类型在《SMTX 虚拟机初始化工具兼容性列表》上的虚拟机。

对于操作系统版本和固件类型不在上述兼容性列表，但在 SMTX 虚拟机服务兼容性列表的虚拟机，请根据 SMTX OS 集群或 SMTX ELF 集群版本参考对应的《cloud-init 虚拟机初始化配置指南》或《cloudbase-init 虚拟机初始化配置指南》进行配置。

## 与虚拟机平台配套关系

SMTX 虚拟机初始化工具仅支持安装在 ELF 平台的虚拟机上，并支持 Intel x86\_64、Hygon x86\_64 和鲲鹏 AArch64 的服务器架构。

## 版本的已知问题

# 版本的已知问题

- 在 Ubuntu 操作系统上使用 Netplan 配置网络时， `/etc/netplan` 目录下如果存在多个配置文件，系统将按照顺序依次执行，可能会导致配置冲突，影响网络设置效果。
- 对于使用已安装虚拟机初始化工具的虚拟机模板创建的 Windows 7 和 Windows server 2016 虚拟机，开机输入密码后会系统进行三次重启，重启结束后可正常使用。
- 通过已安装虚拟机初始化工具的虚拟机模板创建操作系统为 Windows 的虚拟机时不支持配置静态路由，需在创建后为虚拟机手动配置。
- 创建操作系统为 Windows 的虚拟机时，如未选择通过 cloudbase-init 配置网络或配置 DHCP 网络，则 DNS Server 配置不生效。
- 创建操作系统为 Windows 的虚拟机时可能因虚拟机的服务启动速度较快导致初始化配置不生效。可参考《SMTX 虚拟机初始化工具用户指南》的 Q&A 中提供的方法进行解决。
