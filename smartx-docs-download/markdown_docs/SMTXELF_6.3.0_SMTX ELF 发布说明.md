## 版本更新说明

# 版本更新说明

## 新增

### 虚拟化

- 虚拟机高可用（HA）
  - 支持虚拟机 HA 成功或失败触发报警。
  - 支持为虚拟机网络故障场景配置 HA 动作为热迁移或重建虚拟机。
  - 支持为挂载了 SR-IOV 直通网卡、vGPU 以及直通方式为 MDEV 的加密控制器的虚拟机启用 HA 功能。
- 支持为虚拟网卡配置队列数量。
- 支持通过命令行为虚拟机启用 IO Thread 与配置磁盘参数。
- 支持批量选择虚拟机进行 SMTX 虚拟机工具一键升级。
- 支持为虚拟机快照编辑名称和描述。
- 支持为集群启用热迁移加密。
- 支持深度安全防护功能，可对接亚信无代理杀毒平台，提供防病毒及深度包检测能力。

### 网络

- 支持流量镜像，可通过 ERSPAN 隧道将指定网络中的流量复制并发送至远程分析平台。支持灵活选择使用专用的镜像出口网络或复用管理网络作为隧道源。
- 虚拟分布式交换机支持开启 IGMP/MLD 侦听，开启后可侦听 IGMP/MLD 报文，并动态维护组播组成员，以实现组播流量的精准转发，降低网络负载。
- 通过调整部分网络报警阈值参数，缩短探测周期与触发延迟，提升了报警触发的灵敏度与时效性。
- 支持通过命令行配置 vnet 网卡队列长度。

### 运维管理

- 支持统一修改集群中所有主机的 SSH 服务端口。
- 支持通过界面将集群系统时间强制同步为 NTP 服务器时间，并提供更清晰的 NTP 服务器错误提示信息。
- 支持通过命令行修改集群时区。
- 支持通过命令行设置网络服务独占 CPU 资源或与系统服务共享 CPU 资源。
- 支持 RPM 的 GPG 签名验证，提升了系统软件包的完整性与安全性。
- 新增内存分组出现内存压力事件的报警。
- 新增网口收发流量负载压力过高的报警。
- 新增对硬件 RAID 成员物理盘状态的监控和展示。

### 内核

- 内核版本升级至 5.10.0-247.0.0。
- 支持海思 SP680 系列网卡。
- Hygon x86\_64 平台新增驱动支持：sxe 驱动支持 Linkdata 1160-2X 网卡。
- 增加主机 TSC 校准失败的告警。
- 升级 MegaRAID 驱动至 07.735.03.00 版本。
- 支持沐创 RNPGbe N500 系列网卡。

## 改进

### 虚拟化

- 优化跨集群冷迁移、跨集群分段迁移以及克隆虚拟机机制，迁移或克隆后虚拟网卡的 PCI 地址保持不变。
- 优化跨集群热迁移机制，迁移时仅传输有效数据，缩短迁移时间。
- 优化鲲鹏 AArch64 架构下热迁移机制，支持在 CPU 型号不同的节点间进行热迁移。
- 提高鲲鹏 AArch64 架构下虚拟机规格上限，vCPU 最大支持为 512，内存最大支持为 1TiB。
- 优化 OVF 导入机制，在目标集群为 AArch64 架构时，自动将虚拟机磁盘的总线类型从 IDE 修正为 SCSI，避免因架构不兼容导致导入失败。
- 优化 Hygon x86\_64 架构下 CPU 兼容性机制，当虚拟机 CPU 兼容性为主机透传时，默认不指定 Vendor ID，以满足合规要求。
- 优化 HA 机制：在部分故障场景下，若虚拟机暂停无法恢复，系统将先强制关闭虚拟机，再执行本地 HA 恢复。
- 优化 ISO 上传机制，降低任务超时的概率。

### 网络

- 支持通过 network-tool 将 default 虚拟机网络迁移至其他虚拟分布式交换机。
- 增加主机连接跟踪表使用率过高的预报警和连接跟踪表已满的实时报警，提示用户对连接情况进行处理，避免影响主机中的网络通信。

### 运维管理

- 调整 timemachine 相关告警规则，删除不适用于快照计划的规则并调整部分规则的描述。
- 增加升级预检查项，阻止许可过期的集群进行升级。
- 优化了 Mellanox 网卡启用 SR-IOV 功能的流程，减少了重启主机的次数。
- 优化了 `fluent-bit` 服务状态的检查逻辑，避免因检查不通过导致集群升级失败。

## 修复问题

### 虚拟化

- 修复了虚拟机挂载了启用常驻缓存的磁盘可能导致仅迁移计算失败的问题。
- 修复了虚拟机热迁移期间，源端主机因运维操作或故障触发 `libvirtd` 服务重启，可能导致热迁移及后续虚拟机运维操作失败的问题。
- 修复了虚拟机热迁移期间，对 vCPU 运行效率持续进行高水平限制，可能导致虚拟机业务异常的问题。
- 修复了跨集群热迁移时，目标端虚拟机文件可能被用于清理残留文件的定时任务误删，导致迁移失败的问题。
- 修复了集群中定时资源删除任务周期较长可能导致虚拟机跨集群迁移后迁回失败的问题。
- 修复了极少数情况下，节点故障前触发但未完成初始化的虚拟机任务可能导致虚拟机 HA 失败的问题。
- 修复了 AMD x86\_64 架构集群中，CPU 兼容性配置列表可能非预期存在 Dhyana 选项的问题。
- 修复了磁盘锁访问 ZBS 服务失败时，可能释放未初始化内存导致 libvirt 非预期重启的问题。
- 修复了启用 CPU 独占的虚拟机冷迁移至异构节点后，移入回收站可能失败的问题。
- 修复了通过 SMTX 虚拟机工具编辑 Ubuntu 或 Debian 虚拟机的网卡 IP 地址时，可能导致虚拟机原有 DNS 配置变化的问题。
- 修复了虚拟机开机后可能因已有 DHCP 网卡未自动连接，导致添加新网卡失败的问题。
- 修复了通过 SMTX 虚拟机工具为虚拟机配置 DNS 服务器后，虚拟机重启可能导致网关配置无法生效的问题。
- 修复了跨集群热迁移可能出现进度计算不准确的问题。
- 修复了将虚拟机网卡分配的虚拟机网络更改为关联同一 VDS 的其他虚拟机网络后，虚拟机热迁移可能导致 Everoute 对该虚拟机进行的限速与流量镜像等功能失效的问题。
- 修复了通过 SMTX 虚拟机工具为虚拟机配置静态 IP 可能因操作系统 nmcli 工具版本较低导致失败的问题。
- 修复了当虚拟机挂载了同时启用常驻缓存与数据静态加密的虚拟卷时，跨集群热迁移可能失败的问题。
- 修复了所属关联存储为分层模式时，虚拟机监控中磁盘逻辑空间展示不正确的问题。
- 修复了在集群执行一键提升副本数操作后，使用该集群内虚拟机克隆生成的模板，通过快速拷贝创建虚拟机可能失败的问题。
- 修复了多网卡虚拟机在使用 SMTX 虚拟机工具配置静态 IP 后重启，IP 地址可能不符合预期的问题。
- 修复了使用 SMTX 虚拟机工具编辑 DNS 服务器地址后，可能因部分配置项被错误清除导致短域名配置失败的问题。
- 修复了 Linux 虚拟机通过 SMTX 虚拟机工具配置 DNS 后可能非预期启动 `NetworkManager` 服务的问题。
- 修复了虚拟机网络配置文件中 HWADDR 与网卡 MAC 地址不匹配时，编辑网卡 IP 地址可能会失败的问题。
- 修复了从 6.0.0 版本升级或迁移而来的虚拟机，在 CPU 兼容性跟随集群设置为 Cooperlake、Icelake-Client 或 Icelake-Server 时，首次 HA 重建可能失败的问题。
- 修复了虚拟机跨集群热迁移期间修改集群部分设置可能导致虚拟机进入非预期状态的问题。
- 修复了 QEMU 中的 USB 1.0 模拟设备优化可能导致虚拟机在无 VNC 连接时 CD-ROM 读取性能下降的问题。
- 修复了跨集群热迁移在特定的网络故障下可能长时间卡死且无法取消的问题。
- 修复了集群中 Windows 虚拟机启用 Windows 优化后可能由于 TSC 重置不正确导致重启卡住的问题。
- 修复了当集群资源不足时，故障节点快速恢复后 HA 可能无法自动恢复虚拟机的问题。
- 修复了当主机 `elf-vm-monitor` 服务未运行时进行虚拟机迁移，可能导致 HA 非预期触发虚拟机重建的问题。
- 修复了 SMTX 虚拟机工具执行命令异常后可能无法正常工作的问题。
- 修复了集群中虚拟机不满足放置组策略触发报警后，即使迁移使其满足放置组策略，报警可能仍无法自动解决或重复误报的问题。

### 网络

- 修复了网口绑定模式为 `balance-tcp` 时，网口连通性检测结果异常的问题。
- 修复了由于健康检查服务不稳定，导致频繁出现网口异常报警的问题。
- 通过调整监控图表中的丢包率计算方式，修复了丢包率误报的问题。

### 运维管理

- 修复了集群扩容后，新节点升级工具版本与现有节点不一致，导致后续升级失败的问题。
- 修复了端口扫描时 ntpm 的 http 指标数量暴增，导致监控报警功能服务异常的问题，默认关闭相关指标的采集。
- 修复了浏览器存在缓存数据可能导致集群部署时出现非预期现象的问题。
- 修复了节点的部分风扇与其他风扇转速差异过大的问题。
- 修复了在主机上安装 ISO 时无法选择 removable 类型的物理盘的问题。
- 修复了因 NTP 服务器最大往返延迟阈值设置过于敏感，导致误触发主机无法与 NTP 服务器时间同步报警的问题。
- 修复了日志轮询配置写入路径错误导致日志无法轮询的问题。
- 修复了因 root 账户过期导致计划任务失效，进而引发日志轮转失败与磁盘占满的问题。
- 修复了 cgroup 检查不通过导致集群扩容失败的问题。
- 修复了虚拟机挂载 USB 设备后，在节点退出维护模式时虚拟机无法自动回迁至源节点的问题。
- 修复了强制关闭集群节点后，节点网络无法 ping 通的告警触发时间超出预期的问题。
- 修复了节点日志采集超过 3 GB 后，短时间内触发大量清理日志任务，导致系统服务的 CPU 和内存资源被大量消耗，进而错误地触发非预期的报警的问题。
- 修复了当集群的日志文件大小超过 7 GB 后，部分节点下载的日志文件可能解压失败的问题。
- 修复了集群 NTP 服务器使用无法解析的域名时，未能正确触发异常告警的问题。
- 修复了因节点系统时间跳变导致 MongoDB 日志文件占用磁盘空间异常的问题。
- 修复了在非预期网口配置下，停止或重启网络服务可能导致 DNS 配置被清空的问题。

### 内核

- 修复了 Intel X710 网卡的 LLDP 功能失效的问题。
- 修复了在使用 CCP 加密设备的 Hygon x86\_64 服务器中部署 SMTX ELF 集群出现宕机的问题。

## 版本配套说明

# 版本配套说明

## SMTX ELF 与服务器类型配套说明

SMTX ELF ⽀持直接部署在 Intel x86\_64 服务器、AMD x86\_64 服务器、Hygon x86\_64 服务器、兆芯 x86 服务器、鲲鹏 AArch64 服务器或飞腾 AArch64 服务器上。

## SMTX ELF 与关联存储版本配套说明

SMTX ELF 支持关联 SMTX ZBS 和 SMTX OS 集群，并使用其存储资源，不支持对接第三方存储。

本版本的 SMTX ELF 集群仅⽀持关联以下版本的集群：

- SMTX ZBS 5.5.0 及以上
- SMTX OS 6.0.0 及以上

## SMTX ELF 与 CloudTower 版本配套说明

SMTX ELF 的管理服务由 CloudTower 提供，本版本 SMTX ELF 集群只⽀持由 CloudTower 4.8.0 及以上版本进⾏管理。

## SMTX ELF 与 SMTX 虚拟机工具配套说明

本版本的 SMTX ELF 集群配套使用 4.1.0 版本的 SMTX 虚拟机工具。

## SMTX ELF 与其他产品版本配套说明

SMTX ELF 可与其他 SmartX 产品配合使用，但每个产品只有部分版本支持与当前版本的 SMTX ELF 配合使用。请查阅对应产品的发布说明文档获取两个产品的版本配套关系。

- Everoute
- 可观测性平台
- 巡检中心
- 升级中心

# 特性支持说明

本版本软件支持启用以下功能和特性。使用这些特性对服务器的架构、软硬件配置均有一定的要求。关于下述特性的配置要求与使用限制，以及对其他功能的影响，请参考随软件发布的[《SMTX ELF 特性说明》](/smtxelf/6.3.0/elf_property_notes/preface-generic)。

## 功能说明（AArch64 架构）

# 功能说明（AArch64 架构）

SMTX ELF 部署在鲲鹏 AArch64 服务器或飞腾 AArch64 服务器上时，下列与虚拟机服务有关的功能将不支持使用。

- E1000 模式网卡
- Cirrus、VGA、QXL 模式的显卡
- BIOS 固件模式
- CPU 兼容性配置
- vGPU 功能
- 深度安全防护功能

当 SMTX ELF 部署在飞腾 AArch64 服务器上时，以下功能也不支持使用。

- GPU 直通
- PCI 直通网卡
- SR-IOV 直通网卡
- 加密控制器
