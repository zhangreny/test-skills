## A-J

# A-J

半连接
:   客户端向服务器发送连接请求，但服务器未做出回复，双方没有完全建立起连接。

---

## K-O

# K-O

连接
:   两个通信实体之间的通信路径，它提供了一个可靠、有序的数据传输通道。连接的建立通常经历握手过程，确保通信的可靠性和顺序性。

流量
:   在网络通信中指的是通过网络传输的数据包。流量的存在与否取决于是否有实际的数据包传输，而不仅仅取决于连接存在与否，因此，可能存在有数据流但无连接的情况。

## P-T

# P-T

Pod
:   Kubernetes 中创建和管理的最小可部署计算单元，代表了一个独立的应用程序运行实例，该实例可能由单个容器或者几个紧耦合在一起的容器组成。
:   本产品文档中的 Pod，如无特殊说明，均指已启用 EIC CNI 插件的 SKS 工作负载集群中的 Pod。

全连接
:   通信双方完成了三次 TCP 握手过程，已经成功建立连接的状态。

RTT
:   RTT（Roud-Trip Time，往返时间）是指从发送方发送数据到接收方，然后接收方发出的响应数据再回到发送方所经历的时间。RTT 是衡量网络延迟和响应时间的一个重要指标，RTT 的值越小，表示网络通信的时延越低，通信效率越高。

数据包
:   在计算机网络中进行数据传输的基本单元，它是将数据划分为小块并添加一些必要信息（如源地址、目标地址、校验和等）以进行有效传输的数据结构。

数据流
:   一个完整的网络会话从源端传输到目标端，数据在各个网络节点（集群、虚拟机、交换机、物理主机等）之间传输和处理。

SMTP
:   SMTP（Simple Mail Transfer Protocol，简单邮件传输协议）是一种用于在网络上传输电子邮件的协议。

SMTP 服务器
:   遵循 SMTP 协议（RFC 5321标准）的专用系统，负责处理电子邮件的发送、中继和投递，在邮件客户端与接收方服务器之间建立可靠传输通道，通常使用 25 或 587 端口。

SNMP
:   SNMP（Simple Network Management Protocol，简单网络管理协议）是一种用于管理和监控网络设备（如路由器、交换机、服务器等）的通信协议。

Syslog 服务器
:   用于接收、集中存储和管理通过网络设备、服务器及应用程序发送的 Syslog 协议日志消息的专用系统，通常通过 UDP 514 端口实现日志的标准化收集与分析。

## U-Z

# U-Z

系统服务
:   以虚拟机形式运行，为集群提供管理和高级功能的服务，例如 CloudTower、Everoute 服务、备份服务、复制服务、SMTX 文件存储等。
