# inspector_cn/1.2.1/巡检中心用户指南
## 关于本文档

# 关于本文档

本文档介绍了通过 CloudTower 使用巡检中心对 SMTX OS、SMTX ELF 和 SMTX ZBS 集群进行巡检，查看并导出巡检结果的具体操作。

阅读本文档的对象需了解 SMTX OS 超融合软件、SMTX ELF 虚拟化软件、SMTX ZBS 分布式块存储软件、CloudTower 管理平台和硬件设备，具备丰富的数据中心操作经验。

## 概述

# 概述

巡检中心是由北京志凌海纳科技股份有限公司（以下简称 “SmartX”）自主研发的集群巡检服务，旨在为您部署的 SmartX 超融合基础设施提供主动的健康状况检测。

通过巡检中心对目标集群的配置、使用情况、性能和服务状态进行检查，由此识别集群当前存在的问题及潜在的风险，并为每条风险项提供针对性的解决方案，可帮助您快速了解集群信息、明确优化方向，确保集群的资源和运行处于最优状态。

---

## 巡检内容

# 巡检内容

巡检中心支持对 SMTX OS、SMTX ELF 和 SMTX ZBS 集群进行集群、硬件、系统、网络、存储和虚拟机六大类巡检项的逐项检查后提供风险提示，并收集、展示集群的基本信息和资源使用情况等环境检查信息。

## 风险情况说明

# 风险情况说明

巡检中心将根据每一个巡检项的检查结果判断其对集群的影响程度，通过高、中、低三种风险情况提示您及时解决问题或规避风险。

| 图标 | 风险情况 | 说明 |
| --- | --- | --- |
|  | 高风险 | 存在紧急的资源或服务状态异常，建议在巡检完成后立即处理 |
|  | 中风险 | 检测到可能会造成严重影响的风险，建议尽快修正 |
|  | 低风险 | 配置不是最优方案，或发生需要在日常运维中注意的事件，建议按照最佳实践建议进行操作 |

## 创建巡检任务

# 创建巡检任务

**前提条件**

- 巡检服务运行正常。
- 待巡检的目标集群需满足如下条件：

  - 集群的类型和软件版本符合[要求](/inspector_cn/1.2.1/release_notes/release-notes-2)；
  - 与 CloudTower 关联且连接正常；
  - 未被包含在其他进行中的巡检任务里。
- 请确保待巡检的目标集群当前未执行 SSH 端口编辑操作，否则该集群将巡检失败。

**操作步骤**

1. 在 CloudTower 的**巡检中心**界面右上角单击 **+ 创建巡检任务**。
2. 在弹出的**创建巡检任务**对话框中选择目标集群。您可以勾选一个或多个数据中心，也可以单击某个数据中心，选择数据中心中的一个或多个集群。每个巡检任务最多可以选择 50 个目标集群。
3. 选择巡检项，默认为全选，您可根据需求仅保留其中几项巡检项。
4. 单击**开始巡检**，巡检任务开始后将自动跳转至巡检记录页面。

## 管理巡检任务

# 管理巡检任务

巡检中心支持查看所有巡检任务，还支持停止、删除巡检任务，并且针对以下两种场景提供了重新执行巡检任务的功能：

- 当前巡检任务`全部失败`或`部分失败`，解决导致失败的问题后可快速执行该巡检任务。
- 当前巡检任务`全部成功`，且巡检任务的巡检集群和巡检项无需改变的情况下可快速执行下一次巡检任务。

**操作步骤**

1. 在 CloudTower 的**巡检中心**界面单击**巡检记录**，巡检记录列表内将展示所有巡检任务的执行时间、执行结果、巡检对象、巡检项数、总风险项数、各级风险项数和用时。将鼠标移至每条巡检任务的巡检对象字段可展示该巡检任务内所有目标集群的名称。
2. 单击巡检任务的右侧 **...**，选择**重新执行**、**停止巡检**或**删除记录**，即可对巡检任务执行对应的操作。

   | 巡检任务的执行状态 | 当前可执行的操作 | 备注 |
   | --- | --- | --- |
   | 巡检中 | 停止巡检 | 如在任务停止前已完成部分集群的巡检，可以查看已完成的集群的巡检结果 |
   | 巡检完成  包含如下三种巡检结果：  - 全部成功 - 部分失败 - 全部失败 | 重新执行 | 重新执行巡检任务时不支持修改目标集群和巡检项，且会在巡检记录中生成一条新的巡检任务 |
   | 删除记录 | - |

## 查看巡检结果

# 查看巡检结果

在 CloudTower 的**巡检中心**界面单击**巡检记录**。在巡检记录列表内单击巡检任务的**执行时间**即可查看巡检结果。

- 执行结果为`全部成功`或`部分失败`的任务的巡检结果包含巡检总结和各集群的巡检结果详情。
- 执行结果为`全部失败`的任务的巡检结果仅展示集群的名称、集群类型、计算平台、软件版本和失败原因。

---

## 查看巡检结果 > 巡检总结

# 巡检总结

巡检总结主要包含概要和风险项。

主页面的上方将展示如下信息：

- 巡检任务基本信息，展示集群数量、巡检项数、用时。
- 风险信息，展示通过率、各级风险项数量。

在页面左侧导航栏中单击**概要**将展示如下信息：

- 集群基本信息，展示集群的名称、执行结果、集群类型、计算平台和软件版本。
- 集群风险分布，展示巡检成功的各集群中的各级风险项数量。

在页面左侧导航栏中单击**风险项**将展示所有风险项的详细信息，包括巡检项名称、所属集群、风险情况、类型、说明、检查结果以及操作建议。如某条风险项的建议操作右侧出现 **...**，您可以单击 **...** > **查看相关对象**，系统将自动跳转至有助于您解决该问题的页面。

> **说明**
>
> 检查结果中如出现 `CLUSTER`，表示该问题是集群级别的问题。

## 查看巡检结果 > 各集群巡检结果

# 各集群巡检结果

巡检成功的集群的巡检结果包含集群信息、环境检查、评估和改进建议。

巡检失败的集群的巡检结果仅展示集群名称、集群类型、计算平台和失败原因。

## 查看巡检结果 > 各集群巡检结果 > 集群信息

# 集群信息

在巡检结果页面左侧导航栏中单击目标集群的**环境检查**或**评估和改进建议**，主页面的上方将展示如下信息：

- 集群基本信息

  展示集群名称、集群类型、软件版本、计算平台、IPMI 配置信息。
- 集群风险信息

  展示巡检项的通过率和各级风险项的数量。

## 查看巡检结果 > 各集群巡检结果 > 环境检查

# 环境检查

您可以通过风险项中的检查结果结合环境检查信息定位问题。

在巡检结果页面左侧导航栏中单击目标集群的**环境检查**将展示集群的基本信息、硬件、系统、网络、资源、性能和虚拟机信息。

不同集群类型收集的信息范围不同，展示的环境检查信息将不同，详情见[环境检查项](/inspector_cn/1.2.1/user_guide/user_guide_12)。

> **说明**：
>
> 在查看**系统**>**事件**>**报警**时，由于巡检中心在集群不存在报警但未获取到报警状态数据时也会提示`存在报警`，如报警状态显示`存在报警`，请您查看 CloudTower 的**报警**界面或在集群的管理界面选择**全部**>**报警**，确认是否存在未解决的报警信息。

## 查看巡检结果 > 各集群巡检结果 > 评估和改进建议

# 评估和改进建议

巡检中心根据巡检项类型整理了目标集群的风险信息和操作建议，您可以根据操作建议对集群进行调整或优化。

在巡检结果页面左侧导航栏中单击目标集群的**评估和改进建议**将展示风险类型分布以及风险详情信息。

- **风险类型分布**

  展示不同巡检项类型里的总风险项数和各级风险项的数量。
- **风险详情信息**

  默认按照巡检项类型分类展示所有巡检项、风险情况、检查结果和建议操作，并根据风险项从高至低排序。

  如某条风险项的建议操作右侧出现 **...**，您可以单击 **...** > **查看相关对象**，系统将自动跳转至有助于您解决该问题的页面。

  您还可以在右侧导航栏中启用**只看风险项**以关注本次巡检所发现的问题。

> **说明**
>
> 检查结果中如出现 `CLUSTER` 字样，表示该问题是集群级别的问题。

## 查看巡检概览

# 查看巡检概览

巡检中心将保留最近一年的巡检记录，您可以通过概览页了解每个集群的巡检频率，以制定巡检计划。

在 CloudTower 的**巡检中心**界面单击**概览**，展示如下信息：

- **集群巡检记录**

  展示近 12 个月内，CloudTower 所关联的各集群月度成功的巡检任务数量。

  将鼠标移至某一集群在某月的巡检数量格，将展示该集群在该月的所有成功巡检任务，单击其中一次巡检任务的**详情**可以查看巡检结果。
- **巡检服务**

  展示巡检服务的版本、架构、支持的集群类型和不支持的集群数量。将鼠标光标悬浮至不支持字段上方，可查看不支持巡检的集群名称。
- **上次巡检**

  单击右侧的**查看详情**或者单击所有卡片均可查看上次巡检任务的巡检结果。

  如上次巡检执行结果为`全部失败`，则仅展示**基本信息**的内容，单击右侧的**重新执行**可以重新执行此次巡检。

  - **基本信息**

    展示上次巡检任务距离今天的时间、开始的时间、集群数和累计巡检项数。
  - **风险概览**

    展示上次巡检的通过率、累计巡检项数和各级风险项数量。

    其中通过率是巡检结果为**正常**的巡检项数量占**累计巡检项**的百分率，通过率越低表示风险项越多。
  - **集群风险分布**

    展示上次巡检的各集群的风险分布情况。

## 导出巡检报告

# 导出巡检报告

巡检中心支持导出 `.docx` 格式的巡检报告。报告将汇总巡检任务中所选集群的状态、资源使用率以及风险情况，并为每条风险项提供针对性的解决方案。

**前提条件**

巡检任务的执行结果为`全部成功`或`部分失败`。

**注意事项**

建议单次导出不超过 50 个集群，否则可能因内容过多而导出异常。

**操作步骤**

1. 在 CloudTower 的**巡检中心**界面单击**巡检记录**，进入巡检记录列表。
2. 在巡检记录列表内单击巡检任务的**执行时间**进入巡检结果。
3. 单击巡检结果右上角**导出报告**，进入导出报告界面。默认导出包含此次巡检任务所有集群的所有巡检项的巡检报告。
4. （可选）在左侧导航栏中根据您的需求保留需要导出的数据中心或集群。
5. （可选）在左侧导航栏中单击已勾选的集群的**评估和改进建议**，在右侧的导航栏里可以根据您的需求保留巡检项或风险项。
6. 单击右上角**导出选中的检查结果**，巡检报告将自动下载至本地电脑。
7. 浏览器的下载内容中出现命名为`InspectorReport-巡检时间`的巡检报告后，再退出导出模式或关闭浏览器页面。

> **说明**：
>
> - 在本地电脑打开巡检报告时，部分版本的 Microsoft Word 可能会提示无法读取或无法打开该文档，请单击**是**或**确定**，即可打开巡检报告。若仍然无法打开，请尝试使用其他文字处理软件（如 WPS Office）打开文档。
> - 对于巡检中心 1.2.0 以下版本执行的历史巡检记录，巡检报告可能导出失败，或导出的巡检报告可能不完整。

## 升级巡检中心

# 升级巡检中心

巡检中心包含在 CloudTower 中，无需单独安装。安装或升级 CloudTower 至指定版本后，将自动安装巡检中心。

巡检中心不允许卸载，但支持对自身进行升级。巡检中心支持随 CloudTower 升级和单独升级两种自升级方式。

---

## 升级巡检中心 > 随 CloudTower 升级

# 随 CloudTower 升级

当巡检中心随 CloudTower 版本发布时，可在升级巡检中心之前先升级 CloudTower，此时巡检中心会随 CloudTower 一并升级，可参考目标 CloudTower 版本的《CloudTower 安装与运维指南》中的升级 CloudTower 小节进行升级。

---

## 升级巡检中心 > 巡检中心自升级

# 巡检中心自升级

当巡检中心不随 CloudTower 发布时，可为巡检中心单独进行升级。

**前提条件**

- 已获取 **tar.gz** 格式的升级文件，且 CPU 架构和当前 CloudTower 相同。
- 巡检中心未处于升级中状态且巡检服务运行正常。

**操作步骤**

1. 在**巡检中心**主页面单击右上角的**设置**按钮，选择**服务升级**。该页面的**当前版本**区域展示了当前巡检中心的版本和 CPU 架构。
2. 在**服务升级**区域选择从**本地选择**或**从 URL 上传**，并将升级文件上传至 CloudTower。
3. 单击**升级**，弹出升级确认对话框。
4. 单击**升级**。

   升级过程中可在任务中心查看升级进度。升级完成后，刷新浏览器页面即可正常使用巡检中心。
