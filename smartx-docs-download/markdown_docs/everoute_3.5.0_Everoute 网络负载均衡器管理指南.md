# everoute/3.5.0/Everoute 网络负载均衡器管理指南
## 关于本文档

# 创建虚拟服务

每个虚拟服务属于一个 LB 实例或一个 LB 实例组，需要关联一个服务器池或服务器池组来处理后端流量。服务器池的主备与 LB 实例的主备机制相互独立，即无论虚拟服务属于 LB 实例还是 LB 实例组，均可以按需选择关联服务器池或服务器池组。

LB 实例或 LB 实例组中的 LB 虚拟机在虚拟服务中负责通过 `VIP + Port` 的方式接收客户端请求，经过 NAT 地址转换后，根据服务器池中配置的调度策略将请求分发至后端服务器，实现流量负载均衡。

使用 NAT 地址转换模式时需注意如下事项：

- **FullNAT 模式**

  - 同一个虚拟服务的 VIP 和 LIP 可以位于不同的网络中，即关联的与客户端通信和与服务器池通信的网络可以不同。
  - 请确保 VIP 地址与客户端连通、LIP 地址与后端服务器连通。
- **DNAT 模式**（仅适用于 VLAN 网络）

  - VIP 和后端服务器必须属于同一个虚拟机网络，即关联的与客户端通信和与服务器池通信的虚拟机网络必须为同一个。
  - 虚拟服务的 VIP 地址将作为服务器池中后端服务器的网关，因此后端服务器与 VIP 之间需要二层可达，后端服务器与客户端之间需要二层不可达，以免后端服务器直接将响应信息发送至客户端。
  - 创建虚拟服务完成后，您需要在后端服务器中手动将虚拟服务的 VIP 地址配置为缺省网关。

**前提条件**

- 已创建所需 [LB 实例](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_46)或 [LB 实例组](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_41)。
- 若计划虚拟服务属于 LB 实例，需为 LB 实例提前关联至少一个与客户端通信的网络以及一个与服务器池通信的网络，具体操作请参考[关联网络](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_48)。
- 若计划虚拟服务属于 LB 实例组，需为 LB 实例组提前关联了至少一个与客户端通信的网络组以及一个与服务器池通信的网络组，具体操作请参考[关联网络组](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_52)。
- 若计划使用服务器池组，需提前[创建服务器池组](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_56)。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**虚拟服务**。
2. 单击 **+ 创建虚拟服务**。
3. 根据配置向导，依次配置虚拟服务的[基本信息](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_60)、[网络](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_61)、[服务器池](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_62)以及[其他参数](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_63)。
4. 单击**确定**。

## 概述

# 概述

Everoute 负载均衡（Load Balancer，LB）作为 Everoute 网络与安全产品的功能，可为基于虚拟机、容器或物理服务器的应用提供第 4 层负载均衡服务。它根据网络通信数据包中的 IP 地址和端口信息，按照设定的负载调度算法，将对应用的访问流量分配到多台后端服务器，以增强应用的并发处理能力和可靠性。

Everoute 负载均衡是部署在 SmartX ELF 虚拟化环境中的虚拟网络功能。它采用“软件定义”方法进行管理，管理员可以在 SmartX 的管理平台 CloudTower 上对 Everoute 负载均衡功能及其相关组件进行统一配置和管理。Everoute LB 虚拟机具有灵活关联 ELF 虚拟化环境中的不同虚拟网络的能力，能够简化流量的转发路径。通过结合多活与主备的机制，多个 Everoute LB 虚拟机之间实现了高可用和高效率，从而避免了单点故障、提高了服务质量。

## 概述 > 产品功能

# 产品功能

- **支持虚拟服务及业务高可用**

  - 支持将流量通过负载均衡的方式转发到多台后端服务器，即使某台后端服务器故障，其它服务器仍可以继续提供服务。
  - 支持将两个服务器池组成服务器池组，主服务器池中所有成员故障时，可由备服务器池中的成员继续提供服务。
  - 每个虚拟服务由 LB 实例中的两个 LB 虚拟机以主备方式提供高可用，避免由于单点故障导致负载均衡服务不可用。如需进一步提升可靠性，可使用 LB 实例组，以避免由于 LB 实例整体故障而导致服务不可用。
- **提供丰富的负载均衡调度策略**

  支持轮询、加权轮询、最少连接、加权最少连接、源地址哈希、目标地址哈希六种调度策略，能够灵活地适应不同场景的应用负载均衡需求。
- **支持主动健康检查**

  支持 TCP、HTTP、UDP、ICMP 四种主动健康检查方式，可周期性检查后端服务器的健康状况。

  当检测到后端服务器出现异常时，可以将客户端请求自动转发至其他健康的后端服务器；当异常的后端服务器恢复正常运行后，可以将其自动恢复到负载均衡服务中，继续承载业务流量。
- **支持会话保持机制**

  支持在特定的时间段内，将所有源地址相同的连接送往同一个后端服务器，以保证会话的连续性和稳定性。
- **支持业务流量控制**

  支持灵活控制虚拟服务处理的出入峰值流量，避免单个虚拟服务占用过多的网络流量资源，实现合理的资源分配。
- **支持客户端访问控制**

  支持通过白名单或黑名单限制允许访问或禁止访问的客户端 IP 地址，有效保护虚拟服务资源，避免恶意请求的破坏，提高负载均衡器的安全性和健壮性。
- **支持客户端并发连接数控制**

  支持灵活限制同一时间下客户端允许与虚拟服务建立的连接数量，避免恶意攻击消耗过多资源。

## 概述 > 应用场景

# 应用场景

- **虚拟化应用场景**

  支持在虚拟化网络场景下，通过负载均衡器连接不同的虚拟机网络，为不同虚拟机网络下的业务提供网络负载均衡服务。使用负载均衡器进行业务流量分发，使用应用服务器弹性扩缩支撑业务变化。
- **容器化应用场景**

  支持容器化环境下的四层负载均衡器，无论是用户自建的 Kubernetes 环境还是 SMTX Kuberentes Service 管理的 Kubernetes 环境，均可以灵活方便地使用此功能，为 Kubernetes 集群内部应用提供对外的四层负载均衡能力。

## 快速入门

# 快速入门

**注意事项**

根据提供应用服务的后端服务器所在的位置不同，负载均衡器的配置方式存在些许差异，具体如下：

- **后端服务器位于 LB 实例或 LB 实例组所在集群**

  - 将后端服务器使用的虚拟机网络关联至 LB 实例或 LB 实例组，作为与服务器池通信的网络，并在此网络中配置 LIP 地址。
  - 由于后端服务器与 LIP 地址位于同一个子网，可以二层互通，因此无需为服务器池配置网关。
  - 创建虚拟服务时 NAT 模式既可以选择 FullNAT，也可以选择 DNAT。
- **后端服务器位于 LB 实例或 LB 实例组所在集群之外的其它集群，或者为物理机**

  - 当后端服务器与已配置的 LIP 地址之间可以二层互通时，配置方式与**后端服务器位于 LB 实例或 LB 实例组所在集群**时相同。
  - 当后端服务器与已配置的 LIP 地址不在同一网段时，需保证后端服务器与虚拟服务中用于与服务器池通信的虚拟机网络三层互通。
    - 需要在服务器池中配置网关，以确保客户端请求能够被正常转发至后端服务器。
    - 创建虚拟服务时 NAT 模式仅可以选择 FullNAT。
- **后端服务器位于虚拟专有云网络**

  - 将后端服务器所在的 VPC 关联至 LB 实例或 LB 实例组，作为与服务器池通信的网络，并在此网络中配置 LIP 地址。
  - 所有后端服务器可以位于同一个 VPC 内相同或不同的 VPC 子网。
  - 由于后端服务器与 LIP 地址位于同一个 VPC，因此将默认使用 VPC 子网的网关，无需单独在服务器池中配置网关。即使手动配置，也仍会通过 VPC 子网的网关通信。
  - 创建虚拟服务时 NAT 模式仅可以选择 FullNAT。
  - 如果与客户端通信的网络也位于 VPC，则与服务器池通信的网络必须与其处于同一个 VPC。
  - 当后端服务器同时关联多个虚拟服务时，所有虚拟服务使用的与服务器池通信的 VPC 子网应保持一致。

**操作步骤**

在部署 Everoute 服务时，您可以启用网络负载均衡器功能；如未在部署阶段启用，也可单独[启用网络负载均衡器](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_39)功能。

启用网络负载均衡器功能之后，即可按照如下步骤说明开始使用网络负载均衡器。

1. [创建 LB 实例](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_46)或[创建 LB 实例组](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_41)：由于启用网络负载均衡器功能时仅支持创建一个 LB 实例，您可以按需创建更多 LB 实例；也可以将两个具有相同 LB **虚拟机数量**、**配置类型**、**支持 VPC 状态**的 LB 实例组成 LB 实例组，以提供 LB 实例的高可用。

   - 使用 LB 实例时，您需要为 LB 实例[关联网络](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_48)。
   - 使用 LB 实例组时，您需要为 LB 实例组中的主、备 LB 实例分别关联网络，组成[网络组](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_52)。
2. [创建健康检查监视器](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_23)：健康检查监视器用于周期性监控服务器池中后端服务器的健康状态。您可以单独创建健康检查监视器，之后在创建服务器池时，直接选择已创建的健康检查监视器；也可以在创建服务器池的过程中，直接添加健康检查监视器。
3. [创建服务器池](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_28)或[创建服务器池组](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_56)。

   - 服务器池是为应用提供服务的一组后端服务器，负责接收并响应负载均衡器分发的客户端请求。您可以单独创建服务器池，之后在创建虚拟服务时，直接选择已创建的服务器池；也可以在创建虚拟服务的过程中，直接新建服务器池。
   - 服务器池组是将两个服务器池建立主、备关系，其中一个作为主服务器池、另一个作为备服务器池，以提供后端服务器的高可用。服务器池组既可搭配 LB 实例使用，也可搭配 LB 实例组使用。您需要在创建虚拟服务前创建服务器池组，以便在创建虚拟服务的过程中直接选择。
4. [创建虚拟服务](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_33)：每个 Everoute 服务可以包含多个虚拟服务，每个虚拟服务对应一个应用，您可以根据实际应用情况创建多个虚拟服务。

   创建虚拟服务时需要配置如下内容：

   1. 配置基本信息：包括所属 Everoute 服务、属于 LB 实例或属于 LB 实例组、协议类型、NAT 模式、监听端口号等。
   2. 配置网络：属于 LB 实例时，您需要选择步骤 1 中关联的网络；属于 LB 实例组时，您需要选择步骤 1 中关联的网络组。
   3. 配置服务器池，包括两种方式：

      - 服务器池：
        - 直接选择步骤 3 中创建的服务器池。
        - 新建服务器池，并关联健康检查监视器。
      - 服务器池组：直接选择步骤 3 中创建的服务器池组。
   4. 配置其他参数：包括会话保持、QoS 控制、并发连接控制以及访问控制。

   创建并启用虚拟服务后，即可实现该应用业务的流量负载分担。

## 启用网络负载均衡器

# 启用网络负载均衡器

启用时仅支持创建一个 LB 实例，即一组用于部署 LB 服务的虚拟机。如需创建更多 LB 实例或创建 LB 实例组，可在启用后进入 **LB 实例**或 **LB 实例组**页面进行配置。

**前提条件**

- 确认已开启相应的防火墙端口，请参考《Everoute 安装与升级指南》的[规划端口](/everoute/3.5.0/er_installation_guide/installation_guide_08)章节。
- 规划 LB 虚拟机，请参考《Everoute 安装与升级指南》的[规划 LB 虚拟机](/everoute/3.5.0/er_installation_guide/installation_guide_34)章节。
- 如当前 Everoute 服务已启用网络安全导流之外的其他功能，则 Everoute Controller 配置类型至少为`中配`。如不满足，请参考《Everoute 安装与升级指南》的[提高 Everoute Controller 配置](/everoute/3.5.0/er_installation_guide/installation_guide_31#%E6%8F%90%E9%AB%98-everoute-controller-%E9%85%8D%E7%BD%AE)章节 。
- 如 LB 需启用**支持 VPC** 功能，请先参考《Everoute 安装与升级指南》[配置虚拟专有云网络功能](/everoute/3.5.0/er_installation_guide/installation_guide_35)。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**设置**，选择待启用网络负载均衡器的 Everoute 服务。
2. 在**服务设置**中单击**基本信息**。
3. 在**网络负载均衡器**功能右侧单击**启用功能**。
4. 进入**启用网络负载均衡器功能**页面，参考《Everoute 安装与升级指南》的[配置网络负载均衡器](/everoute/3.5.0/er_installation_guide/installation_guide_16)章节填写配置信息。

   > **说明**：
   >
   > 启用**支持 VPC** 功能时，若提示暂无可用 TEP 网络 IP 池，请参考《Everoute 虚拟专有云网络管理指南》[设置 TEP IP 池](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_47)。
5. 配置完成后，单击**启用**。

## 查看概览

# 查看概览

## 查看 Everoute 服务概览

进入 CloudTower 的**网络与安全**界面，您可以在概览页查看当前所有已部署的 Everoute 服务的信息。

每个 Everoute 服务概览卡片包括如下内容：

- **服务名称**：卡片标题即为 Everoute 服务的名称。
- **版本**：展示当前 Everoute 服务的版本。当存在可升级的版本时，单击**可升级**，可进入**服务升级**页面，选择版本完成升级操作。
- **Controller 状态**：展示 Everoute Controller 的数量及当前状态。单击 **Controller 状态**可查看 Everoute Controller 信息。

  > **说明**：
  >
  > - Controller 状态包括正常运行、异常、正在部署、正在卸载。
  > - 绿色表示正常状态的 Controller 个数，红色表示异常状态（包括异常、正在部署、正在卸载）的 Controller 个数，黄色表示 vCPU、内存、存储容量使用率大于 80% 的 Controller 个数。
- **网络负载均衡器**：若当前 Everoute 服务已开启网络负载均衡器功能，将展示功能**已开启**，否则将隐藏该项。单击**网络负载均衡器**可进入 **LB 实例**页面。

## 查看网络负载均衡器概览

当存在 Everoute 服务启用了网络负载均衡器功能时，可查看网络负载均衡器的概览信息。

网络负载均衡器概览包括如下卡片：

- **虚拟服务**：展示当前虚拟服务总数，以及已启用和未启用的虚拟服务数量。
- **服务器池**：分别展示当前服务器池和服务器池组的总数。
- **LB 实例**：分别展示当前 LB 实例和 LB 实例组的总数。
- **负载均衡虚拟机**：展示当前 LB 实例和 LB 实例组中的 LB 虚拟机总数，以及健康状态和异常状态下 LB 虚拟机数量。
- **关联网络**：单击右上角的 **LB 实例**，展示当前 LB 实例中已关联的网络总数，以及与客户端通信的网络和与服务器池通信的网络数量；单击 **LB 实例组**，展示当前 LB 实例组中已关联的网络组总数，以及与客户端通信的网络组和与服务器池通信的网络组数量。

## 管理 LB 实例

# 管理 LB 实例

LB 实例是一组用于部署 LB 服务的虚拟机。每个实例由 2-4 个 LB 虚拟机组成，承担负载均衡的实际数据转发任务。

**使用场景**

- 在多集群场景下，您可以根据业务需求为每个集群单独部署 LB 实例，实现集群间的业务流量隔离，确保各集群拥有独立的负载均衡资源，便于管理与扩展。
- 在多租户场景下，您可以为每个租户单独配置 LB 实例，每个租户独享其对应的负载均衡资源，有效避免资源争抢，提升网络安全性。

## 管理 LB 实例 > 创建 LB 实例

# 创建 LB 实例

一个 Everoute 服务中可以创建一个或多个 LB 实例，每个 LB 实例中的所有 LB 虚拟机需位于同一集群，多个 LB 实例可位于相同或不同的集群。

**注意事项**

- LB 虚拟机默认开启 CPU 独占，LB 虚拟机数量要求不多于集群中能够为其提供足够 vCPU 和内存的主机数量。
- 如您后续计划将创建的 LB 实例用于组建 LB 实例组，则组内两个 LB 实例的**虚拟机数量**、**配置类型**、**支持 VPC 状态**需保持一致。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**设置**，选择 Everoute 服务。
2. 在**服务设置**中单击**网络负载均衡器**，默认进入 **LB 实例**页面。
3. 单击 **+ 创建 LB 实例**，进入**创建 LB 实例**页面，参考《Everoute 安装与升级指南》的[配置网络负载均衡器](/everoute/3.5.0/er_installation_guide/installation_guide_16)章节填写配置信息。

   > **说明**：
   >
   > 启用**支持 VPC** 功能时，若提示暂无可用 TEP 网络 IP 池，请参考《Everoute 虚拟专有云网络管理指南》[设置 TEP IP 池](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_47)。
4. 单击**创建**。

   创建完成后，您可以在 **LB 实例**页面查看当前已创建的所有 LB 实例的**名称**、**描述**、**健康状态**、**配置类型**、**关联网络**等信息。当 LB 实例的健康状态为**异常**时，相关虚拟服务将停止工作，请及时排查、恢复至**正常**状态。

   当 LB 实例同步状态异常时，页面上方会出现异常提示，此时您可以在**同步状态异常**提示信息右侧单击**恢复**，以重新同步并恢复 LB 实例的状态。

   单击某一 LB 实例右上角的虚拟机图标，可查看实例中每个 LB 虚拟机的详细信息，以及 vCPU、内存的总量和使用率。当某个 LB 虚拟机的健康状态为**异常**时，可能会影响 LB 服务的稳定性，请尽快恢复至**正常**状态。

   > **说明**：
   >
   > 如果 vCPU、内存的总量和使用率数据无法显示，有以下两个可能的原因：
   >
   > - LB 虚拟机所属集群未部署高级监控且未关联 1.2.0 或以上版本的可观测性服务。
   >
   >   当集群已采集的监控数据量即将达到基本监控的预留资源上限时，虚拟机的一些指标将停止采集。请您参考《CloudTower 使用指南》的**管理高级监控**章节部署高级监控，或参考《可观测性平台用户指南》的**部署可观测性服务**章节部署可观测性服务。
   > - LB 虚拟机异常。
   >
   >   虚拟机状态处于`更新中`、`暂停`、`关机`或`未知`时无法获取相关指标，请先将虚拟机恢复正常。

## 管理 LB 实例 > 编辑 LB 实例

# 编辑 LB 实例

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**设置**，选择 Everoute 服务。
2. 在**服务设置**中单击**网络负载均衡器**，默认进入 **LB 实例**页面。
3. 单击某一 LB 实例右侧的 **...** > **编辑**，在弹出的编辑 LB 实例页面，可编辑如下内容。

   - 编辑名称、描述信息
   - 提高 LB 实例配置：如虚拟服务或后端服务器增多，超出当前 LB 虚拟机的管理能力时，可提高 LB 实例的配置。需要注意的是，Everoute 不支持将部署在 AArch64 架构且版本低于 6.0.0 的 SMTX OS（ELF）集群中的 LB 实例的配置提高。
   - 启用或关闭支持 VPC：关闭前请确保与客户端通信和与服务器池通信的网络均未关联任何 VPC 子网。关闭后，将自动释放已配置的 TEP IP 地址，再次启用时需重新配置。
   - 编辑网络设置：编辑 LB 虚拟机的**子网掩码**、**网关**、**管理网络**相关信息将导致 Everoute 系统服务的管理网络短暂中断。编辑 LB 虚拟机的 **TEP 网络**相关信息将导致集群中正在使用 VPC 网络的虚拟机连接短暂中断，建议在业务空闲时或业务下线后谨慎操作。

     > **说明**：
     >
     > 编辑 LB 虚拟机的管理 IP 时，建议将 LB 虚拟机管理 IP 所在网段与 LB VIP 所在网段严格分开，以避免可能存在的路由覆盖导致的网络异常。
4. 单击**保存**。

## 管理 LB 实例 > 关联网络

# 关联网络

创建 LB 实例之后，您需要为 LB 实例关联至少一个 VLAN 网络或 VPC 子网，并指定其用途，为虚拟服务提供数据传输能力。单个 VLAN 网络或 VPC 子网可同时用于与客户端通信和与服务器池通信。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**设置**，选择 Everoute 服务。
2. 在**服务设置**中单击**网络负载均衡器**，默认进入 **LB 实例**页面。
3. 单击某一 LB 实例区域的 **+ 关联网络**，进入**关联网络**页面。
4. 按需选择**网络类型**和**用途**。

   不同网络类型和用途对应的配置内容存在差异，具体说明如下：

   - **VLAN 网络**

     - [与客户端通信的 VLAN 网络](#%E4%B8%8E%E5%AE%A2%E6%88%B7%E7%AB%AF%E9%80%9A%E4%BF%A1%E7%9A%84-vlan-%E7%BD%91%E7%BB%9C)
     - [与服务器池通信的 VLAN 网络](#%E4%B8%8E%E6%9C%8D%E5%8A%A1%E5%99%A8%E6%B1%A0%E9%80%9A%E4%BF%A1%E7%9A%84-vlan-%E7%BD%91%E7%BB%9C)
   - **VPC 子网**

     - [与客户端通信的 VPC 子网](#%E4%B8%8E%E5%AE%A2%E6%88%B7%E7%AB%AF%E9%80%9A%E4%BF%A1%E7%9A%84-vpc-%E5%AD%90%E7%BD%91)
     - [与服务器池通信的 VPC 子网](#%E4%B8%8E%E6%9C%8D%E5%8A%A1%E5%99%A8%E6%B1%A0%E9%80%9A%E4%BF%A1%E7%9A%84-vpc-%E5%AD%90%E7%BD%91)
5. 配置完成后，单击**关联**。

## 与客户端通信的 VLAN 网络

1. 网络类型选择 **VLAN 网络**，用途选择**与客户端通信**。
2. 选择**虚拟机网络**。
3. 单击 **+ 添加 VIP 地址池**。

   - 允许在一个虚拟机网络下创建多个 VIP 地址池，分别设置**名称**、**起始 IP**、**结束 IP**、**子网掩码**和**网关**信息。后续虚拟服务使用此网络时，可直接从地址池中自动分配 VIP 地址。
   - 您也可以暂不添加 VIP 地址池，后续可通过编辑关联网络创建 VIP 地址池。
4. 关联完成后，在 **LB 实例**区域可以查看已关联的 VLAN 网络名称。

> **说明**：
>
> - 当 LB 虚拟机的管理 IP 地址与 Everoute Controller 的管理 IP 地址不位于同一网段时，此处配置的 VIP 地址池所处网段不允许包含 Everoute Controller 的管理 IP，否则将导致此条静态路由覆盖管理网络的网关生效，进而导致管理网络不通。强烈建议将 VIP 所在网段与管理 IP 所在网段严格分开，避免可能存在的路由覆盖导致的网络异常。
> - 关联与客户端通信的 VLAN 网络后，系统将自动为每个 LB 虚拟机从 169.254.0.0/16 中分配 IP 地址，作为内部 IP，用于探测主、备虚拟服务的健康状态。

## 与服务器池通信的 VLAN 网络

1. 网络类型选择 **VLAN 网络**，用途选择**与服务器池通信**。
2. 选择**虚拟机网络**。
3. 为每个 LB 虚拟机分配所选虚拟机网络中的 **IP 地址**（LIP），用于虚拟服务与服务器池通信。

   支持设置一个**起始 IP** 后为多个 LB 虚拟机批量填充 IP 地址。
4. 设置**子网掩码**。
5. 关联完成后，在 **LB 实例**区域可以查看已关联的 VLAN 网络名称以及每个 LB 虚拟机的 LIP 地址。

## 与客户端通信的 VPC 子网

**前提条件**

- 当前 LB 实例已启用**支持 VPC**。
- 已参考《Everoute 虚拟专有云网络管理指南》在虚拟专有云网络中[创建所需的 VPC](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_60) 及 [VPC 子网](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_65)。

**操作步骤**

1. 网络类型选择 **VPC 网络**，用途选择**与客户端通信**。
2. 选择**所属 VPC**，建议选择客户端所在的 VPC。如客户端处于 VPC 外部，需单独在 VPC 中配置网关服务，用于支持 VPC 内的 VIP 地址与外部客户端之间的通信。
3. 选择 **VPC 子网**，用于在此子网内配置虚拟服务 VIP 地址。
4. 为每个 LB 虚拟机分配所选 VPC 子网中的 IP 地址，作为内部 IP，用于探测主、备虚拟服务的健康状态。

   支持设置一个**起始 IP** 后为多个 LB 虚拟机批量填充 IP 地址。留空则将从 VPC 子网中自动分配。
5. 关联完成后，在 **LB 实例**区域可以查看已关联的 VPC 子网名称、VPC 子网所属的 VPC 名称、以及每个 LB 虚拟机的内部 IP。

> **说明**：
>
> 关联与客户端通信的 VPC 子网后，VPC 子网将默认作为 VIP 地址池使用，无需单独配置。

## 与服务器池通信的 VPC 子网

**前提条件**

- 当前 LB 实例已启用**支持 VPC**。
- 已参考《Everoute 虚拟专有云网络管理指南》在虚拟专有云网络中[创建所需的 VPC](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_60) 及 [VPC 子网](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_65)。

**操作步骤**

1. 网络类型选择 **VPC 网络**，用途选择**与服务器池通信**。
2. 选择**所属 VPC**，建议选择服务器池成员所在的 VPC。如后端服务器位于 VPC 外部，需单独在 VPC 中配置网关服务，用于支持 VPC 内的 LIP 地址与外部服务器池成员之间的通信。
3. 选择 **VPC 子网**。
4. 为每个 LB 虚拟机分配所选 VPC 子网中的 **IP 地址**（LIP），用于虚拟服务与服务器池通信。

   支持设置一个**起始 IP** 后为多个 LB 虚拟机批量填充 IP 地址。留空则将从 VPC 子网中自动分配。
5. 关联完成后，在 **LB 实例**区域可以查看已关联的 VPC 子网名称、VPC 子网所属的 VPC 名称、以及每个 LB 虚拟机的 LIP 地址。

## 管理 LB 实例 > 编辑关联网络

# 编辑关联网络

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**设置**，选择 Everoute 服务。
2. 在右侧的**服务设置**中单击**网络负载均衡器**，默认进入 **LB 实例**页面。
3. 在某一 LB 实例区域找到待编辑的关联网络，单击 **...** > **编辑**。
4. 对于已关联但暂未被虚拟服务使用的网络，您可以更换网络用途或取消用途，但需至少保留一个用途；您也可以为已关联网络增加用途。

   更换用途或增加用途需完成相应的网络配置，具体参数说明请参考[关联网络](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_48)。
5. 对于已关联的与客户端通信的 VLAN 网络，您可以编辑 VIP 地址池相关参数。

   - 当 VLAN 网络中的 VIP 地址池均未在虚拟服务中使用时：

     - 支持编辑 VIP 地址池的名称、起始 IP、结束 IP 、子网掩码、网关。
     - 支持增加新的 VIP 地址池。
     - 支持删除 VIP 地址池。
   - 当 VLAN 网络中存在 VIP 地址池已在虚拟服务中使用时：

     - 支持编辑 VIP 地址池的名称、起始 IP、结束 IP、子网掩码、网关。需要注意的是，对于已使用的 VIP 地址池仅允许扩大地址池范围，即编辑后的 VIP 地址池中必须包含编辑前已有的 IP 地址。
     - 支持增加新的 VIP 地址池。
     - 不支持删除已使用的 VIP 地址池。
6. 单击**保存**。

## 管理 LB 实例 > 解除关联

# 解除关联

解除 LB 实例和网络的关联前，请确保该网络未在属于当前 LB 实例的任何虚拟服务中使用。如已在虚拟服务中使用，则需要先在对应的虚拟服务中通过**编辑网络配置**的方式更换使用的网络，之后再解除关联。

**风险提示**

关联关系解除后，将释放对应的 VIP 地址池、内部 IP 地址以及 LIP 地址，再次关联此网络时您需要重新设置。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**设置**，选择 Everoute 服务。
2. 在右侧的**服务设置**中单击**网络负载均衡器**，默认进入 **LB 实例**页面。
3. 在某一 LB 实例区域找到待解除关联的网络，单击 **...** > **解除关联**。
4. 在弹出的确认对话框中单击**解除**。

# 删除 LB 实例

4. 在弹出的确认对话框中单击**删除**。

## 管理 LB 实例组

# 管理 LB 实例组

LB 实例组由两个 LB 实例组成，其中一个作为主 LB 实例、另一个作为备 LB 实例，用于提供实例的高可用。

**使用场景**

在灾备场景中，您可以在主集群和备集群中分别部署 LB 实例，组成 LB 实例组。默认由主集群中的 LB 实例提供服务，当主实例发生故障时，系统将自动切换至备集群中的 LB 实例，保障服务的连续性和可用性。

---

## 管理 LB 实例组 > 创建 LB 实例组

# 创建 LB 实例组

您可以将两个具有相同 LB **虚拟机数量**、**配置类型**、**支持 VPC 状态**的 LB 实例组成一个 LB 实例组，两个 LB 实例可以位于不同的集群或位于同一集群的不同物理节点。

同一时刻，LB 实例组中的所有虚拟服务仅运行在其中一个 LB 实例中，即当前处于**活动**状态的实例。

**前提条件**

已[创建所需 LB 实例](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_46)，且 LB 实例未关联任何 LB 实例组，也未关联任何与客户端通信或与服务器池通信的网络。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**设置**，选择 Everoute 服务。
2. 在**服务设置**中单击**网络负载均衡器**，默认进入 **LB 实例**页面。
3. 单击 **LB 实例组**页签，进入 **LB 实例组**页面。
4. 单击 **+ 创建 LB 实例组**，填写 LB 实例组的**名称**和**描述**。
5. 选择满足要求的**主 LB 实例**和**备 LB 实例**。

   创建后默认主 LB 实例负责处理所有虚拟服务的转发请求，备 LB 实例则作为主 LB 实例的冗余，在主 LB 实例发生故障时接管虚拟服务的转发请求，以确保负载均衡服务的连续性和高可用性。
6. 单击**创建**。

   创建完成后，您可以按需编辑 LB 实例组的**名称**和**描述**信息。

## 管理 LB 实例组 > 查看 LB 实例组

# 查看 LB 实例组

3. 单击 **LB 实例组**页签，进入 **LB 实例组**页面。
4. 在 **LB 实例组**页面，您可以查看当前已创建的所有 LB 实例组的如下信息。

   - LB 实例组的**名称**、**描述**、**配置类型**、**支持 VPC 状态**、**关联网络组**。
   - 主、备 LB 实例的**所属集群**。
   - 主、备 LB 实例的**健康状态**，显示为**正常**或**异常**。
   - 主、备 LB 实例的**活动状态**，实际运行虚拟服务的 LB 实例状态显示为**活动**。
5. 单击主、备 LB 实例右上角的虚拟机图标，可查看主、备实例中每个 LB 虚拟机的详细信息，以及 vCPU、内存的总量和使用率。

   > **说明**：
   >
   > 如果 vCPU、内存的总量和使用率数据无法显示，有以下两个可能的原因：
   >
   > - LB 虚拟机所属集群未部署高级监控且未关联 1.2.0 或以上版本的可观测性服务。
   >
   >   当集群已采集的监控数据量即将达到基本监控的预留资源上限时，虚拟机的一些指标将停止采集。请您参考《CloudTower 使用指南》的**管理高级监控**章节部署高级监控，或参考《可观测性平台用户指南》的**部署可观测性服务**章节部署可观测性服务。
   > - LB 虚拟机异常。
   >
   >   虚拟机状态处于`更新中`、`暂停`、`关机`或`未知`时无法获取相关指标，请先将虚拟机恢复正常。
6. 当 LB 实例组中主、备 LB 实例同步状态异常时，页面上方会出现异常提示，此时您可以在异常提示信息右侧单击**恢复**，以重新同步并恢复主、备 LB 实例的状态。

## 管理 LB 实例组 > 切换活动实例

# 切换活动实例

在 LB 实例组中，活动 LB 实例负责运行虚拟服务，非活动实例不运行虚拟服务。

- **自动切换**：当系统检测到主 LB 实例健康状态为**异常**时，如果备 LB 实例健康状态为**正常**，则系统自动将备 LB 实例切换为**活动**实例，并将所有虚拟服务及其流量自动切换至备 LB 实例。当主 LB 实例恢复**正常**状态后，系统不会自动切换活动实例至主 LB 实例，您需要手动切换。
- **手动切换**：当系统未自动检测到 LB 实例异常或主 LB 实例故障恢复后，您可以按需手动切换活动实例。

**风险提示**

切换活动实例时，使用 LB 实例组的虚拟服务会有短暂的网络中断，手动切换建议在业务空闲时或业务下线后谨慎操作。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**设置**，选择 Everoute 服务。
2. 在服务设置中单击**网络负载均衡器**，默认进入 **LB 实例**页面。
3. 单击 **LB 实例组**页签，进入 **LB 实例组**页面。
4. 在某一 LB 实例组区域单击 **...** > **切换活动实例**，切换主、备 LB 实例的活动状态。若虚拟服务中使用了服务器池组，您可以按需选择是否同步切换服务器池组的主、备服务器池活动状态。
5. 单击**确认**。

5. 单击**确认**。

## 管理 LB 实例组 > 关联网络组

# 关联网络组

创建 LB 实例组之后，需分别为 LB 实例组中的主、备 LB 实例关联至少一个 VLAN 网络或 VPC 子网，组成网络组，为虚拟服务提供数据传输能力。单个 VLAN 网络组或 VPC 子网组可同时用于与客户端通信和与服务器池通信。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**设置**，选择 Everoute 服务。
2. 在**服务设置**中单击**网络负载均衡器**，默认进入 **LB 实例**页面。
3. 单击 **LB 实例组**页签，进入 **LB 实例组**页面。
4. 单击某一 LB 实例组区域的 **+ 关联网络组**，进入**关联网络组**页面。
5. 设置关联网络组的**名称**，按需选择**网络类型**和**用途**。

   不同网络类型和用途对应的配置内容存在差异，具体说明如下：

   - **VLAN 网络**

     - [与客户端通信的 VLAN 网络组](#%E4%B8%8E%E5%AE%A2%E6%88%B7%E7%AB%AF%E9%80%9A%E4%BF%A1%E7%9A%84-vlan-%E7%BD%91%E7%BB%9C%E7%BB%84)
     - [与服务器池通信的 VLAN 网络组](#%E4%B8%8E%E6%9C%8D%E5%8A%A1%E5%99%A8%E6%B1%A0%E9%80%9A%E4%BF%A1%E7%9A%84-vlan-%E7%BD%91%E7%BB%9C%E7%BB%84)
   - **VPC 子网**

     - [与客户端通信的 VPC 子网组](#%E4%B8%8E%E5%AE%A2%E6%88%B7%E7%AB%AF%E9%80%9A%E4%BF%A1%E7%9A%84-vpc-%E5%AD%90%E7%BD%91%E7%BB%84)
     - [与服务器池通信的 VPC 子网组](#%E4%B8%8E%E6%9C%8D%E5%8A%A1%E5%99%A8%E6%B1%A0%E9%80%9A%E4%BF%A1%E7%9A%84-vpc-%E5%AD%90%E7%BD%91%E7%BB%84)
6. 配置完成后，单击**关联**。

## 与客户端通信的 VLAN 网络组

1. 网络类型选择 **VLAN 网络**，用途选择**与客户端通信**。
2. 分别为主、备 LB 实例选择**虚拟机网络**。主 LB 实例对应**主网络**，备 LB 实例对应**备网络**。
3. 单击 **+ 添加 VIP 地址池**，所添加的地址池将被主、备 LB 实例的 VLAN 网络共享。

   - 允许在一个虚拟机网络下创建多个 VIP 地址池，分别设置**名称**、**起始 IP**、**结束 IP**、**子网掩码**和**网关**信息。后续虚拟服务使用此网络时，可直接从地址池中自动分配 VIP 地址。
   - 您也可以暂不添加 VIP 地址池，后续可通过编辑关联网络组创建 VIP 地址池。
4. 关联完成后，在 **LB 实例组**区域可以查看已关联网络组中主、备网络的 VLAN 网络名称。

> **说明**：
>
> - 当 LB 虚拟机的管理 IP 地址与 Everoute Controller 的管理 IP 地址不位于同一网段时，此处配置的 VIP 地址池所处网段不允许包含 Everoute Controller 的管理 IP，否则将导致此条静态路由覆盖管理网络的网关生效，进而导致管理网络不通。强烈建议将 VIP 所在网段与管理 IP 所在网段严格分开，避免可能存在的路由覆盖导致的网络异常。
> - 关联与客户端通信的 VLAN 网络后，系统将自动为每个 LB 虚拟机从 169.254.0.0/16 中分配 IP 地址，作为内部 IP，用于探测主、备虚拟服务的健康状态。

## 与服务器池通信的 VLAN 网络组

1. 网络类型选择 **VLAN 网络**，用途选择**与服务器池通信**。
2. 分别为主、备 LB 实例选择**虚拟机网络**。主 LB 实例对应**主网络**，备 LB 实例对应**备网络**。
3. 分别为主、备 LB 实例的每个 LB 虚拟机分配所选主、备虚拟机网络中的 **IP 地址**（LIP），用于虚拟服务与服务器池通信。主 LB 实例对应**主 IP**，备 LB 实例对应**备 IP**。

   支持在主、备 IP 中分别设置一个**起始 IP 地址**，以便为多个 LB 虚拟机批量填充 IP 地址。
4. 分别为主、备 LB 实例设置**子网掩码**。
5. 关联完成后，在 **LB 实例组**区域可以查看已关联的网络组中主、备网络的 VLAN 网络名称，以及主、备网络中每个 LB 虚拟机的 LIP 地址。

## 与客户端通信的 VPC 子网组

**前提条件**

- 当前 LB 实例组中的主、备 LB 实例均已启用**支持 VPC**。
- 已参考《Everoute 虚拟专有云网络管理指南》在虚拟专有云网络中[创建所需的 VPC](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_60) 及 [VPC 子网](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_65)。

**操作步骤**

1. 网络类型选择 **VPC 网络**，用途选择**与客户端通信**。
2. 选择**所属 VPC**，建议选择客户端所在的 VPC。如客户端处于 VPC 外部，需单独在 VPC 中配置网关服务，用于支持 VPC 内的 VIP 地址与外部客户端之间的通信。
3. 选择 **VPC 子网**。主、备网络的 VPC 子网相同，以便 VIP 地址能够在主、备 LB 实例间漂移。
4. 分别为主、备 LB 实例的每个 LB 虚拟机分配所选 VPC 子网中的 IP 地址，作为内部 IP，用于探测主、备虚拟服务的健康状态。主 LB 实例对应**主 IP**，备 LB 实例对应**备 IP**。

   支持在主、备 IP 中分别设置一个**起始 IP 地址**，以便为多个 LB 虚拟机批量填充 IP 地址。留空则将从 VPC 子网中自动分配。
5. 关联完成后，在 **LB 实例组**区域可以查看已关联的网络组中主、备网络的 VPC 子网名称，VPC 子网所属的 VPC 名称，以及主、备网络中每个 LB 虚拟机的内部 IP。

> **说明**：
>
> 关联与客户端通信的 VPC 子网后，VPC 子网将默认作为 VIP 地址池使用，无需单独配置。

## 与服务器池通信的 VPC 子网组

**前提条件**

- 当前 LB 实例组中的主、备 LB 实例均已启用**支持 VPC**。
- 已参考《Everoute 虚拟专有云网络管理指南》在虚拟专有云网络中[创建所需的 VPC](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_60) 及 [VPC 子网](/everoute/3.5.0/er_administration_guide_vpc/administration_guide_vpc_65)。

**操作步骤**

1. 网络类型选择 **VPC 网络**，用途选择**与服务器池通信**。
2. 选择**所属 VPC**，建议选择服务器池所在的 VPC，主备 VPC 子网需位于同一个 VPC 内。如后端服务器位于 VPC 外部，需单独在 VPC 中配置网关服务，用于支持 VPC 内的 LIP 地址与外部服务器池成员之间的通信。
3. 分别为主、备 LB 实例选择 **VPC 子网**。主 LB 实例对应**主网络**，备 LB 实例对应**备网络**。
4. 分别为主、备 LB 实例的每个 LB 虚拟机分配所选 VPC 子网中的 **IP 地址**（LIP），用于虚拟服务与服务器池通信。

   支持在主、备 IP 中分别设置一个**起始 IP 地址**，以便为多个 LB 虚拟机批量填充 IP 地址。留空则将从 VPC 子网中自动分配。
5. 关联完成后，在 **LB 实例组**区域可以查看已关联的网络组中主、备网络的 VPC 子网名称，VPC 子网所属的 VPC 名称，以及主、备网络中每个 LB 虚拟机的 LIP 地址。

   更换用途或增加用途需完成相应的网络配置，具体参数说明请参考[关联网络组](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_52)。

     - 支持编辑 VIP 地址池的名称、起始 IP、结束 IP 、子网掩码、网关。
     - 支持增加新的 VIP 地址池。
     - 支持删除 VIP 地址池。
   - 当 VLAN 网络组中存在 VIP 地址池已在虚拟服务中使用时：

     - 支持编辑 VIP 地址池的名称、起始 IP、结束 IP、子网掩码、网关。需要注意的是，对于已使用的 VIP 地址池仅允许扩大地址池范围，即编辑后的 VIP 地址池中必须包含编辑前已有的 IP 地址。
     - 支持增加新的 VIP 地址池。
     - 不支持删除已使用的 VIP 地址池。
7. 单击**保存**。

## 管理健康检查监视器

# 管理健康检查监视器

健康检查监视器负责检查服务器池中一组后端服务器（即服务器池中的成员）的健康状态。

当检测到后端服务器出现异常时，会将客户端请求自动转发至其他健康的后端服务器；当异常的后端服务器恢复正常运行后，可将其自动恢复到负载均衡服务中，继续承载业务流量。

> **说明**：
>
> - 每个服务器池至少需添加一个健康检查监视器。
> - 健康检查监视器可在不同的服务器池中复用。

---

## 管理健康检查监视器 > 创建健康检查监视器

# 创建健康检查监视器

您可以单独创建健康检查监视器，之后在创建服务器池时，直接选择已创建的健康检查监视器；也可以在创建服务器池的过程中，直接添加健康检查监视器。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**健康检查监视器**。
2. 单击 **+ 创建健康检查监视器**，根据如下参数说明完成配置。

   | 参数 | 描述 |
   | --- | --- |
   | 名称 | 健康检查监视器的名称，在系统中唯一标识健康检查监视器。 |
   | 所属 Everoute 服务 | 选择健康检查监视器所属的 Everoute 服务。  仅支持选择已启用网络负载均衡器功能且状态正常的 Everoute 服务。  **注意**：在服务器池中直接添加健康检查监视器，默认与服务器池属于相同的 Everoute 服务，无需指定。 |
   | 协议类型 | 健康检查监视器检查后端服务器状态时使用的协议，具体如下：  - **TCP**：向后端服务器发起一个 TCP 连接请求，根据后端服务器是否响应或超时，判断服务器健康状态。 - **HTTP**：向指定的 URL 发起一个 HTTP GET 请求，根据返回的结果或者状态码，判断服务器健康状态。 - **UDP**：向后端服务器发起一个 UDP 请求，根据 ICMP 的错误消息，判断服务器健康状态。 - **ICMP**：向后端服务器发起 Ping 包检测，根据返回结果判断服务器健康状态。 |
   | 请求路径 | 选择 HTTP 协议时，您需要设置健康检查请求的 URL 路径。请求路径默认为 `/`，您可以按需修改。  设置完成后，实际请求地址为 `http://server_address:server_port/请求路径`。其中，`server_address` 为后端服务器的真实地址（RIP），`server_port` 为检查端口。 |
   | 检查正常状态码 | 选择 HTTP 协议时，您需要设置健康检查正常状态码，用于确认后端服务器状态是否正常。  您可以指定多个数值或一个数值区间。例如指定为 `200-299`，则健康检查时后端服务器回复 200-299 中的任意一个状态码，均认为后端服务器状态正常。 |
   | 发送内容 | 可选。选择 UDP 协议时，您可以按需指定健康检查时发送给后端服务器的内容。 |
   | 接收内容 | 可选。选择 UDP 协议时，您可以按需指定健康检查时期望从后端服务器接收的内容。  当后端服务器对健康检查响应的内容中包含此字符串时，即认为后端服务器状态正常。 |
   | 检查端口 | 健康检查监视器检查后端服务器健康状态时使用的端口。  - **默认端口**：选择默认端口时，使用服务器池中为每个后端服务器指定的端口进行健康检查。 - **指定端口**：选择指定端口时，您需要输入**端口号**，支持 1 ～ 65535。指定端口后，将对关联了此健康检查监视器的服务器池中所有的后端服务器，统一使用该端口进行健康检查。 |
   | 检查间隔 | 发起健康检查的时间间隔，默认为 10 s。 |
   | 超时时间 | 接收健康检查响应时预期等待的时间，默认为 5 s。若后端服务器在此时间内没有正确响应，则视为本次健康检查失败。 |
   | 重试次数 | 健康检查的重试次数，默认为 3 次。  - 对于处于**健康**状态的后端服务器，连续检查失败的次数达到重试次数后，将状态标记为**不健康**。 - 对于处于**不健康**状态的后端服务器，连续检查成功的次数达到重试次数后，将状态标记为**健康**。 |
   | 重试间隔时间 | 每两次重试之间的时间间隔，默认为 3 s。 |
3. 单击**创建**。

   创建完成后，您可以在**健康检查监视器**列表中查看当前已创建的所有健康检查监视器的配置信息，以及监视器关联的服务器池。

## 管理服务器池与服务器池组

# 管理服务器池与服务器池组

服务器池是为应用提供服务的一组后端服务器，负责接收并响应负载均衡器分发的客户端请求。

服务器池组是将两个服务器池建立主、备关系，其中一个作为主服务器池、另一个作为备服务器池。当主服务器池中的成员全部故障时，可自动切换至使用备服务器池中的成员，以提供后端服务器的高可用。服务器池组既可搭配 LB 实例使用，也可搭配 LB 实例组使用。

后端服务器可包括以下类型：

- 位于 LB 实例所在集群中的虚拟机
- 位于其他集群中的虚拟机或物理机
- Kubernetes 集群中的容器应用
- 位于 VPC 网络中的虚拟机

当存在两个数据中心做主、备使用时，建议采用 **LB 实例组 + 服务器池组**的方式配置，将主 LB 实例及主服务器池中的所有后端成员部署在主数据中心，同时在备数据中心单独部署一套备 LB 实例和备服务器池。当主数据中心发生故障时，可以同时将 LB 实例和服务器池切换至备数据中心，以最快速度的恢复业务运行。当主数据中心恢复正常后，可手动将活动 LB 实例迁移回主 LB 实例，并可以选择自动尝试将所有使用服务器池组的虚拟服务切换使用主服务器池。

## 管理服务器池与服务器池组 > 管理服务器池 > 创建服务器池

# 创建服务器池

一个 Everoute 服务中可以创建多个服务器池。每个虚拟服务可关联一个服务器池，一个服务器池也可以同时关联多个虚拟服务。

您可以单独创建服务器池，之后在创建虚拟服务时，直接选择已创建的服务器池；也可以在创建虚拟服务的过程中，直接新建服务器池。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**服务器池**。
2. 单击 **+ 创建** > **服务器池**。
3. 配置服务器池的**基本信息**。

   | 参数 | 描述 |
   | --- | --- |
   | 名称 | 服务器池的名称，在系统中唯一标识服务器池。 |
   | 描述 | 服务器池的描述信息，您可以按需填写。 |
   | 所属 Everoute 服务 | 选择服务器池所属的 Everoute 服务。  仅支持选择已启用网络负载均衡器功能且状态正常的 Everoute 服务。 |
   | 调度策略 | 选择调度策略，网络负载均衡器将根据选择的调度策略，决定如何将客户端请求调度至服务器池中的各个成员（即后端服务器）。  - **轮询**：将客户端请求依次分发到每个后端服务器。 - **加权轮询**：采用轮询的方式，根据权重大小决定每个后端服务器处理的请求数量。 - **最少连接**：将请求分发到连接数最少的后端服务器。 - **加权最少连接**：将请求分发到连接数较少且权重较大的后端服务器。 - **源地址哈希**：根据请求的源地址，作为散列键从静态分配的散列表找出对应的后端服务器，若该服务器是可用的且未超载，则将请求发送到该服务器。 - **目标地址哈希**：根据请求的目标地址，作为散列键从静态分配的散列表找出对应的后端服务器，若该服务器是可用的且未超载，则将请求发送到该服务器。 |
   | 启用状态 | 设置服务器池的启用状态，默认为**启用**。  **注意**：状态为**未启用**时，您将无法在虚拟服务中选择此服务器池，健康检查监视器也不再对该服务器池进行周期性健康检查。当您暂时不希望在虚拟服务中使用此服务器池时，可设置为未启用。 |
4. 配置服务器池的**成员信息**。您可以单击 **+ 添加成员**，依次添加多个成员；也可以单击**批量填充**，将根据您填写的成员数量、起始 IP 等信息批量填写所有成员信息。

   > **注意**：
   >
   > - 添加的成员无须位于当前 Everoute 所在集群，仅需确保成员与用于与服务器池通信的虚拟机网络中所有 LIP 地址三层连通。
   > - 如您在使用**批量填充**前手工添加过成员，则批量填充将会覆盖已有成员信息。

   | 参数 | 描述 |
   | --- | --- |
   | 成员 | 服务器池中为客户端提供应用服务的后端服务器。 |
   | IP 地址 | 为客户端提供应用服务的后端服务器的真实 IP 地址（RIP）。 |
   | 端口号 | 为客户端提供应用服务的后端服务器的端口号。 |
   | 权重 | 设置成员的权重值。 当调度策略为**加权轮询**或**加权最少连接**时，将通过此权重进行加权计算，留空则默认权重为 1。 |
   | 启用 | 设置服务器池中成员的启用状态。  **注意**：状态为**未启用**时，表示该成员暂不接受调度。此时，负载均衡器不会将客户端的请求分发至该成员，且健康检查监视器也不会检查该成员的状态。当您暂时不希望使用该成员承载业务流量时，可设置为未启用。 |
   | 备用成员 | 设置服务器池中的备用成员。您可以设置多个主成员、一个备用成员，也可以设置一个主成员、一个备用成员。当服务器池中主成员均不健康时，将由备用成员负责处理流量。  适用于后端为单个服务器池的场景，用于提供主成员故障时的冗余保障。如需实现跨服务器池的主备切换，请使用服务器池来组建服务器池组。 |
   | 网关 | 当服务器池的全部成员在 VPC 外，且和虚拟服务中用于与服务器池通信的 VLAN 网络之间二层不可达时，您需配置网关，以确保客户端请求能够被正常转发至相应的成员。当存在成员位于 VPC 中时，将使用 VPC 子网网关，无需配置。 |
5. 配置服务器池的**健康检查监视器**。

   单击 **+ 添加健康检查监视器**，您可以**从已有中选择**之前创建的健康检查监视器，也可以**新建健康检查监视器**。具体参数说明，请参见[创建健康检查监视器](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_23)。

   > **注意**：
   >
   > 当一个服务器池中配置多个健康检查监视器时，只有所有健康检查监视器检查均为**健康**，才认为后端服务器状态**健康**。
   >
   > 根据主成员和备用成员的数量，健康检查方式有所不同：
   >
   > - 当服务器池中配置了多个主成员、一个备用成员时，仅为主成员提供健康检查，无法为备用成员提供健康检查。流量默认在主成员中调度，只有主成员均不健康时，流量才调度至备用成员。若服务器池后续计划用于组建服务器池组，则不支持将其配置为该模式。
   > - 当服务器池中配置了一个主成员、一个备用成员时，可为主备成员均提供健康检查，并根据健康检查结果进行流量调度。
   >
   > 建议为服务器池至少配置一个健康检查监视器。未配置健康检查监视器时，无法监测服务器池成员的健康状态，也无法使用备用成员。若此服务器池后续用于组建服务器池组，将无法触发主、备服务器池故障自动切换。
6. 单击**创建**。

   创建完成后，您可以在**服务器池**列表中查看当前已创建的服务器池信息。

   - 单击某一服务器池，可查看该服务器池的详情，包括基本信息、成员信息、健康检查监视器信息以及已关联的虚拟服务。
   - 单击健康检查监视器或者虚拟服务的名称链接，可跳转至相应功能的详情界面。
   - 单击**服务器池**列表右上方的**导出报表**图标，可将当前的服务器池列表导出成 `.csv` 文件至本地保存。

## 管理虚拟服务

# 管理虚拟服务

一个 Everoute 服务中可以创建多个虚拟服务，每个虚拟服务通常对应一个应用。为应用创建并启用虚拟服务后，即可实现该应用的流量负载均衡。

## 管理虚拟服务 > 创建虚拟服务

# 创建虚拟服务

每个虚拟服务属于一个 LB 实例或一个 LB 实例组，需要关联一个服务器池或服务器池组来处理后端流量。服务器池的主备与 LB 实例的主备机制相互独立，即无论虚拟服务属于 LB 实例还是 LB 实例组，均可以按需选择关联服务器池或服务器池组。

LB 实例或 LB 实例组中的 LB 虚拟机在虚拟服务中负责通过 `VIP + Port` 的方式接收客户端请求，经过 NAT 地址转换后，根据服务器池中配置的调度策略将请求分发至后端服务器，实现流量负载均衡。

使用 NAT 地址转换模式时需注意如下事项：

- **FullNAT 模式**

  - 同一个虚拟服务的 VIP 和 LIP 可以位于不同的网络中，即关联的与客户端通信和与服务器池通信的网络可以不同。
  - 请确保 VIP 地址与客户端连通、LIP 地址与后端服务器连通。
- **DNAT 模式**（仅适用于 VLAN 网络）

  - VIP 和后端服务器必须属于同一个虚拟机网络，即关联的与客户端通信和与服务器池通信的虚拟机网络必须为同一个。
  - 虚拟服务的 VIP 地址将作为服务器池中后端服务器的网关，因此后端服务器与 VIP 之间需要二层可达，后端服务器与客户端之间需要二层不可达，以免后端服务器直接将响应信息发送至客户端。
  - 创建虚拟服务完成后，您需要在后端服务器中手动将虚拟服务的 VIP 地址配置为缺省网关。

**前提条件**

- 已创建所需 [LB 实例](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_46)或 [LB 实例组](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_41)。
- 若计划虚拟服务属于 LB 实例，需为 LB 实例提前关联至少一个与客户端通信的网络以及一个与服务器池通信的网络，具体操作请参考[关联网络](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_48)。
- 若计划虚拟服务属于 LB 实例组，需为 LB 实例组提前关联了至少一个与客户端通信的网络组以及一个与服务器池通信的网络组，具体操作请参考[关联网络组](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_52)。
- 若计划使用服务器池组，需提前[创建服务器池组](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_56)。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**虚拟服务**。
2. 单击 **+ 创建虚拟服务**。
3. 根据配置向导，依次配置虚拟服务的[基本信息](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_60)、[网络](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_61)、[服务器池](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_62)以及[其他参数](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_63)。
4. 单击**确定**。

## 管理虚拟服务 > 创建虚拟服务 > 配置基本信息

# 配置基本信息

1. 配置虚拟服务的**名称**和**描述信息**。名称在系统中唯一标识虚拟服务，一个虚拟服务对应一个应用。
2. 选择虚拟服务**所属 Everoute 服务**。仅支持选择已启用网络负载均衡器功能、状态正常的 Everoute 服务。
3. 设置虚拟服务**属于 LB 实例**或**属于 LB 实例组**，并选择已创建的 LB 实例或 LB 实例组。
4. 根据如下参数说明配置**协议类型**、**NAT 模式**、**启用状态**和**监听端口号**。

   | 参数 | 描述 |
   | --- | --- |
   | 协议类型 | 您需要根据实际应用的传输层协议选择与客户端通信的协议类型，比如 HTTP 类型应用的传输层协议为 TCP。  - **TCP**：一种面向连接的、可靠的传输层协议。适用于注重可靠性，对数据准确性要求高的场景，如文件传输、发送或接收邮件、远程登录。 - **UDP**：一种无连接的传输层协议，可靠性相对较低。适用于关注实时性而相对不注重可靠性的场景，如视频聊天、游戏、金融实时行情推送。 |
   | NAT 模式 | 选择网络负载均衡器使用的地址转换模式。  - **FullNAT**：默认模式，该模式下将进行源地址转换（SNAT）和目标地址转换（DNAT）。   - 接收请求时，自动将客户端源地址（CIP）转换为负载均衡器虚拟机的固定 IP 地址（LIP），目标地址（VIP）转换为后端服务器的真实 IP (RIP)，与后端服务器通信，转换后服务端将不会感知真实的客户端地址。   - 响应请求时，将后端服务器的真实地址（RIP）转换为虚拟服务的 VIP 地址，负载均衡器虚拟机的固定 IP 地址（LIP）转换为客户端地址（CIP），与客户端通信，使客户端不感知真实处理请求的服务端 IP。 - **DNAT**：当您希望后端服务器能够感知到真实的客户端地址并进行相关分析时，可选择 DNAT 模式。此时将禁用源地址转换，仅进行目标地址转换。   - 接收请求时，保持客户端源地址（CIP）不变，将目标地址（VIP）转换为后端服务器的真实 IP (RIP)，与后端服务器通信，此时服务端可以感知真实的客户端地址。   - 响应请求时，将后端服务器的真实地址（RIP）转换为虚拟服务的 VIP 地址，向真实的客户端地址（CIP）发送响应信息，客户端不感知真实处理请求的服务端 IP。 **注意**：DNAT 模式下，与客户端通信和与服务器端通信的网络均需为 VLAN 网络。 |
   | 启用状态 | 设置虚拟服务的启用状态，默认为**启用**。  **注意**：设置为 **未启用**后，该虚拟服务不再接收客户端的请求。当您暂时不希望使用虚拟服务时，可设置为未启用。 |
   | 监听端口号 | 设置监听的端口号，取值范围为 1 ～ 65535。负载均衡器将通过 VIP 地址和监听端口号接收客户端的请求。  **注意**：不同虚拟服务中，使用的 `VIP 地址 + 端口号` 的组合不允许完全相同。 |
5. 单击**下一步**。

## 管理虚拟服务 > 创建虚拟服务 > 配置网络

# 配置网络

根据上一步中所选 NAT 模式不同，支持的使用场景有所不同：

- FullNAT 模式支持如下三种场景：

  - **VLAN 场景**：与客户端通信、与服务器池通信均使用 **VLAN 网络**。
  - **VPC 南北向场景**：与客户端通信使用 **VLAN 网络**、与服务器池通信使用 **VPC 子网**。
  - **VPC 东西向场景**：与客户端通信、与服务器池通信均使用 **VPC 子网**。
- DNAT 模式仅支持 **VLAN 场景**，即与客户端通信、与服务器池通信均使用 **VLAN 网络**。

在不同场景下，VIP 地址的使用存在以下限制：

- **VLAN 场景**中使用的 VIP 地址不允许和 **VPC 南北向场景**中使用的 VIP 地址重复，即两种场景不可相互复用 VIP。
- **VPC 南北向场景**中 VIP 所在的网段，不允许与 **VPC 东西向场景**中 VPC 子网的网段重复。
- **VPC 南北向场景**中，如需在多个虚拟服务中复用 VIP 地址，则多个虚拟服务中与后端服务器通信的网络必须属于同一个 VPC。

参考如下说明完成配置后单击**下一步**。

# 配置服务器池

1. 选择服务器池类型，支持**服务器池**和**服务器池组**。
2. 根据服务器池类型完成如下配置。

   - 当类型为**服务器池**时，您可以直接选择**已有服务器池**，也可以**新建服务器池**并关联**健康检查监视器**。

     >

   - 当类型为**服务器池组**时，您可以选择已有的**服务器池组**。
3. 单击**下一步**。

## 管理虚拟服务 > 创建虚拟服务 > 配置其他参数

# 配置其他参数

根据如下说明，完成会话保持、QoS 策略等其他参数配置。

| 参数 | 描述 |
| --- | --- |
| 会话保持 | 选择是否为虚拟服务启用会话保持，会话保持时间当前默认为 300s。  启用后会根据客户端源 IP 地址，将相同地址发出的、属于同一个会话的请求调度至相同的服务器池成员。  **说明**：   - 当服务器池的调度策略为**源地址哈希**时，由于从同一源地址发出的请求本身就会调度至相同服务器，因此无需配置会话保持。 - 当后端服务器被健康检查监视器标记为**不健康**时，如仍有客户端会话绑定至该服务器，系统会立即将会话切换至其他可用服务器。如需在服务器发生故障时继续会话保持，可联系 SmartX 售后工程师，通过命令行方式将该配置修改为**继续会话保持**。 |

| 访问控制 | 选择是否为虚拟服务启用访问控制。  启用后可限制虚拟服务只接收或不接收某些客户端的请求，以提供更灵活的负载均衡业务级别的访问控制。   - **白名单**：需配置白名单 IP 地址，虚拟服务只接收来自白名单地址的客户端请求，来自白名单以外地址的请求均会被拒绝。 - **黑名单**：需配置黑名单 IP 地址，虚拟服务不接收来自黑名单地址的客户端请求，来自黑名单以外地址的请求均会被接收。 |

## 管理虚拟服务 > 切换活动服务器池

# 切换活动服务器池

在服务器池组中，活动服务器池负责为应用提供服务，非活动服务器池不为应用提供服务。当主、备服务器池健康状态均为正常时，默认主服务器池为活动服务器池。

- **自动切换**：

  满足以下任一条件时，系统将自动切换活动服务器池：

  - **主服务器池故障**：当主服务器池中的所有成员均处于不健康状态时，系统自动将备服务器池切换为活动状态。
  - **备服务器池故障**：若当前使用备服务器池，且其中所有成员均处于不健康状态，同时主服务器池中至少存在一个健康成员时，系统自动将主服务器池切换为活动状态。
  - **跟随 LB 实例切换**：在 LB 实例组页面手动[切换活动实例](/everoute/3.5.0/er_administration_guide_lb/administration_guide_lb_43)时，可选择同步将相关虚拟服务中服务器池组中的主/备服务器池切换为活动状态。例如，将活动实例切换为备 LB 实例时，可同步将备服务器池切换为活动状态。
  > **说明**：
  >
  > 若当前活动服务器池未关联健康检查监视器，则默认所有成员状态健康，不会触发自动切换。
- **手动切换**：当非活动服务器池中至少存在一个主成员状态为健康时，您可以手动触发活动服务器池切换。

**注意事项**

切换服务器池活动状态的过程中，当前虚拟服务会有短暂的网络中断，建议在业务空闲时或业务下线后谨慎操作。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，在左侧导航栏中单击**虚拟服务**。
2. 单击某一虚拟服务，进入虚拟服务的详情页面。
3. 在**服务器池组**的信息展示区域右上角单击**切换活动服务器池**。
4. 在弹出的确认窗口中单击**确认**。

     > **注意**：

## 管理许可 > 许可方式

# 许可方式

网络负载均衡器功能提供如下两种许可方式：

- 按 **vCPU 数量**许可：

  该许可方式规定了所有 LB 实例中 LB 虚拟机占用的 vCPU 数量的上限。
- 按 **CPU 插槽数**许可：

  该许可方式规定了已部署 LB 实例的集群中所有主机的 CPU 插槽数之和的上限。集群扩缩容后，消耗的许可数将自动更新。

  例如，若您在 3 节点集群中部署了 1 个 LB 实例，该实例由 2 个 LB 虚拟机组成，尽管 LB 虚拟机分别运行在集群中的 2 个主机上，此时仍会按照集群中 3 个主机的 CPU 插槽数计算许可。若集群扩容为 5 节点，则将按照 5 个主机的 CPU 插槽数计算许可。

  当在同一个集群中部署更多 LB 实例时，不会额外消耗许可。若某集群中的所有 LB 实例均被删除，则该集群占用的许可将被自动释放。

首次部署网络负载均衡器功能后，将根据功能的部署完成时间，默认生成该功能的**试用许可**，许可信息如下表所示。

| **许可信息** | **说明** |
| --- | --- |
| 版本 | 标准版 |
| 许可类型 | 试用许可 |
| 许可方式 | vCPU 数 |
| 许可数 | 不限 |
| 过期时间 | 90 天（自功能首次部署完成开始计算） |

- 许可额度达到上限，希望提升许可额度。
- 转换许可方式。
- 升级许可类型。例如，某一功能的**试用许可**需升级为**订阅许可**或**永久许可**。
- 延长订阅时长。

更新许可时，请将所有当前软件许可信息提供给 SmartX 客户经理，在完成商务流程后，客户经理将提供新的许可码。

**注意事项**

当 CloudTower 从 4.1.0 以下的版本升级至本版本后，如您原先同时具有分布式防火墙和网络负载均衡器的正式许可（**订阅许可**或**永久许可**），则升级后将仅具有分布式防火墙的正式许可，网络负载均衡器会变更为**试用许可**。请联系 SmartX 客户经理，重新申请并更新网络负载均衡器的许可。

**操作步骤**

1. 进入 CloudTower 的**网络与安全**界面，选择**设置**，单击**软件许可**，将许可码粘贴在**新许可码**处。
2. 单击输入框外部的空白区域，载入新许可信息。
3. 确认新许可信息无误后，单击**更新**，完成许可更新。

## 卸载网络负载均衡器

# 卸载网络负载均衡器

当您不再需要某个 Everoute 服务中的网络负载均衡器功能，想要卸载该功能时，请在知晓卸载功能可能导致的风险并确认满足卸载的要求后，参照操作步骤进行卸载。

**风险提示**

卸载网络负载均衡器后，将造成如下影响：

- 已关联的所有虚拟机网络将被解除关联。
- 后续再次开启网络负载均衡器功能时，需要重新初始化所有功能。

**卸载要求**

Everoute 服务的网络负载均衡器功能中不包含任何 LB 实例或 LB 实例组。

**操作步骤**

1. 登录 CloudTower，进入**网络与安全**界面，选择**设置**，选择待卸载功能的 Everoute 服务。
2. 在**服务设置**中单击**基本信息**，进入基本信息界面。
3. 在待卸载功能名称右侧单击**卸载功能**。
4. 确认卸载功能后，单击**卸载**。
