# sks_cn/1.5.1/SMTX Kubernetes 服务发布说明
## 关于本文档

# 关于本文档

本文档介绍了相较于 1.5.0 版本，SMTX Kubernetes 服务 1.5.1 修复的问题，以及该版本相关的配套信息和兼容信息。阅读此文档您还可以了解 SMTX Kubernetes 服务 1.5.1 的部署和升级注意事项。

# 文档更新信息

## 版本更新说明

# 版本更新说明

- 支持以节点组粒度对 Worker 节点标签、污点、注解进行持久化配置。同时移除了原有的节点非持久化配置功能。
- 修复了手动对节点组的节点的网络进行差异化配置后，在界面编辑节点组配置导致差异化配置被覆盖的问题。
- 修复了工作负载集群配置 HTTP 容器镜像仓库后，新节点无法正常访问仓库的问题。
- 修复了使用 EIC 的工作负载集群中，指定静态 IP 的 Pod 重建后通信失败的问题。Pod 重建后 MAC 地址会发生改变，Pod 未有效刷新其他网络设备对 Pod IP 的 ARP 缓存，仍使用重建之前的 MAC 地址与 Pod 通信，导致通信失败。

## 版本部署和升级注意事项

# 版本部署和升级注意事项

- 仅当 SKS 版本为 1.4.0 或 1.5.0 时，才支持将 SKS 升级至本版本。若 SKS 版本低于 1.4.0，则需先将 SKS 升级至 1.4.0 版本，再升级至本版本。
- 升级 SKS 至本版本前，需确保 CloudTower 为 4.7.x 或 4.8.x 版本，并且未启用高可用。
- 若升级前的 SKS 版本为 1.4.0，则升级 SKS 至本版本前，需确保当前所有工作负载集群的 Kubernetes 版本均为本版本 SKS 兼容的 Kubernetes 版本（1.25.x 和 1.26.x 版本），并且当前所有工作负载集群的功能版本不低于 1.4.0。
- 部署本版本 SKS 或将 SKS 从 1.4.0 版本升级至本版本前，对于管控集群和启用 Calico CNI 插件的虚拟机集群（部署场景仅需关注管控集群），若虚拟机节点所属 SMTX OS 集群的主机 CPU 属于海光一号或二号处理器系列，需确保 SMTX OS 集群的“虚拟机 CPU 兼容性”设置满足以下要求，否则请先修改设置，再执行部署或升级。

  | CPU 处理器系列 | SMTX OS 集群的“虚拟机 CPU 兼容性”要求 |
  | --- | --- |
  | 海光一号 | `主机透传`或 `Opteron_G3` |
  | 海光二号 | `EPYC`、`EPYC-IBPB` 或 `Opteron_G3` |

## 版本配套说明

# 版本配套说明

## 与 CloudTower 版本配套说明

您可以通过满足以下配套要求的 CloudTower 部署和管理本版本 SKS。

- **软件版本**：4.7.x、4.8.x 版本
- **是否启用高可用**：不启用
- **主机操作系统**：CentOS 或 openEuler
- **环境配置**：高配

  若 CloudTower 的环境配置等级为**低配**，可参考《CloudTower 安装与升级指南》中的**变更 CloudTower 环境配置**章节，通过修改 CloudTower 虚拟机占用的资源，将其环境配置等级变更为**高配**。

## 与 SMTX OS 版本配套说明

本版本 SKS 管控集群、SKS 容器镜像仓库、以及通过本版本 SKS 创建的工作负载集群的虚拟机节点均运行在 SMTX OS 集群中。SMTX OS 集群须满足以下配套要求。

- **虚拟化平台**：ELF
- **软件版本**：
  - **不启用双活特性**：5.0.5 及以上的 5.0.x、5.1.x、6.0.x、6.1.x 或 6.2.x 版本
  - **启用双活特性**：6.0.x、6.1.x 或 6.2.x 版本
- **主机操作系统**：CentOS 或 openEuler

## 与 SMTX ZBS 版本配套说明

通过本版本 SKS 创建的工作负载集群启用 SMTX ZBS CSI 插件时，支持使用满足以下要求的 SMTX ZBS 集群作为持久化存储后端。

- **软件版本**：5.5.x 版本、5.6.x 版本
- **是否启用双活特性**：不启用
- **主机操作系统**：CentOS、openEuler 或 TencentOS

## 与可观测性服务版本配套说明

本版本 SKS 系统服务、通过本版本 SKS 创建的工作负载集群均支持关联 1.4.2 及以上的 1.x.x 版本的可观测性服务，使用其监控、报警、日志、事件和审计功能。

## 与 Everoute 版本配套说明

通过本版本 SKS 创建的工作负载集群启用 EIC 插件时，工作负载集群所在的 SMTX OS 集群需关联 2.0.3 及以上的 2.x.x、3.0.x、3.1.x、3.2.x、3.3.x 或 3.4.x 版本的 Everoute 服务。
