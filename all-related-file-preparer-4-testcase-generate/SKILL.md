# All Related File Preparer for Testcase Generate

按顺序执行：

**Step 1：校验文件 -> Step 2：拿到已确认的分析结果 -> Step 3：搜索并确认相关 Drive 文件 -> Step 4：下载到本地**

如果本技能是被 `../testcase-generate/SKILL.md` 调用：

- 优先使用上游已经确认的摘要、产品、关键词。
- 不要重复让用户确认同一轮产品归属或关键词，除非上游没有提供最终结果。
- 不要在本技能里展开测试角度或测试用例生成说明。

---

## 输入约定

接受两类输入：

1. 文档路径模式：用户提供一个或多个 `.docx`、`.pdf`、`.md` 文件路径。
2. 上游交接模式：调用方同时提供
   - Step 1 校验后的有效文件路径列表
   - 已确认的分析结果，至少包含：
     - 功能摘要
     - 主体产品及关系
     - 1-2 个最终关键词

支持类型：

- `.docx`
- `.pdf`
- `README` / `readme` / `README.md` / `readme.md`
- 任意 `.md`

---

## Step 1：校验文件并确定有效输入

- 如果用户没有提供任何文件路径，立即停止并提示补充路径。
- 遍历每个路径：
  - 路径不存在：标记为“文件不存在”。
  - 类型不支持：标记为“类型不支持”。
  - 路径存在且类型受支持：标记为“有效”，加入后续流程。
- 输出有效文件列表，并保留这些路径用于后续推算下载目录。

如果没有任何有效文件，停止后续流程。

---

## Step 2：拿到已确认的摘要 / 产品 / 关键词

### 2.1 优先使用上游已确认结果

如果调用方已经提供最终确认过的分析结果，直接沿用，不要重新分析。

### 2.2 上游未提供时，调用根目录分析技能

读取并严格遵循 `../smartx-doc-2-product-and-keywords/SKILL.md`。

- 输入使用 Step 1 的有效文件列表。
- 让该技能返回结构化分析结果，而不是直接替当前流程做最终决定。

### 2.3 需要用户确认时的处理规则

如果返回结果中候选产品数量大于 1，必须输出以下 JSON 作为本次回复的唯一内容，然后停止等待用户选择：

```json
{
  "question": "这个功能最相关的 SmartX 产品是哪个？",
  "type": "single_select",
  "options": [
    "产品1",
    "产品2",
    "产品3"
  ]
}
```

如果返回结果中候选关键词数量大于 2，必须输出以下 JSON 作为本次回复的唯一内容，然后停止等待用户选择：

```json
{
  "question": "以下关键词中，哪些最能代表这篇文档的核心功能？（请选择 1-2 个）",
  "type": "multiple_select",
  "max_selected": 2,
  "options": [
    "关键词1",
    "关键词2",
    "关键词3",
    "用户自行输入 1-2 个关键词"
  ]
}
```

确认完成后，本步骤必须得到一份最终分析包，至少包含：

- 功能摘要
- 主体产品 + 与功能的关系
- 1-2 个最终关键词

---

## Step 3：调用 keyword-related-file-from-google-drive 搜索相关资料

读取并严格遵循 `keyword-related-file-from-google-drive/SKILL.md`。

输入使用 Step 2 的最终分析包，至少包含：

- 功能摘要
- 最终关键词
- 如果有的话，再附带主体产品信息，帮助收紧搜索范围

输出要求：

- 按该技能的规范展示相关文件名称、id、链接和筛选结果。
- 明确请用户确认这些链接是否满足需求。
- 如果用户要求放宽 / 收紧关键词、排除某类文件或重新筛选，在本步骤内调整后重跑。
- 只有用户明确确认后，才能把当前结果中的 `id` 列表交给 Step 4。

---

## Step 4：调用 google-drive-file-download 下载到本地

### 4.1 推算下载目录

根据 Step 1 的有效文件路径列表推算 `output_dir`：

- 若所有有效文件都在同一目录，取该目录。
- 若不在同一目录，默认取第一个有效文件所在目录，并向用户确认。
- 若无法推算，要求用户明确提供一个已存在的本地目录。

向用户说明：

`将把 Step 3 中确认的相关文件下载到：{output_dir}。`

如果用户修改路径，先确认路径存在，再继续下载。

### 4.2 下载执行

读取并严格遵循 `google-drive-file-download/SKILL.md`。

- 输入目录：确认后的 `output_dir`
- 输入文件：Step 3 已确认结果中的全部 `id`

按该技能要求逐个调用下载脚本，收集成功、失败和具体错误。

### 4.3 输出并确认

- 按下载技能的规范展示下载结果。
- 如果用户要求重试失败项或修改路径，在本步骤内处理并重新输出。
- 用户明确确认后，本技能流程结束。

---

## 小结

- 直接调用时，本技能可以自己完成“文件校验 -> 分析确认 -> Drive 搜索 -> 下载”。
- 被上游技能调用时，本技能应跳过重复分析，直接使用上游已确认的摘要 / 产品 / 关键词进入 Drive 搜索和下载。
