# All Related File Preparer for Testcase Generate

根据用户提供的文档路径（docx、pdf、readme），依次执行：**文件校验 → 文档分析 → Drive 关键词搜索 → 本地下载**。最终得到与文档相关的 Google Drive 文件 URL，并可将其下载到**第一步中用户提供的文件所在目录**。Step 4 的下载目标目录在执行时根据 Step 1 已确认的有效文件路径列表推算，并请用户确认或修改后再下载。

如果本技能是被 `../testcase-generate/SKILL.md` 调用：

- 仍按本技能流程准备资料；
- 由调用方统一承接对用户的解释和后续分析；
- 不要在本技能中重复展开测试角度或测试用例生成说明。

---

## 输入约定

- **接受格式**：用户提供一个包含一个或多个文件的**文件夹路径**，指向本机或工作区内的文件夹。
- **支持类型**：仅接受以下类型，其余一律视为不满足：
  - `.docx`（Word 文档）
  - `.pdf`（PDF 文档）
  - **Readme 类**：`README`、`readme`、`README.md`、`readme.md` 或任意以 `.md` 结尾的 Markdown 文件

---

## 四步流水线总览
```
Step 1: 校验文件类型与上传 → 输出文件列表 → 通过
    ↓
Step 2: 调用 smartx-doc-2-product-and-keywords 分析 → 输出分析结果 → 用户确认/修改 → 通过
    ↓
Step 3: 调用 keyword-related-file-from-google-drive 处理分析结果 → 输出相关 URL（含 id）→ 用户确认/修改 → 通过
    ↓
Step 4: 根据 Step 1 的有效文件列表推算下载目标目录 → 用户确认路径/修改 → 调用 google-drive-file-download 下载 → 通过 → 结束
```

### Step 4 下载目标目录的推算与确认

- **推算规则**：根据 Step 1 用户确认的**有效文件路径列表**得到「文件所属目录」。若所有文件在同一目录下，取该目录；若不在同一目录，取**第一个有效文件的所在目录**（或向用户询问下载目录）。
- **Step 4 使用**：向用户展示「将把 Step 3 中的相关文件下载到：**{output_dir}**（由第一步中您提供的文件路径推算）。如需更改下载位置，请直接说明目标路径。」用户确认或给出新路径后，再执行下载。

---

## Step 1：判断文件类型与是否上传

### 1.1 是否上传文件

- 若用户**未提供任何文件路径**（只说了需求、未给路径），则**立即停止**并提示：
  - 请上传或提供至少一个文档的路径，支持类型：docx、pdf、readme（含 .md）。

### 1.2 判断文件类型是否满足

- 遍历用户给出的每一个路径：
  - 若路径对应**不存在**的文件：列出该路径，标记为「文件不存在」，不进入后续分析。
  - 若扩展名/文件名**不在**支持列表中（见「输入约定」）：列出该路径，标记为「类型不支持」，不进入后续分析。
  - 仅当路径存在且类型为 docx / pdf / readme（.md）时，标记为「有效」，加入待分析列表。

## Step 2：调用 smartx-doc-2-product-and-keywords 分析

### 2.1 调用方式

- 读取并严格遵循 `smartx-doc-2-product-and-keywords/SKILL.md`（即 smartx-doc-analyzer 工作流）。
- 使用 Step 1 确认的**有效文件路径列表**作为输入，对该批文件执行该技能定义的完整流程（读取文档 → 产品识别 → 关键词提取 → 输出结果）。
- 若该技能依赖 `references/products.md` 等，从 `smartx-doc-2-product-and-keywords/` 目录下读取对应 reference 文件。

### 2.2 分析子 skills 返回结果

收到 smartx-doc-2-product-and-keywords 技能的返回结果后，按照如下规则进行判断：

### 候选产品数量 > 1 时 → 必须输出 JSON 并立即停止

> ⚠️ **强制中断规则**：smartx-doc-2-product-and-keywords 返回的候选产品数量 **大于 1 个**时，无论置信度高低、无论是否自认为可以判断，**严禁自行选择**。必须将以下 JSON 作为本次回复的**唯一内容**输出，输出前后**不得添加任何文字**（包括解释、过渡句、标题或 Markdown 标记），然后停止，等待用户回复。
```json
{
    "question": "这个功能最相关的 SmartX 产品是哪个？",
    "type": "single_select",
    "options": [
        "产品1",
        "产品2",
        "产品3"
    ]
}
```

> **为何必须停止**：前端会检测此 JSON 并渲染多选框。如果在 JSON 前后添加任何解释文字，或者跳过 JSON 自行决定关键词，前端将无法识别，整个交互流程会断裂。

收到用户选择后，确认所选产品在 `references/products.md` 的列表中，以该选择作为主体产品，继续 Step 3。

### 候选关键词数量 > 2 时 → 必须输出 JSON 并立即停止

> ⚠️ **强制中断规则**：smartx-doc-2-product-and-keywords 返回的候选关键词数量 **大于 2 个**时，无论置信度高低、无论是否自认为可以判断，**严禁自行删减**。必须将以下 JSON 作为本次回复的**唯一内容**输出，输出前后**不得添加任何文字**（包括解释、过渡句、标题或 Markdown 标记），然后停止，等待用户回复。
```json
{
    "question": "以下关键词中，哪些最能代表这篇文档的核心功能？（请选 1-2 个）",
    "type": "multiple_select",
    "max_selected": 2,
    "options": [
        "关键词1",
        "关键词2",
        "关键词3",
        "用户自行输入 1-2 个关键词"
    ]
}
```

> **为何必须停止**：前端会检测此 JSON 并渲染多选框。如果在 JSON 前后添加任何解释文字，或者跳过 JSON 自行决定关键词，前端将无法识别，整个交互流程会断裂。

如果用户选择了 "用户自行输入 1-2 个关键词"，则提示用户输入关键词，并将其作为最终关键词，进入 Step 3。
如果用户没有选择 "用户自行输入 1-2 个关键词"，则以选中的 1-2 个词作为最终关键词，进入 Step 3。

---

## Step 3：调用 keyword-related-file-from-google-drive 得到相关 URL

### 3.1 调用方式

- 读取并严格遵循 `keyword-related-file-from-google-drive/SKILL.md`。
- 将 Step 2 用户确认的**完整分析结果**（必须包含「文档对主要功能点的描述摘要」与「关键词」区块）作为该技能的输入，执行其工作流（校验输入 → 提取关键词 → 执行 Drive 搜索脚本 → 相关性筛选 → 返回结果）。

### 3.2 输出并等待确认

将 Step 3 的**搜索结果**（相关文件的名称、id、可点击链接等）按该技能规定的格式（如 Markdown 表格）输出给用户。

并明确说明：

- **请确认以上相关文件/链接是否满足需求。确认后将进入第四步（将上述文件下载到第一步中您提供的文件所在目录）。如需调整（例如放宽/收紧关键词、排除某类文件），请说明，可在本步内重跑或调整后再呈现。**

若用户要求调整：
- 在该步内根据用户意见调整（如修改关键词、重新筛选），再次输出结果并请求确认。
- 重复直至用户**明确确认**。

只有用户确认后，才将**当前确认的搜索结果**（含每个文件的 id）作为 Step 4 的输入。

---

## Step 4：调用 google-drive-file-download 下载到本地

### 4.1 目标目录的推算与确认

- **推算**：根据 Step 1 已确认的有效文件路径列表，按「Step 4 下载目标目录的推算与确认」中的规则得到 `output_dir`（若无法推算或文件列表已不可用，则请用户明确指定一个已存在的本地目录）。
- **向用户展示**：「将把 Step 3 中的相关文件下载到：**{output_dir}**（由第一步中您提供的文件路径推算）。如需更改下载位置，请直接说明目标路径。」
- 若用户**要求更改路径**：以用户给出的目标路径作为 `output_dir`，确认该目录存在后再执行下载；若目录不存在，提示用户并停止。
- 若用户**确认路径无误**：直接使用当前 `output_dir` 执行下载。

### 4.2 调用方式

- 读取并严格遵循 `google-drive-file-download/SKILL.md`。
- **输入**：目标目录 = 上述确认后的 `output_dir`；文件 ID 列表 = Step 3 用户确认的搜索结果中的全部 `id`。
- 按该技能要求对每个 id **循环调用**下载脚本（每次只传一个 `--id`），收集 stderr 与退出码，按该技能规定反馈成功/失败及具体错误。

### 4.3 输出并等待确认

将下载结果（成功数量、失败列表及错误信息）按 google-drive-file-download 技能规定的格式呈现给用户。

并明确说明：

- **请确认以上下载结果。确认后下载流程结束，进入后续用例分析和生成环节；若有失败项可针对失败 ID 重试或调整路径后再次执行第四步。**

若用户要求重试或修改路径：在该步内按用户意见修改路径并重新执行下载，再次输出结果并请求确认。重复直至用户**明确确认**。

用户确认后，流程结束。

---


## 小结

- **输入**：用户提供的 docx / pdf / readme（.md）文件路径。
- **Step 1**：校验类型与是否上传，输出有效文件列表 → 用户确认/多轮修改 → 确认后进入 Step 2。
- **Step 2**：用 **smartx-doc-2-product-and-keywords** 分析确认后的文件，输出分析结果 → 用户确认/多轮修改 → 确认后进入 Step 3。
- **Step 3**：用 **keyword-related-file-from-google-drive** 处理确认后的分析结果，输出相关 URL（含 id）→ 用户确认/多轮修改 → 确认后进入 Step 4。
- **Step 4**：根据 Step 1 的有效文件列表推算 `output_dir`，用户确认或修改路径后，用 **google-drive-file-download** 将 Step 3 的 id 下载到该目录 → 用户确认结果后结束。
